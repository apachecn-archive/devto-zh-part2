# 我自学编程学到的 6 件事。

> 原文：<https://dev.to/theminshew/7-things-ive-learned-teaching-myself-to-code--1j67>

我写这篇文章是为了回应我昨晚看的阿莱士·史密斯的一篇文章。我很受鼓舞，并希望分享我的学习经验，希望将这种美好的感觉传递下去。请注意，我主要从事 web 开发，所以我的堆栈反映了这一点。

**1。愿意抛弃一切，最好的、正确的解决方案才是最重要的。**

这适用于从代码到语言到思想的一切。编程变化如此之快，他们处理问题的方式如此之多，所以记住解决方案是编程的原因是很重要的。这一切都是为了解决问题，而不是为了旅程。当你花了两天时间编写代码或创建页面，有人走过并指出一个更快/更好的解决方案时，你非常容易生气或产生抵触情绪。

如果你专注于最好的解决方案，它有助于更容易地抛弃那些代码。我最近花了两天时间为一个项目开发 flexboxes，我的商业伙伴(完全不是技术人员)认为没有这个功能对我们客户的愿景更好。我的本能是发疯，无论如何都要保持代码，但几次呼吸后，经过思考，我意识到我的合作伙伴是对的。

另外，如果我诚实的话，我的客户不会关心我在 x cool widget 上花了多少时间或者 y doodad 的代码有多漂亮。他们只是希望他们的产品尽可能的好。

**2。你学习的语言并不像每个人说的那样重要。**
我从来不是某个特定技术的粉丝，我喜欢 windows 10，也有一台同样受欢迎的运行 Kubuntu 的笔记本电脑。我甚至喜欢用我妻子的 MacBook。我以同样的方式学习语言，我很感激我这样做了。我决定学习一些我在工作中需要的语言(HTML/CSS/JS *我知道 HTML 和 CSS 不是真正的编程语言。)然后我会挑几个我想学的，(SQL，Python，Perl，Java)然后我加了 C，因为我想用一个更基础的语言来弥补我没有 compsci 学位的不足。

在所有的学习和阅读之后，我明白了他们都有自己的位置和价值。除非你迫切需要学习一门特定的语言，而不是尝试几种语言，并找到合适的感觉。哪种语言在你使用时流动。然后不管流行文化怎么说都要学。如果你对汇编有热情，那么这种热情将比你对 python 的一点点兴趣带你走得更远。

**3。编程真的像大家说的那样难。**
当我开始的时候，我真的认为我是一个超级天才，可以在几个小时的学习中解决所有问题。c 和 CSS 已经让我卑微了。我花了几天时间试图弄清楚哪怕是基本的 CSS 概念，这让我很尴尬，因此我把我的调子改为“给我足够的时间，栈溢出，文档和 2009 年的论坛帖子，我可以弄清楚任何事情。”

如果你理解编程是困难的，并且你这样对待它，我相信你有更好的机会不放弃，这也缓和了我经常感到沮丧。现在，我不再对每一个错误或破损的页面感到沮丧，而是理解了这个过程的一部分，不再那么沮丧了。在凌晨 3 点找了 4 个多小时都找不到分号后，我现在只能发誓。(这是骗人的，我发誓任何时候我都用几乎日常的 CSS。)

**4。尝试不同的学习方法。**
我从书籍开始，然后是在线文本课程，真的很难，我尝试了论坛，但太新了，我甚至不知道如何正确地提问。我终于抓到了一些视频课程，事情真的开始点击。

愿意尝试其他学习方式。视频、书籍、在线课程、论坛甚至编码游戏等等都是很棒的工具，如果你能试着去使用它们的话。如果我坚持只看书，我会很沮丧，可能会彻底放弃编程。我从未使用过视频，也不喜欢将它们用于大多数其他学习，但对于编码来说，能够跟随并在教师旁边暂停是令人惊讶的。

我通过整体学习概念学得最好，所以我一下子学习了 7-8 种语言。我完全理解这通常是一个非常糟糕的想法，但对我来说，它很有效。我现在已经专注于 5 种一致的语言，其余的只是为了兴趣而涉猎。我比以前一次只学一种语言要超前好几光年。我也很容易感到无聊，所以我可能会停止一起学习，但能够从 web 开发切换到构建 SQL 数据库，到阅读 C 中的指针，再到学习 Git。这是我的救命稻草，帮助我度过了最初的学习难关，坚持编程。

这对我有效，但对你可能无效。我的观点是，不管别人怎么说，尝试不同的东西是值得的。

尝试你能尝试的一切，然后选择最适合你的。

我喜欢尝试新的文字编辑，这是一种病态。我无法告诉你在选定当前版本之前，我尝试了多少不同的 linux 发行版。愿意尝试不同的事情，如果你喜欢并且富有成效，那就去做吧。不要让别人告诉你什么是最好的编辑器、操作系统甚至语言。只是尝试一下。我爱上了在某些方面非常反主流的东西(我喜欢 php，我控制不了自己)，但我也在其他方面随大流，因为大众是对的。(linux for development 更好地改变了我的整个工作流程)

试试看，如果你喜欢就留着吧。永远不要因为某个论坛或科技博客上的某个人说某样东西是有史以来最好的，就坚持下去。

**6。保持谦逊，**
这很难，但保持谦逊，愿意倾听他人并以谦逊的态度接近他人是无价之宝。我没有太多的机会与其他开发人员一起工作，我的大多数客户根本不知道或不关心代码。我必须记住，我只是选择学习这些技能，我不是上帝给人类的礼物。在我进入编码领域之前，我就知道了与那些没有很好的沟通技巧或傲慢自大的开发人员或程序员一起工作是什么感觉。这从来都不好玩，而且大多数时候双方都不满意最终产品。

从另一方面来说，与一个专注于团队、谦逊的开发人员或编码人员一起工作绝对令人惊叹，团队可以做非常酷的事情。非常重要的是要记住，不知道如何打开电脑的人可能是另一个领域的绝对天才。保持专注于团队合作，努力培养领导力和沟通技巧，这样大家都会赢。

我是一个新手，以至于一个分号可以让我每周都感到谦卑。然而，我已经注意到，有时我在和别人交谈时，会带着一种恼怒的“为什么你就不能理解这个简单的概念呢？”的态度。人们很容易忘记“简单概念”需要几天或几周的研究才能理解。自我是敌人

**> > > > >【最后的想法】< < < < <**
希望这对正在阅读的人是鼓励和有用的。编程很难，但绝对值得。我喜欢这种感觉:因为某件事的难度而讨厌它，然后又因为你最终得到了它而爱上它，现在你可以做令人惊奇的事情了。我喜欢精确但不精确的编码(任何人都喜欢),喜欢在键盘上输入一串符号，点击网页上的刷新按钮，看着你的创作变成现实的快乐。

我不知道有多少其他领域允许创造力、解决问题、对相同概念的极端憎恨和绝对热爱，并让我整夜从事一些愚蠢的个人项目，就像编程和 web 开发一样。感谢阅读并继续编码！！！
