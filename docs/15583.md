### 为什么我一开始想去参加一个编程聚会？

主要原因是人们。我认为和那些你有可能一起工作的人交往并向他们学习总是一个好主意。不管一个人的能力如何，你总能从他身上学到一些东西——在这些聚会上，我非常信奉这一原则，试图与尽可能多的人交谈。

在我看来，最吸引人的方面是听到公司如何在结构层面上工作，以及他们使用的技术。例如，我与一位在银行工作的人交谈，他正在使用 Node 重写他们的整个堆栈。我发现这很有趣，因为 Node 是一项相对较新的技术。我问他他们是如何处理银行软件、规模和安全性这些关键领域的。由于我本人从未在如此关键的环境中使用过 node，因此获得这样的见解是非常有趣的。

讲座也是学习新技术的好方法。我的第一次 meetup 有一个题为“麻瓜的机器学习”的演讲，并向你介绍了如何使用 Azure 的机器学习功能，以及机器学习如何工作的概述。处于技术的最前沿，获得行业领先的教学是令人惊讶的，远远超过了我在网上找到的任何内容。此外，让一位该主题的专家来发表演讲，可以让你提出一些在网上很难找到答案的问题。

第三个也是最后一个原因是，任何好的聚会都会有免费的披萨！🍕

### 你是如何找到你的聚会的？

对此有一个简单的答案:[meetup.com](https://www.meetup.com/)。我查看了该地区所有的编程聚会，找到了一个看起来很活跃的聚会，并且有一段我很感兴趣的谈话历史。我注册了几个，我会建议任何“第一次见面”的人也这样做——分散你自己，你永远不知道你可能喜欢什么。尽管我目前的工作是 PHP 程序员，但我最终还是参加了一个 Javascript 聚会！不要把自己局限在针对你当前使用的语言的聚会上。看看这些演讲，看看它们是否让你感兴趣。正如我所说的，我的第一次聚会是诺丁汉 Javascript 小组，但演讲本身是关于机器学习的，与 Javascript 有很大关系。

一般来说，谈话会围绕某项技术展开，比如说 Docker 针对 meetups 语言的技巧。例如，在一个基于 PHP 的 meetup 上关于 Docker 的谈话可以被命名为“在 Docker 中设置 PHP 开发环境”。即使你不懂 PHP(或者 meetup 旨在开发的任何语言)，你也会发现这些讲座很有价值，而且很多人和你一样，并不使用这种特定的语言。

### [](#what-should-i-do-at-a-meetup)在 meetup 上应该怎么做？

现在你只能去听演讲，但那只是一半的经历。我强烈建议(当然是在抓了一些披萨之后)接近人们并介绍你自己。程序员通常是一群害羞的人，但最终，我们都有共同的兴趣，所以会有很多话题可聊。

起初，我发现每个人似乎都在自己的小圈子里互相交谈；如果是这样的话，就走过去对他们说“嗨，你好吗？我是乔希(显然用你的名字代替我的名字，但你明白我的意思)。询问人们的工作和他们在那里做什么，他们正在使用什么令人兴奋的技术，以及他们是否觉得谈话有趣。几乎每一个和你交谈的程序员都会有一些他们正在从事的兼职项目——谈谈这个吧！这可能会引发一些最令人兴奋的讨论，因为通常人们会尝试一些他们无法在日常生活中使用的尖端技术。

根据他们所说的，一直思考你可以问他们什么问题，这表明你在倾听他们，并且真诚地参与到对话中。

其他谈话要点可能包括，询问他们关于他们工作的公司，它有多大，他们的总部在哪里等等。

结束谈话时，不要找借口去洗手间(除非你真的需要)，简单地说“*我现在要向其他人介绍我自己，但我很想继续这个谈话，也许我可以收下你的电子邮件，我们可以谈谈*”。由于这是与工作相关的，人们可能会对交出他们的电话号码有点谨慎，所以选择与业务相关的沟通渠道，电子邮件，twitter 和 linkedin。

总而言之，不要害怕。这些是你的人，和你一样害羞和书呆子。所以，走出去，看看你会遇到谁。外面有很多有趣的人！如果所有这些都失败了，看看你是否能说服一个同事或朋友一起来。
