## [](#interesting-subject)有趣的主题

大约 11 年前，我写了这篇博文“[一个理工学生必须知道的 10 件事](https://geshan.com.np/blog/2007/11/10-web-by-things-technology-student-or/)”然后 4 年前，我在我完成学士学位的大学做了一次演讲，题目是“[我希望在读理工学士/本科时知道的事情](https://geshan.com.np/blog/2014/08/things-i-wished-i-knew-while-doing-my/)”。再次回到 2 年前，2017 年 11 月，我在加德满都的几所大学与 QnA 进行了另一次谈话，并进行了题为“[科技毕业生的职业道路](https://geshan.com.np/blog/2016/06/career-paths-for-to-be-tech-graduates-slides/)”的互动。我还在 2017 年 2 月的一次聚会上做了另一个名为“[我希望作为初级开发人员知道的事情](https://geshan.com.np/blog/2017/02/things-i-wished-i-knew-as-a-junior-developer-slides/)”的演讲。在过去的十年里，这一直是我感兴趣的话题。

## 为什么学生在课程结束后找不到技术工作？

让我们面对这一点，这是教育系统和学生个人的混合体。我在尼泊尔的加德满都攻读了我的本科/学士学位，也在荷兰攻读了我的硕士学位。当然，非常不同的教育体系和教育观。有一点是相同的，那就是这两者都不能让学生成为就业的现成产品。个人方面，不要因为你朋友在做，就去做一门课程。简单的逻辑，首先问问你自己，你认为自己在 5 或 3 年后会在哪里，然后沿着原路返回。

## [](#how-to-fix-this)这个怎么修？

我不是教育专家，但我已经在多个国家从事科技工作十年了。有一点是肯定的，技能和成绩是首要的，学位是次要的。大学应该努力弥合学生和真实市场需求之间的差距。就像我不知道有哪所大学会教学生，比如说`git`你每天在工作中都会用到它。那也不代表教的东西就是错的:)。是的，我在这里咆哮了一下。

从学生的角度来看，你需要研究什么与市场相关，自己学习，打造自己的品牌，这样你才能更快地推销自己的技能。举例来说，如果我面试一个拥有优秀科技博客的候选人，他的博客上有很多有用的和定期的帖子，甚至可能是中等的。你被聘用的几率已经上升了大约 5%。因为能够把你的想法/学习用语言表达出来是一个软件工程师的基本技能。

## 现在让我们开始行动吧

在我的长篇大论之后，让我们把注意力集中在作为一名技术专业的学生如何在完成学位后尽快找到工作上。以下是你需要做的事情，以增加你获得第一份技术工作的机会。

### [](#1-in-person-networking)1。面对面交流

每月至少参加一次聚会/会议/培训/活动，并在 Linked In 上添加至少两个可以帮你找到工作的新朋友。更不用说会后给他们发信息/邮件来建立关系了。聚会是结识你平时不会遇到的人的好方法。取决于你所在的城市，比如你在悉尼，你可以每周去一次科技聚会。如果你在加德满都，你可以每月参加一次聚会。去参加一个通常免费的聚会在很多方面都是有益的，你可以从听别人说话中学到新东西，你可以建立关系网...不要低估与人见面的力量。

所以你在问为什么这个人会推荐我？对吗？答案很简单`Many companies have referral bonus`。如果他们推荐你，你被选中，得到这份工作，完成试用期，他们会得到一笔钱。对你、推荐人和公司都是一个双赢的局面。电子邮件，然后打电话，并要求咖啡会议之前，要求工作推荐建议。当然，更不用说你应该有一份出色的简历。

### [](#2-use-recruiter-help)2。使用招聘人员帮助

一开始，应该是你给科技招聘人员发邮件，甚至打电话。想法很简单，招聘人员的工作就是为技术职位寻找合适的候选人。所以，坦诚地告诉他们你正在寻找初级或刚开始的工作，并努力找到一个匹配的。或许招聘人员知道一些你非常适合的职位空缺。根据城市的不同，有专门针对应届毕业生的实习和毕业生职位。在你有了几年的工作经验后，你可能需要逃离招聘人员，但那是另一回事了。

### [](#3-contribute-to-open-source)3。为开源做贡献

我知道很多公司会先看你的 GitHub 简介，然后再看你的简历。所以`actions speak louder than words`。首先，您不需要开始编写代码。你可以从帮助文档等开始。有很多方法可以回馈开源。[2018](https://hacktoberfest.digitalocean.com/)Hacktoberfest 两天后就要开始了，好好利用吧。所以学习 git 和 GitHub 就成了这一步的先决条件。如果你是最后一个学期学技术的学生，不知道 git =你在技术方面没有非常光明的未来:)。

### [](#4-make-a-list-of-companies-with-probable-roles)4。列出可能扮演角色的公司

如果你没有目标，你将如何实现它，如果你有目标，你可以有计划去实现它。列出你完成学位后想要工作的公司和职位/角色。在课程快结束的时候，如果你不确定是选后端还是前端还是 UI/UX 还是 DevOps/SRE 还是数据工程还是 DBA 或者...对不起，但是没有人能帮助一个困惑的人；).如果你认为自己是 Atlassian 的前端工程师，你需要为之努力。另一个方便的建议是保留一份你申请的所有工作的电子表格，并注明日期，以便做适当的跟进。对于缺乏经验的候选人来说，接到面试电话并不容易，所以检查第一步推荐会有很大帮助。

### [](#5-learn-a-new-tech-thing-each-month)5。每个月学习一项新的技术

现在你已经列出了你想要工作的公司和职位，显然你已经阅读了职位描述。所以你现在知道市场和/或你想加入的公司需要什么样的技能。开始学习它，每周/周末花几个小时学习它。最好的学习方法是尝试建立你最喜欢的项目，它可以是任何让你兴奋的事情。不要只是上了一堂课就说是的，我知道 docker。展示它在一个项目中的运行，然后你将真正地学习它。如果你已经知道公司需要教给另一个初级软件工程师的 10 件事情中的 7 件，你得到这份工作的机会自然会更高。

## [](#conclusion)结论

你还需要做很多其他的事情，比如一份清晰且令人印象深刻的一页简历，至少有一个在线项目。比如，对你想从事的领域有一定的了解和深度，了解自己的技能并进入正确的技术领域也会有所帮助。你需要在课程结束前 6-9 个月完成上述步骤，否则仓促行事，到最后也不会有多大帮助。

全球科技行业每年都需要新人。从积极的角度来看，技术需要越来越多的新人。另一方面，你的竞争对手不仅在你的城市，而且在你的世界的任何地方，所以你的技能和学习能力让你脱颖而出。祝你好运寻找你的第一个技术角色！
