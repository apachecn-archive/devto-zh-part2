# 让你成为更好的程序员的软技能

> 原文：<https://dev.to/amangautam/softer-skills-that-make-you-a-better-programmer--2g3e>

我写代码已经快 11 年了。我完全确定的一件事是，要成为一名真正优秀的程序员，你必须做的不仅仅是写代码。变得伟大很简单(不容易！).

这里有一些重要的实践，可以让任何人成为更好的程序员。

## 写给其他人类看

为一个初级开发人员写代码，他将在两年后来修复你代码中的问题。这意味着:

1.  选择更好的变量和函数名
2.  使用行内注释解释代码为什么(而不是做什么)要做什么
3.  编写更好的提交消息
4.  有一个带有安装指南和编码指南的自述文件
5.  将文档视为需要不断审查和重构的代码库的一个组成部分

写你的代码库，因为它将来会被一个白痴管理，因为最终，它可能会被管理。

PS:为机器优化代码也很重要，但是试着从人类同胞那里抽象出来。

## 编写测试用例

我用过的最好的产品的测试覆盖率在 85-95%之间。这看起来显然是一个很大的承诺，特别是当你是一个独立开发者的时候。

测试所有的东西是不可能的，但是至少要为那些很可能失败或随时间变化的部分编写测试用例。

编写单元或集成测试有时看起来是一件昂贵的事情，但最终它们总是比解决生产问题要便宜。

## 坚持用你的工具

在你熟悉你的工具之前，需要做大量的工作。让工具成为你潜意识的一部分需要更多的努力。

令人惊讶的是，大量开发人员非常频繁地改变他们的工具、语言和框架。我并不反对获得使用所有这些工具的经验，但我肯定反对经常更换这些工具。

你和你的工具之间不一定要有婚姻，但是你也不应该有太多的一夜情。冷静一段时间，孩子！

## 部署吧！

我已经记不清有多少项目从未公开过。大多数开发人员都有两个惊人的性格特征:

1.  完美主义者:不允许你做任何不完美的事情
2.  拖延者:即使你没有一个好的产品，也不要让这些特点占据你的主导地位，阻止你发布产品。不要让你梦想中的应用程序死在一个私有的 git 回购中！

## 捍卫你的最佳实践

如果医生在给你做检查前洗手，这被认为是正常的(事实上，这是一个好习惯)。

但是，如果开发人员想在编写实际代码之前编写测试用例，许多雇主/客户会觉得这很奇怪。

每个行业都有一些应该严格遵循的好习惯。有时候，成为一名伟大的销售人员是成为一名伟大的程序员的必经之路。推销您的最佳实践！

你的客户/雇主可能不知道科技领域的最佳实践。教育他们是你的责任，如果需要，甚至告诉他们如果你不遵循最佳实践会发生什么。(用一两个历史例子吓唬吓唬他们也行)。

## 学习一门新的编程语言

每年学习一门新语言。如果能用在工作上，牛逼！如果没有，也没关系。一门新的语言不仅会给你额外的技能，还会给你新的思考方式。

例如，来自 Java 和 C++背景，我花了一些时间来消化这样一个事实，即我们可以在没有`main`函数的情况下用 PHP 运行程序。

在研究了基于 PHP-Apache 的 URL(例如 home.php)之后，Python 实现路由和 URIs 的方式几乎令我震惊。

学习一门新的语言会帮助你质疑你当前语言的标准。
