# 在阅读一篇完整的教程时，你是愿意在一篇长文中阅读还是多篇短文中阅读？

我正在写一篇教程，但是基于我所概述的部分，我想对于一篇文章来说可能会有点长(例如超过 4000 字)。

想想构建一个完整(虽然简单)的应用程序的教程。目标不是应用程序本身的最终结果，而是以一种初学者友好的方式走过开发阶段，将事情联系在一起。

我想知道你更喜欢哪种方法以及为什么。谢谢！
