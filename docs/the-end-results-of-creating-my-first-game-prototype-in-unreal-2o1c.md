# 在虚幻中创建我的第一个游戏原型的最终结果

> 原文：<https://dev.to/alexgwartney/the-end-results-of-creating-my-first-game-prototype-in-unreal-2o1c>

首先，我想说我希望每个人都度过了愉快的一周。说到这里，让我们开始吧。所以这篇文章有一点背景。我一直朝着进入游戏/图形编程的目标努力。到目前为止已经做了很多工作。但我很享受看到自己技能提高的回报。至此，我终于在虚幻中完成了我的第一个游戏原型。在我进入细节之前，我想说的是，如果你正在寻找一个游戏编程的起点。我一直在学习 https://www.gamedev.tv/.Who 的课程，这真的有助于开始行动。

因为有了这些人，我才能够从对 c++和虚幻一无所知到能够很好地构建我的第一个简单原型。所以这是我在这篇文章中真正想涵盖的内容。从我的游戏原型开始。我花了多长时间？我如何保持我的范围小，最后是什么是我面临的上山战斗。

所以我想出的游戏原型是基于我从 gamedev.tv .上的课程，那就是建造一个简单的逃生室。你可以在这篇文章的结尾看到一些游戏的图片。我决定从这个想法中得到的是建立一个原型，用户必须得到两把钥匙。为了逃跑。为了设计这个想法，我把一把钥匙放在笼子里，另一把放在笼子外面。在这一层的附属楼层。在一楼，我又拿了两个盒子和一个压力板。这两个盒子允许玩家打开附赠的钥匙以便逃跑。现在游戏机制的诀窍是一个立方体会杀死玩家。另一个允许玩家打开钥匙。它们是基于重量的，玩家必须猜测哪一个有效。

现在你已经知道游戏原型是什么了，这到底花了多长时间？所以实际上花了几个月的时间。这背后有很多原因。一个即使游戏超级简单小巧。代码的幕后其实有很多东西。我从未做过的事。因此，在没有一步一步教程的帮助下，我自己解决了这个逻辑。让事情变得更加困难。这也确实有助于脱离教程循环。我花了这么长时间的另一个原因是我还必须重新学习 blender 中的基本建模。我也经历了几次房子模型的迭代，然后才决定继续前进的设计。我怀疑我做得越多，我就会做得越快。

接下来我想谈的是项目范围。当制作一个游戏，尤其是一个原型时，这是一件很重要的事情。当我继续讲述我是如何做到的。得到这个建议的最好地方和我从哪里学到的。额外学分是“制作你的第一个游戏:最小可行产品-范围小，开始正确-额外学分”[https://www.youtube.com/watch?v=UvCri1tqIxQ](https://www.youtube.com/watch?v=UvCri1tqIxQ)

最后，让我们谈谈我面对的一些上山战斗。让我告诉你，他们有很多。我遇到的第一个问题是十亿个未知数和老骗子综合症。我想我做不到。其次，我遇到的另一场艰难的战斗是完成这个项目。所有这些想法一直在我脑海中闪现。我必须时刻牢记我的最终目标，那就是从 A 地到 b 地，完成它，然后继续前进。最后，因为我不想把这篇文章写得太长。我面临的最后一场艰难的战斗是找出项目的所有未知因素。有很多次，我面对可怕的空白代码编辑器，不知道从哪里开始。会变得非常非常困难。但是就像任何项目一样，让我告诉你总会有失败的时候。成为这个啊哈时刻，你就会突破它。所以你只要继续努力。
