# 如何做一个好的审稿人？

> 原文：<https://dev.to/rvprasad/how-to-be-a-good-reviewer-3l5b>

独立于被评审的工件(例如，设计文档、代码、测试、包装、演示、书籍)，一个好的评审者应该做以下事情:

1.  识别工件中的问题**(什么)**
2.  提供问题成为问题的原因**(为什么)**
3.  建议如何解决已发现的问题**(如何)**

仅仅做 1 和 3 是不够的。有必要做两件事

1.  评估你的判断的有效性
2.  帮助作者学习和提高。

这里有两个例子:

#### 例 1(代码)

1.  这种字段访问可能导致空指针异常。
2.  字段访问表达式中的接收者是从这个方法调用返回的对象，它可以返回 null。
3.  在访问字段之前，请确保 receiver 对象为非空。

#### 例 2(文档)

1.  该指令要求用户设置一个仿真器。
2.  用户可能不知道如何设置仿真器。
3.  提供设置仿真器的说明，或者明确指出用户应该熟悉仿真器的设置。
