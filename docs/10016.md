# 实现与开源合作的梦想

> [https://dev . to/avellino/实现-梦想-全职-com-开源-2322](https://dev.to/avelino/realizando-sonho-de-trabalhar-full-time-com-open-source-2322)

看到刚开始贡献开源的朋友想安排一天时间贡献项目(通常，让 Github 保持绿色)是非常常见的-我...。页:1。像很多人想象的那样，为一家公司工作，只为贡献开源真的很酷吗？

我常说，贡献开源是上瘾的，它使你花费数小时和数小时去了解某人(或某公司)理想化的项目，直到到了发送第一个 PR ( *pull request* 、 *patch* 等)并为其实施辩护的时候了。开发人员(您)需要进化的第一个**【技能】**(如果您还没有)。是的，我说的是**销售**。不出售产品，而是从自己的想法(`aprender a se comunicar via texto - em geral em inglês`)来看对方*【问题】*，因为文字没有传递感情。今天我觉得很好，主要是想到一个贡献，思考一下软件的演变，而不仅仅是把我们相信的带入软件中(如果你保持开放源代码软件，对新的想法/需求持开放态度)。

起初是的，但就像任何梦想或一切都是可预见的(尤其是当你从未担任过这个职位时)。写这篇博客文章的想法是揭露一个你可能没有分析过的情景。当我们恋爱时，我们认为前进的道路是由花组成的。

## [](#vamos-direto-ao-ponto)让我们直奔主题吧！

当您参与任何项目(主要是开源项目)时，您需要花费几个小时(天、月甚至几年)来了解它，了解它之后，有两行:

*   -提出/落实你有什么需要；
*   -承担某人提出的任何问题。

也就是说，(自主性)的选择掌握在自己手中。这就是我想达到的地步:自主性。

但是当我在一家公司里的时候，会不会是一样的呢？
那么，很可能不会。你知道为什么吗？因为你得到报酬是为了实现公司拥有的需求而不是你的需求！
如果到了这一步，情况可能会更糟，尤其是如果你正在为一个大型**开源项目作出贡献的话。**

 **你有没有想过你**工作的公司**可能需要你不同意在项目中拥有的资源？而你必须捍卫一个你不同意的想法，因为你得到报酬？这一点很多想得到开源捐款的人都不会想到，我挑战你思考一下，当你走过这种情景时(如果你想担任这样的职位的话)，你会怎么做。

*   你会怎么做？

记住没有什么是永远的，明天你可能会在另一家公司，但是倡导/提议/斗争/揭发(使用适合你世界的词)的人是你，而不是付钱给你的公司！

## [](#ent%C3%A3o-n%C3%A3o-devo-aceitar-trabalho-para-contribuir-com-open-source)那么我不应该接受工作来贡献开源吗？

我相信出现了以下疑问:`Durante quase todo o texto parece que você (Avelino) apoia o fato de que a pessoa deve fazer aquilo que ela está sendo paga para fazer, mas no último parágrafo parece que você (Avelino) muda de ideia e traz a mensagem de que é o indivíduo que contribui e não a empresa e que deve-se ter cuidado.`

不存在对错，我写这篇博客文章是为了让你反映出**付钱给**开源可能没有很多人想象的那么有趣。我的建议是你发展以下技能:

*   -销售:不一定要销售“产品”，而是您的想法；
*   -沟通:不是每个人都会拥抱你的想法(这是好的也不是坏的)；
*   -学会说不:我常说学习说不随年龄增长而来。在这种情况下，*【不】*这个词表示你不同意任何提议的东西，不是你反对*的意思，即不是否定该人，而是否定(在这种情况下提议)要遵循的上下文；*
**   -学会讨论想法，而不是信念；*   -学会倾听:极其重要。*

 *最后，我认为你最重要的科学是:**开源由人维护**
