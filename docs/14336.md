### [](#hi-im-roseanna-%C2%A0a-kiwi-working-on-the-google-assistant-in-london)大家好，我是罗珊娜——一个在伦敦使用谷歌助手的新西兰人。

### 是谁或什么让你开始编程的？

我对讲故事的热爱:大约 2001 年的尼奥宠物(HTML/CSS)和 RPG Maker。

[![](../Images/b7ecad17f420d04d792b9d144d824751.png) ](https://res.cloudinary.com/practicaldev/image/fetch/s--3C9GcTnr--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://static1.squarespace.com/static/58bfb8c059cc68cc36914ea9/t/5987c4fcbf629a081ff2a857/1502070016598/%3Fformat%3D1000w) [ ![](../Images/79bbd8c30d10d87dc785ae564361f532.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--yU8zJbup--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://static1.squarespace.com/static/58bfb8c059cc68cc36914ea9/t/5987c50b914e6b456544736e/1502070031573/%3Fformat%3D1000w)

我想为我的 Neopets 创建背景故事，丰富多彩的网站和公会，Neo 在网站上有 HTML 教程。RPG Maker 在我不知情的情况下教会了我全局变量、循环和 if 语句！直到高中结束，我才真正学会了一门编程语言。

### [](#what-does-a-partner-technology-manager-do-what-does-your-day-to-day-look-like)合伙人技术经理是做什么的？你的日常生活是什么样的？

我本质上是一个技术顾问。我的角色是谷歌工作家族的一部分，称为技术解决方案顾问(TSCs)，他们有技术背景，但也希望更直接地与客户合作。我负责管理外部合作伙伴，他们正在使用谷歌平台上的[动作，与](https://developers.google.com/actions/)[谷歌助手](https://assistant.google.com/)集成。

我的日常工作包括解决技术问题、编写代码样本、合作伙伴方面的一点项目管理，以及将合作伙伴的需求传达给销售和工程部门。不同的合作伙伴需要不同的支持，所以这是非常多样的工作。我非常期待未来几个月去欧洲宣传这个平台。我也有一个不同的团队做一些前端 web 开发 20%的项目。

### [](#what-is-the-tech-scene-in-london-like)伦敦的科技场景是怎样的？

势不可挡。像伦敦的其他地方一样，每时每刻都有很多事情发生。聚会、会议、代码营、共同工作空间、孵化器..有很多有趣的初创公司从伦敦金融业和科技公司“五大巨头”中吸引人才。

### 你在小公司和大公司都工作过，你注意到了哪些不同？有什么偏好吗？

我在小公司有更多一对一的指导，知道公司里每个人的名字是一种很酷的感觉。这让我感觉更像一个家庭，我也感觉到了更多的支持，尤其是作为一名初级开发人员。我所参与的影响和范围是全国性的，而不是全球性的。小公司仍然可以是政治性的，但它没有那么复杂，我也没有管理“链”本身。我和设计师、测试人员、产品经理和开发人员打过交道。

现在，由于我的团队所处的位置和谷歌的规模，利益相关者包括市场营销、多个工程团队、多个产品经理、业务开发、政策/律师、语言学家……所有这些都在不同的国家和办事处。成为某件大事的一部分很酷，但这是一个更大的组织挑战。我不认为我有什么偏好，不同规模的科技公司有不同的课程可以教你。

### 迄今为止，在你的软件生涯中，你学到的最难的一课是什么？

求助不会让你变得软弱，不代表你不配拥有现在的地位，也不代表你没有能力。说实话，我还在学这个！我认为这很难，因为寻求帮助的想法会加剧冒名顶替综合症。

### 对于成功的软件职业生涯，你的第一条建议是什么？

建立你的人际网络并使之多样化:尽可能多地接触来自各行各业、不同技术领域的人，培养同理心，了解成功软件中的不同角色。如果你天生不擅长与人交往或建立联系，加强这块肌肉是值得的，这需要时间，只要练习和灵活它:)

### 对于想在谷歌这样的知名公司找工作的开发人员，你有什么建议？

学习学习学习，使用尽可能多的媒介(白板、纸张、谷歌文档、IDE、命令行、网站)进行练习。我们的面试风格是有据可查的，你可以在网上浏览很多账户——你可能认识参加面试的人！与他们交谈，但最终不要因为拒绝而气馁。我第一次没有被录取。你可以走另一条路，建立你的技能，如果它仍然符合你的职业目标，重新申请。请注意，面试过程，尤其是谷歌的，仍然存在固有的缺陷和“最大努力”。反转一个链表并不一定能证明你和其他人一起开发软件的能力。我们会问你关于算法、数据结构和软件工程角色潜在的语言特征。我的面试有角色扮演的故障排除/调试环节，描述了网络的各个方面是如何工作的，一些行为上的“如果……你会怎么做？”以及技术面试。
