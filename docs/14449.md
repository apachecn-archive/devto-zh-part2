# 作为开发者的一周

> 原文:[https://dev.to/cindywu/one-week-as-a-develubber-37 G1](https://dev.to/cindywu/one-week-as-a-develubber--37g1)

我在 2017 年 1 月 21 日写了这篇文章，在我开始称自己为开发人员(也就是工程师)一周后。

## [](#one-week-as-a-develubber)作为开发者的一周

杰夫给我发了这个[博文](https://www.vikingcodeschool.com/posts/why-learning-to-code-is-so-damn-hard)。我今天早上读了它，它让我对自己上周的进步感觉更好。

在过去的一周里，我读了一本反应书。我想我理解了其中的 50%。我试图将我的学习应用于修复 wizard 中的一些 bug，失败了(但学到了)。然后，我开始应用我所学的 react 来帮助构建我们新的引用功能。

周二到周五我有机会和杰夫搭档。我觉得在和杰夫一起工作的这段时间里，我学到了最多的东西。我想第一天我觉得我理解了我们正在做的事情的 40%。周三，我觉得我理解了我们正在做的事情的 20%。周四，因为在周三，我感觉我更失落，我更不乐观。但是周五的时候，我觉得我理解了 90%我们正在做的事情。

我认为阻碍我的最大的事情之一是，当 Jeff 不在的时候，我对代码没有安全感，但这种情况在周五开始改变，当时我们分开了，我们开始并行处理相同的问题。耶！

真正帮助我的一件事是，Jeff 向我展示了他是如何编写注释，概述如何将较大的功能分解为较小的尺寸。我相信我完全理解需要执行什么，我只是不知道如何用代码语言来写，在这里是 javascript。

我认为阻碍我的最大的事情之一是我自己的自信和我不熟悉特定语言的事实。当被要求用 javascript 写一个 if then 语句时，我的大脑一片空白。我可以用英语解释需要发生什么的概念，但是我不能把英语翻译成 javascript。我想在这方面努力。我想如果我努力理解我们使用的特定语言的语法，我可以克服自信的问题。

我现在的总体感觉是，我在概念上理解构建我们试图构建的东西涉及哪些步骤，但由于缺乏经验，我缺乏所有工具如何工作的基础知识。这是一件好事，因为我有了一个好主意，可以采取具体的步骤来提高我的基础知识(例如，学习 javascript 语法，使用在线 CSS 资源来熟悉常用的属性，等等)。).

不要放弃。不要太气馁。继续练习。熟能生巧。
