我们的代码评审聚焦是一个持续的系列访谈，所以你可以了解我们的顶级评审。通过下面的代码回顾，了解更多关于 Dylan 不断学习的动力:

#### 1。你的背景是什么？

我从 2011 年开始做网页编程。我过去经常使用 Rails 和 Ruby，尤其是在我最后一次在 Blue 围裙担任技术主管时。我帮助公司从 12 名工程师扩大到 100 多名。

我目前是 Grubhub 的技术负责人，我现在更多地使用 Java，但在我心中 Ruby 将永远占据一席之地。事实上，我曾经为 crystal-lang 做过贡献，这是一种类似 Ruby 的语言，经过编译、静态类型检查，速度类似于 c。

#### 2。为什么代码质量很重要？

当你写代码时，你通常只想完成工作，但更重要的是确保它以一种易于理解、可扩展且无副作用的方式运行。如果你的代码是高质量的，你会确保所有的复选框都被选中。你让你未来的自己(和你的同事)生活得更轻松。

#### 3。为什么良好的代码评审过程很重要？

简单地说，代码评审让你的代码更好。这是一种指导过程，使您的变更得到更高质量的审核，也帮助您作为一名工程师得到发展和成长。

如果评审者发现了你没有考虑到的错误或改进，评审有时会感到沮丧。然而，如果代码审查使您的更改质量更高，您以后会为此感谢自己。你和你的评审者都站在同一边，一起工作向用户交付高质量的软件。

#### 4。你为什么要用 PullRequest 复习？

PullRequest 允许我回到 Ruby 的世界，让自己跟上速度。我在 Grubhub 的工作是用 Java 编写的，所以代码审查可以确保我的 Ruby 技能保持最新。此外，我还可以看到我所在行业之外的所有类型的商业应用程序的代码；观察其他科技公司的内部运作总是很有趣。

#### 5。你发现了做第三方评审的哪些优势？

团队经常会陷入他们固定的方式中。让另一方(比如我)加入进来，向他们介绍他们原本不会考虑的可能的改进，这很好。每个人都带来了他们自己的背景和专业知识，然后团队能够对他们的最终产品做出更好、更明智的工程决策。

#### 6。您看到的一些常见问题是什么？

我看到的第一件事是对数据库属性以及应用程序代码在这些场景中的行为缺乏了解。开发人员需要理解事务、隔离、原子性和数据库给你的所有其他属性，以及当事情出错时会发生什么。

#### 7。你对开发者有什么建议？

提高您对正在使用的任何框架或技术的内部工作方式的理解。如果你正在使用 ORM，理解幕后的魔法是什么。如果您正在使用消息队列，请理解是否保证至少一次传递，以及当消息未能被处理时会发生什么。这样做会让你在以后的道路上免去许多头痛，而且你会因此变得更好。我保证这对你的职业生涯有好处。
