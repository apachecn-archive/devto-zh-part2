作为一名软件开发人员，同时也是一名社区关系经理，这让我意识到开发人员社区的重要作用，事实证明，它们是分散在整个技术行业的公司或组织发展的重要组成部分。他们的激情和创新精神使他们成为产品的终极增长黑客，这一点已经被脸书、谷歌等全球顶级公司利用他们的开发者社区成为他们所做的最好的。

开发者社区是一个快速发展的社区，可以包括技术领域的每一个人，从高级软件工程师到尝试新技术的高中生。他们使用技术的独特性和创造性方法使开发人员用户与其他用户完全不同，因为他们不仅仅是为了钱而工作/使用产品，而是因为他们对自己的技术充满热情。由于这个原因，许多技术组织对他们的开发者社区进行了大量投资。

建立一个开发者社区从来不像看起来那么容易，开发者喜欢以不同于通常用来接近新用户的营销策略的方式被接近，他们喜欢被视为产品不可或缺的关键，因此如果关键步骤没有执行，就很难吸引到他们。

下面是一些我用来帮助建立一个的指导步骤:

#### 找到并定义你的受众:

开发人员跨越技术的所有子领域，包括 web 开发、移动开发、桌面应用程序开发、游戏开发、数据科学、人工智能、机器学习、机器人等。对于广阔的领域，总是建议你仔细看看你的开发人员将从加入中获得的目标和好处，这样做有助于筛选。

例如:在 [Findworka](http://www.findworka.com) ，我们专注于构建和使用 web 和移动产品，因此我们很好地关注了对我们的 [Findworka PRO 平台的 web 和移动开发感兴趣的 onboarding PRO 开发人员。这样做可以为您节省一些时间，并有助于轻松地与您的开发人员交流、互动并最终参与进来。](http://www.findworka.com/pros)

#### 创建入职流程:

你如何把开发者带到你的平台上，对他们在你的社区中产生长期或短期的影响有很大的帮助。您的入职流程使您能够控制减少开发人员在开始使用您的产品时会遇到的常见障碍和问题。

你的入职过程也可以被看作是你给开发者创造关于你的产品或社区的第一印象的方式。许多公司通过他们的产品文档来做这些，这些文档旨在解释开发人员应该知道的关于使用产品的所有知识，一些专注于研究的社区通过组织初学者友好会议来做这些，以使开发人员熟悉他们的目标和研究小组的目标。已经被证明对开发人员有更持久影响的入职流程往往包括其他开发人员的案例研究，这些开发人员要么使用该产品，要么在加入社区后取得了一些成就。

#### 衡量指标:

指标有助于你更好地了解你的开发者成员，他们在哪里以及如何被重定向到你的平台，以及他们在你的平台上的活动。

简单的衡量标准，如检查您的社区聊天组的参与度、社交媒体上的关注者数量、成员对特定主题或语言的兴趣、对某些产品或技术的兴趣等，在利用您的社区的潜力和增长方面有很多技巧。

#### 打造领导者:

让特定的成员担任领导角色有助于保持社区更加集中和团结，较小的角色(如团队领导、小组领导、管理员、大使)或较高的角色(如开发人员倡导者、开发人员社区经理等)会带来更多的信心，并为其他开发人员创造更多的信任，因为这些领导往往更了解他们的需求，并能与他们的技术情况相关联。

在构建开发人员社区的过程中，以下是一些表明发展的特征:

1.  创意:许多人犯的错误是没有很快意识到开发人员属于产品部门，而不是营销或销售部门，这就是为什么许多销售和营销策略对他们不太有效的原因。如前所述，开发人员喜欢被认为是产品不可或缺的一部分，可以很容易地看穿所有的营销策略，他们喜欢公司或组织信守承诺，最终兑现承诺甚至更多。保持透明，目标是在避免垃圾邮件的同时为他们的生产力增加更多价值。
2.  **可见性**:发布更多关于您的社区或组织的信息会增加信任度并增加成员数量，让您的开发者拥护者或社区经理代表社区参加以开发者为中心的活动不仅会建立良好的关系，还会表明您的社区或组织始终是可访问的。 请记住，开发人员不会对销售代表的传统营销策略作出回应，因此他们更倾向于与来自技术背景的代表打交道，而不是仅仅专注于向他们推销组织的产品，而是愿意讨论和分析其他产品和开发人员工具，这些产品和工具有时证明是更大的竞争，但唯一的目的是教育他们并为他们增加价值。
3.  **高度重视反馈**:了解哪些计划或行动在你的社区中可行或不可行的一个很好的方法是接受反馈，因为它们有助于你的社区或产品更快地迭代，社区有很多成员反馈，因为它们是他们首先开始的唯一优先事项，问题跟踪、功能请求或 Q & A 是获得反馈的一些示例方法。对从反馈中得到的问题或建议采取相当多的行动会让你的成员更加信任，因为这有助于他们意识到他们的意见被认为是重要的。
4.  **伟大的指标**:指标的增加在很大程度上表明增长，并表明正在采取正确的步骤。
5.  **奖励或激励:**在不同的场合发放奖励或奖励，如礼品包、t 恤、免费技能获取、奖学金等，鼓励会员参与和参与更多，许多开发者也因为社区提供的奖励和激励而加入某些社区，谁不喜欢免费赠品和好东西呢？

希望开始构建一个或改进一个现有的，这里有一些我优先考虑的要点，你可能想注意一下:

**Starter pack** :对于一个刚刚起步的社区来说，拥有以下内容有助于在初始/开始阶段在成员和社区所有者之间建立必要的联系

1.  时事通讯(帮助传递有关新计划、当前趋势等的信息)
2.  行动守则
3.  聊天平台(便于交流)
4.  支持系统(一些社区将它与聊天平台结合在一起，但是在新成员遇到困难的时候有这个帮助)

对于一个已经存在了几个月或几年的社区来说:

1.  博客:张贴技术文章，解释关于你的产品或几种技术的各种技术概念，这些文章可以由职业作家或来自社区的客座撰稿人撰写
2.  集成反馈系统
3.  板载社区专家:开发者倡导者/开发者社区经理等
4.  进入高级阶段:以教程、黑客马拉松、演示等形式创建更高级的技术概念

**Master Pack** :针对会员率激增的现有社区:

1.  会议:举办你自己的开发者大会。

#### 结论:

开发者社区可能非常令人兴奋！它们可能是人们从事编码工作的唯一原因，也可能是创业成功背后的原因，或者是为什么开发人员喜欢他或她的工作的原因。他们促进成员之间的联系和互动，并帮助弥合差距。

建立一个开发者社区需要耐心，但是如果你有热情并且愿意在做的时候找到快乐，这也会是一个有趣的旅程。
