# 开发一个物联网的 App 之前应该了解什么？

> 原文:[https://dev . to/patelmiteshb/what-should-I-know-before-a-app-for-the-internet-of-things-3e3](https://dev.to/patelmiteshb/what-should-i-know-before-developing-an-app-for-the-internet-of-things--3ie3)

到目前为止，数十亿台联网设备正在传输大量实时数据，这是我们都深信不疑的物联网潜力。作为前进的一步，企业已经将他们的重点从仅仅连接的 DevOps 转向高度可靠的 DevOps，并且 80%的物联网支出将被 B2B 市场夺走。

让我们来看看一些有助于您开发更好的物联网应用的见解。

**寻找一个随时可以“部署”的平台**

随着物联网变得越来越占主导地位，企业渴望通过“随时部署”平台进行投资，以便普通开发人员能够更轻松地完成开发工作，并超越传统开发，构建“下一代应用”这种平台嵌入了摄取和分析层，而开发人员可以专注于核心 SDLC 流程。

苹果、微软和谷歌等领先企业更喜欢使用预构建的环境，而不是从头开始。

**考虑对短期目标的长期影响**

从头开始构建一个物联网应用程序可能是一项艰巨的任务。所以，如果在某件事上投入时间和精力，要着眼于更大的图景和长期影响。当想到物联网、自动化家庭时，看一看更大的画面，如联网城市可以是该任务的最终目标。鉴于物联网设备不再处于起步阶段，考虑到可扩展性需求

随着对物联网技术的大量了解，以及该技术仍处于起步阶段，这些技术和不同形式的物联网将在未来几年呈指数级增长，可能是 5-10 年。因此，从长远来看，考虑应用程序的可伸缩性会给您带来很多好处。所以，就把它想象成一种资源，它的获取应该尽可能广泛！

**查看设备的固件安全性**

传统应用和[物联网应用](https://www.solutionanalysts.com/)的不同之处在于，它们与事物而不是人进行交互。因此，用于开发这些应用程序的固件应该是安全的，并定期更新其安全框架，以便在未来时间内检测到的任何新漏洞都可以在应用程序和设备中远程处理，就像防病毒软件一样。

此外，包括保护数据的安全功能，从而使用安全的硬件组件。虽然安全性是 DevOps 的一个关键问题，但加密和认证细节必须包含在应用程序中，以便保护数据免受任何漏洞或非法黑客行为的影响。

**明白物理攻击也很严重**

我们传输到设备的数据被管理和分析，以便在这些设备上安装的芯片上进行进一步的计算。例如，这些芯片可以是集装箱中的 GPS 芯片，或者空调上的恒温器等。这些芯片在制造时应该考虑到物理上的弱点，比如，它们应该很难被移除或篡改。

最后，

**找到合适的开发人员**

根据惠普最近对不安全物联网应用的研究，来自硬件供应商的软件开发开发者比其他人更容易受到安全威胁。也就是说，由于缺乏风险识别方面的实践知识，不具备软件应用程序专业知识的硬件供应商可能不是最佳选择。

让最好的开发人员加入进来。寻找项目的“相关开发人员”，而不是“可用开发人员”。不要忽视组织交叉训练课程，让你的团队在每个级别都充实自己的基础知识。

当我们稳步迈向激动人心的 2020 年时，从一开始就掌握基本知识将有助于消除运行时突然出现的性能问题。
