## 可视化依赖图

在对我们的系统设计进行建模时，在图表中可视化我们的代码是一个很好的思考框架。设计的早期可视化可以成为与同事交流想法和计划/协调团队工作的优秀工具。图表还可以帮助[独立开发](https://www.essentialdeveloper.com/articles/consequences-of-ineffective-communication-in-ios-teams)，因为团队可以快速规划新功能设计(在白板上 5-10 分钟)，并行构建它们，然后集成各个部分，而不会与其他开发人员的工作冲突。

此外，图是一个很好的诊断工具，因为它们体现了组件和模块的依赖图，揭示了紧密耦合，甚至保留了可能导致僵化和[内存泄漏](https://www.essentialdeveloper.com/articles/the-minimum-you-should-do-to-prevent-memory-leaks-in-swift)的循环。学习如何绘制和阅读图表是开发人员技能的一个极好的补充，他们的目标是创建可持续的代码库。
