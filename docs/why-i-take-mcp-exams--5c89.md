# 我为什么参加 MCP 考试。

> 原文：<https://dev.to/kenakamu/why-i-take-mcp-exams--5c89>

我不仅喜欢微软的认证，也喜欢其他的认证。但是我的朋友经常问我为什么要参加考试，因为他们大多数人认为证书没有用。嗯，我能理解他们为什么会有这种感觉，但这就是我如何利用考试来学习更多东西。

# 作为学习目录

当然，我不是什么都知道，因此我搜索或学习的内容仅限于我现在知道或感兴趣的内容。为了拓展我的技能和知识，我需要有人或事告诉我可以学习哪些技术或产品。这就是为什么我去 [MCP 考试列表](https://www.microsoft.com/en-us/learning/exam-list.aspx)看有哪些考试，这给了我学习新事物的选择。

# 作为学习的准则

当我决定学习哪种服务或产品时，我需要有人告诉我从哪里开始。

*   大多数考试给我**技能测试**部分，在那里我可以看到我应该知道的术语或知识。
*   我可以按照章节逐一了解服务或产品。

# 为真实场景

在我完成了我的学习和实践之后，我仍然需要真实的体验，需要了解场景。

*   每次考试给我们 30-100 个问题
*   有些考试提供了案例研究基础，这是检验知识的好方法

我知道有人说考题没道理。但是，考试质量其实越来越好。如果你愿意，你也可以给出反馈。

# IT pro 考试？

我是一名开发人员，但我参加了 IT 专业考试。为什么？

*   因为我可以从另一个方向学到东西。例如，我可以通过了解产品的特性来学习服务或产品的设计模式。
*   我可以看到 IT 专业人员面临的挑战，这样我就可以更好地与他们合作。现在是开发时间，所以无论如何我都需要了解它

好吧，就这样。因为我学习新的东西来提高自己，我从来没有为了通过考试而学习，因此我经常失败。这实际上很好，因为我又有了一个深入学习的机会:)
