# 内容安全策略

> 原文:[https://dev.to/funkysi1701/content-security-policies-30n2](https://dev.to/funkysi1701/content-security-policies-30n2)

内容安全策略(CSP)是一个 HTTP 响应头，它定义了可以在网页上加载哪些内容源。这是一种对抗跨站脚本(XSS)攻击的方法。

### 那么什么是 XSS 攻击呢？

当你加载一个网页时，它也会加载各种其他资源，比如图像、一些 css 样式表、你想要运行的各种 javascript 文件，可能还有许多其他东西。

你怎么知道你可以信任所有这些事情？如果你创造了他们，他们生活在你的控制之下，那么答案可能是肯定的。然而，最近你可能想使用互联网上的资源，比如 youtube 视频、谷歌分析、disqus 评论、cdn 的 jquery 库等等，你不能确定他们到底在做什么。

假设您有一个页面，您可以将任何文本添加到表单中，然后显示出来。恶意用户可以添加恶意 javascript 或让浏览器从互联网上的任何地方加载恶意代码。

### CSP 来救援了！

CSP 只允许浏览器从您指定的来源加载。您可以指定加载您自己站点的资源，但不加载邪恶的脚本。

让我们看一些例子

`Content-Security-Policy: script-src 'self'`
