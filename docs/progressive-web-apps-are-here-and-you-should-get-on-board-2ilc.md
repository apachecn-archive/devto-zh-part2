# 进步的网络应用已经出现，你应该加入进来

> 原文：<https://dev.to/asaf_g6/progressive-web-apps-are-here-and-you-should-get-on-board-2ilc>

## 炒作和议论

进步的网络应用程序最近被大肆宣传，这是有原因的。它们为大多数类型的应用程序提供了强大的优势。在你的手机上安装一个应用程序来做任何事情，对我来说没有任何意义..然而，在我看来，通过网络浏览器提供应用程序是不可避免的。

那么有什么好大惊小怪的？

用一句话来说——通过浏览器提供移动应用，仅此而已。
但在我们谈论它的好与坏之前，让我带你去一趟我想象中的时间机器之旅。

## 假设我们有一台时间机器

让我们说，我们已经完成了时间机器提供的所有很酷的事情——去听吉米·亨德里克斯的音乐会，看《生活大爆炸》(真实的一部，不是连续剧),见爱因斯坦或任何你想做的时间旅行的事情。让我们回到 2000 年左右，看看你的台式机或笔记本电脑上有什么应用程序。我的猜测是，你有一些实用程序，如防病毒，解压缩之类的东西。一些通讯应用程序，如 icq 或 skype，一些“专业”软件，如文字处理器或照片编辑器，也许还有一些游戏。当然，最后但并非最不重要的一点是网络浏览器。随着时间的推移，越来越多的软件从在你的机器上运行变成了在你的网络浏览器上运行。现在，我确信在你的桌面上有一个新闻应用程序会感觉很奇怪。事实上，有可能你的电脑上只有很少或没有应用程序，而且你对只有一个网络浏览器也很满意。手机有点不一样，我们也不是在 2000 年，但是有人说历史一定会重演。

## 那么渐进式 Web 应用有什么好的呢？

#### 观感

渐进式 web 应用程序看起来和感觉上都像本地应用程序。

它们在用户体验方面的特点几乎相同:

*   全屏工作。
*   主屏幕按钮。
*   离线工作。
*   通知。

#### 方便开发者

对于渐进式网络应用程序，你让浏览器来做繁重的工作。

*   一个适用于所有平台的代码库。
*   适合任何尺寸的屏幕。
*   您的所有用户始终拥有您的最新版本
*   没有来自应用商店的限制。

如果你想知道支持哪些硬件特性，你可以检查一下[https://whatwebcando.today/](https://whatwebcando.today/)。

#### 便于营销

用户可以通过搜索引擎发现你的网络应用，或者点击一个点击链接。
今天，你给一个 app 做广告，流程是这样的:用户看到你的广告，点击它。然后，用户会被重定向到您的应用商店页面。在那里，用户必须下载你的应用程序，打开它，当然还要使用它。

渐进式网络应用:用户看到你的广告并点击它。用户要么在你的应用上，要么被重定向到浏览器，然后在你的应用上。它要短得多。

#### 更好的为用户服务

最后，但同样重要的是:

*   没有安装过程。
*   最小的磁盘空间消耗。
*   更快的加载时间。

## 我们到了吗？

嗯，不。还有一些障碍要跨越。

现在的用户在应用商店里寻找应用，所以如果你的应用不在那里，他们可能找不到。你如何向用户解释渐进式网络应用？不是很清楚，大部分人都没有意识到。

那么这是一个移动网页？还是一个 app？我必须下载它吗？

还有主屏幕按钮的问题。用户必须首先访问 web 应用程序，然后进入浏览器菜单并点击“添加到主屏幕”。这不直观，也不够好。

许多用户的手机上有多个浏览器，可以同时打开你的应用。这对你和用户来说都是非常混乱的。用户可能会在他们的主屏幕上看到你的应用两次，你可能会认为你有两个不同的用户。

## 不是每个人都适合的

我相信有些应用程序总是需要安装在设备上。无论是性能问题还是应用的本质。看到一个闹钟渐进式的 web 应用程序真的会很奇怪，对吧？真的没有意义。游戏通常需要更好的性能，我们可以期待它们在正确安装后会有更好的表现。
