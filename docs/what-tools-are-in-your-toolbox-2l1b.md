# 你的工具箱里有什么工具？

> 原文：<https://dev.to/lemuelogbunude/what-tools-are-in-your-toolbox-2l1b>

你使用什么工具？除了编程语言，我们工具箱中的下一组项目通常是生产力工具，帮助我们快速完成工作，减少错误的东西。

我将分享一个我使用的工具列表，我是通过一个朋友或者可能是我读过的一篇文章了解其中一些工具的。因此，本文的目的是让那些可能不知道这些工具的人了解这些信息。

这不是一个详尽的列表，所以请随意提及一些你也使用的，我很想知道😉。

## [t1](#git)[去](https://git-scm.com/)

[![Git](../Images/017985d2cf1e2dd14c6d222ca44a0a8d.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--v4NY2Ei---/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/dze5gp97ij5ra07jks2q.png)

您可能正在使用 Git 进行版本控制，如果您以前没有使用过，我强烈建议您检查一下。你可能会复制并粘贴你的代码，并标注日期，老实说，我在开始使用 Git 之前就这么做了🙄。

当你对 Git 很熟悉的时候，你可以去看看 GitHub，在那里你可以存放你的代码。GitHub 还有其他替代方案(以防你对微软心存警惕😅)比如 GitLab。

## [t1](#git-kraken)[去破解](https://www.gitkraken.com/)

[![Git](../Images/fca6e9e6436189cbb47c2ceeaf1d49c5.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--jsQLaWHv--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/zimjjadkblvrsj6eew4i.jpg)

使用 Git 一段时间后，您可能需要一个工具来帮助您可视化您的 Git 提交，并帮助您以一种更加用户友好的方式执行 Git 操作。有不同的 Git 客户端非常棒，其中之一是 GitKraken。如果你的目标是变得更有效率，看看 GitKraken:)。

## [Visual Studio 代码](https://code.visualstudio.com/)

[![Git](../Images/28a69a4112a785d8c984259e6e83a78d.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--WWggTk9u--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/4f3hu07y3qxs51zosqyn.jpg)

我猜有很多“VS 代码拥护者”正在阅读这篇文章，有趣的事实是我并不是 VS 代码的真正粉丝。哈哈坚持住！先不要关闭浏览器！故事结局很好:)

不是不喜欢 VS 代码，只是没看出什么特别。嗯，直到我的一个朋友把它介绍给我。从那以后，我就在工具箱里添加了 VS 代码。

我在工作中使用 Android Studio，所以我并不真的*需要* VS 代码，但是它太好了，我不得不把它作为我工作工具的一部分。我真的很感谢那些努力为 VS 代码做巨大扩展的人。

## [面料](https://get.fabric.io/)

[![Git](../Images/75f7f4895e131c9354854a9951085b84.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--VOjmPJpK--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/a1zjly7zxd93ejhq5709.png)

Fabric 是一个平台，可以帮助您的移动团队构建更好的应用，了解您的用户，并发展您的业务。我主要用它作为一个崩溃报告工具。

借助 Fabric，您可以获得强大而轻量级的崩溃报告，以便快速修复问题，简化和自动化您的测试版分发和应用部署流程，深入了解您的用户及其在您的应用中采取的操作，发现与健康和不健康用户相关的行为，推动应用安装并监控活跃用户。

综上所述，就是牛逼！

## [InstaBug](https://instabug.com/)

[![Bug](../Images/c37c3fbaeb270a46a310293ca0699f67.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--vSUuK-0x--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://i.ibb.co/cvmNGwt/Screenshot-2019-12-11-at-5-02-41-PM.png)

有许多崩溃报告工具可供选择，但是，如果你想要的不仅仅是一个崩溃报告工具，InstaBug 可能是你的解决方案。用户可以报告错误，并能够通过应用程序与你聊天，发送反馈，甚至进行应用内调查，这将有助于你更好地了解你的用户。

## [滚动条](https://rollbar.com/)

[![Git](../Images/f5be2f98db8a585fd029b903d9f2f9dc.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s---SCBJcs1--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/yr6z98xnqna4i0xo4917.jpg)

类似于 Fabric 的 RollBar 帮助我在用户受到影响之前解决错误。从我个人的经验来看，我真的很喜欢 RollBar，你可以选择将它与你已经使用的工具集成在一起。您还可以使用 webhooks 及其全功能 REST API 来创建定制集成、导出数据等等。我用 RollBar，我建议你去看看。

## [【bug snag】](https://www.bugsnag.com/product/)

[![Git](../Images/6cd98130744f821508bbcd9d88ebd332.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--05ta5G5G--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/rykt8dtto9jp9bv78e9x.png) 
Bugsnag 是我之前提到的其他错误报告工具的替代工具，使用 Bugsnag，您可以为 web、移动和后端自动收集诊断信息，并根据根本原因对错误进行分组。您还可以评估每个错误的影响，跟踪发布状况，发现回归和峰值。
除此之外，你可以看到崩溃的代码行，深入挖掘困难的错误，跟踪修复进度。

总之，这是一个非常棒的错误报告工具。

## [Codacy](https://www.codacy.com/)

[![Git](../Images/68c3045ba84ac0dfd25680d383138ef0.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--0jhSGmRA--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/abcdex9wwx0a9fms82pb.png) 
Codacy 非常棒，它可以帮助你进行自动化的代码审查和代码分析，如果你以前没有检查过，我也强烈建议你这样做👨‍💻👩‍💻。您可以在每个提交和拉取请求中自动识别新的静态分析问题、代码覆盖、代码重复和代码复杂性演变。太棒了，太好了。

## [代号气候](https://codeclimate.com/)

[![Git](../Images/084a27c7ec942a7743a291b7aaaacf9e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--vwro24IF--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/6o2ww71uo1nbb51ds6go.jpg)

Code Climate 是另一个可以用于自动化代码评审的工具，这类工具可以帮助你更快地发布更好的软件。代码气候为您提供详细的分析，例如代码的认知复杂性、代码变动、可维护性以及类似的分析。这是一个令人敬畏的非常有用的工具。

## [圈词](https://circleci.com/)

[![Git](../Images/5f28e0b2d157fb2902e680a835107f1f.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--AWqxAE7Z--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ly4myq9343sf64rvhwi5.png)

如果你有一个团队希望快速构建东西，在创纪录的时间内交付产品，你将需要一个工具来帮助尽早快速地解决问题。CircleCI 是持续集成，是软件团队正在使用的一种开发实践，允许他们在多个平台上更容易、更快速地构建、测试和部署应用程序。

您可以将 CircleCI 与 GitHub 上的项目集成在一起，这样每次提交时都会运行测试，因此可以更早、更容易地发现错误。还有其他持续集成工具，如 Travis CI 和 Jenkins。
