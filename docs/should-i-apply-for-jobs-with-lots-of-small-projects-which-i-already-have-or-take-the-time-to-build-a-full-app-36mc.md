# 我应该申请已经有很多小项目的工作，还是花时间建立一个完整的应用程序？

> 原文：<https://dev.to/ermal/should-i-apply-for-jobs-with-lots-of-small-projects-which-i-already-have-or-take-the-time-to-build-a-full-app-36mc>

我认为我足以胜任初级工作。我在 HTML，CSS 和 JavaScript 方面很强。我已经用 mongo，react，甚至 wordpress 创建了电子商务模板(前面的部分)(很多年前，wordpress 不会出现在投资组合中)。

但是这些模板不处理支付，react 模板甚至没有后端——所以没有办法向数据库添加更多的项目。**我没有这么做的主要原因是，我认为雇主无论如何都不会看到这一点(他们不会有认证信息),因此我把重点放在他们能看到的东西上**

我搜索了 dev.to，强烈建议专注于一个项目并充实它，我会这样做，但在专注于我生活的另一部分 6 年后，我真的想找一份入门工作。上次我检查时，我有以下内容

*   电子商务布局:在我专注于设计的地方，它是有反应的，我努力不保持传统。
*   艺术家作品集:我虚构了一个人，好像他们是画家，需要一个网站。
*   图书电子商务:有折扣、本周作者、时事通讯、博客文章等板块
*   图书作者网站(想想维基百科上的几个作者):时间线，引文，书籍，简历都在一个页面上
*   毕加索的网站:完全不同的设计

简而言之，如果工作描述是强大的设计、css 和 html 技能，并知道如何使用 react(消费 API)，**我知道一个事实，如果其他一切都是平等的，我就是这份工作的人**我有设计的眼光，注意细节，努力不令人厌烦

(**我听起来像个大头目某某，对不起。**)

我感到害怕、害羞，我脑子里的某种东西告诉我，我六年没工作，把事情搞砸了。我的另一部分说，我今年迫切需要进入这个领域(我很快就 30 岁了)。

假设这些静态网站是我所认为的那样(独特、反应灵敏，并显示出我擅长 JS 和 react)，**你认为我有很大的机会获得一份入门级工作**还是我应该真正专注于开发一个完整的应用程序？

我觉得就我的记录来看，我所拥有的全部热情是专注于改善我生活中需要改善的部分(完成了残疾人的事情，现在是职业生涯——但与改善我的残疾不同，我觉得职业生涯需要别人让我参与)，以及一个投资组合。我问的是上述组合是否足够好，或者我应该在考虑申请之前专注于建立一个完整的应用程序。

我可以构建完整的应用程序，我已经创建了关于身份验证和制作完整应用程序所需的一切的迷你演示，我只是喜欢在上面工作，现在我觉得创建一个让我感到自豪的应用程序可能需要几个月的时间！？！

我想做的应用程序是:github 管理器，和一个文本编辑器(基于电子或网络)。没有一个是独特的想法，但是...
