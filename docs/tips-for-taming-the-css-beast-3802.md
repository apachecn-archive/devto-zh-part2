## 我:

嗨！谢谢，我很高兴你喜欢这篇文章。😁是的，CSS 对很多人来说很难。一个问题是，通常有许多不同的方法来做同样的事情，都有不同的利弊，不同的选择正在流行和过时。我认为你能做的最好的事情是:

### 让自己充满伟大

让自己沉浸在来自真正擅长 CSS 的人的内容和代码中。韦斯博斯有几门免费课程。查一下 [JavaScript 30](https://javascript30.com/) 和他的 [CSS 网格课程](https://cssgrid.io/)。当您浏览这些内容时，您会看到一些非常有用的最佳实践。登录 Codepen，四处看看，关注那些用 CSS 做酷事情的人。莎拉·德拉斯纳是我想到的一个人。看看他们的代码。克里斯·科伊尔和其他人写的关于 CSS 技巧的文章几乎都很优秀。看一看[引导](https://getbootstrap.com/docs/3.3/css/)和其他中的实际 CSS 文件。查看`.text-center`或`.btn btn-primary`类后面的 CSS，了解它们是如何工作的。

### 练习

建造东西。构建 100 万个使用 CSS 和 HTML 的小项目(同样，Codepen 很适合这个)——如果你愿意的话，还可以使用 JavaScript！你练习的越多，你就越能发现问题，如何解决问题，以及好的设计。

### 获得帮助

求助！如果你被困在一个特定的问题上，不要害怕寻求帮助(就像你现在做的一样)🙂).我总是很乐意看一些 HTML/CSS，并尝试解释问题是什么，以及一些解决问题的好方法。无论如何，希望这有所帮助。如果你还有其他问题，请告诉我。

> 重要的是不要试图记住所有的属性和标签。

理解浏览器如何布局的基本概念，行内元素与块元素，盒子模型，以及选择器和特性如何工作。看看像 BEM 这样的好的实践(只是一种布局你的 CSS 的方法)。一旦你开始了解这些基本原理，你就会感觉到“我想有一种特性可以满足我的需求，但我不知道它叫什么。可能是边框什么的？我去谷歌上查一下。”我一直都这么做。W3Schools FTW！😁
