# 何时停止一个项目:后退一步，休息一下

> 原文:[https://dev . to/tee katwo/when-to-call-it-quit-on-a-project-592h](https://dev.to/teekatwo/when-to-call-it-quits-on-a-project-592h)

我试图用我刚开始学习的框架 Angular 为第 6 届 Scotch.io 挑战赛制作一个番茄计时器。一开始还挺顺利的。我复制了想要的设计外观。我已经设置好了组件，只需要创建 javascript 来让计时器从给定的时间开始倒计时。这是它急剧下滑的地方。

## 问题来袭

我就是无法让计时器倒计时，无论我怎么尝试，我觉得我有一个很好的游戏计划。我想我知道该怎么做，这是没有问题的。结果完全不是那样。我花了几个小时试图寻找任何做过类似事情的人的例子，但毫无结果。没有一个例子是我使用 Angular CLI 的设置。他们都使用依赖于$scope 的设置，我对此一无所知，也不知道它与我的设置有什么关系。

我是一个很有决心的人，所以我花了一晚上的大部分时间试图打破这个“问题驼峰”。想着如果我足够努力，一切都会到位，一切都会变得有意义。对我来说通常就是这样，我做研究，试错，然后事情就成功了。我有点明白我需要做什么，然后这是我学到的东西。

## 宣布退出

快到凌晨 3 点了。我非常累，非常生气，生自己的气。我太想这么做了，我以为我可以做得更好。这实际上既不公平也不真实。从某种意义上说，这与我的能力无关，我是个失败者，但更重要的是，我在这方面确实是个新手。为什么我应该对一个我才开始使用不到一个月的框架了如指掌？有一次我意识到，我觉得在凌晨逼自己去学习 Angular 的一切是很傻的。

我不想这样做，但我把这个项目搁置了，制作一个带棱角的番茄定时器。我在 [Vue](https://codepen.io/Vpugh/pen/GxmmwO) 成功了，并不后悔。这是一个我已经体验了几个月的框架，我感觉非常舒服。花了大约两天时间，但我让它工作起来，并把它充实成一个功能齐全的番茄定时器。

## 最终结果

如果我对我的棱角分明的项目过于沮丧并放弃了，我永远也不会走到那一步。我知道当我在使用 Angular 方面有了更多的经验，并且对它的工作原理有了更好的理解，我会回来完成这个项目。我毫不怀疑。

有时候你需要放弃，以后再回来。如果你不知道为什么它不起作用，或者你觉得你错过了什么，退后一步，换个角度看问题。没有足够的经验并不可耻。不要放弃，你随时可以回来。
