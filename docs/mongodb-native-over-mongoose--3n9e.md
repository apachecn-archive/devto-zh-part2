# MongoDB-原生于猫鼬之上？

> 原文：<https://dev.to/tojacob/mongodb-native-over-mongoose--3n9e>

大家都用猫鼬！我不喜欢 mongoose，它实现的所有抽象在处理需要一个以上的查询才能工作的复杂事务时都不是很有用。而且，我最后学的是“mongoose”而不是 mongodb。

然而，我认识到这个方案的重要性，因为它“方便”了团队中其他程序员阅读代码。

通常(说到 api)，我对传入的数据做的第一件事是检查和验证它们，以便控制器只处理“干净的”数据，而不必在任何传入的数据不正确时徒劳无功。

由于之前的事情，在模型中实现方案变得没有必要，因为数据是有效的。我喜欢这一点，但仍然有一个问题，程序员必须分析验证器，以了解他们正在处理什么类型的数据。

最简单的解决方案是创建模式，即使数据已经过验证，但是这将导致在整个应用程序中重复大量代码。

你认为这些计划被高估了吗？我错过了什么吗？
你经历过类似的事情吗？
他们是如何解决的？对新手有什么有用的建议吗？

我知道这可能看起来很傻，但是最近我“稍微”更加关注架构，并且我发现没有计划和模型的工作违背了 MVC。
