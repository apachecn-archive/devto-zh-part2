# 你对创建新用户的想法

> 原文:[https://dev . to/jessachandler/your-thoughts-on-creating-a-new-user-3 nke](https://dev.to/jessachandler/your-thoughts-on-creating-a-new-user--3nke)

为了处理某人成为付费 web 应用程序用户的情况，作为开发人员，“我们”需要采取几个步骤:

*   将它们添加到数据库中
*   验证他们的电子邮件
*   向他们发送欢迎消息
*   改变他们
*   重定向他们
*   还有别的吗？

大多数开发指南包括“将它们添加到数据库”和“重定向它们”。我已经创建了应用程序中的所有步骤。我天真地采取了所有的步骤，并为一个新用户加入了路由处理程序。

当我站在真正重要的发展的边缘时，我在想:

*   操作的顺序重要吗？(显然，我们不想在实际创建帐户并向他们收费之前重定向到一个经过身份验证的区域，但我们应该等到他们点击“确认”电子邮件后再创建帐户吗？)
*   你有关于你的经历的故事吗？
*   如果您通过验证服务验证电子邮件，是否仍要发送验证/确认电子邮件？
*   我是不是错过了什么大事？
