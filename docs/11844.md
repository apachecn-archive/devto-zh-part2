# 推杆

[推手](https://pusher.com/?utm_source=dev.to&utm_medium=referral&utm_campaign=Devtocontest)是我们的第一个*白金赞助商*，这意味着我们正在和他们做一个超级特殊的合作关系。这种形式就是[推手大赛](https://dev.to/devteam/first-ever-dev-contest-build-a-realtime-app-with-pusher-4nhp)。

我们正在使用 Pusher 自己为[开发到](https://dev.to/)构建几个实时特性。Pusher 是一家非常受尊敬的工程组织，我们很乐意在他们的基础设施上进行建设。

## 数字海洋

[数字海洋](https://try.digitalocean.com/performance/?utm_source=devto&utm_medium=display&utm_campaign=Devto_2018_Brand)是一个备受喜爱的云计算平台。我总是对数字海洋核心产品的良好构建*和良好记录*印象深刻。说起来容易做起来难，他们已经能够保持领先。

数字海洋很容易上手，它可以与您的 Pusher 竞赛参赛作品配对，以托管您的应用程序的云存在。只是一个想法。😄

## [t1【bug snag】](#bugsnag)

[Bugsnag](https://www.bugsnag.com/?utm_source=dev.to&utm_medium=cpc&utm_content=recognition_post&utm_campaign=dev.to-2018-may) 为 web、移动和服务器应用提供错误监控。它在前端是轻量级的，在后端是非常复杂的。它几乎可以与任何东西集成，并且有一个免费层。团队中也有很多优秀的人。

说真的，良好的监控对晚上睡个好觉至关重要。看看 Bugsnag。

## 木材

木材是伐木业中最有趣的产品之一。围绕可读性和可用性构建，Timber 是一个您实际上很乐意使用的记录系统。我们自己也是测试版时代的木材用户*，现在我们和他们共用一个办公空间*。他们正在做伟大的工作。

Timber 目前与 Ruby、Elixir 和 Node 应用程序的集成最为无缝。这是我们团队极力推荐的。
