# API 的定义

> 原文：<https://dev.to/dandevri/the-definition-of-apis-116l>

> 如果一个人不至少 50 次提到 **API** 这个词，你就不可能和一个开发人员交谈。在这篇文章中，我想探究 API 的定义是什么，有哪些类型的 API。

## 定义

连通性是一件神奇的事情。不同的应用程序和设备如何相互连接？这是应用程序编程接口。

> API 创建应用程序和设备之间的连接。

大多数计算机程序以**用户界面**的形式为人类提供数据，以使数据的消费尽可能令人愉快。操作系统都提供 GUI(图形用户界面)来与系统交互。用户并不真正关心技术方面的东西。类似地，API 为开发人员提供了一种更简单的方式来与其他类型的软件进行交互。数据需要由另一个程序而不是“用户”来使用。

> 如同为人类量身定制的用户界面一样，软件需要一个易于使用数据和/或功能的界面。

API 是信使,它接收请求，告诉系统做什么，并返回响应。例如，您可以访问另一家公司的数据库。API 有一个接口，可以询问其他系统的信息。你可以说 API 是不同程序之间沟通的**中间人**。

API 层帮助**标准化编码过程**。在 web 上，它可能是基于 JavaScript 的 API，是底层过程的抽象。

## 比喻墙上插座

可编程网络出版了一个伟大的系列；什么是 API，它们是如何工作的？他们有一个很棒的小类比，我想引用一下:

> 你可以把一个 API 想象成一个墙壁插座。电力是服务，而设备是该服务的消费者。在这种情况下，消费者将其电力需求外包给服务提供商。电插座有可预测的模式。插头是标准的，它提供的功率(伏特)是一致的，并且几乎是相同的值。同样，API 规定了软件组件应该如何相互交互。

想象一下，如果没有这样的标准，你将如何获得权力。

## 好处

API 不仅可以帮助开发人员编写更好的程序，它还可以帮助围绕程序的社区作为一个整体，因为 API 基础结构:

*   如果 API 是公开的，每个人都可以插入它并开始项目工作。
*   其他程序可以很容易地移动，数据保持不变。
*   它给底层服务增加了一个抽象层。
*   API 让开发人员更有效率(他们不必从头开始)

在网络上，这意味着你可以将功能外包给浏览器，并从其他服务中添加数据。浏览器为网络应用程序提供了一种标准的访问方式，例如照相机、音频和蓝牙。Slack 或 Twitter 等服务/应用也是如此。您可以使用实时 slack API 发送消息，而不需要 slack 接口，只需要几行代码。

## 原料药的种类

可以用特定的`parameters`打电话给`endpoints`询问具体信息。您基本上是从服务中请求大型数据库的子集。开发人员应该如何将自己的应用程序连接到 API 并没有规则，但是已经出现了一些标准。

首先要区分的是 API 是**公共的**还是**私有的**。

*   如果 API 是完全公开的，你可以请求数据而不需要任何额外的信息。
*   如果 API 是 *public* 的，你可以用服务提供的`public key`请求数据。通过在发出请求时添加密钥，您可以进行 API 调用。请注意，使用公钥，大多数 API 都会有限制。如果 API 是*私有的*，你将需要一个如上所述的密钥，但是这个密钥是不公开的。服务人员需要亲自为您提供一把钥匙。

## 认证

如果您想要操作来自特定用户的数据(用户必须登录),大多数 API 使用 OAuth open 协议。

> 一种开放协议，允许以简单标准的方式从 web、移动和桌面应用程序进行安全授权

它允许您与用户的特定数据中的受保护数据进行交互。比如说；无需使用 Spotify 界面，即可将播放列表添加到特定的 Spotify 帐户。

## 休息 API

网络上的大多数界面都使用`HTTP protocol`进行连接。您可以通过 HTTP 协议使用可在 web 上寻址的 API 端点来请求数据。(GET，PUT，POST & DELETE)你可以向一个特定的端点发出请求，因为 HTTP 请求总是一个往返过程，所以你得到的响应可以在你自己的应用程序中使用。在网上，大多数时候这种回应是以`.json`的形式出现的。

## RTM API

一些服务采用了稍微不同的方法，它们不使用`HTTP protocol`而是使用基于 WebSocket 的 API，允许你实时接收事件。API 可以向您推送信息，而不是询问信息。

通过**轮询**，您可以使用 Res API 获得类似的结果。(只是非常频繁地调用端点)轮询类似于刷新按钮。它有效，但不是最佳解决方案。

## Web API

要明确；这篇文章的大部分内容都是关于从其他服务请求数据的 API。还有特定的 Web APIs。它们用于操作 DOM，可以使用 JavaScript 和对象类型来访问。Web APIs 扩展了 Web 浏览器的功能，因此是特定于浏览器的。

*   标准 JavaScript APIs。这些是您可以在标准 web 应用程序中使用的相同的核心 JavaScript 和 DOM APIs。
*   HTML 特性和其他浏览器特有的新兴 API。
