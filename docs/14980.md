# 如何与你的雇主进行冷战

> 原文:[https://dev . to/spirodonfl/how-to-pay-cold-your-on-your-employees-e9d](https://dev.to/spirodonfl/how-to-wage-cold-your-on-your-employers--e9d)

这是[“如何对你的开发人员发动冷战”](https://dev.to/spirodonfl/how-to-wage-cold-war-on-your-developers--17n0)的后续文章，在那里我提供了公司吓跑开发人员的轶事经验。我在这篇文章中重复使用这个术语，但是把它翻过来。

# [](#overpromising)夸下海口

我们都知道这个。雇主问“你认为这需要多长时间？”在你的内心深处，你在想“可能一个月，但是……如果我这么说，客户会跑掉，或者他们会认为我是个笑话”，所以你嘴上说“两个星期”。

我这样做过几次了。情况不妙。这对你自己和你的雇主都是一种伤害。

想想你同意什么。实质上，你同意在两周内每周工作 80 小时，而不是正常工作时间。如果你有家人或朋友，或者两者都有(你真幸运)，那么这会杀了你。

它也几乎总是以项目失败而告终，从而给你带来坏名声。

当然，这也意味着客户损失金钱和时间。此外，根据项目对他们的业务意味着什么，他们可能会赔钱。在某些情况下，人们会失业。

这有什么关系？它导致…

# [](#not-caring-enough)不够关心

我以前也有过这种心态:“这不是我的公司，所以不是我的问题。我想怎么编码就怎么编码。我会利用我需要的任何时间。我需要以我需要的方式完成我的工作。”

其中一部分是自我。部分原因是缺乏对他人的关心。公司变得有点人性化了。你可以谈论苹果或谷歌，就好像他们是一个人一样。那肯定有优点。

然而，公司是由人组成的。公司为多个家庭提供收入，在某些情况下，为整个经济体提供收入。技术可以成为所有这些的巨大组成部分。你可能会被要求为他们拿着那把钥匙，提供他们需要的支持。

如果*是*你的公司呢？你希望得到怎样的待遇？黄金法则，如果你愿意的话，有时可能是一条值得思考的重要法则。

和其他事情一样，也有例外。但是请考虑一下不在你正前方的方面。

# [](#not-respecting-time)不尊重时间

首先，你自己的时间是你拥有的最有价值的商品。

如果你打算打卡上班，打卡下班，拿一份薪水，跳过这一步。跳过整篇文章。

如果你在乎并充满激情，那么我建议你考虑一下你在这份工作上花费的时间。总的来说，鞭策自己、按时完成任务、努力工作并没有错。

我有一个家庭，其中包括一个新生儿，这是我们的第一个孩子，我意识到我花在他们身上的时间很少，更不用说这么多的工作了。

不要忽视你自己、你的家人或你的朋友。你自己的理智和快乐对你在工作中的成功至关重要，当你意识到这种关联时，可能已经太晚了。

此外，尊重你雇主的时间。我已经简单地提到了这一点，但是请记住，你的雇主支付你的是工作，而不是猫视频。休息是你的权利。但是尊重他们的时间也是尊重你的时间。如果你追踪一周内，你花在工作上的每个小时和你应该工作的每个小时，你可能会感到惊讶。

作为一名自由职业者，这实际上对你更重要，原因显而易见。当你在工作的时候，你的生产力是非常重要的。记住这一点。

# [](#not-respecting-requirements)不尊重要求

同样，自我可能会在这里发挥作用。我有时反对满足某些需求，不一定是因为这些需求不好，而是因为我不同意达到这些需求的方法。

公司和客户通常会在业务需求上花很多心思。这对他们来说很重要。不是经常为了好玩或者只是尝试什么或者烧钱。

对你不同意的需求进行第三次或第四次回顾，并问自己为什么不同意它们。如果你有充分的理由反对，那么你的论点(通常)对雇主来说会很清楚，他们也会同意你的观点。

# [](#skirting-the-mundane)超脱世俗

唉，我不得不从代码库中取出所有这些查询，并把它们移到这里，包装在一个函数中？那太无聊了。大三学生也能做到。

我以前也这么想过。如果你曾经有过不想做的无聊事情，在这里插入一个类似的抱怨。

当你是一个中级或高级员工时，很容易说“我已经把我的猴子时间放进去了”。

也许那是真的。

我会这么说。有很多时候我不想做简单的事情。我这样做了，并最终发现了一些问题，否则我永远也不会发现这些问题。或者，当其他事情突然出现，我确切地知道该去哪里时，这些无聊的知识就派上了用场。

不要小看无聊。

# [](#carrying-a-large-ego)带着很大的自负

我真的不会写太多，因为这是显而易见的。

我曾经和一些开发人员一起工作，他们告诉我(当我还是个大三学生的时候)，他们懒得花时间在琐碎的事情上。我让一个人直接说“我对这个太重要了”然后离开。

检查你自己。你不是上帝给编程世界的礼物。你不是史蒂夫·乔布斯或沃兹尼亚克或任何你认为是技术大师的超级英雄。你就是你，你必须和别人一起工作。这包括那些可能对你有所要求和期待的客户，他们可能会觉得你“高高在上”。

我也不得不学这个。这是可以克服的。

# [](#conclusion)结论

这里似乎有一个总的主题:尊重自己和他人，不要自负。

这说起来容易，但是现在，当你和一个初级开发人员或者一个客户的女主角一起看一个技术问题时，很容易忘记。

我也建议你花些时间在朋友和家人身上，如果你还没有的话。即使是开发伙伴中的朋友也比没有好。说真的。即使在最小的层面上，你的精神状态也会随着社交互动而改善。如果你愿意，请联系我。
