# 作为初级开发人员，你会如何辞职？

> 原文：<https://dev.to/yokim/how-would-you-go-about-resigning-as-a-junior-developer--gnl>

## 前言

所以这不一定是一个我迫切需要帮助的话题，我只是认为这将是一个有趣的讨论！

### 场景

你是一名初级开发人员，你去参加了一些面试，你申请的一家公司刚刚给了你一份比你目前得到的任何东西都更有价值的工作。你接受。

你最初的兴奋和喜悦很快就被不得不把这个消息告诉你的老板，以及不得不继续工作 4 周左右的想法所压倒，因为这是你的合同中规定的。

作为开发人员，你将如何从书面和口头的角度处理这种情况？

### 我的想法

我认为直截了当、开门见山会减轻我在递交辞呈时可能有的任何犹豫，但让我踌躇不前的一件事是，我不得不公开、诚实地告诉你的老板你为什么要离开，告诉他们你为什么如此不开心，以至于要找另一份工作等等。

一想到必须向老板敞开心扉倾听，我就觉得这可能会伤害我们的关系，而且可能会在将来反咬我一口，例如:

如果你接受了新工作场所的邀请，他们联系了我的老板，想了解我是一名什么样的员工，我担心要么是前任老板告诉我的即将成为雇主的品质是不真实的，要么是纯粹的诽谤。

* * *

这些只是一小部分想法。

你的呢？
