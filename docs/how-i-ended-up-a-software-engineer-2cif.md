# 我如何成为一名软件工程师

> 原文：<https://dev.to/omawhite/how-i-ended-up-a-software-engineer-2cif>

**注意:我之前在我的旧博客上写了这篇文章，现在正准备把它移到我的新博客上，并在 dev.to** 上建立交叉帖子

前几天，我在清理收件箱时，看到一条来自[代码学院](https://www.codecademy.com/)的消息。我很快删除了这条消息，并取消了他们的电子邮件更新，因为我已经很久没有使用这个平台了。在这样做的过程中，我意识到我已经走了很长一段路。不管你如何衡量，从我高中的最后一年开始，或者仅仅是在过去的一年里，我已经朝着我不断前进的目标迈出了令人难以置信的步伐，那就是成为一名伟大的工程师。

很久以前，当我还是一名高中生，试图决定在大学里学什么时，学习编程对我来说似乎是一项不可能完成的任务。我对技术感兴趣，甚至涉足过一些 HTML 和 javascript 的复制和粘贴，但还没有真正学会任何重要的编程。我试着自己学习，浏览教程或者使用代码学院之类的东西，但是没有什么能真正记住。由于无法自学，我想参加一个课程可能会有所帮助，所以我报名参加了学校的计算机编程班。一开始很棒，我们开始用 [scratch](https://scratch.mit.edu/) 学习，这是一种简单的基于视觉的编程语言。它非常容易上手，让我可以轻松地创建简单的游戏和其他有趣的东西。因此，当我们转向 Visual Basic 并最终转向 Java 时，我更有信心了。这种信心很快就动摇了，因为我发现一条更难的学习曲线在等着我。

在课程的这一部分，我们被给予阅读的书籍，并期望在没有太多指导的情况下一章一章地完成练习，这对我来说是非常困难的。我没有从阅读这些书中获得太多的价值，也没有讲座，因为老师自己没有任何编程知识。他只是在我的学校里教授一些商业选修课的老师，不知何故编程属于他的管辖范围。所以在这学期剩下的时间里，我一直在与 Java 和 VB 作斗争，我完全相信编程不是我的职业。我仍然很想学习技术，所以我决定去印第安纳大学学习信息学，希望最终成为一名顾问或商业分析师。

幸运的是，我最终再次尝试了编程和计算机科学，这一次是在大学二年级，周围的学习环境要好得多。在和一个朋友约定辅修计算机科学，尽我们所能成为最好的工程师之后，我下定决心要战胜这个曾经对我来说似乎不可能的挑战。这的确很困难，但这次也不一样。我可以带着问题去找教授，听他们讲课，做适当的笔记，我可以和周围的其他学生一起工作，向他们学习。事实证明，这种学习环境正是我最终取得突破所需要的。慢慢地，但肯定地，计算机科学的基本概念开始吸引我，一旦发生，我就无法满足。

随着我在 Target 开始软件工程师的职业生涯，这种学习一直在继续。在我到这里的前 6 个月，我有幸与我在云平台工程团队中遇到的最有才华的工程师团队一起工作。当我开始的时候，我几乎不明白码头是什么，为什么集装箱化是有价值的。我将带着对 [Spinnaker 项目](https://www.spinnaker.io/)的一些开源贡献离开，当我下周进入我的新团队，专注于前端工程时，我确信我的 Javascript 技能/知识也会显著增加。

我写这些是为了尝试理解以下观点。如果你正试图学习编程或沉浸在技术世界中，但事情并没有停滞不前，不要气馁。每个人都是不同的，虽然不是每个人最终都会成为一名软件工程师，但在技术世界里，各种各样的人都有一席之地。棘手的部分是弄清楚我们适合哪里，我们如何学习，什么样的事情会激励我们。所以，如果你现在正为此而挣扎，不要担心！找一个导师或朋友一起尝试学习，不断尝试不同的学习方法，直到有所收获。对于我们中那些已经过了最初学习阶段的人，向那些还没有过的人伸出援手。鼓励刚起步的人，给他们一些指导，给他们一些帮助你的资源，回答他们的问题。这样我们就可以继续发展这个行业，并确保它充满新鲜血液、新观点和不同的人群。
