# 黑客重构现实指南

## 标题是怎么回事？

我不是向导，这篇文章本身也不是。相反，我希望创建一个讨论，在那里我们可以作为人和程序员引导彼此达到更大的正念。作为程序员，我们用隐喻说话，因为这能让我们更好地理解难以直接描述的事物。说到底都是开关和电子，所以谈论物体更容易。

正念是非常相似的，因为你所描述的是如此与生俱来和基本的，以至于我们几乎经常忽略它。称之为正念和慈悲也就到此为止了，如果我们要走这条路，我们必须更好地描述它。这里是我尝试使用编程隐喻来塑造我的理解，我希望它能帮助你重构你的理解。

## 为什么要正念

我们的生活充满了精彩的时刻。例如，当我写这篇文章时，我正坐在晨光中，我的狗在我身边，这是我所珍视的东西。但是当我坐下来欣赏这一刻的时候——我哼着一首[歌的第一小节](https://www.youtube.com/watch?v=Uy5T6s25XK4),所以我也在我对这首歌的想象创造的另一个世界里。所以，在我心中的现实中，我既是这个世界上的一个存在，也是我自己创造的一个泡泡中的一个存在。

这种二分法不是要解决的问题，因为它是一种自然的生活方式。然而，简单地将注意力转移到我们所占据的现实的泡沫上是有帮助的。如果你对脚下的土地一无所知，你可能会失足。因此，为了更接近我们的人生道路，我们可以练习留意自己和周围的环境。

## 正念

在任何情况下，当我们开始训练对我们此时此刻的状态以及我们周围事物的认识时，正念就实现了。虽然通向这种瞬间觉知的步骤就像经历它们的时刻和事物一样不可计数，但道路的起点总是近在咫尺。

找到第一步，以及随后的步骤，与软件指令的开发没有什么不同。我们实验，我们出错，我们修复它，我们运输它并拥有它。这可能看起来很肤浅，但是在我们的头脑中，这些相似之处可以引导我们使用工具来塑造我们自己，就像我们塑造我们的代码一样。

我们已经是一个伟大的软件开发团队了，那么我们如何应用这些良好的实践来开发对外部编辑和终端有帮助的技能呢？如果软件开发技能可以在其他环境中重用，对于有头脑的程序员来说，生活会更加枯燥。

## 例子:

这次讨论的目的是实验软件开发技巧的应用，以提高我们在现实生活中的正念。这些上下文混搭的标语可能是“应用硬技能来提高软技能”。一些例子:

1.  应用调试原则来解决关于差异的分歧。
2.  应用测试驱动开发来改善与团队成员的交流。
3.  应用领域建模为重要的接收者写一封简明的邮件。

### 🐛排除分歧

我从我的经历中举一个例子，其余的让大家思考。作为开发人员，我们遵循一个过程来确定问题的根本原因，从它在系统中的外围影响开始。当面对一个个人问题时，我们能应用同样的过程从问题到解决方案吗？

这种个人问题的一个例子可能是对代码更改的棘手分歧。您以特定的方式编写了解决方案，而另一位开发人员留下了反馈，要求您以不同的方式重写代码。在这种情况下，理想的做法可能是继续持有不同的观点，但在实际意义上放弃自己的信念，作为一个团队致力于一个单一的行动过程。

就像我们寻找最接近问题根源的 bug 修复一样，当你对你依恋的根源有所了解时，放手是最容易的。而且，就像错误的影响并不总是立即表明其原因一样，你继续分歧的直接理由可能比立即明显的更有深度和细微差别。在这样做的过程中，我发现我对这些事情的感觉没有我通常认为的那么清晰，实际上有很大的空间来达成一致！

要找到你依恋的根源，首先从你的角度列举一个情境的要素，并确定你对它们的感觉。它们可能很大，如产品的方向，也可能很微妙，如围绕该问题的沟通的语气和方式。选择正确的东西只是那些想到的东西。如果你发现什么值得注意的东西，记下来！当你感觉完成时，再尝试想象一个协议。你找到了吗？
