## [](#azure-databricks)[蔚蓝数据报](https://docs.microsoft.com/en-us/azure/azure-databricks/what-is-azure-databricks)

[![DataBricks](../Images/bdd8551a3a866757e66a41fc4501595a.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--r6AZHxO0--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/bjbvwqfghxf7dmwt5wl6.png)

我在去年的 Microsoft Connect 上听说过 Azure DataBricks，但一直没有机会谈论它。这个星期我又遇到了它，所以我想也许我有机会在这里谈论它！

这是他们文件的摘要:

> Azure Databricks 是一个基于 Apache Spark 的分析平台，针对微软 Azure 云服务平台进行了优化。Databricks 由 Apache Spark 的创始人设计，与 Azure 集成，提供一键式设置、简化的工作流程和交互式工作区，支持数据科学家、数据工程师和业务分析师之间的协作。

[Apache Spark](https://spark.apache.org/) 是一个高性能&分布式数据分析平台，包括用于流传输、机器学习等的工具。DataBricks 是 Spark 的云托管平台，由 Spark 背后的同一批人构建。这是在云中做 Spark 工作的首选平台。有了 Azure DataBricks，你可以访问 Azure 提供的一切。这使得使用 Spark 更快更容易。从 Azure 获取数据，将其传输到 Azure DataBricks 并在其中进行处理，然后将结果发送到 Azure 应用程序或可视化。

如果你从事数据科学，这绝对是值得尝试的。😃

## [](#react-preload%C2%A0link)[反应预载链接](https://www.npmjs.com/package/react-preload-link)

> React 路由器链接组件的超级版本，在导航前预加载您的数据。与 React 一起构建并用于 React。

就像上面说的一样。使用 React Router 的常规链接组件，即使您需要的数据尚不存在，您也会立即被重定向到该页面。React 预加载链接包装 React 路由器链接组件，并在导航到页面之前首先加载数据。查看这里的演示:[https://sandervspl.github.io/react-preload-link/#/](https://sandervspl.github.io/react-preload-link/#/)。

## [](#docker-squash)[码头工人壁球](https://github.com/jwilder/docker-squash)

> 挤压 docker 图像使其变小

像`COPY`和`RUN`这样的命令会在 Docker 图像中创建额外的层，每一层都会使图像变大。Docker Squash，嗯，把这些层挤压成一层，是为了让图像变小。
