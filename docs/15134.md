# 如何开始一个开放源码软件项目

> 原文:[https://dev.to/hardkoded/how-to-start-an-oss-project-4 纳尔](https://dev.to/hardkoded/how-to-start-an-oss-project--4nal)

## [](#introduction)简介

OSS 项目和你日常工作中的项目有很大的不同。起初，这就像在一个童话故事中，但是，如果没有纪律，你的梦想项目可以很快成为 *<在这里插入一个无聊的电影参考>* 。

我在 2017 年 10 月开始了[木偶师夏普](https://github.com/kblok/puppeteer-sharp)项目，虽然我还在学习如何做这个我们称之为 OSS 的东西，但这里是我迄今为止学到的东西。

## [](#open-up-to-the-community)对社区开放

有时候，我们的自我是我们最大的敌人。我们不想显得愚蠢。我们不想展示半途而废的东西，“人们会怎么想？”

> "与社区分享你的想法是 OSS 项目成功的第一步."

当我在一次测试运行之前就在 GitHub 上获得了第一批“星星”时，我感到很惊讶！受到鼓舞，我想“有些人对此感兴趣。我们来编码吧！”。这种接触有助于你继续前进。如果你把你的项目保密，没有人知道，他们不会知道你是否让它不完整。但是，如果你向公众开放，你可能会得到“嘿！我需要这个工具，你打算什么时候完成？”你会被推动着前进。

## [](#dont-buy-fake-progress)不买假进步

发现自己买的是假进步的时候我都不敢相信。"我整个职业生涯都在做项目！"，我想，这怎么可能呢？

问题是很容易陷入假进度。企业项目倾向于从 CRUD 页面开始；各国 CRUD 文件污物等。分散我们对难以解决的实际问题的注意力，但对客户(和我们自己)来说似乎是进步的事情。

同样的事情也发生在我身上。我从小类开始，比如键盘或者鼠标。感觉很好。我在按代码！但是一个月后，我发现我实现了 50 多个类，但是没有一个绿色测试。很逊吧？

所以，我改变了方向，把所有的精力都放在了让我的第一次测试绿色上。

> “你可能有一千个类和一百万行代码，但是如果你没有一个绿色测试，你就不能出货。”

先做重要的事情。不要买假进步。

## [](#clear-roadmap)清晰的路线图

路线图对于企业项目是强制性的。无论你使用路线图、项目计划、里程碑还是其他什么，设定期望值是很重要的。但是当你开始一个 OSS 项目时，你做它(通常)是为了好玩。你有一个很酷的想法，并且想要实现它。你最不想做的就是项目管理！！

如果你想花几天时间写代码，找点乐子，那就去吧，你不需要计划。但是，如果你想完成一个 OSS 项目，你需要一个路线图。

> 如果你想完成一个 OSS 项目，你需要一个路线图

对不起，我知道这很无聊，但你需要花一些时间来构建路线图，而不是编写代码。至少以我的经验来看，好的一面是你不一定需要设定日期。虽然人们想知道他们什么时候可以使用某些功能，但是他们也知道你是在空闲时间使用这些功能。项目追随者不应该强迫你在一定时间内实现一个特性(除非你的项目达到了 Newtonsoft JSON 或 ReactJS 的受欢迎程度)。

当我意识到我需要一个木偶师夏普的路线图时，我把我知道的库应该有的每一个特性都放到了不同的版本中。我没有提到我将在什么时候实现这些特性，而是说，我将如何对它们进行优先级排序。这不仅帮助了对项目感兴趣的人，也帮助我按顺序编码。

## 你的考试是你的灯塔

单元测试是 OSS 项目中的基本工具。它帮助您度量代码质量并解释您的代码是如何工作的。但这也是一个很好的方式来看到(和展示)你的进步。

当我发现自己买的是假进步的时候，那是因为我没有任何测试。不要评判我。在 Puppeteer Sharp 中运行一个测试需要很多行代码。但是，我一次测试都没有。我迷路了。我没有灯塔。

警告
如果你有数百个测试，但没有一个测试一个重要的特性，你可能会买到虚假的进展。

我不是说小单元测试不重要。但是，让测试显示出您在路线图中包含的特性是很重要的。只有当你对特性进行绿色测试时，你才取得了进步。

## 我知道我会完成这个项目

我怎么知道我会完成这个项目？因为我承诺每天至少给这个项目做一个[番茄](https://en.wikipedia.org/wiki/Pomodoro_Technique)。约翰·瑞西格在这个话题上有一篇[很棒的帖子](https://johnresig.com/blog/write-code-every-day/)。

> 但是，一天 25 分钟够吗？

当然不会比一天花 8 个小时，或者一个周末花 20 个小时好。但是，这种稳定、有序的进展将帮助我在不忽视我的日常工作、家庭和健康的情况下实施这个项目。

不要停止编码！
