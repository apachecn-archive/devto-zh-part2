# 开发者更应该关注什么:结果还是过程？

> 原文:[https://dev . to/essential developer/whats-more-essential-for-developer-focus-on-result-or-process-4245](https://dev.to/essentialdeveloper/whats-more-essential-for-developers-to-focus-on-result-or-process-4245)

在这个视频中，我想问你一个重要的问题，但在此之前，让我告诉你我们认为软件开发人员可以掌握的最佳技能之一:**长期愿景**。

这听起来可能很明显，但根据我们的经验，大多数开发人员在无休止地追求*银弹*和*魔法解决方案*时，关注短期收益，而最快的途径通常是耐心。

为什么长期如此重要？我们生活/事业中的大部分结果都是不同步的(好的结果需要时间)，甚至超出了我们的控制。我们可以想象结果是什么(长期愿景)，而不是试图控制结果，但是**关注实现目标的过程**。更好的是，要灵活，欢迎沿途的变化！

例如， **[测试驱动开发](https://www.essentialdeveloper.com/professional-ios-engineering-season-1)** 是一个可以让你控制开发过程的规程。为了让 TDD 工作，它需要一致性。 **[一致性是关键](https://www.essentialdeveloper.com/articles/the-importance-of-discipline-for-ios-programmers)。**一致性是流程的重点。TDD 只是一个例子，你可以选择自己喜欢的流程。但是要遵循流程。每天都是。

现在，我想问你一个重要的问题:现在，你更关注结果还是过程？
