## [](#what-is-dep)什么是 DEP？

因为我们在 python 社区有 PEP(Python 增强提案)，所以从开发社区得到一些 dep(开发增强提案)可能是个好主意。

## 这个 DEP 会是什么样子？

简而言之，这仅仅是:

*   目前的
    *   我喜欢的。
    *   我不喜欢的是。
*   将来的
    *   我希望看到的。
    *   我不希望看到的。

## [](#what-i-like)我喜欢什么

*   发展自然

我将 dev 视为“开发者的脸书”，你与开发者交流，与开发者分享想法，与他们开玩笑，最重要的是**了解开发**的趋势。

*   超级快

Dev 带来**出色的用户体验**在搜索/浏览时，今天我在搜索无服务器架构的可测试性，所以我写了“tdd 无服务器”，瞧，它在**非常精确的第二秒**就带来了结果！

## [](#what-i-dont-like)我不喜欢什么

*   通知链接

当你打开某人评论的通知时，你点击它，该死！它打开了一个他评论的小窗口，你想检查/阅读他评论的上下文，你打开完整的主题，玩滚动游戏到达他的评论...咄！

为什么不在完整的主题窗口中打开他的评论并自动滚动到它，然后在他的评论上做一些黄色高亮显示？！

*   添加图片

上传以上= >复制链接= >在编辑器中粘贴链接=> meh！

为什么不复制图片(从剪贴板)并直接粘贴到编辑器上？

并为手机/平板电脑用户保留该按钮。

## [](#what-i-hope-to-see)我希望看到什么

*   表情符号

说真的，在书面文字是唯一媒介的地方，没有一个像话语中那样好用的表情符号(你只要写一个冒号，它们就都出现在你面前)，你怎么和别人互动。

*   减价实时预览

在实时预览中检查我写的东西，而不是从一个点击跳到另一个点击(降价/预览)来检查它看起来怎么样，这不是更好吗:|

*   更有条理/更简单的用户界面

主页面 UI 有点混乱，可以更简单

举例:你觉得左边和右边哪个更好？

## [](#what-i-hope-to-not-see)我希望没有看到的东西

*   原生 iOS/Android/无论什么应用

是的，我是认真的...甚至他们已经在回购中开始使用 iOS 应用[！](https://github.com/thepracticaldev/DEV-ios)

你启动的每一个回购，你编写的每一个软件，你想应用的每一个小创意，总会有一笔技术债务。

这是开发团队的负担(对于两个额外的回购:iOS 和 Android)，错误肯定会发生，他们必须解决它。

PWA 在很多方面都绰绰有余，而且更加令人愉快**尤其是更新的事情**...我不想看到一个错误的、需要通过应用商店更新的、对开发团队来说负担沉重的应用程序。

*   敌对环境

对帖子没有(不喜欢/愤怒的表情)反应...仅仅因为你不喜欢某人的想法，并不意味着你要对他们表现出愤怒的表情，忽略它，在不破坏别人幸福的情况下开心地生活。

*   本地化/国际化

我知道这是一个社交网络...但是我们不要忘记，更多的语言意味着将社区分散成不同语言使用者的更小部分:(

是的，“英语是编程的通用语言”，伟大的 Jeff(stack overflow 的联合创始人)在他的文章中如是说。

*   工作平台

这里的大多数人都在他们的个人资料中提到了 LinkedIn 账户，那么在 dev 中添加一个招聘平台(带有烦人的邮件/通知)有什么意义呢...鉴于 HR 可以查看自己的帖子/评论/github。

我们都看到了 stackoverflow 是如何被工作宠坏的。
