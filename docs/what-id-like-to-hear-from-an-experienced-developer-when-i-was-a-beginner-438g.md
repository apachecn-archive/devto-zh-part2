# 当我是一个初学者时，我希望从一个有经验的开发人员那里听到什么

> 原文：<https://dev.to/ivancrneto/what-id-like-to-hear-from-an-experienced-developer-when-i-was-a-beginner-438g>

当我们还是学生，开始第一份工作时，许多问题浮现在脑海中:

*   应该学什么编程语言？
*   我应该追求更科学的职业吗？
*   与我想从事的技术相关的工作类型有哪些？
*   在哪里工作？什么样的公司？
*   最近有什么变化？
*   和许多其他人...

这样的问题在当时是正常健康的。在生活中，我们不得不做出许多个人和职业的决定。在这篇文章中，我将展示我所做的决定，我对关键问题的看法，并分享一些发生在我身上的有趣案例。我开始时想听的事情。

### 编程语言

[![](../Images/d90c7b408b45d1e89761fddf8c6a9d05.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--c5_OBm2P--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/851eg8sa7mwu7ybv89ik.png)

这确实是被问得最多的问题之一。该不该学 Java？Python？c？露比？JavaScript？XYZ？我应该学习世界上所有的语言吗？

#### 语言重要吗？

简而言之:没有。

公平的回答:是和不是。他们重要的是了解他们的差异，范例，他们如何实现面向对象，设计模式，他们如何处理内存和操作系统。一段时间后，它们仅仅是你在一两天内学会如何使用的工具。你可能会坚持用一个，用另一个做一点，用另外两个玩得开心。最终，这只是一个为合适的工作选择合适的人的问题。

几年前，我和我的一个朋友学习了 Python 和 Java。他很快找到了工作，而我没有。我真心喜欢 Python，他为了找工作选择了 Java。我可以找到一份非常好的工作，并且乐在其中。不过，如果我辞职了，就很难再找到另一份工作了。他的工作没那么好，但他能很快找到另一份工作。有一个不可避免的权衡。

### 技术

关于技术，我想说的是差不多的，但是一旦你把你的知识缩小到某个领域，我们可以很容易地找到我们想要学习的每一个类别。它们本质上代表了以下技术:

1.  我们一起工作；
2.  与我们一起工作的人有关。
3.  补充一下；
4.  我们只是想和他们一起玩。
5.  我们找机会学习它们。

作为一名 web 开发人员，我用 Python 语言，Django 框架作为相关技术，MySQL，PostgreSQL，Redis，JavaScript 来补充我的工作。这是我的典型筹码。我在 Swift 工作过，因为我想学习 iOS 开发，而且我在 Pyramid 有一个很好的机会。

### 机遇

专业人士最重要的一点是其稀有性。技术在继续找工作和被公司争议之间的转变中起着关键作用。

身边有很多 Python 开发者。没有多少人有 OpenCV 和 OCR 相关技术的经验。在某家公司，有一个机会去做一个普通的项目，检测房间里肮脏和破损的墙壁。dev 选择继续工作，所以现在她仍然是这个团队的一员。

每当我有机会掌握新东西或做别人拒绝做的事情时，我都很想这样做。实现这一目标的后果往往非同寻常。一旦我了解到一个只有一个人维护的遗留系统，当他去度假时，我可以接管。一点一点地，我从他那里接过了“遗留系统维护”的角色，所以他最终被释放去做更重要的事情，而我也变得不可或缺。当我们不得不改造它的时候，我知道公司的大部分商业规则。

另一个有助于加快学习过程的主要因素是代码审查。即使你是一个 Jr 开发人员，也要继续复习一个 Sr 开发代码。只问关于实现、方法或业务规则的问题。接下来，在本地运行代码并进行修改。检查模块、类、函数返回。这随着你在任何地方的成长而大大改善。

### 市场

关注市场的动向也是增加获得更多机会和成为不可多得的专业人士的非常好的一步。公司和工作对你有要求，但市场也有要求。

如果我们从 2000 年到现在，有可能跟踪网络的发展，例如，许多与添加丰富的应用程序、音频和视频相关的工作的出现。我们看到了智能手机的出现，我们可以专注于移动应用程序的开发。我们也有一个前端繁荣，JavaScript 框架，聊天和家庭自动化，大数据，数据科学的许多可能性，最近，我们有进步的网络应用程序，加密货币和区块链。

#### 企业还是创业？

你应该两个都试试，看看哪一个最适合你。我先尝试创业，因为我可以有更多的自由，在不同的领域工作，我没有机会在大公司工作。我与客户交谈，做演示，收集需求，实施解决方案，部署，甚至在公司的财务部门工作。

#### 我应该考虑远程工作吗？

毫无疑问。我已经远程工作了七年，四年来一直是 Toptal 的一部分，我说找一份远程工作真的很简单。甚至非科技公司也开始允许远程员工，而且效果很好。有些雇佣远程人员一段时间，有些有远程人员和现场团队，有些只由远程人员组成。甚至是 CEO！

如果想从事远程职业，一定要按照[这个](https://www.toptal.com/freelance/how-to-work-remotely-and-still-be-the-best#book-tested-programmers)走。
