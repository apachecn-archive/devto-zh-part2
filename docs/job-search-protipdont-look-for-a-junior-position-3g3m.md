# 求职建议——不要找初级职位

> 原文：<https://dev.to/zellwk/job-search-protipdont-look-for-a-junior-position-3g3m>

[https://www.youtube.com/embed/Lr312DGmPWQ](https://www.youtube.com/embed/Lr312DGmPWQ)

今天，我想分享一个关于找工作的令人不安的真相。这可能会改变你对如何找工作的想法，尤其是如果你刚开始工作的话。

准备好了吗？开始了。

不要以获得初级职位为目标。以获得中级职位为目标。

如果你正在寻找一个前端开发人员的角色，不要找初级角色。找一个正常的前端开发工作。

这个建议听起来可能违反直觉。如果没有先获得足够的大三经验，如何成为前端开发者？

要回答这个问题，你找工作的时候要考虑自己想要什么，公司想要什么。

## 自己想要的 vs 什么公司什么。

当你寻找初级角色时，你想要的是:

1.  学点东西
2.  培养你的技能

也许 2-3 年后，当你学够了，你想找一份更好的工作。

这种想法会让你失去找工作的机会，因为公司不做慈善。他们不想雇佣你只是为了训练你。他们想雇你为他们工作。即使他们在寻找初级开发人员，这一点也不会改变。

看到你想要的和他们想要的区别了吗？

我不是说每个公司都不愿意雇佣和培训初级开发人员。我是说它们很稀有。

问题是，你想把找工作的事情交给运气吗？找到一家帮助你、训练你、培养你的公司？

或者说，你想决定自己的命运，你得到什么工作？

如果你想决定自己的命运。你不能指望找到一家好公司。它们很稀有。你需要等待别人来选择你。

为了能够在你想去的任何地方找到工作，你需要停止像初级开发人员那样思考，开始像专业人员那样思考。这意味着你需要训练自己。你不能等待任何人来训练你。

你需要学习基础知识。你需要学习 CSS。你需要学习 JavaScript。你需要有信心去建造你能想到的任何东西。你可能以前没有建造过它们，但是你有信心在需要的时候建造它们——无论何时有人让你建造什么东西，你都可以说“只要给我 X 天的时间，我就能完成”。

如果可以，问问你为什么建要建这个？为什么这个不是那个？这些问题是你有的，这就是你作为专业人士的态度。当你能做到这些，我认为你找工作不会有问题。

## 学习

你需要尽可能多的自学。有很多资源，有免费的也有付费的。两者你都可以学，真的没关系。

重要的是你学会如何学习，因为开发人员 90%的工作不是编写他们已经知道的东西。是为了解决问题。当你解决问题的时候，你会发现你不知道的事情。当你发现你不知道的东西时，你必须学习和实现新的东西。

所以不知道怎么学就麻烦了。即使你处于初级职位，也是如此——你仍然需要学习和实现一些东西。
