## [](#applications-of-machine-learning-in-cyber-security)机器学习在网络安全中的应用

机器学习(没有人类干预)可以收集、分析和处理数据。在[网络安全](https://dev.to/apium_hub/cybersecurity--crossing-the-last-frontier-2hn8)的情况下，这项技术有助于更好地分析之前的网络攻击，并制定各自的防御对策。这种方法实现了一个自动化的网络防御系统，只需要最少的熟练网络安全力量。

根据信息数据公司( [IDC](https://www.idc.com/getdoc.jsp?containerId=prUS41878616) )的数据，人工智能(AI)和机器学习将从 2016 年的 80 亿美元增长到 2020 年的 470 亿美元。正如[谷歌](https://www.blog.google/products/g-suite/keeping-your-company-data-safe-new-security-updates-gmail/)所分享的，Gmail 上 50-70%的邮件都是垃圾邮件。在机器学习算法的帮助下，谷歌正在以 99%的准确率阻止这种不想要的通信。[苹果](https://www.apple.com/privacy/)也在利用机器学习来保护用户的个人数据和隐私。在这里，我们涵盖了机器学习在网络安全中的应用。

## [](#5-cyber-security-threats-that-machine-learning-can-protect-against)机器学习可以防范的 5 种网络安全威胁

**鱼叉式网络钓鱼**

机器学习在网络安全中的应用之一就是对抗鱼叉攻击。传统的网络钓鱼检测技术缺乏可靠地捕捉所有恶意链接的速度和准确性，使用户处于危险之中。因此，该问题的解决方案在于预测性 URL 分类模型，该模型基于最新的机器学习算法，可以识别揭示恶意发件人电子邮件的模式。这些模型被训练来识别微观行为(关键特征)，如电子邮件标题、正文数据的子样本、标点符号模式等。因此，这些经过训练的模型可以用来检测电子邮件是否是恶意的。

网络安全提供商 [Barracuda Network](https://barracuda.com/) ，Inc .开发了一种基于机器学习的引擎，可以根据客户的业务性质和要求，在没有人工干预的情况下分析独特的通信模式。正如该公司所披露的，该引擎实时研究异常信号的通信模式，并向管理员通知鱼叉式网络钓鱼攻击的详细信息。

**水坑**

黑客将跟踪用户经常访问的网站，这些网站位于用户私人网络之外。这就是水坑的概念。考虑这个例子，假设有一个受欢迎的咖啡店，来自某个特定组织的许多人从那里点餐。因此，用户进入网络浏览器，输入咖啡店的网址，然后下订单。黑客所做的是，他们不访问一个组织的网络，而是访问咖啡店的网站。因此，他们会利用咖啡店网站的漏洞来获取用户的凭据。

机器学习算法可以通过分析网站的路径遍历来测试或确保 web 应用服务的安全标准。它可以检测用户在通过目标路径时是否被定向到恶意网站。机器学习路径遍历检测算法可以用来检测这些恶意域。机器学习还可以监控进出网站主机的罕见或异常的重定向模式。

网络防御服务提供商 Paladion 开发了一个专有的 RisqVU 平台，以有效应对水坑攻击。这是[人工智能](https://prescouter.com/how-artificial-intelligence-will-impact-your-industry/)和大数据分析的结合。水坑攻击需要同时分析来自代理、电子邮件流量和口袋的数据。RisqVU 是一个应用来自多个来源的分析的大数据分析平台，这种基于人工智能的研究有助于可视化攻击的单一视图。

**Webshell**

Webshell 是一段恶意加载到网站中的代码，目的是允许攻击者修改服务器的 web 根目录。这意味着获得了对系统数据库的完全访问权。如果是电子商务网站，攻击者可能会频繁访问数据库，以收集客户群的信用卡信息。

使用 web shell 的攻击者的目标通常是后端电子商务平台。电子商务平台的主要风险与在线支付相关，在线支付被认为是安全和保密的。就在线业务而言，系统管理员希望运行在支付处理器服务器上的重定向托管支付页面是安全的。但是，恶意加载的 web shells 可以通过不同的路径修改网站以根交易数据。这些数据通过攻击者自己的服务器传输，同时传递到托管的支付页面。这是一个并发流程。因此，当支付过程外包时，网站更安全是一个神话。事实上，一旦攻击者控制了系统并更改了流程，就可以利用该漏洞。

机器学习能有什么帮助？可以检测正常购物车行为的统计数据，并且可以训练机器学习模型来从恶意行为中识别正常行为。可以在受监控的独立系统上执行已识别的恶意文件，以便进一步训练模型。这些机器学习算法可用于先发制人地识别 web 外壳，并在它们利用系统之前将其从系统中隔离出来。

由于 webshell 攻击的性质也被称为无文件攻击，网络安全公司 [CrowdStrike](https://www.crowdstrike.com/) 开发了 Falcon 来检测无文件 webshell 攻击，这些攻击依赖于高级漏洞而不是传统的恶意软件。

**勒索软件**

勒索软件是赎金+软件的组合。它是指任何种类的软件，要求任何种类的赎金，以换取用户被绑架的文件的加密密钥。加密密钥基本上是解锁用户的锁定文件的密钥。锁定的文件可能是用户计算机依赖的多媒体文件、办公文件或系统文件。

有两种勒索软件:加密文件(将数据转换成秘密代码)的文件编码器和锁定屏幕，锁定计算机并阻止用户使用它，直到支付赎金。

如果可以训练数据集来正确分析勒索软件攻击的微观行为，神经网络和深度学习算法可以检测未知的勒索软件。基本上，这个训练过程需要一个更大的勒索文件集和一个更大的干净文件集。该算法的任务是为数据集中的每个文件找到一些关键特征。然后，可以将这些特征分类成子集，以便为获取的数据集训练模型。当勒索文件攻击系统时，可以根据训练好的模型检查该文件，并在它加密整个文件系统或锁定对计算机的访问之前采取必要的安全措施。

云数据保护公司 Acronis 开发了一种新的网络安全解决方案，目标是零攻击日。Acronis 使用机器学习研究来分析脚本。

**远程利用**

最后但同样重要的是，机器学习在网络安全中的应用是远程利用。远程利用，也称为远程攻击，是一种针对一台或一个计算机网络的恶意操作。通过机器或网络的易受攻击点，攻击者可以访问系统。远程攻击的目标是利用和窃取系统中的敏感数据，或者通过引入恶意软件来破坏目标计算机网络。远程利用可以通过多种方式发生:

*   拒绝服务攻击:这是一种通过向服务器发送虚假的客户端请求来使服务器对用户不可用的技术。它造成了巨大的使用高峰，使服务器冻结，并让它们忙于处理大量的未决请求。
*   DNS 中毒:DNS 服务器是将人类记忆深刻的域名如 facebook.com 翻译成相应的数字 IP 地址的系统。DNS 系统用于识别和验证互联网上的资源。毒害 DNS 服务器基本上意味着欺骗它们接受伪造的数据来源作为真实的，并且访问这些被毒害的 DNS 服务器的用户被重定向到不知不觉地将恶意软件或病毒下载到系统中的站点。
*   端口扫描:计算机端口用于发送和接收数据。端口扫描器可用于识别数据的漏洞，并通过利用这些漏洞来控制计算机。

机器学习算法可用于分析系统行为，并识别与典型网络行为不相关的异常情况。可以为多个数据集训练算法，以便它可以预先跟踪利用有效载荷。

Juniper Networks 正在使用机器学习来分析自动收集的海量数据。机器学习使自动化能够使用算法从数据中学习，并做出决定和预测。
