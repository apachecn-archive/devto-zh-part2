# 如何写好项目 ou 交接的技术文档？

> 原文：<https://dev.to/nsrivastava2/how-to-write-a-good-technical-document-for-project-ou-handover-20d0>

我一直想知道写技术文档的最好方法是什么。我在谷歌上浏览了几个，甚至向朋友要了一些样本，但是当我作为一个程序员阅读它们时，我觉得它们并不像我希望的那样有用。它们只是又长又无聊的文件。我试图创造一种新的形式。我需要同样的反馈。

这个文档很简单，它包含了应用程序的主要流程，以及受影响的主要数据库表和字段名。因为它是由我开发的，我可能会对它有偏见，所以我想知道你是否可以看看同样的东西并提供你的反馈。
