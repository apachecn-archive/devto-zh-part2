# 让你的兴趣成为你的动力

> 原文:[https://dev.to/lunaceee/let-your-interest-be-your-driver-5h3h](https://dev.to/lunaceee/let-your-interest-be-your-driver--5h3h)

感觉就像几年前，我分享了我如何以软件工程师的身份加入旧金山一家初创公司的故事。现在，虽然我在 Netlify 仅仅工作了几个月，但我确信我在正确的公司选择了正确的职业。

我最近从事的最有趣的项目之一是重塑我们的对外电子邮件生态系统。有几个因素(嘘！即将发布公告)激励我们重新设计所有的电子邮件模板，以便它们与应用程序的工作方式保持一致。我们还想建立一个新的邮件体系结构，将三个令人敬畏的开源项目[雨果](https://gohugo.io/)、 [Heml](https://heml.io/) 和 [Netlify CMS](https://www.netlifycms.org/) 结合起来，创建三层抽象。我们使用 Heml 来生成多客户端兼容的电子邮件模板，而不需要通过嵌套表。Hugo partials 可以方便地允许不同的模板共享可重用的代码，例如电子邮件的页眉和页脚。最后但同样重要的是，与 Netlify CMS 的集成有助于提取代码，并允许内容编辑人员使用现有模板创建新的电子邮件内容。

作为项目的所有者，我接触过技术和非技术方面的挑战，包括项目范围、跨部门沟通、静态站点生成器工作流、模板语言……还有更多，但你已经明白了。

但是每一次挑战，都有足够的动力让我坚持下去。一个具体的例子是接受与 Netlify 的开源 CMS 项目合作的挑战。作为一名营销人员出身的软件工程师，我的梦想是制造有利于讲故事的技术。内容编辑人员无需接触一行代码即可创建和编辑电子邮件模板，并实时预览更改，这为他们提供了更多空间来帮助向用户讲述精彩的故事。

我如此积极地致力于 Netlify CMS 集成的一个原因是因为[文档](https://www.netlifycms.org/docs/)写得非常好，使得这个项目非常容易理解。旁注:向项目的文档团队大声欢呼，他们实际上是由令人敬畏的[@ very through](https://dev.to/verythorough)领导的一个女人乐队。

我要成为一名资深程序员还有很长的路要走。我敬佩所有留在科技行业并帮助其他女性进入该领域的出色女性。对于其他刚刚开始学习代码的女性和非二进制人士，我建议找出你喜欢和擅长的东西，设定自己的目标，并坚持练习。此外，不要犹豫向你周围的社区伸出援手，因为这让我们共同成长。
