## 那么什么是 *MVP* ？

MVP 代表*最小可行产品*。这是一个带有少量功能的应用程序，你可以用它的*功能*来消费。

MVP 帮助新开发人员用有意义的项目填充他们的投资组合。这将有助于他们从其他新开发者中脱颖而出。这是指代码是否干净，结构是否良好，项目是否有效。

你总是可以*在 MVP 上迭代*,而不是试图完善一个项目:没有一个软件项目是完美的。你还不如*号船*。

> *真正的艺术家之船。*—史蒂夫·乔布斯

## 挑选一个 MVP，开源它

开源并不是软件工程师狂热追求的时尚。它是当今软件开发的生命线。

使用像 [GitHub](https://github.com) 、 [GitLab](https://about.gitlab.com/) 或[bit bucket](https://bitbucket.org/)<sup id="fnref1">[1](#fn1)</sup>这样的开源平台，将向未来的雇主展示你正在做的工作。它还显示了你正在做的事情做得有多好，它还显示了你的成长和学习过程。

## 开源却不去追星

很多时候，包括我自己在内的新开发人员都想在 GitHub 上一炮而红，在一个项目上获得成千上万的明星，并在[趋势库列表中列出](https://github.com/trending)。

停下来。这绝对是个陷阱。

[![](../Images/0e4b8982d780be86cb6a8616ae767e6f.png)T2】](https://i.giphy.com/media/RKiOMSy7LLcA0/giphy.gif)

在 GitHub 上有一个流行的回购很少见；GitHub 上有 3.5 亿以上的用户，甚至更多的库。机会对你不利，至少一开始是这样。

当你开始的时候，尝试开源，培养你的技能，是你能做的最好的事情。一旦你对开源有了感觉，你就可以构建一些更有意义的东西。如果开发者社区觉得有趣，明星们就会来。

## 经历完美主义

我想让我现在的应用程序 Check Yo Self 成为一个漂亮的全栈环境来检查你的降价博客帖子。它经历了多次迭代，但我停止了建设。每一个闪亮的新东西，我想加入它，使它变得完美，淹没了我；它是残废的。

我做这个应用已经一年半了，现在快两年了。考虑到它的简单前提，它应该结束了。范围蠕变，感觉它需要比类似的应用程序看起来更好让我瘫痪。我无法在它上面工作。它扼杀了我的生产力，我停止了几个月的工作。

在看了 Dan 的演讲和他的推文后，我决定我应该建立一些功能。然后，在它们建立之后，我可以迭代它。

我还在开发一个 CLI 工具。我已经为此努力了 8 个月，也因为同样的原因花了这么长时间。

我去掉了所有多余的选项，并在过去的几天里一直在努力。

考虑到所有的小型 npm 工具，我可以制作一个带有一些有用的小功能的工具。就是这样。

这大大减轻了我在过去一年中感受到的压力。现在我知道我可以运送一些东西，而不会有[被抛弃](https://dev.to/avalander/my-github-graveyard-172b) [被回收](https://dev.to/domcorvasce/my-github-graveyard-1hdo)的重量在我的 GitHub 个人资料上乱丢。

## 结论

即使有类似的工具，你也可以构建你想要的任何东西。你有独特的视角，你可以把这种视角带到别人已经完成的项目中。你会用不同的方式建造它。

在 [GitHub](https://github.com/) 、[上建立一个回购](https://help.github.com/articles/create-a-repo/)，为它建立一个小文档，并在 Twitter、LinkedIn、Hacker News 等任何地方分享它。

然后，你就有了一些值得骄傲的东西来添加到你的投资组合和简历中。你总是可以在你已经拥有的基础上再接再厉。
