# 做开发者是一种怎样的体验？

新手或高级开发者提出的做什么、学什么或用什么的问题，往往是带着单数人的主观性来回答的。这个人可能是研究分布式神经网络的计算机科学博士，或者是自学的 JavaScript 开发人员。不难理解，如果不帮助初学者寻求建议，他们的答案会非常不同。

这个帖子的标题直接灵感来源于[“做一只蝙蝠是什么感觉？”](https://en.wikipedia.org/wiki/What_Is_it_Like_to_Be_a_Bat%3F):这是美国哲学家[托马斯·内格尔](https://en.wikipedia.org/wiki/Thomas_Nagel)的一篇论文，首次发表在 1974 年 10 月的《哲学评论》上。

没有人的经历可以和另一个人的经历相比。这是冒名顶替综合症的根本原因。每个开发者都以自己的方式走向他想要成为的人，如果有些路径是相同的，如果我们考虑到[品质](https://en.wikipedia.org/wiki/Qualia)的话，我们就不能得出相似的体验。

感受性被定义为主观的、有意识的经验的个体实例。

这意味着，如果你得到一个建议，你必须把这个建议转化成你自己固有的经验，然后才能建立某种经验。

# 建议的例子

## 努力

对我来说，努力工作意味着大量阅读和练习。我需要用我能操作的具体例子来面对我从理论中理解的东西。

这对另一个人来说不一定是一样的。

## 明白自己在做什么

你怎么知道我的理解和你的理解是一样的？我可以转述并试着解释我所理解的。但是永远也不能保证我理解了你试图向我解释的东西。

另一方面，一些天才能够做他们理论上不理解的事情。这不一定是个问题。

我们可以总结为，你可以试着在其他类似的环境中复制你所做的，这可能足以验证这个建议。

## 避免教条

我需要明白我在做什么，这给我带来了信心。否则，我会很快变得有压力，并质疑我的整个生活和我所有的选择。这就是为什么我试图解释一切来寻找教条。

但是我相信并不是每个人都这样。其他人可以相信某件事而活得长久而快乐，而不必面对理性、经验或理论。而且能做某件事不一定是问题(如果要教人的话可能是问题)。

# 做开发者是什么感觉？

要成为一名优秀的开发人员，你应该在做开发人员该做的事情时寻求积极的感受。最终，这是唯一重要的事情。

请忘记比较和竞争。通过倾听他们的问题来帮助你的随从，毕竟在我们的职业中经常起作用的是[橡皮鸭技术](https://en.wikipedia.org/wiki/Rubber_duck_debugging)。
