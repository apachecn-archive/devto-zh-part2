# C#中的元组是什么？

> 原文：<https://dev.to/flippedcoding/what-is-a-tuple-in-c-18ln>

大多数 web 开发人员并不了解这一点，因为元组在 web 开发中并不常用。这不是因为它们没用，而是因为它们肯定有用。元组只是被埋没在所有其他好东西下面的许多好东西之一。所以我们把它挖出来润色一下。

首先，元组是一种数据结构，它包含许多不同数据类型的元素。要记住的一件重要事情是，元组是引用类型，而不是值类型。这意味着它不会将数据存储在自己的内存空间中。它们存储数据在内存中的位置。类是引用类型如何使用内存的一个很好的例子。

不管怎样，元组有几种用法。

*   当你想暂时保存一些值而不创建一个单独的类时。它还可以用来保存数据库记录。

*   您可以从一个方法返回多个值，而不需要 *out* 或 *ref* 参数。

*   您也可以将多个值传递给只有一个参数的方法。

现在你已经挖掘出了什么是元组，是时候开始打磨它了。使元组与众不同的一点是它们是不可变的。这意味着一旦它们被创建，你就不能改变它们。该元组中的任何属性都将是只读的。这也意味着元组的大小在创建后是固定的。

有一些好消息！在 C# 7 中，元组已经更新并变得更好。您不必担心您的元组有多少个值(最初，一个元组中只能有八个值)。您还可以命名元组属性，而不是使用默认的超级通用属性名。最棒的是，你所要做的就是安装系统。ValueTuple NuGet 包。
