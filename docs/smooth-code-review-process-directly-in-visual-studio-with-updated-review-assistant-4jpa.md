# 使用更新的审阅助手直接在 Visual Studio 中平滑代码审阅过程

> 原文：<https://dev.to/devartsoftware/smooth-code-review-process-directly-in-visual-studio-with-updated-review-assistant-4jpa>

Devart 是一家公认的为开发人员和数据库管理员提供专业数据库工具的供应商，它推出了 Review Assistant 的新版本，这是一个方便而强大的 Visual Studio 代码审查插件，支持 TFS、Subversion、Git、Mercurial 和 Perforce。主要的变化与代码评审工作流有关，它有一个新的完善的界面，新的评审状态和评论，一个单一的编辑器来查看所有的变化，改进的电子邮件通知，等等。

以下是更新后的审阅助手中实现的主要增强功能的独家新闻:

*   促进代码协作的新审阅状态新版本的审阅助手现在包括“需要响应”状态。这种状态通知代码作者，有一个问题他们应该回答，以保持代码审查过程的流动。此外，被拒绝的状态现在被称为需要返工，这是对代码状态更精确的描述。

-自动审核状态消除过多工作
通过新的更新，审核者现在只需点击完成审核即可自动获得审核状态，无需手动定义。自动状态基于指定注释的类型。

-扩展了评审注释的类别以无缝协作
除了简单的注释，用户现在可以添加带有不同标签的注释:
-“问题”以通知代码作者需要响应
-“应该修复”以指示不严重的代码缺陷
-“必须修复”以显示代码错误

-单个编辑器将所有代码更改收集在一个屋檐下
现在，单个 diff-tool 窗口显示对不同文件所做的所有更改，而无需打开单独的文档。

-整理电子邮件通知
在此版本中，电子邮件通知获得了更好的外观和可读性。此外，所有审查通知都被简化为消息链。

除了上面提到的升级，新的评审助手现在提供:
-代码评审委员会的改进界面和功能
-更方便地将评审导出到 XLS 文件
-改进客户端和服务器版本之间的不兼容性验证
-支持 MS SQL 2017
