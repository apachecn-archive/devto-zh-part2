# 高级 iOS 编程概念——它们真的“高级”吗

> 原文：<https://dev.to/essentialdeveloper/advanced-ios-programming-concepts--are-they-really-advanced-1j2l>

根据我们的经验，给主题贴上“高级”iOS 编程概念的标签非常棘手。“*基础*、*中级*、*高级 iOS 编程理念*”，应该由读者来评判，而不是作者！我们认为，尽管这可能是一种有效的营销策略，但将内容标记为高级可能是一个错误。

比如在我职业生涯的初期，C 是我用的最多的语言。C 语言承载了许多被认为是“高级”的概念，比如指针、分配和内存管理。然而，如果你像我一样开始用 C 编写代码，在学习过程的早期阶段理解这些概念是很自然的。初级 C 开发人员应该能够毫不费力地处理指针和内存分配。当然，初级 C 开发人员的知识会有差距，但随着时间的推移，他们可以掌握它。

现在，中高级 Swift 开发人员在指针和内存分配方面不如初级 C 开发人员熟练的情况并不少见。缺乏使用指针的经验并不是一件坏事。Swift 语言有助于内存分配，并对开发人员隐藏了许多指针操作/复杂性。这是 Swift 语言的创造者的意图，让开发者更容易创建更安全的代码。

## 我们可以给话题贴上“高级 iOS 编程概念”的标签吗？

当然可以，但是当我们将主题标记为高级时，我们通常会得到两种不同类型的响应:

1.  这不高级！这是我职业生涯刚开始时学到的。

2.  这就是高级！我从来没学过这个。

当一个新人开始编程时，一切都不知何故地“*高级*”随着我们职业生涯的进展，一些话题开始变得更加“*基础*如果我们开始给事物贴太多标签，一个奇怪的现象可能会发生。学生可能会说“如果这是高级的，我已经知道我能从这个来源学到的一切。”这种心态甚至会阻碍[学生在教育上投入更多](https://www.essentialdeveloper.com/articles/the-importance-of-discipline-for-ios-programmers)，因为在**先进的 iOS 编程概念**之后会发生什么？感觉就像旅程的终点。然而，专注的开发人员知道精通是一个无止境的旅程。

我们看到的一些常见的“高级 iOS 编程概念”有“OO”、“FP”、“网络”、“数据库”、“测试”和“架构”有趣的是，这些都是我们在 Essential Developer 教给初学者的概念。同时，我们还向拥有 20 多年经验、缺乏或需要更新知识库的高级开发人员传授相同的概念。

很难判断人们对于“高级”iOS 编程概念已经知道或不知道什么。在我们的课程中，我们专注于教授概念。我们尽量不要给概念贴标签，因为这是由学生来决定它们听起来是基础、中级还是高级。如果你已经熟悉了一个话题，而且看起来很容易，那就太好了！我们可以继续前进，关注阻碍你进步的知识差距。
