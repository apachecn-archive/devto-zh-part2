# 思想。

自从这个过程开始以来，很大一部分过程都集中在设计上，并且在头脑中有 UX 用户体验的想法。在很大程度上，我们已经能够做到这一点。直到小挫折。

> 2 周前，我在设计和开发方面没有取得太多进展，主要是因为我的电脑损坏了，第二周就修好了，所以项目回到了正轨。对于那些孜孜不倦地等待系列更新的人来说，真的很抱歉这个懒散的一周。

继续，我已经能够完成应用程序的设计。让我们看看在这个过程中取得的进展(听起来像说唱)。

## 缺点和成就。

在这个过程中，有一些缺点，也有一些成就，这些包括如下。

**缺点**

*   我在设计方面很慢，通常需要多花一点时间，但在一周的工作过程中，我已经能够改进了。

**成就**

*   我已经能够清楚地掌握整个项目的结构，并准备进入实施阶段。
*   所有设计都已完成##周目标。

新的一周(本周)将专注于这些设计的实现，想法是建立初始结构，然后专注于软接触(可以改进的领域)。

接口将在 react 中实现，我希望在本周完全完成的接口包括。

*   登录页面
*   登录页面
*   注册页面
*   主页仪表板

## 结论。

建设是一项任务，但大部分情况下，它与自我管理和过程有更多的关系。这意味着管理软件开发周期以及项目计划的过程，在决定项目的生存能力和可行性方面有很大的帮助。

作为一个一个人的团队，有时会感到力不从心，这就是为什么把所有的步骤都安排得很详细，并分成几个部分，以使一个人能够以一种有组织的方式发展。
