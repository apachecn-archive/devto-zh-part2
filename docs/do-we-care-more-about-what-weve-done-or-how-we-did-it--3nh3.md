# 我们更关心我们做了什么或我们是如何做的？

> 原文：<https://dev.to/scottshipp/do-we-care-more-about-what-weve-done-or-how-we-did-it--3nh3>

## 我爬行的皮肤

我刚从我的 LinkedIN 上尖叫着跑了。你可以想象，我在 LinkedIN 上的大多数朋友都是软件行业的资深人士。他们中的许多人现在是经理，而我选择留在技术领域。

是什么让我逃跑了？

嗯，这是一个熟悉的比喻，我刚刚达到了我的极限。你可能会说是稻草压断了骆驼的背。

我看到了另一个帖子，敦促我用 Go、Kubernetes 和 React 开发一些令人兴奋的东西。

## 科技名牌吐槽

开发人员和前开发人员有一个坏习惯，那就是关注他们的工作，而不是他们工作的结果。我看到太多对算法和技术名称的关注，而不是对它们的使用实际上完成了什么的关注。我们帮助了谁吗？我们让某人的一天变得更美好了吗？我们是不是至少让那些烂东西不那么烂了？

不要！但是我们使用了 Kubernetes、Docker、Go、React.js、Webpack、Swagger，以及谁知道还有哪些闪亮的、贴满标识的、被过度宣传的库、框架或工具。

你希望我说什么？

“oooohhhhhh！！！?"

“Aaaaaahhhhh！?"

不要！去做点真正的工作吧！

## 关机

抱歉，这变成了一场咆哮，但我厌倦了 LinkedIN、Reddit 和其他社交媒体网络上的开发人员的帖子，他们在最新的公共云平台上部署了最新的容器技术，他们使用了最新的热门语言和框架包，每个人都在告诉我们。

这让我想起了我想通过成为一名软件工程师来逃避的浅薄。我的意思是，如果我想听一堆人体测量，我已经知道去哪里(我当地的健身房)。“哦哇，看那个。。。多亏了我的乳清 2000 和 Bowman-self-musculator，我才有了这些岩石般坚硬的腹肌。”

对我来说，这一切都像是肤浅的加利福尼亚人。就像 Blink 182 的歌“Mutt”

> 他暂停刮胡子，告诉自己他是炸弹
> 她做好了卷发器，用信用卡支付了资金
> ...而且这太不健康了
> 通常他们以前都渴望得到关注

这种态度在软件中真的有一席之地吗？

我不这么认为。
