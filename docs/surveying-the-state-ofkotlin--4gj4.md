# 调查科特林的状态

> 原文：<https://dev.to/zmarkan/surveying-the-state-ofkotlin--4gj4>

我最近发起了一项调查，探索 Kotlin 生态系统的状态，以及开发者围绕🌍正在捡起来。这是我在[推手](https://www.pusher.com)作为开发人员传道者开始工作的第一批项目之一。
我邀请你帮我把它变成有史以来最伟大的科特林调查！

Kotlin 由 JetBrains 于 2011 年创建，用于他们的 ide 和工具中。由于其现代化的方法和与 Java 的简单互操作性，它很快俘获了 Java 和 Android 生态系统中开发人员的心。自 2017 年 5 月以来，它还获得了谷歌对 Android 开发的官方支持。

# 我们为什么要这样做

*   首先，我想了解开发人员如何使用 Kotlin，无论是在工作中还是在他们的业余项目中，是什么让他们进入 Kotlin，以及他们用于学习的资源:演讲、博客、书籍、课程或其他完全不同的东西？
*   其次，我很想知道 Kotlin 的一些最有趣的特性，比如协程和扩展函数，是如何被采用和使用的。

我的灵感来自于 JS 的状态，这是一个分析 JavaScript 生态系统中最流行的特性和工具的优秀资源。我做这个调查完全是出于好奇，想给社区一些回报。

我们在 Pusher 为所有新的 Android SDK 开发使用 kot Lin——包括[推送通知](https://pusher.com/push-notifications)和[聊天工具](https://pusher.com/chatkit)。
**科特林开发者社区棒极了**💯提供有价值的见解和打造最佳实践。

我的计划是在几周内公布调查结果——很可能在三月份。我希望这个调查能够帮助那些还没有决定是否使用它的组织和团队采用它。
