# 我还没准备好成为巴别塔的维护者

> 原文：<https://dev.to/hzoo/i-was-not-ready-to-become-the-maintainer-of-babel-2j6>

当 Babel 的创作者 Sebastian McKenzie 开始远离这个项目时，他给了我和其他一些人回购/出版权。我知道我还没有准备好成为[巴别塔](https://babeljs.io/)的维护者。毕竟，我以前从未发布过自己的 npm 包，也没有探索过很多代码库。但是慢慢地(有时候真的很慢)我就习惯了。我记得 [Kent Dodds](https://dev.to/kentcdodds) 说过，如果你想成为一个项目的维护者，就行动起来，做维护者做的事情。

当你在学习新东西时，你会发现有多少东西你不知道。你可能会发现[这不是冒名顶替综合症](https://rachsmith.com/2017/i-dont-have-imposter-syndrome)。你不知道其他人知道多少，也不知道你已经走了多远。

有[不同层次的进展(一系列推文)](https://twitter.com/i/moments/789192306343542784):可能是你的第一个问题或评论，第一次公关，第一次发布，第一次回购等。**接下来的事情似乎总是那么势不可挡。记住这意味着你学到了更多，这很有帮助。从一无所知开始，不代表你不合格。如果你处于任何位置(训练营、开发、贡献者、维护者)，这意味着有人*希望*你在那里。互相鼓励吧！**

也许这是一个信心问题，就像在做演讲甚至参加 TC39(指定 JavaScript 的委员会)之前感到紧张一样。每个人都有不同的东西要拿出来，我们不应该遵循我们对开发人员的刻板印象。

我们如何建立信心？一部分是学习关心你所做的事情，但要思考你如何定位自己，以及你是否能从工作中脱离出来。如何面对反馈、舆论等？对我来说，这很大程度上来自于学会如何忘记自己:不要高估自己，甚至低估自己，而是更少地考虑自己。通过与他人和自己的共情，我可以不断提高自己前进的能力。我试着给自己犯错的空间，给别人犯错的空间。我认为在那种自由中，我们可以无所畏惧地尝试。

> 我听过一些我经常说的话，就是“我认为”说得太多了😁所以这是我可以改进的地方。

然后，我们可以开始把自己视为开发人员，而不是想出理由认为我们不够好。让我们作为一个团体在这个旅程中互相鼓励。我们真的准备好了吗？

今天我全职维护巴别塔。我能做到这一点很大程度上归功于捐赠，所以我会感谢[对我的 Patreon](https://www.patreon.com/henryzhu) 的任何贡献。我将会分享更多类似这样的短文，来阐明我的过程和 JavaScript 的前景。很高兴在这里参与讨论！
