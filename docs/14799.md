# 让我们来谈谈发布管理

> 原文:[https://dev.to/michaelgv/lets-talk-release-management-2k 47](https://dev.to/michaelgv/lets-talk-release-management--2k47)

我经常致力于改进我自己的发布，并想众包一些信息——你是如何发布的？

我是这样做的:

1.  在私人 gitlab 服务器上有一个关于如何发布每个产品的报告，以及另一个报告中的部署和试运行指南、devops 代码

2.  当你发布新版本时，你写一个信息更新日志，广播即将发布的版本

3.  在发布上线之前，有一个 devops 冻结，防止任何人部署到临时和生产，以便我们可以测试这个 RC

4.  获得批准后，第三方将进行一次检查，并最终合并变更日志。一旦变更日志被合并，发布就被标记，CI 开始构建

5.  完成构建和分发后，我们升级我们的生产服务器，这总共需要大约 20 分钟

6.  在准备和开发阶段解除开发运维冻结，周期结束

7.  发布经理记录他们做了什么，花了多长时间，下次的想法/主意，并提交给发布管理回购
