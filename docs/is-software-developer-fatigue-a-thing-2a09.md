# 软件开发人员疲劳是一件事吗？

> 原文：<https://dev.to/rhymes/is-software-developer-fatigue-a-thing-2a09>

“JavaScript 疲劳”是一个众所周知的概念(它也带来了很多好处),是由近年来 JS 社区的爆炸式增长引起的，但我也想就软件技术的其余部分展开讨论。

我们是否有太多的编程语言？有时我发现自己在思考这样一个事实，即我们(作为地球上的开发者)花费如此多的时间用无数种语言编写“日期/时间库”,因为每一种新的语言或框架显然都需要一个。我不是在这里争论优点，每一种语言对某些人来说都是惊人的或有用的，但它们都需要相同的组成部分。我们会为每个领域找到一种“真正的语言”吗(我知道，我的推理是不可靠的),并停止每次都重写函数将一个字符串拆分成一个数组吗？
