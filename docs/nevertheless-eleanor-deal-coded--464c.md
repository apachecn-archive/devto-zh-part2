# 尽管如此，埃莉诺交易代码

> 原文：<https://dev.to/unoxymoronic/nevertheless-eleanor-deal-coded--464c>

## 我开始/继续编码是因为...

有人认为我值得雇佣。我很震惊。

我在大学中途意识到，我选择了错误的专业，夹杂着许多个人问题，我勉强毕业，对音响技术学位为我设定的任何职业都不感兴趣。但是我的课程确实帮助我意识到我对工程比对艺术更感兴趣；因此，我在最后一年申请了各种基于工程/技术的研究生项目。

我没有被任何地方接受。我参加了一家大型咨询公司的面试，面试官无视我的陈述，嘲笑我让他想起了他的女儿。我放弃了尝试进入技术行业，而是专注于找到任何我能找到的工作。

我申请了私人助理、呼叫中心的工作，任何不需要特定学位的工作。我把我的简历贴在求职网站上。几个月过去了，我几乎得不到一次面试机会。然后，不知从哪里，我接到了一个电话，是一个毕业的 IT 招聘人员打来的，他提出要把我培养成一名开发人员，并把我安排在他们的客户那里获取经验。

突然，我有了一份工作，在一个让我兴奋的行业，我甚至没有想过申请！

我的第一个开发角色周围都是人，他们带我了解基本的 SQL/UNIX/HTML 知识，并教我在我们的组织中取得成功所需的框架和语言。他们让我走上了自信之路，并鼓励我表达自己的观点。

他们还介绍我去，也就是我真正爱上编程的时候。突然，事情对我来说变得更加自然，我可以考虑设计决策而不是语义，我的信心开始增长。

## 我想吹嘘一下...

为自己挺身而出，掌控自己的事业。

1)在最初的两年里，我指导其他毕业生，当我们的团队没有高级开发人员时，我代替他们，当他们需要支持时，我被要求在其他团队工作。我还和我的经理发生了几次口角——他告诉我，我需要调整我的态度，我太“消极”了，我想得太多了，我还没有为我申请的晋升做好准备。

所以我对他提出了投诉，并寻找新的工作。我知道了自己的价值，然后离开了。

2)我的下一份工作让我加入了一个开发遗留 Java 应用的团队，完全超出了我的舒适区，但在危机中我让自己变得足够有用，客户创建了一个新的运营团队，并让我领导它。

我与跨平台的 3 个团队合作开发跨微服务的日志记录标准，与架构师合作将平台的所有日志和指标转发到聚合器，实现平台的第一个警报，并编写标准来指导每个团队创建更多。

这与其说是我掌握了控制权，不如说是对管理团队的女性的感谢，感谢她们认识到我的潜力，让我进入一个可以茁壮成长的环境，而不是放弃我。

3)我有了一个机会，可以为一个政府项目提供咨询，与我喜欢的技术合作，与激励我、鼓励我的人合作。我的公司认为这太过分了；我们没有足够的人来建立一个团队，所以不值得花时间申请这个项目。我不同意，自己接手了申请。我撰写、提交并赢得了合同，并招募了一个 10 人的团队与内部团队密切合作，共同交付项目。

我从没有这种政府采购的经验，到建立并赢得一个主要合同的投标。我还把自己列为技术负责人，并在合同过程中逐渐成长为这个角色。

现在人们不断地对我说，我在这么短的时间里取得了这么大的成就——我一定是在管理的快车道上了。每次有人告诉我他们知道我的下一步职业发展，我都会有一个非常简单的回答:

“我已经赢得了自己决定的权利。如果我想留在原地，那由我决定。我足够固执，也足够自信，不会被迫做任何事情。”

## 我最近克服了...

我对不显得不专业的执着。

第一次管理一个团队，我不得不雇佣和解雇一些人——在这一过程中的每一步，我都担心自己会做错什么。我会问错误的面试问题，我会因为解雇某人而让自己陷入法律纠纷。

我也被推到了个人极限，不得不为我在工作环境中能做什么和不能做什么直言不讳——这涉及到很多眼泪、自我反省和处理以前的创伤。

我意识到，和我进行过所有这些艰难、可怕对话的人，事后也同样尊重我。我仍然能够有效地完成我的工作。所以也许我不需要那么担心方式的对错，接受别人和我一样都是瞎编的。

## 我仰望...

社区组织者设法变得活跃和积极，而不是完全精疲力竭！任何人在日常工作之外还能抽出时间来指导、提供指导或协调会议发言人，真的让我大吃一惊。

## 我对支持女性和非二进制代码的盟友的建议是....

当你在聚会上看到某人独自一人时，去介绍你自己。提供一些关于你自己的信息来减轻其他人的负担。尽量避免问“什么风把你吹来了？”因为这可能会被视为“你看起来不属于这里，证明我错了”。

走进一个似乎每个人都互相认识的聚会，可能会让人感到非常害怕，尤其是当你意识到自己是少数群体的时候。盟友悄悄地伸出手来，弥合这一鸿沟，可以决定一个人是否觉得自己是社区的一部分。
