## 阅读

### [基于状态的组件与香草 JS](https://gomakethings.com/state-based-components-with-vanilla-js/)

作为一个在过去的一年里专注于提高我的普通 JS 技能的人，Chris Ferdinandi 的每日建议对我来说是无价的。在 Javascript 框架的海洋中，看到一些关于用 vanilla JS 构建基于状态的组件的文章令人耳目一新。

### [分数](https://ethanmarcotte.com/wrote/fractional/)

CSS 网格，现在很热。伊森分享了他对`fr unit`的新欢。

> 简而言之，`fr`是一个近乎神奇的值，它让我们能够快速寻址网格区域中的剩余空间。

### [关于设计和建造拨动开关](https://www.sarasoueidan.com/blog/toggle-switch-design/)

Sara 在 CodePen toggle 上发现了一个奇怪的行为，这导致了一个关于构建一个可访问的 CSS 驱动的 toggle 的深入教程。

*   风格和功能是相互关联的；因此，在设计易访问性时，同时考虑这两个因素会有所帮助(实际上，设计任何东西都是如此)。
*   总是，总是在构建组件时开始考虑标记和可访问性，不管它们看起来有多小或多简单。

## 工具

### [偷懒](https://github.com/aFarkas/lazysizes)

Lazysizes 是一个全功能的 JS 库，可以帮助你轻松加载媒体。你不仅可以加载内嵌图片，lazysizes 还提供插件来加载那些过大的背景图片。你的手机用户会感谢你的。

*   [如何在 YouTube 上用 lazysizes](https://www.youtube.com/watch?v=Uwnmn65cMec) 加载图片
*   [如何在 YouTube 上用 lazysizes](https://www.youtube.com/watch?v=UFKlvJO6GoA) 加载背景图片

### [前端性能检查表](https://github.com/thedaviddias/Front-End-Performance-Checklist)

> 性能是一个巨大的主题，但它并不总是一个“后端”或“管理”主题:它也是一个前端责任。前端性能清单是一个详尽的元素列表，作为一个前端开发人员，您应该检查或至少了解这些元素，并将其应用于您的项目(个人和专业)。

### [boile form](https://boilerform.design/)

> Boilerform 是一个 HTML 和 CSS 样板文件，可以减轻使用表单的痛苦。

我早就用 [WTF 了，表格？](http://wtfforms.com/)作为我工作中表单元素的起点，但 Boilerform 看起来很坚固。它还提供了一个基本的验证接口，扩展了本地 HTML 验证功能。
