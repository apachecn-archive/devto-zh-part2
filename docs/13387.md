# 避免精疲力尽，回到基础

> 原文:[https://dev . to/restore dev/avoid-burnout-and-get-back-to-the-basics-4j 46](https://dev.to/restoreddev/avoid-burnout-and-get-back-to-the-basics-4j46)

有没有因为要学的东西太多而感到不知所措？学习 Redux、CSS Grid、TypeScript、Go、Rust、Elixir、Kotlin 或 Elm 的压力是否让你感到沮丧？软件业变化如此之快，很难跟上所有的最新趋势。因为软件发展很快，我认为它会在程序员中产生一种潜在的恐惧，担心如果他们不学习每一项新技术就会落后(我知道对我来说是这样)。这种恐惧会导致“冒名顶替综合症”,即知道得不够多，以及因为试图学习太多东西而筋疲力尽。我就掉进了这个陷阱，很难对抗。那么治愈的方法是什么？我认为重要的是回到我们工艺的基础，而不是跟随潮流和流行竞赛。那么，你要怎么做才能回到基础呢？

### 不要试图学习每一种新的编程语言

尝试学习十种不同的编程语言是非常诱人的，这样你就可以在简历中列出每一种语言。然而，将你的经验传播出去会让你在一项技术上缺乏深度。仅仅因为你可以用五大函数式编程语言创建一个“hello world”并不意味着你可以为他人构建有意义的软件。现在，我喜欢为了乐趣而探索其他编程语言，这种追求没有错，但是不要让探索阻止你更好地掌握你已经知道的语言。我主要使用 PHP 和 JavaScript。PHP 已经多次被宣告死亡，但它仍然是世界上十大编程语言之一。虽然因为 PHP 的名声不好，很容易将它抛在我的学习之外，但我想成为一名更好的 PHP 程序员，并每天学习更多的相关知识。我的最终目标是写出好的软件，只要软件运行良好，我使用哪种编程语言并不重要。

### [](#learn-to-write-clean-code)学会写干净的代码

我最近通读了罗伯特·c·马丁的《干净的代码》( Clean Code )( T1 ),他提供的建议对我最近的一些项目非常有价值。什么是干净代码？对我来说，干净的代码可读性很强，很容易理解，也很容易扩展或改进。写干净代码的能力并不依赖于最新的趋势或者黑客新闻顶端的编程语言。干净的代码可以用任何语言编写，这是一项在任何未来编程迭代中都很重要的技能。你不会后悔学习编写更好的代码，而不是阅读最新的 JavaScript bundler 的文档。

### 专注于编写好的测试和安全的软件

我知道，我知道单元测试和安全是无聊的话题。它们不如最新的“Ruby on Rails 杀手”有趣。然而，我认为，除了编写代码本身之外，这两方面的专业知识比任何其他主题都更重要。能够编写好的测试极大地提高了代码的质量和快速修改代码的能力。保持适当的安全性可以节省您的资金，让审计员远离您，并保持您的客户的信任。像编写干净的代码一样，能够编写测试和维护安全性是包含每种语言和技术的两项技能。通过掌握这两种技能，你对你的公司或潜在雇主来说将比了解每一种编译成 JVM 的语言更有价值。

最后，回到基础就是关注简单、好的编程。它不是关于学习复杂的构建工具或硅谷使用的流行语言。这是关于增加你知识的深度，而不是分散它。回归基础就是把你自己从需要知道每个愚蠢的名字如 Gulp 或 Guzzle 中解放出来，只是改进你如何用你最喜欢的语言写一个类。这是关于放下学习一切的压力，专注于学习几个小而重要的话题。回归基本不会让你得到一个电视节目或一条热门推文，但它会让你安心，并提供更好的产品。这不就是全部的原因吗？
