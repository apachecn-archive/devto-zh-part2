# 老狗，软件开发者的新把戏

> 原文:[https://dev . to/rodney ringler/old-dogs-new-tricks-for-a-software-developer-55nj](https://dev.to/rodneyringler/old-dogs-new-tricks-for-a-software-developer--55nj)

最近，我换了家公司，发现自己不得不学习一种全新的开发方式。我花了 6 年时间在一个完整的 Windows/上。网店。一切都是 C#、Windows 服务器、SQL Server dbs、Visual Studio、TFS、Windows/。网络生活。这是一个非常同质的架构，所有的开发人员都是 C#专家。

在我的新公司，尽管我是 C#/招聘的。NET 专业知识，我发现自己被交给了一台 Mac，作为我的开发机器和一个完全不同的世界。他们主要使用 linux 服务器。我发现。NET 真的只用于少数人。NET 核心 api 在 linux 上运行(arrgghh)。他们的平台/语言/架构完全不匹配。后端是一大堆 php，node js，python，还有 go，几乎每一个你能想到的前端(angular，django/python，vue，react，straight html 等等。).它们完全是 AWS 云。所有开发人员都生活在 bash 终端窗口中，使用某种代码编辑器(vs code、sublime、atom 等)。)而不是一个成熟的 ide。

我的新开发团队无法相信有人不在 mac 上开发。他们认为成熟的 ide 过于复杂，终端窗口、代码编辑器和控制台打印语句是调试的最佳方式。不要成为精英程序员，这对于简单的东西来说似乎很好，但是一个复杂的多线程应用程序需要调试的不仅仅是控制台日志语句。嗯，这对我来说是一个很大的改变，但我已经做出了改变，因为我厌倦了，想学习新的东西，并看看一群年轻人的企业网上商店是什么样子。我不知道我会学到多少东西。也许这是中年危机，谁知道呢。我确实第一次留了胡子，还纹了身。我开始觉得又酷又时髦了。
现在到了文章的主题，我对这两种截然不同的环境和架构的优缺点的观察。

作为开发机器的 Mac。我喜欢。我发现自己讨厌承认这一点，但 mac 是一个非常可爱的机器。我确实有 Parallels 和 Windows 10 虚拟机，以备那些时候我想做一些核心 C#的东西，但我发现自己越来越多地生活在 mac 上的终端窗口和 vs 代码中。

说到终端窗口。对我来说这是一个完整的循环。早在 90 年代初，我就开始使用 vax/vms 终端和一些 unix 机器。我喜欢终端窗口的私密性，并且离操作系统更近。也就是说，我仍然是为你处理底层事物的好工具的粉丝。我的怪胎喜欢终端窗口。在我看来，有经验的开发人员喜欢一套好的工具来处理底层的东西。

代码编辑器与成熟的 ide。对于 node、python、php 和更简单的东西，我开始喜欢上了，而不是代码。我已经决定在崇高和原子。也许这是我的 visual studio 根源，但我更喜欢它。我可以在里面做一些基本的调试，它和 git 集成的很好。但是对于严肃的开发来说，没有什么比一个成熟的 ide 更好的了。在我看来，完整的 Visual Studio 体验是无可匹敌的。它的工具是无与伦比的。我发现与我共事的大多数年轻开发人员都不喜欢它陡峭的学习曲线。他们觉得太复杂，不想花时间去精通它。我认为这是一个有点短的网站，但他们所做的大部分是不需要 visual studio 的项目。创建一个查询数据库并返回一些 json 数据的 api 并不是什么难事。此外，由于它主要是一个混合环境，而且基于 linux，visual studio 的许多优势都被削弱了。但是在微软和 Visual Studio 的插件中，他们已经为他们的 linux 工具添加了很多东西。

多种语言。我从来没有处在这样一个精神分裂的环境中。这是类固醇的流行趋势。他们认为，这部分是由于收购。这样做的好处是，它允许很大的自由，允许选择使用最适合特定任务的语言。这样做的缺点是有大量的技术债务，每个人都成为所有人的大师，却没有一个人是专家。我发现这是最大的缺点。每个人都是我所说的精通多种语言的人，但没有一种是大师。成为您所开发的语言和平台的专家是有好处的。你知道最佳实践、最佳设计模式、最新特性和趋势等。我相信通过这种方式，你可以开发出更干净、更易于维护和扩展的代码。对我来说，这是我看到的新环境最大的缺点。但是从学习新事物的角度来看，每天使用多种语言而不是只有 C#是很好的。

总之，我认为在这两种环境之间的某个地方可能是两个世界中最好的。自私地说，学习和接触所有这些语言、平台和趋势是非常棒的。但是如果是我的公司，我肯定会采用某种定义好的架构。我很惊讶我有多喜欢 mac、终端窗口和 vs 代码。对于小任务来说，这是一套很好的工具。

我期待更多的学习和成长。永远不要过于偏见或停留在你的专业平台上，以开放的心态尝试新事物。在生活和软件开发中，我试图以此为生。
