# 然而，亚历克莎·塔斯基编码了

> 原文：<https://dev.to/lasertuskey/nevertheless-alexa-tuskey-coded--392c>

## 我开始/继续编码是因为...

从美好的 Myspace 时代开始，我就对代码着迷了！当我发现在我的个人资料页面的标记中有我可以学习操作的模式和序列时，我就着了迷。我在高中上了一堂 Java 课，这让我大开眼界，因为我知道我不仅可以改变屏幕上的东西，还可以准确地告诉计算机我希望它在我的程序中做什么。

当我不得不在大学选择专业时，我在平面设计和计算机科学之间左右为难。我一直是一个视觉型的人，对于 CompSci 学位的微积分和其他数学要求让我害怕，所以我选择了前者。虽然我喜欢学习如何设计，但我的后脑勺总有一个小小的声音在说，如果呢？

快进到我职业设计生涯的第四年。离开我短暂的平面设计生涯后，我发现我的激情在于设计网站。我在一家较小的公司工作，各部门紧密合作。我发现自己很嫉妒我开始合作的前端开发人员。他们实际上是在建网站！我记得我是多么喜欢修补代码，并感受到重新开始学习的动力。没过多久，我那位观察力敏锐、支持我的经理就意识到了我对编码潜在的热爱，并帮助我在职业生涯中追求一条新的道路；一个我可以设计和建立网站的地方。

## 我最近克服了...

我的许多新技能都是在工作中从我的开发伙伴那里学到的，他们中的大多数人要么上过编程学校，要么已经做了很多年了。很容易将自己归类为“新手”，因为我现在只做了一年左右的前端开发。然而，我开始意识到“新手”这一点很棘手，因为它有时会成为我的借口。“我学不会那个，太难了！我只是没有足够的经验去尝试！我不可能在那个项目上帮忙！”

虽然我当然知道在最初的几年里我会继续奋斗，但我已经意识到前进和尽可能多地学习的唯一方法是尽可能多地接受这些挑战，并接受我可能会失败或不得不寻求帮助，但最终，我会学到一些东西。有一天，我的一个同事向我寻求帮助，解决 IE 中的一个 Flexbox bug，他是一个有多年经验的开发人员。我的第一反应是说“我不确定，问问其他前端开发人员，”但我忍住了恐惧，想了想，意识到我确实知道答案！我能够帮助他摆脱困境，并且觉得我真的开始相信我自己和我的能力。

## 我对女性和非二进制编码人员的建议是....

相信自己的能力，相信自己的实力，相信自己的聪明。这说起来容易做起来难，但它对你学习和前进的速度和信心有很大的影响。记住你已经走了多远，知道这一点会给你多少力量。要知道，寻求帮助会让你变得强大，而不是软弱，大多数高级开发人员喜欢帮助你解决问题。把它交给其他需要你帮助的开发人员，即使是帮助解决一个最小的问题，对一个人来说也意味着一切。
