# 我读过的最差的科技文章

> 原文:[https://dev . to/laszlolm/the-worst-tech-article-ive-ever-read-32lj](https://dev.to/laszlolm/the-worst-tech-article-ive-ever-read-32lj)

这是一个黑暗的周一早晨，Mashable 确保通过他们的广告接触到的人会更加黑暗。

我的一个朋友从脸书发给我一张截图，上面写着“Java 仍然是你需要学习的编程语言的 4 个理由”
[![The screenshot I got](../Images/f7ea6265584387929a271f385ea17aff.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--G4rFAbph--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/yb0ohiefta1pupgjm5a2.jpeg)

这个到目前为止没什么问题。不要误解我，我不是一个讨厌 Java 的人，我不想指出关于 Java 的谬误，但是当你浏览这篇文章时，标题变成了【Javascript 仍然是你需要学习的编程语言的 4 个理由”这个事实已经提高了我对它的紧张程度。

所以我继续读下去..

第一段告诉我 1995 年 JavaScript 第一次出现的时候发生了什么。没关系，没什么不好，只是我不确定希拉里·克林顿成为第一夫人和 JavaScript 的出现有什么关系。

让我们跳到第一个副标题:**“1。Java 的钱在哪里”**
等等，我仍然不能确定他们在这篇文章中写的是 Java 还是 JavaScript，以及重点是什么。每种编程语言都有投资和使用它的组织。例如，Erlang 的平均工资远远高于 JavaScript 或 Java。

所以下一段他们还在写 Java。事实上，他们引用的是 Java 而不是 JavaScript 的工资估算(第一个是基于快速谷歌搜索的平均 73000 美元，第二个是 115000 美元)。他们还引用了使用 Java 的谷歌。在某种程度上，谷歌的一些产品确实使用 Java，但其大部分基础设施都是建立在 Python、JS 和一些非常低级的技术上，如 C/C++。Java 只在团队只由 Java 开发人员组成的产品中使用。非常罕见。

他们也称 Java 为开源。据我所知，OpenJDK (Java 开发工具包)是开源的，但 Java 不是。你称之为 Java 的任何东西都需要符合 TCK 标准并得到甲骨文的批准。此外，由甲骨文公司制造的原始 JDK 不是开源的。

这里他们在文章中添加了一个 Java 课程商店的链接，所以我认为这篇文章实际上是关于 Java 的。但是为什么要把 JavaScript 掺进去呢？

最后一点是关于学习它的难易程度。嗯，我可以反对这一点，但这取决于个人喜好。我只是不知道这篇文章的作者是否在里面写过一行字。

为什么科技出版商不对他们的文章进行事实核查？这实际上是一个非常短的问题，如果没有错误的信息，它甚至可以给没有经验的人一些启示。事实上，他们正在社交网络上大肆宣传，这意味着这可能会影响到很多对这个行业一无所知的人。

### 这篇文章可能是我最近读过的最差的文章，但是我很想听听你最差的开发者文章。
