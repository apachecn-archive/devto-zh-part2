# 你如何处理客户端超时？

> 原文:[https://dev.to/orkon/how-do-you-handle-client-timeouts-4558](https://dev.to/orkon/how-do-you-handle-client-timeouts--4558)

您有一个端点进行一些有价值的计算，它应该在 3000 毫秒内做出响应。因此，客户端将 3000 毫秒作为请求的超时设置。不管出于什么原因，有些%的请求开始需要 6000 毫秒。现在的情况是，在客户端断开连接后，执行继续进行。结果就被丢弃了。

问题是:用你目前的技术，你如何处理它(如果有的话)？你会中止计算吗？你是怎么做的？
