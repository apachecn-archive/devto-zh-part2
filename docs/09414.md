你有没有看过一个网站，并认为:“所有的东西要么是土豆，要么不是土豆……”以及更多(或更少？)重要的是:“这个网站作为一个应用程序会很棒！”？如果你做了，你并不孤单，伙计！

互联网总部的一些非常酷的人同意你的观点(土豆的事情*和网站作为应用的事情*)(不要去核实事实，只是盲目地相信它的存在)!*哇！看你走了！*)。他们引入了一个概念，简称 PWA。

****它被称为渐进式网络应用，因为它会随着你的使用而逐渐变得更好****

* * *

PWA =进步网络真棒！
有几件关键的事情会让你想用 PWAs 摇滚起来

*离线模式*

想象一下这个场景。你在一个新闻网站上，刚刚读完一篇文章，过了一会儿你想重读这篇文章或者给别人看，但是有一个问题！

*你不能上网！*

如果你想读的文章在非 PWA 网站上，你会很不高兴，得到一大堆“没有网络连接”的屏幕，但是如果文章在 PWA 网站上，你会很好地阅读你的文章。

PWAs 所能做的是保存你访问过的网页，即使你不能上网也可以使用它们。对我来说，这是一个*游戏规则改变者*，它减轻了每个网站最大的弱点，在某些情况下，完全消除了它！

*推送通知*

让我们看看另一个场景。

你拥有一个电子商务网站，看着你的网站分析报告，你得出的结论是，你的许多访问者来到你的在线商店，看看商品，考虑购买一些东西，甚至可能经历了一些购买过程，然后就离开，再也不会被看到。

PWAs 可以为你做的是发送*推送通知*，这些通知感觉自然，看起来像你的访问者正在使用的设备——即使浏览器没有在他们的设备上打开——让他们知道他们之前一直在看的商品正在打折，并且只需点击一个按钮就可以让他们回到他们离开的购买过程。

外卖很明确。拥有一个 PWA 网站就像让一个销售人员日以继夜地为你带来新客户或重新吸引老客户，让他们知道你仍然重视他们的业务。

Jumia 就是一个很好的例子。博客后面有一个链接，一定要去看看。

*网页分享*

在这种情况下…

*我开玩笑的！没有这个场景。唷！*

撇开玩笑不谈，在现代社会，在应用程序和平台之间共享内容的能力是绝对必要的。T3】

Web share 调用设备的本机共享功能，这意味着任何允许在 PWAs 上共享的应用程序或平台都将能够与该应用程序或平台共享内容。

*我说过我们会经历一个场景*

假设你拥有一家企业，分享内容是你在线营销策略的关键部分。当你第一次雇佣一个网络开发工作室时，他们整合了脸书和推特共享，这在当时是完美的，但后来你的业务已经发展壮大，比如说，扩展到了中国。*好看！*

现在需要整合微信上的分享。为了让您的 web 开发团队集成微信，他们需要阅读开发人员文档(可能会也可能不会翻译成您的开发人员熟悉的语言)，接下来他们需要编写代码，之后他们需要测试新代码是否有效，以及它是否不会破坏任何旧功能。

所有这些工作都要花钱——如果不是成千上万的话，也有几百美元可以用在其他地方，比如你的产品营销。

有了 PWAs，所有这些都可以避免，因为编写一个共享功能可以覆盖所有相关平台，供您的 web 应用程序访问者使用。
*分享走我任性的儿子！*

*削减成本*

在大多数情况下，企业只需要一个网站就可以在网上展示和相关，但在其他情况下，他们需要只为本地设备保留的一小部分功能。

这迫使企业投资昂贵的 Android 和 iOS 本地应用，价格在 2 万到 12 万美元之间。

pwa 希望通过使网站更像本机应用程序并利用设备本机功能扩展其功能来解决这一问题，结果是大幅削减了成本，因为公司现在只需为一个平台而不是三个平台开发。

*谢谢 PWA！*

*成功案例*
还不相信 PWAs 是牛逼的酱？好吧，让我们问问这些正直的企业:

*全球速卖通([链接](https://developers.google.com/web/showcase/2016/aliexpress) )*
速卖通是一个受欢迎的电子商务网站，让你可以直接从中国订购商品。

将他们的站点迁移到 PWA 后，他们看到了以下结果:

*   *所有浏览器的新用户为 104%；iOS 转化率增加 82%*
*   在所有浏览器中，每个用户每次会话访问的页面增加了 2 倍
*   在所有浏览器中，每次会话花费的时间增加了 74%

*朱米亚([链接](https://developers.google.com/web/showcase/2016/jumia) )*

Jumia 是非洲领先的电子商务网站。我将强调他们面临的挑战(摘自上面的链接)

在 Jumia 的移动网站上，每天有近四分之一的购物车被遗弃。为了应对这种情况，他们通过电子邮件提醒顾客完成购买。但是让人们提供他们的电子邮件地址是有挑战性的，并且电子邮件打开率很低。由于超过 65%的 web 流量来自移动浏览器，Jumia 希望通过推送通知等先进的 web 应用技术来解决他们的问题，并重新吸引放弃购物车的移动用户。

PWA 为他们做了什么:

*   38%的开放率
*   网络推送用户先前放弃的购物车的转化率提高了 9 倍
*   网络推送用户先前放弃的购物车的转换率为 7.85%，而本地应用的转换率为 4.5%

推特([链接](https://developers.google.com/web/showcase/2017/twitter) )
拜托…我不会跟你解释推特，你知道它是什么。

Twitters PWA 在 2017 年 4 月成为移动设备上的默认 web 体验。

**工程负责人提示:
Twitter Lite 是目前最快、最便宜、最可靠的 Twitter 使用方式。
这款网络应用的性能可与我们的本地应用相媲美，但与 Android 版 Twitter 相比，它只需要不到 3%的设备存储空间。**

结果:

*   每次会话页面增加 65%
*   发送的推文增加了 75%
*   跳出率下降 20%
