# 学习生成艺术 7 天的感受

> 原文:[https://dev . to/oh barye/what-I-feed-through-7-days-learning-generative-art-1fka](https://dev.to/ohbarye/what-i-felt-through-7-days-of-learning-generative-art-1fka)

正如我在[我学习生成艺术的第一步](https://dev.to/ohbarye/my-first-step-in-learning-generative-art-3674)中所写的，我开始学习生成艺术。我很高兴有人对这篇文章和[关于它的推文](https://twitter.com/ThePracticalDev/status/991301465405902848)做出了一些反应，所以我决定画一幅画，连续七天每天都贴出来。

## 开发环境

有一个名为[处理](https://processing.org/)的语言和环境，但我使用了 [p5.js](https://p5js.org/) ，因为我想编写熟悉的 JavaScript 代码。

我在 https://alpha.editor.p5js.org/的或 https://www.openprocessing.org/的[用网络上的交互式编辑器画出了它们。这些网站非常棒，即使你是初学者也可以画一幅画。启用自动刷新后，可以快速查看外观，非常方便。但是要小心，如果你在代码中犯了导致无限循环或 OOM 的错误，浏览器会挂起，你在编辑器中的代码可能会丢失。那是我面临的最痛苦的事情。](https://www.openprocessing.org/)

## 印象&我学到了什么

因为我使用过 HTML Canvas，所以开始用 p5.js 画图的门槛并不高。

“嗯，我可以用`line`函数画线，简单”。“哼，我可以用`rect`函数画一个矩形，好吧”。

有道理，我认为这是完全可以理解的——直到我在地球上看到了[高水平的真正艺术品](https://www.google.co.jp/search?q=generative+art&source=lnms&tbm=isch&sa=X&ved=0ahUKEwj7sLzemfHaAhVr1oMKHQMKCuUQ_AUICigB&biw=1440&bih=776)。看到他们的那一刻，我无意中对自己说了一句“哼！？！?"。

说了这么多，我就知道，习惯的唯一办法就是耐心使用。开始动手动脚后，很快就发现很迷人。

与真实商业中的代码不同，生成艺术没有确切的答案可供选择，在我承认它好之前没有尽头。即使当我想“我要完成了”，比较一些模式，我不能决定选择哪一个，然后我会花一些时间。然而，**随着时间被无限吸走，我发现我还记得编码的快乐**。

虽然很难继续每天张贴，但我想在未来继续绘制生成性艺术，以便我永远不会忘记这种感觉。

代码从这里开始变得越来越复杂，以后似乎很难再读回来。
