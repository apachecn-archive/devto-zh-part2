## 什么是 Forem？

Forem 是用于构建社区的开源软件。为你的同行、客户、粉丝、家人、朋友以及任何其他需要聚集在一起成为集体一员的时间和空间的社区[参见我们的公告帖子](https://dev.to/devteam/for-empowering-community-2k6h)以获得 Forem 是什么的高级概述。

[dev.to](https://dev.to) (或者只是 dev)由 Forem 托管。这是一个软件开发人员的社区，他们撰写文章，参与讨论，并建立自己的专业档案。我们重视支持性和建设性的对话，为所有成员追求伟大的准则和职业发展。该生态系统涵盖从初学者到高级开发人员，欢迎所有人找到自己的位置…

每次我们有机会让社区以新的方式做出贡献，我们都会感到惊讶和惊叹。自从几年前项目开始以来，我已经一次又一次地看到这种情况发生。这不是一个库，这是运行这个平台的代码库。而且分享起来很有乐趣！

这一新篇章是我们努力维护和提高透明度的持续努力的一部分，当然，也是改进和增长我们可爱的代码堆的一个机会。正如我在关于主题的上一篇帖子[中提到的，我们非常兴奋地看到，有了您的贡献，这个平台最终可以重新用于我们范围之外的社区和生态系统。](https://dev.to/ben/the-devto-codebase-will-go-open-source-on-august-8-2kia)

我们将继续维护和发展我们的开发者社区，但作为一个强大的开源项目。我们认为这种方法可能会产生许多有趣且有价值的用例。我们希望一些贡献者带着支持和改进 [dev.to](https://dev.to/) 本身的目标进来，而其他人作为独立的实体来考虑可能的用例。

因为这是一个起点，贡献指南和开源开发者体验需要我们不断改进。因此，如果你遇到了绊脚石，我们会在这里帮助你，但我们也一定会随着时间的推移让事情变得更好。

如果您来自网络上的其他地方，并且不熟悉 [dev.to](https://dev.to/) ，我们是一个大型的软件开发人员在线社区，致力于相互教学，建立我们的职业生涯，并且通常使软件开发成为一种更加协作、人性化的努力。我们主持从初级到高级的文章和讨论，随着我们的成长，我们总是努力营造一个支持不同用例的建设性环境。只要是关于代码或开发者体验/生活的，任何形式的博客帖子和讨论都是受欢迎的。

我们的成员写帖子，开始讨论，并建立他们的专业档案。当我们开源时，我会提醒人们我们是一家盈利公司，但我可以向你们保证，我们主要是由构建一个包容的和令人敬畏的软件生态系统的价值观所驱动的。我相信每个人都这么说，但我认为我们把它戴在袖子上了——开源增加的透明度只会有助于随着我们的发展而提高这种责任。

我们是一个 [Ruby on Rails](https://github.com/rails/rails) 应用，前端有 [Preact](https://github.com/developit/preact) 。我们还将推出本地应用程序([🍏](https://github.com/thepracticaldev/DEV-ios)，[🤖](https://github.com/thepracticaldev/DEV-Android))。技术选择将随着时间的推移而演变。我们远不是任何一种风格的信徒，虽然我们不会在一夜之间改变太多，但我们会鼓励健康的讨论和辩论。我们也非常依赖一些外部服务，但随着我们的发展，这些都需要改变。这些讨论将会有一半的乐趣！

核心项目是在 Affero GPL 3.0 的许可下进行的，这是一个类似项目使用的版权许可。在我们未来的任何时候建立或提取的任何图书馆都将在麻省理工学院下可用。

最后，我想提醒大家，我们对自己的[行为准则](https://dev.to/code-of-conduct)非常认真，我们希望每个人都能尊重所有相关人员。
