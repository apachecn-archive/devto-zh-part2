### [](#introduce-yourself-who-are-you-where-do-you-work)自我介绍！你是谁？你在哪里工作？

嗨，我是 Johna*，我是 Credera 的顾问。

我在阿拉斯加出生并长大，后来搬到德克萨斯州，在南方卫理公会大学学习计算机科学、心理学、俄语和中文，在大四前的那个夏天，我通过招聘会和实习意外地找到了我梦寐以求的工作。我只参加过一次发展面试，那是在 Credera 实习时——自 2016 年毕业以来，我一直在那里全职工作。

Credera 是一家全方位服务的咨询公司，从战略管理咨询到技术解决方案，无所不包。我是一名技术顾问，这意味着我的大部分时间都是在协作的团队氛围中为我们的客户进行定制软件开发。

*它的发音是 John——呃，在现实生活中大多数人都弄错了，这就是为什么我在网上用 Jones。

### 是谁或什么让你开始编程的？

我的父母总是鼓励我尝试所有的活动——从油画到植物学到选美。在我高中二年级后，他们给我在当地大学报了一个为期三周的工程课程，包括建造自主机器人，并给它们编程，让它们互相战斗并通过迷宫。

在一个 60 人的班级里，我是三个女孩之一，和我学校的一群男生被分在一个队里。我团队里的人对机器人的硬件很着迷，我没有办法接近它，所以我开始为比赛写程序。第一周，这些人不使用我的代码，但我注意到其他一些团队在努力，所以我开始为他们的团队编写代码，他们开始赢得比赛。到营地结束时，我的团队已经在使用我的代码了，我成了任何需要帮助的团队的雇佣兵；我发现编码对我来说是一种特别的宣泄(而且我非常擅长)。我对机器人没什么好感，但从那时起，我知道我想成为一名开发人员。

### 你似乎很重视工作/生活/爱好的平衡，为什么你认为这很重要？

我的爱好之一是与陌生人交谈，我最常听到的观点是，“我希望我仍然[插入活动]”——这种空白可以用任何东西来填充，从演奏乐器到与他们的家人建立更密切的关系。当你忙的时候，很容易忘记对你来说重要的事情，把它们扔掉，并认为你以后会再捡起来。不幸的是，许多“重要”的东西不是十年或二十年后你可以随便“捡起来”的:你的孩子会长大并搬出去，你的父母会退休，你对外语的听力下降，你的体能下降，嗯，你明白了。每周花一个小时去适应你的爱好并不困难，你只需要有意识地去做。

我认为良好的平衡对防止倦怠也很重要。平衡并不一定意味着在每件事情上花费相同的时间，但它确实意味着在工作接管或者生活接管你的职业道路的繁忙季节，你可以依靠生活的其他领域来帮助你减压。

### 在 Instagram 上保持活跃对你的职业生涯有什么帮助？它有什么缺点吗？

在 Credera 内部，这是打破老板-员工对话障碍的一个很好的方法，除了成为一名优秀的开发人员之外，还能获得其他方面的认可。我永远不会忘记有一天，我们的首席用户体验主管在走廊上拦住我，告诉我我的 Instagram 内容“非常有效”——我不认为我们在那一刻之前说过话，我有点星光熠熠，他甚至知道我是谁。

这也是一个很好的招聘工具，我有不止一个学生在招聘会上找到我，说他们听说过 Credera，因为他们在 Instagram 上关注我。这最终让我看起来很好，因为虽然商科学生往往了解咨询，但计算机科学学生往往不了解(尽管这是我们工作的很大一部分)——潜在的开发人员找到我们，已经对我们的工作有了现实的想法，并希望为我们工作，这是一个很大的优势。

缺点与任何类型的内容创建平台相同，主要涉及持续创建内容所需的时间和精力。在有意这样做了将近一年之后，我确实有了“照片疲劳症”,似乎没有任何可能的主题/图片没有被报道过。

与工作相关的最大挑战是谈论我的工作，同时还要为客户保密——这不一定是一个缺点，但它增加了额外的偏执，我必须仔细检查我所有的帖子，以确保任何识别信息都被删除。

### [](#what-do-you-think-the-future-of-social-media-engagement-is-for-developers-where-should-we-focus-our-efforts)你认为对开发者来说，社交媒体参与的未来是什么样的？应该把精力放在哪里？

我不认为在社交媒体上有一个放之四海而皆准的标准，人们应该发挥自己的优势。我一直喜欢摄影，所以 Instagram 作为一个社区建设平台对我来说很有意义。如果你喜欢采访人，制作一个播客或博客，并在其中加入时事通讯选项。如果你喜欢研究/分享链接，看看 Twitter 和 LinkedIn。如果你想教人们如何做事，去 Youtube 上制作一些可爱的 DIY 视频(或者启动一个 GitHub 项目并寻求贡献者)。

如果你不喜欢创作内容，那就积极参与你喜欢的任何平台。一个人在网上发帖并不能创建一个社区，回复内容的人才能创建。

让社交媒体如此酷的不是“媒体”部分，而是它的“社交”部分。开一个技术博客来展示你的专业知识是很棒的，但是如果没有人阅读(或者想要阅读)，那么你就错过了与人交流的机会。

### 迄今为止，在你的软件生涯中，你学到的最难的一课是什么？

学会在一天结束时放下工作。

大学课程倾向于以任务为基础:你有一周的时间做一个项目，24/7 工作(可能会熬几个通宵)来完成它，然后直到下一个项目。然而，现实世界是一场马拉松。总会有另一个 bug，如果你过度劳累，熬了几个通宵，你不会有四个月的假期来恢复。

### 对于成功的软件职业生涯，你的第一条建议是什么？

想办法在你完全不感兴趣的事情中找到激情。

无论你是自由职业者还是在工业界，总会有这样的时候，人们让你做你不感兴趣的事情，但为了支付账单或保住工作，你不得不答应。如果你能在这些项目中找到乐趣，从长远来看，你会更加快乐。

### 工作之外，你有什么爱好吗？你认为它们对你的科技事业有任何帮助吗？

哦，是的，一百次是的——有些人比其他人以更间接的方式提供帮助。社交游戏化是我目前最耗时的爱好:我找到一个有公开排名指标的系统(像亚马逊评论、Yelp、Instagram 等。)并使用数据驱动的方法，我试图弄清楚算法是如何工作的，它背后的心理(如果有人类互动的话)，它需要多长时间，以及其他人如何才能重现相同的结果。这不仅是一次很棒的鸡尾酒会，还把我引入了一个我甚至不知道存在的技术社区。

我的其他一些爱好包括人像摄影、阅读、宠物看护、交际舞(我在大学时跳过交际舞)、烹饪、听播客、园艺和特色咖啡。我的男朋友是一名演员，所以我们也花很多时间去看戏剧、阅读和制作试听带。开发人员仍然因为粘在我们的电脑上而成为笨拙的交流者而受到很坏的指责，所以我认为任何帮助你学会以一种与技术无关的方式与他人相处的东西都可以像技术敏锐度一样帮助你的技术职业。

### [](#whats-the-difference-between-being-a-technology-consultant-and-working-as-a-software-developer)做技术顾问和做软件开发有什么区别？

当公司想要招聘软件开发人员时，他们通常会考虑特定技术的技能组合——比如 Java 开发人员、前端开发人员、Ruby on Rails 的开发人员等等。除非你被一家小公司雇佣，在那里你最终会做所有的工作(或者是一个独立的自由职业者)，否则作为一名软件开发人员，你很有可能最终会在一个非常特定的领域工作，并且很好地学习一些语言或技术。

作为一名技术顾问，你(理论上)必须能够胜任任何技术。这意味着你快速学习和与客户交流的能力比你对最新版本 Java 的复杂知识更重要。是的，你绝对应该努力对你想专攻的技术有一个更深的理解，但是你必须准备好接受一个使用你以前从未接触过的技术栈的项目，并且迅速弄清楚如何足够好地使用它来创建生产质量的软件。

### [](#what-booksresources-would-you-recommend)你会推荐什么书/资源？

[裸体](http://amzn.to/2kLov84)是 Credera 推荐给新员工的第一本书，也是对咨询感兴趣但还不知道咨询是什么的人的绝佳介绍。

Fred Brooks 的《人月神话》无疑是我最喜欢的面向开发的项目管理书籍，它似乎是那种从来不阅读材料，但每个人都以某种方式独立阅读的书。

从高中开始，我就在宿舍和公寓里随身带着查尔斯·佩佐德的代码。它涵盖了逻辑，编号系统，开关和微处理器的高水平。

Brian Christian 的《最有人情味的人类》也是图灵测试和人工智能概念的伟大读物——它是让我意识到以人为中心的发展的重要性的第一批书籍之一。

[回复所有人](https://gimletmedia.com/reply-all/about)是一个令人愉快的科技导向播客。这几集是在现实生活中调试技术问题的好例子(网络钓鱼、奇怪的电话、网络社区中人们的故事)。
