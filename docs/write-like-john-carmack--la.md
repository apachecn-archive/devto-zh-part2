# 像约翰·卡马克一样写作

> 原文：<https://dev.to/ben/write-like-john-carmack--la>

约翰·卡马克，世界上最富传奇色彩的软件开发人员之一，[在《脸书邮报》上发表了一篇关于他最近参加的代码务虚会的文章。你知道我喜欢它什么吗？它读起来像脸书邮报。它没有被过度抛光。它不是专门为任何目标人群写的，读起来像是写给朋友的便条。](https://www.facebook.com/permalink.php?story_fbid=2110408722526967&id=100006735798590)

尽管这本书很大程度上超出了我的理解范围，但它仍然是一本令人愉快的读物。

> 作为对我目前 Oculus 工作的一种改变，我想从零开始编写一些 C++神经网络实现，并且我想用一个严格基于 OpenBSD 的系统来完成。有人评论说这是一个相当随机的配对，但结果还不错。

约翰·卡马克是我们这个领域的传奇人物，但我相信这种程度的非正式对于像你我这样的凡人来说是可能的。这是一种健康的分享方式。冒名顶替综合症让分享你的真实经历成为一项相当艰巨的任务。没关系。这正是我职业生涯大部分时间的感受。但是如果你有勇气的话，我肯定会建议你分享，就像几个没有偏见的朋友一样。

我能说的是，如果你真的分享了 [dev.to](https://dev.to/) 的内容，我们会努力不让那些混蛋毁了你的一天。但这并不是为我们的社区做宣传，只是一个关于我和我的团队将在哪里提供个人帮助的说明。

我对卡马克的个人生活知之甚少，除了从他的写作风格中汲取灵感之外，这并不是对任何东西的认可。非正式推动我们的行业向前发展。实用主义和天马行空的梦想都是从自然对话中诞生的，而不一定是从华丽的展示中诞生的。

最后一点:你的工作有足够的空间来润色，每个人都可以自由选择方法，包括永远不写博客！但是，如果你从来没有想到像这样给软件朋友写一封短信，考虑简单地给朋友写一封短信，也许没有人会读。

值得一提的是，我觉得 Twitter 并不理想，因为环境就是这样一场人气竞赛。其形式因素的限制也不一定有利于自由流动的表达。我经常使用 Twitter，但不一定像其他人那样是为了这个目的。

快乐的编码。〈希莫丝〉
