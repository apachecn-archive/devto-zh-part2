# 如何获得支持以解决“破坏性”问题

> 原文:[https://dev . to/jaymeedwards/how-to-get-support-fix-a-disruptive-problem-13j 8](https://dev.to/jaymeedwards/how-to-get-support-to-fix-a-disruptive-problem-13j8)

软件项目一天比一天复杂。

你比以往任何时候都更容易被某个问题所阻碍。

我曾多次遇到过这种情况，我不得不向我的老板或经理解释我遇到的问题。

当你被封锁的时候已经够糟糕了，直到你解决了问题。

但是当你意识到解决问题*可能会阻碍其他人完成工作*时，情况会更糟！

当这种情况发生时，我的老板有时会假装没有问题。

如果你的软件项目压力很大，阻止多人来解决一个问题对管理层来说是不可能的。

尤其是当经理们关注人们完成了多少工作的时候。

我在[另一个视频](https://www.youtube.com/watch?v=0yxfb-drlE4&index=3&list=PL32pD389V8xt_znF-pvOl7OP_xNfnkGgq)中与你讨论了关注一个团队完成了多少工作*实际上会损害一个产品为你的公司赚了多少钱*。

因此，在这段视频中，我想分享一些让你的经理或老板支持你的方法——即使你需要*扰乱他人来解决问题*。
