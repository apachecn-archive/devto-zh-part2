# 不要憎恨

> 原文:[https://dev.to/jdsteinhauser/dont-be-hatin-4387](https://dev.to/jdsteinhauser/dont-be-hatin-4387)

"我讨厌做傻事."

多年来，我设计了数据系统、算法和高效的数据处理流程。我觉得很舒服。我知道如何解决这些问题。但是我真的把自己归类为故意避免任何类型的 UI 工作。我会开发一个快速分析脚本，然后拼凑一个 UI，这样其他人就可以在没有我的情况下完成这项工作。但是事情是这样的:用户会抱怨需要我做额外工作的事情。

*   "我找不到让东西运转的按钮。"
*   "我不小心在田里划了一条斜线，结果整个事情都搞砸了."
*   “哦，伙计，如果你能让这个工具做 **x** 、 **y** 和 **z** 就好了。我知道 z 真的很难，但是它会节省我几分钟的时间。就像 4 行代码一样，对吗？”

呃，用户。如果没有他们疯狂的想法，软件会更容易开发。

这就是我过去*年*的态度。我像躲避瘟疫一样躲避 UI 工作或者前女友。因为后见之明都是 20/20 之类的东西，我现在知道**我限制了自己**没有花时间*理解*为什么用户会有问题。原料药的消费者？我已经控制住了。图形界面的用户？没有那么多。正因为如此，我也在近十年的时间里避开了任何网络开发。

## [](#consequences-of-selflimitation)自我限制的后果

我的妻子和我正在开发一个现场逃生游戏(这是其他几个帖子的故事)，我们决定为房间和游戏主人推出我们自己的用户界面。我当时决定，这是一个好主意，刷掉我的网站开发技能的蜘蛛网。我是说，说真的，web dev 在 12 年里改变了多少？

那是在我遇到巴别之前。做出反应。JSX。节点。Websockets。谢天谢地，没有那么多要补上的！

我们完成 UI 的时间很紧。我有大约两个月的时间来做一些事情...同时做另一份全职工作，花时间和孩子们在一起，自己动手建造逃生室，雇人...那是忙碌的几个月！我不得不赶上这么多科技项目，这也没什么帮助。我中途实现了几个东西，拼凑出了一些“有效”的 ui。

...你知道，如果我足够关心用户体验并跟上 web 开发的步伐，我会开发出更漂亮、更有用、必要时更容易修改的东西。*和*我会用更少的时间来交付它。为什么这很重要？

*   我会花更多的时间和家人在一起。整个工作/生活平衡只是在*而不是*那里。一天，当我在商店埋头苦干时，我妻子给我带了些晚餐，还带着我们最小的女儿。当我女儿拥抱我的时候，我泪流满面，因为她拥抱我的方式就好像我已经离开城镇一周了。
*   这影响了我的日常工作。无数个小时的“副业”工作意味着我要努力保持清醒，专注于我的主要收入来源。
*   这可不是什么好代码。我花了很多时间进行堆栈溢出驱动的开发，以获得一些实用的东西。我之前没有公开这个项目，因为我们打算在事情“尘埃落定”后出售我们的逃生室软件。我现在不公开它，因为它是令人尴尬的代码。我敢肯定，如果我再次进入就业市场，它会拒绝未来的雇主。它仍然藏在我自己的私人回购中，提醒我不要做什么。

TL；DR - I 花了过多的时间开发令人尴尬的代码，同时对我的工作和(更重要的是)我的家庭生活产生了负面影响。这都是因为我让我的 web 开发技能烂掉了，主要是因为我甚至厌恶做 UI 开发的想法。

## [](#more-consequences)更多后果

在我们开设了第二个密室后不久，我换了家公司，我加入了一个为政府“字母机构”更新软件的项目。其他一些人在我之前加入了这个项目，并致力于许多底层算法。负责这个项目的人问我“你为什么不试试开发我们的配置 UI？”就在那时，我又说了一遍。“我讨厌做 ui。”

在这个项目中，虽然我和我的领导意见相左，但他说了一些我以前没有考虑到的深刻的东西。**“别再说了。这会让你看起来不像一个团队成员。”**我知道拓展业务和学习新事物对我有好处，但我没有想过这会如何影响团队动态。为了与他人合作愉快，你不能像那样设置人为的障碍。*但我们都这么做了！当然，总有一些事情是你不想做的。我并不特别喜欢在家里倒垃圾，但我对此并不抱怨。这是必须完成的事情，这样我们才能继续做我们*喜欢做的事情。**

## [](#what-did-we-learn)我们学到了什么？

我妻子决定她想开始在工作中制作分析仪表板，于是她开始学习 [D3](https://d3js.org) ，以及专注于 ui 的 web 开发。她特别拿起了一本书，引起了我的注意:[为头脑设计](https://www.manning.com/books/design-for-the-mind)。我相信也有其他推荐的好的 UI 开发读物，但是我把过去的感觉放在一边，拿起了这本书...我着迷于 UI 设计背后的心理学。进行用户访谈，看看你是否正在构建一个对他们的生活或工作流程有益的工具。以这样一种方式构造用户界面，引导用户走上“快乐之路”各种我以前从未考虑过的事情！我现在实际上发现 UI 工作是一种享受。我还是更喜欢开发算法吗？当然可以，是的！但是现在我明白了*ui 应该如何构建，这使得任何可视化的开发变得实际上可以理解和令人满意。*

## [](#conclusion)结论

你可能不喜欢做某事，这没关系。但是向你的同事抱怨呢？这对你的工作环境不利，对团队动态也有负面影响。放弃那个领域的任何工作，因为你不了解它，也不想了解它？嗯，当你不得不做这件事的时候，它会花费很长时间，并且可能会打破你的生活/工作平衡。想办法享受吧。停止“憎恨”那些对你的职业或生活有积极影响的事情，想办法让它变得愉快。

就像我之前说的，我不太喜欢倒垃圾。然而，在我回去之前，我会停下来欣赏夜空，在我的前廊呼吸一些新鲜空气。我很期待。
