# 是时候结束 vi/Emacs 之争了

> 原文：<https://dev.to/realedwintorres/time-to-end-the-viemacs-debate-343i>

*我最初在博客@CACM 上发布了这篇[文章](https://cacm.acm.org/blogs/blog-cacm/226034-time-to-end-the-vi-emacs-debate/fulltext)。*

程序员之间有一个由来已久的争论。这是激烈的哈特菲尔德/麦科伊，喷气机/鲨鱼，和密歇根州/俄亥俄州的世仇。程序员们继续争论最好的文本编辑器: *[vi](https://en.wikipedia.org/wiki/Vi)* 或者 *[Emacs](https://en.wikipedia.org/wiki/Emacs)* 。在过去的 42 年里，程序员们已经[在](http://www.slate.com/articles/technology/bitwise/2014/05/oldest_software_rivalry_emacs_and_vi_two_text_editors_used_by_programmers.html)的任何一方扎根，几乎没有希望转换。我必须公开我从 1987 年开始使用 vi。还记得同学们任意挑 vi 或者 Emacs。我当时试过 emacs，但不喜欢它。也许我没有给它一个公平的机会？现在已经太晚了。我终身在 vi 营。

现在开发商发话了。Stack Overflow 刚刚发布了其 [2018 开发者调查结果](https://insights.stackoverflow.com/survey/2018/?utm_source=Iterable&utm_medium=email&utm_campaign=dev-survey-2018-promotion)。vi 的改进版本 Vim 的受欢迎程度为 25.8%，而 Emacs 的受欢迎程度仅为 4.1%。这是一个惊人的差异。为了进一步衡量受欢迎程度，我统计了 22，582 个用 vim 标记的问题和 15，667 个用 Emacs 标记的堆栈溢出问题。你可能会说问题越少意味着可用性越高。我把这解释为活动的迹象。

我知道 Atom 和 Sublime 文本编辑器现在非常流行。但是 30 年前让 vi 吸引我的东西今天仍然适用。我真的很喜欢这些简短的命令。我很少需要把手从键盘上拿开。要删除字符，请键入“x”。要插入字符，请键入“I”。如果你仍然按 Ctrl+x，Ctrl+s 来保存文件，我想我们不能成为朋友。我喜欢我的大脑被硬连线到 vi 的方式。我可能无法向您描述大多数 vi 命令，因为使用 vi 就像呼吸一样:您只需做就行了。麻省理工学院和斯坦福大学[都有漂亮的小 vi 参考卡，可以放在一页纸上。](https://pangea.stanford.edu/computing/unix/editing/viquickref.pdf) [GNU Emacs](https://www.gnu.org/software/emacs/refcards/pdf/refcard.pdf) 参考卡长两倍。调用 emacs 编辑器需要三个额外的字母；随着时间的推移，这些额外的击键次数越来越多。最后，vi 是个很酷的名字。我有一个叫 Vi 的表弟。你见过叫 Emacs 的人吗？

如果你还在用 Emacs，我同情你。也许是时候让 vi 试试了。我认为我很幸运从一开始就选择了 vi。您可以尝试说服我使用 Emacs，但是说服我回到 Visual Basic 会更容易。
