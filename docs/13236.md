# 解决 IT 中的代际转换

人们担心，劳动力整合、流失、退休、招聘冻结以及新 It 员工对基础架构元素的兴趣降低，可能会导致许多依赖大型机关键业务系统的公司出现技能短缺。幸运的是，你不需要让代际劳动力技能的转变让你措手不及。你现在可以采取一些行动来充实你的员工和人才库。

Adabas & Natural 的一些客户已经找到了如何通过成功管理团队内部人才的代际变化来保持其核心应用程序长期运行的方法，他们吸引并留住年轻的开发人员，并让所有代人一起工作，分享技能和知识，并进行创新。以下是他们成功管理员工代际变化的秘诀。

## 发现新的人才

首先，必须引进新的人才来替代退休的劳动力。寻找渴望通过在现代环境中工作来有所作为并实现自我实现的个人——是的，我说的是大型机和遗留系统环境中的现代环境。随着开源 Eclipse 平台的迅速采用，几乎每种语言和架构都有了集成开发环境(IDE)和平台，包括 Java IDE、C/C++和第四代语言(4GL) Natural。这些可扩展的平台拥有丰富的附加工具，可用于创建桌面、web 和云应用程序。

美国武装部队互助协会(AAFMAA)的首席信息官 Amarish Pathak 注意到，随着基于 Eclipse 的 Natural IDE Natural one 的推出，征兵成功率有所提高。“NaturalONE 已经为 AAFMAA 大学一毕业就招募开发人员打开了大门，”他说。“由于大多数大学使用 Eclipse 教学，学生们已经了解了 NaturalONE 环境。我们已经能够充实我们的团队，以提供长期的连续性。”

有了有影响力的领导和同级培训，你可以期待新员工快速提升学习曲线。Alte Oldenburger 的软件架构师 Stefan Macke 曾说过一句非常有用的话:“要管理员工队伍的代际变化，你需要一个有个性的领导者来激发创造力；热情、有说服力、有能力带动他人的人。”

虽然找到已经熟练的程序员可能是可取的，但找到适合你的团队文化并愿意学习的人可能更有益。如今，发展是一项团队运动，因此找到适合你的组织的人并亲自培训他们就变得更加重要。

## 自己培养新人

不要担心寻找已经知道你的关键任务应用程序背后的编程语言的人——学习很容易。随着软件供应商提供的在线教育服务的激增，培训自己人才的能力比以往任何时候都更便宜，需要的旅行也更少。用创造性的导师计划来补充这一点，你的新人才也将了解你所在的行业和组织的细微差别，从而使你在市场中具有竞争力。

学徒制在欧洲非常流行，用来培训新一代的开发人员。无论是教实习生，还是招收勤工俭学学生，如果你明确承诺学徒/实习生将获得长期就业，你将开发出一个有前途的招聘渠道，具有经过检验的真实前景。

确保所有开发人员都参与到游戏中，强制要求每个开发人员都必须了解 it 基础设施中支持的每种语言。这将有助于消除同事之间争夺被认为是“最酷”项目的职位。它还将确保所有开发人员都能够熟练地处理您的所有应用程序，让管理人员能够更自由地按需转移资源。新员工越明白编程语言只是达到目的的一种手段；他们就越不会被自己使用的“语言”所困扰，而会专注于更重要的解决问题的技能。

一家著名的欧洲保险公司吸引并留住了年轻的开发人员——平均年龄为 34 岁，其中一半是 20 多岁的年轻人——通过一个强大的培训项目，教授 C#、Ruby、Java 以及主要是自然语言，即其核心系统的语言。该公司 20 名开发人员中有 8 名是在过去 5 年中加入的。

## 留住你的新员工

既然你已经在招聘和培训上投资了，留住你的新员工是最重要的。给年轻的开发人员开发新想法的机会，并向更多的观众展示——让他们拥有自下而上的战略影响力，并有机会塑造这种影响力。对新的想法保持开放，也允许创造力蓬勃发展。这可能需要你和你的员工改变心态。当我们在同一个环境中待了很长时间后，我们经常会陷入“那是以前试过的”或“那不是我们这里的做法”。虽然一个新想法可能行得通，也可能行不通，但它会让你的新员工有能力去寻求解决方案，发挥创造力，并感到被赏识。知道自己在团队中有发言权的新员工更有可能留下来。

确保每个员工都参与传统项目和创新项目。只有当您拥有现代化的开发运维方法、基于下一代架构的现代化基础设施的工具以及有助于业务战略的优先项目时，您才能吸引和留住新人才。提供最先进的敏捷开发工具和方法。许多软件供应商目前提供现代 DevOps 工具和最佳实践，可以很容易地加以利用。

即使对于已经存在了 50 多年的企业来说，也不需要对如何做生意有什么陈旧的东西。通过使用现代化技术，隐藏在核心系统中的逻辑可以被解锁并用于移动设备、web、云和物联网(IoT)上的新应用。

## 把几代人聚集在一起

最后，培养一种企业文化，尊重老一代和年轻一代为共同目标做出的贡献。这可以通过实施促进各代人积极参与的计划来实现。

引入指导计划，帮助促进机构知识和技能的传承。您的运营面临的最大威胁之一是，您的应用程序主题专家在将他们的知识记录下来并传授给未来负责维护和更新您的业务关键型应用程序的员工之前就退休了。你也可以改变指导模式，让新员工帮助你的有经验的员工学习新的发展流程和工具。

一家欧洲公司的新员工证实:“向老同事学习对我帮助很大，尤其是在刚开始的时候。他们不仅在医疗保险部门的事务上支持我，而且在我熟悉 Natural 时，每位同事都随时准备帮助我并回答我的问题。”

我们的客户分享的另一个很好的建议是实现结对编程——一种敏捷软件开发技术，其中两个程序员在一个工作站一起工作。两个开发人员在相同的设计、算法、代码或测试上合作——经常交换导航员和驾驶员的角色。您的新开发人员可能已经熟悉了这种流行的方法。

“最近我有机会向 IT 部门展示一个配对程序，”一家欧洲公司的新员工说道。“团队很喜欢它。我喜欢作为一名实习生，我有自己的声音，我的同事会听我的。结对编程引发了学员和其他团队成员之间的活跃对话。这真的帮助了两代人作为一个团队一起工作。”

当新员工被给予尊重新想法以及发挥影响力和创造力的机会时，你将会在建立长期的员工队伍方面大有作为。

## 行动中的菜谱

Josh 是美国国家机构的千禧一代程序员，他可以直接讲述现代平台如何简化开发、测试和集成，使支持大型机系统比以往任何时候都更容易。当这位程序员开始在该机构工作时，为了学习如何支持大型机，他有一个很大的学习曲线要爬。正如乔希指出的，“如果有问题，你只是不能谷歌的答案。我不得不依靠身边经验丰富的开发人员来弥合差距。”幸运的是，由于他的热情和学习意愿，他得到了很好的指导。这位千禧一代程序员也很好地融入了公司文化，并与经验丰富的同事建立了相互尊重的关系。

多亏了现在运行在“新”产品上的“旧”产品，比如 Eclipse，开发、测试和操作大型机的每一项任务都变得更加容易完成。开发人员不再需要编写详细的程序来测试缓冲区和输入/输出功能。Eclipse 和其他现代平台通过内置函数处理所有这些小细节。

Josh 希望该机构雇佣新的程序员来支持其核心系统，并认为招募、培训和留住这些程序员并不困难。“由于所有这些新工具，新 IT 专业人员的学习曲线不会像我一样陡峭，”他反思道。

今天，这位千禧一代程序员看到了一条有利可图的职业道路，那就是留在培训他的组织中。他不仅觉得自己的技能比他在其他公司写 Java 的朋友得到了更好的补偿，而且他看到了工作稳定和令人满意的职业道路的未来。

显然，这也是该机构的一个胜利，它现在确保了他们的任务关键型应用程序将在未来的业务中继续发挥重要作用。网络、移动、大数据、云和物联网带来的新的数字机遇也可以加以利用，因为该机构已经获得、培训和保留了解决 IT 代际过渡所需的人才。
