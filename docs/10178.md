# 随处可见的应用程序！

> 原文:[https://dev.to/satansly/apps-apps-everywhere-3kp0](https://dev.to/satansly/apps-apps-everywhere-3kp0)

[![Apps Meme](../Images/cf0e9ff71743b123914df8ee140fd9f7.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--nm15necQ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ao36ym8eeo3muxhila24.png) 当人们认为自己提出了一个将改变世界的想法时，他们应该问自己的问题是，我需要一个应用程序甚至一个网站来验证这个想法吗？

用技术和设计精美的 UX 来吸引用户是很常见的。但是这导致了几个小时的计划、设计、构建、测试，然后在一天结束时，可能会失败。基于自己的假设和直觉的创业想法有可能卖不出去，而且只局限于一个人或一群人。这个想法需要尽快得到验证。

为了验证一个想法，需要尽可能早地在一小群人中进行测试，你会问，你需要一个应用程序吗？可能不是。如果你的创业想法的核心并不依赖于技术(例如，使用 p2p 网络或视频通话的服务)，并且可以在不构建任何东西的情况下进行测试，那么首先测试这个想法。

如果你正致力于一个革新向顾客交付商品的想法，在小范围内，你不需要一个应用程序。你所需要做的就是通过合适的渠道(小册子、社交媒体、口头宣传)营销你的服务，并通过电话预订和送货上门。如果你能让几个真正的顾客(不是朋友和家人)为你的服务付费，并得到积极的反馈，你手头就有了一个好主意。现在，您可以开始考虑扩展和自动化您业务中最普通的任务。

一旦一个想法得到验证，就建立你的第一个 MVP。MVP 最初需要专注于解决一个问题。你最初开始用你的想法去解决的问题。你可能仍然不需要一个应用程序。只有当应用程序需要特定于移动设备的功能时，您才需要它。网络应用的覆盖面更广，在电脑上的易用性超过了在移动设备上使用服务的体验。许多有用的信息可以放在一个网页上，并且可以减少许多导航点击。

总结这个小声明和总结，专注于用最少的精力验证一个想法，只有当你有了初步的动力并准备扩大规模时，才建立一个网站或应用程序。
