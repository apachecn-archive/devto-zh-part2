# 肯特·C·多兹给新手的 3 条建议

> 原文：<https://dev.to/swyx/3-tips-from-kent-c-dodds-for-people-just-getting-started-ik8>

我最近在 [6 Figure Dev 播客](https://6figuredev.com/podcast/episode-061-react-with-kent-c-dodds/)上听到了这个，并不得不分享伟大的 KCD 的笔记:

*   **软件很硬**。如果你在挣扎，那是因为这不容易。忽略那些告诉你这很容易的人。它是不断变化的，努力提高和跟上是正常的。
*   进行基于动机的学习。你不必先学习所有的基础知识。了解你需要什么来建造你想要的东西。当事情变得困难时，这是推动你度过难关的动力。当你对框架源代码等感到好奇时，再深入研究。
*   你将通过建造来学习新的东西——教他们。如果有人说过什么，不要担心。你学到的东西对这个世界来说不会是新的，但对你来说会是**新的**，我向你保证世界上还有其他人对它也是新的。**通过博客帖子或聚会交谈，边教边学**。您可以接触到当前内容无法接触到的人。比如非英语的听众，或者更近期的初学者(高级工程师经常被“知识的诅咒”所折磨，再也无法有效地向初学者解释事情)。

如果你想知道更多关于他如何提升自己职业生涯的故事，我推荐这个演讲:[软件开发零到 60 岁:如何快速启动你的职业生涯——Forward 4 Web Summit](https://www.youtube.com/watch?v=-qPh6I2hfjw&list=PLV5CVI1eNcJgNqzNwcs4UKrlJdhfDjshf)

就是这样。只是不得不分享。去做吧。
