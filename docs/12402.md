# 专心做正确的事情

> 原文:[https://dev . to/noquierouser/dipping-into-something-done-rightly-1c 23](https://dev.to/noquierouser/dipping-into-something-done-properly-1c23)

几周前，我做了一个我经常做的决定:仅仅为了尝试新事物而尝试。实际上，它并不那么经常，也绝对不是为了它而去*，尽管如此，我还是做了，我喜欢它，我想继续下去。不管怎样，好奇心不就是每个程序员曾经的燃料吗？*

 *## 《T2》前情提要 …

我有一个 IT 技术学位——或者类似的学位，职业命名在各国都很奇怪——在那里我学到的大多是老技术，除了一件永远不会过时的事情:**如何学习**。这就是我如何在学习期间接触到许多编程语言，直到我毕业，然后…嗯，找工作并不容易，特别是当你积极避免在贵国唯一的——非常臃肿和拥挤的——大城市现场工作的时候。但是我跑题了。

在我的职业生涯中，作为一项持续了几周的任务，我与 C#和 ASP.NET 有过一段时间的接触，没有什么特别难忘或壮观的，我有点后悔这样做，因为不知何故我期待着与这些技术打交道，特别是 C#。当我还是个孩子的时候，我就希望有一天我能做一个 Windows 应用程序，不管它有多小，多不相关，所以有这样一个不起眼的任务是很平淡无奇的，顺便说一下，我成功了。

然而，在那种情况多年后，我决定再试一次，重新点燃我与 C#和 Windows 应用程序开发的关系。到目前为止，我开发的主要是基于 Wordpress 的网络应用，所以我必须为*一顿难忘的蒸文档午餐做好准备……*几周前，当我决定再次涉足 C#并尝试完成一些事情，而不仅仅是*一个 Hello World* 的时候。

## [](#the-combat-scenario)战斗场景

简而言之:我在家乡的一个非营利性社区电视网工作，担任首席技术人员和 it 支持。我的任务很多，因为我们是一个非常小的团队，试图完成电视网几乎所有可以想象的任务。在这些众多任务中，**日常编程**是最重要的任务之一。每天 17:45 该频道的节目开始，不同的节目在节目运行期间的特定时间开始。基本上，**一个播放列表**。

现在，由于我们的资源稀缺，我们几乎没有专门的设备来进行这项工作，所以节目是在一台装有 6 年前你能买到的最强大的显卡的电脑上运行的，使用 T2 的 vMix 作为我们的播放和混音软件。到目前为止，一切顺利。

但我有一个很大的不便:当我制作播放列表时，我不知道这些节目什么时候播放，什么时候结束。我只知道播放列表在 17:45 开始，节目持续 12 秒到 2 个半小时，但我无法知道一个节目是在 18:00 还是 19:30 播出，还是在 20:30 或 20:34 结束。必须做点什么。

一开始，我用的是一个在线时间计算器，这在当时是有效的，但是我的思想分裂了，不知道下一个节目是什么，也不知道在下一个节目播出之前我还有多少时间。这是一个缓慢而繁琐的过程，我没有办法向别人解释。

但是，作为一个程序员，我知道我可以用一个小程序解决这个问题…

## [](#a-cild-was-born)一个 c#ild 诞生了

我知道我不能用另一个 Wordpress 插件来解决这个问题，所以我认为现在是时候重新审视 C#和它的力量来一劳永逸地解决这个问题了。因此，我去咀嚼一些文件，用一个简单的 Hello World 重新获得一些信心，并带来了将拯救我们所有人的圣杯。

但是面对一种有自己做事方式的编程语言，这是一个很难克服的学习曲线。主要是因为，作为一名 Wordpress 开发者，我以前从来没有如此繁重地处理过类。或者强类型，因为 C#和 PHP 是非常不同的怪物。与 HTML 和 CSS 相比，即使是 XAML 也有很大的不同……所以我在那里，从下面看着这个巨大的怪物，用智能感知作为我信任的副驾驶，谷歌搜索我脑海中闪过的几乎每一个想法。

最终，在黑进了 Big Shell 之后，警报阶段开始了，我走出了阿森纳装备，胜利了: [vMixListMaker 诞生了](https://github.com/noquierouser/vMixListMaker)。现在，投入生产 3 周了，我作为一名程序员的生活比以前轻松多了。

## [](#the-part-where-the-main-title-makes-sense)主标题有意义的部分

正如之前在 dev.to 上多次被告知的那样，*工作代码比没有代码好*。但是现在代码工作了，那么，是时候改进它了。我的代码是 *hacky* ，边界*垃圾*，但它需要改进，主要是因为我想给现在看起来像*小而不相关的* Windows 应用程序添加功能。换句话说，我在寻求帮助。

在开发这个小应用程序时，我看到了许多关于 MVVM(模型-视图-视图模型)及其奇迹的文章，关于测试驱动开发是当今最酷的开发方式，以及许多人仍然在他们的 vMix 平台上使用 Windows 7，并且无法运行该应用程序，因为它是一个通用的 Windows 应用程序。

所以我寻求帮助，不是为了拉请求或其他什么，**而是为了给我指明正确的方向**。到目前为止，我已经了解了一些关于如何用 C#和 UWP 编程的事情，但我想了解更多关于如何实现 MVVM 解决方案，如何使用 TDD 进行开发，如何从我的应用程序中分离一些逻辑，并使其可用于同一工具的两个不同版本，这样我就可以开发一个与当前 UWP 版本并行的 Windows 7 WPF 应用程序。

[查看我的 GitHub repo 中的代码](https://github.com/noquierouser/vMixListMaker)。我乐于接受关于如何改进我的代码、最佳实践和库的建议。什么都欢迎。

我已经看过很多理论文章，但现在我需要看到例子、练习、演练和文档。想学习用 C#适当开发，最佳实践和推荐。因为一旦你开始写代码，你就必须写好代码。
