# 第一次经历编码面试？常见问题和答案。

> 原文：<https://dev.to/interviewcake/first-time-going-through-coding-interviews-common-questions-and-answers-c0j>

### 科技公司的面试流程是怎样的？

它实际上与大多数其他公司有很大不同。

下面是一份编程工作的面试。

### 我需要知道这个“大 O”的东西吗？

[大 O 符号](https://www.interviewcake.com/big-o-notation-time-and-space-complexity?utm_source=dev)是我们用来谈论数据结构和算法效率的语言。

它会出现在你的采访中吗？嗯，看情况。面试有不同的类型。

有经典的算法编码面试，有时被称为“谷歌式白板面试”它专注于数据结构和算法([队列](https://www.interviewcake.com/concept/queue?utm_source=dev)和[栈](https://www.interviewcake.com/concept/stack?utm_source=dev)、[二分搜索法](https://www.interviewcake.com/concept/binary-search?utm_source=dev)等)。

这就是我们的全部课程为你准备的。大玩家就是这么面试的。[谷歌](https://www.interviewcake.com/google-interview-questions?utm_source=dev)，[脸书](https://www.interviewcake.com/facebook-interview-questions?utm_source=dev)，[亚马逊](https://www.interviewcake.com/amazon-interview-questions?utm_source=dev)，[微软](https://www.interviewcake.com/microsoft-interview-questions?utm_source=dev)，[甲骨文](https://www.interviewcake.com/oracle-interview-questions?utm_source=dev)， [LinkedIn](https://www.interviewcake.com/linkedin-interview-questions?utm_source=dev) 等。

对于创业公司和小商店来说，这是一个大杂烩。大多数人至少会问几个算法问题。但是它们也可能包括一些特定于角色的东西，比如针对后端 web 工程师的 [Java 问题](https://www.interviewcake.com/java-interview-questions?utm_source=dev)或 [SQL 问题](https://www.interviewcake.com/sql-interview-questions)。他们会对你在没有太多指导的情况下发布代码的能力特别感兴趣。你可能最终会做代码测试或结对编程练习，而不是白板会议。

为了确保你学的是合适的东西，你应该问问你的招聘人员会期待什么。发一封电子邮件，问这样一个问题，“这次面试会涉及数据结构和算法吗？还是会更专注于用 X 语言编码。”他们会很乐意告诉你的。

如果你从来没有学过数据结构和算法，或者你觉得有点生疏，看看我们的[数据结构和算法的直观指南](https://www.interviewcake.com/data-structures-and-algorithms-guide?utm_source=dev)。

### 应该用哪种编程语言？

公司通常让你选择，在这种情况下，你应该使用你最舒服的语言。如果你知道一堆语言，喜欢一种让你用更少的字符和更少的代码行表达更多的语言，比如 Python 或 Ruby。它能让你的白板更干净。

试着在整个面试过程中坚持使用同一种语言，但有时你可能想换种语言来回答一个问题。例如，在 Python 中逐行处理一个文件要比在 C++中容易得多。

不过，有时候，你的面试官会问一个特别的问题，比如说，C 语言特有的问题。如果你在简历上写 C，他们会问的。

所以请记住这一点！如果你对一门语言没有信心，那就在你的简历中清楚地表明这一点。把你不太擅长的语言放在像“工作知识”这样的标题下

### 穿什么好？

一个好的经验法则是穿得比人们通常在办公室穿的稍微多一点。对于大多数西海岸的科技公司来说，标准的工作就是牛仔裤和 t 恤。如果你担心过于随意，问问你的招聘人员办公室是什么样的。

### 该不该发感谢信？

感谢信很好，但并不是真的需要。送一个就随便一点。不需要在精美的信纸上手写便条。选择给你的招聘人员或招聘经理发一封简短的电子邮件。感谢他们在整个过程中对你的帮助，并请他们向你的面试官转达你的谢意。
