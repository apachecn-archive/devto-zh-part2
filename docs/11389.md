# 黑手党:最初的想法和前进

> 原文:[https://dev . to/Christian Howard/mafia-initial-thinks-and-moveing-forward-3a1o](https://dev.to/christianhoward/mafia-initial-thoughts-and-moving-forward--3a1o)

几周前，当我看到 Pusher 竞赛的电子邮件出现在我的收件箱时，我看到了挑战自己的机会，在有限的时间内开发一个产品，并使用一项我没有太多经验的技术。我抓住了这个机会，开始思考我能做些什么，以及如何去做。首先，我一头扎进 Pusher，真正弄清楚它是做什么的以及它是如何工作的。

# [](#first-experience-with-pusher)第一次体验推杆

我的第一个想法是构建一个实现聊天功能的简单游戏，这样玩家就可以互相交谈。因此，我开始钻研推送教程，看看如何使用推送通道来实现这一点。一旦我觉得自己对数据流有了很好的理解，我就选择投入一些沙箱风格的测试，看看应用程序的其余部分是如何工作的。正是在这个沙盒中，我发现 Pusher 不仅在聊天方面是需要的，在其他方面也是需要的(稍后会有更多的介绍)。意识到这一点后，我发现我完全可以实现我的想法，制作一个基于网络的游戏黑手党。

# [](#mafia)黑道

我的黑手党游戏将是流行的破冰船/派对游戏黑手党(在某些地区也被称为狼人)的网络版。用户可以提供用户名和房间名来加入其他玩家所在的房间。在那个房间里，每个玩家都可以和其他玩家聊天，并根据他们在游戏中的角色做出决定。一旦一个玩家被淘汰，他们将不再能够聊天或者做出会影响游戏的决定。

对于那些不熟悉游戏的人来说，玩家可以分为两类:黑手党或村民。黑手党的玩家知道黑手党的其他成员是谁。每天晚上，黑手党成员会从游戏中淘汰一个人。其他人都是村民。在村民群体中，有两个特殊的角色:医生和侦探。

医生可以在每个回合拯救一个成员，包括医生，但是他们不能在连续回合中选择同一个玩家。侦探有能力知道玩家在每个回合扮演什么角色。村民和黑手党不会知道谁有这些特定的角色，医生和侦探也不会知道谁有其他的特殊角色。不是黑手党的人、医生或侦探都是普通的村民。他们没有特殊的权力，但有能力投票决定被指控的市民是否是黑手党。如果对个人进行投票，所有留在游戏中的玩家都将投票，由多数人决定。

游戏的妙处在于演绎的艺术！

# [](#architecture)建筑

我最强的技能在于全栈 JavaScript，所以这个项目将在前端使用 React，在后端使用 Node/Express 来构建。Pusher 将在应用程序中提供所有行动更新。

我认为 React 组件和状态的使用将对管理游戏的各个方面有极大的帮助。我有在我的个人项目中构建游戏的经验，并且发现在状态中管理游戏元素非常有帮助。

我的初始线框将游戏分为三个区域，一个玩家面板显示游戏中的所有玩家，一个聊天面板显示游戏中的对话，一个动作面板执行游戏中的各种动作。
