# 什么是 Web 开发中的 DLL？

> 原文：<https://dev.to/flippedcoding/what-is-a-dll-in-web-development-30n8>

在您作为 web 开发人员的旅程中，您最终会对 DLL 文件做一些事情。这真的没有听起来那么糟糕。你只需要对它们的工作原理和用途有一点了解。

首先，DLL 代表动态链接库。这些文件只是保存其他程序可以用来做不同事情的指令。这意味着多个程序可以同时以不同的方式使用相同的代码。作为开发人员，它可以节省您大量的时间。

这就是你如何能够在你所有的项目中使用不同的库和 NuGet 包。每当你安装了一个新的库或 NuGet 包，你就在项目中添加了一个新的 DLL 来保存你需要的代码或数据。

这些文件可能看起来很简单，事实也确实如此，但是它们比看起来更强大。你通常不能访问文件中的代码，除非你通常不想对它做任何改变。

DLL 文件使用像 C 或 C++这样的语言，尽管你会更经常地看到 C++。如果您愿意学习如何做，您可以编写自己的 dll 来运行一些您需要的代码。这对你的项目很有价值，当然它也能让你看起来更好。

好的一面是，您不太可能不得不从头开始编写一个 DLL 文件。DLL 文件很可能会在您的代码中导致一些错误。

您将在项目中使用的那些 NuGet 包和库有它们自己的 DLL 文件。当有版本变化时，问题有时会突然出现，你最终得到的是那个特定库的 DLL 文件的旧的或不兼容的副本。

通常情况下，一个 DLL 文件依赖于另一个 DLL 文件。当独立文件有一个过时的版本，但是依赖文件是最新的，您将开始在您的开发窗口中看到大量的红色文本。在您完成了所有常规的调试步骤(调试指南)之后，可能是时候看看这里了。

在深入研究 DLL 文件之前，请确保您的所有包和库版本都是最新的并且兼容。如果世界上所有的更新和安装都不能修复它，那么它要么是其他的东西(很可能)，要么是时候卷起袖子，通过 DLL 代码工作了。
