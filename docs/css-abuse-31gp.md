## [汉诺塔](https://codepen.io/finnhvman/pen/gzmMaa)

著名谜题 [*汉诺塔*](https://en.wikipedia.org/wiki/Tower_of_Hanoi) 。它由三根杆和许多大小不同的圆盘组成，可以滑到任何杆上。拼图开始时，圆盘整齐地叠放在一根杆上，按大小升序排列，最小的在顶端，这样就形成了一个圆锥形。

该难题的目标是将整个堆栈移动到另一个棒，遵守以下简单规则:

1.  一次只能移动一个磁盘。
2.  每次移动包括从一个堆叠中取出上面的盘并放置
3.  它在另一个堆栈的顶部或在一个空杆上。不能将磁盘放在更小的磁盘上。

[![Tower of Hanoi](../Images/7b03d5ef1850192c5a5d394f559a3935.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--JiKdm7ow--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://cdn-images-1.medium.com/max/900/1%2AI1foFh3KAIX3RH4VKrkV_w.gif)

当你点击一个磁盘堆栈时，你选择最上面的磁盘，然后你可以将它移动到另一个杆，无论你如何努力，你都不能将一个大磁盘放在一个小磁盘的上面(按照游戏规则)，所有这些都是纯粹的 CSS 魔法。

我认为这个游戏的作者应该在他的简历上增加两项新技能一个算术天才和一个有远见的人。

## 最后的想法

如果你想知道，为什么他们要在纯 CSS 中这样做？我猜答案在这里，因为他们可以。

这里的挑战不是创造一幅令人敬畏的画或创造网页版的马里奥赛车游戏，而是使用一个我们都熟悉并能很好使用的工具(CSS)来创造这个，尽管这个工具不是为这样做而设计的。

当然，如果他们使用 JavaScript 或 SVG，他们可以以一种不那么痛苦的方式获得相同的结果，甚至更好的结果，但也是以一种不那么令人兴奋的方式。

[![lynn fisher](../Images/e99c1d6e90a8e08fbb1afc8d1c866e44.png)T2】](https://twitter.com/lynnandtonic/status/991436541850411008)

通过这种方式，他们可以炫耀他们能做什么，并表明无论你使用什么工具，你的创造力都是无限的。工具将永远是工具，但在正确的手中可以创造这种魔力。
