# 从网站到移动应用——你的 4 个最佳选择

如今，创业者可以选择的移动应用开发技术非常丰富。它们之间的主要区别在于实现速度、成本和最终产品质量。理解主题之间的差异以及实现它们的各种方法是棘手的部分。

因此，让我们解开四个流行的移动应用技术，将说明它们之间的共同差异。这些将帮助你确定最实用的方法来验证你的商业想法，并促进其伴随的产品开发。

## 渐进式网络应用

这项由谷歌开发的相对较新的技术使移动设备能够在智能手机的主屏幕上添加网站或网络应用程序，并能够离线与之互动。

要将常规 web 应用程序转变为渐进式 web 应用程序，您只需添加一个主屏幕图标、web 应用程序清单和服务人员，使网站能够离线工作、加载速度更快，并发送推送通知。请注意，当您在手机浏览器中加载渐进式网络应用程序时，它会提示设备将该网站添加到主屏幕。目前，渐进式网络应用程序在 iOS 设备上并不完全支持，但希望在不久的将来这种情况会有所改变。

## 混合移动应用

阿帕奇科多娃

Apache Cordova 是一个框架，使开发人员能够使用 HTML、CSS 和 Javascript 构建移动应用程序。

使用 Apache Cordova 构建的应用程序可以在不同移动平台(Android、iOS)上的内置浏览器环境(WebView)中工作，并且可以从 App Store 或谷歌 Play 商店下载。它们带有一个位于主屏幕上的图标，可以与智能手机硬件 API 进行交互，以实现设备功能(地理定位、摄像头等)。

反应自然

如果您的 web 应用程序是用 React.js 构建的，React 是一个很好的选择。它是混合应用程序领域的最新技术之一，从现有 web 应用程序迁移可以非常快地完成。因此，您可以获得使用智能手机操作系统原生组件(原生按钮、输入和其他设备功能)的移动应用程序。性能很好，因为您的源代码会转换为本机移动应用程序，而不是在内置的浏览器窗口中运行。

## 原生 app 开发

就性能和产品质量而言，为每个平台构建原生应用是最佳选择，但这也是最昂贵的方法。如果你已经有了一个网络应用程序，你只需要为 Android 移动应用程序和 iOS 移动应用程序构建移动客户端，它们将连接到你的网络客户端所在的后端。后端可能仍然需要一些小的改动，但这不会花太多时间。
