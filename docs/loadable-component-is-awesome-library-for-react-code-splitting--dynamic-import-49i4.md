# loadable-component 是一个很棒的库，用于反应代码分割/动态导入

> 原文：<https://dev.to/terrierscript/loadable-component-is-awesome-library-for-react-code-splitting--dynamic-import-49i4>

我发现了很好的 React 代码分解库，这篇文章记录了这个: [`loadable-components`](https://github.com/smooth-code/loadable-components) 。

* * *

当我想在 React 应用程序中使用动态导入和代码拆分时，我必须实现包装器，这很麻烦。

在 [React 文档](https://reactjs.org/docs/code-splitting.html#libraries)的代码拆分部分使用 [`react-loadable`](https://github.com/jamiebuilds/react-loadable) 。这是一个很好的库，但遗憾的是这个库是禁用的。

我看了 React 文档的问题，发现这个问题和`loadable-component`。

[https://github.com/reactjs/reactjs.org/pull/573](https://github.com/reactjs/reactjs.org/pull/573)

我觉得“啊，这是伟大的图书馆！”下面是可加载组件的我最喜欢的点。

# 最喜欢的点:概念

从[动机](https://github.com/smooth-code/loadable-components#motivation)

> 主要目标:减少 API，使其对开发者来说尽可能容易

我看了样本，觉得“这当然比其他库容易！”

# 最喜欢的一点:好自述(文档)。

`loadable-components`的 README 列出了需要的方法和功能，清晰而详尽。

仅供参考:我觉得最吸引人的功能是[预取悬停](https://github.com/smooth-code/loadable-components#prefetch-on-hover)。也有这个功能，但是我不容易找到。
(因为`react-loadable`的自述有一些非常大的图片..)

# 最喜欢的一点:干净代码&测试

有许多功能，我预测代码很难。
但是当我看`loadable-component`源代码时，我惊讶地发现代码保持得非常干净。
如果我遇到问题，我将能够解决或发布拉动式请求。
当然也有测试。

# 最喜欢的一点:开放式问题

库的所有者可以选择禁用或启用问题。但是我想选择接受 isses 的图书馆。
问题是可以张贴我的问题，也可以作为一个伟大的知识库。

# 结论

我喜欢[可加载组件](https://github.com/smooth-code/loadable-components)。
