# 构建您的 MVP 应用程序

> 原文：<https://dev.to/lollipop_media/building-your-mvp-app-12c1>

我们喜欢与初创公司合作，我们看到他们在想法发展的不同阶段来到我们这里，并继续取得不同程度的成功。我们看到的一种常见行为是，在向世界展示你的宝贵想法并获得有价值但令人畏惧的反馈之前，你会受到尽可能多发展的诱惑。

[![](img/826d5f8796cceaf563452f5ff332782f.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--cSS3n6iG--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1600/1%2A3zwnuQuTbNwk3ZoxwVIXZw.jpeg)

这是可以理解的，优秀的创始人对他们的想法充满热情，可能已经形成了一个非常生动的观点，关于他们的产品将做什么，以及它将如何处理最模糊的场景。

即使在你开始开发你的应用程序之前，避免这种诱惑也是非常重要的，因为它可以帮助你向他人解释，甚至向自己澄清你试图解决的核心问题是什么。

在你业务的关键时刻，重要的是不要浪费时间和金钱去开发对整体业务没有太大影响的东西。把每一个功能都当作机会成本来挑战，以你能创造的最小的东西为目标，为客户提供最大的价值和洞察力。

[![](img/c119bc6fb728472665458855441a79dd.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--w9vXnONQ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1600/1%2AEr_ZehZyhD38RrEZTPjgJQ.jpeg)

在这个决策过程中保持节俭可以让你处于一个更好的位置:

*   你可以测试你的想法并从客户那里获得反馈，而不需要大量的资源。
*   通过尽早获得反馈，你给了你的想法一个发展的机会。
*   您可以让您的开发团队专注于这些核心特性，并真正将它们钉牢。
*   节省下来的时间不要纠结于不太重要的功能，这可能意味着你在市场上击败了你的竞争对手。

考虑到这些好处，这里有一些规划你的
MVP 的高级步骤。

### 第一步——定义你的业务

*   简单回答"**你的项目是做什么的，对我有什么好处？**”。这不仅有利于为谈话的投资者和潜在客户做一个好的电梯间推介，在内部阐明你的 USP 也是聚焦你的 MVP 的一个好的开始。
*   定义是什么让这个项目成功，**在开发的初始阶段，你希望达到什么样的里程碑**？像这样的问题可以帮助塑造一个初始的应用程序构建需要什么，它是为了给投资者留下深刻印象的一次性原型，还是为了扩大规模而快速上市。

### 第二步——定义你的受众

*   谁会使用你的产品，什么是典型的用户之旅？
*   你如何向这些人推销，有什么免费的方法来引起他们的注意吗？
*   在你的想法形成之前，和他们谈谈，问问他们对你的想法有什么看法——这可能会帮助你更好地定义你试图解决的问题。

### 第三步——定义你需要构建的东西

*   利用你已经获得的信息，决定你的应用程序需要具备哪些特性。
*   列出特性并确定哪些是 MVP 绝对必要的，将特性分成小块并评估它们的成本/收益通常会有所帮助。
*   将这些功能分解成单个的冲刺工作，按照让你和 beta 用户尽可能快地尝试端到端用户旅程的顺序来做。

### 选择合适的工具去做。

由于是 React Native 的早期采用者，我们发现其他框架最显著的特点是它能够在不牺牲质量的情况下，比其他替代方案更快、更便宜地将您的想法推向市场。许多精通技术的企业家已经注意到这一点，并经常要求他们的应用程序使用这一技术。
