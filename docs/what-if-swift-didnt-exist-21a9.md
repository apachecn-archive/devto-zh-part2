我完全听到你在说什么！这种讨论在苹果开发者中并不新鲜。我不想在这里挑起战火。我是地球上最不可能站在哪一边的人，而且，由于我的客观经验相当有限。事实上，在过去一年左右的时间里，我获得了一些 Swift 知识，但这主要是因为我对语言的兴趣，而不是因为苹果开发者生态系统。我开发 iOS 应用程序，喜欢这样做，无论是作为一种爱好，还是在专业层面上，但我喜欢编写 Swift，因为我相信这种语言的潜力远远超过苹果生态系统的限制(如果你想聊聊在服务器上运行 Swift 应用程序，请 ping 我😉)

然而，当你看到苹果开发者社区在新一波 Swift 程序员和那些选择 Objective-C 而不回头的人之间被撕裂时，这有点令人沮丧。对许多人来说，这不是他们自己能做的决定。项目的成熟度和特定的需求通常会迫使开发人员做出选择。每个都有明显的优点和缺点，正如已经说过的，我不会站在哪一边。这些讨论让我开始思考，如果苹果公司选择不开发一种全新的语言，而是试图更新一种为公司服务了几十年的老黄牛，会怎么样？

看起来有点太晚了？我很想听听你对此的看法。
