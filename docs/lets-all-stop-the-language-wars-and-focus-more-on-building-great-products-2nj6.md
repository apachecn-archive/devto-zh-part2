# 让我们停止语言战争，把更多的精力放在开发伟大的产品上

> 原文：<https://dev.to/brpaz/lets-all-stop-the-language-wars-and-focus-more-on-building-great-products-2nj6>

如果你是一名开发人员，你可能在互联网论坛、博客评论区或社交网络上看到过许多无休止的讨论和争论，比如，“什么是最好的编程语言”或“为什么这种新的闪亮的语言是有史以来最伟大的东西”与“为什么有人使用这种狗屎语言/如果你用 X 语言编程，你就是一个低劣的开发人员”。

令人难以置信的是，有多少本应是聪明人的开发人员倾向于如此刻薄和咄咄逼人地捍卫他们最喜欢的语言或框架，并批评每个敢于喜欢或使用他们认为不好或过时的语言来构建东西的人。

PHP 开发人员和框架的用户，如 [Electron](https://electronjs.org/) 通常是最严厉的*批评家最常见的受害者，导致了这样的迷因:*

这些讨论在我看来是愚蠢而无用的。

作为开发人员，我们有机会使用技术改变世界和改善人们的生活，难道我们不应该支持每一个正在开发对某人有用的东西的人，而不是仅仅因为他们在使用“低级”语言就抨击他们吗？

当然，每个人都有自己的偏好，根据不同的用例，也有比其他语言更好的语言，但是为什么会有这种厌恶呢？我们不能有更多建设性的批评吗？

每种编程语言都有它的用途，应该受到尊重。如果没有 PHP，互联网就不会有今天。我不喜欢 Wordpress，但它是一个革命性的软件，当它问世时，允许许多人与他人和许多公司分享知识，并在互联网上占有一席之地。如果没有像 Electron 这样的框架，许多开发者不会关心桌面应用程序。

仅仅因为它不在黑客新闻的首页，它超过一年或者它没有最干净的语法和最好的性能，并不意味着它是世界上最糟糕的东西，所有使用它们的人都应该下地狱。

一个顶尖的开发人员可以让一个 PHP 应用程序飞起来，一个平庸的开发人员可以让一个 NodeJS 应用程序非常慢。

能够使用最新的技术很好，但是在现实世界中，稳定性非常重要。你会使用最新闪亮的 JavaScript 框架建立一个企业吗？没有人知道两年后它是否还会存在。

作为一个对创业和组织非常感兴趣的人，上周我有机会阅读了 Pieter Levels 的惊人的 makebook ,其中写道:

> 讨论什么编程语言最好的人不是在发布产品。那些不在乎自己使用什么编程语言的人正在发布产品。他们会在任何需要的时候使用任何需要的工具。

我完全同意他的说法。

作为开发人员，构建优秀的产品应该是我们的主要关注点。使用你认为更适合这项工作的工具。每个项目都不一样。

最终用户并不关心它是用哪种语言构建的，只要它能解决他们自己的问题。

当然，开发者体验非常重要。如果你使用一种你喜欢的语言来开发产品，你会更有效率和快乐，这种语言易于理解和维护，并且为你提供了很好的工具和质量库来更快更高质量地构建你的产品。

许多语言都可以提供，这也是一个非常主观的话题。我真的很喜欢用 PHP 工作，并且可以非常高效地使用它。你可能更喜欢 Ruby 或者 Node。这是个人的选择。构建高质量的产品和干净的代码并不是只有一种方法，这也取决于很多因素，比如你对语言的经验和你产品的需求。

理解这一点至关重要，因为有些语言比其他语言更适合完成特定的任务。

我不会用 PHP 开发“实时”应用程序，用 Java 开发个人博客，或者用 Node 开发需要复杂计算的应用程序。

了解每种语言的优缺点，如何以及何时应用已知的设计模式，最终将会有所不同。

此外，许多开发人员倾向于过度设计和过早优化。这也是引用书中一段非常有趣的话:

> 老实说，除非你在编写飞船程序，否则超越一门语言是很难的。它们都基于相同的计算原理。你可以用大多数语言构建任何东西。

这就是现实。并不是每个人都在谷歌或脸书的规模上构建应用程序，在那里性能和可伸缩性扮演着非常重要的角色。

PHP、Ruby、Node、Python、Golang，我想说几乎任何维护良好的编程语言都可以成为大多数项目的有效起点。

今天，在微服务时代，你不再局限于每个应用程序使用一种语言。您可以通过用不同的语言构建一些组件来轻松优化您的应用程序。

## 结论

让我们停止这种无意义的语言战争，开始互相尊重和合作，以创造更好的产品和解决真正的问题。

没有“万能的语言”，每种语言或框架都有其优点和缺点。理解它们，以及如何应用到你的产品中，以满足你的用户的期望，最终会有所不同。
