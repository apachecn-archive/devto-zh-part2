# 单页应用程序还是多页应用程序——哪一个更适合 Web 开发

单页应用程序，更广为人知的名称是 SPAs，目前在 web 应用程序开发领域非常流行。大多数企业都在谈论它，开发人员也在探索它，以便围绕它提出更好的想法和见解。但与此同时，优秀的老式多页面应用程序(MPAs)表现稳定。
如果你比较两者在市场上的权威、价值和影响力，在目前的情况下，两者在比例上几乎相等。那么，你如何发现哪一个更适合 web 开发呢？好吧，在我们进一步剖析和比较它们不同的效用和价值参数之前，让我们先了解一下当你说单页应用和多页应用时它实际上意味着什么。

# [](#going-by-the-meaning)顺其意

SPA 是一个 web 应用程序，可以放在一个 HTML 页面上。当在不同的屏幕尺寸和显示格式下打开时，它会被重新加载和调整。它安排其外观，接收和风格，以微调和适应不同的视觉通风口。因此，如果你在手机屏幕上看到一个 SPA，然后进入全尺寸的监视器屏幕，它在结构和相关性上看起来都是一样的，但会以不同的视觉容量和格式呈现，以便与屏幕很好地匹配。使用 SPAs，当你执行一个动作时，界面保持不变。谷歌、脸书、Trello 都是 spa。大部分推广或登陆页面也是 SPAs。

当我们说某个 web 应用程序是多页面设置时，这意味着它是由多个层次的 UI 组成的，并且分布在不同的 HTML 布局中。这些大多在尺寸上更重，在视觉属性和范围上更广。当您请求一个特定的内容或点击一个链接时，它会显示属于相似页面的数据，或者将数据提交回服务器，请求从服务器向浏览器呈现一个新页面。大多数现代的多页面 web 应用程序使用 AJAX 来使这个渲染过程快速而流畅，不像传统方法那样缓慢而庞大。尽管如此，正如预期的那样，这些页面需要时间来构建，并且不能像 SPAs 那样排序和快速，但有利于构建更大的网络应用。

## [](#pros-and-cons)利弊

如果我们把它们放在一起比较，从用户的角度来看，应该会揭示更多的信息。所以。以下是单页和多页 Web 应用程序的主要优缺点。

### [](#singlepage-apps)单页应用程序

**优点**

1.  单页应用程序快速、灵活、流畅，因为它们结构紧凑、形式简洁。

2.  水疗中心的开发方法简单而流畅，因此您在建设水疗中心时面临的复杂性更低。

3.  通过监控和调查网络操作、页面元素和其他数据资源，可以使用 Chrome 轻松调试 spa。

4.  spa 可以轻松有效地缓存本地存储。

**缺点**

1.  这不是一个简单的任务来优化搜索引擎优化的 spa，因为基于一个单一的页面格式，它有最少的范围来索引页面专有的关键字。

2.  它下载速度很慢，因为它意味着加载繁重的客户端框架。

3.  要让它工作，需要 JavaScript。这意味着如果您在浏览器上与 JavaScript 断开连接，您将无法正常运行应用程序。

4.  由于跨站点脚本，SPAs 更容易出现漏洞和风险，因此不太安全。

### [](#multiplepage-applications)多页应用

**优点**

1.  最适合于那些不能很好地呈现和理想地以单页格式结束的较重的应用程序。

2.  最适合喜欢使用可视化地图和菜单导航来浏览页面的用户。

3.  从 SEO 的角度来看非常有帮助。因为你可以更有针对性和资源性地优化页面。

4.  允许更大范围的安全性和数据完整性。

5.  MPa 提供了更多的可伸缩性。

**缺点**

1.  多页面应用程序不允许你拥有与移动应用程序相同的后端。

2.  前端和后端是不变地耦合在一起的，这使得在两端独立地应用更改有点僵硬和封闭。

3.  MPA 的开发比 SPA 的开发稍微复杂一些。因为，开发者不能同时在框架的客户端和服务器端工作。

## [](#spa-or-mpa)SPA 还是 MPA？

在找到您应该使用 MPA 还是 SPA 的答案之前，您应该首先确定和定义您的范围以及针对底层性能和服务目标的要求。在这里，您应该很好地定义您需要忍受的偏好和优先级，以最好地致力于您的解决方案。

因此，举例来说，如果你的想法是开发一个应用程序，允许不同教育背景和学科的学生参加考试，并参考不同学科的学习资源，你应该选择 MPA。这将允许你为不同的主题创建不同的页面，并且理想地服务于你的目的，即在一个定义良好的界面布局、类别和位置中容纳所有的页面，这些页面被很好地分离，以更相关和更有组织的方式为不同的用户服务。

除此之外，如果你必须让你的用户有更安全和一致的平台，MPa 比 spa 更可靠。在这里，除了允许更好和更广泛地接触应用程序之外，它还将有助于优化不同的搜索应用程序页面。

否则，如果你想简洁明了，只是以简洁的形式展示，如果你的想法最适合直截了当的一页纸的交流布局，spa 就是你的选择。比如，你必须推广一种抗病毒的产品。在这里，你只需要把它的描述，凭证，品牌，证明以及一系列的功能。最好是用一个能与不同显示类型兼容的单页应用程序来呈现。这一页最好是通过相互关联的部分来汇编和展示您的解决方案的整个推介和价值，从而全面传达您的信息。这个细长的单个页面还可以用 URL 锚来呈现，以聚合通信资源和分散的离散设计元素的形式来构建浏览器导航和偏好功能，以将用户连接到服务。
