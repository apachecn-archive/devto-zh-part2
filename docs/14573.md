# 我应该继续黑 WordPress 还是用框架建一个自定义网站？

> 原文:[https://dev . to/vino DMS/should-I-continue-hacking-WordPress-or-build-a-custom-website-using-a-framework-4l2c](https://dev.to/vinodms/should-i-continue-hacking-wordpress-or-build-a-custom-website-using-a-framework--4l2c)

我作为一名开发人员加入了一个团队，该团队一直在使用 WordPress 托管他们的 web 内容。在我加入这个团队之前，有一个精通 HTML 和 CSS 知识的网站管理员在一些插件的帮助下维护网站。

当我用 WordPress 托管一个有几个插件的博客网站时，我很好，但是我真的不喜欢做定制开发，因为我发现 WordPress 太粗糙了。

我们现在的 WordPress 实例使用了将近 30 多个插件，并且变得缓慢和难以管理。因为我不想在加入团队后马上提出新的改变，我继续黑 WordPress 来实现新的特性。主要是扩展已安装的现有插件的功能。

但是现在我真的不喜欢在 WordPress 上开发，因为我发现它太粗糙了。
【在 functions.php 添加 PHP 代码，在主题的 custom.css 文件中添加额外的 CSS，以及带有 snippet 插件的 JS 代码，这些都太繁琐，很难管理。不仅如此，我们经常被缺失的内容所困扰，插件设置有时会回到默认设置，或者小部件内容会被旧内容覆盖。]

我的一个问题是，在这一点上寻找一个不同于 WordPress 的选择是明智的吗？我主要担心的是投入时间构建新的东西，并在我们的业务和实际交付物中失去动力。

如果我真的应该寻找不同的选择，我们确实需要有一个 CMS，因为团队的其他成员应该能够轻松地管理内容，同时允许我开发。那么我有什么选择呢？我应该找一个更强大的 CMS 比如 Drupal 吗？因为它提供了一个框架，比如 symphony，使得开发更加结构化和有趣。

如果你有什么评论或者经历过这样的情况，请在这里分享你的想法。
