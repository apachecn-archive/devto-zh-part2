# 你不能凭空想出项目创意

> 原文:[https://dev . to/gauravchaddha 1996/you-cant-pull-project-ideas-out-of-thin-air-GfK](https://dev.to/gauravchaddha1996/you-cant-pull-project-ideas-out-of-thin-air--gfk)

*这篇[帖子](https://www.chaddhag.com/pullProjectIdeas)最初是为我的博客写的。*

我注意到随着我目前的项目完成，我有了一些空闲时间。现在我没有那种凭空想出点子的创造力。因此，需要来了——到哪里去寻找这些灵感。

第一个——我需要决定我将要从事的领域——一个应用程序，或者一个前端登陆页面——或者一个后端可扩展的成熟平台。
我决定建立一个成熟的平台。这是很重要的，因为我的搜索会相应地吸引。我建议选择领域而不是语言。而是用最适合需求的语言来构建项目。

**第二**——去打猎！

在我能从中找到一颗钻石之前，我需要整理出一系列的想法。

1.  我搜索了自己手机上的应用程序。我有一些绝妙的想法可以重新组合，还有一些我想重新做。我对它们进行了分类，我的分类是社交媒体、包装客户端(如 reddit 的 Boost 等。)和服务(记笔记等)。

2.  接下来，我探索了公共 API，旨在找到一些容易做的事情。也许在已经建立的 API 之上编写一个客户端。

    *   Todd 座右铭公共 APIs Github 回购是一个很好的开始寻找的地方- [回购链接](https://github.com/toddmotto/public-apis)
    *   [Abhishek Banthia Public-API](https://github.com/abhishekbanthia/Public-APIs)
    *   [安全 API](https://github.com/deralexxx/security-apis)
    *   [更多 API](https://github.com/TonnyL/Awesome_APIs)

    您还可以将不同的 API 组合成一个通用的客户端，使人们更容易使用多个平台。

3.  然后我上了流行语*机器学习*，尝试用 github 寻找机器学习模型。
    其他流行语还有*人工智能、深度学习、AR、VR、区块链*等。

4.  在我看来，我徒步去过的最好的地方是 *Github Trending* 。你可以探索你所选领域的不同语言，看看有什么新的东西。将这些 github repos 的想法互相整合——你就有了新的东西！

5.  使用 Github 搜索得到了一些有趣的东西——一个由[个项目想法](https://gist.github.com/MWins/41c6fec2122dd47fdfaca31924647499)组成的列表(很多)。

6.  每天花一些时间看黑客新闻。我在那里发现了一些非常有趣的科技新闻和 ML 模型提交。

有些东西我没有亲自做过，但你可以尝试一下:

*   找到所选语言的 sub-redddit 并留意它的提交。
*   Devrant [collabs](https://devrant.com/collabs) (当然——在允许的情况下采纳想法)
*   探索 PlayStore 上的新应用
