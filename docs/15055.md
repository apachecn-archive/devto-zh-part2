# 每个开发人员都需要的最被低估的技能

> 原文:[https://dev . to/_ Patrick god/the-most-低估技能-每个开发者都需要-27c7](https://dev.to/_patrickgod/the-most-underrated-skill-every-developer-needs-27c7)

假设你是一名计算机科学的学生。现在想象一下你是如此幸运，你作为一名勤工俭学的学生得到了一份工作，在那里你能够提高你的编码技能，你写的代码甚至被用于生产。

真正的客户，真正的用户，可能会使用你已经实现的特性。

我永远不会忘记我第一次意识到这一点的那一刻。坐在办公室里，没有任何专业经验，我突然想到:“天哪，过去我写的每一行代码要么是为了练习，要么是为了学习。现在真正的客户将使用我的代码。我太幸运了！这太神奇了！”

是的，它是。如果你很幸运，你得到了作为一名专业软件开发人员的报酬，你应该**永远不要认为这是理所当然的**。

你应该心怀感激，你真的欠你的同事很多。

这就是为什么有一件事，你真的应该在和他们一起工作的时候练习。

而这个东西就是:**沟通**。

惊讶吗？

我这么说是什么意思？

当你作为一名学生工作时，你很可能每天都不在办公室。即使你在远程工作，你也不可能每天都这样做。

这意味着，你可能不是每天都有空。

如果你现在正在做对下一个功能或多或少重要的任务，**请与团队交流你在做什么以及你已经完成了多少**。你的责任重大。

我经常看到它。

全职人员处理重要的任务，这样整个团队就可以完成这个冲刺。

然后他们想给工作的学生或任何每天不在办公室的人一件东西。

这些任务可能没有团队第二天需要的那么紧急，但是可能有这样的情况，团队需要知道这个特定的任务完成了多远。如果没人知道，**任务甚至可能会执行两次**。这真的很糟糕。

像这样的情况可能会非常令人沮丧，也非常昂贵。

你必须四处打听是否有人知道这项任务的进展。你必须讨论现在继续做是否合适，尽管有做两次的风险。你可能试图联系兼职同事，但没有成功。你可能需要找到不同的解决方案。你甚至会非常生气。总而言之，**这是不尊重人的行为，人们会试图给你一些不太重要的工作。**

如果这是你想要的，很好。如果不是，那就换。

好吧，我可能夸大了整个主题，但是根据我自己的经验和从其他开发人员那里学到的教训，这确实是一个需要考虑的重要主题。

如果你不能与你的团队进行适当的沟通，你甚至可能最终丢掉工作。

而且，说真的，给整个团队写一封电子邮件或一条松散的消息，或者简单地使用你的 Scrum board、Team-Foundation-Server、吉拉、Trello 或任何你正在使用的工具，并不难。

请练习一下。晚上，当你完成你的代码时，推送它，告诉每个人你在几个小时或几天不可用之前做了什么。git 中的提交消息可能不是每次都足够。

而且要搞清楚，不仅仅是打工学生，实习生或者兼职。即使是全职的人也可能缺乏这种技能。但至少我们大多数人都有自己的日常单口相声，这在这种情况下很有帮助。

抱歉，我说了这么短的话，但这是必须要说的。谢了。
