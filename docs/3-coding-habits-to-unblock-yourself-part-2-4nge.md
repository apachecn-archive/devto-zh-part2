# 错误参考

在第一部分中，我谈到了控制台日志，以及它如何帮助像我这样的初学者看到错误以及错误所在的位置。在这篇文章中，我将讨论学习错误信息的习惯。

特别是 JavaScript 编码，由于其复杂的本质，很容易出错。错误发生在你最意想不到的地方，这也是 JS 开发人员困惑的主要原因之一。

但不用担心，我相信如果你能养成学习甚至熟悉一些 JavaScript 常见错误的习惯，这个习惯可以帮助你更有效地排除 bug 或问题。

在 JavaScript 中，你会遇到很多错误，比如 ReferenceError、SyntaxError、RangeError 等等。它们都有不同的定义和解决方案。因此，我鼓励你开始注意你收到的一些错误代码，并对它们做一些研究。谁知道一旦你遇到相同的错误代码不止一次，你甚至可以建立一个雷达，而编码和条件自己不犯同样的错误。

顺便说一下，你可以在这里查看 JavaScript 错误代码:
[https://developer . Mozilla . org/en-US/docs/Web/JavaScript/Reference/Errors](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Errors)

## 几个常见的 JavaScript 错误

### 常见语法错误:

1.  If 错误:运算符“==”执行比较，而运算符“=”赋值，很容易混淆。
2.  括号不匹配，区分大小写，整体拼写错误。

### 常见类型错误:

1.在非数组对象上使用数组方法会导致错误发生，因为您将在未定义的对象上调用方法。
