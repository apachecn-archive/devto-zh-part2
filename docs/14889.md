# 关于变新变好

> 原文:[https://dev.to/brandonelance/on-being-new-and-getting-good-3ok 8](https://dev.to/brandonelance/on-being-new-and-getting-good--3ok8)

前提 1:有大量的数据。前提 2:这些数据经过组织后最有用。
推论 1:需要组织大量数据(最好是相关的表)。
前提 3:我应该在需要的领域拓展我的技能。
结论:我应该学习 SQL。

这是有道理的吧？！反正现在已经太晚了。我不仅从未学习过形式逻辑，而且我已经买了这本书。事实上，我几个月前就买了。

### “我为什么在乎，”你问。

事情是这样的:我不是 CS 专业的。我有计算物理学硕士学位。为什么这很重要？因为如果你处于类似的境地，想要在数据科学、分析或统计建模等领域有所作为，你完全没有准备。

显然不是我们没文化！就我而言，我对 Python 相当熟悉。我处理和清理过脏数据。我将机器学习算法应用于大型数据集。这些对潜在雇主很重要，对吗？

所以我浏览了招聘信息，就像一个人做的那样。我应用了必要的过滤器，这样我就只盯着一墙的入门级职位。好了，我对 Python 有所了解，对 MATLAB 有很多了解，对 C++也有一点经验...但是 Java 呢？SQL？r？露比？SAS？这些都**都**在一张入门级别的招聘启事上！

### “你的观点是什么，好先生？”

由于我自己的领域有些饱和，我试图打开的职业选择*可以说是*有点回避，但我最初对自己似乎准备不足感到非常震惊。否则，我会准备得多么充分。但是我没有停滞不前或者为下一步该做什么而烦恼，而是决定学习一些新的东西。

你知道的我不知道去吧，上帝。

如果没有别的，它有助于我的理智。如果你和我一样，你需要学习，需要坚持，需要感觉你正在做的事情会对你以后有帮助。

这就是我的观点。即使你不在你想去的地方，投资一项有用的技能也不是个坏主意。事实上，如果你没有前进，你很可能会落后。

### 回 SQL...

所以我在这里，学习 DCL 的来龙去脉。啊...我在骗谁呢？我已经和权限斗争了几个小时了。

### 哦，对了

我是新来的，这是我的第一个帖子，所以大家好！
