# 开发人员的项目想法

> 原文：<https://dev.to/codnificent/project-ideas-for-developers-506o>

让我们花几分钟时间来谈谈项目想法。很多人似乎会问这类问题:“我已经学会了`this`、`this`和`that`。接下来我该学/做什么？”。然后人们会告诉他们:“建立大量的项目”。这个时候通常是他们觉得自己知识有差距的时候。我是凭经验说这些的。这发生在我们大多数人身上...

“嘿老兄！去做一些很酷的东西吧！”这是他们通常会对你说的话。并不是说他们错了。只是，你似乎不知道该建造什么。
而且，有些无聊的部分是，当你搜索项目想法(构建什么)时，你通常会看到类似待办事项列表、计算器、番茄定时器、[看到这个我已经建立了😄](https://codnificent.github.io/tomatoTimer)等等。

这些东西还不错。但是我猜你想要一些实际上有用并且更有挑战性的东西。如果那是你想要的，不要担心。我有事要告诉你。但我要告诉你的这件事，我觉得你不会喜欢的。
好吧，这个怎么样？想象一下，你和你的朋友在一起，你说了一些他们不喜欢的话。就这样，他们都脱下鞋子，开始用鞋子打你。

> 我想当我告诉你这个的时候，你会这么做——用你的鞋打我！

#### 好吧。问题是，你不应该搜索项目的想法

是啊。你不应该搜索。我之所以这么说，是因为只有当我们密切关注我们的环境时，才会有很多项目创意。

[https://thepractical dev . S3 . Amazon AWS . com/I/n 4 MH 5 ixe ru 4 f 6 q3g 8 lsn . jpg](https://thepracticaldev.s3.amazonaws.com/i/n4mh5ixeru4f6q3g8lsn.jpg)

以此为例。Vue.js 背后的人[尤雨溪](https://twitter.com/youyuxi?ref_src=twsrc%5Egoogle%7Ctwcamp%5Eserp%7Ctwgr%5Eauthor)发了一条 [twit](https://twitter.com/youyuxi/status/1049694911665266688) ，他在 twit 中说“CSS 很难”。看看上图的右下角。你会看得更清楚。

这就是我所说的——密切关注我们的环境。这并不是说谷歌做得不对。只是 CSS 代码在不同的设备上工作方式不同。这是我的观点。

看到这个 twit 之后，我决定做点东西——用户反馈箱。这个想法是因为开发者不能在不同的设备上看到他们的代码，所以在他们的网站/网络应用中集成一个反馈框是非常有意义的。

通过这个反馈框，用户可以匿名分享他们使用网站/web 应用程序的体验。嘿，对你来说这可能听起来很垃圾，但是当我思考这个问题的时候，它对我来说非常有意义。

事实上，我看到了构建 box(最简单的 react 应用程序之一)的巨大成功😆)直到我决定部署它。我试过 Netlify，gh-pages，都没用。我读了很多文件，但我不能把它做对。所以我放弃了...☹️

> 哦不！我太过分了。这里主要是和大家分享如何获得项目创意。

你只需要思考，问自己问题。感谢大自然，人类是有创造力的，不像人工智能。你想到的项目对你来说似乎太大了。但是，你总是可以和别人一起建造它。

> 专业建议:首先建立一个博客/网站，在那里你可以分享你学到的东西。就像这个[一个](https://mordernweb.com)

想一些对你或其他人有用的事情——不要总是做无聊的事情。不要！又来了...挑战你的能力总是一种有趣的学习方式。所以，不要想多了。当你开始问问题和阅读文档时，你会惊讶于你能做多少事情。

> 嘿，太棒了，不要像我一样，因为无法部署而放弃我的反馈箱

“嗨伙计们！我已经有了这个项目的想法，应用程序的用户可以做这个和那个。唯一的问题是，我觉得我一个人做不到。有人愿意和我一起工作吗？”。

你需要做的是很好地解释你的想法。仅此而已。你会有机会和别人一起工作。编码需要大量的协作。你会有机会向这个人学习。只是不要在大的想法上有所保留。
哦！你害怕有人会窃取你的惊人想法吗？我也是！但是我们能做些什么呢？如果我们继续保留它，等到我们准备好实现它的时候，其他人可能会做出更好的东西。游戏变了，变了很多！

因此，你可能会决定思考并问自己一些问题，以便提出有用的项目想法

或者，你可能决定脱下你的鞋子来打我。不幸的是，你联系不到我😝

> 也许，你会继续问:“对于初学 JavaScript 的开发人员来说，最好的项目是什么？”，“作为一名中级 python 开发人员，我应该从事什么项目？”，问题还在继续...

不管你决定做哪一个，我祝你好运。再见！👋
