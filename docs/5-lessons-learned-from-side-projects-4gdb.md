从我成为专业软件工程师的第一天起，我总是花一些时间开发我自己的兼职项目。虽然有些副业没有意义(就像上面的推文)，但我看到副业对我的职业生涯有巨大的好处，所以我决定写一些我学到的如何最大化副业效果的经验。

# 上线！

是的，你听到了…去和你的副业一起生活吧！最近我建立了我的挑战网站，( [www.mypopchallenge.xyz](http://www.mypopchallenge.xyz) )，虽然 UI 不是很令人印象深刻，但我发现在部署方面存在非常有趣的挑战。事实上，我在 AWS 的 1 年免费层已经结束，这让我编写了一些脚本，以便在 now.sh 实例和 Netlify 部署之间进行同步，这样我就可以同时使用它们，并将我的费用保持在最低水平。

上线不仅仅是将你的应用发布到应用商店或者将你的网站部署到网络上。它可以是向 npm 发布一个库，甚至只是开源代码，这样人们就可以看到它，使用它，并从中学习。

所以这是我的第一个建议——去现场直播！

# 不要试图从中赚钱

当你试图从你的应用中赚钱时，你开始变得疯狂。你可以想象自己有成堆的钱，并且你正在朝它努力。这样不好吗？我觉得很神奇。这是一种企业家精神，许多好产品就是这样诞生的。但这不是一个副业。你肯定会学到很多东西，但当你开始看到从中赚钱时，就很难放弃它，这有时是正确的事情——更专注于你的日常工作或开始一个你想学习的新技术的新副业项目。

这是我的建议——为了乐趣而不是为了钱去做兼职项目。

# 增量学习

所以你有一个想法:这是世界上从未见过的最好的 ToDo 应用程序…你听说过这个惊人的前端框架，它在 GitHub 上有 10 亿颗星星，服务器必须与那个数据传输热框架在一起，你还知道 3 个你必须尝试的酷 JS 库，你想试试你作为微服务架构师的技能。听起来很多？太多了！兼职项目的本质是完全由动机推动，因此你需要很快看到结果，否则动机会随着下一个酷想法而消失。猜猜看，一个月后，您将拥有另一个半成品回购，包括 2 个微服务、一半数据层和一部分前端。

我的建议是——为你的副业项目一次选择一项新技术。

# 独自工作

一个软件开发团队 90%的问题都是沟通问题，剩下的都是人与人沟通不够好无法解释的问题。当你增加一个成员到你的项目组时，你需要花时间沟通。虽然与他人的交流很棒，但你不会得到你希望你的副业项目所给予的好处。(我不是在代码通信方法上编码。我所有的兼职项目都有文档记录，并有完整的自述文件)
如果你不想解释为什么你决定改变技术栈，或者花 25%的时间在团队内部同步化上，那就别管它了。

我的建议是——兼职项目正是独自享受纯粹的编码体验的时候，这是我们大多数人在选择学习如何编码时喜欢的体验。

*注意:有时当项目的范围太大时，你已经上线了，你想要更快，或者你刚刚开始你的辅助项目，以便你可以学习如何更好地沟通，这实际上是有意义的合作。*

# 一起码

你独自工作的事实并不意味着没有人应该看到你的代码。每当有人向我寻求改进的建议时，我都会告诉他这个建议——“找个人来审查你的代码”。我写临界区代码的时候，总会拿给别人看。我问了很多关于设计的问题，我试图让尽可能多的人参与到一个友好的、提供建议的位置中来。如果没有人帮你做这件事，外面有很多人会很乐意帮忙的。(有趣的事实是——当朋友们审查我的代码时，我从来没有得到不好的评价。我想愿意帮别人检查代码和拥有良好的编程技能之间是有关联的)

我的建议是——尽可能多地分享你的代码，以获得反馈并变得更好。

# 结论

副业很有趣。我喜欢向人们展示我的一些作品的感觉，以及看到人们与我的产品互动的兴奋感。我从我的兼职项目中学到了很多，我上面给出的经验是非常适合学习的。如果你从你的副业项目中学到了其他经验或教训，请写一篇评论。此外，如果你想分享你的副业，我希望看到评论中的链接。
