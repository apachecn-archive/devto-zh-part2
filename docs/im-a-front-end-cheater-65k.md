# 我是一个前端骗子

> 原文：<https://dev.to/catriname/im-a-front-end-cheater-65k>

我是一个全栈开发者。我用 Visual Studio 开发，用 C#编码。NET 核心，部署到 Azure 我必须承认，我是一个前端骗子。

来吧。NET Core Web App demo 本身加载了预装的 Bootstrap 和 JQuery。在某种程度上，我们都是骗子，但我正试图让我的欺骗多样化，并消除我的依赖性。

这是我的最新策略。

# 再见自举，你好布尔玛！

Bootstrap 能做的，布尔玛能做得更好！

嗯，不完全是，但是让我解释为什么。

## 布尔玛没有 JS 文件。

唯一一次我觉得需要 JS？可能是 JQuery 数据表，但是我做了一个特定的“DataTableLayout”并且只在需要的时候使用 JQuery 表。Bootstrap 和 JQuery 的这种耦合本身就是我试图摆脱的。

其他的几乎都有？列，表，英雄形象？布尔玛无所不为！

## 我要风格就是风格。

我不喜欢 JS 做我的悬停、表单验证等工作。HTML5 增加了它的验证选项，CSS 在转换方面也很丰富。为什么我被迫引入 JQuery 和各种动画库，只是为了获得一些简单的文本动画。

令人沮丧的是，当我可以用一个单独的 CSS 文件:布尔玛来做栏目、表格、封面设计等等的时候，却要依赖这么多代码。

# 我要天赋！

这很棒，所有东西都加载得很快，但是没有“动画”，没有“天赋”。

现在怎么办？我应该屈服，继续导入 JQuery 和 WOW.js 吗？我是说我可以...

或者，我可以挑选我的动画，比如: [Animista](http://animista.net) 。它有一个很棒的用户界面，我可以挑选*正是*我需要的，没有别的。如果有任何问题，那么，我注释掉 CSS，回到布尔玛计划，我们走！

# 举例

所以，我想做一个简单的鼠标。用户鼠标经过图像和文字出现。我遵循了 W3Schools 中的基本 [CSS/HTML 示例，并且很快就完成了(我用“flip”类替换了“container”以避免布尔玛冲突)。](https://www.w3schools.com/howto/howto_css_image_overlay.asp)

出于兴趣，我访问了 [Animista，寻找一个稍微模糊以聚焦单词](http://animista.net/play/text/focus-in)的例子，并努力将它的动画与我的鼠标悬停结合起来。这就是结果...

下面是 HTML:

```
<div class="flip">
    <img src=img/flip_images/branches.jpg" alt="Avatar" class="flip-image" style="width:100%">
    <div class="flip-middle">
        <div class="flip-text"><a href="/Home/Locations">LOCATIONS</a><br /><div class="is-size-6">Over 80 Years of Service</div></div>
    </div>
</div> 
```

下面是 CSS(W3+Animista 的组合):

```
.flip {
    position: relative;
    padding: 0.3rem;

}

.flip-image {
    opacity: 1;
    display: block;
    width: 100%;
    height: auto;
    transition: .5s ease;
    backface-visibility: hidden;
}

.flip-middle {
    transition: .5s ease;
    opacity: 0;
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
    -ms-transform: translate(-50%, -50%);
    text-align: center;

}

.flip:hover .flip-image {
    opacity: 0.3;
}

.flip:hover .flip-middle {
    opacity: 1;

    /* below credit of Animista*/
    -webkit-animation: text-focus-in .5s cubic-bezier(0.550, 0.085, 0.680, 0.530) both;
    animation: text-focus-in .5s cubic-bezier(0.550, 0.085, 0.680, 0.530) both;
}

.flip-text {
    font-size: 2rem;
    font-weight: 600;
    line-height: 1.125;
    color: #565656;
}

/* ----------------------------------------------
 * Generated by Animista on 2018-8-7 13:40:21
 * w: http://animista.net, t: @cssanimista
 * ---------------------------------------------- */

/**
 * ----------------------------------------
 * animation text-focus-in
 * ----------------------------------------
 */
@-webkit-keyframes text-focus-in {
    0% {
        -webkit-filter: blur(12px);
        filter: blur(12px);
        opacity: 0;
    }

    100% {
        -webkit-filter: blur(0px);
        filter: blur(0px);
        opacity: 1;
    }
}

@keyframes text-focus-in {
    0% {
        -webkit-filter: blur(12px);
        filter: blur(12px);
        opacity: 0;
    }

    100% {
        -webkit-filter: blur(0px);
        filter: blur(0px);
        opacity: 1;
    }
}
/* ---------------------------------------------- */ 
```

结果是一个简单的动画:你将鼠标放在一幅图像上，它会褪色，出现一个透明的白色覆盖图。一个动画单词在死点变得模糊，这样你就可以点击链接。

纯 CSS！(布尔玛+阿尼斯塔 mods)

# 那么，为什么这么麻烦呢？

我去年有一个网站，它过去和现在都很重要。它接受付款。它需要一个非常特殊的 JQuery 版本来运行自己的库，加密信用卡号码并发送给 API(因此不能存储和避免 PCI 遵从)。这很好，但是根据我当时的设计，我有了一个不同版本的 JQuery，以及大量的库。这些库做了所有的事情，从封面背景图片，到动画，到表单验证等等。我把所有东西都拆了，它确实工作了。然而，在那里的某个地方，其中一个库与 Edge 的新版本发生了冲突，并通过 API 引起了大量关于无效信用卡的投诉。所有与边缘中的表单下拉相关的错误。

Chrome 和 Firefox 处理得很好。

## TLDR；总结，经验教训:

最小化你的依赖性。这就是我开始做的事情。
