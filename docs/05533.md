# 我的博客帖子工作流程:从主题到发布

> 原文:[https://dev . to/aspittel/my-blog-post-workflow-from-topic-to-publication-4n 78](https://dev.to/aspittel/my-blog-post-workflow-from-topic-to-publication-4n78)

写博客可能相当棘手——很难想出一个主题来写，更不用说实际创作内容了。我写了越来越多的博客，我想给我写帖子的建议——从主题到按下帖子按钮！如果你对我关于开博客的建议感兴趣，我在这里也有一篇关于这个的文章。

## 1。寻找话题

有时候，为一篇博客文章找一个主题是创建它最困难的部分之一！以下是一些寻找主题的技巧:

*   不要太担心有一个完全独特的主题——绝对不要抄袭别人的帖子或复制他们的想法，但人们仍然会阅读我的 [React intro](https://zen-of-programming.com/beginners-guide-react) 即使那个主题已经被做了很多次。我对这个主题有不同的看法，所以我决定无论如何都要写下来。

*   写一篇有助于超越你的博文。当你在寻找某样东西却没有找到，或者你发现教程没有多大帮助时，记下它，这样你以后就可以自己写了。*非常感谢[拉奎塔](https://twitter.com/loudblkgrl)提出这个建议！*

*   我做的一件事就是接触你的社交网络，看看人们想要什么样的帖子。我会偶尔发一条推特，询问人们想要什么类型的帖子。人们也经常向我寻求建议或意见。我会记下我觉得特别有趣或与更广泛的受众相关的内容。你也可以看看 Quora 或 Reddit 这样的网站，看看人们在那里发布了什么。如果你对一个问题特别感兴趣，你可以写一个详细的回答。

*   你有一个独一无二的故事——分享吧！分享你的背景故事，你是如何开始写代码的，或者你职业发展的故事，甚至只是你解决的一个问题。成为个人并在网上分享你的故事可能很难，但它真的能帮助人们。

*   我坚信创造力源于约束。当我在工作中有着装规范时，我总是穿得更好，因为我必须创造性地思考以符合指导方针，而不是随便穿上什么。当我开始写博客时，我遵循的主题是每周写一篇关于我用一种对我来说新的技术开发的应用程序的文章。我想学的技术太多了，我总是有东西可以写——唯一让我停下来的是时间限制！Vaidehi Joshi 大获成功的 BaseCS 系列是另一个紧跟主题的优秀例子。

我用 [TickTick](https://www.ticktick.com/) 保存了一个博客帖子想法的运行列表——这是我使用的待办事项列表应用程序。当我想到主题的时候写下来是很重要的——如果我不写的话，我会完全忘记的。月初我做一个内容日历，上面写着那个月我实际想写的帖子。我并不总是坚持内容日历，但提前知道我想写什么，并知道不同主题和类型之间的平衡是很好的。

## 2。草案初稿

一旦我有了一个话题，我通常就开始写我对它的任何想法。我的过程在这里确实是杂乱无章的——我会在帖子里从一个地方跳到另一个地方，概述一些部分，然后在其他部分写段落。我让自己真正有创造力，我记下我所想的一切。当我写这篇文章的时候，当前的段落已经完全写完了，但是文章的开头只是一些短语，比如“来自约束的创造力”和“分享你的故事”其他人可能更有条理，但这对我很有效。

然后，我仔细检查并确保我已经完整地写出了每一部分，并且我对这篇文章很满意。

关于内容，我关注几件事:

1.  它是准确的——当然有技术含量，事实需要正确，代码需要工作。我通常从我写的完整应用程序中复制并粘贴代码，这样我就能肯定它会工作。

2.  这是可以理解的，并且适合我的目标读者——关于你的博客文章的另一个重要的事情是确定你的目标读者，并使你的文章面向他们。你的目标读者是想学习 SQL 库 Knex 的节点开发者吗？或者，你的读者是一个完全的新手，当你在构建你的节点应用程序时，他需要解释函数的概念。一般来说，你希望你的帖子能吸引相当多的读者，所以我通常会把帖子发给那些知道基本原理但不知道我所说的具体工具的人。你通常想把你的帖子瞄准相对相似的受众，这样人们就会对你的帖子感兴趣。

3.  我不使用文字墙。在互联网上，人们习惯于用易于浏览的视觉效果来分割短文本块。在文章中添加项目符号、图片和副标题，增加视觉趣味。我的帖子一般在 2000 到 3000 字之间，但感觉没那么多，因为没有巨大的文字块。

我用我的文本编辑器[和代码](https://zen-of-programming.com/vs-code-setup)写我的内容。对 Markdown 的支持非常棒，我有一个拼写检查器，可以随时发现问题。它有一个预览工具，所以我可以看到最终的降价会是什么样子，如果我也想！我喜欢能够定制我的设置，所以我真的很喜欢在这个过程中使用 VS 代码。

## 3。修订本

接下来，我通过[语法](https://app.grammarly.com/)运行我的帖子——我有专业版。我喜欢它如何让我选择我试图在帖子和观众中使用的语气，以及我如何“教”它更准确地用于我的写作。它还会收集滥用的单词，这也很有帮助。我将我的文本复制并粘贴到桌面编辑器中，然后检查每个错误，确保内容是好的。

## 4。画面

我的下一步是为社交媒体创造视觉效果，并给我的帖子增加视觉趣味。这些通常有 TL；我的帖子的 dr 版本，或者只是标题。我把它们做成所有社交媒体网站的首选尺寸，这样当你点击分享按钮或者分享一篇文章的链接时，图片应该看起来很好(至少在我的网站上)！

我在[草图](https://dev.to/aspittel/designing-with-sketch-42jp)中画出这些，我有一个样板设置，我只是复制并粘贴包含不同大小图像的不同画板。然后，我只需添加我想要的帖子设计并格式化它们！

## 5。出版

最后一步是按下发布键，把你的文章发送到世界上！我强烈建议一开始就发布到别人的网站上，或者至少交叉发布——除非构建和维护博客基础设施是你的主要目标！现在，我在自己的网站 dev.to 和 Medium 上发表文章。我也试过一次 LinkedIn，但它很难格式化，只有大约 100 人阅读，所以以我的经验来看不值得花这么大力气。Medium 也可能很棘手，但是 [Markdown to Medium](http://markdowntomedium.com/) 使它变得简单一点——它将您的 Markdown 代码转换成 Gists，所以您不必担心怪异的 Medium 代码格式！

老实说，对我来说，发布常常是最难的部分。我的电脑上有很多未发表的帖子，我可能会也可能不会在某个时候发布。老实说——由于紧张，我经常在写完文章几天后才发表。我想列出一些我常见的担忧，以及我是如何促使自己最终坚持发表文章的。

没人会看我的帖子:那又怎样。我已经把所有的努力都投入到写这篇文章中，最坏的情况是我从写这篇文章中受益，但是没有人受益。

**我没有足够的经验来写这个**:如果你有一篇博文包含了大部分正确的信息，或者至少是你对这个话题的解释，那么你就足够有经验了。从新手的角度来看，有很多优秀的帖子，它们真的很重要！

如果我的代码错了怎么办？还是我弄错了一个细节？:果然如此！我以前的文章也有 bug！很多人也是如此。当然，有些人可能会很粗鲁，但以我的经验来看，大多数人只是想帮忙。另外，你从写文章中学到了一些新的东西，你可以很容易地编辑了！

如果人们在网络上很刻薄怎么办？这是我的新问题——看到我的博客有所增长真的很酷，但随之而来的是负面影响。现在社交媒体上有很多对我无礼的评论和不适当的挑逗。我在处理这个问题上肯定不完美，但我会努力记住积极的评论比消极的评论多得多，希望我的帖子能帮助到一些人。我试图忽略它们，当我的帖子出现在 Reddit 或 HackerNews 等网站上时，我已经不再看帖子了，因为匿名可能会产生更多的毒性。是的，人们可能很粗鲁，但是更多的人是友好的，并且从你所做的事情中受益。

所以，出去发表一些博文吧！我迫不及待想见到他们。
