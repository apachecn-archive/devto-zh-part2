# 尽管如此，Silwing 编码

> 原文：<https://dev.to/silwing/nevertheless-silwing-coded--56b0>

## 我开始/继续编码是因为...

...解决问题和找到高效的做事方法是令人难以置信的满足。我的第一个“代码”是在 FrontPage 中创建的一个哈利波特粉丝网站。如果我没记错的话，我那时大约 11 岁。我很快厌倦了像 FrontPage 这样的 WYSIWYG 编辑器的局限性和低效率，转而使用源代码。所以 HTML 和 CSS 成为我第一次真正的编码体验。

在写硕士论文时，我开始四处寻找工作机会。我参加了目前工作的面试。我记得面试官问我的一个问题是“你认为自己五年后会怎么样？”。我当时的回答是，我在想象走建筑师的道路。获得几年的编码经验，然后成为一名架构师。一年后，我告诉我的经理，我必须收回那句话。架构师做的代码太少了。

解决问题的满足感永远不会变老，这是让我继续编码的原因，也许会一直这样。

## 我最近克服了...

...更改我的默认交互式外壳。受 dev.to 文章“我能看看你的终端吗？”我从 bash 跳到 zsh，到目前为止我很喜欢它。改变可能令人畏惧，但是我们没有抓住的机会通常是我们后悔的。

## 我仰望...

...我那些极其熟练和聪明的同事们。我学到了很多，也教会了他们一两件新东西。分享是关怀，分享知识是任何开发者社区的最大优势之一。

我也很崇拜晚餐骨头，自从他加入《我的世界》团队后，我就一直在推特上关注他。作为一名开发人员，他处理如此大型和受欢迎的项目的方式，同时仍然保持非常个人化的感觉，这是令人惊讶的。

## 我目前正在入侵...

...让我正在从事的项目的开发人员的日常生活变得轻松的 shell 脚本。让我们的流程更加顺畅和高效是一件好事。

## 我对其他想编码的女性和非二进制人士的建议是....

...永不放弃。

从某种意义上说，我是幸运的，我从未经历过，我所读到的其他女性或非二进制的人们所经历的。但这也意味着我没有第一手经验可以与他人分享，我可以说的是，如果你想从事编码工作，你有权这样做。这应该是显而易见的，但我们都感觉到冒名顶替综合症不时压在我们的肩膀上，如果别人告诉我们我们不适合这样做，那就更是如此。所以值得重复。如果你热爱编码，那就编码吧。

你能做到的！
