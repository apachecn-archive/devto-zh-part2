# 什么是 JAMstack，为什么您应该尝试它

> 原文：<https://dev.to/lauragift21/what-is-jamstack-and-why-you-should-try-it-5f8h>

你曾经想要建立一个现代的网络应用程序，而没有所有的噱头吗？那么你应该已经听说过`JAMstack`，其中`JAM`代表`JavaScript API's and Markup`。

### "JAMstack:名词\ ' jam-stak \ '

### 基于客户端 JavaScript、可重用 API 和预建标记的现代 web 开发架构

不断变化的网络世界总是让我们惊叹不已。静态网站很受欢迎，超级高效，速度极快，这使得它非常适合博客、简历、作品集、登陆页面和文档。是的，Jamstack 很酷，你问我为什么要用它？

### 您应该考虑 JAMstack 的原因

*   **更好的性能**:如果你想在旅途中以完美的速度和性能构建你的网站，你最好的选择就是使用 JAMstack。仅在通过 CDN 提供的部署上构建您的文件。

*   **便宜而快速的部署** : Jamstack 网站可以轻松部署，而无需经历配置数据库、保护。htaccess 等等。我们只处理静态文件，所以很容易集成一个[快速 CDN](https://www.slant.co/topics/2256/~static-website-hosting-provider) 来启动和运行。

*   **令人敬畏的开发者体验**:学习和使用 JAMstack 并不像你想象的那么困难。仅仅凭借 HTML、CSS 和 JS 经验，开发者就可以构建复杂的网站。JAMstack 不需要复杂的语言，它还允许为站点生成器选择 CMS 选项，消除了为内容和营销维护单独堆栈的需要。

*   **高安全性**:静态网站存在漏洞的可能性非常低，因为它只是通过 CDN 提供的 HTML 文件和外部 API 处理。这降低了网站上任何可能的风险或攻击。很容易将 SSL 证书与[集成在一起，我们来加密](https://letsencrypt.org/)。此外，利用服务器端微服务 API 来保护网站。

*   伟大的社区:jam stack 社区正在不断发展壮大。对各种功能的新支持每天都在涌现。更令人兴奋的是正在创建的[当地社区聚会](https://jamstack.org/community/)。[在线社区](https://gitter.im/jamstack/community)也令人惊叹。

另一个值得考虑的特性是将核心动态功能集成到 JAMstack 网站中的能力。JAMstack 或多或少比你想象的更加动态，这项技术有能力处理动态的服务器端功能，考虑处理后端功能工具，如 [Webtask](https://webtask.io/) 和[无服务器](https://serverless.com/)。使用[表单礼包](https://formspree.io/)或[网络表单](https://www.netlify.com/docs/form-handling/)随时处理表单。完整的搜索功能[谷歌搜索](https://cse.google.com/cse/)、 [lunr.js](https://lunrjs.com/) 和 [Algolia](https://www.algolia.com/) 有相当多的工具可以与 JAMstack 集成。该列表在[此处](https://www.thenewdynamic.org/tools/)和[此处](http://cloudcannon.com/tips/2014/12/12/the-ultimate-list-of-services-for-static-websites.html)可用

很多公司已经加入了这股潮流，那么你还在等什么呢？
