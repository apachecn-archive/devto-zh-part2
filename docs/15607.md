# 所有人的代码

> 原文:[https://dev.to/gonzalezanguita/a-quick-tip-2d1](https://dev.to/gonzalezanguita/a-quick-tip-2d1)

代码对于开发者来说就像语言对于国家来说一样，有些区域甚至说着同一种语言，你很难理解其中的一两件事。

```
 const a = true;
const b = true;

const something = a && b;

if (something) {
    const text = "Its because is well coded (???)";
    console.log(text);
} 
```

只需付出一点努力，你就可以交流，那么为什么不在编码方面做同样的事情呢？

```
 const isWellCoded = true;
const didYouTakeYourTime = true;

const youCanReadThis = isWellCoded && didYouTakeYourTime;

if (youCanReadThis) {
    const whyCanYouReadThis = "Its because is well coded";
    console.log(whyCanYouReadThis);
} 
```

有一次一位老师对我说，

> 代码必须以易于阅读的方式编写。你应该努力去理解它所说的话中没有的内容。

如果你很投入，经过一些练习后，你会停止思考质量，因为它是从你的头脑中预先格式化出来的。
