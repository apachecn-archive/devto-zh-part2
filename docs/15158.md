# 开始新项目的步骤

> 原文:[https://dev.to/grahamcox82/steps-to-starting-a-new-project-3p 9](https://dev.to/grahamcox82/steps-to-starting-a-new-project--3p9)

我是一个连续的项目启动者。即使我想否认，我也不能否认。这不是一件好事，但这意味着我已经很好地掌握了入门知识。

每当我开始一个新的个人项目，我总是努力做同样的基本事情，这是我为其他可能感兴趣的人列出的清单。我不打算详细说明，因为它们应该都是不言自明的。根据项目的计划规模，并不是所有这些步骤都是需要的，或者都是值得努力的，所以这是一个挑选有意义的步骤的问题。

*   新版本控制报告-通常是 Github
*   问题跟踪——针对 bug 和实际的开发案例
*   建立依赖管理——我是 Java 项目的狂热爱好者
*   设置相关性版本检查以保持最新
*   单元测试
*   构建尽可能多的时间集成测试——使用尽可能多的真实依赖项。例如，我为此使用了嵌入式 postgres 数据库
*   所有可部署组件的 Docker 映像
*   全栈端到端测试。这就是 docker 容器派上用场的地方。
*   每一次推送的持续集成——包括分支和拉取请求。我最近开始使用 Shippable，但是有很多选择
*   持续部署——heroku 在这里很好，但是有很多选择

只有在这一点上，我才会开始实际的开发工作。但是在这一点上，我确信我正在做的事情得到了很好的维护并且工作正常。
