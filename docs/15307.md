# AutoCAD 的 3 个开源替代方案

> 原文:[https://dev . to/xeroism/3-open-source-alternatives-to-AutoCAD-5ega](https://dev.to/xeroxism/3-open-source-alternatives-to-autocad-5ega)

# [](#open-source-computeraided-design-software-may-not-have-all-the-functionality-of-proprietary-solutions-but-it-may-have-everything-you-actually-need)**开源计算机辅助设计软件可能不具备专有解决方案的所有功能，但它可能拥有你实际需要的一切。**

[![autocad banner](../Images/01cd6cf7db64c16524bd1c4655a3f988.png)T2】](https://i1.wp.com/fossnaija.com/wp-content/uploads/2018/02/autocad_alternatives_banner_fossnaija.png?ssl=1)

CAD——计算机辅助设计或计算机辅助制图，取决于你问谁——是一种技术，旨在更容易地为现实世界的对象创建规范。无论你正在建造的物体是房子、汽车、桥梁还是宇宙飞船，它都有可能是从某种类型的 CAD 程序开始的。

其中最著名的 CAD 程序是 AutoDesk 的 AutoCAD，但还有许多其他的，专有的或开源的。那么，AutoCAD 的开源替代品怎么样呢？答案取决于你打算如何使用它们。

让我们首先诚实坦率地说明一些事情:如果你正在寻找现有 CAD 程序的替代程序，它将提供相同的功能和工作流程，而不对你的流程进行任何更改，你将会失望。但我认为你失望的原因与产品的许可无关——长期用户对他们的软件有特定的需求和期望，很难用简单的替代方法来替代复杂的程序。决定一个软件的替代品，不管是开放的还是封闭的，对你来说是一个好的选择的诀窍是梳理出你到底需要什么。

情况和发现坚持自己“需要”Photoshop 的人只是用它画了几个几何图形，从照片上去掉红眼没什么区别；他们真正需要的是一个可以替代那些特定功能的图形编辑工具。如果这些功能已经付费但没有使用，那么它是否拥有原版的所有功能就无关紧要了。

我个人的开源 CAD 程序之旅也不例外。我在研究生院时曾短暂地使用过 AutoCAD，所以当我想尝试绘制一些东西的三维平面图时，这几乎是我所知道的全部。但仅此并不能使 AutoCAD 成为最佳选择。

随着我努力用开源选项取代生活中越来越多的软件， [Blender](https://www.blender.org/) 被证明同样适合我的 3D 建模需求，无论我是在玩为 3D 打印机创建的模型，还是在看从其他程序导出的风景。

对于相对简单的规划我的家庭景观项目的任务来说， [Sweet Home 3D](http://www.sweethome3d.com/) 是一个很好的开源选择。

如果您的需求更具体一些，并且您确实需要一个专用的 CAD 程序，这里有一些很好的开源选择供您考虑:

## [](#brlcad)

 **[![BRL-CAD banner](../Images/c252ff4d56ef0b0a5fa9d2df42666978.png)T2】](https://i2.wp.com/fossnaija.com/wp-content/uploads/2018/02/brlCAD_banner_fossnaija.png?ssl=1)

BRL-CAD 是一个跨平台的 CAD 工具，可以追溯到 1979 年，尽管在开源许可下发布源代码需要 25 年时间。事实上，BRL-CAD 是如此的古老，以至于它已经被[认为是目前正在开发的应用程序中最古老的源代码库。](http://blog.openhub.net/2007/08/worlds_oldest_source_code_repositories/)

BRLCAD 最初由陆军研究实验室的 Mike Muuss 开发，几十年来一直被美国军方用于建模武器系统，但它也用于更多的日常设计任务，从学术到工业设计到卫生应用。

那么超过 35 年的发展给你带来了什么？BRL-CAD 由 400 多种不同的组成工具和应用程序组成，分布在 100 多万行源代码中。不是所有的部分都在同一个许可下，许可范围从 BSD 到 LGPL 到简单的公共领域；SourceForge 上项目的[源代码](https://sourceforge.net/projects/brlcad/files/BRLCAD%20Source/)内的[复制](http://svn.code.sf.net/p/brlcad/code/brlcad/trunk/COPYING)文件有更多细节。

## [](#freecad)**FreeCAD**

[![freeCAD banner](../Images/2c1ce7ecc68101d583901a14290592cc.png)T2】](https://i1.wp.com/fossnaija.com/wp-content/uploads/2018/02/freeCAD_banner_fossnaija_1.png?ssl=1)

[FreeCAD](http://www.freecadweb.org/) 是一个[参数化](https://en.wikipedia.org/wiki/Solid_modeling#Parametric_and_featurebased_modeling)开源 CAD 程序，它能够设计“任何尺寸的真实物体”，尽管很明显许多展示的[用户创建的例子](http://forum.freecadweb.org/viewforum.php?f=24)是更小的物体，但没有具体的理由它不能用于建筑应用。FreeCAD 主要是用 C++编写的，如果你是一个 Python 程序员，你会希望利用它的 Python 接口来扩展和自动化 FreeCAD。

FreeCAD 可以导入和导出各种常见格式的 3D 对象，其模块化架构使其易于使用各种插件扩展基本功能。该程序有许多内置的界面选项，从草图编辑器到渲染器，甚至机器人模拟能力。目前，FreeCAD 正处于测试阶段，正积极开发常规版本，但开发人员警告说，它可能还不适合生产使用。FreeCAD 的[源代码](https://github.com/FreeCAD/FreeCAD)托管在 GitHub 上，并在 [LGPL](https://github.com/FreeCAD/FreeCAD/blob/master/COPYING) 许可下作为开源提供。

## [](#librecad)**自由 eCAD**

[![libreCAD banner](../Images/cabffca7be590ded139f65f1c8a25725.png)T2】](https://i0.wp.com/fossnaija.com/wp-content/uploads/2018/02/libreCAD_banner_fossnaija_2.png?ssl=1)

LibreCAD 是另一个 CAD 程序，它可以在 Windows、Mac 和 Linux 上运行。作为 QCAD 的一个分支(下面会提到)，LibreCAD 有一个 AutoCAD 用户熟悉的界面，默认情况下，它使用 AutoCAD DXF 格式进行导入和保存，尽管它也可以使用其他格式。

不过，LibreCAD 只适用于 2D，所以如果你打算用它来做平面图或类似的东西，嗯，平面图会更有意义。LibreCAD 是在 [GPL](https://github.com/LibreCAD/LibreCAD/blob/master/LICENSE) 下授权的，你可以在 GitHub 上找到它完整的[源代码](https://github.com/LibreCAD/LibreCAD)。
