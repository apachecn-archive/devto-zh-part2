# 我为什么喜欢网络:兴趣、成就和目标🎯

> 原文：<https://dev.to/mauricehayward/why-i--the-web-interests-accomplishments-and-goals--jg9>

我喜欢网络，因为它让我可以接触到资源，实现我的抱负。现在，我正在学习法语，我真的很想生活在一个法语国家，让自己沉浸在这种语言中。使用网络，我不仅可以从事我喜欢的职业，还可以在任何我想去的地方工作！🏖️

网络给了人们选择的自由。这就是为什么用户体验(UX)和可访问性是重要的。每个人，无论是否残疾，都应该能够平等地享受网络并获取网络资源。

## UX 令人激动！

UX 就是根据用户的需求和反馈来做决定。过去几个月，我一直在深入研究 UX 的基本面。我喜欢它！

我觉得这种以用户为中心的心态真的让我成为了一个更好、更快乐的开发者。我从尝试制作大量功能转向尝试对用户产生最积极的影响。如果你问我，后者是一个更好的成功标准😀。

## 过去一年的成绩！

这是我吹嘘自己过去一年成功的地方！

#### 1。我成功地把实习变成了全职工作！

2017 年 1 月大学毕业后，我花了 5 个月自学 web 开发，找工作。之后，我找到了一份前端开发实习，做了 7 个月。2018 年 1 月，我实习的地方给了我一份全职工作！我的正式头衔是 UI 开发者(前端开发)。

#### 2。我在工作中发起并实施了响应式设计。

我说服产品负责人给我时间来构建响应式设计。这一重大变化为这个项目提供了更多的机会，让它对用户更有用。

#### 3。我提高了团队中的 UX 意识。

在我的文章中，你在无意识地模仿 UX😴，现在让我们使它有意识⏰而且是故意的💪，我讲述了我是如何发现自己已经在不知不觉中应用了 UX 原理，然后将它介绍给我的团队。这让我们更加关注我们的用户体验。(如果你想了解更多这方面的内容，请阅读文章！)

#### 4。通过帮助解决 [#CauseAScene 运动](https://hashtagcauseascene.com/)，我提升了我的设计&开发技能。

在这个项目之前，我从未与 Wordpress，php，字体或从头设计一个网站。在各种线上和线下资源的帮助下，我能够迭代地为 **#causeascene CONF** 创建一个网站。这个网站现在正被用来扩大和造福世界各地的边缘人群的声音！这里是[会议页面](https://hashtagcauseascene.com/conference/)，这里是[会议页面](https://hashtagcauseascene.com/event/san-francisco-2018/)。(请支持 **#causeascene CONF** 来到你的小镇！)

#### 5。我和我的同事为 IT 暑期实习生发起了一个导师项目。

对我们来说，导师是指你可以与之交谈、倾诉、学习的人，而不是你的经理。在前一年的实习中，我和我的同事感到需要导师的帮助。所以我们开始了一个每周一次的项目，实习生可以在那里得到他们的问题的答案。

我们的目标是创造一个真实的、稍微非正式的环境，实习生可以提问而不用担心被评判。我们试图回答尽可能多的问题，并邀请有资格的嘉宾回答我们不能回答的问题。

我甚至有机会给实习生灌输。我做了一个关于 JSON 格式的讲座，另一个是关于 UX 基金会的

## 目标

#### 1。为我公司的内部应用程序创建 UX 设计策略

在我的组织中，在组织的业务(面向客户)方面有 UX 和设计团队，但在内部应用(面向同事)方面没有。这让我有了一个雄心勃勃的目标，那就是将 UX 设计融入我们的内部应用文化。为了让这个目标看起来不那么难以实现，我把它分成了几个小步骤:

1.  了解有关业务方面、我们的应用程序用户和客户的更多信息。
2.  调查并确定 UX 将提供最大价值(业务和用户方面)的机会，然后创建价值主张文档。
3.  调查如何将[网页内容可访问性指南(新 WCAG 协议)](https://www.w3.org/WAI/WCAG21/quickref/?versions=2.0)整合到 UX 设计策略中。
4.  启动内部应用程序设计系统的草案或大纲

#### 2。努力成为一名 UX 工程师

现在我是一名 UI 开发人员(前端开发人员),我接受业务需求，并从技术上实现它们。我想扩大这个角色，这样我就能在确保需求和设计为应用程序用户带来价值方面发挥更大的作用。

#### 3。在会议上发言

我以前从未在科技会议上发言，所以我可以等着有机会。我目前有两个提议，一个是关于 Javascript 的闭包，另一个是让 UX 成为一个团队的成果。
