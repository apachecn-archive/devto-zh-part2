# 开发人员回顾#8:本周前 7 项，已讨论

> 原文:[https://dev . to/dango lant/dev to-review-8-本周七大热门话题讨论-51op](https://dev.to/dangolant/devto-review-8-top-7-of-the-week-discussed-51op)

本周，当 Malik 和我再次聚在一起谈论本周关于 dev.to 的 7 篇热门文章时，我们对一组真正杰出的文章进行了热烈的讨论。

[https://w.soundcloud.com/player/?url=https://soundcloud.com/malikanddan/devto-review-episode-8&auto_play=false&color=%23000000&hide_related=false&show_comments=true&show_user=true&show_reposts=false&show_teaser=true](https://w.soundcloud.com/player/?url=https://soundcloud.com/malikanddan/devto-review-episode-8&auto_play=false&color=%23000000&hide_related=false&show_comments=true&show_user=true&show_reposts=false&show_teaser=true)

首先，我们有[“一个开发人员的三个阶段”](https://dev.to/stealthmusic/the-three-stages-of-a-developer-hp4)，它讨论了你在职业发展过程中可能经历的三个主要阶段和态度。

之后，我们继续讨论[“教授函数式编程:两种主要方法”](https://dev.to/eddroid/teaching-functional-programming-two-big-picture-approaches-3nli)，在这里，Ed Toro 为读者提供了两个广泛的框架，通过它们可以理解函数式编程，同时还有一堂生动的历史课和一些实际例子。

在[“没人告诉我的关于成为一名软件工程师的事情”](https://dev.to/anaulin/things-nobody-told-me-about-being-a-software-engineer-3pa5)中，作者详细列举了她迄今为止在职业生涯中学到的实际经验，包括技术和社会经验，这引发了我和马利克之间的一场可能持续了一周的对话。真的是必读。

继续，我们讨论一个在 dev.to 上发布的有趣的新项目，发布在帖子[“Gitote 现在出来了！”](https://dev.to/gitote/gitote-has-been-launched-m35)，并且说说社区驱动项目的价值。

用[“Unix 方式...或者为什么您实际上想要使用 Vim"](https://dev.to/gypsydave5/the-unix-way-or-why-you-actually-want-to-use-vim-3n8p) ，我们谈论一个用户对 Vim 的核心优势和 Unix 哲学的想法，以及这与我们之前关于可组合性的对话的关系...

这让我们很自然地进入了一个关于[“探索 Vim”](https://dev.to/vintharas/exploring-vim-4k1i)的密集而精彩的帖子，作者在帖子中回顾了 Vim 的简史，他自己的旅程，以及关于从它开始并坚持使用它的提示&。

最后，我们讨论[“DEV 上令人敬畏的 PHP 资源🚀 🎉 🎢"](https://dev.to/biros/awesome-php-resources-on-dev----3a18)，这篇文章向你指出了 dev.to 上的所有各种资源，你需要从 0-60 岁开始成为一名 PHP 开发人员。

我们本周的“主持人的选择”是[“开发者体验:错误消息”](https://dev.to/stereobooster/developers-experience-error-messages-5cbn)，从开发者生活质量的角度对各种语言的优缺点进行了系统的比较。
