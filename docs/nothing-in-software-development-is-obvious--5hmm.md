有一件事，我也想提一下，虽然可能很明显，但是说出来也无妨，那就是始终应用最小特权原则。

对此罗布回应道:

[![robdwaller profile image](../Images/b98a8480454eb9de42f026cca6783289.png) ](/robdwaller) [ Rob Waller ](/robdwaller) • [<time datetime="2018-02-22T14:59:47Z" class="date-short-year"> Feb 22 '18 </time>](https://dev.to/robdwaller/comment/2c65) 

有意思，多说说。我以前没听说过那件事。

在我们的领域没有什么是显而易见的。专业知识分支到如此多的方向，以至于任何人都可能对任何概念有盲点。软件开发中没有“什么都学”的课程。这是从智慧共享经济中不断获取知识的大杂烩。我们永远不能假设我们在对话中的投入不是在帮助某人第一次抓住一个话题。

我认为 Nick 的评论更像是一个有用的额外提示，给未来想探索更多概念的读者。Rob 的评论很好地说明了为什么这很有价值。

软件开发知识得到了传播，因为我们在如此多精彩的在线对话中变成了墙上的苍蝇。每次你以这种方式向世界发出信号，你都在为这个领域的未来做出贡献。最小特权原则足够标准[到有一个维基百科页面](https://en.wikipedia.org/wiki/Principle_of_least_privilege)，一旦你理解了它，这是一个相当直观的概念，但这并不意味着人们*就知道它*。可以想象，一个开发人员可以在他们职业生涯的第一天或第 5000 天学习这一点。你只是不知道。

花时间有益地提及你认为可能是显而易见的事情，对网络上的潜伏者可能会有难以置信的帮助，他们今天可能会学到一些新东西。这也是一个重申你在这个问题上的正确性的机会，只要后续的评论是建设性的。

不断分享智慧，这是我们成长的方式。
