# AWS 在 5 分钟内

> 原文：<https://dev.to/itscoderslife/aws-in-5-minutes-lif>

这篇文章将让你对什么是亚马逊网络服务有一个基本的了解，其中的组件是如何工作的，每个组件在一个非常高的层次上负责什么——所有这些只需要 5 分钟。

这篇文章让你对 aws 中使用的组件有一个简单的了解。当 aws 开发人员在工作中讨论话题和问题时，这肯定会帮助你理解事情。

**EC2**–**弹性云计算**实例是您运行 web 应用服务器或微服务服务器的地方。这是 AWS 的主要组成部分之一。您可以在多个可用性区域中创建多个服务器实例，这样，如果一个区域出现故障，您对最终用户的服务不会受到影响。可以在这里或 ELB 配置会话维护

**ELB**–**弹性负载平衡器**自动在多个目标之间分配传入的应用流量。它处理来自公共网络(用户/移动电话/消费者)的流量。该组件的主要功能是将流量路由到不同的服务器，平衡传入流量的负载。它可以处理单个可用性区域或多个可用性区域中应用程序流量的变化负载。

面向公共网络的 ELB 将位于公共子网内，以使其可以访问。除了此实例和 S3 实例之外，所有其他组件都将无法公开访问，因为它们将配置有专用子网。这将是拥有会话粘性的最佳位置。

**S3**–**简单存储服务**用于存储文档、音频、视频、图像文件等所有静态文件。该服务仅用于存储和检索文件。Glacier 用于连接 S3，以归档最少使用或未使用的文件，从而减少存储空间。

数据库:

**RDS**–**关系数据库**是存储和管理应用程序数据的地方。该服务支持 RDBMS 解决方案，如 MySQL 等。该服务还支持主从架构。

如果 web 应用程序解决方案需要基于文档的数据库，如 DynamoDB 等，则使用 NoSQL 数据库。

**安全组**–用于管理任何人都可以拥有的服务访问权限。理想情况下，每个服务都必须定义一个安全组。

**Route 53** 是一个高可用、可扩展的云域名系统(DNS) web 服务。它旨在为开发人员和企业提供一种将最终用户路由到您的应用服务器的方法。

**IAM**–**身份和访问管理**是您可以添加/更新/删除用户及其权限的工具。通常会有最终用户、管理员和开发人员。

**弹性缓存**是一种托管的、内存中的数据存储服务。您可以选择 Redis 或 Memcached 来支持实时应用程序。ElastiCache 是游戏、广告技术、金融服务、医疗保健和物联网应用的热门选择。

对于管理员: **CloudWatch** 针对 AWS 云资源和您在 AWS 上运行的应用程序的监控服务。它会关注你的整个设置，如果使用 **SES** 服务的服务器上出现任何警告或警报，它会使用 **SNS** 服务触发通知，给管理员发电子邮件。

服务通常用于采用无服务器架构设计的解决方案。

希望这在某种程度上会有用，至少让你对 AWS 有一个初步的了解。
