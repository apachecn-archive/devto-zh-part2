大家好，

首先，我想说我很高兴在这里写我的第一篇文章！在我来到这里的短暂时间里，我注意到这是一个多么伟大的社区，我希望积极参与进来！

所以，言归正传:我计划开始写一些我自己的项目，只是一些我想了解更多的一般主题，以及我将如何将其应用于数据集。这最初是作为一个个人的、私人的事情来计划的，但是我注意到获得一些在线的存在是很方便的。既然这些东西可以很容易地结合在一起，为什么不直接去做呢？

我写了一些基本的帖子来尝试 GitHub 页面上的功能，现在我有足够的信心向你展示我的网站的第一次设置。它离我想要的结局还差得很远，但是我想得到一些关于它当前状态的反馈和建议。此外，我不认为我会保持目前的网站名称，但我们会看到未来的道路...

我最大的疑问之一是我如何向我的读者展示使用过的代码:现在，我有一种剧透元素，它隐藏代码，直到用户点击它。我选择这种方法是为了增加文章本身的可读性。下面是我[最新帖子](https://mvdsman.github.io/blog/Simple-pairwise-alignment/)的一个例子。

当然，当我试图将相同的帖子发布到其他平台如 Dev.to 使用 Atom 时，我必须做一些手动编辑，因为剧透没有被很好地导入。在这些情况下，我认为只保留显示段落间代码差异的标准方法。

那么，你的方法是什么，你对我决定如何组织我的帖子有什么看法？

PS:对其他事情的任何反馈也非常感谢！
