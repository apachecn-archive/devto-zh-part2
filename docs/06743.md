# 什么是 AWS S3？

AWS**S**imple**S**storage**S**service，称为 **S3** ，是一个对象存储解决方案，可以可靠地存储和检索任意数量的数据。

# 它能做什么？

它可以存储您的文件，构建包，报告，图像和任何类型的数据，你可以想到以后检索，保证 99.999999999%的持久性。此外，它可以使用非常灵活的规则来限制对特定数据的访问。

# 为什么要用？

S3 极其可靠和灵活。如果您在某个时候需要存储和检索:

*   用于分析的海量数据
*   供客户访问的文件
*   为您的应用程序构建文件
*   图像和视频等资产

此外，它还与 AWS 中的其他服务进行了大量集成。

# S3 的一些基本概念

*   ## 木桶

    在某种意义上，Bucket 就像 Linux 中的顶级目录，它是第一级中的一个节点。那么，bucket 就是存储一组对象或更多目录的“目录”(没有引号，因为现在这些是正确的目录)。更多澄清如下。

*   ## 物体

    对象就是文件本身。一个桶可以包含许多不同的对象。

*   ## 斗策

    S3 存储桶策略是一组规则，明确定义了谁有权或无权访问您的存储桶以及该存储桶中的对象。

*   ## ACL

    一个访问 **C** 控制 **L** ists 是定义对你的桶和对象的访问的另一种方式。然而，这是一种遗留的访问控制机制，所以最好关注 bucket 策略。例如，需要 ACL 的情况是当存储桶策略变得太大时——它们的大小被限制在 20 kb。或者当您希望限制对存储桶中特定对象的访问时。

*   ## 生命周期规则

    生命周期规则是 S3 可以根据对象存储的时间对一组对象应用的操作。例如，您可以删除在您的存储桶上超过 90 天的对象。

除了这些概念，关于 S3 最重要的事情之一是它的一致性模型。

S3 的一致性模型称为**写后读一致性**。例如，如果您发出一个 PUT 请求来在 bucket 上创建一个对象，那么下一个 GET 请求将**总是**拥有想要的对象。

但是，如果您尝试先发出 GET(对象不存在)，然后发出 PUT，再发出另一个 GET，您*可能*没有找到想要的对象。按照这个顺序发出请求，一致性模型现在被称为最终一致性。

换句话说:

写后读一致性:

*   PUT /my-bucket/photo.png -> 200
*   GET /my-bucket/photo.png -> 200

最终一致性:

*   GET /my-bucket/photo.png -> 404
*   PUT /my-bucket/photo.png -> 200
*   GET /my-bucket/photo.png -> 404

当您删除一个现有的对象时，您还可以获得最终的一致性。也就是说，您*可能*看到一个对象列在您的存储桶上，尽管您几秒钟前已经删除了它。

这是因为对 S3 存储桶所做的更改需要一些时间来通过 AWS 服务器传播和复制。

# S3 的一些用例

*   存储 AWS 负载平衡器日志以供进一步分析
*   托管静态网站
*   存储静态网页的资产
*   为客户存储导出的数据
