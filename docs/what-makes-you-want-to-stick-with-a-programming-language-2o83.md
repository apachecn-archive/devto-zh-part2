# 是什么让你想坚持使用一种编程语言？

> 原文：<https://dev.to/17cupsofcoffee/what-makes-you-want-to-stick-with-a-programming-language-2o83>

我之前的帖子产生了很多非常有趣的评论，所以我带着另一个语言设计问题回来了——尽管这次有点抽象！

> 除了写“Hello，world”之外，还有哪些因素让你想继续在一门编程语言上投入时间？或者反过来说，是什么事情让你扔下一堆砖头一样的语言，跑回到`${FAVOURITE_LANGUAGE}`温暖诱人的怀抱？

对我来说，在早期阶段让我保持动力的最重要的东西是文档——特别是，写得很好的指南/教程，而不是简单的 API 文档。我注意到最近几年有一种趋势，那就是语言会有一本“书”,介绍语言的所有特性，我非常喜欢这个概念！这方面最好的例子是 Rust 编程语言，它非常好，当它出来的时候，我可能会订购它的印刷本< 3

另一方面，我认为让我远离一门语言的主要原因是糟糕的工具(包管理、命令行工具、编辑器扩展等)。).这足以浇灭我对其他美妙语言的热情——我喜欢 Lua 的设计，但我发现每当我使用它时，我真的很想念代码完成！

其他人呢？
