# Web 开发中的后端是什么？

> 原文：<https://dev.to/flippedcoding/what-is-the-back-end-in-web-development-4afp>

前端和后端开发之间的划分很强。因为他们处理完全不同的事情。你可能已经了解了前端开发。这就是 HTML、CSS 和 JavaScript 的全部内容。对我来说，这是 web 开发中相对较酷的部分。

但是就像冰山一角一样，幕后发生的事情比任何用户需要知道的都要多。后端是所有这些数据库调用和身份验证以及其他有助于保持数据安全的验证发生的地方。这是每个网站背后的隐现。

您通常只需要关注一种后端编程语言，这比前端所需的三种语言更容易学习。您必须注意的一件事是不同的后端框架。

有了 C#，你就有了 MVC 框架、ASP.NET 和其他任何 ASP。在 PHP 中，有 Laravel、Cake、Yii 和其他一些听起来很有趣的名字。我都不打算碰 Java。它可以留在那里。

但是，当您开始在后端工作时，您将专注于处理数据。每次有人提交表单时，您都必须确保信息的格式正确，没有恶意代码，并且被写入正确列的正确表格中。

您还将负责从数据库向用户提供数据。如果你曾经在网上登录你的银行账户，你会看到数据库被查询。作为后端开发人员，您需要知道如何将正确的数据与正确的用户凭证相匹配。如果你搞砸了，人们的敏感信息很容易被黑客窃取。这就是为什么后端开发人员更诚实地获得报酬。

作为后端开发人员，你甚至不需要了解前端开发，反之亦然。这两层是如此的分离，以至于你可以在不接触另一层的情况下掌握其中的一层。当你在后端写代码时，你可以通过 API 让前端开发者访问它。这样，他们可以获得制作用户界面所需的信息，而您可以完全控制如何引入、过滤和显示数据。

如果你既能做前端开发又能做后端开发，那你就相当不可思议了。尤其是如果你能在高水平上做到这两点。通常，当你开始着手一个更大的项目时，选择一个方面来关注，然后开始涉足另一个方面是很好的。后端开发有一点危险。

与前端不同，后端编程语言非常不同。PHP 甚至不是面向对象的，所以有这种情况。当你承诺学习一门后端语言时，确保你正在学习的这门语言能让你得到一份你可能想要的工作。一些语言在不同的领域更受欢迎，在你真正开始投入时间和金钱学习一门新技能之前，了解这一点是很重要的。

希望这是一个有点深刻的见解。你们觉得怎么样？不管你走哪条路，你都不会错的。前端和后端开发人员都有这样的需求，这有点奇怪。
