# 跳水前先划水

我认为作为一个程序员新手，我和许多其他人都犯了同样的错误。话说回来，我最持续的痛苦就是完全缺乏计划。

我希望我早一点知道，拿出纸笔粗略地勾画出一个实施方案或想法的简单行为会让我的生活变得更加轻松 **1000** 。

我们很容易对一个想法或一项任务的快速解决方案过于兴奋，一头扎进去，突然撞上一堵砖墙。通过花 *10 分钟*讨论这个想法，一些问题很快就会得到解决，甚至可能出现一些实际的边缘情况。几乎不用说，编程行为的一半以上是关于解决问题的，但是很容易忘乎所以，试图在运行中解决问题。

## 减速

简单地勾勒出几个想法或讨论出一些抽象的属性的行为会迫使你以一定会节省时间和精力的方式来思考问题。

更进一步，征求同事的意见，尤其是如果他们的水平和你差不多，你只会增加他们的意见。

你不是在向他们寻求解决方案，只是多一双眼睛来看看你错过了什么。

通过了解这一点，我可以省去很多麻烦，但这只是过程的一部分！所以，去拿一支铅笔和一些纸，潦草地写下你的宏伟计划，希望能避开陷阱。
