## 我能在这里找到什么？

*   [Express](https://expressjs.com/) 颜色
*   [反应](https://reactjs.org/) + [冗余](https://redux.js.org/)，ES6，异步/等待
*   [Web 组件](https://developer.mozilla.org/en-US/docs/Web/Web_Components/Using_custom_elements)(自定义元素)集成
*   反应路由器
*   国际化
*   厚颜无耻
*   PostCSS (autoprefixer)，所以不需要-webkit、-moz 或其他前缀
*   利用 CDN、[缓存破坏](https://www.keycdn.com/support/what-is-cache-busting/)支持构建脚本配置**开发、试运行、生产**
*   构建将 JS、CSS 与 sourcemaps 捆绑在一起的脚本
*   单元测试[笑话](https://jestjs.io/)，[酶](http://airbnb.io/enzyme/)
*   E2E [柏树](https://www.cypress.io/)测试
*   [ESLint](https://eslint.org/)
*   Ghooks(预提交单元测试和 eslint 验证)
*   代码覆盖率( [https://codecov.io](https://codecov.io) )
*   Travis CI 运行单元和 E2E 测试，并向 codecov 报告

## 快速启动

创建 React + Redux 应用程序结构在 macOS、Windows 和 Linux 上工作。如果有些东西不工作，请提交一个[问题](https://github.com/shystruk/create-react-redux-app-structure/issues/new)。

#### npm

`npm i -g create-react-redux-app-structure`

#### 故事

`yarn add global create-react-redux-app-structure`

```
create-react-redux-app-structure my-app
cd my-app/
npm run fast-start 
```

[http://localhost:8080/](http://localhost:8080/) 会自动打开。
