# MVP 关闭

> 原文:[https://dev.to/kyam/mvp-innit-1b 67](https://dev.to/kyam/mvp-innit--1b67)

美登学院的第三周已经结束了，这是我们第一次完整的从头到尾的网站建设。我们被要求设计和创建一个作品集网站，向未来的雇主展示我们的技能，最酷的事情是这是我们第一次真正实现 Scrum，从 sprint 规划到 sprint 回顾，以及中间所有的实际工作。另外，在实际项目的上下文中使用 Git 使它看起来不那么可怕了(即使我们中的一些人有一些问题必须用...黑魔法)。

我发现设计方面真的很有挑战性。我不仅发现很难动笔并决定一个线框，大约在 sprint 进行到一半时，我意识到我需要一些与我的设计非常不同的东西，并花了大约 15 分钟来确定我是否可以重新开始(当然，我不能)。所以我不得不继续不开心。

<u>好东西</u>
虽然我为这个设计绞尽脑汁，但我从中学到了很多东西(老实说，这也是我来学院的原因)。我试图将内容塞进页面的每个部分，而不是利用空间&图片，并且还认为*使用不同大小和不同方面的组合图片(当然是这样)不会让*感到痛苦。另外，我知道下一次我会在之前，使用大量的媒体查询，让我的初始页面尽可能多的响应不同的屏幕尺寸。

其他“好东西”是团队真正拥抱了 Scrum:没有人在他们自己的世界里工作，我们分享想法和问题，代码审查的负担真正在整个团队中分担。这是一种令人耳目一新的工作方式，感觉是合作而不是竞争。

所以总的来说，有些好东西，有些困难，但我们仍在学习和成长，我迫不及待地想知道接下来的 13 周会发生什么。尽管我对自己的设计不太满意，但最终还是满足了所有的要求。MVP，不是吗。

可怕的消息是，在本周末，我们已经完成了四分之一的进度...
