# 编程很难！

> 原文：<https://dev.to/michael/programming-is-hard-2bph>

在过去的几周里，我一直在教我的姐夫和他的妻子 JavaScript。他们从未接触过任何种类的编程，但却有兴趣学习。挑战不是他们缺乏编程经验，而是语言障碍。

我妹夫的母语是德语，他和妻子用韩语和德语交流。我的嫂子是说韩语的。我的母语是英语，只懂初级韩语和基本的德语短语。

尽管如此，我们还是决定让他们踏上学习 JavaScript 和编程基础的旅程。我选择的教学语言是韩语和英语的混合。

令我感兴趣的是，编程的世界是围绕着英语展开的。这是我在说英语的环境中出生和长大时认为理所当然的事情。

教 JavaScript 中的各种关键词以及它们的作用，尤其是看我嫂子处理她正在学的东西，很有趣。这是因为她必须首先理解关键字是什么，翻译成韩语，这样她就可以用她认为合适的语言来处理它。然后当她要用它做例子的时候，在她脑海中把它从韩语重新翻译回英语。

在教学过程中有帮助的是，通过使用愚蠢的例子和给他们简短的练习来强化概念，努力使学习过程变得有趣。因此，如果我的解释在翻译过程中有任何遗漏，例子和练习会强化或纠正他们的理解。

我也很好奇语言障碍是否可以通过使用图表和绘图来缓解，或者像变量名这样的东西是否可以用学习编程的人说的母语来表达。

出于好奇，我试着创建了一个名为“한”的变量，并为它设置了一个字符串“Han”。“한”是韩文的第一个音节，韩文是韩国字母表中的单词。然后我在“한”上运行了一个`console.log()`，果然我得到了返回给我的值“Han”。

```
let 한 = "Han"
console.log(한)
// > Han 
```

我也很好奇世界上有多少非英语母语的人想学习编程的材料。我个人并没有寻找这样的资源，因为对我来说从来都不需要，但是在教我的姐夫和嫂子的时候，我确实认为需要这样的资源。

如果你的母语不是英语，并且曾经学过编程，我很好奇你是如何学习编程的，以及在你的旅途中你找到了哪些有用的资源。你能发推特给我@michaelsoolee 让我知道吗，这样我就可以把这些信息传递给其他可能正在寻找它们的人。此外，如果在旅程的早期有什么特别帮助你学习的东西，请告诉我。

编程很难，我不知道(直到现在)它可以更难。
