*有哪些不使数组再次变异的 Javascript 数组方法？*

为什么页面加载需要这么长时间？

为什么我的代码不起作用？！

我们都经历过。想象一下:当你发现一个问题时，你正在编码。您的代码中的某些部分不起作用。什么？这是怎么回事？所以过了一段时间(或者几天)，你仍然不知道为什么代码不工作。你是做什么的？

不要担心！有很多很好的资源可以帮助你。有些你可能已经熟悉了，有些你可能不熟悉。以下是我在需要帮助时经常使用的一些资源。

## [](#google)**谷歌**

在寻找编码问题的解决方案时，这可能是最常用的(也是最明显的)方法。然而，你会惊讶于 Google 在寻找编码问题的解决方案上是多么的有用！通过谷歌，你可以筛选大量的资源，从博客到 YouTube 视频，再到解决类似问题的 StackOverflow 解决方案。

记不住一个代码语法？谷歌一下。不知道控制台出现错误意味着什么？谷歌一下。不知道为什么你的代码中的某些东西不能工作？很有可能，其他人也经历过同样的事情，或者写了博客。谷歌一下！

## [](#blogs)**博客**

有很多博客都有一些非常棒的文章，涵盖了前端、后端以及两者之间的所有内容。以下是我最喜欢的一些:

#### [](#css-tricks)[**CSS 诡计**](https://css-tricks.com/)

这是我作为一个新手遇到的第一批网络开发博客之一，直到今天，它仍然是教程、指南和学习 CSS 小知识的绝佳资源。您可以找到教程、指南、CSS 参考指南、CSS 片段、视频截屏等等。虽然该网站主要关注 CSS，但您可以找到其他主题的教程，如 React 和 Vue 等 JavaScript 框架。

#### [**stack overflow**T5】](https://stackoverflow.com/)

一个开发者社区，他们互相帮助解决编码问题并分享他们的知识。虽然我不是 StackOverflow 的成员，也没有在那里提问过，但是在寻找各种编码问题的解决方案时，我会广泛使用它。

#### [](#medium)**中等**

世界各地的人们在 Medium 上撰写了数以千计的故事，涵盖了教育、医疗保健、经济、宗教和政治等众多主题，其中有许多出版物涵盖了发展和技术相关主题。其中包括:

*   Freecodecamp :这里最受欢迎的关于开发媒介和技术相关文章的出版物之一。老实说，我怎么推荐 Freecodecamp 都不为过！Freecodecamp 不仅收集了大量涵盖各种主题的文章，还提供了完整的网络开发课程(免费)，一个包含教程和讲座的 YouTube 频道以及一个绝对令人惊叹的播客。如果你还没见过 Freecodecamp，那就去看看吧。相信我，你不会失望的！
*   Codeburst 展示了来自世界各地的网络开发教程和文章。你会发现每天都有许多主题的文章和教程发表，所以你一定会找到你想要的。(免责声明:我是 Codeburst 的作者，但是这是一个很好的资源！).

#### [**dev .**](https://dev.to/)

Dev.to 是开发人员寻求帮助的另一个令人惊叹的平台。该平台不仅有大量以开发为重点的文章和教程，而且还允许您通过启动一个概述您遇到的问题的线程来寻求平台上其他人的帮助。平台上的人都非常乐于助人，支持我们，不做评判。

该平台允许您使用标签在网站中进行搜索，标签代表平台中涵盖的主要主题，例如 React。他们还启动了直播流，有经验的开发人员在那里教其他人编码，这不仅允许你学习新东西并与观看直播流的其他人互动，还可以与正在教学的开发人员互动并向他们提问。在过去的 9 个月左右的时间里，我一直在 dev.to 上发帖，我看到这个平台呈指数级增长，并在这段时间里做了令人难以置信的事情。对于从新手到有经验的所有级别的开发人员来说，这是一个不可思议的资源，并且对每个人都有好处。

## [](#gitter)[**吉特**T5】](https://gitter.im/)

Gitter 是一个开源的网络和即时消息平台，它连接了来自世界各地的开发者。它拥有 100 多万会员，来自 100 多个国家的 30 多万个聊天室。你可以找到一个聊天室来满足你所有的技术需求！

这里有几个 Gitter 聊天室可以帮你入门:

*   [GitterHQ/developers](https://gitter.im/gitterHQ/developers) -开发者通用聊天室。
*   Freecodecamp/help —一个可以获得一般帮助的聊天室。
*   [Freecodecamp/help frontend](https://gitter.im/FreeCodeCamp/HelpFrontEnd?source=orgpage)—您的前端项目需要帮助吗？这是你的地方！
*   [Freecodecamp/help Javascript](https://gitter.im/FreeCodeCamp/HelpJavaScript?source=orgpage)—纠结于 Javascript？当你需要帮助时，这是一个很好的聊天室。
*   [Freecodecamp/code review](https://gitter.im/FreeCodeCamp/CodeReview?source=orgpage)—需要有人审查你的代码吗？从开发伙伴那里获得关于算法挑战解决方案、前端和后端项目的建设性反馈。

## [](#twitter)**推特**

Twitter 不仅仅是一个发布你最近在做什么或者了解世界上正在发生什么的地方。你还可以找到一个由来自世界各地的开发人员组成的令人敬畏的社区，从前端开发人员到后端开发人员再到数据科学家，他们开放并愿意帮助其他开发人员解决他们的编码问题。甚至有些人愿意指导其他需要指导和指引的开发人员。

欢迎提问、提供建议和/或指导的开发人员名单已经整理好，可以在这里找到(感谢二项式的开发人员和创始人斯蒂芬妮·赫尔伯特，她请她的追随者来帮忙！).

## [**DevPal**T5】](https://devpal.io/)

DevPal 由拥有近 10 年经验的前端开发人员安迪·贝尔创建，在这里你可以提出你的编码问题，并从经验丰富的开发人员那里得到答案。

我过去用过 DevPal，有过很棒的体验；安迪给了我很大的帮助，为我的问题提供了我需要的答案。网站上的反应很快，回答也很详细。这是一个很好的资源，尤其是对于代码新手来说！
