## [](#stimulus)[刺激](https://stimulusjs.org/)

> 一个适中的 JavaScript 框架

[![Stimulus](../Images/f530af9652824c4a58bf37df0a0f54fc.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--4UZ5K-IU--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/qqkck8q3atdiqul2rp2s.png)

Stimulus 侧重于为现有的 HTML 添加功能。这意味着即使激励还没有加载，你的代码也会立即显示。这是与 React、Vue、Angular 等不同的处理前端 web 的方式。刺激计划是由 Basecamp 建立的，所以我们可以有某种保证，这个项目将会得到很好的支持。

从 **[刺激手册](https://stimulusjs.org/handbook/introduction)** 开始学习刺激。

## [](#flutter)[飘起](https://flutter.io/)

> Flutter 是谷歌的移动 UI 框架，用于在创纪录的时间内在 iOS 和 Android 上制作高质量的原生界面。Flutter 与现有代码一起工作，被世界各地的开发人员和组织使用，并且是免费和开源的。

[![Flutter](../Images/7278d616e2b456be1d73c7638152499a.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--ix44wt6K--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/wh347srqfiv7ry6o06q8.png)

颤振现在，终于，在测试中！它允许我们使用 Dart 语言创建带有 web 代码的移动界面。它已经成为 alpha 版本有一段时间了，我很高兴看到这个社区是如何围绕它发展的。我唯一的缺点是，我们仍然需要学习 Dart，但是它与 JavaScript 并没有太大的不同，所以它可能并不难学。在我看来，React Native 是一个非常强大的竞争对手。如果你是一名 React 原生开发者，想要尝试一下 Flutter，请查看 React 原生开发者指南中的 [Flutter。](https://flutter.io/flutter-for-react-native/)

## [](#react-static)[反应静止](https://react-static.js.org/)

> React-Static 是一个快速、轻量、强大的框架，用于构建静态渐进的 React 应用程序和网站。它被精心设计以满足最高标准的搜索引擎优化、网站性能和用户/开发者体验。

[![React Static](../Images/ddea0e0c1294d3e100b15b2c0a2fa8ff.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--ONfCw95J--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ed1ch26h92svyqgusah0.png)

如果你做静态网站，比如 React，你可能曾经用过盖茨比。React Static 有望成为比 Gatsby 更快、更轻、更用户友好的替代品。我正在一个博客项目中使用它，我喜欢目前为止的体验。它非常容易使用。如果你用 React 制作静态站点，我鼓励你尝试一下。

## [](#static-gen)[静创](https://www.staticgen.com/)

> 顶级开源静态站点生成器

[![StaticGen](../Images/0eb2fb62894e0dc93cf93effacfa8ce6.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--QqJfjnyo--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/p1jnofsc1v1rhsmrwzi0.png)

当我们谈到静态站点时，Netlify 建立了一个站点，列出了最流行的静态站点生成器，如 Jekyll、Hugo、Hexo、Gatsby 等等。你可以通过星号、叉号等对列表进行排序。并按语言过滤。我个人最喜欢的是 React Static。

## [](#publii)[Publii](https://getpublii.com/)

> Publii 是一种新的博客体验；一个直接在桌面上建立静态网站的应用程序。

[![Publii](../Images/a77449fdc7649919cd628e6c459b2367.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--xhGQrVS0--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/277a7lennp7baax09lab.png)

对于想写博客但不知道如何编码的人来说，Publii 是一个很好的选择。Publii 是一款面向 Windows 和 Mac 的免费开源应用，允许用户离线创建内容，然后上传到 S3、Netlify、Github 页面，或者只是通过 FTP 上传到共享的托管服务器。它使用起来很有趣，并且有很棒的现有主题。这是一个很好的博客应用程序。

## [](#headlesscms)[headlessCMS](https://headlesscms.org/)

> JAMstack 网站的内容管理系统列表

[![headlessCMS](../Images/42e2502b542be28d87de65160816b9fe.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--2tmG_AOD--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/0r451k9853l5o8ehet91.png)

Headless CMS 系统允许我们在不编码的情况下轻松地添加内容——对于不知道如何编码的内容管理员尤其有用。WordPress 与传统 CMS 系统的不同之处在于，一个无头 CMS 只返回数据，而不是一个网页，这使得它非常有效，并且可以添加更多的优化。然后，您可以使用静态站点生成器来解析和呈现由 Headless CMS 返回的数据。现在，您可以完全控制内容的设计和显示。

[![Prismic](../Images/2b31c0b26b47ac64d5f1441fb303944e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--UlKU9CQM--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/143jngv2uqflqxufbr19.png)

Netlify 的 headlessCMS 网站列出了最流行的 Headless CMS 库和平台。您可以根据类型、它使用的静态站点生成器以及它拥有的许可来过滤它们。我试用过 **[Prismic](https://prismic.io/)** ，因为谷歌、易趣、福特和其他公司都在使用它——而且它与 React Static 集成得很好，React Static 有一个我更喜欢使用的 Prismic starter pack。我用 Prismic 的唯一缺点是，如果有 1 个以上的用户，你必须付费，但因为只有我一个人，所以没有问题。你一定要看看这个。

## [](#skaffold)[斯卡福德](https://github.com/GoogleCloudPlatform/skaffold)

> Skaffold 是一个命令行工具，有助于 Kubernetes 应用程序的持续开发。您可以在本地迭代您的应用程序源代码，然后部署到本地或远程 Kubernetes 集群。Skaffold 处理构建、推送和部署应用程序的工作流。它还可以在自动化环境中使用，例如 CI/CD 管道，以便在将应用程序转移到生产环境时利用相同的工作流和工具。

如果你和 Kubernetes 一起工作，斯卡福德是一个真正的享受。它由 Google 构建，允许您轻松设置您的开发和生产流程，对我来说，使用 Kubernetes 进行开发有时很困难，因为您必须以某种方式设置它，以便让代码中的更改真正反映在开发环境中，因为您正在使用 Docker 容器。Skaffold 的一个特性是在代码重建时重建受影响的容器，因此我们不需要为此使用卷和其他东西。一旦我们安装并配置了 Skaffold，我们只需设置一个`skaffold.yml`文件并运行`skaffold dev`。

## [](#roughjs)[Rough.js](https://roughjs.com/)

> 创建具有手绘草图外观的图形

[![RoughJS](../Images/a94e5d761127706f449a7d31b18cde8c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--NCsVyADw--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/23evo26d3upnc2ve3qcq.png)

## [](#pagedraw)[PageDraw](https://pagedraw.io/)

> 毫不费力地将模型转化为功能代码

[![PageDraw](../Images/33044fbe03b6039b158100535c9b141f.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--iBwvUv59--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/raqhg40rt9450x66o64r.png)

PageDraw 无疑是我本周的最爱之一。它将你的草图模型转换成 React 组件！对其他模型格式、React Native 和其他前端库的支持即将到来！如果你有不编码的设计师，这肯定是有用的。您只需将它插入其中，进行一点配置，然后就可以得到一个可以导入的 React 组件。如果你是 React 开发者，一定要看看这个！
