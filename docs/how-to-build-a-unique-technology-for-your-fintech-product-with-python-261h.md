# 用 Python 构建金融科技产品的独特技术

金融科技是个迷宫。对于软件开发来说，这是一个激动人心且极其复杂的行业。存在州级法规、与不同服务和机构的集成、银行 API 连接等。去处理。另一个挑战是经营金融、抵押贷款、投资等业务所需的终端用户的高度信任。反过来，这些又要求最高级别的安全性、功能性以及与需求的一致性。

我想说的是，软件越独特，价值越高。没有一个正常工作和值得信赖的软件，任何金融风险都会逐渐消失，失去价值。人们需要持久的金融技术，我将告诉你我们是如何在开发金融科技产品时利用 Python/Django 技术栈实现这一点的。在 Python 已经成为世界上最流行的编码语言之后，说起来特别令人愉快。

## 金融科技:独一无二的重要性

在金融界，有两股潮流仍然共存。一方面，千禧一代在掌握非接触式支付、使用网上银行和各种数字金融服务的同时，辉煌地迈向未来。为了避免老派的官僚作风，他们以一种前无古人的方式构建自己的生活。

另一方面，还有传统的融资服务。这是一个地狱般的机器，数百年的历史，你不能轻易停止。即使它承认新技术对金融的影响，它仍然不认为这是一种威胁，也不是一个有价值的竞争对手。

这种态度在最发达的国家尤其典型，比如 G7，它们拥有最多的资金。我要补充的是，大部分的旧钱。以及大多数准备好运营它的人和最高科技的创业公司。然而，问题是，他们的金融体系是如此陈旧和僵化，以至于它并不总是准备好改变。

德勤在其 2017 年的[统计数据中证明了这一点，该数据准确显示了 G7 如何看待和使用金融技术，而不是世界其他地区。德勤的研究人员指出:](https://www2.deloitte.com/insights/us/en/industry/financial-services/talent-technology-future-of-financial-services-survey.html)

> *“令人惊讶的是，关于移动支付，40%的美国高管认为对他们的行业影响很小甚至没有影响。需要注意的是，样本规模相对较小，17 家美国银行中的 7 家(41%)几乎没有受到移动钱包和其他支付技术的影响，而 36 家非银行银行中的 14 家(37%)没有受到影响*

与此同时，发展中国家的金融部门存在许多黑洞，为增长提供了空间。这些黑洞正在缓慢但肯定地被金融科技接管。通过这样做，它给这些国家的人们提供了更多的机会，比如与发达国家合作，轻松而安全地获得报酬。金融科技消除了金融边界，这是我最喜欢的一点。

无论 G7 对金融科技有多么怀疑，科技仍在改变着金融。原因之一是技术更加灵活，能够适应新用户的需求，比如千禧一代的需求和诉求。这一代人有着新的习惯、高度的数字敏感度和数字存在，他们觉得有必要在醒着的每时每刻都保持高效，浪费不起时间，经常旅行，无论身在何处都重视财务自由。

举例来说，[华尔街日报](https://blogs.wsj.com/experts/2018/06/07/the-uncomfortable-relationship-between-mobile-payments-and-financial-literacy/)证实了我的想法，称支付的便利性吸引了那些对技术感到舒适、生活忙碌的人。移动支付的用户大多受过高等教育，全职工作，以男性为主，并且在金融方面也非常活跃。例如，与非移动金融应用用户相比，他们更有可能拥有银行账户、退休账户和/或自有住房。同样，他们对汽车贷款和抵押贷款并不陌生。

《华尔街日报》在文章中使用的统计数据基本上表明，**移动支付用户在金融方面更活跃，**使用各种金融产品，比非用户赚得更多。与此同时，他们对自己的开销更加粗心，负债累累，甚至从自己的退休账户中取钱。这就是为什么专家预计金融科技将出现一个全新的领域——帮助千禧一代更好地管理他们的资金的简单工具。据报道，使用移动支付的千禧一代面临更大的财务困境和管理不善的风险，尽管他们的收入和教育水平更高。

在数字颠覆的时代，金融业必须对新的客户需求特别敏感。当你的服务变得更加普遍和必要时，他们会使用你的服务吗？你现在能创造一个产品，当千禧一代长大并开始赚大钱时，它能成长和发展以服务他们吗？与当前金融体系所针对的一代人一样。当涉及到像**抵押贷款、投资和财富管理这样的分支机构时，这一点尤其重要。**

正如我上面所说的，提供一种为满足客户需求而定制的独特技术的重要性怎么强调都不为过。到目前为止，不可能避免与传统金融和国家机构的整合。你必须确保你们的合作进展顺利，并且他们认为你足够可靠，可以选择你作为合作伙伴；选择你的技术而不是别人的，或者更糟，创造自己的技术。

当 Moneypark 收购 Defferrard & Lanz 并成为 Helvetia 的一部分时，我们意识到了顶尖技术的重要性，money park 以前是 urs 的初创客户，现在是瑞士最大的技术型抵押贷款中介。所有这一切的发生是因为他们的技术解决方案和商业方法是最有说服力的。

## Python 如何在金融中使用& Fintech

那么，你从哪里得到这种足够强大的技术来承受全球金融动荡的压力，但又足够灵活来适应所有新的变化和客户的需求呢？我们选择在 Python 中使用 Django 框架，并继续发掘它的威力。我们并不想说 Python 是救世主和银弹，但是我们确实知道 Python 对于金融有什么优势。

**1。Python/Django 堆栈让您更快地进入市场。**很简单:这个组合可以让你快速建立一个 MVP，增加你找到适合你的产品/市场的机会。

金融科技相对于传统银行服务的优势之一是其快速变化、适应客户需求以及根据客户意愿提供额外服务和改进的能力。要做到这一点，你必须能够快速进入市场，面对现实生活中的问题，不断改进和成长。这是金融科技能够与传统银行和金融业竞争和/或合作的唯一方式。

该技术必须灵活，并为众多附加服务提供坚实的基础。显然 MVP 是重要的，但是项目的复杂性并不总是允许快速开发它。然而, **Python/Django 框架组合考虑到了 MVP 的需求，并允许节省一些时间。**它们基本上像乐高积木一样工作——你不需要从头开始开发诸如授权或用户管理工具之类的小东西。你只需从 Python 库(Nimpy、Scipy、Scikit-learn、Statsmodels、Pandas、Matplotlib、Seaborn 等)中获取你需要的任何东西。)并打造 MVP。

Django 在 MVP 阶段给你的另一大优势是一个简单的管理面板或 CRM——它是内置的；您只需为您的产品进行设置。当然，在 MVP 阶段，产品并不完整，但是你可以测试并轻松完成它，因为它非常灵活。

MVP 完成后，这个技术栈允许修改部分代码。这意味着在您验证了 MVP 之后，您可以很容易地更改一些代码行，或者甚至编写新的代码行，如果这是产品完美运行所必需的。

千禧一代是习惯生活在快节奏世界的人。他们觉得他们必须在醒着的每时每刻都保持高效，这是他们对其他人和他们所使用的服务的期望。没时间犯错了。最大限度的透明度和高质量的服务对他们来说至关重要，我认为他们不会很快放弃这一点。

比方说，不管我有多爱优步，只要他们犯了一个小错误，比如找司机找得太久，我就会非常恼火。我相信我们都期望并值得比这更好的。我甚至不能开始描述接管人们的恐慌，如果天堂不允许，Slack 崩溃。

**这就是客户发展如此重要的原因——整整一代人都有赖于此。**因此，你越早将产品推向市场，就能越快收集反馈，也能越快做出改进。金融中的 Python 编程，让你双手背在背后就能做到这一点。

**2。Python 是数学家和经济学家的语言。** Fintech 显然离不开这两个群体，他们大部分时间都在使用——等等——Python 来计算他们的算法和公式。虽然 R 和 Matlab 在经济学家中不太常见，但 Python 成为了金融领域最有用的编程语言，[也成为了数据科学的编程“通用语”。因为经济学家使用它来进行计算，当然这使得他们更容易与基于 Python 的产品集成。然而，技术合作伙伴的出现和与技术合作伙伴的沟通仍然很重要，因为有时即使是用同一种语言编写的代码也很难集成。](https://medium.com/@metjush/4-reasons-why-economists-make-great-data-scientists-and-why-no-one-tells-them-524478845ec2)

**3。Python 具有简单的语法，更易于协作。在我看来，成为“通用语”只是时间问题。由于它的简单性和易于理解的语法，Python 非常易读，每个人都可以学习它。Python 的创造者[吉多·范·罗苏姆](https://medium.com/the-renaissance-developer/python-101-the-basics-441136fb7cc3)将其描述为*“一种高级编程语言，其核心设计理念是关于代码可读性和语法，这使得程序员可以用几行代码表达概念。”***

不仅技术专家容易理解，客户也很容易理解。可以想象，双方参与开发过程的人对技术的理解程度是不同的。使用 Python，工程师可以更容易地解释代码，客户也可以更好地理解开发的进展情况。

正如《经济学人》对 Python 的评论:

> 这种语言的两个主要优点是简单和灵活。它简单明了的语法和缩进空间的使用使它易于学习、阅读和分享。被称为 Pythonistas 的狂热实践者已经将 145，000 个定制软件包上传到一个在线知识库。这些涵盖了从游戏开发到天文学的所有内容，可以在几秒钟内安装并插入到 Python 程序中。”

这就引出了下一点。

**4。Python 有开放的库，包括用于 API 集成的库。**开放库有助于在最短的时间内开发产品和分析大量数据，因为您不必从头开始构建工具。这样可以节省大量的时间和金钱，在打造 MVP 的同时尤为可贵。

正如我提到的，金融科技产品需要与第三方进行大量集成。Python 库使得通过不同的 API 将您的产品与其他系统集成更加容易。在金融领域，API 可以帮助您收集和分析关于用户、房地产和组织的所需数据。**例如，在英国，你可以通过 API 获得人们的信用记录，**这是进行进一步金融操作所必需的。在在线抵押贷款行业中，您还需要检查房地产数据，并且您总是需要验证某人的身份，这使用 API 要容易得多。通过使用和组合不同的库/包，您可以一键获取数据或过滤数据，而不必为此开发新的工具。

例如，Django Stars 使用 [Django Rest 框架](https://www.django-rest-framework.org/)来构建 API 或与外部集成，以及使用 Celery 来排队或分发任务。
