# 我的$女巫到$软件开发

> 原文:[https://dev . to/Meg _ gut shall/my-switch-to-software-development-2fm 5](https://dev.to/meg_gutshall/my-witch-to-oftware-development-2fm5)

2017 年底，我在费城外的一家小型 IT 咨询公司担任技术协调员和营销主管，这是我在科技行业的第一个职位。由于我对这个行业一无所知，我决定自己做研究，让自己沉浸在最新的新闻和最佳实践中。我阅读了关于不同技术和编程语言的书籍和文章，参加了该市的许多社交活动，甚至通过当地的社区大学报名参加了计算机科学课程。我从来不害怕提出问题，并建议在公司实施新的概念，但我的老板似乎从来不认可我的努力，事实上还劝阻我的行为，说这不是我的工作描述的一部分。在为公司奉献了一年半之后，我开始意识到我们在面试时讨论的想法永远不会有结果，坦白地说，我觉得自己的角色没有得到充分的重视。在我看来，我非常想成为职业的那个职位并不成功。

作为我工作的一部分，我负责招聘新的顾问和管理薪资，这意味着我对每个人的薪资细节以及他们需要多少年的工作经验了如指掌。这种洞察力，加上我在社交活动中遇到的许多已经转型到科技行业的女性，在我的脑海中引发了一个想法，并让我思考，“为什么我不能也这样做？”

好吧，我博客的标题可能有点误导。我在招聘公司遇到的工资、福利和工作津贴绝对吸引了我的眼球，但在我第一次操作 HTML 代码并看到有形的变化反映在网站上之后……你可以说我爱上了。

我今天的状态始于 2012 年我第一份工作的一个简单项目——一个曲棍球训练设施。在获得一个行政职位的面试机会后，在谷歌上搜索“什么是曲棍球？”，经历了迄今为止我经历的最奇怪的 2.5 小时面试，我被邀请下周一开始工作。我的任务之一是给他们通过 WordPress 开发的网站添加内容。我开始自学如何利用网上提供的大量免费教育信息来浏览这个平台，很快就从更新花名册和张贴图片，到创建锦标赛画廊，添加简化的注册功能(具有在线支付功能)，以及破解打开的文件来调整 HTML 和 CSS 代码。我甚至创建了一个子域来突出俱乐部在新西兰的旅行，其中包括博客帖子和合作画廊。

在第一份工作上干了两年半后，我决定在 2015 年夏天辞职，花些时间去旅行。到家后，我通过 LinkedIn 上的一些关系找到了我在 IT 咨询公司的下一份工作。(网络——真的有用！)尽管我有一大堆的职责(当你在一家小公司做行政管理时，你会身兼数职)，我仍然能够接触到公司网站的代码。我主要是更新博客帖子和调整一些功能，但当我的经理决定雇佣一名专业人员来完全重做我们的网站时，我也有机会进行网站测试。我的工作是在网站上线前尝试打破一些东西，我对此很在行！

2017 年底，我不得不休短期残疾假，我的雇主无法为我保留职位，所以我被解雇了。我挣扎着支付账单，除此之外，我质疑我生活中的一切，以及是什么导致了我现在的处境。在一些密集的自我反思和与职业教练的几次会面后，我发现自己走上了开始软件工程职业生涯的道路。

通过参加网络活动和技术讲座，我学到了软件工程可以用来帮助社会以及改善他人生活的无数方法。费城是一座充满激情的城市——这也延伸到了他们的科技产业和创业文化。无论是解决一个简单的困境，还是为一个系统性问题创造解决方案，我所在城市的人们都愿意分享他们的想法，然后合作并采取行动。我本人有解决问题和帮助他人的热情，我认为这可能是我这样做的完美出路。此外，该领域永远不会有停滞不前的风险，因为技术正日益成为我们日常生活的一部分。

科技行业似乎渴望创新和好奇心，所以这一之前让我受到斥责的特质在适当的环境下可能会产生完全相反的效果。我的一位导师已经在她的社交媒体营销公司向我寻求帮助，包括更新她的网站。现在，我发现当我有非常规的建议或在项目中付出额外的努力时，我要么会得到表扬，要么会得到方向性的反馈，建设性地鼓励我走上另一条路。我不再感到被束缚或被微观管理。再加上当我告诉她我要回学校读软件工程的时候，她为我欣喜若狂！

谢天谢地，我已经学会了在科技行业立足所需的工具(在招聘公司工作时你会学到很多)，现在我只需要发展知识，通过[熨斗学校的在线软件工程项目](https://flatironschool.com/career-courses/coding-bootcamp/online/)，我将获得这些专业知识。
