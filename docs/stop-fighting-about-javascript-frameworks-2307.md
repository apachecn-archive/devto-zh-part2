## 反应 vs 角度 vs Vue vs...

这是一个古老的故事。关于为什么 React 更好，Angular 更好，Vue 更好，Vanilla JS 更好等等，我看了看，看了多少文章，我也说不清。这些职位的范围很广。从非常公平和开放地处理每个框架的最好和最差之处，并邀请用户决定什么对他们有用，到完全有偏见，本质上说框架 X 很烂，框架 Y 是最好的，很少提供推理。

所有这些都是为了培养一种优越感和守门文化。你不是一个真正的开发者，因为你使用了劣质的框架。除非你用的是最热门的框架，否则你不值得我雇佣，也不值得创业。

## 它伤害了所有人

像这样说和宣传想法对整个社区都是有害的。以 JS 调查的状态为例:调查对象的数量大约是 20，000 名开发人员。与行业相比，这是非常小的比例。Stack Overflow 的调查有超过 10 万名受访者。

还有一点要说的是，调查的制作者是拥有大量 React 开发人员的 React 库作者。因此，很自然地，仅仅因为他们迎合的受众，就会有大量的 React 开发者响应并分享他们的调查。

问题是，这项调查是作为 2018 年 JavaScript 的权威观点提出的。如此有限和集中的样本量，结果可能会偏向某些群体。这对于开发人员来说是有问题的，他们拿着这些结果给他们的老板看，说“看看这些结果，Angular 死了，我们需要迁移我们所有的代码来反应或 Vue”。我真的和人们讨论过这个问题。它不是基于什么是最好的选项，而是根据一个声称是 JavaScript 世界权威观点的小型调查或博客得出的最受欢迎的选项。

这对每个人都是有害的，因为公司和开发者会花很多时间去做一些人认为最受欢迎的事情，而不是对公司最有利的选择。例如，从 Angular.js 重新编写一个应用程序以做出反应，尽管由于概念和基础相似，Angular.js 到 Angular 的转换可能更快更容易。我还听说过一个开发人员被一家初创公司雇佣，他告诉他们 Angular 是最好的框架，他们需要用 Angular 重写他们的整个 React 应用程序，这需要花费几个月的时间和金钱。

## 停止热拍

我个人对所有宣称为什么一个框架比另一个好，或者宣称“框架之战”的赢家的权威人士都很反感。在我看来，这是一种我们作为一个社区不需要的守门形式。为什么一定要有战争？我认为没有，或者至少不应该有战争。尤其是 2018 年，React，Angular，Vue，Aurelia 等。都是优秀的现代框架，都有令人惊奇的理由去使用它们。

利用让你开心的事。如果是反应，棱角，Vue，完全不同的东西，如果它是令人愉快的，并帮助你更有效率，那么它是一件好事。让我们请停止使用这些调查和文章来试图说服其他人，你的选择是正确的，他们的是错误的。让我们彼此相爱，爱自己的选择。

*披露:我是 Angular 和 React 的快乐用户。本文无意低调“Angular 为什么没拿第一”。希望听起来不是这样。我只是想帮助结束这场“战争”。*
