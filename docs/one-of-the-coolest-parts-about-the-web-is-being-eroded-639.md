我的意思不是说我们的检查工具越来越差。事实上，他们变得越来越好，越来越强大。我爱我的 Chrome 网络工具和 Firefox 等不仅仅是保持同步。但这通常更多是针对桌面开发的。显然，在手机上检查一个页面的来源更加困难。如果你还没有一个好的设置，这是一件非常痛苦的事情，很多开发者可能从来没有在移动设备上检查过源代码。尽管大多数“工作”仍然发生在桌面上(有争议)，但显然这不是大多数计算机使用发生的环境。

都是史蒂夫·乔布斯的错。我在开玩笑——算是吧。我们知道他和沃兹关于 Apple II 应该有多少插槽的争论，这种争论从未停止过。苹果孜孜不倦地将他们的产品——包括硬件和软件——放进一个黑盒子里。我完全支持用户友好的外壳，但绝对遗憾的是，现在进入引擎盖下是专家的事，而不是任何想偷看一眼的人。

其他公司，如谷歌，也紧随其后。每一个新的操作系统似乎都让检查齿轮变得更加困难，这是一种耻辱。我们保持普通用户快速进入源代码的能力*重要吗？也许是，也许不是。我个人会说是的，其他人也会，但不管怎样，失去这种心态都是一种耻辱。代码检查特性并不需要*首先需要*存在于浏览器中，但是它变成了一个标准的东西，令人难过的是它不再是所有设备的标准。*
