接下来是这篇发人深省的关于测试自动化的文章。作为一个相对较新的概念，看起来我遇到的大多数自动化测试人员都是正在寻找提升技能的测试人员，或者是正在寻找开发途径的毕业生。疯狂的事情是，以前涉足自动化是，这是非常困难的，这在本质上是另一种形式的编程。这篇文章真的让你思考我们在这个问题上的方法，以及本质上我们是如何做错的。

### [Timsort:现实世界问题的最快排序算法。](https://dev.to/s_awdesh/timsort-fastest-sorting-algorithm-for-real-world-problems--2jhd)

## 文章不再可用

算法很棒，所以如果你创造了一个，为什么不用你自己的名字命名呢！这正是蒂姆·彼得斯在 2002 年对 Timsort 所做的。如果你不知道这个故事，或者不知道为什么它是 Python 的事实排序算法，那么请阅读这篇精彩的概述。
