# 新的微软

> 原文:[https://dev.to/jeikabu/the-new-microsoft-17b8](https://dev.to/jeikabu/the-new-microsoft-17b8)

最近宣布 Hyper-V Quick Create 现在支持 Ubuntu ,这让我想起了一些我原本认为是谨慎娱乐的事情，但现在已经存在了足够长的时间，我实际上可以接受了。微软不再是曾经的“邪恶帝国”。

## [](#in-no-particular-order)排名不分先后

C#和 Windows 2k/7 是众所周知的“门槛”。前者(imho)是 Java 本应成为的托管语言/运行时，后者(同样，imho)是来自 Redmond 的第一个操作系统，作为一个最终用户很难真正抱怨它。

微软对 Xamarin 的收购很有趣。然而，除了成为 Visual Studio 的一流组件之外，似乎并没有太多的收获。

Visual Studio 的“Express”版本已经存在了很长时间，但是 [Visual Studio 社区](https://visualstudio.microsoft.com/vs/community/)感觉是第一个可以被专业人员使用的真正产品。

对大多数人来说，Orleans 可能用处有限，但它在分布式系统领域做出了有趣的贡献。

当 Linux 的 Windows 子系统宣布的时候，它可能已经被冻结了。好吧，我实际上没有用它做任何事情，据我所知，它可以与 [cygwin](https://www.cygwin.com/) 或 [msys](http://mingw.org/wiki/msys) 相媲美，但仍然…

在用了 3 年 Emacs 和将近 15 年的 Vim 作为我的通用文本编辑器之后， [Visual Studio Code](https://code.visualstudio.com/) 已经将它们置之不理。

[。网芯](https://github.com/dotnet)好像真的把 C#下的火点着了。我对它的热爱。Net 标准几乎无法包含。

就在前几天我发现 [PowerShell 也是多平台的](https://github.com/powershell/powershell)。我上次(2015 年)看到它时，它的远程功能与 ssh 相比有些复杂/笨拙，但我一直渴望它的命令行参数自动完成。而且，如果 Windows 是你的世界的必需品，那么 cmd.exe 的一流替代品是非常需要的。

注意， [OpenSSH 是一个可选的 Windows 组件](https://blogs.msdn.microsoft.com/powershell/2017/12/15/using-the-openssh-beta-in-windows-10-fall-creators-update-and-windows-server-1709/)。

不能忽略备受争议的 Github 收购案。

荣誉奖颁给了蓝色的。没用过，但听过一般的好东西。老实说，亚马逊/AWS 需要竞争。

只是[“拥抱，延伸，灭掉”](https://en.wikipedia.org/wiki/Embrace,_extend,_and_extinguish)传奇的最新章节？也许吧。
