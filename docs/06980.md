# 开源维护者:你如何处理特性请求？

> 原文:[https://dev . to/Patrick/open-source-maintainers-how-do-you-deal-feature-requests-46mn](https://dev.to/patrick/open-source-maintainers-how-do-you-deal-with-feature-requests-46mn)

我是该领域一个相当受欢迎的开源项目的共同维护者，我们收到了很多功能请求，但我并不总是知道该怎么处理它们。对于一些，我可以清楚地看到优点，它们也是我希望在项目中看到但只是没有想到的东西，我很乐意将它们添加到我们的“功能请求”github 项目中。对于其他人，我可以明确地说，它不会被添加到项目中，因为它非常小众或者完全超出了项目的范围。

但也有介于两者之间的，其中大部分都属于这一类，我不是这一类的粉丝，但我不能清楚地说明和解释为什么不应该添加这一特性。

你们是如何处理这种请求的？你是像我们一样将它们添加到特性请求列表中，还是对你将接受什么类型的特性请求有某种指导方针？此外，如何关闭这些功能请求，而不让用户感觉被关闭或忽略？

感谢您的参与，我真的很期待这次讨论，并想知道您是如何处理这类事情的。
