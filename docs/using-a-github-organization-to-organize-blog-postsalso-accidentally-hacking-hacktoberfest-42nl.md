我发现我的 GitHub 库列表变得难以管理，也许这只是因为我分叉所有的东西而不是开始，尽管有时我也这样做。

我想尝试做一些事情来帮助组织我的存储库——创建一个 GitHub 组织来存放我在博客文章中使用的代码。

这个组织非常简单，希望我这样做不会违反任何规则。现在，我所有博客文章的代码(不多)都组织在一个地方！

[https://github.com/Kritner-Blogs](https://github.com/Kritner-Blogs)是我的组织所在地，你可以看到我目前(在撰写本文时)有两个代码库:

> 在 10 月期间编写代码并提交 PRs，并获得免费的东西！

无意的(但欢迎！)创建 GitHub 组织的结果是，我提交到我的组织/博客帖子存储库的提交(PRs)计入了我 10 月份的 5 个 PRs。诚然，当我发现这一点时，我已经有了 5 个 PRs，但这仍然是一个很好的惊喜。
