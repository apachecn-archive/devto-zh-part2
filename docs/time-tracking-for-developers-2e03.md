# 开发人员的时间跟踪

> 原文：<https://dev.to/olicsan/time-tracking-for-developers-2e03>

一方面，时间跟踪很麻烦。但另一方面，它给你硬数据，可以帮助你确定时间下沉。另外，当你为客户工作时，你必须掌握时间。

## 自动时间跟踪器

跟踪你花在项目和开发上的时间的最省事的方法是在你选择的文本编辑器/IDE 中安装一个自动时间跟踪器 [WakaTime](https://wakatime.com) 。

然后，当你工作和编码时，跟踪器在后台工作并计算你在每个回购/项目上花费了多少时间。

您可以在 WakaTime dashboard 中访问您的时间报告，并查看按日期、项目、编程语言和许多其他标准细分的生产率。

## 项目时间跟踪器

WakaTime 最棒的一点就是你不用动一根手指。但是如果其他人需要你的时间日志呢？例如，您的经理、工资单或客户？

在这种情况下，您可以使用[时钟化](https://clockify.me)。这是一个更加用户友好的项目时间跟踪器，支持团队中的时间跟踪。唯一的缺点是，你必须启动/停止定时器或手动输入小时。

所以，最好的解决方案是:

1.  安装 WakaTime
2.  工作/代码
3.  在一天结束时，检查你在项目/文件/回购上花了多少时间，并在 Clockify 中手动输入时间，以便其他人可以使用这些数据

[![Time tracking in Clockify](../Images/e2cff591a0d28dbf0d647ec99243da43.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--yHT_sK1E--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/q5omftrx791rj5p7q8l5.png)

这样，你就不必记得你在一天结束时做了什么，也不必为启动/停止定时器而烦恼。

## 使用问题跟踪器？

如果你在使用 JIRA、YouTrack、Gitlab/Github 甚至 Trello 这样的问题跟踪器，你可以安装 [Clockify Chrome 扩展](https://clockify.me/chrome-time-tracking)并在 JIRA 或你正在使用的任何东西内为问题启动计时器
