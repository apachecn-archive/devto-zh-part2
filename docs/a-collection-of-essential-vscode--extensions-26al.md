# 基本 Vscode 扩展的集合

> 原文：<https://dev.to/lauragift21/a-collection-of-essential-vscode--extensions-26al>

代码编辑器是一个开发者只要写代码必不可少的工具之一。但是一个有趣的事实是，每个开发人员都喜欢改进他们的工作流程，提高工作效率。我的意思是，既然有工具可以完成任务，为什么还要浪费时间去完成呢？这就引出了 Vscode 的扩展主题。这些工具在帮助开发人员维护他们的代码库和提高生产力方面大有帮助。自从我切换到 Vscode 后，我发现下面的扩展是我的圣杯。我几乎离不开它们，不要误会我的意思，我是说它们是很棒的工具。我们开门见山吧。

**[更漂亮:](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode)** *是一个保持代码库一致性的强大工具。它是一个固执己见的代码格式化程序。它通过解析您的代码并根据自己的规则(考虑到最大行长度，必要时换行)重新打印代码来强制执行一致的风格。*

[![prettier](../Images/4e3d6e9e89e3feeefffd8f233ffdfa3f.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Ml5Xx5uV--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://github.com/remimarsal/prettier-now-vscode/raw/master/prettier-now2.gif)

**[ESLINT:](https://github.com/Microsoft/vscode-eslint)** *我对这个工具又爱又恨，因为它是自动化的，我一眼就能知道我做错了什么。该扩展使用安装在打开的工作区文件夹中的 ESLint 库。如果文件夹没有提供，扩展会寻找一个全局安装版本。如果您没有在本地或全局安装 ESLint，那么可以在本地安装时运行工作区文件夹中的`npm install eslint`,或者在全局安装时运行`npm install -g eslint`。*

[![esint](../Images/ce8a63369d8636dbd573d337b10076bd.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--an8Rpr6P--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://cdn-images-1.medium.com/max/800/1%2AW1gdb9Ae62PH0W2ZnquVCg.gif)

**[GITLENS:](https://github.com/eamodio/vscode-gitlens)** *这个扩展的强大之处是你无法想象的。它能够在 CodeLens 中提供 Git 信息(最近的提交、作者列表)、按需内联责备注释、责备资源管理器，以及将更改与工作树或先前版本进行比较的命令。*

[![](../Images/c5cbe670d839df8b15e0ac9e2d43e010.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--De_WmsyI--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://raw.githubusercontent.com/eamodio/vscode-gitlens/master/images/gitlens-preview.gif)

**[VUE 2 片段:](https://marketplace.visualstudio.com/items?itemName=sdras.vue-vscode-snippets)**
*这个扩展将增压你的 VUE 工作流程。轻松地搭建 Vue 指令，如 v-for、v-if、v-show 等，而无需查看文档来确认语法是否正确。*

**[MARKDOWNLINT:](https://marketplace.visualstudio.com/items?itemName=DavidAnson.vscode-markdownlint)***MARKDOWNLINT 是一个 Visual Studio 代码扩展，它包括一个规则库，以鼓励 Markdown 文件的标准和一致性。它由 markdownlint for Node.js 提供支持，后者基于 markdownlint for Ruby。*
