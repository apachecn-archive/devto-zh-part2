## DevOps 阅读清单:今年你应该读的 30 本最好的 DevOps 书

1.[李·艾奇逊的《面向规模的架构:不断增长的应用的高可用性》](https://www.amazon.com/gp/product/B01IAK49FS/ref=as_li_tl?ie=UTF8&tag=devopslinks0e-20&camp=1789&creative=9325&linkCode=as2&creativeASIN=B01IAK49FS&linkId=7ae0818196e164312964ba6b0f626955&ref=booksfordevopshandcuratedtechbooks)

我们 DevOps 阅读清单上的第一篇文章是《面向规模的架构》。这是一本理解扩展和管理关键应用程序的真实范例的优秀书籍。这本书涵盖了 5 个不同的元素:可用性、风险管理、服务和微服务、扩展应用程序和云服务。这本书也可以称为实用指南，它展示了如何防止应用程序随着增长而变得缓慢、不一致或完全不可用。此外，在这本书里,“扩展”这个词解释得很好，因为它不仅仅是处理更多的用户；它还涉及管理风险和确保可用性。

1.  * * * *[《devo PS 手册:如何在技术组织中创造世界级的敏捷性、可靠性和安全性》,作者:Gene Kim、Patrick Debois、John Willis、Jez Humble、John Allspaw](https://www.amazon.com/gp/product/1942788002/ref=as_li_tl?ie=UTF8&tag=devopslinks0e-20&camp=1789&creative=9325&linkCode=as2&creativeASIN=1942788002&linkId=9406eb8618e5862472fe314d406dd3f1&ref=booksfordevopshandcuratedtechbooks)

我们 DevOps 阅读清单上的第二个是 DevOps 手册。这可能是 DevOps 领域最知名、最有影响力的书籍之一。众所周知，有效的技术管理对企业竞争力至关重要。许多团队和公司仍然在努力平衡敏捷性、可靠性和安全性。DevOps 手册是一个非常好的指南，其中包含了经过深入研究的工作、案例研究和示例。

1.  [AWS 认证解决方案架构师官方学习指南:Joe Baron、Hisham Baz、Tim Bixler、Biff Gaut、Kevin E. Kelly、Sean Senior 和 John Stamper 的助理考试](https://www.amazon.com/gp/product/1119138558/ref=as_li_tl?ie=UTF8&tag=devopslinks0e-20&camp=1789&creative=9325&linkCode=as2&creativeASIN=1119138558&linkId=315510a4cf908b0c0f473dda9437ae2d&ref=booksfordevopshandcuratedtechbooks)

DevOps 阅读清单上的第三个是 AWS 认证解决方案架构师官方学习指南。这是另一份优秀的实用 DevOps 指南，涵盖了多项目标，包括设计高可用性、经济高效、容错、可扩展的系统、实施和部署、数据安全、故障排除等。如果你想学习云计算，丰富你在 AWS 上创建基于云的基础设施和服务的知识，这本书绝对适合你。这本书提供了关于主题的重要评论，包括将多层架构映射到 AWS 服务，通过 AWS RDS 了解托管 RDBMS，了解松散耦合和无状态系统，AWS 服务中的一致性模型，路由表，访问控制列表，防火墙，NAT 和 DNS，大规模分布式系统，弹性和可伸缩性概念，与 AWS 相关的网络技术等。

1.  Sam Newman 的《构建微服务:设计细粒度系统》

我们 DevOps 阅读清单上的第四个是构建微服务。这本书写得很好，容易理解。在这里你可以找到团队建设、版本管理、日志、监控、微服务等主题。这本书提供了大量的例子和实用的建议，全面介绍了系统架构师和管理员在构建、管理和发展微服务架构时必须考虑的主题。

1.  [Mitesh Soni 的 Web 开发 DevOps】](https://www.amazon.com/gp/product/1786465701/ref=as_li_tl?ie=UTF8&tag=devopslinks0e-20&camp=1789&creative=9325&linkCode=as2&creativeASIN=1786465701&linkId=49f33d1ac3869b189c013b5d310a98ad&ref=booksfordevopshandcuratedtechbooks)

DevOps 阅读清单上的第五个是 DevOps for Web Development。这本书结合了 web 应用程序部署和系统配置的技巧，对端到端自动化的简单解释和大量截图。在这里，作者谈到了 DevOps 的关键工具，如 [Jenkins](https://dev.to/apium_hub/advantages-of-jenkins-12) ，Chef，AWS，Azure， [Docker](https://apiumhub.com/tech-blog-barcelona/top-benefits-using-docker/) ，New Relic，Nagios。此外，他还介绍了如何进行部署和监控的方法。

通过这本书，你一定会学到如何为 Amazon EC2、AWS Elastic Beanstalk、Microsoft Azure、Azure web Apps、Docker Container 等多种云平台部署 Web 应用。您将了解如何使用 Jenkins 进行持续集成 Chef 配置管理工具、Docker 容器、AWS、Microsoft Azure 和 Docker 中的持续交付和持续部署，使用 Jenkins 2.0。我们非常喜欢这本书，因为每一章都像一个实用的动手项目。

1.  [AWS 认证解决方案架构师官方学习指南:Joe Baron、Hisham Baz、Tim Bixler、Biff Gaut、Kevin E. 这本书收集了关于云自动化、基础架构和 DevOps 的案例研究。

写得很好，也挺全面的。你将从世界领先的公司学习如何做 DevOps:网飞，亚马逊，Etsy 等。

1.  [有效的 DevOps:建立大规模的协作、亲和和工具文化，作者 Jennifer Davis，Ryn Daniels](https://www.amazon.com/gp/product/1491926309/ref=as_li_tl?ie=UTF8&tag=devopslinks0e-20&camp=1789&creative=9325&linkCode=as2&creativeASIN=1491926309&linkId=e3c935dc3ce4b505b5788cc4a1d965da&ref=booksfordevopshandcuratedtechbooks)

这本书绝对应该在 DevOps 的阅读清单上。有效的 DevOps 不仅仅是一个技术指南，它还是一个文化和管理指南。DevOps 是关于团队协作和文化挑战的，这本书更关注这些人性的一面。这本书联系了 DevOps 的技术和文化方面，描述了失败和成功。此外，它还提供了不同的方法来改进团队内部的协作，创建团队之间的纽带，选择正确的工具和工作流，以及扩展有效的实践。

1.  站点可靠性工程:谷歌如何运行生产系统，作者贝琪·拜尔、克里斯·琼斯、詹妮弗·佩托夫、尼尔·理查德·墨菲

谷歌如何运行生产系统？这是本书的主要问题。我们喜欢这本书的一点是，每一章都是独立的，涵盖了一个特定的主题。这本书是散文和文章的集合，其中谷歌网站可靠性团队的关键成员解释了他们对整个生命周期的承诺如何以及为什么使公司能够成功地构建、部署、监控和维护世界上一些最大的软件系统。您还会发现一些原则和实践，让 Google 工程师能够使系统更具可伸缩性、可靠性和效率。

1.  [devo PS 2.0 工具包:利用容器化微服务实现连续部署管道的自动化，作者 Viktor Farcic](https://www.amazon.com/gp/product/152391744X/ref=as_li_tl?ie=UTF8&tag=devopslinks0e-20&camp=1789&creative=9325&linkCode=as2&creativeASIN=152391744X&linkId=2d8ae95f254a9d3ade4147ad815dc2d2&ref=booksfordevopshandcuratedtechbooks)

这本书解释了如何构建一个完整而稳定的 CI/CD 管道，并支持基于微服务架构构建的现代分布式应用。它涵盖了现代技术，如 Docker、Kubernetes、Ansible、Ubuntu、Docker Swarm 和 Docker Compose、Consul、etcd、Registrator、confd 等。这本书用具体的例子和解释的用例描述了 DevOps 的技术基础。简而言之，这本书涵盖了帮助我们以更好、更有效的方式构建软件的技术，将微服务打包成不可变的容器，持续测试并部署到自动配置了配置管理工具的服务器上。它是快速、可靠和连续的部署，具有零停机时间和回滚能力。它是关于扩展到任意数量的服务器，设计能够从硬件和软件故障中恢复的自我修复系统，以及关于集群的集中日志记录和监控。这本书展示了微服务开发和部署生命周期的整个过程，使用了一些最新和最棒的实践和工具。

1.  [DevOps:软件架构师的观点，作者 Len Bass，Ingo Weber，朱丽明](https://www.amazon.com/gp/product/0134049845/ref=as_li_tl?ie=UTF8&tag=devopslinks0e-20&camp=1789&creative=9325&linkCode=as2&creativeASIN=0134049845&linkId=ed9a5e6387b1f81e0215dc2af26629e6&ref=booksfordevopshandcuratedtechbooks)

这本书绝对是必不可少的，应该包括在开发人员的阅读清单中，以建立一个可扩展的平台。对于软件架构师来说，这本书是理解和掌握 DevOps 环境的非常好的参考。它由 3 名软件架构师编写，涵盖了许多有趣的主题，如开发运维需求、虚拟化和云计算、运营、调整系统以与开发运维实践良好配合、敏捷方法以及 [TDD](https://apiumhub.com/tech-blog-barcelona/advantages-of-test-driven-development/) 。通过这本书，您将了解如何进行故障检测、升级规划、将安全控制、角色和审计集成到开发运维中、为开发运维的采用、推广和测量准备业务计划以及使软件适应开发运维管道和工作流。作者回顾了软件架构师为了实现 DevOps 的目标必须做出的决定，并阐明了其他 DevOps 参与者可能如何影响架构师的工作。它们还提供了更有效地部署 DevOps 所需的组织、技术和操作环境，并审查 DevOps 对每个开发阶段的影响。作者解决了连接多个功能的交叉问题，提供了对合规性、性能、可靠性、可重复性和安全性的实际见解。

1.  约阿金·维罗纳的实用开发

我们喜欢这本书是因为它涵盖了架构、编码、测试、部署、监控和跟踪问题的实际部分。此外，作者还讨论了 DevOps 环境下的物联网。这本书帮助读者熟悉 DevOps 效率所需的工具，并教他们如何使用 DevOps 实践设计适合连续部署系统的应用程序。它还教导读者使用不同的选项(如 Git、Gerrit 和 Gitlab)有效地存储和管理代码，然后测试、部署和监控代码。
