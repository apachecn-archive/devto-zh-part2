# 我们还需要招聘机构吗？

> 原文:[https://dev . to/Peter dev/do-we-still-need-recruiting-agency-4gpk](https://dev.to/peterdev/do-we-still-need-recruitment-agencies-4gpk)

作为一个找工作的考生，我从来没有和任何招聘机构合作过。但作为一名负责技术面试的高级开发人员，我被迫与一些人力资源公司合作，因此我遇到了一些非常奇怪的情况。有时候很烦，有时候就是逗我笑。无论如何，由于我糟糕的经历，我很难找到任何理由向人力资源/人才机构支付佣金。我一直不喜欢“中间人”。然而，许多公司除了自己努力寻找人才之外，还通过中介机构寻找人才。

让我简单解释一下我们公司制定的标准招聘流程。我们没有 HR 部门，所以整个过程由我和 CTO 主导。首先，我准备了一份反映我们当前需求的工作邀请。然后，CTO 在各种平台上发布了该报价。我们收到了不同的简历，看了看。如果你没有一个有意义的 GitHub 概要文件，我们通常要求做我们简单的招聘任务:为现有的单元测试编写一个 PHP 购物车实现；种 TDD，大部分人 4 小时以内解决。我们还发送了一些技术问题，看看我们是否没有浪费您(和我们)的时间在办公室与您会面。尤其是如果你住在几百英里以外。如果我们喜欢您的回答和我们任务的解决方案，我们将邀请您参加一个个人会议(大约 1.5 小时)，包括一个软面试和技术面试。最后，你会收到我们的报价...如果你不喜欢就把它关了。

很简单，不是吗？只是两个互相做生意的政党。我们是客户，我们购买你提供的服务。我们协商交易，如果每个人都满意，就告诉我们你什么时候可以开始。我提到的描述是通用的；我们试图根据提供的材料、成熟的经验和许多其他因素单独接触每一位候选人。如果你找到了另一种炫耀的方式，你就不必解决我们的编码任务。有趣的事实:我们从来不关心你的正规教育。

不幸的是，我们从求职网用户那里收到的简历是不够的，因为公司不想把钱投入到更大的猎头努力中，比如去参加会议或招聘全职人力资源人员...它决定借助外部人力资源机构的帮助。我们会向代理机构提出我们的需求，代理机构应该会找到合适的人选。我们收到了每位候选人的书面推荐信。成功聘用一名员工后，公司向中介支付佣金。

# [](#how-recruiting-with-agencies-went-wrong)招聘机构是如何出错的

如果这些公司做得很好，我不会抱怨，但我所经历的是:

1.  这些建议泛泛而谈，毫无用处。每个候选人都被描述为积极主动的团队成员，以个人发展为目标。等等等等。如果你去掉那些废话，你会发现你在和一个平庸的程序员打交道，他的投资组合很无聊。公司和候选人都在浪费时间和代理机构交谈，因为无论如何，我们都必须自己解决大部分问题。最有趣的事实是那些不被记录在案的。

2.  一家机构曾经将这些建议保存为 DOCX 文件。我在 Ubuntu 上工作，LibreOffice 没有正确地呈现这些文件，从页面中丢弃了一些内容。我不得不切换到 Windows，启动 Microsoft Word，为自己准备一个 PDF 文件，然后再切换回来。最终，人力资源部的人学会了如何自己创建 PDF。真是如释重负。
    [![How do I convert to PDF?](../Images/31bb356a458d5c3b066af4270c444282.png)T3】](https://res.cloudinary.com/practicaldev/image/fetch/s--kZK6TF37--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/k2zna3qgk9m5ukphjuqz.jpg)

3.  同一家机构剥离了我的招聘任务。它最初是一个 GitHub repo，由根目录中的一些重要文件(`composer.json`、`docker-compose.yml`、`phpunit.xml.dist`和最重要的`README.md`)组成！)和一个`tests`目录。你必须写一个简单的实现，通过我所有的测试。我惊讶地发现，那家机构送来的候选人都是自己重写了`composer.json`。我问他们:“你们为什么要这么做？整个自动加载器都是在那里定义的！."原来中介给候选人发的只是`tests`目录。当然，他们没有发送完整的回购协议——候选人可以找到是哪家公司完成了这项任务。目标公司的名字在开始时是最高机密——我鲁莽地把它作为 PSR-4 名称空间的供应商部分。

4.  失踪的情况还涉及到一名候选人，他厌倦了与一家机构没完没了的会面，以至于在得知他申请的是哪家公司后，他拒绝与该机构合作，直接将简历寄给了我们。我不知道那个故事，我很惊讶地看到他已经解决了我们的任务。

5.  说到没完没了的会议——如果有人决定与人力资源机构分享他或她的经历，我总会听到这种说法。当你看到一则花哨的招聘广告，比如“为我们的客户招聘一名市场领导者……”你投了简历，你被邀请参加公司总部的面试。然后你在家接受我们的任务。然后他们邀请你参加另一个会议...但不是和我们一起！如果会议成功，你最终会去我们的机构。你去那里，可能会听到你在以前的会议上已经回答过的同样的问题，因为不称职的机构没有正确总结你的答案，我们没有足够的关于你的数据。

6.  有时，我们不会收到完整的简历，而只会收到一个已解决的任务(类似于盲目招聘)。有时候我们会收到一份姓氏被洗掉的简历。有时我们只收到推荐信，而不是原始简历。

7.  有时候候选人会被迫和一个人力资源部的人一起去拜访目标公司，他会确保我们不会在他们背后做交易，也不会虐待你。对我来说，这就像是来采访你爸爸一样！他是一个中间人。这不是雇员和雇主之间的交易。幸运的是，你的“爸爸”只停留了一个“软面试”，他在技术面试(由我主导)时离开了。他去买了杯咖啡，在那里等着，直到我们结束。所以基本上他在我们办公室喝咖啡是有报酬的。

8.  有趣的是，一个外向、乐观的人事经理如何带着一个惊恐的候选人去参加会议。乍一看，我会雇用人力资源的家伙，而不是一个害羞，害怕戴夫。

9.  当我决定离开公司时，我的同事收到了一家代理机构提供的职位。

10.  我的老板收到了一家机构的聘用通知，要聘用一位正要离开公司的同事。

11.  当你被录用，工作一段时间后，中介会时不时给你打电话，问你工作怎么样。这是我的同事告诉我的。作为一名员工，我会觉得这很烦人。

我没有办法说服我即将离职的雇主停止在人力资源机构上浪费钱(或者投资正确的机构)。但我仍然想知道，为什么波兰的优秀开发者会对那些没有披露目标公司名称的机构发布的招聘广告做出回应。工资范围也没有规定。如果我花时间和人力资源部的人交谈，我想提前知道我将为谁工作，我期望的薪水是多少！

# [](#carving-my-own-path)雕刻自己的路

就像我一开始说的，我从来没有通过中介找工作。我有一些基本的谷歌技能，但让我给你看一个我职业生涯的简短故事，它提供了一些更多的技巧:

1.  我在高中期间遇到了我的第一个雇主。那是一家当地新闻公司。我的朋友已经为他们工作过，他问我是否可以在当地的活动中拍些照片。还是学生的我，就在业余时间开始了卑微的摄影生涯。在我高中考试之后，上大学之前，我得到了一份永久的工作。他们发现我可以为食物编写 PHP 代码，所以他们雇佣我白天在他们的新网站上工作，晚上拍照片。奇怪的安排，但这份工作给了我很多生活经验。

2.  五年后，我遇到了我未来的女朋友。我决定搬到离我家乡大约 300 公里的格但斯克。我开始浏览波兰求职网站 pracuj.pl，很快就找到了一份来自教育公司的工作。我投了简历，完成了一项招聘任务，三周后我得到了一份工作！我已经在那里工作了四年多了。在那段时间里，我的工资翻了一倍，我从一名中层开发人员变成了团队领导。

3.  我和我的女朋友决定改变我们的生活方式，多去旅行。我需要一份完整的远程工作。我记得在波兹南的一次 PHPers 会议上遇到了一家不错的软件公司。我寄了简历。不幸的是，他们只是停止了他们的招聘过程。但是一个月后，我收到一封电子邮件，邀请我参加一个新的流程，我成功通过了。我通过电话和 Skype 参加了三场面试:入门、技术和软技能。我真的很喜欢那些采访。我的一位面试官说他看过我的 PHPers 关于数据库优化的演讲，并且已经知道了我的名字。我得到了一份薪水允许我体面地生活，购买更多的音乐设备(哦耶！)甚至为退休存钱。

如你所见，我 10 年左右的开发生涯中没有和任何招聘机构达成任何协议。重要的是，我不是一个随和、乐观和外向的人。我不擅长敲门砖，但至少我学会了如何恰当地制作简历，找到一个可能的雇主，并让他对我的提议感兴趣。我花了很多时间浏览波兰 nofluffjobs 的公告栏，那里所有的工作都简单明了，薪水都在前面。

还要做什么？我想我应该参加更多的会议，做更多的演讲，这样人们就会知道我的名字。我应该写更多的博客文章，并尽可能为开源做出贡献。这样，我将发展我的个人品牌，并希望在没有人力资源机构的情况下做我的生意。

如果你在人力资源/人才中介机构工作，请提高你的技能。我知道 IT 猎头很难，滥用 LinkedIn 一整天只是收到一堆粗鲁的回复(或者根本没有回复)可能真的令人沮丧。但如果你想做好一个猎头，你需要了解候选人和公司的行为，以及他们真正需要的是什么。我们都是来做生意的，对吧？

我在波兰生活时获得了以上所有的经历。我很想听听你们对其他国家就业市场的看法。随意评论！
