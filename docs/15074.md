# 反应是我在山羊中发现的宝藏

> 原文:[https://dev . to/Maxwell _ dev/react-is-the-treasure-I-found-in-the-goats-PCE](https://dev.to/maxwell_dev/react-is-the-treasure-i-found-among-the-goats-pce)

像许多开发者一样，我想我的第一反应是被解雇。

*   这只是另一种很快就会消失的 JavaScript 趋势
*   用 JavaScript 编写 HTML 是愚蠢的
*   我永远不会厌倦 jQuery
*   今晚我一定会早点睡觉。

事实证明我在所有方面都是错的，在做了实际研究后，我开始改变我的想法。特别是最近的一篇文章给出了一些令人惊讶的数据。React 的增长已经超过了其他 JavaScript 框架，并且以这种速度将超过 Angular 的使用。它赢得了用户满意度，包括对 Vue 的满意度。在 indeed.com，React 击败了 jQuery，成为十多年来第一次失利。

因此，我放下自尊，戴上学习帽，开始研究 React。我通读了文档，阅读了分解利弊的文章，还做了一些教程。但是我的最终测试是我的 JavaScript 成年礼——用它来重现著名的 Monty Hall 难题。我在学习 jQuery，Angular，和 [Vue](https://codepen.io/max1128/pen/OxQYzW) 的时候也是这么做的。现在轮到 React 了。

结果呢？React 以优异的成绩通过了，并突破了我最后一层玩世不恭。检查[最终结果](http://maxwellantonucci.com/react-app-monty-python/)和[代码报告](https://github.com/maxx1128/react-app-monty-python)。

这篇文章实际上不是关于我如何编写这个应用程序的——我不认为有足够有趣的技巧来证明一篇完整的文章。相反，这是关于我对 React 的看法是如何随着我的制作而改变的。

### 等等，为什么天魔堂会陷入困境？

如果你以前没有听说过，蒙蒂霍尔难题(MHD)是一个众所周知的小逻辑难题。你有三扇门，一扇门藏着财宝，两扇门藏着山羊。你选择了一扇门，但是在那扇门打开之前，主人打开了另一扇有山羊的门。你现在有一个选择:坚持你选择的未打开的门，或者换到另一个未打开的门。据统计，[通过切换](https://betterexplained.com/articles/understanding-the-monty-hall-problem/)，你的胜算会翻倍。

我发现 MHD 击中了学习工具或框架的甜蜜点。对于一个相对初学者来说，处理起来足够简单，但对于创造性地解决和克服严肃项目中的常见问题来说，又足够复杂。最重要的是，到最后我知道我是否对新工具感到满意。

最后，它允许我制作像这里这样吸引人的、点击量大的博客文章标题。这难道不是最终最重要的吗？

### React 为什么做得这么好？

这是一个很难回答的问题，因为我喜欢 React 的原因非常广泛。大多数已经在许多 React 文章中讨论过:

*   React 组件和它们的风格是极其模块化的，这非常符合我对面向对象编码和原子设计的偏好。
*   React 的虚拟渲染引擎让它可以在网络之外使用，例如移动应用和虚拟现实。
*   上述虚拟渲染引擎，结合单向数据绑定和条件渲染，使得 React 非常快。谁不爱快应用？
*   [CSS 模块](https://glenmaddern.com/articles/css-modules)。说够了。

但是我喜欢 React 的主要原因似乎与直觉相反:没有“框架魔法”

#### 有了 React，你需要基本面

你可能想知道为什么“需要更多的工作”是一个加号。我使用过的框架，比如 Ember 和 Rails，用“约定胜于配置”的思维为开发人员做了很多工作。我并不反对这一点，尤其是对于需要快速启动和扩展应用程序的更有经验的开发人员来说——这可以节省时间，让事情顺利进行。React 还有“Create React App ”,它是一个基本的、预配置的 React 项目，所以它也不在此之上。

但在本质上，React 是一个库，而不是一个框架——它没有那种魔力。即使使用 Create React App，我也不能忽略基本的 JavaScript 函数、ES6、作用域、闭包、导入和导出、析构等等。我不能忽视应用程序的工作方式，它的结构，或者数据的来源。我也只会通过整体上编写更好的 JavaScript 来编写更好的 React，这是我最终想要的。

这对我来说很重要，因为使用基于更好的基础的工具有助于我作为一名初级开发人员学习，并防止冒名顶替综合症。React 并不要求对库本身有太多的了解，但是它要求对 JavaScript 有很强的了解。对于初学者来说，这个更大的学习曲线是我推迟 React 这么久的原因，但我认为我误解了它。这种曲线来自更多的学习语言，而不是图书馆。这是一个我更加渴望克服的曲线，我会尽我所能去战胜它。

在我构建这些基础知识时，React 将是一个很好的工具。可能限于边项目几年，但即便如此，我还是会继续用。我已经有了几个想法，包括用 [Gatsby.js](https://www.gatsbyjs.org/) 为[5 月 1 日重启挑战](https://www.may1reboot.com/#/)重新制作我的个人网站。

通常我不会从一个在山羊中寻找宝藏的游戏中学到很多。从来没有想到这里的“宝藏”是一个新的最受欢迎的 JavaScript 库！
