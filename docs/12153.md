## 我们作为数据科学家和数据工程师面临的挑战

数据科学是关于想法、实验和分享见解的。数据科学人员在所有这些方面都很棒，但一旦涉及到向团队以外的人公开，他们似乎就会遇到障碍。

最重要的是将结果从数据科学家的办公桌上传到企业，以便他们可以使用这些结果来制定决策。动力更多的是让实验正确并快速进行，无论如何都要实现。随着时间的推移，这会导致技术债务(和依赖性地狱)，如果我们重新审视实验，这可能会使我们的生活变得困难。

## Docker 是什么？它能做什么？

Docker 很牛逼。人们通常将它用于:

*   自动化，共享和复制实验，
*   打包和部署数据科学应用程序，
*   创建易于使用的数据科学沙盒，用于启动和实验，以及
*   云环境中的大规模数据分析和机器学习。

## 但是 Docker 是什么？它能为数据工程做什么？

> Docker 是一个开放平台，开发人员和系统管理员可以在笔记本电脑、数据中心虚拟机或云上构建、发布和运行分布式应用。(官方文件)

Docker 使用容器来简化应用程序的创建、部署和运行。容器使程序员或数据工程师能够隔离和打包应用程序及其所需的所有依赖项(文件、库等)。

整个产品作为一个包提供，可以在任何运行 Linux 操作系统的机器上运行，完全复制打包的环境，而不管目标机器是如何配置的。

重要的是，Docker 在初创企业和大型企业领域也有同样的追随者，证明了它在工程任务领域的实用性。

*   网飞在其运营基础设施中运营着一支集装箱船队。
*   大多数云提供商都有某种形式的支持或基于平台的应用程序来运行容器(更准确地说是 Docker)。
*   举个例子，AWS 提供了 [AWS 容器服务](https://aws.amazon.com/ecs/)，它有助于创建高性能、可伸缩且易于编排的本地容器服务。

## 学习 Docker 有多难

Docker 的学习曲线是可以管理的，但是对于较少接触技术方面的数据科学人员来说，可能会发现它很陡。我们将讨论如何开始您的 Docker 之旅，但首先让我们看看 Docker 真正闪光的用例。

### 1。你需要分享和复制你的分析和实验吗？

数据科学最大的痛点之一是共享和复制实验。这方面的挑战包括:

#### 不同的操作系统

*   每个人都会做出选择，这些选择会破坏许多数据科学库中的安装。
*   在基于 Linux 的系统中顺利安装的库在 Windows 中可能不那么容易。

#### 版本化

*   不同的 python/R 版本，不同的库版本会在共享分析时造成巨大破坏。
*   人们一直在努力简化这一过程，但上述操作系统问题仍然是一个重要因素。

#### 缺少流程

*   很多时候，在较小的团队中，甚至在企业级别，在共享这些实验的同时创建过程的工作要么是不存在的，要么是与常规的手工过程的报告工作相一致的。
*   数据科学实验是创造性的和非线性的，这使得它们很难整齐地适应系统中已经存在的流程。

Docker 在以上所有方面都有帮助。因为 Docker 映像是不可变的，所以冲突操作系统的问题得到了解决。操作系统和库的版本保持不变(除非更新)，我们可以设置工作流，将 Docker 集成到数据科学流程中的每个人(所有技术人员)中。

### 2。构建概念验证(或生产级)仪表板、Web APIs 和后端 Cron-jobs 是您工作的一部分吗？

数据科学应用有多种类型:

*   使用条纹/破折号/散景的仪表板
*   为服务于机器学习模型而创建的 Web APIs
*   使用机器学习模型定期运行的脚本(批处理)

向最终用户共享数据科学应用是一项艰巨的任务。你不想公开分享，只分享给特定的子集。概念证明(POC)的仪表板是一项艰巨的任务，如果用户是非技术人员，直接共享 RShiny 或 Python 代码并不能真正解决问题。它违背了基于网络的用户界面易于使用的全部目的。Docker 及其生态系统可以帮助数据科学家至少尝试为有限的用户群发布数据科学应用。

### 3。您是否需要创建数据科学沙盒来进行入门和实验？

对于许多大组织来说，入职是一个关键问题。公司防火墙、ServiceNow 要求安装 Anaconda 是一些消耗生产时间的技术障碍。

基于 Docker 的映像可以帮助新用户快速使用他们的笔记本电脑/台式机，只需几个命令。快速可行性分析的沙盒也可以用 Docker 解决。如果团队想尝试新的东西，而不需要安装的麻烦，我们需要的只是一个 Docker 映像。

### 4。您是否希望改进数据科学应用程序的部署流程？

部署是一个大难题。在如何将机器学习或数据科学应用程序集成到系统中的问题上，数据科学小组和 DevOps 小组之间存在巨大的信息不对称是很常见的。

这两个组都是独立工作的，通常常规的软件工程部署对于与机器学习应用程序相关联的独特问题是无效的。如果数据科学小组能够进行小规模部署，或者有一个混合团队与数据科学小组协同工作，则 Docker 可以帮助解决上述问题。

### 5。您是否希望在云环境中更轻松地执行大规模数据分析和机器学习培训

随着数据分析(主要是由于量)和机器学习(训练模型)的复杂性不断增加，需要更多的计算能力。大多数云提供商都可以通过他们的平台访问他们的计算能力。

在这些情况下，时间就是金钱，Docker 以及 Chef/ Puppet/ Terraform 等工具可以帮助节省设置和管理云基础架构的时间。

### 使用 Docker 进行数据科学的挑战

1.  没有足够的资源用于使用 Docker 的数据科学应用。

2.  学习曲线陡峭的新技能。

3.  让整个团队改变他们的工作流程来整合 Docker。
