很久以前，当 netscape 还是新生事物，家庭互联网还处于起步阶段的时候，对于一个网站来说，你真的只有一个选择，那就是使用 HTML 对其进行硬编码，多年来这是标准，这使得更新网站变得困难、痛苦，并且通常超出了标准企业所有者的能力范围。随着时间的推移，WordPress 最终将网站开发带给了普通大众，有了 WordPress，你不仅可以从一个(当时)易于使用的界面来构建和更新你的网站，你还可以实现一系列动态功能，如联系人表单、社区网站，甚至最终实现电子商务系统。有了广泛的可用功能，似乎唯一的限制就是你对能做什么的想象力，你需要的只是一个开发人员和预算。有了如此强大的市场地位和广泛的功能，为什么静态网站现在又开始流行了呢？

# 什么是静态站点？

如下所述，静态网站通常只是 HTML、CSS 和 JS。任何内容更改都必须通过手动修改 HTML 代码来完成。如果你不熟悉 HTML，这可能是令人生畏的，因为一个错误的修改可能会导致你的整个网站看起来完全错误。既然有这么多的障碍，你为什么要费心去下载一个 WordPress 主题，然后根据你的需要调整一下风格呢？进入静态站点生成器。

## 静止站点发电机

第一个大的静态站点生成器(SSG)之一是 Jekyll，Jekyll 让你可以两全其美，你可以为文章和页面设置一些模板，在 YML 文件中使用 MarkDown 编写内容，运行一个快速 CLI 工具，Jekyll 会处理剩下的事情。这允许你对一个普通的静态站点进行微调控制，而不需要每次你想创建一个新的文章时都创建一个新的 HTML 文件的繁琐过程。从那时起，市场上充斥着你能想到的几乎每一种编程语言的工具和框架。

## 静态网站优势

**性能**

与动态网站相比，静态网站的一大优势是性能，因为网站不必在每个页面上对数据库进行处理器密集型查询，静态网站的导航速度很快，真的很快。通过适当的文件缩小和图像优化设置，网页可以在几毫秒内加载到现代互联网上。随着谷歌将越来越多的权重放在网站表现上作为排名工具，这对你和你的企业都有巨大的好处。

**灵活性**

以一个 twitter feed 为例，比如一个 WordPress 网站，你可能需要安装一个插件，并使用一个短代码使其出现在前端，这就引入了复杂性、额外的脚本加载和安全问题。在编辑 HTML 文件时，实现 twitter feed 可以简单到从 twitter 复制并粘贴推荐的代码，也可以复杂到手动创建并设计一个定制的 twitter feed。唯一的限制是你的能力。

**提高可靠性**

由于没有后端服务或数据库为您的网站提供内容，因此您的网站运行不再需要额外的可用性要求。流量的突然涌入将不再导致性能问题，恶意用户将更难关闭网站。这不是不可能的，在规划和建设你的网站时，安全性和可靠性肯定是需要考虑的，但它比大多数 CMS 和动态平台要健壮得多。

**更简单的托管需求**

当你托管一个动态网站时，你往往需要考虑很多不同的元素，例如:

*   我的主机可以处理数据库托管吗？
*   它会运行 WordPress 软件吗？
*   设置是否针对加载动态文件进行了优化？

当托管一个静态站点时，你的需求变得简单多了:

*   我可以托管 HTML 文件吗？
*   平台用 NGINX 还是 Apache？

虽然根据您的部署计划可能有其他要求，但这是任何静态网站的两个基本要求，大大简化了您的主机托管，并经常在额外的软件和支持成本方面节省大量资金。

## 静态站点的下行

综上所述，我相信你们中的很多人都想知道它的坏处是什么？有一些。

复杂性

即使是像 Jekyll 这样的系统，你仍然需要创建原始的布局，不像许多现代的 WordPress 主题，Jekyll 没有一个花哨的拖放站点设计器。你需要创建的是使用 HTML，CSS 和 JS。这可能会导致您需要引入外部人员来进行初始设置，从而产生不可预见的成本。此外，创建新帖子可能很棘手，你的内容工具通常只是一个标准的文本编辑器，虽然这有助于简化事情，但取决于你使用的 SSG，如果你需要学习 Markdown，它也会使事情变得复杂。有像 [Netlify](https://www.netlify.com/) 、 [PrismicIO](https://prismic.io/) 和 [Contentful](https://www.contentful.com/) 这样的网络用户界面，但是根据我的经验，在我推荐它们之前，在文档和支持方面与它们集成还有很长的路要走。

**功能**

如果你需要一个具有自动营销功能和会员区的大型电子商务平台，那么静态网站是不适合你的。由于该网站完全由 HTML/CSS/JS 文件组成，任何比联系表单或 twitter feed 更复杂的东西都是不可能的，像 WordPress 这样的动态网站或 CMS 平台将会更加合适。

# 哪个好？

正如你在上面看到的，CMS/动态网站和静态网站都在网上有自己的位置，决定实际上归结为以下几点:

*   预算
*   项目要求/复杂性
*   你的技能和能力
*   您的用户群

如果你需要一个 7 页的快速网站，并通过联系方式为一个活动做广告，那么尽一切可能走 SSG 路线，即使你不得不雇用外部开发人员来开始，你也会看到更大的投资回报率。如果你需要一个全方位的会员网站，有电子学习和商店，那么基于 CMS 的网站将是一个更好的选择。
