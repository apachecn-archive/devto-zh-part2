## 规划不一定是敏捷的敌人

[*https://HBR . org/2018/09/planning-dont-have-to-be-the-敌人的敏捷*](https://hbr.org/2018/09/planning-doesnt-have-to-be-the-enemy-of-agile)

大多数科技公司现在都将遵循敏捷的开发周期。计划是管理的重要基石，协调和控制接下来会发生什么。然而，敏捷和计划并不一致。看看这篇伟大的文章是什么意思。
