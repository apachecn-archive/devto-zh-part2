****Windows steady state:功能&局限性****

SteadyState 是微软为 32 位版本的 Windows XP 和 Vista 提供的免费工具。它帮助 It 人员管理和维护学校、机场、公共图书馆和医院中的共享计算机环境。Windows SteadyState 提供了完整的磁盘保护功能，可以通过重新启动来恢复最终用户所做的更改，并将系统配置恢复到 IT 管理员保存的所需状态。T3】

Windows SteadyState 的工作原理是将用户在硬盘上所做的所有更改重定向到一个临时缓存中。在其“丢弃模式”下，Windows SteadyState 可以在计算机重新启动时清除这些缓存，同时将其回滚到管理员定义的基线。启用“持久模式”后，它可以在重启后保持缓存完整，it 管理员可以选择保留更改。否则，缓存将在指定的持续时间后过期。在“提交模式”下，Windows SteadyState 不会将任何内容重定向到缓存，因此所有更改都会对系统产生永久影响。IT 经理也可以使用这个工具来定制用户环境。可以限制用户执行不适合特定计算环境的若干动作。例如，它允许禁用 Windows 注册表、命令提示符或关闭控制面板。T3】

Windows SteadyState 已于 2010 年被微软停产。这使得许多依赖 SteadyState 的公共工作站容易受到恶意软件、病毒和持久系统变更的攻击。Windows SteadyState 不支持任何 64 位版本的 Windows 操作系统(OS)。因此，管理运行在最新 Windows 版本上的系统的 IT 专业人员迫切需要具有类似 SteadyState 功能的替代解决方案。T3】
