# 工作与娱乐

> 原文：<https://dev.to/jvarness/work-vs-play-4i2e>

过去，我已经在博客上写了很多关于我的 Pebble watchfaces 和我学习 Dart 的旅程，但今天我想谈谈我最近一直在想的其他事情。

**这不是关于什么技术是最好的**的讨论，而是关于我如何注意到我的*职业*与我的*兴趣*之间的差异，以及我一直在*学习的不同东西*的叙述。

从历史上看，在我的职业生涯中，我一直使用 Java 和 C#进行大部分企业开发。谈到我的职业生涯，Android 应用程序、Windows 应用程序、Java 服务器和服务都消耗了我的大部分时间。我甚至设法用杜兰达尔/奥雷利亚在里面加入了一点网络开发。甚至在大学里，我们总是用 Java 做大部分的作业。Java 和 C#一直是我非常习惯使用的语言，我仍然认为它们是我最了解的两种语言。

然而，我最近注意到的是，我开发的那些没有投入生产的东西与我为了娱乐而学习和开发的东西大不相同。

我在 Visual Studio/C#和 Eclipse/Java/Maven 之外编写代码的大部分时间都花在了 Visual Studio 代码上，研究 React、Angular 或 Amber 的教程，或者维护用 CoffeeScript 编写的 Hubot 脚本，或者使用 NPM 开发其他各种 JavaScript 项目。我已经开始慢慢变得越来越适应其他语言，经常发现它们的特性和工具比我在职业生涯中使用的更好。

这种反差对我来说相当惊人。这就好像我作为一名工程师快乐地过着两种不同的生活，每一种都与另一种相吻合，并不断发展，在完全的技术和堆栈方面变得越来越好。

在你的专业工作和个人工作中，你注意到了哪些方面？你在职业上做的事情和你个人做的事情一样吗？或者你做的专业工作和我注意到的完全不同？

如果不一样，你会喜欢这种不同和变化吗，或者你会希望你的个人生活和职业生活更加接近吗？如果不是，是什么让你无法突破自己的舒适区？
