# 对于有些感兴趣的人来说，JavaScript 框架的超级简史

> 原文：<https://dev.to/_adam_barker/the-super-brief-history-of-javascript-frameworks-for-those-somewhat-interested-3m82>

JavaScript 框架已经走过了漫长的道路，任何新的开发人员不仅面临着学习一门语言，而且选择首先学习框架。

今天可用的选择是在相对较短的时间内一些难以置信的创新的结果。就在 2004 年，谷歌发布了 Gmail，这被认为是第一个我们今天所知的单页应用程序的浏览器产品。

如果你刚刚开始学习 JavaScript，你会发现这是一个好时机，但是如果你想在浏览器中做一些有趣的事情，并不总是有那么多的选择和灵活性。

# 动态 HTML

在 21 世纪早期，JavaScript 很大程度上还处于起步阶段，主要用于对页面进行基本的修改。亮点包括鼠标悬停、下拉菜单和滚动文本——这些都是开发人员今天对 CSS 的威力习以为常的事情。

当时，Internet Explorer 主宰了网络浏览器领域(我知道，很奇怪吧？)并且从 1999 年开始，包含了微软为他们自己的电子邮件产品 Outlook 开发的库的包装。这个可爱但名字却令人生畏的 XMLHttpRequest 对象最终成为了跨浏览器的标准，并成为我们今天体验网络的门户。

至关重要的是，这个 XMLHttpRequest 允许将请求发送到服务器，并处理响应，而不必刷新浏览器或执行完整的往返。在此之前，像点击标签或发布表单这样的事情意味着服务器处理请求并决定如何呈现响应，浏览器将旋转并最终显示结果。

开发人员欣然接受了这一功能，并看到了在 web 上创建真正的桌面替代应用程序的潜力。异步 Javascript 和 XML，简称为 AJAX，是用来描述这种新发现的能力的术语。

# jQuery

从技术上讲，jQuery 并不是第一个 JavaScript 框架，但它在 2006 年发布后迅速流行起来。jQuery 旨在解决开发人员在 web 浏览器实现中面临的许多微妙差异问题，它将开发人员想要的许多功能抽象到一个清晰、易于学习的框架中。

由于有可能构建利用 web 便利的应用程序，开发人员构建了越来越大的解决方案来测试 jQuery 的可维护性。像 Gmail 这样的产品已经展示了可以实现的目标，但是很明显可能需要更多的企业友好工具。

# 骨干&骨干

2010 年，Jeremy Ashkenas 发布了 Backbone，这是第一个旨在创建单页面应用程序的框架。Jeremy 已经看到了一个巨大的 jQuery 应用程序会变得多么混乱，他希望有一种更干净的方法来补救数千个选择器和交织在一起的事件处理程序。有趣的是，Backbone 不需要 jQuery，它是一个独立的框架，但是如果 jQuery 存在的话，某些功能是可以启用的。

大约在同一时间，亚当·艾伯龙斯和米斯科·赫韦瑞创建了 AngularJS，不久后，当赫韦瑞在谷歌工作时，该公司受到了谷歌的照顾。Angular 是第一个为前端应用程序开发提供完整架构的框架。

AngularJS 的主要特性之一是双向数据绑定，并提供了一种将模型数据绑定到 HTML 标记并实时更新更改的方法。开发者称之为“automagic”。Angular 还支持依赖注入和创建可重用组件的能力。

# 击倒，流星，反应过来

Knockout 是另一个为双向数据绑定提供解决方案的框架，尽管它只提供了 AngularJS 的一小部分功能。Meteor 也有大量的追随者，但在 2013 年，脸书发布了它的 React 框架，并从此享有巨大的人气。

许多新开发人员第一次听说 React，并把它作为他们的首选框架。尽管如此，React 确实有一个相对陡峭的学习曲线，但它也受益于优秀的文档和巨大的社区。

* * *

对于一个开发人员来说，看到框架试图解决其前辈问题的方式是非常有趣的。我认为 Angular(当然是最近的版本)与 React 有很多共同之处，反过来，他们也与 Vue 这个新成员有着共同的目标。

另一个变得越来越重要的考虑是这些框架支持移动目标的容易程度。Ionic 和 React Native 已被证明是相当重要的参与者，因为团队试图将 JavaScript 作为一种适用于所有方式的语言进行标准化:具有 NodeJS、桌面和移动浏览器的服务器端，以及在其他框架的帮助下针对本机设备。

* * *

总之，JavaScript 是一种很好的学习语言，有了这些框架，几乎任何事情都可以在任何设备上实现。随着框架的每一个新版本的发布，都有很多从以前的版本中学习的证据，所以现在是开始构建你的第一个应用程序的最佳时机。
