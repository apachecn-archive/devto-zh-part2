# 三个月的开发者生涯

> 原文 https://dev.to/cindywu/three-months-as-a-develubber-335n

自从转换角色以来，我们已经接近三个月了。做一名开发人员很难，但没有我最初想象的那么难。

作为一名开发人员，最大的好处是问题空间通常被很好地定义。不太好的一点是，通常有无数种方法来解决问题。对于像我这样的新手来说，更糟糕的是，我可以写一些有用的东西，但不太可能是一个有经验的人会写的。我不知道我做的东西是否好吃，但它是可以吃的。

#### [](#i-dont-know-what-i-dont-know)不知道自己不知道什么

这还是个问题。我不知道这什么时候会不再是个问题。不可能对一种语言了如指掌。如果那是不可能的，很多事情都是不可能的。只是没有足够的时间。而且，即使你有无限的时间去学习所有的东西，也可能不是最好的利用某人的时间。

#### [](#learn-by-doing)边做边学

我得出的结论是，最好的学习方法是边做边学。所以，在接下来的一个月里，我会做得更多。这意味着做更多的编码项目。多做一些让我不舒服的事情，坚持下去。

我一直告诉自己的是，我不是想让阿波罗 11 号登陆月球。我试图通过一些互联网管道移动一些像素和一些数据。如果我做不到这一点，还有很多其他的事情我也做不到。

#### 我觉得自己很安全

我在自己身上看到的最大变化之一是，大多数时候，我一个人感到安全。因为我意识到我有可能独自完成事情。如果某件事在过去是真的，那么它在未来也可能是真的。如果有数据显示我在过去成功地自己修复了一个 bug，就有理由相信我在将来也能自己修复一个 bug。

#### [](#pair-more-not-less)一对多，一对少

杰夫和我已经开始少配对了。我认为这是一个好的迹象。这意味着我可以自己做一些有用的事情。然而，我认为为了保持我学习的速度，我们需要更多而不是更少的配对。

#### [](#i-still-lack-the-foundations)我还是缺乏基础

我昨晚一整晚都在学习主持人。我现在有点明白什么是主持人了。我今天在一个演示者中编写了一个方法，但是没有将该方法从项目演示者移到更新演示者中。我希望在 Ruby、Rails 和 Javascript 方面有更扎实的基础。我对 CSS 感到很舒服，尽管我的 CSS 很乱。接下来的一个月，我将致力于理解我们应用程序中的 Railsy 部分。当我遇到更多我不理解的 Rails 内容时，我会花时间尽我所能去理解它们。

#### [](#do-one-thing-and-do-it-well)做一件事，把它做好

当我只是编码时，我会取得很大的进步。当我被要求考虑其他事情时，我不擅长编码。如果我想做好这件事，那么我应该只做这件事，不做别的。在过去的两周里，我取得了很大的进步，这是因为我没有做其他的事情。我应该禁止自己去想代码以外的任何事情。然后在我所有的空闲时间里，我用创造性的活动来填满它，比如烹饪、写作、演奏音乐、锻炼和探索纽约城。

#### [](#fixing-procrastination)修正拖延症

上周日，我没有完成作业。我试着思考为什么。最后的百分之十很难。我喜欢开始项目，但我不太擅长完成它们。我需要加强我的自律。

我也明白了如果我不想编程，那么最好的办法就是不编程。对我来说，最好的办法是做一些需要我全神贯注一个小时的事情，然后再回来编程。我不应该做的是在网上阅读东西。我能做的最好的事情之一就是阅读实体书、写作或锻炼。

我仍然努力拖延那些看起来困难的事情。对此我没有很好的补救方法，但也许这里的配对会有所帮助。

我的梦想是有人能告诉我他们想要建什么，为什么要建，这样我就可以担心我要怎么建了。我想成为一名全职工程师。我确实非常喜欢它。与其说我喜欢做开发人员，不如说我喜欢学习。

#### [](#making-progress)不断进步

做一名开发者的好处是，当你取得进步时，这是显而易见的。你修好了一个东西。你创造了一个能做一件事的东西。

这个星期是我第一次自己制作邮件。这周是我第一次自己提交真正的 PR，而不仅仅是修复 bug 的 PR。这一周我第一次觉得如果我想，我真的可以建造任何我想建造的东西。

作为一名开发人员有时会感到沮丧，但我知道那些沮丧的时刻仅限于很短的一段时间，这在很大程度上是我自己可以控制的。

我喜欢杰夫在我们一对一谈话时给我看的这张图表。他说迈克的房子在最右边。有一天，我希望我们都能和迈克做邻居。
