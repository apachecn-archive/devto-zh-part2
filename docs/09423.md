### [](#introduce-yourself-who-are-you-where-do-you-work)自我介绍！你是谁？你在哪里工作？

大家好，我是 Steven Cooper，虽然大多数人都知道我是 developersteve。我是 Telstra 的开发者布道者，负责 TelstraDev 的社区参与。此前，我是一家初创公司的首席技术官，也是 Xero 的高级传道者，并在 PayPal/Braintree 担任了两年的 APAC 开发者代言人。

我有南半球(不，是世界上)最大的混合贴纸收藏，现在放在一个 A3 大小的文件夹里，重约 2 公斤。大约在 2014 年，它从一个小三明治袋开始，随着我参加越来越多的会议，收集剩余的贴纸带到下一次会议，它一直在增长。

* * *

### 是谁或什么让你开始编程的？

这是我一直感兴趣的事情，也是我一直非常喜欢的事情。从我年轻的时候起，我就能拿起一件科技产品，并让它超出它的能力。我天真地记得当我大约 8 岁的时候，在雅达利上玩 basic，虽然我从来没有想过我会最终追求它，因为在我的脑海里它是任何人都可以做的事情。

即使在作为分析师的早期职业生涯中，我仍然不认为自己是一名开发人员，尽管我已经建立了一个基于网络的报告系统，使用 2 台备用台式机作为 web 服务器来汇总销售数据(大多数情况下，我不必继续手动运行它们)。我开始从事自由职业，建立自己的网上商店，主要是为了与同行进行比较，挑战自我。

2009 年，PayPal 改变了我的想法，当时澳大利亚办公室打电话问我，他们是否可以让我作为两名开发人员之一从澳大利亚飞往旧金山参加 PayPalX 会议。以前没有参加过开发者大会，但去了旧金山的一个大型会议，哇，真是大开眼界。正是在来自世界各地的 1000 名开发人员中，我发现我是一名开发人员，我意识到的事情是，虽然任何人都可以编写代码，但这是旁观者的独特体验，没有任何一个开发人员会以同样的方式处理问题或编写代码。

2010 年和 2011 年，我被 PayPal 接回来参加创新大会，上次去那里时，我发现他们打算在 APAC 招聘一名福音传道者，尽管这需要几年时间，但我还是欣然接受了。对我来说，这是更多的回馈，并帮助 PayPal 做更多他们作为开发人员为我做的事情，让 PayPal 这样的全球支付品牌在你认为自己是开发者之前相信你是开发者，这是你永远不会忘记的事情。

### [](#what-is-a-developer-evangelist-and-what-do-they-do)什么是开发者传道者，他们做什么？

我一直被教导的看待它的方式是，你是公司内部一个受资助的初创公司，这意味着你做从视频制作、活动设置、博客帖子、媒体参与、代码(样本/SDK/调试)、客户参与和社交媒体的一切事情。通常第二年和第三年传福音一旦你有了一个既定的计划，将会有最多的旅行，因为你倾向于做活动跳跃和背靠背会议很多，与社区接触最多。

在旅行方面，自 2014 年以来，我已经进行了近 50 万公里的旅行，其中包括 113 场活动演讲，我使用 tripit 和 jetitup 对其进行跟踪，然后在我的[网站](https://developersteve.com)上保留我正在演讲和已经演讲的所有活动的列表。

* * *

### 你如何跟上你所宣传的技术的发展，你是否怀念全职软件开发？

这是我作为一名传道者的热情的一部分，我喜欢发现最新的技术，推动它的边界，然后能够与每个人分享。通常我会结合一些资源来寻找新的和新兴的技术。它是网络、社区和社交平台的混合体。

现在我正在玩 Flutter，用我们新发布的事件检测 API 在 dart 中构建一个模块。这真的很酷，因为你可以在开发应用程序时在 2 个模拟器中开发应用程序(flutter 很棒)，我们刚刚发布的新事件检测 API 意味着你可以创建内置无缝网络定位和认证体验的应用程序。

* * *

### 作为一名开发者传道者，你有过的最酷的经历是什么？

我已经能够与语言和服务的创始人见面/聊天，例如 Matz (ruby)、Rasmus (PHP)和 Igor (Nginx)等等，看到一些最常用的平台和服务背后的人和思想真的很酷。我最喜欢看的是普通人的日常生活

* * *

### 对于一个成功的开发者传道者，你的第一条建议是什么？

做所有的事情，对于 devrel 圈子里的人来说，我认为这是一个既定的任务，因为这个角色是代码、业务和演示(以及所有这些之间的部分)的混合，你将成为一个多用途的人，前一分钟做指导，下一分钟做演示，然后是代码调试和支持。对于那些看过我表演的人来说，尤其是我的秘密舞台天赋通常是当场就想出双关语和俏皮话，我不知道这是怎么发生的，但这只是我一直都能做到的事情，甚至我的一些 dev 笑话也只是突然凭空出现。

### 迄今为止，你职业生涯中最难学到的一课是什么？

在 devrel 圈子里，这是你真正擅长的事情，尤其是对每个内部团队来说都是不同的，并且是少数几个可以作为多个团队来完成工作的团队之一。我认为另一件事是学会说不，作为一个开发人员，甚至在 devrel 中，有时候你真的真的很想做一些事情，但时间表和资源并不总是允许。特别是在 devrel 中，有时候你只需要停下来避免精疲力尽。

### 工作之外，你有什么爱好吗？你认为它们对你的科技事业有任何帮助吗？

我不认为我曾经把布道看作是一种职业，这只是我所做的，我认为这是成功的部分秘诀，这只是我和也只是极客的开发人员一起极客。我想如果我把它看作是一份工作，那我就必须找些别的事情做。当我可以的时候，我很喜欢玩 Xbox，这取决于我当时想玩什么。在过去的几个月里，我也一直在摆弄 3D 打印，并设法将我自己的一些创作成果贡献给 thingiverse。
