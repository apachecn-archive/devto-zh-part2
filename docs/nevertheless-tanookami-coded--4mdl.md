# 然而，Tanookami 编码

> 原文：<https://dev.to/tanookami/nevertheless-tanookami-coded--4mdl>

## 我开始/继续编码是因为...

解决问题是绝对令人兴奋的，我喜欢构建满足需求的东西。就经验而言，我的出身在于插画，所以我已经知道进入“区域”的满足感，在那里创造力很容易从脑海中浮现出来。我的第一份工作是一个偶然的网页设计职位，它训练了我 HTML 和 CSS 与同一个创意区的相关性令人惊讶。再加上结果很容易用一个网址显示出来，所以我就迷上了。

现在，在我的导师和合作伙伴的指导下，我通过学习更多关于后端系统的知识再次找到了快乐，我们计划以结对编程的方式一起学习最新的前端语言。

## 我最近克服了...

重构旧代码时令人沮丧的模糊时期。目标是减少游戏的战斗系统产生的垃圾邮件，使它更像一个浓缩的摘要，这意味着访问一个旧的系统和代码不是我写的。加上编码的中断，围绕这个项目的恐惧感很难接近。

我的建议是:进去吧。即使只是阅读并逐渐理解代码也是一种进步，你惊人的大脑会在你睡觉的时候继续解决问题。不要因为你今天改变或写的几行而气馁。阅读它，学习它，构建你的作战计划。那明天就回去吧。[目前卡住了？我最近收藏了这个；也许对你也有帮助。](https://dev.to/lpasqualis/understanding-and-overcoming-coders-block-5m6)

## 我想吹嘘一下...

通过不放弃和仍然兴奋地学习更多东西来感受启发！我开始意识到愤怒、沮丧和“放弃”的欲望是多么短暂。

我最自豪的时刻之一是和我的搭档度过了富有成效的一天，我们进行了大量的交流，同时也达成了共识。那天做了这么多事！那是在用一个可靠的计划编写新代码的时候，这给了我一些想法，我们可以通过首先一起审查旧代码并达成一致来解决重构旧代码的问题。所以我想我是在吹嘘写这篇文章是如何激励我进步的。

## 我给盟友的建议是支持女性主义者的守则....

永远不要认为任何人会被此时此刻的处境所限制:我们都是从底层开始的——我们都在成长。鼓励奋斗成长，因为奋斗本身会带来进步，无论是肌肉还是精神。

就我而言，从小到大，我一直相信智力是一种固定的特质，所以普遍的倾向是，除非我们已经确定自己会成功，否则永远不要说话/伸出手。否则，我们就有暴露局限性和“弱点”的风险。相信固定的特征会导致即使是最有才华的人犹豫不决并停止尝试，因为害怕他们会发现一些限制。
