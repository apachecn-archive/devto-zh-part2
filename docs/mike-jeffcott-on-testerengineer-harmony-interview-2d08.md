### 自我介绍！你是谁？你在哪里工作？

我是迈克·杰弗科特。我在澳大利亚墨尔本的 Zendesk 工作，担任测试方面的高级软件工程师。我与 Web Widget 背后的团队一起工作，这是一个 iframe，我们的客户可以将它放在他们的网站上，这样他们的客户就可以在不离开他们的网站的情况下做 Zendesk 的事情。

我的时间基本上被分成了以下几部分:从事我们的自动化浏览器测试，照看测试基础设施(比如 staging environments，我们用来运行测试的 Jenkins 实例，以及 Sauce Labs 之类的东西)，在浏览器开发工具中分离东西，以及在测试策略和自动化测试实践方面指导开发团队。

### 谁或什么让你进入编程和测试领域？

像许多科技界的人一样，我从小就对计算机感兴趣。在大学之前，我没有很多与编程相关的机会，所以就编程而言，我基本上是在攻读计算机科学学位。最后我读了 4 年的理学学士，计算机科学和心理学双学位。

实际上，我大学毕业后对全职编程并不热衷，所以我投身于测试，作为一种充分利用我在整个学位期间积累的技能和知识的方式。最终，我越来越多地转向自动化测试，作为一种使整个过程更好的方法。如今，我兜了一圈，无论如何我都称自己为全职程序员。

### 比起做产品，做测试的工程师更吸引你的是什么？

我真的很喜欢为我的工作解决各种问题。在开发 Selenium 脚本时，我通常会深入研究 Chrome 开发工具，试图找到最好的 CSS 选择器来定位元素，以及与 UI 交互的最快速有效的方式。有时候，我不得不做一些非常奇怪的事情来测试 web 应用程序，因为事实证明，有时候浏览器自己也会做一些非常奇怪的事情。

观看测试运行并试图跟上浏览器中发生的事情也很有趣。

### QA/测试经常受到开发人员的指责——你认为开发和测试之间成功的合作是怎样的？

我认为很大一部分原因是缺乏对测试的理解。我获得了计算机科学学位，却从未真正被教授过测试知识。我们基本上只学到了一堆语言和概念，但对这个行业的真实情况了解得还不够。与一个理解任何类型的测试给他们的产品带来的价值的开发团队一起工作会有很大的不同。

在 Zendesk，像许多其他软件公司一样，我们遵循在每个开发团队中嵌入一个测试人员的模型(如果是一个大团队，有时不止一个)。这是培养这种伙伴关系的一个很好的方法，我们已经到了一些产品工程师开始承担编写和维护他们自己的自动化测试的责任的时候了

### 你对那些想从事软件测试的人有什么建议？

浏览 Selenium 教程，将你熟悉的事情自动化，比如做一些谷歌搜索。不要过于简单化，它会让你体验到大多数软件测试是什么样子的。如果你不测试 web 应用程序，这将是一件类似的事情，只是使用不同的工具。

### 迄今为止，在你的软件生涯中，你学到的最难的一课是什么？

在大多数计算机科学课程中，覆盖率测试是多么的少。我知道在我整个本科学位期间，我并没有真正接触到任何测试策略和测试设计，并且从我职业生涯中迄今为止与同事的交谈和工作来看，似乎大多数人得到的最好的东西是单元测试的介绍。

除了帮助他们定义他们团队的测试方法之外，与开发团队一起提出测试策略，并为他们提供各种类型的自动化测试覆盖范围的工具或框架，正成为我工作中越来越大和重要的一部分。大多数时候，我发现和我一起工作过的开发人员并不是不愿意承担通常在许多公司留给 QA 的工作，只是他们没有接触过或者没有以正确的方式展示出来。

### 对于成功的软件职业生涯，你的第一条建议是什么？

尽可能地练习诚实和同理心。这听起来有点模糊，但这些“软技能”可以以许多重要而微妙的方式表现出来，可以帮助你和你的团队建立一个更好的工作环境和更好的产品。这里有几个例子:

*   如果你不知道或不明白某件事，问问别人或查一下。

*   承认你的错误，但不要因此而自责。我们都在犯错误，并将继续犯错误，直到时间的尽头。

*   尽可能多地考虑终端用户对你在他们的环境中所构建的东西的体验。

*   质疑一切，既是为了澄清信息，也是为了挑战假设。

*   试着写所有的东西，从代码到文档，就好像是让不同团队的人去理解一样。

### 澳大利亚墨尔本的科技场景是什么样的？

它很大，而且还在不断增长。此时，你能想到的任何语言或学科都会有一个聚会(这提醒了我，我应该关掉 meetup.com 发来的关于新语言或学科的邮件)。Zendesk 全年在我们的墨尔本办公室举办一系列活动，以 9 月为例，我们有关于 Ruby、心理学、JavaScript、一般 web 开发、初级开发人员、机器学习和人工智能以及软件测试的活动。

一些国际软件公司在这里设立了工程办公室，包括 Zendesk(显然)、Xero、Slack 和 AWS，因此工作机会也越来越多。墨尔本也是一个非常适合居住的地方，所以这肯定会有所帮助。
