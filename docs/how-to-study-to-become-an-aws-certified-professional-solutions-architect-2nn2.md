# 如何学习成为 AWS 认证专业解决方案架构师

> 原文：<https://dev.to/kylegalbraith/how-to-study-to-become-an-aws-certified-professional-solutions-architect-2nn2>

作为同时获得 AWS 解决方案架构师和 DevOps 专业认证的人，我被问得最多的问题是，我是如何为他们学习的。这个问题比你想象的要难回答。然而，今天我将分享我如何为这两个考试学习的方法，并提供我对考试体验的见解。

### 专业解决方案架构师(SA)考试概述

首先，要参加专业解决方案架构师考试，您必须通过[助理级别考试](https://aws.amazon.com/certification/certified-solutions-architect-associate/)。我在[的另一篇博客文章](https://dev.to/kylegalbraith/how-to-best-the-aws-certified-associate-solutions-architect-exam-6g5)Dev 上分享了如何准备这场特殊考试的细节。

先修课程结束后，让我们快速了解一下专业 SA 考试评估的内容。在高层次上，它验证您是否具有在 AWS 平台上设计分布式应用程序和系统的技术技能。更详细地说，您能否证明自己在以下方面的能力:

*   利用 AWS 设计容错、可伸缩和高度可用的系统。
*   选择适当的服务，最好地解决给定的问题和规定的需求。
*   交流如何将复杂的工作负载迁移到 AWS。
*   向支持大型企业的 AWS 部署大规模应用程序。
*   了解 AWS 成本以及如何优化成本。

这些技能在考试中会在多个领域进行评估。域的划分如下所示。

### 如何准备专业解决方案架构师(SA)考试

首先，我想说的是，在获得专业认证时，接触 AWS 并亲身体验会让你脱颖而出。即使学习简单的用例，比如托管、保护和部署静态网站，也会对你的考试有所帮助。**在 AWS 上构建应用是最好的学习方式。**例如，我在参加考试之前，已经有大约五年使用 AWS 的经验。

也就是说，我仍然需要为专业考试而学习，因为我接触的 AWS 只有 25-30%的服务。

在为考试而学习的问题上，没有放之四海而皆准的方法。但是，鉴于我在 AWS 的现有背景，我可以分享我的学习方法。以下是我大致按时间顺序采取的步骤。

1.  下载[考试蓝图](https://d1.awsstatic.com/training-and-certification/docs-sa-pro/AWS_certified_solutions_architect_professional_blueprint.pdf)以确定考试的细分领域。知道这一点很重要，这样你就可以先把你的学习目标放在百分比最高的领域。
2.  通读考试蓝图，标出你可能不熟悉的地方。例如，网络是一个需要理解的大问题，因为它是在 AWS 中运行工作负载的关键支柱。你对人际关系了解得越多，你的情况就会越好。
3.  通读域后，记下与该域有联系的 AWS 服务。例如，EC2 Spot 车队及其与 AWS 中优化成本的关系。
4.  对于手头的服务，我根据我对它们的了解程度将它们排在 1-3 位。这个分数给了我一个基线，让我开始按优先顺序研究各种服务。 *1 =没用过，需要学习。2 =知道但不了解细节，需要更深入地了解。3 =使用它并了解主要场景，在边缘情况下更新。*

### 从你的弱点开始

现在你已经确定了你的薄弱环节，是时候开始熟悉它们了。这个过程会根据你学习的最佳方式而有所不同。对我来说，我是一个动手用自己的话写出来的学习者。所以我改善自己弱点的方法如下:

##### 首先在 AWS 控制台中进入您正在学习的服务

[![AWS Neptune](../Images/0fbe151e1b40d43815082b93c7292402.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--prbJa3_8--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/deypz4wxj7rfr8al30sr.PNG)

这是一个开始学习的好地方，因为它列出了许多高水平的内容，可以帮助您入门。这些页面通常包含以下重要信息:

*   服务的工作方式及其主要价值主张
*   如何开始使用服务(即提供服务)
*   示例使用案例
*   该服务提供的功能和优势
*   费用

##### 了解服务试图解决的问题

对于成为认证解决方案架构师来说，用例部分非常重要。你需要理解为什么客户想要利用一个给定的服务。使用案例部分帮助您了解客户希望从使用该服务中获得什么。

##### 通过 AWS 控制台手动提供服务

阅读关于这项服务的一页纸并不能给你提供工具来判断这项服务是否是考试的正确答案。您还必须了解如何提供服务。因此，请继续执行该特定服务的入门指南中的手动步骤。

在启动服务的过程中，请注意您不确定的任何配置设置，无论是现有的还是缺失的。如果是存储服务，这可能是静态加密之类的东西。就存储服务而言，了解它是否存在非常重要。

##### 如果可能，使用服务

您可以将应用程序部署到或合并到开发工作流中的大多数服务。不要在这一步花上几天时间，但是在一个示例项目中使用几个小时的服务将有助于理解可能会让您吃惊的部署模型、怪癖和边缘情况。

这里的一个例子可能是使用 AWS Lambda。一旦创建了一个函数，作为开发人员，我将如何开发代码来使用它。我如何将我的代码部署到函数中？我如何允许我的功能访问其他资源，如 S3 或 RDS？

实际使用你不擅长的服务的练习会比文档更快地提升你的水平。

##### 阅读文档了解术语、限制和成本

这是我心里的最后一步。我不是从这里开始的，因为我想知道一项服务的价值主张，并获得实践经验。如果你先通过阅读来学习，你可能想从这里开始。

当我完成这一步时，我会在谷歌文档中做大量笔记。我首先浏览文档，并对不同的术语做笔记。例如，如果我正在学习 AWS Elasticbeanstalk，我的术语注释可能是这样的:

**应用:**包含环境配置和应用版本的 Elasticbeanstalk 中的顶层对象。
**环境:**可以部署到的应用程序的基础结构配置。

一旦记录了术语，请仔细阅读该服务的局限性。如果这是在规模上使用，什么是硬限制和软限制。例如，S3 最初有一个 100 桶的软限制，您可以通过支持票请求更多。

了解使用该服务如何收费至关重要。考试中可能会问你如何优化该服务的成本，因此深入了解该服务如何向客户收费至关重要。这里一个例子可能是弹性贴图减少。了解 map-reduce 作业中工作节点的配置以及如何向客户收取费用非常重要。一旦你知道他们是按需定价，你可以看看使用现货实例。

如果用自己的话重写东西对你没用，那就别做。无论哪种方法对你来说都是不错的。然而，术语、限制和成本是你需要了解的关于任何服务的关键事情。

### 三三两两

说到您之前列为(2)或(3)的服务，也需要引起注意。他们可能不需要太多的关注，但你应该向自己保证这一点。

要做到这一点，你可以使用我们上面用过的同样的方法，但是要用一种简化的格式。我对这些服务的态度是确保我能回答以下五个问题:

1.  这项服务解决了什么用例/问题？
2.  我如何调配此服务，关键参数是什么？
3.  本服务中的关键术语是什么？
4.  这项服务有什么限制/约束？
5.  这项服务如何向客户收费？

如果你能回答这些关于一项服务的问题，你就有足够的知识来评估它是否适合考试。

### 我对专业考试的建议

在 NDA 制度下，我不能谈论考试内容。但是，我可以提供我对参加 AWS 专业认证考试的建议和见解。

1.  考试前，每天学习 1-2 个小时。但是，你考试的那天早上，没有学习。这可能会损害你参加考试的信心，并导致你过度思考答案。
2.  前一天晚上睡个好觉。
3.  考试前吃一顿健康均衡的饭，上厕所的时间计入考试时间。
4.  如果你准备好参加考试，时间不是问题。
5.  阅读整个问题，然后跳到答案。
6.  确定问题问的是什么，并确定它涉及的领域。
7.  马上排除肯定错的答案。如果是回答里一些你没听说过的晦涩难懂的服务，极有可能淘汰掉。
8.  一定要参加模拟考试，不要过多考虑自己的表现。它们有助于理解问题的格式以及回答问题所需的时间。然而，根据我的经验，它们经常充满了棘手的问题，这不是考试的标准。

### 结论

获得亚马逊网络服务认证是你的一大荣耀。但是，不要只拿一个去拿一个。它们有利于你的职业生涯，但也教会你很多关于 AWS 生态系统的知识。在不了解生态系统的情况下获得认证对你的益处远远小于了解生态系统并获得认证。另外，请注意，对于 DevOps 专业考试，我使用了同样的学习策略。不同之处在于我需要学习什么，而不是我如何学习。
