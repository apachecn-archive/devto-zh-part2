# 我学习新技术的四个阶段

> 原文:[https://dev . to/DOM nikl/my-four-stages-to-learn-new-tech-stuff-1f0c](https://dev.to/domnikl/my-four-stages-to-learn-new-tech-stuff-1f0c)

在过去的 15 年里，我已经成为一名专业的软件开发人员，我学习了超过 16 种编程语言和几种框架、数据库系统和其他工具，它们帮助我掌握了软件技能。

其中一些我学到了，因为我在一个客户的项目中需要它，但其中一些技术我只是好奇。如果你是一名软件开发人员，你就会知道我们的世界是如何快速地向新技术发展，用新技术来解决新类型的问题。

所以你需要快速掌握新技术，并且知道如何有效地学习。这是我今天学到的，一出分为四个阶段的戏剧:

# [](#stage-1-syntaxterminology)第一阶段:语法/术语

[![some JavaScript minified code](../Images/69f35896e6ed18de48570e1d3226fc11.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--NJolYGAl--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/mi7p8tznirk4fhkvtuf0.jpg)

关于一门新的编程语言或工具，你需要学习的第一件事是语法和术语。如果你已经知道另一种语言或工具，你已经学会了用这种语言解决问题的某种方法，但现在你必须找出如何用这种新语言解决类似的问题，并需要学习它背后的概念。

说实话，我学过一次语法，反正后来就忘了。我不得不经常用谷歌搜索这些东西。在我看来，最初了解它的最好方法是留意公案。

Koans 是一些小练习，你必须修改或编写小部分代码，并且通常有单元测试支持，你必须通过单元测试才能进入下一个单元。看看劳拉·黛安·汉密尔顿的综合 Koans 列表。

大多数工具都附带文档，并包含一个简单的教程来帮助您开始使用它。如果你打算学习 Docker，在官方 Docker 文档中有一个很好的例子。

第三方文档可能也非常有用。当使用 Go 时，我总是不得不谷歌一些东西，比如如何让 goroutines 使用通道与另一个通信，我总是最终使用 [Go by example](https://gobyexample.com) ，这是另一个很好的例子。你可以在[Y 分钟内学会 X 语言](https://learnxinyminutes.com/)找到其他语言的列表。

但大多数时候，我并不打算只学习一门新语言的语法。我想四处玩耍和探索。在这方面帮助我的是[代码大战](https://www.codewars.com)。通过解决小问题，它迫使我学习语法并探索有效解决问题的惯用方法。

# [](#stage-2-ecosystem)第二阶段:生态系统

[![the ecosystem](../Images/c973aff0a1f948d65eae3ce3cdee2823.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--y3TzNnG2--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/w1jabhpvpntalo8lur4q.jpg)

> "如果说我看得更远，那是因为我站在巨人的肩膀上."-艾萨克·牛顿

您已经了解了语法和术语，但是接下来该怎么做呢？在我看来，学习新语言或数据库系统的工具和库是非常重要的(也是令人兴奋的)。无论您是否需要日志记录、记录指标或连接到特定数据库系统的客户端，都有可以构建的(开源)库，如果您想用一种新的编程语言构建一些有用的东西，您需要了解它们。

幸运的是，GitHub 上有所谓的“[牛逼列表](https://github.com/sindresorhus/awesome)”，包含了你能想到的每种语言所需的一切。

在这个阶段，我试着用它做一些有用的事情。这意味着我可以使用我构建的工具，或者至少从制作它的过程中学到一些东西。通常我会在事后把它扔掉。不要对代码过于感情用事！毕竟，在你的职业生涯中，你会写很多这样的作品，但只有很少一部分会被制作出来或在那里停留很长时间。

所以大多数时候，我只是用我的新语言重写一个 Unix 程序。我用 C 写了`cat`，用 Python、PHP 和 Java 写了 Go 和 Kotlin - `tail`等等。当然，我从来没有使用过它们，但是它们是我非常熟悉的工具，并且有很好的文档记录，所以我有一个明确的目标。

除此之外，我经常尝试用另一种语言解决我过去已经解决的问题，并尝试用一种新的语言以更简单的方式再次解决它，这有时是一项令人信服的任务。

# [](#stage-3-best-practices-amp-experience)第三阶段:最佳实践&经验

[![an old wise man](../Images/9c1db6d7e290a3c39aabaeac01783ea8.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--TkwtFFCW--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/bdoqw9uqjg6d6q9xheoa.jpg)

当采用一种新语言时，语法和库并不足以构建稳定可靠、可维护且易于扩展的软件。你还需要获得使用新技术的经验。

通常这意味着在生产中运行它，遇到问题并知道如何解决它们。但是，还有另一种不那么有害的方法，那就是通过向他人学习和提前了解最佳实践。这个阶段是关于使用一种语言或系统的惯用方式。

我这里主要有两条路径:第一，有 Google。您可以在(官方)文档或博客帖子中找到许多关于最佳实践的内容，这些内容可以帮助您发现最佳实践。在会议上听到人们谈论他们使用这种技术的经验是很有帮助的。

# 第四阶段:摒弃旧的观念和习惯

在学习了一种新的工具或语言之后，你也不应该忘记忘掉一些东西。如果你拥抱并接受技术变革(如果你来到这里，我猜你会这样)，你也应该接受你必须在某个时候抛弃旧的习惯、模式和术语。毕竟在我们的行业中，每时每刻都在以非常快的速度发生着很多变化。

因此，为了不在学习新技术及其术语时感到困惑，你应该简单地忘记旧技术。举个例子:“表”在关系数据库系统出现之前肯定有另一个意思，人们不得不在 20 世纪 70 年代忘记它，那时那些 RDBMS 是新的尖端技术。

这就是我学习的四个阶段。我不认为这将是它的最后一次迭代，因为我不会停止学习，并因此进化我的学习过程。我也很乐意听到你学习灵感的过程。
