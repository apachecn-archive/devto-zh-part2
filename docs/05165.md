我不得不说我在[尝试中度过了一段美好的时光！今年 Swift 纽约](https://www.tryswift.co/events/2018/nyc/)。这些年来，我参加过许多开发者大会，但最近我主要参加商业和创业会议(即 [MicroConf](https://www.microconf.com) 和 [Release Notes](https://releasenotes.tv) )。

此外，这是我第一次参加苹果开发者大会，而不是由[主持](https://www.meetup.com/Lansing-CocoaHeads/)，出席，[在一个联席会议上发言](https://www.meetup.com/A2-CocoaHeads/events/246212307/)。这是一个新的环境。

我想感受一下 Swift 的发展趋势，所展示的主题有一些有趣的趋势:

### [](#arkit-and-coreml)ARKit 和 CoreML

大惊喜没错！🙃

苹果展示和支持这些技术已经有一段时间了。除了偶尔尝试创建模型之外，我在这方面做得还不够，而这些演讲强化了我这样做的需要。[艾琳·尼尔森](https://twitter.com/tryswiftnyc)真的分解了机器学习的工作原理。 [Berta Devant](https://twitter.com/bertadevant) 解释了 ARKit 是如何进行面部追踪的，并可以使用。 [Audrey Tam](https://twitter.com/mataharimau) 解释了如何使用苹果的新工具构建自己的模型。健太郎·松前展示了他的*超分辨率*技术的一个伟大应用。总体而言，在机器学习技术上投入时间的需求将继续增长。

### [](#swift-on-the-server)服务器上的 Swift

服务器上的 Swift 是真实的，并且有真实的公司支持它。我碰巧参加了 IBM 在 Kitura 举办的研讨会，初步了解了如何使用 Docker 和 Kubernetes 进行开发和部署。 [Gopal Sharma](https://twitter.com/gopalkri) 谈到了他在服务器方面面临的 Swift 挑战。 [Johannes Weiss](https://twitter.com/johannesweiss) 谈到了 [SwiftNIO](https://github.com/apple/swift-nio) ，这是客户端和服务器端通信的未来。[克里斯·贝利](https://twitter.com/Chris__Bailey) & [伊恩·帕特里奇](https://twitter.com/alfa)详细讲述了 Kitura 的部署过程及其未来[洛根·赖特](https://twitter.com/logmaestro)谈到了[蒸汽](https://vapor.codes)。

如果公司继续采用 Vapor 和 Kitura，我们将继续看到该领域的增长。这是 Swift 开发人员应该认真花些时间理解的事情。

### [](#podcasting)播客

播客仍然是热门的增长领域。我是两个播客的主持人: [OkProductive](https://okproductive.com) 和 [EmpowerApps。显示](https://www.empowerapps.show)。我关注过像 [Relay Fm](https://www.relay.fm) 和 [Gimlet](https://www.gimletmedia.com) 这样的播客网络，以及像 [Marco Arment](https://transistor.fm) 和[贾斯廷·杰克逊](https://transistor.fm)这样的软件创作者。因此，看到《断路器》的 Leah Culver 讲述他们的故事以及她对如何发展一家软件公司的了解真的很有趣。看看这个新行业中还有哪些创业公司蓬勃发展将是一件有趣的事情。

### [](#conclusion)结论

试试！Swift 是一次伟大的会议，也是认识 Swift 领域众多杰出人士的绝佳机会。如果你在 Swift 领域，尤其是初学者，我肯定会推荐你去看看。
