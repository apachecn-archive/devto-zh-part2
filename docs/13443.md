# 反对预制 CSS 框架(和设计系统)的案例

> 原文:[https://dev.to/bitersen/case-against-premade-css-frameworks-3jn](https://dev.to/bitersen/case-against-premade-css-frameworks--3jn)

如今，人们倾向于使用预制的 CSS 框架来处理任何事情。它们易于使用，文档也很丰富。CSS 框架已经变得如此普遍，以至于有些人可能认为这是每个人制作网站的方式。

[![Material design](../Images/bcfe61d0a6a0e3c946cd360132e7071f.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--_vEMQiR2--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/0qugrqghdtqhigf2kaao.png)

**错了！**

如果你正在使用这些 CSS 框架，尤其是像 Material design 这样的流行框架，你实际上是在窃取它所服务的公司的视觉身份。在这种情况下，你使用谷歌的设计系统，他们的品牌颜色和几乎所有的东西，除了他们的标志。

当谈到其他流行的框架，如 Bootstrap 或 Foundation，问题是你使用的风格与其他网站的风格一样！试着理解这个数字有多大，它对这些风格的独特性有什么影响。

## 为什么这对你来说是件坏事(对制造它们的公司来说是件好事)？

最重要的问题——你的公司在任何方面都不突出，这是**不值得纪念的**，你是**没有创造品牌知名度的**。如果你想创建自己的品牌，尤其是作为一家初创公司，你必须从一开始就做好，因为事后改变用户的看法或行为是极其困难的。

当您在工作中遇到必须对既定设计进行更改的情况时，可能是因为您希望根据反馈或数据进行一些改进，这些更改通常很小且很微妙。在这一点上，开发你自己的设计系统是非常危险的，因为它可能会疏远大部分已经习惯了你的网站或应用的用户。

与此同时，你基于材料设计的流行应用程序，例如，只推广谷歌和安卓。他们获得了品牌认知度、声誉……所有这一切。

试试这个实验——挑选几个非谷歌网站或应用程序，来自同一个行业，用材料设计制作，并把它们放在一起。看一看它们。试着在使用它们的几分钟内尽可能多地记住它们。一周后，试着回忆他们的设计中是否有什么特别突出的地方，让你能在视觉上联想到这个特定的品牌？

你肯定很难记起来。想象一下，同样的经历会在普通消费者心中留下什么样的影响。

## 什么时候可以使用预制的 CSS 框架？

*   样机研究
*   创建管理面板，不打算为大量观众或公众使用

如果你想快速创建一些原型，比如未来应用的模型，使用成品 CSS 框架是可以的。您可以将开发过程的这一部分比作工作设计阶段的实体模型创建。完成的设计不会与实体模型保持一致，就像完成的应用程序不会与 UX 原型保持一致一样。

对于你的应用程序中不公开的部分，比如管理面板，风格独特并不重要。拥有它当然很好，但它通常对业务或品牌认知度并不重要。

要非常小心，因为此时你正在让你的顾客习惯这些风格。如果他们倾向于在管理面板工作了很多，这些风格将变得非常接近他们的喜好，一段时间后，他们可能会要求你以同样的方式设计前端。在这一点上，祝你好运向他们解释任何关于品牌和设计的事情。

## 有什么解决办法？

如果你没有养成为顾客创造和提供独特风格的习惯，从长远来看，你会对他们不利。雇佣一个[网页设计师](https://dev.to/bitersen/web-design-in-2018--456f)或者 UI 设计师&开发人员，将他们与你的开发团队整合在一起。这并不难。因此，从长远来看，您将为您的客户创造更多价值，您的公司将因提供独特的高质量解决方案而闻名。
