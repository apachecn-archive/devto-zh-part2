# 保留还是不保留

> 原文：<https://dev.to/yechielk/to-retain-or-not-to-retain-574b>

#### 我如何才能记住我在训练营学到的所有信息？或者我应该？

尽管我在一年半前从 Flatiron 毕业，但我在 Flatiron Slack 频道上仍然相对活跃。我认为这是对所有在我起步时帮助过我的好人的回报。另外，这是我所知道的最支持和包容的社区之一！

最近有人向我提出了一个问题，由于我经常看到这个问题出现，我决定(在他们的允许下)向更多的人公布我的答案。

> 你好，叶切尔！
> 
> 我正在参加的在线训练营即将结束，但遇到了一些困难。我很好奇，想知道你在那个时代或任何时候是否经历过这种情况。
> 
> 我觉得我没有从课程和实验中记住太多的信息。我不知道这是因为我知道我已经接近尾声了，我的大脑已经不堪重负，还是因为我没有有效地学习。
> 
> 你有过这种经历吗，或者你认识的人有过这种经历吗？你是怎么克服的？
> 
> **感谢您的宝贵时间，**

你好，

是的，记住信息可能很难，尤其是当试图在短时间内塞进很多东西的时候(就我个人而言，我在课程的早期就遇到了这个问题，如果你现在只是抱怨你已经接近尾声了，那一点也不坏……:P)

我可以分享一些帮助我记住信息的东西(我会在信息结束时这样做)，但要提醒的是，每个人学习和记住信息的方式都不一样，所以对我有效的可能对你无效，反之亦然。

然而，在我分享对我有效的方法之前，我认为值得记住的是，弄清楚如何保留你正在学习的东西可能不是现在最需要关注的事情。

我的意思是，你将在训练营中获得的主要技能不是你正在学习的特定技能(Rails、React 等等)。相反，你应该自学的主要技能是如何自己学习和寻找你需要的信息。

例如，在我目前的工作中，我们使用 PHP、SQL 和非常简单的 JavaScript，这是否意味着我在 Flatiron 学习的 Rails 和 React 是浪费时间，因为我不再使用它们了？一点也不！我花了几个月的时间在熨斗自学这些东西，这意味着当找工作的时候，我已经准备好面对他们对我的任何考验。我参加过 Ruby、JavaScript、PHP、Java 和 Go 的面试，当我在没有任何 PHP 经验的情况下加入一家新公司时，我能够很快上手。事实上，在短短两周内，我已经用一种新语言在一个不熟悉的代码库中提出了我的第一个拉请求。

[![Woman looking at a computer frustrated](../Images/144715c96d783a17422630b13a6e9d97.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--wtwyIUAl--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/600/0%2ApgqteWNCJY8fl5Gf.jpg)

因此，实际上，每次你忘记了 React lab 的语法，回到相关的课程或谷歌如何将道具传递到组件中，你都在练习一项你肯定会使用 IRL 的技能；知道在哪里以及如何查找所需信息的技能。另一方面，如果你在第一次之后就记住了语法，你所能做的只是练习一种反应技能，这种技能在毕业后可能都不会用到。

所以总结一下；不要把精力放在记住此时此地的信息上，而要更多地关注提高技能，在你需要的时候帮助你查找信息:)

现在，为了不让你失望，我答应分享对我有效的方法:

记笔记:写东西帮助我记住它们(甚至不用回头去看笔记)。重复做某件事:我做某件事的前 2-3 次总是要回去查，做 5-6 次通常意味着我会记住一段时间。

最后，但绝对不是最不重要的，教导他人:我总是试图在 Slack 和 Flatiron 的提问论坛中提供帮助。我发现，向落后我几节课的人解释一些事情，有助于我巩固自己的概念，并更好地记住信息。
