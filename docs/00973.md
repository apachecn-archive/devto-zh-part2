## 问题:写代码很有趣…但是发布却不是

经常阅读的读者知道我在这里的使命是帮助人们过渡到 web 开发职业。我的部分指导是**回顾**学员已经走了多远，**展望**他们想要去的地方，并使用这些试金石**找到下一步**。

在与我的学员合作的过程中，我发现很多新手只需要运送一些东西。当你在学习的时候，制作东西真的很容易，但是，因为你不需要运送任何东西，所以那件东西经常被放在后面。**运输困难**，所以它被搁置，以利于开始下一个令人兴奋的项目。

然而，当你专业地做这件事时，**运输是*唯一*重要的事情**。当你带着一堆半成品项目去参加面试时，你可能会一无所有。当你试图获得自由职业者的工作时，如果你不能给别人一个网址让他们看到你做的东西，他们就不会相信你真的能帮助他们。**你就像一个读过一堆食谱但从未真正做过任何菜的厨师。**

## 解决方案:出货 It 学校📦

写代码和发货是**两种完全不同的技能组合**。编码是设计算法并用你选择的语言的语法来编写它们。发货比编码更早*开始*首先回答“我在解决什么问题？”它比编码晚结束*:当软件对现实世界可用时(无论是消费者还是单个客户)。发布是要面对 1000 个小障碍，并打破每一个障碍，直到你的软件发布。*

**Ship It School** 的创建**就是为了填补这个空白**并向所有人展示软件实际上是如何发布的:通过**务实的妥协**、**关于发布所必需的批判性思考**，以及**足智多谋**。

## 第一季

## 布拉德利·塞利斯

[![Bradley Serlis headshot](../Images/ef4151538ea0a38b3d6e916b1dd08ed4.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--Ss9VotoN--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://raddevon.com/wp-content/uploads/2018/11/bradley.jpg)[Bradley Serlis](https://www.bradleyserlis.com/)是一名刚刚从训练营毕业的学生，开始了他在 web 开发领域的新工作。他通过在 React Native 中构建他的项目，在他的训练营同伴中为自己开辟了一个利基。当他们构建在网络浏览器中运行的项目时，Bradley 构建了移动设备。

他一直在找工作，想听听我对如何着手的建议。我告诉他我对几乎所有处于他这种位置的人都说过的话:去找些自由职业者的工作吧，T3，T2。问题是这样的:布拉德利想做移动开发，但他还没有向任何移动应用商店发布移动应用。

他现在可以接其他的网络开发工作，他甚至可以接一份做移动应用的工作。当他的潜在客户要求看他的一些作品时，问题就来了。在 Mac 上展示一个运行在 iOS 模拟器中的应用程序**不如在实际手机上展示这个应用程序**强大。

## JamMate

[![Jammate splash screen](../Images/c729c56ec4769979d951f7eaa6e4deb6.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--v8yY_dSV--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://raddevon.com/wp-content/uploads/2018/11/jammate.jpg)

JamMate 是一个音乐人的社交网络。它可以帮助你根据人们喜欢的类型和他们演奏的乐器来找到他们。当我见到布拉德利时，他已经在这个项目上投入了相当多的工作，但它缺少了拼图中的关键部分:**它还没有发货**。

于是，布拉德利和我开始全力冲刺。我的参与微乎其微。我几乎没写过什么代码。(我完全是 React Native 的新手。)我的主要职责是**帮助布拉德利找出我之前提到的那些障碍**并**找出如何消除它们**。

## 壁垒

1.  布拉德利希望实现用户之间的消息传递，以允许用户开始他们的音乐合作。
2.  JamMate 使用脸书进行认证，而脸书要求你在使用他们的 API 时**提交你的应用进行审核**。
3.  数据库没有安全性。任何用户都可以读或写任何东西。
4.  该应用需要**构建并提交给应用商店**进行审核。

这些是最大的问题，但是布拉德利也有一个其他小问题和错误的洗衣清单。

*   用户列表显示了每个用户，即使他们离你太远，无法进行快速的会面和协作。
*   相关:由于他使用的是 Firebase，所以无法查询限制用户列表。他必须把所有用户拉进来，在设备上对他们进行过滤。
*   用户列表显示了每个用户…甚至是你。
*   用户的位置以一组坐标的形式从设备中获取，然后通过 API 运行并存储为邮政编码。在他达到 API 的免费上限后，这最终会产生费用。这也使得通过邻近过滤变得更加困难。
*   无法删除您的用户帐户
*   部分用户界面很粗糙

## 突破

以下是我们为突破这些障碍而采取的步骤:

1.  **明确了目标**。我们实际上想在这里完成什么？是为了建立一个广受欢迎的社交网络并获得 10 亿用户，还是为了看看发布一个应用程序是什么感觉？这将驱动所有后续步骤。
2.  毫不留情地将问题列表削减为我们必须在第一次发布前解决的问题列表。
3.  **发现拦截器**。我们仔细检查了我们的关键问题清单，发现了任何可能阻止我们前进的问题。我们找到的唯一一份主要报纸是《脸书评论》。这必须在应用程序提交到应用程序商店之前发生。
4.  **先处理拦截者，然后再处理其他事情**。通过尽最大努力解决所有的阻碍因素，我们能够在很短的时间内发货。在 Ship It School sprint 概念形成后的几周内，我们能够将应用程序提交给苹果公司进行审查。

但事实远不止如此。例如，我们发现我们不需要获得脸书评论，因为应用程序只使用它进行身份验证(🎉)，*但是*我们了解到我们需要一个应用程序的公共隐私政策(💩).在这个过程中，我们学到了数百个这样的小教训，但是，更重要的是， **Bradley 现在已经非常接近拥有绝对的证据，证明他可以发布移动应用**。(应用上线后，我会进行事后分析。)
