## [](#que-intro)阙简介

美好的一天开发到社区！我希望外面的每个人都成功或者坚持不懈。今天是我比后者更成功的一天。为什么？嗯，因为今天我要庆祝我与 [Pillar Technology](https://pillartechnology.com/about) 合作一周年！今天也标志着我停止在这个平台上发帖的结束。所以作为这个街区的新成员，我应该介绍一下我自己。

[![alt text](../Images/3d5745453fbd870b8e6e36e0ccbc4ecf.png "Hello World")T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--J_7KijwA--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/5h7qs2mnae8xmva0gyjo.jpg)

我的名字是马修·奎因。我是 Pillar Technology 的一名软件工匠，如上所述，今天标志着我在该公司工作一周年。为了庆祝这一年的成功，我将回顾到目前为止的旅程，然后制定第二年的目标。

## [](#journey-to-pillar)前往石柱

我的 Pillar 之旅是一次考验我毅力的旅程。回到 2014 年夏天，我即将进入大学四年级，我在一家财富 1000 强公司实习，这家公司把我平淡无奇的简历变成了值得一看的东西。我在学校学习营销分析，打算找一份工作，让我可以用自己对统计学和数据的热爱来讲述一个故事。这次实习是我进入这个领域的契机。因此，那个夏天的每一天，我都面带微笑走进那家公司，并努力尽我所能提供价值。这得到了回报。实习快结束的时候，我收到了一份全职工作邀请，要求我在毕业后继续工作。我被极客了！我的第一份“真正的工作”。我再高兴不过了！我也不是，直到我不是。

[![alt text](../Images/6f3744e0000cbbcd527b3721244b5859.png "Happy Face, Sike!")T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--BKTfZLwF--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/d4x3xu4eza08o4lgjsch.png)

不舒服的文化、缺乏联系、没有导师指导的挑战减少，这些因素累积到我不再快乐的程度。我仍然喜欢我正在做的事情的某些方面。然而，我意识到职业生涯不仅仅是冷静对待工作。所以，没有计划，没有真正的下一步，不知道接下来会发生什么，我离开了。我一点也不知道，这一举动引发了一系列事件，将我带到了现在的位置。

[![alt text](../Images/9d4e1952e1959f7d6d4c781806460b27.png "Timeline of Events")T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--P8vRxzwf--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/qskrq3654themn1xhcvt.png)

我不会深入讨论以上时间线中发生的所有事情(我会在其他帖子中分享这些经历)，但我知道这是一个颠簸的十个月。从从副业项目中赚钱，到突然加入一个编码训练营，再到在拒绝我的第一份工作之前被 27 家公司拒绝，等待 Pillar 的回复，这是一次疯狂的经历，其回报超出了我的预期。

### [](#looking-back-at-year-one)回首一年

我在 Pillar 的第一年过得很糟糕。我完成了学徒计划，毕业并获得了我的第一个客户，在我们公司的会议上发言，并与非常有才华的人建立了联系。过去一年我经历的成长是惊人的。不过，有一件事我没有做到，那就是设定要达到的实际目标。最初，我只是想变得更好，从学徒项目中毕业，但对于第二年，我希望我的经历不止于此。因此，我为下一年设定了目标。

### [](#goals-for-year-two)第二年目标

[![alt text](../Images/1f2586576532e7f496f7b34ea962694b.png "GOALS")T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--QwCyjMPZ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/b06i6kcaxbgd2o6bo11m.jpeg)

1.  **深入研究 JavaScript 并做出反应**:为咨询公司工作意味着所使用的技术因客户而异。这最终转化为软件工匠需要能够学会如何学习。然而，由于在 Pillar 之前没有任何技术经验，我发现我周围的许多人对*的东西*很满意，而我却不满意。因此，第二年的目标是深入研究 JavaScript 和 React。

2.  **交流**:我相信巩固你所知的最好方法是交流你所学的。任何与我在技术世界的经历或我对 JS 和 React 的发现有关的东西，我都想交流。这将通过在网站上发布，在闪电谈话和午餐和学习中介绍，提问，以及阅读其他人如何执行相同的实践来完成。

3.  **创造。创造。创造**:学习的另一个重要部分是通过创造完成的。我将尽我所能用我正在深入研究的技术创造更多。小项目。项目的各个部分。大型项目。没关系。我只是要创造。

4.  **Dev.to** :我成为 Dev.to 社区的一员已经有一段时间了，作为一个读者而不是作家。但今天我要改变这一切。我会养成习惯，在这个平台上至少两周发布一次帖子。

最后，我感谢你们花时间阅读这篇文章，我期待着成为这个社区更积极的一员！

周一快乐！
