### 自我介绍！你是谁？你在哪里工作？

嗨！我叫 Shelley，是一名游戏开发人员，住在澳大利亚墨尔本，但来自新西兰。我现在的正式头衔是极客联盟的 UI/UX 程序员。我倾向于关注 UI 特性，但是在一个相对较小的团队中工作意味着我可以在需要时参与游戏的各个领域，从游戏性到移植到音频编程。我从 2011 年开始从事游戏工作，在 2016 年搬到这里之前在奥克兰的一些工作室工作。

### 谁或什么让你进入游戏编程？

事实上，我陷入其中主要是靠运气！我一直喜欢玩游戏，但不知道创造一个游戏需要什么。当我高中毕业，不得不选择一个大学专业时，我最终选择了计算机科学，因为我“喜欢计算机”。我甚至不知道什么是编程——我小时候或在学校里从未接触过它。幸运的是，事实证明这是我真正喜欢的事情，我们 CS101 班的最后一项任务是创建一个 2D 简单游戏，那时我知道游戏实际上可以成为一个真正的职业。在毕业前的最后几周，我在我们的课程网站上找到了一个当地游戏工作室的实习名单，并设法得到了这个角色。我一考完试就去了那里，从那以后就一直在这个行业工作！

### 游戏开发者和非游戏开发者的区别在哪里？帮助一个人成功的关键差异是什么？

我发现，游戏通常有一个不太正式的过程，决策是不断变化的。正因为如此，我倾向于将“游戏感觉”置于“干净的代码”之上(这可能是也可能不是最好的做法)。我认为对游戏开发的不同领域有基本的了解和兴趣是非常重要的。虽然我实际上不能创造艺术资产，但我知道艺术管道，这使得与艺术家的沟通容易得多。对于某些角色来说，对创造性方面的兴趣也很重要——UI 不是很多程序员喜欢实现的东西，但是我发现创建 UI、调整那些动画和计时并增加额外的修饰是非常令人满意的。

### 澳洲的游戏开发场景是什么样的？澳大利亚和新西兰在世界舞台上竞争吗？

我们仍然相对较小，但正在增长，我们绝对是在世界舞台上竞争。在过去的几年里，已经有一些非常受欢迎的游戏出自 ANZ 产业。不幸的是，政府对游戏产业的支持不多(在维多利亚，我们很幸运，有电影《维多利亚》的支持)，这肯定是阻碍我们前进的原因。

### 迄今为止，在你的软件生涯中，你学到的最难的一课是什么？

不要感情用事，对你工作的批评不是人身攻击——这适用于那些想为你如何改进提供建议和见解的同事/导师，也适用于公众和你产品的消费者。这一点在游戏中尤其重要，因为游戏中有大量有资格的玩家对你的游戏应该如何运作有自己的看法，他们对此非常直言不讳，并且基本上认为有必要告诉你应该如何工作。

### 对于成功的职业生涯，你的第一条建议是什么？

不要自负，永远要乐于学习。行业是不断变化的，没有人什么都知道。我的每一个新角色都是一个巨大的新学习机会，充满了不同的挑战。

### 工作之外，你有什么爱好吗？你认为它们对你的职业生涯有任何帮助吗？

我有个坏习惯，拿起新东西，几个月后就扔掉了。我喜欢你第一次学习并取得快速进步时的那种满足感，但当你过了基础阶段，进步变慢时，我很快就失去了兴趣。摄影是我在过去一年左右一直在玩的一件事，最近开始变得更加认真，我期待着继续尝试它！虽然最近我大部分空闲时间都在新的蜘蛛侠游戏中的照片模式中度过！我绝对认为拥有与职业无关的爱好和接触不同的视角会有所帮助。

### 你会向那些想走上与你相似道路的人推荐哪些书籍/资源？

我很幸运，因为我总是能够在工作中学习-从零游戏经验开始我的实习，我花了第一周做 Unity 教程并熟悉引擎。从那时起，我通常做 50/50 的游戏和 UI 任务，所以我现在的角色感觉是自然的下一步。我建议查看一些 GDC 讲座(有很多免费的)，教程(Unity 在他们的网站上有一个不错的范围)和诚实的 Twitter。Twitter 可能很糟糕，但是它上面的 gamedev 社区非常开放和有帮助，我在那里遇到了一些很棒的人。如果可能的话，也可以看看当地的聚会。这是我早期做的很多事情，它真的帮助我走出去，认识其他开发人员，它肯定对我在 LoG 之前的角色起到了一定的作用。

### 最后，做出你的大喊！你想让读者去看什么？

去支援检查站([https://checkpoint.org.au](https://checkpoint.org.au))！他们是一个慈善机构，专注于提高对心理健康问题的认识，并提供专注于游戏行业和社区的资源。他们做了一些非常惊人的工作。
