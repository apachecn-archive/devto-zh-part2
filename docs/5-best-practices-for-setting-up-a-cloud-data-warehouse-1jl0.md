# 设置云数据仓库的 5 个最佳实践

数据仓库是来自不同来源的集成数据的中央存储库，已针对分析和报告进行了优化。云数据仓库采用了一种新的方法，将数据仓库功能作为服务提供，而不是在传统的内部数据中心提供。

利用数据仓库即服务(DWaaS)方法有几个好处，包括:

*   低成本数据仓库，因为它消除了建立和维护内部数据中心的高昂费用。
*   将更多时间放在从数据中获得的价值上，而不是管理数据中心的复杂性上。
*   轻松的可扩展性，能够轻松适应增加(或减少)的数据量和工作负载。
*   让更多可能没有资源来构建自己的内部数据中心的公司更容易访问数据仓库。

对于企业来说，向云数据仓库的过渡可能会很困难。无论您的公司是从传统的内部部署迁移，还是云代表您首次涉足数据仓库，这五个最佳实践都将帮助您从一开始就做好准备。

## 了解云数据仓库架构

每个云数据仓库服务提供商都有自己独特的架构，但是，在选择供应商之前，您需要熟悉一些常见的架构趋势。

*   [大规模并行处理](http://www.ittoday.info/AIMS/Information_Management/3-02-45.pdf):几家领先的云数据仓库供应商使用大规模并行处理架构，将数据划分到许多服务器或节点上，每个服务器或节点都有自己的处理器。MPP 设计同时协调所有处理器上的查询工作负载，以实现更快的数据查询。
*   列存储:在列数据库中，表是按列值存储的，而不是按行存储的。这种类型的设计通过减少回答查询所需的时间来提高性能。
*   无服务器:一些云数据仓库供应商有一个无服务器架构，其中你不需要提供或管理任何基础设施。

[与传统的本地数据仓库架构相比，Panoply 在云数据仓库架构上有很好的资源](https://panoply.io/data-warehouse-guide/data-warehouse-architecture-traditional-vs-cloud/)。

## 满足安全&合规要求

对于考虑云数据仓库服务的企业来说，安全性和合规性问题是重大的绊脚石，这是理所当然的。毫不奇怪，决策者担心云中的信息安全违规风险——当数据被信任在供应商手中时，由于缺乏控制，这可能会产生恐惧。当您考虑到数据仓库是商业智能的核心组件时，对安全问题的担心会进一步加剧。

必须进行尽职调查，确保潜在的[数据仓库云供应商](https://www.datamation.com/big-data/top-data-warehouse-tools.html)能够满足您所有的安全和数据保护要求。需要注意的重要事项包括:

*   强用户身份验证策略
*   数据加密
*   符合相关行业法规，例如， [HIPAA 针对受保护的健康信息](https://aws.amazon.com/compliance/hipaa-compliance/)
*   强调供应商在虚拟化环境和底层硬件上的数据安全、保护、归档和复制政策的详细文档

## 进行分阶段迁移

不要一开始就全押——分阶段的方法最有意义，在这种方法中，首先将最简单、技术含量最低的工作负载迁移到基于云的数据仓库。可变工作负载(如财务报告活动)是初始迁移的理想选择，因为它们不会持续使用数据仓库资源。作为部门数据集市的一部分，业务单元特定的工作负载也适合初始迁移。

持续工作负载(如推动战术业务决策的每日报告)可以稍后迁移，业务关键型可审计工作负载也是如此。循序渐进地做事可以在云数据仓库中建立信心和信任，获得的经验对于迁移更复杂的工作负载和用例将是无价的。

## 验证 ETL 工具

传统上，ETL(提取、加载、转换)工具用于将数据从事务系统移动和集成到数据仓库中。在数据最终进入数据仓库之前，对数据进行整合和转换。对于企业来说，确保现有的 ETL 工具和数据流在支持云原生技术的情况下针对云实现进行验证是非常重要的

对于首次进入数据仓库的企业来说，最好是寻找本机云数据集成工具( [ETL as a service](https://aws.amazon.com/glue/) )，这些工具旨在应对将本地事务性数据库系统中的数据集成到云存储库中的挑战。

越来越多的企业正在使用 [ELT 方法](https://searchdatamanagement.techtarget.com/definition/Extract-Load-Transform-ELT)，在这种方法中，原始数据直接从源系统传输到云数据仓库，在那里根据特定业务用例的需要进行转换。基于云的基础设施的力量使得 ELT 更加可行。

## 确保成本透明

尽管对于数据仓库来说，云无疑是一个更便宜的选择，但保持数据仓库活动所产生的潜在成本的透明度仍然至关重要。不同的供应商使用不同的成本模型，对查询的数据量收取一定的费用。大多数供应商还对数据出口(将数据从公共云移回内部系统)收费。

在所有情况下，预测成本并向相关业务用户传达高效查询驻留在数据仓库中的数据的重要性是一个好主意。其他成本的可见性也很重要，包括数据备份和灾难恢复费用。

## 总结起来

这些最佳实践将有助于指导您选择正确的服务提供商，并在云数据仓库中进行设置。一旦你决定进行转型，确保你的公司与你选择的供应商建立了密切的关系，重点是响应能力、绩效和参与度。
