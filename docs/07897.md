## 所以你想在 Node 中重写你的 Java 库...

我最近遇到了这样一种情况，我想用一种新的语言重新实现一个现有的项目，我试图决定如何最好地管理与现有产品相关的新代码。

我最初的想法是简单地创建一个新的存储库；这具有项目历史和问题之间的清晰区别的优点。

```
java-repo
    > master
        - dir1
        - dir2
node-repo
    > master
        - dir1
        - dir2 
```

```

    > master
        - java
            - dir1
            - dir2
        - node
            - dir1
            - dir2 
```
