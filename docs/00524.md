# 自由的意义

> 原文:[https://dev.to/terceranexus6/the-point-of-freedom-1i68](https://dev.to/terceranexus6/the-point-of-freedom-1i68)

伙计们，你们好吗？我有一个想法已经有一段时间了，我很想和你分享并听听你的想法。这不是很技术性，但我认为它在任何技术性工作中都很重要。

不久前， [DEV 开源了](https://dev.to/ben/devto-is-now-open-source-5n1),这让那些关注网站开发的人非常兴奋。我们中的一些人非常习惯于创建免费和开放的内容，并乐于加入 Linux 讨论、活动等。但是，这有什么意义，为什么我们在这列火车上？如果可以的话，我想先解释一下免费和开放之间的区别。

本质上，软件的自由是由理查德·斯托尔曼在 80 年代提出的四条规则定义的。正如在 [GNU 宣言](https://www.gnu.org/philosophy/free-sw.html)中所解释的，这四条规则是:

[![](../Images/da27c8c55583b85b5c8b1c590a153468.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--KTeLd09h--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.gnu.org/graphics/heckert_gnu.transp.small.png)

*   出于任何目的，按照自己的意愿运行程序的自由(自由 0)。
*   研究程序如何工作的自由，并改变它，使它如你所愿地做你的计算(自由 1)。获得源代码是这样做的先决条件。
*   重新分发拷贝的自由，这样你就可以帮助他人(自由 2)。
*   向他人分发你的修改版本的自由(自由 3)。通过这样做，你可以给整个社区一个从你的改变中获益的机会。获得源代码是这样做的先决条件。

是的，它从*零*开始，因为我们都知道数组是如何工作的；)而且听起来很棒！它谈到了改进、分享、学习...

**开源**是一个术语，正如在[官方来源](https://opensource.org/osd)中所描述的，它按照法律要求处理代码分发的方法，以便让社区可以访问它。两者都丰富了软件开发，据说软件的自由有更深的含义，更像是一种*的生活方式*或一种*的思维方式*。

现在，我想介绍一些疑惑、想法和问题，它们已经在我的脑海里有一段时间了。

在我看来，我们生活在一个非常快速、狂热的日常生活中，这让我们加入了一个快速思考的动态中。你的观点必须总结在一条易读的推文中(最好有图片！)一篇快速可读的文章，一次电梯推销...这是大规模信息社会的正常产物，但它有一个危险的陷阱；快速文化也缩短了我们的思考时间。为什么我现在指着这个？嗯，有时候，在某些情况下，我看到软件自由被用作一个标签，以使一些东西变得更好，或者看起来像是 T2 的程序员友好、社区友好，而不是真正应用这个概念背后的基本精神。我想澄清一下，我不是说软件自由是某种童子军徽章或类似的东西，但是**当我们致力于它的时候，我们真的是这个意思吗？**

软件自由的整个运动基于社区意识，代码的进化不是个人的。也是不应该拥有的东西。贡献的分配旨在规范和发展，而不是个人范围。让每一个提交都值得审查，从而创建更大、更广泛的项目。这需要一个脉动，一个不仅创造，而且不断改进和批判的活跃社区。**需要批评家**为了不陷入绝对正确的一致性，没有批评就不可能有进步。**当软件的自由被用作正确的标签，仅仅是一个形容词时，会发生什么，难道它没有失去它的实体吗？**

同样的概念打破了虚拟墙，进入了硬件社区。硬件的自由，在那些所谓的制造者的带领下，使得令人兴奋的令人惊奇的事情、项目和想法成为可能，并且允许几乎任何人参与到电子产品中，这在几年前是不容易的。事实上，我在创客社区中看到了一些我在软件社区中没有轻易发现的东西:更多的多学科成员。你们当中有人遇到过这种情况吗？许多与技术有关的艺术家加入了创客社区，例如，还有教育家，甚至心理学家。

我坚信科技是任何专业人士都应该参与的事情，为什么不呢？我们都已经扮演了用户的角色，参与了数字世界，为什么还要远离它的发展呢？你认为任何类型的专业人员都能够丰富开发者社区吗？

希望收到你们的来信，伙计们！
