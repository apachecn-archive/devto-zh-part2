# 如何傻逼的总结

> 原文：<https://dev.to/tiptopgs/summary-of-how-to-idiot-1a7f>

今年我将参加#RWDevCon，这让我想起了去年里奇·图尔顿的一个鼓舞人心的演讲“我是个白痴”。如果你错过了他的演讲，你可以点击这里查看。像他这样的白痴是我喜欢与之共事的人。以下是我从他的演讲中摘录的内容。我的笔记很好地提醒了我，在科技领域，一个伟大的目标是追求成为一个白痴，并对此无动于衷。就像里奇·图尔顿说的，把这个世界变成一个更好的地方，让未来的白痴看到你的代码(包括你未来的自己)，这是一种很棒的感觉。

总之，用我自己的话来说，永远不要在没有完全理解的情况下离开一段代码。花时间重写一遍，这样不仅你，下一个傻逼也能看懂。比起简洁，更喜欢清晰。我们为什么要这样做？我们这样做是为了向其他白痴表示善意，最重要的是向我们自己表示善意。
