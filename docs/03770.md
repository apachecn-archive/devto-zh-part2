TypeScript 今天已经六岁了，我想花一些时间来思考一下这项惊人的技术在个人层面上对我意味着什么，以及为什么我认为您真的应该看看它(如果您还没有这样做的话)。

当 TypeScript 第一次被引入时，我正在使用 CoffeeScript，并且我有诸如 C#之类的微软技术背景。我记得在安德斯·海尔斯伯格的主题演讲宣布的第二天，我看了这个演讲，从第一天开始，我就对这个想法感到非常兴奋。

从第一次预览开始，TypeScript 是一项非常好的技术，但随着时间的推移，事情变得越来越有趣。泛型类型和不可空类型是两个重要的改进，其次是映射类型和条件类型。我认为这个旅程对于 web 开发社区中的每个人来说都是非常令人兴奋的，但是当我思考它对我个人的意义时，它是令人惊奇的。

# 与类型开玩笑

在 TypeScript 到来之前，我知道一些强类型编程语言以及一些动态编程语言。然而，我没有使用函数式语言的类型系统的经验，比如 Haskel。

TypeScript 感觉与其他强类型语言不同；我经常发现自己想知道如何表达一个给定的类型。随着时间的推移，由于打字稿类型系统的表现力的增加，这些感觉也增加了。我感觉 TypeScript 时不时地用小的“类型难题”来挑战我，要求我表达一种类型。解决其中一个谜题的奖励是类型安全代码。起初，并非所有这些“类型难题”都能解决，但随着语言的发展，最复杂的难题现在可以通过条件类型或查找类型等特性得到解决，而最简单的难题可以通过类型推理系统的改进得到自动解决。

这很难解释，但我觉得 TypeScript 让我更多地考虑类型和任何其他编程语言。

回顾过去，TypeScript 帮助我学习了很多关于类型和类型系统的知识。我原以为我知道这些话题，但现在多亏了 TypeScript，我发现这是一个极其广泛的主题，需要很多年才能真正掌握理论。我很感谢 TypeScript 团队和社区，因为 GitHub 问题包含了有用的评论，帮助我学习了很多关于 TypeScript 的知识，特别是关于类型系统和编程语言如何工作的知识。因此，TypeScript 帮助我成为了一名更好的工程师，这也是我强烈建议您尝试一下的原因。

# 机遇之地

打字稿对我个人来说也意味着好运。因为我很早就采用了 TypeScript，所以我有机会写一本关于它的书。这本书帮助我赢得了一些自信，这使我尝试公开演讲，并创办了都柏林 TypeScript meetup，这使我获得了微软 MVP 奖。

# 开源

TypeScript 还帮助我学习了很多关于开源项目管理、GitHub 和社区的知识，因为我已经能够观察到 TS 团队如何以绝对透明的方式管理项目和社区。

# 总结

在过去的六年里，我非常喜欢使用 TypeScript。这项技术对我的发展帮助最大，帮助我学习了大量的新东西，也给我带来了一些惊人的机会。这就是为什么我要感谢 TypeScript 团队和社区的其他成员创造了如此伟大的技术。我迫不及待地想看到未来六年它将如何演变。

生日快乐，谢谢你！
