# 10 年后重返 Linux

> 原文：<https://dev.to/rvprasad/returning-to-linux-after-10-years-dc0>

在使用 Windows 和 MBP 进行开发近十年之后，我最近又切换回 Linux 作为我的开发平台。为此，我选择了跑 PopOS！在[系统 76](https://system76.com/) 的 [Oryx Pro](https://system76.com/laptops/oryx) 笔记本电脑上。

# 羚羊 Pro

[![An image of Oryx Pro](../Images/a630310eb5a251761358a02d7dc19071.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--DDpMwPmn--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/xanth5w2y5d7k0lq7733.jpeg)

在能力方面，Oryx Pro 给人的印象非常深刻。它可以容纳一个 NVidia GeForce 1060 或 1070，高达 64GB 的 RAM 和 10TB 的存储空间！！

虽然它不是来自大公司(如戴尔、联想)，但它构造精良、外观时尚、坚固耐用。就我们对笔记本电脑外形的期望而言，它有点笨重；尤其是，如果你是从 MBP 过来的。然而，它在功能方面补偿相当慷慨。

就体验而言，一切开箱即用。没有打嗝也没有死亡。System76 的客户服务和技术支持非常有帮助。

就像生活一样，没有一次经历是没有缺点的:)在使用笔记本电脑几天后，我意识到有三个主要的烦恼。

*   电源端口接受 L 形电源插头，就在 HDMI 端口旁边。因此，当 HDMI 端口被占用时，电源插头无法 360 度旋转。因此，由于插头之间的相互作用，HDMI 端口和电源端口上可能会有过度的压力。我怀疑一次意外的拉扯会造成不小的损失。直插电源插头可以轻松解决这个问题。[ *更新*:使用远离电源端口的迷你显示端口解决了这个问题。]
*   集成的英特尔 GPU 没有连接到三个外部显示端口中的任何一个。这意味着当连接到外部显示器时，NVidia GPU 不可用作专用计算 GPU，这是人们希望使用专用 GPU 进行计算的时候。
*   一次电池充电持续时间不到 90 分钟。这与 Macbook Pro 在类似工作负载下连接外部显示器(由 NVidia GPU 驱动)时的情况相比要少得多。我想知道如果英特尔 GPU 可以驱动外部显示器，情况是否会是这样。到目前为止，我一直在尝试各种各样的调整，它们似乎正在产生一些变化。我需要运行更多的测试来确认观察到的差异是持续的和实质性的。

即使有上述烦恼，如果你正在寻找一个价格合理的可配置的开发就绪型笔记本电脑(CPU 和 GPU)，那么我很高兴地推荐 Oryx Pro。在订购您的笔记本电脑之前，只需与 System76 的销售/技术团队交谈并验证您的假设即可。

# 流行 OS！

我过去使用 Linux 的经验是，如果你能找到并配置合适的设备驱动程序和应用程序，并让它们相互配合，那么 Linux 就是开发者的天堂。查找和配置一直是个难题。令人高兴的是，今天情况似乎并非如此。

Oryx Pro 预装了 [Pop OS！](https://system76.com/pop)，一个基于 [Ubuntu 17.10](https://www.ubuntu.com/desktop/1710) 的发行版。打开 Oryx Pro 后，笔记本电脑中的所有设备都可以运行，并且可以通过软件使用。Fn+F10 切换网络摄像头。VLC 能够显示来自网络摄像头的反馈。USB 端口以最高速度运行，内核标志没有任何变化。GPU 被识别，没有问题。外置硬盘、显示器、键盘和轨迹球(是的！)被零调整地识别出来。我可以在两分钟内配置好轨迹球键。我能够配置 Gnome/X 显示器跨越多个显示器。不到 3 个小时，我就在笔记本电脑上安装了所有需要的软件(包括从网上下载)。是的，我已经到达了我的开发者天堂:)

然后，我面临两个问题。

*   我开始设置我的个人应用程序特定的偏好。与在 PC 或或 MBP 上不同，我必须手动识别并从不同的机器上复制相关的配置文件。这花了一些时间，原因有两个:a)这是一个偶尔的任务，我对此很生疏，它需要相当多的网络查询，b)没有应用程序可以在 Linux 和 PC/MBP 之间迁移我的应用程序特定的偏好。由于这是一个不常见的任务，我认为这是一个次要的便利。
*   我开始玩英伟达设置 app，在英特尔 GPU profile 和英伟达 GPU profile 之间切换。虽然应用程序要求我注销以使切换生效，但我必须重启笔记本电脑以使切换生效。在我看来，这是一个很大的烦恼。

    现在，有可能我在这里做错了什么，所有需要做的就是注销和登录。即便如此，如果一个人在这样的切换时有许多应用程序在运行，这仍然是一个很大的不便。

尽管存在这些问题，但回到一个可以轻松地从二进制文件安装应用程序或从源代码构建应用程序并继续工作的平台上还是令人耳目一新。[要体会这种感觉，应该尝试在 Mac 和 Windows 上安装 Python 的源代码库。]

虽然我有时需要使用 PC 或 MBP 来完成特定的任务(因为它们在 PC 或 MBP 上更容易且得到更好的支持)，但我目前很高兴回到 Linux 上进行开发。
