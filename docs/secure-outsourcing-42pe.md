# 安全外包

> 原文：<https://dev.to/galyna_chekan/secure-outsourcing-42pe>

虽然外包软件开发有明显的好处，但也有风险。

与您合作的软件供应商实际上是您团队的延伸。你可以授权它访问你公司的敏感数据和客户记录，让它深入了解你的基础设施和内部业务流程，甚至了解你的 it 系统的缺陷和局限性。

我们都听说过这样的故事:公司与可疑的离岸公司合作，却发现后者的安全标准不足，他们编写的代码也很糟糕(要么有缺陷，要么是故意编写的逻辑炸弹和后门程序)，这一切都为时已晚。

每一个雇佣离岸软件工程师的企业都潜在地将自己置于一个易受攻击的位置。然而，像甲骨文和微软这样的 IT 界巨头，多年来一直在成功地使用外包开发。

*那么，有什么秘诀呢？*

无论您是寻求利用发展中国家的区域经济(通过利用廉价劳动力来削减成本)，还是寻找渴望的、训练有素的、拥有您当前需要的特定技能的专家，您都必须负责任地解决与外包软件开发相关的风险。鉴于即将出台的 GDPR 法规，这意味着不仅要仔细挑选合作伙伴(根据评估等)。)还要仔细检查供应商的安全管理流程，确保他们所做的一切都符合您公司的政策。

在本帖中，我们会给你一些如何在使用外包开发时保持安全的提示。
