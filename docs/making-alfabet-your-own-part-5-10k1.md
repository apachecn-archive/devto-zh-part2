## 简介

运作良好的 IT 转型计划的关键是信息的完整性、一致性、准确性和流通性。利益相关者必须完成他们在 IT 规划和管理过程中的部分，无论是记录规划元素、研究问题领域、分析性能、创建新的解决方案架构，还是决定新的投资。数据采集流程应尽可能简单直观，以确保知情人士的合作和能力。Alfabet 提供了几种数据捕获模式，例如编辑器、简单的数据输入表单、图表和可视化、表格和数据捕获矩阵——所有这些都是高度可配置的。所有这些元素都可以组合到向导中，以在数据捕获过程中提供端到端的指导。

## 奇才

向导解决了 IT 管理中的一个最大问题:拥有一个以结构化方式捕获信息的系统方法。统一的数据格式对于聚集数据进行分析是必不可少的。标准向导可确保 Alfabet 库存中关于“商品”对象(如需求、外围设备、组件、设备和应用)的一致数据捕获和高质量数据。在一个易于使用的线性流程中，通过配置用户提示，对主题和解决方案知之甚少的用户可以轻松地执行数据输入。因此，向导可以为大型用户社区提供最佳支持，并加快推广速度。

此外，向导使信息需求能够沿着阶段门进行组织。这对于需求和项目尤其重要，因为不同的最低信息集与规划、评估和决策过程中的不同里程碑相关联。通过能够定制在对象上捕获的信息类型，向导以两种方式支持联合 IT 规划和管理:

*   对象的数据责任可以由不同的贡献者分担，例如，合同的商业元素是 IT 采购经理的责任，而使用方面是一个或多个应用或解决方案架构师的责任。每个贡献者都可以拥有一个针对他/她所能贡献的信息而定制的数据捕获向导。
*   不同的业务单位可能对同一类型的对象(如项目)有不同的数据要求。这可以根据对象的特征进一步区分，例如预算低于 5 万欧元、介于 5 万欧元和 25 万欧元之间或高于 25 万欧元的项目，每个项目需要不同的数据。

## 临时调查

由法规或高级管理请求驱动的特别调查有时需要一些信息，这些信息在 Alfabet 存储库中不是标准的。这些调查需要快速可靠地执行，这是传统的手动方法无法做到的。Alfabet 的调查工具能够针对 Alfabet 利益相关方社区快速实施数据收集活动。该设施提供了一个简单的调查程序，自动化程度高，数据质量高，并可报告特定方面的结果，如域或组织以及整个调查项目。使用经过验证的流程来管理数据捕获，以提高效率和完整性。

例如，调查可用于:

*   评估用户对架构元素的满意度，以支持组合决策
*   盘点技术使用情况，以便能够在引入新技术时预测兼容性问题
*   快速回应合规审计员的问题
*   当从较低的成熟度级别发展到较高的成熟度级别时，收集关于对象的附加数据方面

## Alfabet 数据集成框架(ADIF)

Alfabet 数据集成框架(ADIF)是一个可配置的批量更新工具，用于高性能导入、导出和处理大量数据。ADIF 是捕捉来自相邻 IT 管理工具的数据的理想选择。通过在单个数据库事务中映射、格式化和写入数据来实现高性能。ADIF 中的配置是使用 SQL、T-SQL 或 PL/SQL 完成的，它为用户提供了一个简单且完善的工具来定义数据集成。

ADIF 为 CSV、XLS、JSON 和 XML 文件提供自动结构识别，因此一个导入包可以包含不同类型的多个数据源。它提供了导入包的可视化设计和可视化调试器，以在受控的逐步执行中验证所实现的业务逻辑的正确性。导入作业作为同步或异步流程启动，可以嵌入到已建立的企业集成解决方案中。

使用 ADIF 进行数据捕获具有以下优势:

*   快速数据上传/修改——ADIF 脚本的主要部分直接在数据库上执行
*   事务性-所有更改都在一个事务中执行，并在出错时完全回滚
*   各种数据源- ADIF 可以同时处理来自不同来源的数据
*   各种数据格式——微软 Excel、CSV、JSON、XML
*   从数据库中直接读取数据——支持 SQL 和 Oracle 数据库以及 LDAP
*   开发环境提供- ADIF 包含一个完整的脚本开发环境

由于数据的完整性和一致性在 Alfabet 中至关重要，因此 Alfabet 会在导入过程中进行质量检查，以确保原始工具中的不一致和矛盾数据不会被接管。由于不遵守已建立的建模约定，原始数据源经常允许数据差异。Alfabet 通过强制执行正确的输入来对此进行补偿。例如，如果一个类具有开始和结束日期属性，则 Alfabet 要求结束日期晚于开始日期，而对于从中导入此类数据的工具，情况可能并非如此。
