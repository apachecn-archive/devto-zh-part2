# 用于原型语义 UI 的草图模板

> 原文：<https://dev.to/whoisryosuke/sketch-template-for-prototyping-semantic-ui-1h1d>

该样式指南的目标是将所有的 [**语义 UI**](http://semantic-ui.com) 组件记录为一个**草图符号库**，以简化 SUI 项目的原型制作。任务是努力遵循语义 UI 为其设计系统的语义和原子基础制定的相同标准。

在里面你会发现一个由几个不同符号组成的**组成的**组件**。不同的符号允许你从 CSS 框架的面向语义的结构中获得同样的灵活性。需要不同的背景颜色？也许你的按钮需要改变状态？只需点击你的组件，并在 Sketch 的右侧面板中进行定制。**

这个文件也能够根据你定制符号的方式为你生成一个完整的风格指南。您所有的颜色、文本样式、按钮、输入等都可以从 Styleguide 页面导出为 pdf、SVG 等格式。

## 指令

### 使用草图文件

1.  制作新的一页
2.  单击顶部栏上的插入，
3.  然后悬停在符号上
4.  找到你需要的任何组件。

建议您使用组合符号来创建组件。有些符号，如卡片甚至按钮，是由许多符号组成的，这些符号构成了最终产品。例如，条件按钮只是一个只向左和只向右的按钮，条件形状和文本在中间。像标签按钮这样的组件也是以同样的方式实现的，一个标签按钮和另一个常规按钮堆叠在下面。

### 定制样式指南

1.  开文件
2.  转到符号页面
3.  按照那里的说明进行定制

因为 Semantic 做了一些 Sketch 不喜欢的有趣的事情，所以我们有一些容器/边框是专门用于 Semantic 的。你可以在最后一步的底部找到特别的说明。

您还会发现 Sketch 不允许您从侧面板更改文本颜色或对齐等内容。这就是为什么我创建了(在某些情况下)字体颜色和粗细不同的额外按钮。
