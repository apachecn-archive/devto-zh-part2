# 糟糕的代码并不可耻

> 原文:[https://dev.to/canderson93/bad-code-isnt-shameful-2n8o](https://dev.to/canderson93/bad-code-isnt-shameful-2n8o)

实话实说吧。我们都写过糟糕的代码。

有时候，我们会写出绝对*令人震惊的*代码。

在编程社区中，我们痴迷于“好”代码——不难看出为什么。好代码客观上比坏代码好。它更快、更干净、更易维护——你为什么要满足于更少的东西呢？

**因为我们不可能总是写出好的代码**——有时候没有时间。有时会有遗留代码。有时候我们写代码的时候代码很好，但是需求变了。

有时候我们就是不知道怎么做

好的代码是一种理想——我们都应该向往的东西，但是坏的代码是生活的一部分。写糟糕的代码*并不可耻*。写糟糕的代码并不会让你成为糟糕的开发者。

唯一可耻的是根本不去尝试写好的代码。
