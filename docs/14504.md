# 锐化过时的技能

> 原文:[https://dev.to/chrishunt/sharpening-out-of-date-skills-2e 99](https://dev.to/chrishunt/sharpening-out-of-date-skills--2e99)

经历了一系列不幸之后，我发现自己失业了，这是近 20 年来的第一次。就其本身而言，这并不是一个大问题，然而当我更新我的简历时，我注意到的是——我的核心技能被严重忽视了。

### [](#what-happened)发生了什么？

为一家公司工作通常会给你有限的项目，如果没有商业理由升级/重构代码/重写测试以使用最新和最好的软件或库版本，这种学习可能在工作环境中无法获得。

外面工作怎么样？我总是利用我的个人项目来自学新的技能、技术和技巧。啊，是的——但是那些时髦的库，利基技术和第三方服务，我花了几个小时在一个虚构的项目中实现的，这个项目永远不会见天日，不是让你被雇用的东西。这是核心技能，他们需要一些抛光。

我想，如果你使用的技术和你每天在办公室里使用的一样，那么有趣的业余项目就不算有趣了。

### [](#the-knowledge-gaps)知识空白

我确定了几个真正让我担心的地方，因为它们是我简历上的标题技巧。例如:

*   MongoDB 比我在项目中使用的任何版本都要领先两个版本
*   当我的 PHP 项目在 PHP7 上运行时，我没有升级任何代码来利用 PHP7 的任何功能
*   我的大部分前端工作都是使用 Bootstrap 3 构建的。Bootstrap 4 已经经历了 Alpha、Beta 和现在的发布。
*   我的项目在 EC2 上的 Docker 环境中运行，但是“部署”仍然非常手动

### [](#time-for-action)行动时间

我读了很多关于 PHP7、MongoDB 3.6 和 Bootstrap 4 的书，对它们的可用性和新功能有了很好的了解，但我从未使用过。我知道一个事实，阅读本身并不能嵌入知识。我从阅读大量关于承诺、await/async 函数和 Javascript 生成器的文章中了解到这一点。当 a 第一次使用它们的时候，我有些不知所措。

我考虑了如何实现这种知识升级，并决定我需要组织我的行动，以便有超越仅仅为了学习而学习的价值。让这个过程帮助我找工作也是非常重要的。

我考虑了以下方法:

1.  确定需要关注和改进的领域
2.  优先考虑这些领域
3.  阅读并理解每个区域
4.  为每个领域确定一个用例
5.  行动

我不会对每一步都进行调查和发现，因为它们对我来说非常具体，但总的来说，这个过程开始变得很好，沿途有一些不错的发现。

*   我在 MongoDB 大学上找到了一门关于 MongoDB 3.6 新特性的课程([https://university.mongodb.com/courses/M036/about](https://university.mongodb.com/courses/M036/about))。
*   在我的一个个人项目中，3.6 的几个特性完全可以实现
*   我发现了 Bitbucket 管道，它允许我在`git tag`直接构建和部署 Docker 容器到我的服务器上

几周之内，我觉得我的核心技能恢复得很好，我可以自信地谈论新功能，不仅仅是阅读了它们，而是使用它们实现了解决方案。

### [](#lessons-learnt)吸取的教训

作为一名开发人员，我越来越清楚，我不能再忽视我的核心技能了。开发是一个快速发展的领域，作为一名开发人员，我有责任跟上发展的步伐。

显然，工作与生活的平衡很重要，所以我们不能每晚下班回家就花几个小时进行培训，因此我认为在工作环境中更加努力地学习很重要。这显然需要“卖给”公司。正如我在开始时提到的，如果没有业务需求或价值来升级你的内部产品/软件，它通常不会被完成。

我相信对大多数公司来说,**有**的价值，可以让开发者跟上时代的步伐。

*   开发人员通常会更乐意使用新版本的产品，并且允许他们在工作中学习。
*   该公司在招聘开发人员时可能会更有吸引力。
*   升级软件/库中的新功能可能会增加使用它的公司产品的功能集。
*   ...显然，新版本的产品通常在安全性和速度方面都有所改进。

很多这是“隐藏”的价值，这将依赖于开发商向公司出售的想法，公司要有足够的前瞻性思维来看到价值。

随着求职的继续，我会关注这类公司。
