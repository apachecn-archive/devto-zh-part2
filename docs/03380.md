我一进大学就开始了竞技编程，大概在 2014 年左右。

我很认真地继续 CP 直到 2015 年。

我的职业生涯只有一年。但是在那段时间里，我学到了很多东西，因为这是我最优先考虑的事情。

在我们继续之前，你可能会想我为什么离开 CP。

这是因为我当时对开源感兴趣，创建很酷的 GitHub 项目成了我的新爱好。

这是个好决定吗？

唯一一次和泰尔在一起。但是考虑到我的未来计划，这似乎是一个不错的决定。

因此，让我们开始这个竞争性编程指南。

## [](#step-1)第一步

学 C，C++或者 Java。如果你能学习 C，你可以学习 C++，我会强烈推荐你。

为什么？因为 C++有 STL 文件，里面有集合，贴图，矢量等等。这些在解决某些问题时非常有用。

我从来没有用过 Java，但是看到一些专业程序员( [uwi](https://codeforces.com/profile/uwi) )喜欢用它，我认为它也是一个不错的选择。

还是那句话，CP 不要用 Python，Ruby 等。这些都是非常高级的语言，不会给你足够的代码控制权。而且它们慢得令人痛苦。

## [](#step-2)第二步

上 [Hackerrank](https://www.hackerrank.com/) 。

我推荐 Hackerrank，因为它有最好的用户界面，而且对初学者来说入门相对容易。

任何没有 CP/算法经验的人都能够解决前几个问题。

你可能在前 4-5 个问题后卡住了，这是正常的。在这种情况下，请随意查看社论或谷歌寻找解决方案。

找到了，一定要理解，然后自己编码。

首先，解决所有部分的“简单”问题，然后是“中等”问题。

其实解决一些“中等”的问题，就已经好到可以称自己为 CP 专业人士了。

## [](#step-3)第三步

上车[码厨](https://codechef.com/)和[码部队](https://codeforces.com)。

Codechef long 比赛是业内最好的比赛，Codeforces 因其 2 小时比赛而闻名。

尽量两者兼顾，看看自己擅长的地方。

发展你的竞争性编程技能需要你既快又能深入思考问题。

Codechef 长挑战=深度思考和 Codeforces 回合=快速编码

如果你想变好，你应该在这两个方面都努力变好。

但是，如果你只是擅长长时间的挑战(深度思考)或者只是擅长短时间的竞赛(快速思考)，这都将有助于你成为一名更好的程序员。

## [](#step-4)第四步

如果你愿意，你可以试试 Topcoder。或者就这样吧。

有了 CP，如果你的目标是获得谷歌总部的优惠和喜欢，你就必须定期这样做。也就是说，进行竞赛，尝试排名更高，并建立您的评级。

否则，你可以只做兼职，专注于培养其他实用技能。

这些实践技能与你体面的竞争性编程经验相结合，将大大有助于你获得你梦寐以求的工作和实习机会。

## [](#alternative-route)替代路线

你可能已经有了一些 CP 经验，并且可能想学习一些算法。在这种情况下，通过 Codechef 浏览这个[算法列表。即使只练习前 20 名，也会让你成为一个高于平均水平的有竞争力的程序员。](https://discuss.codechef.com/questions/48877/data-structures-and-algorithms)

你也可以在 [GeeksForGeeks](https://www.geeksforgeeks.org/fundamentals-of-algorithms/) 找到一个不错的算法列表。它们按主题分开，你可以学习顶部的或你听说过的。
