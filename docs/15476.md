# 创造还是修复？

> 原文:[https://dev.to/alvarocavalcanti/to-create-or-to-fix-4l0n](https://dev.to/alvarocavalcanti/to-create-or-to-fix--4l0n)

当我们写下人生中的第一行代码时，第一次感到兴奋的是当我们让计算机做某事时。写下我们的名字，向世界问好，甚至打印一个计算出来的总数。这是创造的乐趣。

当然，这种感觉不会持续太久，因为第一个 bug 永远不会太远。然后来回就开始了。编写特性和修复 bug。

这是生命的循环。

沿着这条路走下去，我们最终得到了许多新的特性，以及相当多被粉碎的 bug。哪些是你最喜欢的？

如果我必须选择一个...

它会碾碎虫子！

公平地说，与项目总监交谈真的很令人兴奋。试图更好地理解给定故事的需求，与用户进行启动或桌面检查会议，并在他们的眼中看到该功能的价值。显然是为了实现这一点。

但最终，对我来说，找到一只虫子并碾碎它更有意义。通读日志，更好地理解业务、代码和调查。调查惊心动魄。不幸的是，这些虫子通常被捆绑在一个压力锅中，焦虑的人们希望它们消失，这并不令人愉快。

但即便如此，我还是喜欢调查和修复。

你们呢？
