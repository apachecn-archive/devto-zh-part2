# 关于优先考虑次要项目

> 原文:[https://dev . to/alahmadiq 8/on-prioritizing-side-projects-3kni](https://dev.to/alahmadiq8/on-prioritizing-side-projects-3kni)

在空闲时间，我想做很多事情。这些可以是任何事情，从编程项目，探索新的爱好，或只是读一本书。然而，每当我试图染指一个项目时，我经常以没有完成它而告终。我努力避免在电视、社交媒体或只是闲逛上浪费太多时间。

作为一名前端网站开发人员，我一直想建立一个我的作品组合，发布在我的个人网站上。我以为我有足够多的现有项目可以展示，但事实证明大多数都是半生不熟的废弃项目。意识到这一点让我对自己感到失望，但这也迫使我审视自己的习惯，并试图理解为什么我从不完成兼职项目。我决定更仔细地审视我工作之外的任务管理，我一直在尝试不同的策略来平衡娱乐和兼职项目。

在这篇文章中，我将重点讲述我所学到的如何优先处理项目。我认为选择和优先考虑项目是任务管理的一个重要部分。在决定做什么的时候，我会问自己以下三个问题:

# [](#1-does-it-contribute-to-my-professional-amp-personal-growth)1。它有助于我的职业&个人成长吗？

我想把更多的时间花在帮助我学习新技能的兼职项目上。例如，我想学习更多关于 web 动画、状态管理、WebSockets 等等的知识。这可以是学习一项新技能，参加一个在线课程，或者开发一个我想要使用的应用程序。

# [](#2-does-it-solve-a-specific-problem)2。它解决了一个特定的问题吗？

我想在构建单页应用程序方面得到更多的实践。当我致力于解决一个实际的、具体的问题时，我发现自己最有动力，而不仅仅是在理论上“练习”。它帮助我保持专注，并让我在前进的道路上学习新的东西。

# [](#3-how-passionate-am-i-about-it)3。我对此有多大热情？

致力于一个项目一直是我最大的问题之一。当副业没有回报时，你很难承诺去做。但是我会更有效率，投入更多的时间和精力在我热爱的事情上。因此，当我对要做的项目进行优先排序时，我喜欢给每个项目分配一个热情等级。我是基于 [Hatch 笔记本](https://www.amazon.com/Hatch-Notebook-Organizer-Sketchbook-Productivity/dp/B01M7X5KTL)受到启发才这么做的。

# [](#the-results)结果

利用上面的问题，以下是我选择在下班后做的一些事情:

*   **改造和润色我的个人网站**。我想通过在网上展示我的作品来增加我的销售能力；我对我目前的网站不满意，所以我选择用 [Gohugo](https://gohugo.io/) 从头开始重建我的网站。
*   **创建一个 Connect-4 游戏**。在我目前的工作中，我们使用 Redux 进行应用程序状态管理。Redux 可以有一个陡峭的学习曲线，所以我选择了一个随机游戏来构建。你可以在[这个链接](https://alahmadiq8.github.io/connect-4/)查看最终项目。
*   **为神童创造一个新的特性**。作为解决特定问题的一个例子，我为 Wunderlist to-do 应用程序构建了一个简单的 API 消费者，它从一个行分隔的任务列表中批量添加 todo 任务。这是我需要的一个缺失的功能。你可以在[这个链接](https://wunderlist-parser.herokuapp.com/)查看最终结果。我非常高兴从事这项工作，因为它帮助我解决了应用程序中的一个问题。

工作之余，我很容易保持专注，并把时间投入到这些项目中，因为它们要么有助于我的个人和职业发展，要么解决了一个特定的问题，要么我对它们充满热情。

我仍在试图找出如何平衡工作、副业和娱乐。看来这将永远是一个持续的过程。目前，我认为我走在正确的道路上。
