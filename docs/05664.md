### 课程的重点

总的来说，本课程着重于从解决问题的角度教我们 AWS。在这种情况下，问题在于托管、保护和交付静态网站。这种学习方法在我看来是通过做来学习。

与其花几个小时阅读 AWS 文档，不如让我们开始工作，真正开始构建一些东西。

这允许我们学习像 S3、CloudFront、API Gateway、Web 应用防火墙和 AWS Lambda 这样的服务，同时实际使用它们来解决我们的问题。在课程中，我们学习了在 AWS 上托管、保护和交付静态网站的细节。

但是，更重要的是，我们学习了一个在 AWS 生态系统中学习任何服务的框架。学习云服务的最好方法是实际使用它们。

在课程中，我们完成了学习 6 种不同 AWS 服务的目标。我们通过关注我们的问题、托管、保护和交付静态网站来做到这一点。

在这里，我们把重点放在作为代码的基础设施，以及如何把它应用到我们的静态网站用例中。如果你喜欢[我最近关于这个话题的博文](https://dev.to/kylegalbraith/the-benefits-you-need-to-know-about-infrastructure-as-code-8c4)，你将会真正喜欢这一章。

我们深入 IaC，探索如何利用 Terraform 将我们的静态网站基础设施表示为代码。

#### 如何使用 GitHub 和 AWS 持续部署静态网站

我们都听说过持续集成和持续部署，但它通常看起来很大，很可怕。但是，真的不是。

这一额外的章节涵盖了 CI/CD 的基础知识、它提供的好处，并探讨了 AWS 中的一个实现。

我们可以将我们的静态网站知识与我们的基础设施结合起来，作为代码知识来创建一个持续的集成和部署管道。然后我们可以学习新的 AWS 服务，CodePipeline 和 CodeBuild，来自动将我们的静态网站部署到 S3。

### 我听到最多的两个问题

这两个加分章节，只针对我听到最多的两个问题。

#### 如何提升我在 AWS 中的技能？

从知道什么是 AWS 到利用 AWS 是一个不小的壮举。但是，这也不是要了解现有的每一项服务。它是关于知道如何及时地应用给定的服务来解决你所面临的问题。事实上，在您尝试应用它之前，您可能甚至不知道该服务的真正用途。

因此，提升你的技能并不是把尽可能多的服务塞进你的脑子里。

云计算领域有一个新兴趋势，可以帮助你提升技能。这个趋势完全是关于自动化的。

基础设施的创建、更新甚至销毁都应该尽可能自动化。代码的部署和服务的扩展应该自动处理。

这两个额外的章节通过将我们的基础设施表示为代码以及使用 CI/CD 管道持续部署我们的代码，帮助您利用自动化的力量。

#### 接下来我可以用静态网站用例练习什么？

老实说，静态网站用例充满了你可以实现的东西，以便了解更多关于 AWS 的知识，不考虑这两个额外的章节。

在这些章节中，我们重点学习和实现持续集成和持续部署。这两个概念对任何软件开发都至关重要，并且适用于大多数用例。我们专注于在静态网站环境中学习这些，但是我们也可以将这些概念应用到动态网站中。

我们还学习了如何使用静态网站用例，并在 Terraform 模板中表示它。这允许我们练习将我们的基础设施表示为代码，在拉请求中检查我们的基础设施，并与其他人共享我们的基础设施。

在我们的静态网站用例中，可以应用和学习更多的东西。除了这些奖励章节之外，您可能还想探索一些想法:

*   通过 API Gateway 和 AWS Lambda 增加搜索功能。
*   通过 Lambda 添加聊天功能。
*   通过 AWS CloudFront 的流媒体视频。
