虽然我可以说我在 GitHub 上创建了一些公开的回购，但我不会真的称它们为“开源”,即使它们在技术上是“开源”。

[Jelly Fin](https://github.com/jelly-fin/jelly-fin) 是我第一个*真正的*开源项目。这是我要造的一个。我的自述会被润色，我的贡献者会像家人一样，应用本身也会成为其他人的榜样:代码覆盖率，可维护性，常青树...一句话，坏蛋。

不全是阳光和彩虹。是工作。到目前为止，这是我在旅途中学到的一些东西。

## 形象决定一切

我们实际上是从零开始，我们得到的第一个问题是“我能帮什么忙？”。太神奇了。加入我们聊天服务器的人数比我想象的要多。我认为这很大程度上归功于市场营销。

拥有自述文件是一回事，但不仅仅是这样。这是你的登陆页面。这是人们在 Github 上查看您的回购时首先看到的内容。对于文档来说，这是第一道防线。你必须在一份简短的文件中解释整个宇宙。

我在网上拿了个模板，改编了一下。以后会不会变化比较大？绝对是。

## 最佳实践确实有帮助

还记得你的导师告诉你的一件事是个好主意吗？动手吧。测试、构建过程、文档等。我更关心的是把事情做对，而不是做得又快又草率。我开始这个项目是为了制作一个应用程序，我无法让自己把它拼凑起来。我想让它成为一个例子。

## 兴趣犹如一团闪光的火

最初有很多人感兴趣，后来兴趣减退了。为什么？我觉得我没有准备好。我有合适的聊天工具，足够好的文档，但是对于那些想要加入并提交拉取请求的人来说，还不够肥沃。动机是什么？个人理财应用一点也不令人兴奋，但它很有用，我们将利用酷炫的技术来实现它！

## 共同的愿景大有帮助

有一点很有帮助，那就是有共同愿景的几个人。我们想这样做，因为这是一件需要做的事情，而且是正确的！我们正在解决问题，以使其他人更容易开发应用程序，但更少关注于实际的编码速度。我喜欢这个。我热衷于以设计为中心的开发。规划确实有助于一个可靠的应用程序。

## 我希望我能和你们更多地交谈

有时候我觉得我不能接触到所有对我感兴趣的人，所以也许我应该采取一种被动的方式。所以，我在尝试新的东西。在 Twitter [@sublimegeek](https://twitter.com/sublimegeek) 或 [WhatsApp](https://wa.me/14796575953) 上联系我。我想和大家见见面，看看这个项目有什么吸引你的地方？你喜欢做什么？你最喜欢什么颜色？

## 最后，耐心。

罗马不是一天建成的每一次对话都是朝着正确方向迈出的一步。每一次提交、每一次拉取请求，以及每次有人因为无法构建项目而打开一个问题。它温暖了我的心。

我觉得我可以做得更多，编写更多的代码，做所有的事情，但是我最近被一个简单的短语所束缚:

> 做你自己最困难的事情是意识到你有时需要少写代码，花时间和你的志愿者在一起。
