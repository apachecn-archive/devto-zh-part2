# 尽管如此，杰斯·珀塞尔代码

> 原文:[https://dev.to/jessiepurcey/nevertheless-jess-purcell-codes-3 oaj](https://dev.to/jessiepurcey/nevertheless-jess-purcell-codes--3oaj)

## [](#i-began-to-code-because-i-love-to-make-things)我开始写代码是因为我喜欢做东西

想想看，我在十几岁的时候就开始编码，用 HTML 制作网页。我喜欢从零开始创造东西的解决问题的方式，以及我对最终结果的控制。作为一名创客和建筑设计师，我今天仍然因为这些原因热爱编程。

## 学习编程使我能够开创自己的职业道路

学习编码使我能够自动化过程，不仅为我自己，也为我们架构公司的其他人，让我们有更多的时间专注于设计。

能够编写代码让我能够将我对技术和建筑的热情融入到我热爱的职业中。

能够编程意味着，使用像虚幻引擎这样的工具，我可以更好地控制用户如何与 VR 体验进行交互。

使用可视化编程工具，如迪纳摩 BIM，不仅使我能够为我们的公司自动化工作流程，而且足够容易接近，我可以鼓励其他用户开始自己动手。

## 我最近克服了非常严重的冒名顶替综合症

作为一个从未上过编程学校、只懂少量编程语言的人，我很难认同自己是一名女性程序员。作为一个在学校学过建筑并且大部分时间都在使用基于可视节点的编程工具编程的人，我觉得我没有权利把自己和传统的程序员相提并论。

随着我更多地使用各种工具和语言，我逐渐认识到编程不是关于你知道什么语言，而是关于如何将复杂的问题分解成非常小规模的具体步骤。我成为设计技术专家的旅程让我觉得称自己为开发人员更舒服。

## [](#my-advice-for-allies-to-support-women-and-nonbinary-folks-who-code-is)我对支持女性和非二进制代码的盟友的建议是....

做你喜欢的事情，找到你热爱的事情，并将这些事情结合在一起。技术不会发展到任何地方，如果你能找到一种方法，利用编程为你热爱的其他事情增加价值，那么你将永远有新的学习和成长的机会。
