## [](#functions)功能

从函数开始。在这篇文章中，我想把函数分成三个不同的类别。

1.  正常功能
2.  箭头功能
3.  有界函数

### [](#normal-functions)正常功能

我把普通函数定义为用...

```

function magic() {
    ...
}

// function expression
const magic = function() {
    ...
};

// (or if you hate your life)
// function constructor
const magic = new Function('...'); 
```

## [](#ways-to-call-a-function)调用函数的方式

现在假设我们有一个函数`f`(任何类别)。有两种方法可以调用它。

1.  隐式(直接)调用
2.  显式调用

### [](#implicit-direct-calls)隐式(直接)调用

隐式(直接)调用很无聊:

```
/* f is defined */

// direct call
f();

// or attach it to an object and call it
const obj = {};
obj.fn = f;
obj.fn(); 
```

### [](#explicit-call)显式调用

显式调用更有趣。你可以用`Function.prototype.call`或者`Function.prototype.apply`调用你的函数。

```
/* f is defined */

// Function.prototype.call
f.call(...);

// Function.prototype.apply
f.apply(...); 
```

## [](#quick-recap)快速回顾

让我们快速回顾一下，我们有 3 类功能:

1.  普通函数-使用函数声明/表达式/构造函数创建
2.  箭头功能- `() => {...}`
3.  绑定函数-用`f.bind(...)`创建

调用函数有两种方法:

1.  隐式(直接)调用- `f()`或`obj.f()`
2.  显式调用- `f.call(...)`或`f.apply(...)`

这意味着我们有 6 个不同的场景。

1.  普通函数+隐式(直接)调用
2.  普通函数+显式调用
3.  箭头函数+隐式(直接)调用
4.  箭头函数+显式调用
5.  绑定函数+隐式(直接)调用
6.  绑定函数+显式调用

[![](../Images/0c6963d98719f22f728373554f0376d7.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--rxUSTYMN--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://media1.giphy.com/media/lKZEeXJGhU1d6/giphy.gif)

事实上，箭头函数和绑定函数并不关心隐式/显式调用。所以这就简化为 4 种情况:

1.  普通函数+隐式(直接)调用
2.  普通函数+显式调用
3.  箭头功能
4.  有界函数

## [](#exercises)演习！

给定`magic`定义如下:

```
'use strict';

const magic = function() {
    // a. what is `this`?
    console.log(this);

    const cool = () => {
        // b. what is `this`?
        console.log(this);
    };
    cool();
};

// QUESTION 1
magic();

// QUESTION 2
const apple = { name: 'apple' };
apple.magic = magic;
apple.magic();

// QUESTION 3
const orange = { name: 'orange' };
magic.call(orange); 
```

### [](#question-1a)问题 1.a

按照流程图，我们想在`magic`中找到`this`。

1.  `magic`的类别是普通功能
2.  `magic`被隐式(直接)调用
3.  `magic`用`magic()`调用
4.  所以`this` = `undefined`！！！

### [](#question-1b)问题 1.b

按照流程图，我们想在`cool`中找到`this`。

1.  `cool`的类别是箭头功能
2.  从问题 1.b 中，我们知道`magic`的`this`是`undefined`
3.  `cool`的定义者是`magic`
4.  所以`this` = `magic`的`this` = `undefined`！

## [](#note)注

*   没有“绑定箭头功能”。`(() => {...}).bind(...)`还是原来的箭头功能。
*   对于隐式调用，只有形状(`f()`或`obj.f()`)有关系。`f`来自哪里并不重要。考虑以下代码:

```
const f = obj.f; // obj.f is a normal function
f(); // `this` in the body of `f` is `undefined`!!! not `obj`!!! 
```
