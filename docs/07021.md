## 从来没有更好过，也永远不会变好

我从 2004 年就进入了软件行业，感觉就像一直在喝消防水管里的水。

在这段时间里，有些事情变得更好了。

我们已经从手动设置服务器发展到在云中自动运行整个*集群*。

JavaScript 已经从被认为是玩具语言变成了世界上最广泛使用的语言。

我们已经从手工编码和样式化我们网站上的一切发展到几十个 [CSS 框架](https://www.webdesigndegreecenter.org/ten-best-css-libraries-developers-designers/)和[组件库](https://blog.bitsrc.io/11-react-component-libraries-you-should-know-178eb1dd6aa4)，拥有数百个功能强大、漂亮的预打包模式和组件。

我们已经从没有网格系统，到[单一笨重的 960 像素网格系统](https://960.gs/)，到[不可思议的强大的可定制网格系统](https://css-tricks.com/snippets/css/complete-guide-grid/)嵌入浏览器。

我们从来没有比今天过得更好。

然而。

我们已经从能够简单地加载文件变成了一系列复杂的前端构建工具。

我们已经从没有好的 JavaScript 库，到拥有 [1 个主要工具(jQuery)](https://royal.pingdom.com/2010/03/26/jquery-triumphant-march-to-success/) ，到拥有 [3 个大型前端框架和几十个小框架](https://javascriptreport.com/the-ultimate-guide-to-javascript-frameworks/)。

变化的步伐从未如此之快，而且还在不断加快。

它永远不会变好。我们必须不断学习。

## 三个 mo:决定学什么的框架

我认为只要我还在这个行业，我就需要不断学习。

即便如此，要弄清楚该学些什么还是很困难的。为了帮助我做出决定，我使用了一个简单的三部分框架，我称之为“三个 mo”。动机、动力和金钱。

1.  动机:我对学习这个有多兴奋？
2.  动力:学习这些对我有什么好处？
3.  金钱:商业/金融机会是什么？

### 动机

得知这个我有多兴奋？

当我在学校时，我的成绩与我的兴趣水平的相关性远远大于课程的难度。

我不仅会在学习我感兴趣的东西时获得更多乐趣，而且会学得更好更快。

### 气势

学习这个自我对做或学习其他事情有什么帮助？

学习这个看起来是一项在未来很长时间都有用的技能吗？

例如，我可能会学习基于 JavaScript 的服务器技术而不是非 JavaScript 技术，因为我知道提高我的 JS 技能也会给我在前端带来更多动力。

### 金钱

有人会付钱让我学这个吗？(最好是当前的客户或雇主)。如果不是，这种技能的市场需求是什么？

我的理想情况是字面上的学习获得报酬。对我来说，这通常发生在我能够为邻近地区的客户或雇主提供价值的时候，学习这项技能将帮助我提供更多的价值。

或者当我带着一个已经很有价值的技能水平进来时，我可以用这个技能在一个项目中工作并获得报酬，并在这个项目上真正地完善和提高。

## 将三个 mo 应用于 JavaScript 框架

对于新的 web 开发人员来说，一个常见的问题是“我应该学习哪个 JavaScript 框架？”

我将局限于 React、Vue 和 Angular 的“三大”框架，但如果“三个 mo”中的一个或多个指向另一个，请不要犹豫，将其纳入您的考虑范围。

对于**动机**，只有你能回答。Vue 是现在的“抢手货”，在 Github 上刚刚超过 React，但 React 有很多很棒的东西和更成熟的生态系统，Angular 有很多企业采用。

从**势头**来说，大概是一洗了之。深入学习这些框架中的任何一个，实际上都将为您学习其他任何框架做好准备，因为许多核心概念是相同的。

一旦你深刻理解了基于组件的架构、组件生命周期和 Flux 风格的状态管理，你就可以将这些知识应用到许多框架中。

关于**金钱**，除非你已经有一个对 Vue 或 Angular 感兴趣的特定客户、雇主或领导，否则我认为当前的优势会起作用。

React 的使用率仍然很高，而且 React 上似乎有更多的工作机会&。也就是说，Vue 正在快速追赶，那里有大量的机会。

如果你仍然停滞不前，最后一个决胜局是，Vue 似乎更容易让人们学习，所以如果你选择 Vue，你可能会更快地变得富有成效。

## 如何学习:基于项目的学习

你的第一步是找一个项目。

这是学习一项新技术的最佳方法:找到一个你有动力去完成的项目，并把这项技术应用到它上面。

理想情况下，这是一个你拿工资去做的项目，但也可以是一个私人项目，只要你非常有动力去完成它。

有一个项目会迫使你去学习更全面的东西，而不仅仅是跟随一门课程或教程。

您将被迫处理教程通常不会处理的边缘案例。

你将不得不调整解决方案，而没有“正确的做法”的退路。

基于项目的学习无可替代。

### 从一门课程开始，然后转到谷歌和文档

一旦你确定了一个项目，我通常会从学习我所学技术的概述课程开始。

这对我来说是一个 kickstarter 它让我度过了“我不知道我在做什么”的难关，能够做一些基本的事情。

我很少完整地完成一门课程，但我通常会浏览一大部分来获得一个大概的理解。

一旦我有了这个概览，我就可以简单地做我的项目了。每当我遇到不知道怎么做的事情时，我会使用谷歌和我正在学习的技术的文档来找出解决方法。

这种方法对我来说似乎比纯粹参加课程或纯粹坚持文档和文章更有效。

课程让我更快地了解全局，同时通过阅读文档和文章来解决具体问题帮助我建立更深的知识库。

这也让我用更多的基础知识来增加我的理解。React 课程不太可能在 JavaScript 基础知识上花费太多时间，但是当我用谷歌搜索来解决我遇到的问题时，我通常会深入研究语言基础知识。

## 学习更有策略

虽然我们不知道未来的网络发展会是什么样子，但我相信变化的步伐不会放慢。

我们将不得不不断学习新的东西，但如果我们试图学习所有的东西，我们将永远无法跟上。

相反，我们需要改变我们的学习方法，使之更具战略性。

我们需要优化我们的学习，以保持动力，利用我们自己的动机和能量，并获得学习的报酬。

换句话说，我们需要由类似“三个 Mos”框架的东西来指导。

这完全正确吗？我表示怀疑。它对我很有效，但我打赌它会变得更好。
