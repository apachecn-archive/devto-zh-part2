# 然而，莎拉·奇马编码了

> 原文：<https://dev.to/sarah_chima/nevertheless-sarah-chima-coded--1923>

去年的某个时候，有人告诉我，他认为我不会在编码行业呆太久。“为什么？”我问，他回答:“因为我是女性”...嗯，我现在有工作了，猜猜我是做什么的...我知道你会猜对，lol，我还在编码。

## 我开始编码是因为...

大学毕业后，我想让自己忙起来。早些时候，我参加了一个涉及 C 语言编码的课程，我在业余时间学习了 python 和 C#，但我从来没有真正感觉到我知道如何编码。不过，在一年零八个月多的时间里，我和编码有着很好的关系。以下是我的简短故事。

我从阅读一本关于 HTML/CSS 的书开始，直到我的一个朋友告诉我关于 Freecodecamp，我立即加入了。Freecodecamp 让我再次爱上了编程，我真的很喜欢在那里学习，因为我可以实践我学到的东西。我对编码上瘾了，我发现自己熬夜只是为了解决算法或完成项目。

几个月后，我参加了我现在工作的公司的远程实习，这完全改变了一切。我要在现实生活的项目中实践我学到的东西。这次实习大大提高了我的编码技能。我遇到了一群优秀的软件开发人员，他们耐心地帮助我学习，并一直激励我直到今天。我还学会了如何在项目中与其他人合作，以及如何管理项目。

今天，我有一份前端开发人员的工作，我很高兴我决定在放学后真正学习编码。

## 我很兴奋...

半铸钢ˌ钢性铸铁(Cast Semi-Steel)...:)好吧，我爱 JavaScript，也爱 React。

## 我最近克服了...

冒名顶替综合症。它是真实的，有时它给我们很大的打击。在某种程度上，我觉得自己并不像表面上看起来知道的那么多，而且与和我一起工作的其他聪明人相比，我觉得自己知道的很少。谢天谢地，我和一个朋友谈了这件事，她真的帮了我。她提醒我，我们都在进步。我可能还不知道所有的事情，但只要我坚持不懈地学习和努力工作，总有一天我会知道的。

## 我想吹嘘一下...

我的作品。去年，我接受了一个挑战，坚持写关于编码的文章。虽然我没有遇到挑战，但我在短时间内写了很多文章。让我高兴的不是我写了多少文章，而是人们从这些文章中学到了什么。每当有人在推特上谈论我或我的文章时，我的心真的很温暖。最近，我没有写我想写的那么多，因为我的时间表改变了，我自己的时间更少了，但我会很快恢复定期写作。

## 我对其他女性和非二进制程序员的建议是...

1.  **练习，练习，练习。**擅长编码没有捷径。你必须坚持不懈地练习。在这个过程中，要对自己有耐心。当你第一次尝试学习一个概念时，你可能不会抓住它，但是随着时间的推移，它可能会变得非常容易。也要记住，除非你用它建造了什么东西，否则你什么都不知道。

2.  参加科技活动，向他人学习，结识他人。这是我在编码之旅的早期为自己做的最好的事情之一。我参加了很多聚会，并结识了很多编程朋友，当我遇到挑战时，可以向他们求助。当你需要找一份新工作或你的第一份工作时，这些朋友也可以帮助你。记住，独自踏上编码之旅是很困难的。拥有同样编码的朋友会鼓励你继续努力。如果你还没有这样做，加入 Twitter，关注那些鼓舞人心的人，你也可以向他们学习。

3.  把自己推出去。告诉别人你做过的项目，写文章，让每个人都知道你在做什么。即使你认为你知道的很少，也要分享你知道的一点点。不要觉得人们会看不起你。如果他们这样做，那是他们的问题，与你无关。永远保持自信。

4.  不要放弃。有时候，编码是非常具有挑战性的，你可能想知道为什么你会把自己放在第一位。与其放弃，不如在编码中找点刺激的事情做。有时候，我会玩 CSS 或者解决代码战的算法来保持理智。寻找对你有用的东西，然后去做。

坚强点，继续努力，你会没事的。:)
