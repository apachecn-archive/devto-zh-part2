# 我最近的前端面试经历

> 原文:[https://dev . to/Arden nl/my-recent-front-end-interview-experiences-4b2p](https://dev.to/ardennl/my-recent-front-end-interview-experiences--4b2p)

我非常高兴地告诉你，我最近在里斯本接受了一份工作，并从上周开始和一群很棒的人一起工作。我已经看到了很多关于工作面试的帖子，如何准备面试以及会有什么期待，所以我想在我记忆犹新的时候分享一下我的面试经历。

在八周的时间里，我发出了八份职位空缺的申请，标题分别是*(高级)前端开发人员*或*(高级)JavaScript 工程师*。其中五次导致了 Skype/电话或面对面的面试，从所有这些中，我得到了后续的编码测试和/或第二次面试。最后，我得到了一个拒绝，两个“也许很快”和两个提议。尽管我已经有十多年没有参加过传统的工作面试了，但我发现这种经历非常令人愉快。

## 背景

六个多月前，我在 web development 中写了我的“成年”以及当时我对如何继续下去的疑虑。TL；博士:*“我最近搬家了，除了再次做自由职业者，一切都很令人兴奋。生活是美好的，但也许我应该寻找挑战"*

你可能会认为六个月的时间足以让我做出一个平衡的决定，决定我的职业发展方向，但是我没有。我继续从事自由职业，直到我看到一个有趣的职位空缺，心想“有决定总比没有决定好！”。第一次申请后，我更加确定我想再次在团队中工作。对我来说，了解里斯本的科技领域并在葡萄牙扎根是件好事。

## 制剂

我一只手就能数清我一生中参加过的所有工作面试，而且都是十多年前的事情了！我一直是一名自由开发人员，直到我工作的第一家机构给了我一份我无法拒绝的工作。

所以你可以看到我对这个面试游戏是多么的陌生。从 20 岁出头开始，我就一直不理会招聘人员，也不需要写简历。是时候聪明起来了。

### CV

在我做任何事情之前，我必须做一份简历，我不知道那些看起来像什么。事实证明，一切都没有改变，雇主们仍然喜欢一份可以快速浏览的可打印文件。

现在，我不得不考虑我可以在简历上写些什么，因为我过去十年的经验基本上可以归结为八年的自由职业和两年的代理工作。此外，我没有计算机科学的相关学位或背景，这是许多职位空缺中提到的。我不知道在葡萄牙学位有多重要，但是我不能用我所拥有的去做其他任何事情！或者从积极的角度来看:保持我的简历结构良好、易于浏览是很容易的😅。最后我采用了以下结构:

*   介绍
*   (技术)技能/语言列表
*   工作经历(自由职业者和代理)
*   最后两项研究
*   三个相关的附带项目

这就构成了一份一页半的简历，这已经是最好的了。

### 研究

我申请的一些公司在网上有很多信息。我留意的是公司博客、Linkedin 个人资料、Github 账户和他们在网上分享的其他资源。对于更大的公司，我在 glassdoor 上找到了大量的信息，这对于了解面试过程中会发生什么很有帮助。

我还发现，现在做白板面试、[codibility](https://codility.com/)测试和其他任务也是一件事。这是我从未做过的事，看起来很害怕。我认为《厄拉多塞之筛》是一部印第安纳·琼斯的电影，如果这是我在采访中必须解释的事情，那我就完了。

我申请的公司类型可以分为三种:数字机构、本地和外国初创公司以及在里斯本设有技术中心的跨国公司。有大量的职位空缺，但由于我没有很大的紧迫性，我只申请了我真正认为合适的职位。

### 求职信

我写的第一封求职信，完全是为了我遇到的第一个空缺而写的。当它迈出第一步时，我也为我的后续应用程序重用了这个结构，但是确保为每个空缺重写它。在我的求职信中，我首先介绍了为什么我对他们的空缺感到兴奋，然后列出了关于我自己和我最近正在做的事情的相关要点，最后总结了为什么我认为这是对我的正确挑战。

对于每一封求职信，我都做了一些基本的工作，比如试图找到一个人来写求职信，并提及我感兴趣的职位空缺的具体信息。

## 面试

我总共面试了五家公司，其中两家面对面，三家通过 Skype /电话。在每种情况下，谈话都非常愉快，尽管在某些时候我明显感到紧张。在很多情况下，他们问我为什么搬到里斯本，为什么我想放弃自由职业，重新为一个雇主工作，并回顾了我过去的经历。我一直很坦率地说我没有计算机科学学位，但是没有人真正问过我。

一些公司对我做了一些研究，提到了我的 Codepen 笔、Github 项目和博客帖子。我得到了不止一个关于有条理的积极评价，这是我努力的方向，因为我有点混乱。

因为我对我得到的每一次面试都心存感激，所以我确保重视每个人的时间，并对其他正在运行的应用程序保持诚实。

我真的没有为工资谈判做好准备。我没有真正研究过葡萄牙的市场。虽然我知道这里的工资比荷兰低，但我不知道到底低了多少。在大多数情况下，我试图将这些谈判推迟到以后，在有些情况下，我确实会给出一个数字，这是一种标记方式。

在所有情况下，我从第一次面试中得到了积极的反馈，并被要求做一个小的编码挑战、任务或第二次面试。

### 挑战

编码挑战对我来说也是全新的。我要做的第一件事是一个 Codility 测试，但我马上就搞砸了。我担心这将是一些超级数学计算机科学的东西，但事实上，这些测试真的很实用，主要是关于数据和 DOM 操作。尽管如此，我还是搞砸了，因为我没有做好充分的准备，忘记了时间。

在另一个案例中，我意识到我能够完成构建 Vue 应用程序的任务，但是这将花费我相当长的时间。由于他们需要开发人员尽快加入进来，我觉得自己不够“资深”，没有相当长的入职时间就无法开始这项工作。我决定不做这项任务，并坦率地向公司解释了原因，这似乎得到了很大的赞赏。

在另外两个案例中，我得到了一个 React/Redux 赋值和一个 JavaScript 挑战，两者都没有非常严格的时间限制，我能够做得相当好。

### 反馈

一家公司(简历:真实性测试)很难拒绝，两家公司认为我可能不太适合我申请的职位，但主动提出与我保持联系。两家公司给了我一份工作，我选择了与我第一次联系最密切的那家。

## 总结性的

在一个新的地方申请工作大大超出了我的舒适区，但事实证明这真的很让人放心。我从这次经历中学到了很多，也更加自信了，因为我知道这里肯定有我的机会。现在，我对自己做出的选择感到非常兴奋，并希望与我的新团队一起创造、学习和分享大量很酷的东西！
