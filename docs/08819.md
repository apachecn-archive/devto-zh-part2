# 代码狂怒

> 原文:[https://dev.to/codemouse92/code-rage-22hk](https://dev.to/codemouse92/code-rage-22hk)

最近还有人消极地避开 StackOverflow 吗？我说的最近是指“自从几周/几个月/几年前你因为不正确的存在而被火烤之后”？

从我开始认真编码开始，我就是 StackOverflow 的一员，到目前为止，我回答的问题是我提问的两倍。在每年一月和九月的“家庭作业问题泛滥”期间，我帮助检查和编辑队列，并协助打扫卫生。我甚至认为它是开发者必须加入的社区。我当然不想给人留下我在抨击 StackOverflow 的印象！

但老实说，我一直在回避这个地方。

# [](#problem-words-or-problem-people)问题话，还是问题人？

现在，StackOverflow 真的应该受到称赞。他们已经认识到他们社区中的一种不健康趋势，并决定对此采取行动。

但是，当他们试图让 StackOverflow 成为一个更受欢迎的地方时，那些对创造有毒环境负有责任的愤怒群众(像往常一样)在评论区表达了他们对脚下变化的不满。

[![Bowie Disapproves](../Images/f26e1cafcb4958509c76639b5535b1a0.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--_BIeqEls--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/5dwux36a7nqhxc1see4a.gif)

如果你曾经在 StackOverflow 上，或者实际上在许多编码器社区上提出过问题，你无疑会有过这样的经历:提出问题或提出建议，然后因为你的“大胆”而被剥夺了生命的精华。关于这个话题，我已经写了很多次，也说过很多次，甚至创造了一些幽默的伪拉丁名字来描述这些交流方式，比如*Programmaticus new bieframerous*和*Programmaticus Militaricus*。

但是这种尖酸刻薄的核心是什么呢？这里有一个“宝石”的评论，我认为很好地强调了这一点...

> 这个网站很受欢迎，也很有帮助，只要你努力学习如何使用它。感谢社区通过消除垃圾用户使它成为如此伟大的地方。

...等等，垃圾用户？

[![Garbage?](../Images/fc9ccddadb468a79ad7f564fd951339f.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--DNcou3YB--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/c9sds9dodo7sc6nz127y.gif)

对我来说，这个评论非常有效地展示了弥漫在我们今天的编码文化中的有害气氛背后的驱动力:**我们习惯于通过一个人的行为来评估他的价值。**

现在，这并不是说我们应该忽视有问题的行为。如果是这样的话，我写这篇文章就没有意义了！我很乐意第一个纠正措辞不当的问题或答案，纠正技术上的错误信息，甚至投票结束清楚地表明这个人不愿意自己做工作的问题。但我(希望我)总是带着同情心去做。**即使我的言语或行为看起来很有力，我也不会在这个过程中评价一个人的价值。**

在`dev.to()`的评论部分可以很容易地观察到这种健康的动态。我们内在地、含蓄地将彼此视为有智慧的人，能够进行合理的对话和逻辑辩论。作为一个团队，我们认识到我们都处于不同的学习和成长阶段，并致力于在进一步成熟的过程中相互支持。

一个健康的纠正并不总是令人愉快的，但它永远不会阻止一个人在公众面前认真尝试和认真失败，提出问题(即使是“显而易见的”问题)，并参与明智的讨论。错误和失败是我们成长的方式。

这似乎是显而易见的，那么*这么多人*怎么会错过呢？

# [](#short-fuses-and-sunday-drivers)短保险丝和周日驱动器

在阅读关于 StackOverflow 讨论的评论时，我意识到这一切让我想起了什么:**路怒症！**

可以肯定地说，我们都经历过。你正在开车，一些“傻瓜”表现出他明显看不懂信号。相反，他正以低于限速 10 英里的速度缓慢前行。然后，更糟糕的是，他做了一个意想不到的转变，却懒得通知你。

我们该怎么做？

[![It's called a blinker!](../Images/4ec282db33780fbbae5e813deebeb2a7.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--IR7SvZXq--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/6jhvefp6xlyn3rzxrb88.jpg)

他的驾驶技术很糟糕吗？哦，当然。他可能会接受一些重要的驾驶再教育。他可能是一个真正的混蛋！...也可能不是。也许他今天不在状态。也许他因为连续工作 14 个小时而累了。也许他的妻子在医院，他分心了。也许他真的，认真的，只是没有意识到自己在做什么！

问题是，我们很快就对这个人的内在价值进行了评估，然后我们觉得必须告诉他他的估计市场价值。我们回忆过去，向他伸出一根手指敬礼，也许还会从敞开的窗户向外说几句教育的话。或者也许我们只是在夕阳下开车的时候，用汽车喇叭给他唱首小夜曲。

当我们停下来想一想，我们完成了什么？我们当然提高了他和我们的血压几个点，但是他学到了什么吗？大概不会。如果他今天过得不好，我们只会让他更糟十倍。如果他真的只差几根薯条就能在车里享受一顿快乐餐，他不会因为我们热情的提议而突然醒来闻到咖啡的味道。

同样，这并不意味着我们不做*某事*。我们有汽车喇叭是有原因的——有时是为了警告危险，有时是为了在睡美人发短信穿过绿灯时叫醒她。我们可能需要闪烁警示灯来提醒尾随者，如果我们突然停下来，他们会被强行推到我们的后座。**我们采取适当的行动进行警告和训诫。**

注意目标的不同:后者用于评估*行为*，通常建议纠正措施；前者用来评估*个人的价值*，没有正面的目的。然而，由于我们倾向于愤怒，我们倾向于前者。

现在，愤怒可以是一件健康的事情，激励我们采取积极的行动去纠正错误的事情。毕竟，是我对我提到的那些充满仇恨的评论的愤怒让我写下了这篇文章。但它也很容易，如果不是更容易的话，变成一件极具破坏性的事情。

作家兼演说家奇普·英格拉姆(Chip Ingram，《克服破坏性情绪》)指出，愤怒是一种次要情绪，由以下三种原因之一引起:

*   伤害(未满足的真实或感知需求)
*   挫折(未满足的真实或感知的期望)
*   不安全感(对个人价值的真实或感知的攻击)

[![Anger exploding](../Images/967387582570456a006081fe759808ca.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--XPycfzzJ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/0vd74frcfg64vbxnr4gq.gif)

有鉴于此，我认为有两个主要原因导致了路怒症的发生:

1)认识到他人不遵守交通规则对自己和他人造成的危险。(未满足的实际需求和期望。)

2)对给我们带来的不便感到愤怒:我们可能会迟到，不得不等得比我们喜欢的时间更长，或者我们只是厌倦了排队等候。(未满足的感知需求、期望，甚至可能是感知到的对*我们*价值的攻击。)

你注意到了吗？路怒症会产生反馈循环！我们生气，攻击别人的价值，让他们生气(或更生气)，他们会回敬或转移到别人身上。

一个有趣的补充说明是，一项研究发现汽车保险杠贴纸的数量与司机的路怒倾向之间存在关联。简而言之，定制汽车是一种地域行为。你最近看过你的笔记本电脑背面吗？

# [](#code-rage)代号狂怒

我想我们大多数人也遇到过这种情况。您正在 StackOverflow 或任何其他编码社区中使用工具，您会发现如下内容...

> 我正在尝试编写 python 代码，从网站上获取内容，并将其放入数据库中。我如何从页面上阅读文本？谢谢

很有可能，你会立即退缩。真的吗？这个用 Python 写*又一个 web scraper* 的朋克小子是谁？更糟糕的是，他甚至不想写。他想让你做这项工作！每个人都能看出这里出了什么问题:

发帖人几乎没有花力气去写一篇可读的帖子。他的拼写和语法很糟糕，从他的代码中看不出有什么更好的地方！你肯定不知道，因为他没有显示任何代码。除此之外，他显然没有阅读过多的关于这个主题的文档和教程，这两者都可以通过对“python web scraper”的基本网络搜索来发现。相反，他在问这个令人痛苦的宽泛的问题。你和其他任何一个不太有经验的 Python 开发者都可以在网页抓取器上写*本书*。

对此我们的反应是什么？通常是以下一种或多种情况。

*   令人满意的坚定点击向下投票按钮，然后继续前进。不管怎样，他们注定要失败。

*   一个傲慢的评论，大概是“真的，伙计？LMGTFY！哦，看，有一些文件！也许你应该读一读，试着写自己的代码！”哈，你确实给他看了。

*   如果你对这个问题感兴趣，转向你的社交媒体，喋喋不休地谈论一些愚蠢的人提出的愚蠢的问题。(标记有问题的白痴的加分。)

[![Ya Burnt!](../Images/27d4fc64703d161155d54058e41dc740.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--KB3ItG1R--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ouyv7v7ld3sz8gm3hn58.gif)

完成了告知白痴们他们的愚蠢的任务后，当你遇到其他事情时，你又回到了浏览社区。

> Python 为这类事情提供了一个`pickle`模块，而且非常简单。您可以将数据序列化到一个文件中，稍后再加载它。

哦不，不要再这样了！对于任何问题来说，`pickle`模块都是效率最低的解决方案，这里有另一个推荐它的人。你现在做什么？

*   只是否决他，然后继续前进。你轻声感谢发明了那个奇妙的匿名倾倒按钮的天才。

*   让他在评论里有吧！“哇。你曾经写过产品代码吗？总有比泡菜更好吃的。它是文本数据，把它放在 JSON 文件中，然后继续前进。假设你有文化，你有时可以试着阅读。”(你已经准备好了一些其他聪明机智的抨击，以防他不会立即屈服于你的高智商。)

*   标记它！毕竟，这种技术上的愚蠢应该从互联网上清除。

*   社交媒体再次嘲弄时间。毕竟，这就是我们有 Twitter 的原因。

当我们结束时，我们除了伤害感情和大声宣扬我们的优越感之外，什么也没做。我们滥用了四种武器...

## [](#weapon-1-the-anonymous-downvote)武器 1:匿名反对票

人们激烈地争取用这个工具肇事逃逸的权利，但是停下来想一想。

反对票意味着什么？“我们不喜欢这个内容。”问题是，对于一个还不了解这个社区所有动态的人来说，这通常是令人困惑的，仅仅是因为**人们无法读懂你的想法**。

[![Two Thumbs Down](../Images/9874615dd7b804fc0cf4c4810354b433.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--ASyU9irP--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/nszs7gip7ri5skl0sl6k.gif)

问题是否显示不努力？答案是否技术上不正确？措辞难懂吗？这是重复的吗(老实说，大多数平台都有其他工具来处理这个问题！)仅仅是链接吗？是不是觉得问题太明显了？如果你是诚实的，你会不会干脆不喜欢这个人？

每当你投反对票时，社会责任要求你绝对必须拥有你的推理。那张反对票*对某人来说意味着*什么。这令人沮丧，甚至令人沮丧，但它本身并没有教育意义！写一个简短的，尊重的，建设性的评论来教育对方。

虽然有些人只是不想花时间，但我认为许多反对留下评论的人这样做是因为他们知道他们的理由无法得到尊重的解释。除了发泄他们的愤怒或仇恨情绪，他们没有因为任何其他原因投票。这纯粹是一记耳光，但用语言承认这一点会暴露自己是个混蛋。你不能生活在假装道貌岸然的错觉中，承认你投票否决了某人，因为他们对英语的掌握让你把他们想象成某种多毛的灵长类动物。

现在，我同意一个例外:如果其他人已经清楚地、建设性地陈述了这个问题，你的匿名反对票没有坏处。它只是构成你对评估的认同。

## [](#weapon-2-toxic-words)武器二:毒字

我真的需要在这里多说吗？用作家布伦南·曼宁的话说...

> 在每一次遭遇中，我们要么赋予生命，要么耗尽生命；没有中立的交换。

我们可以用语言来教育、告知和激励他人，也可以用语言来斥责、贬低和打击他人。当我们选择词语时，我们必须考虑到其他人，选择对他们有帮助的词语。

[![Breathing Fire](../Images/9e555edf380b3750e9806aef05762492.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--MmODhPH9--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/m1jzmv8i5eu2g1hrryd6.gif)

不要误解我，这不是某种“滴血的心”的情绪，主张踮着脚绕过别人过度敏感的情绪。告诫有时需要直言不讳。新信息可能不会让人“感觉”良好。纠正可能需要力量。但是我们的目的不同:我们可能会暂时伤害某人的感情，以便给他们带来更大的好处，但是我们不会伤害他人的感情。

## [](#weapon-3-inciting-others-to-harm)武器三:煽动他人伤害

这是我在这里列出的最严重的罪行之一。我们进行攻击已经够糟了，但是当我们煽动其他人加入攻击时，我们也要对他们的行为负部分责任！

这，说白了就是*欺负*。我们正在召集我们的盟友一起作为一个团伙痛击目标。这可能会造成一些最严重的情感和精神伤害。我们也许可以忽略一个人，甚至两个人，但是如果有一大群人说我们是“垃圾”，我们会觉得他们可能是对的。反过来，这激发了自我仇恨，以及由此带来的所有可怕的事情。

这是否决是否合适的一个因素。如果有人已经受到口头批评，一连串的否决将(合理地)被认为是对骚扰的认可。这同样适用于向上投票的攻击性回应。这是你应该谨慎使用投票工具的另一个原因；如果你不花时间去评估情况，你很容易增加对团队的骚扰。

## [](#weapon-4-abusing-moderator-tools)武器四:滥用版主工具

最糟糕的情况是，3 号武器升级到煽动*版主*伤害个人。有很多*很多*合适的情况下，我们应该标记帖子，报告内容，并让审核人员参与进来。然而，除了骚扰、欺凌和/或违反社区规则之外，这样做对*任何事情*都可能造成最深的伤害。这是因为版主代表了整个社区。

[![Thor's Hammer](../Images/095dcbdcfd0ca4ee20893cd5d6102aae.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--mT0Fu9RC--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/br32649vmicylpk4ei82.gif)

换句话说，如果有人因为一个错误而受到攻击和骚扰，版主进来删除受害者的帖子或对他们采取其他行动，这就使虐待合法化。它对受害者说，“我们作为一个社区同意你的攻击者的评估，你是垃圾。”

这也应该让版主和任何在社区中有权威的人停下来。你的话比任何人的话都重要。滥用这些武器就是代表你巡逻的整个社区。

尤其是在 StackOverflow 上，想象一下如果一个版主在评论中辱骂一个新手，他会有什么样的感受。“这个由成千上万的程序员组成的社区认可了对我智力的攻击。我为什么还要尝试？”

# [](#operator-error-please-reinsert-your-empathy)操作员错误:请重新插入您的同理心

我之前描述的那些公路暴怒的动力会和我们的“代码暴怒”相似吗？我相信是这样的，尽管味道略有不同...

*   我们会因为别人没有遵守社区的规则而生气，即使这些规则是没有明文规定的，因此也是学来的。(未满足的期望。)

*   我们对别人支持与我们相反的观点感到愤怒。(感知对个人价值的攻击。)冒名顶替综合征在这其中作用巨大！

*   我们会因为别人没有做我们认为是“他们该做的”事情而生气。(未满足的期望。)这驱动了“RTFM”语句。

*   我们会因为别人没有给我们前进所需的信息而生气。(未满足的需求。)这可能是作者马特·吉梅尔[后来后悔发起](https://mattgemmell.com/hindsight/)的[“你尝试了什么】](https://mattgemmell.com/what-have-you-tried/)运动的核心。

*   我们对别人进入我们舒适的空间，做我们不喜欢的事情感到愤怒。(感知到对个人价值的攻击，未满足的期望。)这又与保险杠贴纸(推而广之，笔记本贴纸)相关的领地行为有关。

所有这些陈述有什么共同点？他们都是关于我们 T2 的！我们的需求、我们的愿望、我们的期望、我们的个人价值。现在，拥有并寻求满足我们的需求和(一些)欲望本身并没有什么错。有适当的时机对破裂的社会契约(未满足的期望)感到愤怒。当我们的价值受到威胁时，我们完全有理由愤怒。

问题是，我们把一切都看得太重要了，我们忘记了我们在帮助满足他人需求方面的作用，我们自己履行社会契约的义务，以及我们自己不攻击他人价值的责任。

简而言之，我们已经变得如此短视，无法想象坐在别人的方向盘后面，或者别人的电脑键盘后面是什么样子。

> 同理心:理解、意识到、敏感地感受和体验他人过去或现在的感受、思想和经历的行为，而没有以客观明确的方式充分交流这些感受、思想和经历

这不正是我们所遗漏的吗？前面提到的那些“垃圾用户”并不是真正的垃圾。许多人在不熟悉的领域里迷失了方向，不知所措，勇敢地朝着他们所知道的最好的方向前进。他们向我们寻求帮助——也许不是以建设性的方式，当他们要求完成的代码而不是指导，或者没有给出我们需要的关键信息时——但是我们真的需要为他们的要求加分！他们正在迈出第一步，这总是最可怕的:**他们公开承认自己没有能力解决问题**。如果有什么是对自我的打击，那就是。

其他敢于尝试回答问题或提供信息的人，再次冒着暴露自己于公众批评的风险。在这个社交媒体的时代，我们知道有无数人愿意因为我们的信仰不同而把我们分开。认真纠正错误的知识不需要一个棒球砸在头上！相反，这需要尊重、理性的论证，同时理解我们没有责任说服*任何人*相信*的任何事情*！毕竟，我们都有犯错的时候，但我们很少会怀疑。迟早，你会在辩论中站在错误的一方。

是的，有时候人们会太固执而不听。也许他们有非常好的理由，或者也许他们宁愿接受疯狂也不愿犯错，但这是一个我们需要证明自己的情况(特别是为了观察者的利益)并让它过去的情况。

在所有这一切中，我们必须**不要**允许自己陷入对他人的评价。我们评价行动，而不是人。我们关注的是行为，而不是身份。

我们都是人，这意味着我们都会生气，不管我们选择承认与否。因此，代码狂怒是我们都必须与之战斗的事情，有些人比其他人更需要战斗。

最后，答案很简单:讲生活！
