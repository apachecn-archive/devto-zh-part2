# 为建筑而战

> 原文:[https://dev.to/yellowbrickcode/fight-for-the-architecture-5k 3](https://dev.to/yellowbrickcode/fight-for-the-architecture--5k3)

我最近在读鲍勃叔叔的《清洁建筑》。我还在这本书的早期，但我刚刚读了一部分，这让我需要放下它，写这篇文章！

## 开发者也是利益相关者

在名为“为架构而战”的章节中，Bob 谈到了作为一名开发人员和/或架构师，你一直在为你认为对公司有利的事情而战。正如管理团队、营销团队等也是如此。这一部分最让我印象深刻的是:

> [有效的软件开发团队]毫不掩饰地与所有其他利益相关者平等地争吵。记住，作为一名软件开发人员，*你是一名利益相关者*。您与需要保护的软件有利害关系。这是你角色的一部分，也是你职责的一部分。这也是你被聘用的重要原因。

这个，这么多这个！我认为这是你在职业生涯早期没有意识到的事情，至少我知道我没有。我还认为，如果公司没有认识到他们的开发团队是一个利益相关者，这是一个产品最终会失败的稳定斜坡。如果没有花时间在架构上，你最终会得到一个越来越难添加和维护的产品。您必须向团队中添加更多的人，但是您的产出不会增加，因为代码库每天都在变得更难工作。每一行代码的编写成本都会逐渐增加，但是你的产出将不再与你的投资相匹配。最终它将需要被重写或抛弃。

因此，对于阅读本文的开发人员来说，请记住，软件开发团队绝对是他们正在开发的产品的利益相关者。有时候，为了产品的利益，我们必须坚持某些东西，并帮助其他利益相关者理解它的重要性。我们处于这些位置是因为我们了解技术含义，也因为我们擅长我们所做的事情。如果你袖手旁观，听之任之，你将会在一堆代码中结束你的工作，这些代码绝对不是有趣的，并且因为所有错误的原因而具有挑战性！
