# 所以，去做吧。

> 原文:[https://dev.to/michael/so-do-it-3a2k](https://dev.to/michael/so-do-it-3a2k)

前几天红灯的时候，我看了看旁边的车，车牌是:

[![A license plate that says, "sodoit"](../Images/bdc3aa4dbd96d638612dc8ec65a6fecf.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--GIXqGLbP--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/fp45lby68c7s56lc3ibh.jpg)

“那么，做吧”

起初我觉得它很幽默，但仔细想想，我发现它很有启发性。这让我想起了那些我为自己不能在那一刻做自己想做的事情找借口的时候。就好像这个简短的短语给了我许可去做我克制自己不去做的事情。

> 我真的希望这个应用程序存在，但我不知道后端开发...所以，去做吧。
> 
> 我没有时间写信...所以，去做吧。
> 
> 我应该拓宽我的关系网...去做吧。

好吧，你可能会读到这里并说，“所以，我明白了，我会做的”。但是不知道怎么做。这里有一些对我有用的建议。

## 80/20 规则

80/20 法则说 80%的结果来自 20%的输入。或者换句话说，专注于仅仅知道足够多，而不是成为专家。

回到希望一个应用程序存在，但我不知道后端开发。我不需要去学习像 Ruby on Rails 这样的 web 框架的所有细节，我可以一头扎进去，学习如何建立和运行一个网站的基础知识。挑选我需要为应用程序了解的东西，而不是专注于学习关于框架的一切。

## 把事物分解成最小的、不可重复的单位

把你想要达到的目标分解成小的、可行的和不可靠的步骤。

小，因为我认为我们经常看不到我们的想法有多大，并被实现它的困难所吓倒。通过分解它，它变得不那么可怕了。

可行，因为你想采取行动，而不是让你的想法只是无所事事。如果你想看到结果，你必须付出努力。

经久不衰，因为它是关于建立势头。当你停下来的时候，需要时间从你上次停下来的地方重新开始。但是如果你一直向前，最终你会更快地看到你的目标实现。

## 利用你所得到的

我坚信[评估你所拥有的](https://michaelsoolee.com/use-what-you-have/)和[充分利用](https://michaelsoolee.com/best-tech-stack/)来实现你的目标。

所以假设你想做一个 app。你能去掉一些功能，只用 HTML、CSS 和 JavaScript 的基础来制作这个应用吗？

或者关于人际关系，也许你是一个内向的人，但是知道人际关系的价值。你可以利用大量的在线资源，比如在线论坛、Twitter、Slack 频道，甚至你可以完全在线参加的会议。或者设定一个目标，每隔几周只和一个人见面。他们甚至不必在你的行业网络中。他们网络中的其他人完全有可能认同你正在做的事情和你的兴趣，他们可以建立联系。

## 所以，做吧！

你有目标，我也有。有些事情会阻碍我们实现这些目标。很多时候，是我们自己在拖我们的后腿。所以，通过这篇文章和一个简单的短语，“所以，去做吧”，让我们给自己实现这些目标的许可。
