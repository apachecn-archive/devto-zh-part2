Lotus Notes 自 1989 年上市以来，一直作为许多组织的 IT 生态系统的一部分存在至今。然而，在最近几年，有一个值得注意的发展，从 Lotus Notes 到现代，远远超过 IBM 的成熟产品。

## 它从迁移到 MICROSOFT EXCHANGE 开始

组织从 Lotus Notes 迁移的最常见原因是更换电子邮件基础设施，这通常达不到用户的期望。在大多数情况下，组织决定过渡到 Microsoft Exchange，因为它是企业用户的市场先锋。

电子邮件迁移往往只是整个组织向现代 IT 软件基础架构迁移的第一步。在 Lotus Notes 上保持业务应用程序的需求激励公司进一步投入微软的堆栈，最终执行 [SharePoint 开发](https://aufait.in/sharepoint)。无论是在内部还是在云中，SharePoint intranet 都提供了许多有价值的现成功能。迁移者最感兴趣的是 SharePoint 列表、库、页面等等，它们只需相对较少的努力就能支持大多数 Lotus Notes 应用程序。第三方工具也是可用的，极大地改善了迁移。

## 最大的挑战:将业务应用程序迁移到 SHAREPOINT

电子邮件服务和基本功能的迁移通常不是迁移的结束。长期以来，Lotus Notes 被大量用于为一些公司交付基于工作流的应用程序。为了避免出现两个相互竞争的 IT 生态系统(Lotus Notes 和 SharePoint)的情况，公司决定将基于工作流的应用程序也迁移到 SharePoint。

与简单的电子邮件迁移不同，没有工具能够将复杂的应用程序自动迁移到 SharePoint 环境中。值得记住的是，许多组织开发了现代化的复杂工作流解决方案，使得潜在的迁移更加困难。在这种情况下，主要的解决方案是在新环境中从头开始重新创建它们。

为了避免。NET 编码来扩展 SharePoint 的初始功能，通常会选择第三方解决方案，从而提供广泛的工作流和表单功能。这允许用户以低代码的方式重新构建他们的业务应用程序，与定制编码相比，这本质上更快更容易。
