## 我是这方面的专家...

我可以对许多话题略知皮毛，但我几乎不是任何方面的专家。尽管人们称赞我的英语写作技巧！当我陷入困境时，我也是一个打扰别人的专家，但这真的是一种技能吗？

## 我目前正在入侵...

我不像大多数人那样看待“黑客”。人们说“我整个周末都在为我的树莓派编码”。Arduinos？覆盆子尿？那些是什么，冰淇淋口味的？

不，我更喜欢简单的东西，但这仍然让我保持动力。我维护着一个名为[节点认知服务](https://github.com/joshbalfour/node-cognitive-services)的开源项目，它不过是一个 API 的包装器。实际上，这是一个有趣的故事，我是如何成为它的维护者的。你可以在我的博客中阅读详细的[。但简而言之，我从提交 PR，到成为投稿人，到看到库被微软使用:)](https://miparnisariblog.wordpress.com/2017/10/31/contributing-to-open-source/)

我还维护着一个更小的库,它是在我曾经开发的一个移动应用程序出现大规模生产错误时出现的。

## 我想吹嘘一下...

我的堆栈溢出声誉！8600，还在继续。)——其实，[我在我的祖国——阿根廷排名前 50！](http://data.stackexchange.com/stackoverflow/query/53058/top-users-by-country)。达到这个数字其实并不难...你给出几个好的答案，人们会不时地投票。有趣的事实是，我的前三个答案都包含图纸。我猜人们*确实*喜欢视觉胜过文字！

既然我被允许在这里吹牛，我可以说我也为自己感到非常自豪。我曾为一家失败的创业公司工作，失业，被许多知名公司拒绝(其中一家甚至在第一次面试后就被拒绝了)...).但后来，在一阵好运之后(这绝对不是因为我的面试技巧)，我得到了一份在亚马逊做 SDE 的工作，并搬到了温哥华，从我到这里开始，我就一直住在温哥华...我该怎么说呢？实现梦想！

## 我开始/继续编码是因为...

我喜欢那种“啊哈”的感觉，当你把头往墙上撞了一个小时，终于发现你的代码有什么问题时，你会有这种感觉。我也喜欢人们欣赏你的工作，回馈社区的感觉棒极了。

## 我仰望...

就工作而言，我钦佩斯科特·汉瑟曼，因为他是一个很棒的演讲者，我喜欢他的博客，他看起来是一个很好的人。我也钦佩我以前和现在的同事，因为我每天都从他们身上学习。

在音乐方面，我钦佩仙妮娅·唐恩。我喜欢她的音乐，她很美，她的故事既悲伤又鼓舞人心。我等不及要看她两个月后的生活了！

除此之外，我钦佩任何敢于冒险的人。

## 我对女性和非二进制编码人员的建议是...

*   如果公司拒绝你，不要气馁。这只是一个信号，表明你需要继续学习。
*   贡献开源。即使你的拉取请求只是修复一个错别字，它也很重要。
*   享受生活！下班后不用写代码。所以不要觉得你“不得不”做兼职项目。编码是一项耗费脑力的活动，如果你做得太多，你会开始讨厌它。周末出去走走，看看书，骑骑自行车。
