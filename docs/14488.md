# “大 O 符号”到底是什么意思？

> 原文:[https://dev . to/jvanbruegge/what-does-big-o-notation-mean-anyway-1 hea](https://dev.to/jvanbruegge/what-does-big-o-notation-mean-anyway--1hea)

如果你是一名程序员，很可能你已经无意中发现了“大 O 符号”这个术语。但这实际上意味着什么呢？

Big-O 符号用于指定程序或数据结构的计算复杂度。这基本上意味着:我需要多少步骤来执行一个特定的动作。这里的“步骤”不是指 CPU 周期或程序中的行，而是具有固定长度**的任意步骤，不管它们有多大。**

让我们从一个例子开始:检查一个元素是否在数组中。在 JavaScript 中，您可以这样实现:

```
function findInArray(array, element) {
    for(let i = 0; i < array.length; i++) {
        if(array[i] === element) {
            return true;
        }
    }
    return false;
} 
```

您可以很容易地看到这一点:循环内部的所有事情都需要一个固定的时间:通过索引和比较来访问数组中的元素。我们把这个定义为 1 个“步骤”。但是因为有了`for`循环，我们做了`n`次。在大 O: `O(n)`。

用 Big-O 符号描述的其他标准例子是排序算法。以最简单的排序算法为例，冒泡排序。它的实现可能是这样的:

```
function bubblesort(array) {
    for(let i = 0; i < array.length; i++) {
        for(let j = 0; j < array.length - 1; j++) {
            if(array[j] > array[j + 1]) {
                let tmp = array[j];
                array[j] = array[j + 1];
                array[j + 1] = tmp;
            }
        }
    }
} 
```

对于**数组中的每个**元素，它会遍历整个**数组，并在需要时进行交换。和以前一样，循环中的一切都是不变的。然后我们有一个循环做这个`n`次。但是我们另外还有第二个循环，它执行第一个循环`n`次。所以我们执行`n`步骤`n`次。用 Big-O 表示法:`O(n*n) = O(n²)`**

但也有其他一些情况，答案并不那么直截了当。以这段代码片段为例:

```
function mapAndFilter(x) {
    return x
        .map(n => n * 2)
        .filter(n => n > 0);
} 
```

这个函数的复杂度是多少？是吗

*   `O(1)`？
*   `O(n)`？
*   `O(n²)`？
*   不一样的东西？

正确答案是`O(n)`。为什么？虽然我们的代码本身是常量(没有直接循环)，但执行其他函数也需要一些时间。在`map`的情况下，浏览器将遍历所有元素并调用它们的函数(所以这将是`O(n)` ), `filter`与我们的`findInArray`基本相同，所以`O(n)`也是如此。我们的函数应该有`O(n+n) = O(2n)`，但是 Big-O 符号并不关心常数因子，所以`2`被删除，导致`O(n)`。

我认为这是一个很好的例子，你不仅要检查自己的代码，还要检查通过库或浏览器导入的代码。每一个在数据结构上运行的算法都有一个计算复杂度。如果您知道这一点，您可以编写尽可能简单的代码。您还可以问自己:“我需要在我的数据结构上做什么操作，有没有另一种复杂度更低的数据结构？”。
