# 什么时候去 `and when to<breathe></breathe>`

> 原文：<https://dev.to/drewclem/when-to-code-and-when-to-breathe-3kfi>

这是我一年前写的一篇旧文章。我只是把一些东西从中号移过来。

[![](../Images/7d8f009073f5f69866487efde5b1858d.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--95DISF1D--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1024/1%2AKw2TcCGoU1-7Ex8RbyjB0A.jpeg)

在过去的几个月里，我一直尽我所能努力工作，成为一名前端开发人员。我下班回家，不去健身房，和妻子匆匆吃了晚饭，然后坐在电脑前学习每一门课程，看录像等等。…我能拿到的东西。在最初的一两个星期里，我取得了一些进步，但是后来我吸收和保留的信息量急剧下降，我对学习的注意力也急剧下降，我讨厌这样。我想学习编码，我想建立网站和应用程序，我想创造数字建筑设计，像埃菲尔铁塔和西斯廷教堂一样成为技术杰作(我对自己有很高的期望，嗯？).

因此，我开始认真审视我一直试图保持和养成的时间表和习惯，并意识到它们是多么的不健康和难以实现。我的大脑需要休息。我需要休息。更重要的是，为了我的幸福、学习能力和我的婚姻，我需要让这些休息成为我日常生活的一部分。我做了几件事来帮忙:

**意识到这不是一场比赛，休息一下也没关系**

我一直(现在仍然)纠结的一件事是，我觉得自己没有把每一分钟都花在学习编码上是在做错事。重要的是，你要退一步，意识到抽出时间是可以的，实际上对学习过程是有益的。

**安排不看代码的日子**

很容易告诉自己，你将完成“一件事”，然后你将离开电脑，在你意识到该睡觉了之前，你已经在电脑前呆了一整天。每周安排几天时间，让你不去想代码，只是享受这一天。去散步。去书店。和你的另一半一起看几集你最喜欢的节目。如果你安排时间从学习中休息一下，你就不太可能因为没有专心读书(屏幕)而感到内疚，因为休息是学习过程的一部分。

**提醒自己进步就是进步，不管有多小**

我感到内疚的一件事是，我对自己太苛刻了，还没有达到 x 点，这是非常不健康的行为。没有人会坐下来一次就弄明白 javascript。没有人在他们的第一次网站建设中就搞清楚了所有关于前端开发的事情，这没关系。这并不是说你每次触摸键盘都要大步前进。你在这里或那里赢得的小战斗累积起来，重要的是你要教会自己认识到这些，并为胜利而自豪。随着时间的推移，足够多的这些小胜利会积累成一吨的知识，当你看到所有这些小胜利时，你会对自己作为开发人员的成长建立信心，并且当你从学习中休息时，再次感到不那么内疚。

如果你做任何事情的时间太长，强度太大，你很容易就会精疲力尽，注意力不集中，或者只剩下一点点力气(精神上或身体上的),以至于你的努力并没有给这项事业带来任何真正的好处。20 磅并不算太重，但即使是最强壮的人也只能举这么长时间，否则他会累坏的。

解决这个问题很容易。无论做什么，都要给自己时间呼吸和放松。以旺盛的毅力朝着一个目标努力没有错，但关键是要确保你以健康的方式管理这种活力，促进持续的进步，并为你的大脑提供充足的休息和娱乐。

学习编码。记得呼吸。
