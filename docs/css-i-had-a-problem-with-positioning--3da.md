我们来看看 CSS 理论中的定位是怎么定义的。CSS 中有五种不同的元素定位(静态、相对、固定、绝对、粘性)。但是我有一个相对和绝对定位的问题，我们将关注这两个。

## 理论上的定位

每个 HTML 元素的默认位置是静态的位置。

> 它总是由页面的正常流向定位。

来源:https://www.w3schools.com/css/css_positioning.asp

什么是正常流量？

> 在正常流程中，块框一个接一个地放置在页面上(按照它们在 HTML 中的书写顺序)。它们从包含框的左上角开始，从上到下堆叠。每个框之间的距离由上边距和下边距相互折叠来定义。

来源:https://www.thoughtco.com/normal-flow-definition-3467023

> 在**相对**定位时，元件相对于其正常位置定位。
> 
> 在**绝对**位置，定位元素相对于最近的定位祖先进行定位。

来源:https://www.w3schools.com/css/css_positioning.asp

如果你检查 Bootstrap 列，你可以看到它的位置被设置为相对的，不像我的*。父元素*。[现在，让我们定义相对于*的位置。父元素*元素](https://codepen.io/machy44/pen/JpqbYZ)

具有绝对位置的元素正在寻找位置不是静态的第一个祖先。什么？我对此一无所知。父级没有相对位置，这是第一个行为的原因。啊哈！！这才是重点。

**绝对位置在层次结构中一直向上寻找第一个相对祖先(在本例中),如果它不存在，它将获得默认相对定位的文档正文。**

如果您尝试调整浏览器窗口的大小，加载微调器将始终位于父元素的中间。
