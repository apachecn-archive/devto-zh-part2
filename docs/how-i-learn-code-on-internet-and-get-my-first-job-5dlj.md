# 我如何在网上学习代码并得到我的第一份工作？

> 原文：<https://dev.to/tranbathanhtung/how-i-learn-code-on-internet-and-get-my-first-job-5dlj>

一年前，我开始了大学的第一个学期，开始在线学习 HTML、CSS 和 javascript 四个月前，我得到了第一份工作，成为一家初创公司的前端 web 开发人员……我想分享我的自学经历……我所知道的一切。

## 我们该怎么办？

当我们开始学习一些东西时，它真的很难，让我们感到气馁。我认为这是最困难的时候，但这很正常。我在某处读到过一句名言:

> 当你想放弃的时候，想想你为什么开始！

这听起来不错，但至少对我没有帮助。相信我，如果你的动力不是太大，那它就帮不了你。

相反，发展自己的方法是习惯。培养习惯、纪律和习惯来引导你。

[![](../Images/f160aa62f4f5454c51119ac4dec4bc04.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--TJLbhXNu--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/221i209miihevzfjoak3.jpg)

我们不需要在一天中花太多的时间来学习…可能需要一天 2-3 个小时，当这一切成为一种习惯，你可以用一种有趣和愉快的方式做“困难”的事情。

新手的一个普遍问题是同时学习太多东西。例如，当你正在学习 python，但是过了一段时间 javascript 变得越来越热…你想切换到 javascript，这样你就可以知道很多语言，但事实上，你将无法做到完美…所以只专注于一种语言…

此外，学习该语言的基础知识、结构、语法……我现在看到有许多趋势追随者，如 React、Vue 或 Nodejs……他们甚至只需一天就可以浏览 javascript，并开始学习这些框架。你应该记住框架是基于那种语言的…也许它只持续几年，但是那种语言将仍然存在。如果你是我说的那个人，那就别闹了，学点基础知识吧。

除了按照课程的教程或者只读文档，给自己做点自己喜欢的东西。例如，如果你了解网络，你喜欢动物或阅读书籍…你可以用学到的知识来建立与它们相关的网站。这将帮助你不那么无聊，并帮助你快速发展技能
这一部分还有一件事是用谷歌搜索…我很确定你脑子里的每个问题都已经在谷歌或 StackOverflow 上了…尽管它真的是一个有用的工具。任何事情都有两面性…当你陷入困境，5 分钟后，你去谷歌搜索，当然一切正常…这将成为习惯，但这是一个坏习惯…它会减少我们的思考…所以不要过度使用谷歌或 StackOverflow

关于库的使用，我想说的最后一点是框架自带的…我们真的需要它们吗？…当然，我们需要…但是什么时候呢？

当我开始学习 react 时…我在 react 上学习的所有课程都是关于 redux 的…这很糟糕，因为这真的很难，我刚刚知道一些关于状态和道具的基本概念…这让我认为 redux 总是 react 的一部分…我总是使用 redux，即使项目很小，我不知道如何用纯 react 创建一个网站。我想很多开始学 react 的人都遇到过这种情况……在我的国家，纯 react 和 redux 用户之间有一场小战役。我意识到为什么我不得不整天使用 redux，尽管我也不需要它… React 16.03 改进了上下文 API，使管理状态更容易，我通过创建状态管理库了解了这一点。

[https://github.com/tranbathanhtung/re-bat](https://github.com/tranbathanhtung/re-bat)

> Redux 并不坏，问题是我们教错了使用它们的方法

我还构建了一个小的库 UI，因为其他库太大了，有时我不需要太多组件。甚至创建你的想法的一个组件也比学习和改变那个库的组件要快，我决定为自己创建一些组件。

[https://github.com/tranbathanhtung/re-jok](https://github.com/tranbathanhtung/re-jok)

学习或创建自己的库很好…

> 如果你有时间，学习它是如何工作的，而不是使用它。

那么，什么时候我们需要使用一个库或框架呢？

> 记住，只有当你真的有麻烦时才使用图书馆。
