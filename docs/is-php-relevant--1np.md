# PHP 相关吗？

> 原文：<https://dev.to/mporam/is-php-relevant--1np>

学术界、申请者甚至其他开发者问我的最常见的问题之一是，“你为什么教 PHP？”或者，有时候甚至是，PHP 不是一门死语言吗？'

让我从一开始就非常清楚这一点。 *PHP 没有死。*

截至 2017 年 12 月， [PHP 占互联网上使用的服务器端语言的 83%](https://w3techs.com/technologies/overview/programming_language/all) 。其中很大一部分是由基于 PHP 的内容管理系统组成的，比如 WordPress，但是即使你把预建的内容管理系统从等式中去掉， [PHP 仍然占据了超过 54%的网络份额](https://w3techs.com/technologies/overview/programming_language/all)。事实上，如果你看看下面的图表，你会发现 PHP 的市场份额在 2017 年一直保持稳定，甚至有所增加。

早在 9 月份，我写了一篇关于“行业相关”技能和技术的博客。在写这篇博客的时候，PHP 和 JavaScript 是就业市场上最受欢迎的语言。情况依然如此。并非巧合的是，它们是我们在学院花时间最多的两种语言。

让我们看看其他语言的一些用法统计，从传授给新程序员的最流行的语言开始。大学倾向于关注 Java 和/或 C 语言，因此许多新的创业公司选择用这些语言来构建他们的应用程序。这意味着 [Java 现在约占 web 服务器端语言的 2.5%](https://w3techs.com/technologies/overview/programming_language/all) 。看不到 C 语言，但 ASP.NET——有时由大学教授的网络框架——占 14.2%。诸如 Python、Ruby 和服务器端 JavaScript 之类的语言是编程训练营最常见的焦点，然而这些语言总共只覆盖了互联网的 1.2%。服务器端 JavaScript (Node.js)是目前[发展最快的服务器端技术](http://www.zdnet.com/article/javascript-explodes-on-the-server-side-with-the-growth-of-node-js/)，尽管仍然相对较小，但它肯定是一个值得准备的技术(我们目前在学院课程中专门花了一周时间学习 Node.js)。

Techrepublic 最近发表了一篇关于 2018 年要学习的[顶级语言的文章，PHP 和 JavaScript 都出现在前六名，PHP 在 2017 年从第 9 名升至第 6 名。](https://www.techrepublic.com/article/7-programming-languages-that-every-developer-should-learn-in-2018/)

那么为什么有那么多人声称 PHP 是一门不相关或者说已经死亡的语言呢？

据我所知，有两个原因。首先，有时这是一种从开发人员传到开发人员的遗留观点。PHP 在早期的时候，是一种相对较慢的语言，有很多不一致的地方，很少有方向。然而，这种语言已经发展了很多年，自从 PHP 5.3 在 2009 年发布以来，大部分这些老问题都得到了解决。最新版本(7.1)是一种非常快速、精简的语言，具有很强的面向对象特性。

其次，PHP 是一种非常灵活、松散类型的语言。这使得很容易拿起笔开始写，但也很容易写不好。你可以说它是自身成功的受害者。但是，如果编写得当，遵循 DRY、SOLID 和 MVC(我们在学院课程中教授的所有概念)等方法，它是一种非常强大、多样和快速的语言，可以提供很多东西。

所以不，PHP 没有死。虽然像任何语言一样，它也有其缺陷，但统计数据确实说明了一切。
