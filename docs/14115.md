这是我的“开发聊天”系列中的一篇文章，在这篇文章中，我大约每周都会与一位出色的开发人员或技术人员交谈。你可以在这里阅读更多。如果你觉得这些对你有用，请在评论中告诉我！

### 自我介绍！你是谁？你在哪里工作？

👋你好。我是莫妮卡·丁库莱斯库。我是一名谷歌表情符号工程师(意思是:用表情符号进行工程设计的人)，我从事聚合物方面的工作。我在会议上谈了很多关于 web 组件的问题，有时会对互联网上的 web 字体加载感到焦躁不安。

### 是谁或什么让你开始编程的？

事实上，我最近一直在思考这个问题。老实说，最简短的回答是，我只是陷入其中，再也找不到其他事情可做。我来自一个程序员家庭；我的父母都是程序员。他们的大多数朋友都是程序员。我在罗马尼亚长大，那里的教育不是开玩笑的，所以在五年级时，我去了一所专门从事编程的学校。我有一小段时间假装自己将成为一名架构师，但是做了这么长时间，编程感觉更容易了。所以我想，从来没有一个世界是我不能成为一名程序员的。

### 我们都爱表情符号，但你对它们的迷恋是什么？

一堆年前，Chrome 还不能渲染表情符号。我当时在做这个，所以我想我可以试着修复它们(我没有)。关于 Chrome，唯一的钓鱼方法就是煮沸海洋，所以为了弄清楚为什么他们不渲染，我必须学习很多关于表情符号和 Unicode 的知识。一旦我有了这些丰富的信息，它们就有点有趣了——它们被创造出来是因为有人想非常容易地交流非常普通的事情，比如天气，现在它们已经变成了一本任何人都可以使用的超过一千种字形的巨大字典。挺可爱的！

### 你的开发风格在开源代码上是否有所改变，比如 Polymer 和封闭源代码？如果有，如何实现？

我倾向于编码，害怕有一天有人会看我的代码，认为我是一个绝对的白痴，这在开源和闭源之间并没有真正改变(尽管在开源中看你代码的人更多)。当我改用 Polymer 时，我注意到我的编码风格发生了变化，那就是我在编码时也倾向于考虑未来的 Monica。我致力于 Polymer Elements，这是一套大约 100+的小部件；你最终会频繁地切换项目和环境，并且很难记住所有的信息。因此，我留下了更多的长篇评论，并在加入团队后记录了更多内容。

### 和小一点的公司相比，在谷歌这样的大公司工作是什么感觉？你觉得有很多不同吗？

我只真正为大公司工作过，所以我不知道！我认为，谷歌有更多的规则，这不一定是坏事。即使是大公司的开源也有大公司的法律含义，所以你会对许可之类的东西更有意识。

### 你认为你在各种场合的公开演讲对你的职业生涯有帮助吗？又如何/如何不？

绝对的！我认为这让我在社区中有了更多的可见性，并且永远感激那些不断邀请我在他们的会议上发言的人！我遇到了这么多可爱的人，其中一些现在是我的朋友，他们不断教我一些很棒的东西。比如如何制作推特机器人和更好的艺术！

### 迄今为止，在你的软件生涯中，你学到的最难的一课是什么？

“你的代码不是你是谁”，但我想我仍在努力学习它。在开源领域工作有时会让人心碎，因为你总是会遇到一些刻薄的人，他们会对你和你的代码说些刻薄的话，让所有人都看到。这是我们文化的一部分，这很糟糕，但在试图改变它的同时，不要把这些评论放在心上，或者把它们视为个人的失败，这也很重要。

### 对于成功的软件职业生涯，你的第一条建议是什么？

天啊，我最近开始开玩笑说这是“带着白人的自信做事”(明目张胆地从一条推特上偷来的)。这很粗鲁吗？那有点粗鲁。不过这并没有错。总之，严肃地说，我经常被问到这个问题，所以我做了一个关于这个问题的头脑风暴[这里](https://gist.github.com/notwaldorf/4df04b5c9023aec6267bc7e66f284106)。

### 你会推荐什么书/资源？

我从来没有真正喜欢过阅读关于编程的书籍，因为它看起来太像家庭作业了。我喜欢以目标为导向的学习，所以当我被困在一个问题上时，我最终阅读的大多数东西都是教程或博客帖子。所以相反，我要脱离脚本(喘气！)并推荐一本非编程书籍。我最近正在读大卫·塞达里斯的《有一天我会说漂亮话》，这本书非常有趣。
