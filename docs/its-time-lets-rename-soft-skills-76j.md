# 是时候了。我们把“软技能”改名吧！

> 原文：<https://dev.to/samjarman/its-time-lets-rename-soft-skills-76j>

[![Not soft skills, core skills](../Images/58334bb6431cf9a48b85f7c4efc8154e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--lgdiWCih--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/32944y90imo67rivrqix.png)

你好，

众所周知，我们的“软技能”非常重要，这些技能包括但不限于:口头沟通、书面沟通、影响技能、同理心、个人技能、推销、倾听、传递/接受反馈等等。

然而，我和许多其他人对术语*软技能*(与“硬”技能相比，如编写代码或了解特定工作的工具或技术)的问题使得这个术语听起来*次要*或*不那么重要。* **这个内涵再深不过了。**

所以，**我认为我们应该将软技能和硬技能重命名为*核心技能*和*技术技能*** (对于 IT 领域，其他领域可能会有除“技术”之外的其他术语)。需要澄清的是，这不是我发明的，我听说过，我想重申这一点。

核心就是，嗯，就是这样...核心！它是你坚实的内在部分。你建造的那部分。在做任何其他事情之前，这一点必须是稳固的。你努力的那一点，要保持坚强。可能会失去力量、变得疼痛、需要不时小心呵护的部位。别人天生就比别人强的那一点。天哪，我喜欢这个术语。

原来如此。核心技能。这并不是说我讨厌“软技能”这个术语，只是这个名称并不是问题的关键——我希望你能在这些技能上努力，不管你想如何命名它们。它们太重要了。但是，让我们努力给它们起一个有意义的名字，希望其他人也同意——这是我们都将从中受益的事情。
