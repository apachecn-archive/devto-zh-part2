# 来自 EmberCamp & JSCamp 2018 的摘要

> 原文:[https://dev.to/ijlee2/recaps-from-embercamp-js camp-2018-49j 4](https://dev.to/ijlee2/recaps-from-embercamp--jscamp-2018-49j4)

周末，我去了芝加哥，向不同的开发者学习技术和领导技能。我将重点介绍 6 场精彩的演讲。(特别感谢[崔克·格洛瓦契基](https://twitter.com/trek)出色的主持人！)

来自 [EmberCamp](http://embercamp.com/) 的所有演讲都将发布在网上(*自我提醒:在此提供链接*)，所以我希望你能查看一下。

### [](#a-peoples-history-of-ember)一部人民的历史余烬

#### [](#by-lily-beauvilliers)作者[百合](https://twitter.com/ukulilybee)

我很喜欢 Lily 的演讲，因为我能够与她作为 Ember 新开发人员的经历联系起来。自 2011 年以来，Ember 已经变得越来越好。不幸的是，Lily 工作的代码库混合了旧 Ember、新 Ember 和“你”Ember(她团队自己的解决方案)。因此，新成员很难知道他们看到的是哪个余烬。

Lily 的解决方案既实用又鼓舞人心。如果你是新成员，看看 ember 的历史，围绕一个困难的概念创造一个故事，并主动提出问题。如果你有经验，记录技术债和自定义代码，作为团队每个人的参考。

### [](#interactive-data-visualization-can-be-easy-in-ember)交互数据可视化可以在 Ember 中轻松实现

#### [](#by-spencer-price)由[斯潘塞价格](https://twitter.com/spencer516)

诚然，在听斯潘塞的演讲之前，我感到有点苦涩，因为我也提交了一份 Ember-D3 演讲，但没有被接受。:)

现在我很高兴，因为斯潘塞的演讲让我意识到我可以使用多个小组件来重构我的 D3 代码。我必须考虑它对我目前的方法的利弊(一个大的组成部分)。斯潘塞的演讲还提醒我，在创建可视化时，我应该解决可访问性问题。

### [](#monster-of-the-week-how-to-be-brave-when-facing-the-horror-stories-of-web-development)本周怪物:面对 Web 开发的恐怖故事如何勇敢面对

#### [](#by-jen-weber)作者[詹·韦伯](https://twitter.com/jwwweber)

Jen 的闭幕主题演讲让我很惊讶，因为它让我感觉到了人性。Jen 分享了她的恐惧和挣扎，并解决了我们作为新的或有经验的开发人员可能面临的问题——这是我们在 dev.to 上经常看到的话题。

珍给她的怪物起的名字和设计真是太棒了。彩虹草稿纸提醒我们，我们把怪物画成彩色的线条和形状，并在中间写下 1 个待办事项，这提醒我们，我们的怪物并不像最初看起来那么可怕。

### [](#data-visualization-for-js-developers)面向 JS 开发者的数据可视化

#### [](#by-shirley-wu)由[雪莉吴](https://twitter.com/sxywu)

Shirley 在 D3 中展示了她的一些专业和个人项目，并在 React 中做了一个现场代码演示，她坚定地为[新发布会](https://www.chicagojs.org/)定下了基调。这是一个让开发者分享他们所创造并引以为豪的东西的地方。

雪莉去分析她最喜欢的音乐剧《汉密尔顿》，并在一次次失败后讲述一个有意义的故事的程度，激发了我想回到一个我尚未完成的数据可视化项目。

### [](#newbies-guide-to-contributing-to-babel)新手投稿巴别塔指南

#### [](#by-josh-justice)由[乔希大法官](https://twitter.com/CodingItWrong)

Josh 分享了他通过为 Babel(他经常使用的开源项目)做贡献来学习新东西的经历。

我发现乔希的演讲令人难忘，因为他的教学技巧。在 5 分钟内，他的简单例子帮助我了解了我们如何使用 [WeakMaps](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/WeakMap) 来实现私有字段，尽管我以前从未使用过 WeakMaps。

通过涵盖理论(抽象语法树)和应用(略读代码以获得要点)，Josh 表明我们可以通过理解巴别塔的一部分来开始为巴别塔做贡献。

### React Native 与 Expo 中的增强现实

#### [](#by-evan-bacon)作者[埃文·培根](https://twitter.com/baconbrix)

Evan 的演讲虽然缺乏让我们开始编写代码的细节，但却有很多现场演示，我想练习 React Native 和 three.js 来创建我自己的 AR 应用程序。Evan 讨论了世博会正在解决的开放性问题，并让观众自始至终保持愉悦。
