# 创造好的类似物

> 原文:[https://dev.to/zellwk/creating-good-analogies-4fi7](https://dev.to/zellwk/creating-good-analogies-4fi7)

许多初学者认为他们不能理解 JavaScript，因为 JavaScript 是抽象的。

但是 JavaScript 不是抽象的。编程也不是抽象的。

人们觉得编程很抽象，因为他们还没有一个合适的钩子来理解它。一旦安装了合适的钩子，编程就会变得真实和可信。

这个钩子以类比的形式出现。

写出好的类比有三个步骤:

1.  选择一个可以理解的类比
2.  展开类比
3.  将类比与概念联系起来

## [](#pick-an-understandable-analogy)挑选一个可以理解的比喻

你的第一步是选择一个你的学生能理解的类比。这应该是他们之前经历过的。

想想你的学生。他们在日常生活中经历了什么？你可以用哪些经历来类比你试图解释的概念？

打开一份文件，写下你可以用来类比的每一个可能的经历。不要过滤自己。把一切都写下来。

然后，选择一个类比。

只有一个。

并展开类比。

## [](#expand-on-the-analogy)比喻展开论述

有些人会很直观地理解你的类比。

但不是每个人都会。

你需要解释你的类比，这样每个人都能明白。如果你不能从学生那里得到即时反馈，这一点尤为重要。在网络上，我们无法获得即时反馈。

这也意味着你选择的类比需要简单易懂。不要选择那些需要学生理解很多规则的东西(比如足球)。

以下是我过去用过的一些类比:

1.  教函数的时候，我讲了[从井里打水](https://zellwk.com/blog/js-functions/)。
2.  当我教授对象时，我像计算机一样将对象与现实生活中的对象联系起来。
3.  我教 scope 的时候分享过一个[单向玻璃](https://css-tricks.com/javascript-scope-closures/)。
4.  当我教 Git 的时候，我谈到了游戏中的保存点数。

当你扩展你的类比时，试着让学生想象你正在谈论的事情。让它生动起来。你的类比越生动，他们记住的时间就越长。

这里有一个关于我如何为函数创建 draw water 类比的例子。注意，你可以想象自己拎起一桶水，走到村子中央。你也可以想象向某人解释这个过程的痛苦。

> 想象你住在一个没有自来水的村庄。要取水，你需要拿一个空桶，前往村子中间的水井，从井里打水，然后回家。
> 
> 你需要一天多次从这口井里打水。每次你解释你在做什么的时候，说“我要拿一个空桶，去井边打水，然后带回家”是一件很麻烦的事情。
> 
> 简而言之，你可以说你要去“打水”。
> 
> 我的朋友，你创造了一个函数。

一旦你解释了你的类比，你需要把它和概念联系起来。

## [](#link-the-analogy-back-to-the-concept)把类比联系回概念

将类比与概念联系起来是至关重要的。如果你错过了这一步，你的学生会奇怪你为什么要提出这个类比。

为了把类比和概念联系起来，你要谈论类比和概念之间的相似之处。

以下是我如何将打水比喻与函数联系起来的。

> 函数是以特定顺序执行任务的代码块，比如拿着空桶，去水井，打水，回家。

一旦你把类比和概念联系起来，你就把概念具体化了，对你的学生来说是真实的。类比的工作已经完成。

接下来是解释他们需要知道的关于这个概念的细节。

## [](#wrapping-up)包装完毕

创造伟大的类比有三个步骤。

1.  选择一个可以理解的类比
2.  展开类比
3.  将类比与概念联系起来

一开始很难进行类比。你可能觉得你的类比不够好。

这没关系。随着时间的推移，你总能改进你的类比游戏。
