这是一个非常有趣的问题。我想把这个问题应用到其他工作中会有助于重新定义它。

你对空闲时间不盖房子的木匠有什么看法？

在我的一生中，我遇到过各种各样的开发人员，他们中的很大一部分人都是那种拥有工作技能的人，他们不在乎在工作时间之外沉迷于工作。相反，他们利用空闲时间造船、冲浪、绘画、写作、烹饪、养家糊口等等。

有些是技术领导，有些是首席技术官，有些是初级人员，其余的则介于两者之间。我曾经共事过的一些最优秀的技术人员非常擅长将他们的工作和家庭生活完全分开。

在我职业生涯的早期，我就注意到(在我看来)开发人员承受着巨大的压力，他们要 24/7 全天候地编码/阅读/开会，以免落后。事实是，随着我逐渐理解，如果有一个支持成长的环境，每周花 40 个小时写软件足以促进成长和进步。

我很幸运能为那些非常重视员工成长的组织工作，因此这些组织提供了充分的机会(在工作中)去涉猎新技术、学习和成长。

作为开发人员，避免停滞的唯一方法是让工作消耗你的生活，这种想法很普遍，但仍然非常有害。

话虽如此，这里没有人试图扼杀任何人的热情。如果你喜欢编码，抓住每一个机会去做。但是在评判那些没有同样热情的人的技能、成长和发展时要小心。

*也许不是你生命的全部，但却是重要的一部分

我们人类给彼此施加了很多压力，有很多竞争激烈、不健康的环境给我们带来了很多痛苦。我们每个人都有不同的个人背景，这些背景最终会比 1 和 0 更多地指导我们的生活。

我会小心地假设他们没有激情，因为他们不会在空闲时间写代码。有一段时间，我是一个单身母亲，除非我在工作，否则我没有时间写代码。这并不意味着我在工作的时候没有学习新的东西，或者我对技术不感兴趣。:)

学习和努力变得更好的激情是人们与停滞不前的区别。

如果你想在业余时间写代码，那就去做吧，但这个职业绝对是一场马拉松，对我来说，能够让自己更多地离开电脑是一种优势。
