# 什么是懒装？

> 原文:[https://dev.to/flippedcoding/what-is-lazy-loading-4pnn](https://dev.to/flippedcoding/what-is-lazy-loading-4pnn)

每个开发人员都需要的一个关键技能是聪明的懒惰。重要的是要尽可能少地编写代码来让函数工作，因为这将有助于防止性能和内存问题。这就是为什么我们今天要讨论懒惰装载。

延迟加载是一个聪明的概念，可以显著提高页面加载速度。它的核心非常简单。所有的延迟加载意味着你的页面直到有人在看它们的时候才加载某些元素。

你可能已经看过很多了。无论你什么时候去那些页面无限滚动的网站，那都是在使用某种形式的延迟加载。通常有某种占位符或加载轮让你知道内容随时会显示。

它在加载图像和视频时使用了很多，因为这些资源占用了大量内存和带宽来加载，这降低了网站的速度。当你偷懒加载图像和视频时，你一开始并没有使用真实的东西。您会有某种占位符来保持空缺。

然后，当有人向下滚动页面，图片进入视野，它神奇地弹出。这是因为您有一个事件监听器，当图片移动到屏幕上时它会捕捉到。这是处理延迟加载的基本方法。它完成了工作，你可以很容易地加速你的网站。

但是还有其他更好的方法。其中一种方法是使用 JavaScript 库为您处理延迟加载。你不用写任何代码。你所要做的就是在库中正确的地方使用正确的参数，然后你就可以开始了。

如果您想更多地接触代码，另一个选择是使用交叉点观察器对象。这将为您提供一些方法，使您能够非常容易地确定一个元素是否在屏幕上以及其他有用的信息。你可以在这里了解更多如何使用交叉点观察器:[https://developer . Mozilla . org/en-US/docs/Web/API/Intersection _ Observer _ API](https://developer.mozilla.org/en-US/docs/Web/API/Intersection_Observer_API)。

现在你知道了什么是延迟加载，以及在代码中使用它的几种不同方式。只是一个专业建议。您可能不想延迟加载页面上的所有内容。如果你可以定期加载图片或视频，将在页面顶部，然后这样做。当页面下方有大量内容时，延迟加载更有帮助，它会降低初始加载时间。
