搭建舞台:你是一个刚进入这个领域的新人。你不想遭受冒名顶替综合症或背负“平民”的可怕标记，但对如何在真正的程序员亚文化中导航混乱的潜流感到困惑。通过练习这些简单的步骤，你可以骑虎难下。

1.  掌握命令行。你将运行大量的脚本，并从那里启动一些东西，所以为什么不流利地使用它呢？
2.  不要在文件名或目录名中使用空格，也不要在其他地方使用空格。如果你被发现使用了一个，你将被认为是“GUI 点击者”,并且无法修复。如果你的代码在一个名字中有一个空格的目录下(或者在一些极端的情况下，两个空格，我不骗你)，它被认为是垃圾，不管它是不是约翰·卡马克自己写的。
3.  学会用类型思考。如果你只是在工作中使用动态编程语言，在家里使用一个类型化的。如果你不能用类型来思考，你的技能会被认为是缺乏的，不管你在私人生活中的生产力或成功与否。
4.  学习一门**类型化的函数式编程语言** (OCaml、F#、Scala、ReasonML、Haskell……)。无论你是一个多么糟糕的程序员，或者无论你被迫在日常工作中使用多么“冷门”的编程语言(PHP、ES5、Java 等等)，如果你能轻松地使用其中的一种，你就会被认为是顶级的。使用 Clojure 或 Elixir 不算数，因为 1)它们没有类型，2)它们被视为现已解散的 Ruby 部落的“潮流跳跃者”目标。如果你能在 Idris 或其他相关类型的语言中高效工作，尽管你从未关闭过你应该处理的工单，你也被认为是编程之神。**额外的警告**:使用 Haskell 可能会被认为“太努力了”。
5.  只在星期五喝酒。这可以确保你在周一午餐时间或周二早上(取决于你的年龄和精力)恢复了你的思维敏锐度和创造力。
6.  尽可能多地解决问题。如果你在当前的任务中没有遇到有趣的问题，帮助其他人解决他们的问题，或者创建一些你可以练习解决新的有趣问题的辅助项目。
7.  想怎么编码就怎么编码。8 小时工作日只能容纳一定数量的代码，如果你坚持这样做，你对你的雇主来说肯定是“足够好”的。但是多编码，就会更好。编程就是练习，练习，再练习。人们通常会告诉你不是这样，但是父母也倾向于赞美他们孩子的歌唱。
8.  如果你必须画画来理解某事，那就在纸笔上画。Git 是你的工作输出应该去的地方。就像树木在森林里倒下时没有人听到的声音一样，没有在 Git 中结束的工作将被视为没有发生的工作。例外:如果你帮助某人在 Git 中获得了一些东西，你可以合理地解释做了一些有用的工作。
9.  如果你是男性，尽可能留胡子。你不必一直戴着它，但至少要确保有一个经过验证的场景，事实上，你被认为有足够多的胡子。如果你是一个女人，不管你是否能长出一个，适度是被建议的。
10.  不要使用 shell 脚本语言(bash 或。bat/。cmd 文件，或者 Powerhell)如果你有 2 行以上的“代码”。请使用 Python(或者 Ruby，如果必须的话)来代替。如果有人没有安装 Python，他无论如何都不配运行你的脚本大作。
11.  有编程以外的爱好。看在上帝的份上，至少读一本书或者玩玩电子游戏。
12.  获得一个你在 Twitter 上关注的所谓“思想领袖”的良好军火库。他们应该主要由 35 岁或以上久经沙场的老兵组成。年轻一代会把你带入 Node 和其他被认为不够程序员化的亚文化的沼泽。另一方面，要警惕那些被认为过时的非常老的思想领袖(比如“鲍勃叔叔”，或者一般的固定模式)。
13.  关注节目新闻。在 Reddit 上， [/r/programming](https://old.reddit.com/r/programming/) 和 [/r/programmingcirclejerk](https://old.reddit.com/r/programmingcirclejerk/) 以良好的规律性提供可承受的价值。HackerNews(也称为“橙色网站”，因为维护者[不会做 CSS](https://news.ycombinator.com/) )有一小群追随者，但 1)你不能从那里获得“编程新闻”和 2)社区(又名 [webshits](http://n-gate.com/) )总体上令人讨厌和冗长，阅读太多他们的输出会导致社交技能的降低，就像他们一样。
14.  **“重要固然好，但更重要的是做好人”**，套用德国哲学家[的话](https://en.wikipedia.org/wiki/H.P._Baxxter)。如果你是一个不错的程序员，但是傲慢或者刻薄，你将会像一台机器上笨重的老齿轮一样呆在那里，因为拔出它需要拆卸太多其他东西。

你的里程可能会有所不同，这篇博文不应该被当作福音。首先，它是由一个作者写的(相对于福音书，写这篇文章的时候有四个作者)。
