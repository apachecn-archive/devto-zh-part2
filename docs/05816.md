# [](#the-ternary-operator)三元运算符

所以我的论点是要一劳永逸地说服每个人，正确的用法要么是**三元运算符**要么是**条件运算符**，说**是三元运算符**是不正确的。

首先，我们必须奠定一个我们都同意的事实基础。没有这个基础，就不可能达成协议。

**事实 1:** JavaScript 有一个三元运算符类型的条件运算符。如果有文氏图，条件运算符应该在三元运算符的内部。

**事实 2:** JavaScript 正好有一个三元运算符。在文氏图中，除了条件运算符圆，三元运算符圆不包含任何圆。

事实 3: 当使用 **A/AN** 时，你指的是一组项目中的一个。例如“一件衬衫”或“一美元”。

**事实 4:** 当使用 **THE** 时，你指的是一个特定的项目，如果有一个集合，该集合中的一个特定项目是已知的，不会与另一个项目混淆。例如，集合中只能有 1 个。例子可以是“猫”或“车”。

因此，如果我们能就这些事实达成一致，那么应该清楚的是，**是三元运算符**的正确用法，因为 JavaScript 正好包含一个三元运算符。****

 ****如果 JavaScript 包含不止一个三元操作符，那么并且只有在那时条件操作符才能被称为**三元操作符。**

# [](#examples)例子

我的房子里只有一只猫，它的名字叫 Mojo。Mojo 是 Cat 类型的，就像条件操作符是 T2 类型的三元操作符一样。所以我也可以要求“把猫给我带来”你就明白了。如果我说“给我一只猫”，这会引起混乱，你可能会认为我要的是一只猫而不是 Mojo。

如果你在帮我做饭，我可以说“给我一个容器”，有不止一个容器可供选择。你选择一个容器类型的搅拌碗。但是一旦你把碗放在柜台上，我可以说“把抱子甘蓝放在容器里”，因为我指的是一个特定的容器。如果柜台上没有容器，我说“把抱子甘蓝放进容器”是不正确的，因为容器不止一个。

“把遥控器递给我”。我家里有多个遥控器，但你很清楚我指的是哪一个。因此，即使有多个遥控器，特定的遥控器也在上下文中，允许我使用。
