# 如何写好一篇编码文章

> 原文：<https://dev.to/zellwk/how-to-write-a-good-coding-article-3mb7>

一篇好文章向学生展示如何思考一个问题。学生会说“哦哦哦！”当他们通读这篇文章时。他们会理解他们试图学习的概念，他们会停止在网上搜索相同的主题。

烂文章反其道而行。学生在阅读这篇文章时会感到更加困惑。他们甚至怀疑自己是否具备学习编程的条件。

把一篇糟糕的文章变成一篇好文章并不需要太多。内容可以保持不变。你只需要答对五个因素。

## 五大因素

这五个因素是:

1.  你文章的目的
2.  这个学生是谁
3.  所用的例子和类似物
4.  使用的语言
5.  阅读的容易程度

## 你文章的目的

想象你在迷宫里。你正沿着一条小路走着，你看到一个岔路口。

你应该向左走，还是向右走？

你不会知道答案的。那是因为你不知道出路。

正在搜索文章的学生也同样迷失了。他们没有答案，这就是为什么他们首先在寻找。

给他们指出正确的方向是你的文章的工作。这意味着你写的每篇文章都应该有一个终点。这个终点就是你这篇文章的目的。

例如，在这篇文章中，我想告诉你要注意的五个因素，这样你就可以避免写糟糕的文章。你创作的每一篇文章至少会是体面的(希望是好的！).

为了给学生指明正确的道路，你必须了解他们是谁，他们的技能水平如何。

## 谁是学生？

你的学生是初学者、中级还是专家？

初学者、中级学生和专家有不同的技能水平。一旦你考虑到他们的技能水平，你就能为他们写更好的文章。

例如，你不会教初学者构建复杂的应用程序。你不会以“npm 安装这些包”开始你的文章。

这是因为初学者很有可能不知道什么是 npm。他们也可能不知道你让他们安装的软件包。

他们迷路了。

此外，当你考虑到学生的技能水平时，你的内容也会发生变化。对于初学者，您将创建包含较小信息块的文章。对于专家来说，你可以在一篇文章中将许多不同的技能结合在一起。

对于这篇文章，我的理想学生是想写好编程文章的人。他们写了不到二十篇文章。

一旦你考虑了目的和学生，是时候看看你用的例子和类比了。

## 所用的例子和类比

编程不是抽象的。编程可以用真实的、贴近内心的方式来解释。感觉很抽象，因为老师用了无意义的变量，比如`foo`和`bar`😑。

所以第一条经验法则是永远不要使用`foo`或`bar`。总是试着用一个学生能理解的例子。

想想你的学生。

他们在日常生活中经历了什么？你可以用哪些经历来类比你试图解释的概念？

决定一个类比，然后解释它，这样那些没有直观地得到类比的学生仍然会明白你的意思。

例如，我用单向玻璃类比来解释 JavaScript 作用域。阅读[这篇文章](https://css-tricks.com/javascript-scope-closures/)，了解我是如何解释的。

接下来是你的语言。

## 所使用的语言

文字有引发强烈情感的能力。如果使用不当，它们有能力让学生觉得自己是白痴。

看到我做了什么吗？

那一句话可能引发了一些忧虑、无用或内疚的感觉，而这些感觉对大多数文章来说是没有建设性的。

如果你是写文章的初学者，把握情绪可能会很困难。但是不要因此而停止写作。

现在，我要你专注于两件事。当你变得更有经验时，我们可以探究你如何在文章中插入和撤回情感😉。

第一，确保你解释了你使用的每一个行话。术语使概念模糊不清。他们感觉就像无法解释的潘多拉魔盒。但是，如果你不能解释一个行话，你怎么能指望别人明白你的意思呢？

这就是为什么你需要做的第一件事是学会用简单易懂的语言解释术语。

第二，注意像 *这样的词显然是*，*只是*，而*只是*。

最后，你会希望通过一些编辑来使你的文章更加流畅。

## 编辑为平滑阅读

你的文章的任务是帮助学生理解你所教授的概念。你想尽力帮助他们理解。

提高他们理解的一个方法是使用简短的句子。永远记住清晰比聪明更重要。这是一门需要一段时间才能提高的手艺。如果你使用像海明威这样的应用程序，你可以加快这个过程。它会告诉你哪些句子需要修改。

另一个提高理解的方法是消除你文章中的语法错误。在这里，你可以使用类似于[语法](https://www.grammarly.com)的 app 来检查语法错误。

## 包装完毕

要写好一篇文章，需要考虑五个因素:

1.  你文章的目的
2.  你的学生的技能水平
3.  你使用的例子和类比
4.  你使用的语言
5.  为流畅阅读进行编辑

有更多的技巧和技术可以让你写出好文章。这五个因素是最基本的。

现在好好享受，去写点文章吧:)
