# 用于 devs 工作流的 Konsole、SublimeText 和 PAC 管理器

> 原文:[https://dev . to/nebojsac/konsole-sublime text-and-PAC-manager-for-a-devs-workflow-a2k](https://dev.to/nebojsac/konsole-sublimetext-and-pac-manager-for-a-devs-workflow-a2k)

首先，我选择的操作系统是 Ubuntu。现在我用的是 16.04，但是我计划在年底前升级到 18.04。唯一缺少的是 Adobe 程序，但这对我来说不是一个大问题，因为我主要处理后端工作。

我使用 Windows 已经很多年了，它还不错，但是对于开发来说并不太好。有一次，我给自己买了一台 Macbook Air，并使用了一段时间，享受终端，尤其是 **iTerm** ，但真的不喜欢*关闭整个 Mac 系统的感觉。我不小心用咖啡炸了它，我不能自己打开它的事实真的让我很生气，所以从那以后我发誓不再吃苹果电脑。我只是太习惯于摆弄我的设备了:我在 Windows 机器上用过，在我的联想 Thinkpad 上也用过。我很高兴地打开它，并添加了固态硬盘和额外的内存，以帮助它一切顺利。*

在很大程度上，我真的很喜欢 Ubuntu，但也有一些事情困扰着我。一个正在发生的问题是驱动程序的问题，因为我的 WiFi 会随机断开一秒钟，而且*上帝保佑*我把笔记本电脑设置为睡眠模式。唤醒它让我没有 WiFi，有时也没有正常工作的音频插孔。

除此之外，我喜欢我能在 Ubuntu 上完成和建立的东西，而且我还在成长。额外加分的是，它基本上是一个练习我的 DevOps 技能的好方法，这是我日常工作的一部分。

# [](#code-editor-sublime-text-3)代码编辑器:崇高文本 3

[![Sublime Text](../Images/fc1f80bf4e5fbd55a6adbb4e0c051faf.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--ACBunTlY--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/x9m6m6h9d0b8q5r9sawy.png)

我最初几年的开发工作用的是 Notepad++。与当时较大的工具相比，我很喜欢它，并且觉得我学习代码的速度比使用自动完成功能要快。结果很好，但只是因为我主要是在做 PHP，而不是像 Java 或. NET 那样急需 IDE 的东西。由于 Notepad++不可用，我开始寻找类似的东西。我发现了崇高的文本 2(当时)，并立即爱上了林。

它有我喜欢的 Notepad++的所有东西，还有额外的铃声和哨声。最重要的是速度和可定制性。就速度而言，唯一胜过 Sublime 的是 VIM，我还没有找到时间去学习它——尽管我希望在某个时候尝试一下。我也非常尊重 Sublime 的创造者，因为他让它可以免费使用(nagware ),以及他带来的创新，其他人后来复制了这些创新。

我可能最注意到的插件是:

## [](#for-syntax)为语法:

*   **Todo.txt 语法** -我在我的“Todo”文件中使用它
*   **降价扩展**和**降价预览** -我们的文档在降价中
*   **VCS 沟** -新的/删除的/更改的行的基本 Git 和 Mercurial 指示器

## [](#for-cleanup-and-readability)为了清理和可读性:

*   **FileDiffs** -轻松比较文件或片段的差异
*   **SQL 美化器** -缩进 SQL
*   **JSON 重入** -重入 JSON
*   HTML 整洁(我们处理很多 HTML 邮件)

## [](#code-styling)代码样式:

*   清理 PHP(缩进、括号等)
*   **PHP 数组语法转换器** -将`array()`转换为`[]`语法
*   **DocBlockr** -生成文档块

还有一些更小的。我最终会时不时地进行“插件清理”,以保持崇高的精神。

# [](#terminal-usage-konsole-ohmyzsh-and-pac-manager)终端使用:Konsole、OhMyZsh、PAC Manager

自从离开 Macs，我只怀念一件事: **iTerm** 。我还没有找到一个如此强大，又如此容易安装的工具。我已经尝试了 Linux 上所有可用的工具，但没有一个工具能帮我做到。我喜欢的是:

*   用于管理会话的绝佳键盘快捷键
*   您可以重复使用的已保存和预定义的会话
*   新会话非常容易设置和使用
*   您可以使用特定的命令和键盘快捷键来启动特定的服务器连接

现在，我知道像*终结者*这样的工具可能可以完成大部分工作，但我不想成为一名 1337 黑客来让我的工具做我想让它们做的事情。我愿意花一些时间在它上面，但是如果我在几个小时内不能弄清楚它，我就继续下一个。我最终在 Konsole 找到了一个很好的平衡点。它给了我一个很好的可定制的开始会话，有一些很好的快捷方式。它对于管理服务器来说还不够好，所以我使用 PAC manager，它有时感觉有点笨拙，但如果你想将大量现成的命令发送到服务器，它实际上是很棒的。这两者都比 iTerm 需要更长的设置时间，但是它们一起做得很好。

有了 PAC manager，我可以很容易地打开十几个 SSH 标签，有些用于查看日志，有些用于监控服务器性能，有些用于运行命令。PAC 的唯一问题(直到写这篇文章我才注意到)是它已经 2 年没有更新了。我在过去的 4 年里一直在使用它，我没有看到一个特别的更新需求，它在我目前使用的每个 Ubuntu 版本上都运行得很好。

看起来像矩阵的数字来自命令`cmatrix`,我把它用作屏保。一旦我开始打字，它们就消失了。我使用 OhMyZsh 作为我的 shell，它只是使 shell 的使用更容易，整体更好。

我使用 Ubuntu 中默认的 4 个工作空间，并将我的窗口组织成 4 组:

*   编辑(崇高)
*   浏览器
*   终端(控制台)
*   服务器终端

各种各样的应用程序会在最合适的时候打开。

一旦我有时间，我想设置 i3wm 并学习如何使用它。我在 Reddit 上看到了一些很棒的设置。我倾向于每个月花几个小时来调整我的设置。很容易迷失其中，所以我限制自己，否则我会纠结好几天！

我做了很多小改动，让我的系统看起来像是我的系统，但这是鸟瞰图。我强烈建议你也和我们分享一下你的设置，因为你总是可以从这些帖子中得到一些提示。
