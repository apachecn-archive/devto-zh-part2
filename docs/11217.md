## 简介

对于那些生活在岩石下的人来说。你可能听说过 Adobe 刚刚开始提供一个免费的等级计划。

我刚刚放弃了 XD，因为它第一次发布时没有自由层计划。这让我在 Youtube 上搜索视频时偶然发现了 Figma，以获取构建 **UI/UX** 设计的知识。

当它与我寻找在 Linux 上运行的免费且有用的设计工具的工作相关时。在 Django 上构建之前，我使用 Figma 设计 UI/UX 原型来节省时间和精力。

## 原因 1 -超棒的免费等级计划

Figma 允许您与您的开发人员、产品经理和客户共享您的设计，而无需他们下载任何软件来使用您的网络浏览器进行查看/编辑。

除此之外，它允许最多两个人实时编辑同一个设计，并有无数的观众观看。

因此，对于拥有小型开发团队的初创公司来说，利用它来构建自己的 MVP 或 UI/UX 设计是非常好的。

## 原因 2——对设计师不友好

我遇到的大多数 UI/UX 设计工具通常是为了迎合不同类型的用户。

一个功能丰富，但有一个巨大的学习曲线，另一个是极简的，除非你为额外的功能付费，否则你不能用它做很多事情。

Figma 提供的是极简主义的设计，就像在 Google Docs 中工作一样，但是 jam pack 的功能可以让你真正完成工作，从而避免构建令人敬畏的设计。

## 原因 3 -开发人员交接

Figma 最棒的部分是 Figma 的开发人员移交方面，

当你把你的 UI/UX 设计交给你的开发者时，它允许 Android 或 iOS 的移动应用开发者下载你的 Figma 设计的代码用于他们的开发。

如果你像我一样是一名网络开发人员，你可以看看 **CSS** 代码，让你的网站看起来几乎和你在 Figma 中的设计一模一样。

此外，您可以将设计的各种组件下载到 SVG，这大大加快了您在 **React、Angular 或 Vue** 上开发前端的时间。

随着最新的更新，有一个 REST API，允许您导出您的 Figma 设计来作出反应。
