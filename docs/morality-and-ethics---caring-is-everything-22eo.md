# 道德和伦理——关心就是一切

> 原文：<https://dev.to/sjwarner/morality-and-ethics---caring-is-everything-22eo>

计算机伦理是一个极其广泛和主观的话题。我最近接触了一些令人惊讶的教育活动/讲座，并提高了认识，这让我意识到这个话题有多么广泛和主观。在我探索道德技术的神秘面纱(以及软件开发人员如何承担道德开发的责任)的过程中，有时我会觉得有更多的问题出现在我的脑海中——通常会成倍增加——但答案却很少。

这本身就有点可怕(也有点让人泄气)。

虽然我不能代表这个领域的每个人，但我愿意把自己看作是一个问题解决者。我可以将算法、已建立的设计模式或某种过程应用于呈现给我的问题，并得到解决方案。一个“答案”。解决问题的东西。

当我做不到这一点的时候，就很可怕。当这些伦理问题被提出时，在我找到答案之前，我不能瞎搞的事实令人难以置信地担忧——尤其是当我们考虑到这些未解决的问题是多么普遍的时候。

*一大堆没有明确答案的问题会让人失去动力。*

在我们深入探讨之前，让我们快速定义一下我所说的“道德和伦理”是什么意思，因为虽然这些词很有力量，但它们有时可能太大而难以理解:

道德——管理一个人的行为或活动进行的道德原则。道德——关于区分对错或好与坏行为的原则。
道德判断——根据某种善的标准，对某种作为或不作为、意图、动机、性格特征或一个人作为一个整体是(或多或少)好还是坏的评价或意见。
就我个人而言，我认为这些术语‘好’和‘坏’是相当主观的，甚至可以归咎于技术伦理中普遍存在的灰色海洋。今年伦敦 QCon 大会上，程延庆的精彩演讲[‘负责任地打碎潘多拉的盒子’](https://qconlondon.com/london2018/presentation/responsibly-smashing-pandora%E2%80%99s-box)展示了当我们做出这些更人性化的决策时，衡量标准的重要性。通过评估成本与收益，我们可以将技术的影响最大化。虽然我们通常认为成本效益分析有点功利主义，因此与更多的道德观点不相容，例如那些专注于道义概念(如伦理)的观点，将“价值”与新工具潜在的改变生活的途径或效果相提并论最初可能看起来有点不近人情，但我肯定能看到它的用途。

*我们值多少钱？现在有一个思想家...*

软件工程是一个不成熟的领域。这是不可否认的。我们太频繁地看到围绕新技术的非理性行为，跳上最闪亮的潮流，随之而来的是具有令人抓狂的复杂程度的过度设计的解决方案。我们经常看到缺乏彻底定义的规范(通常不是因为缺乏尝试)。我们经常看到人们对自己工作的意义视而不见。

然而，人们似乎仍然对伦理技术感兴趣。最近，在我参加 QCon London 2018 期间，第三天有一个令人难以置信的[“技术伦理在行动”](https://qconlondon.com/london2018/track/tech-ethics-action)赛道，它得到了压倒性的高度评价，也是整个活动中一些最令人难以置信和鼓舞人心的演讲的所在地。我有幸参加的另一个道德技术活动是 VHacks 2018(梵蒂冈有史以来第一次黑客马拉松)，它专门设计用于通过软件和硬件解决社会问题，并以国家报纸[文章、](https://www.theguardian.com/world/2018/mar/07/vatican-hosts-first-hackathon-to-tackle-global-issues)[现场新闻报道](https://www.youtube.com/watch?v=rXaugpXTYiQ&index=1&list=WL&t=48s)，甚至[有线帖子](https://www.wired.com/story/inside-vhacks-first-ever-vatican-hackathon/)的形式获得国际曝光！StackOverflow 最近的 insights 调查显示，绝大多数开发人员越来越关心他们的日常开发应该包括什么，以及应该如何进行。

所有这些都表明，作为专业人士，我们对我们所做的东西的道德含义感兴趣。我们想为我们所做的感到骄傲。我们想有所作为。

我们是创新者和潮流引领者。

正如我前面提到的，这些问题不仅影响深远，而且数量众多。虽然总的来说，我们作为开发人员似乎对“对”和“错”有合理的理解，但上面链接的 StackOverflow insights 调查显示，我们看到了许多道德“灰色”——在现实世界的上下文中，我们很少看到黑与白。

stack overflow 的见解调查用了整整一节来讨论软件开发中的道德规范。我认为这表明了我们正在看到的转变，即从确保在学校和其他国家教授编程和计算机科学，并使其更广泛地普及(近年来的巨大发展已经奠定了非常坚实的基础，包括在英国课堂上分发 [BBC micro:bits](http://microbit.org/) )到通过教育一些更好但仍然重要的编程方面，包括在这种环境下道德背后的思想，来成熟这种新的教学渠道。

当然，在我们就这样一个重要的主题进行教育之前(尤其是在世界各地的课堂上)，明智的做法是回顾一下上面的答案，并对当前的问题以及开始解决这些问题的方法有一个清晰的认识。

正如你所看到的，近 80%的被调查者认为开发者作为一个整体有责任考虑他们产品的道德含义。现在，我问你-你最后一次主动这么做是什么时候？在你的公司举行一次小投票，与你的开发团队讨论这个问题，或者坐下来，就你最新项目的道德含义进行一对一的讨论。

在调查中勾选‘是的，我当然关心’是很容易的，但是实际上我们有没有言行一致呢？我倾向于相信少于这个 80%的开发者。

此外，我很想听听那 20%的人的意见，他们认为作为一个职业，我们没有义务考虑我们的行为的道德含义。总的来说，作为一个领域，我们聪明能干。我们应该以更高的标准要求自己，而不是让自己成为可以有偿制造任何东西的雇佣兵。通过把“责备”或“责任”转嫁给别人，我们离控制和管理我们自己的产品——我们应该感到自豪的产品——更远了一步。

StackOverflow 的调查想到的另一个不足为奇但可以补救的问题是，如果开发人员被要求为不道德的目的编写代码，他们应该怎么办？我们如何报告代码的道德问题？对于这两个问题，超过三分之一的受访者回答“取决于是什么”。这非常清楚地说明了当前计算机伦理在行动中的不确定性(甚至害怕说出来)。作为一个行业，我们需要确保有安全的方式来表达开发过程中出现的问题。我确信许多公司已经存在这方面的一些基础设施，但我们可以(也可能应该)对此进行监管，以允许在不引起过度压力和担忧的情况下提出道德问题。

虽然这些都不是解决我们当前面临的问题的具体方案，但一点点的自省和监管(在我看来)将是真正进步的良好开端。

所以现在我们有了。在这场道德(和不道德)技术的旋风之旅中，我的最后三点收获。

1.  我们需要利用技术的天赋为所有人创造可获得的(和有用的)服务，无论社会地位、种族、性别或世代。通过降低我们软件的成本、能耗和资源需求，我们提供了更广泛的潜在改变生活的技术，并为一些最弱势人群提供了潜在的生活质量改善。最重要的是，我们必须明白，不仅仅是价值数十亿美元的跨国公司有能力做到这一点——对伦理技术的最小贡献也会建立起当地的护理者网络，最终汇集成一个能够解决更大规模问题的全球社区。
2.  为了实现这一目标，我们必须理解技术需要跨越国籍、性别和种族的多样性。虽然这些因素中的一些现在已经被意识到并得到了补救(查看约翰最近关于培养你的女儿成为工程师的博客),但这并不是战斗的结束。更好的技术来自多样化。通过在不同的团队中发展(和管理/营销/指导),我们更有可能意识到我们正在做的事情的道德含义，并提高在问题影响最终用户之前发现问题的可能性。
3.  以人为本是让技术造福人类的关键。近年来，我们把科技视为提升人性的一种方式。如果我们要继续这一趋势，我们必须把重点放在可获得和广泛使用的伦理技术的设计上。如果我们与用户交谈，考虑我们的想法的影响，了解产品是为谁制造的，以及我们旨在解决哪些人类问题，我们将重新编程软件工程的价值观。从研究我们正在创造的产品的道德影响开始，我们可能会不断地做出让人类改变世界的项目，并建立服务于人类而不仅仅是股东利益的公司。最重要的是，关心。关心你的产品。关心它如何被使用。关心你所做的事情的含义。

当涉及到开发道德软件时，关心就是一切。
