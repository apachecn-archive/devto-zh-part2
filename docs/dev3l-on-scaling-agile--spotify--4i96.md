# 原则

这是 Henrik 在整个演讲中展示的 Spotify 原则列表。

1.  信任>控制
2.  创新>可预测性
3.  故障恢复>故障避免
4.  实验>争论
5.  混乱>官僚主义
6.  文化>流程

# 文化

有句话叫‘文化早餐吃策略’。Spotify 通过创建一个强调员工满意度和文化重要性的小算法，将这一点整理成文。关注激励团队的因素。听起来#happycows 产奶量更多。当“为什么”排在第一位时，产品会改变(或创造)行业。

`productivity = effort x competence x environment x motivation^2`

Scrum 首先建立秩序...

*   放松实践，关注原则
*   将 scrum master 更名为敏捷教练
*   不能代表某人拥有自己的文化
    *   **你**就是文化
*   每个人的行为和态度的总和

# 自主

> "正常管理，例外处理例外."

不要扩展敏捷，扩展你的组织。Spotify 相信自主的实现。然后，它将此作为公司的指导原则。

自主性是:

*   速度推动者
*   规模
*   动机
*   **如何将**和**接下来的**决定为一个团队

复杂的系统会有正常的事故...快速失败，快速学习——成为“最快的学习者”

# 连续交货

持续交付是最大的商业优势。当反馈循环以周而不是分钟为周期时，快速学习是不可能的。没有稳定的运输渠道，一家公司将会被甩在身后，和其他恐龙一起死去。这个组织的生存原则是。

> “让整个事情运转起来的关键是，小规模的频繁发布”
> “部署应该非常简单，即使是敏捷教练也能做到”

不要给复杂的发布增加结构，让发布变得简单！

Spotify 以 DevOps 自助服务模式为生。它有完整的平台团队来支持组织的其他部分。以德沃普斯咒语为生。隐藏在特性标志之后，所有的东西都会被发布，甚至是那些没有完成的东西。此外，去耦架构为故障创造了有限的爆炸半径。零停机时间部署和金丝雀部署等最佳实践使整个组织能够像独立的快艇一样前进！

一个反直觉的注意:`more bugs != less quality`

# 创新

我们以这种方式推动创新。这个群体越是自主和多样化，你对结果就越是惊讶。

该公司一年有两次黑客周。这是一场免费的创新盛会。全公司参与。想法是培养出来的。整个产品被运送。随着新关系的建立，这对公司来说是健康的，这进一步加强了创新引擎。

# 管理

服务型领导是管理的指导原则。经理不会问“你在做什么？”相反，他们会问你需要什么？’。创造能够实现高度自治和高度一致的环境。

**最少可行的官僚机构**

*   “解决混乱比解决官僚主义更容易”
*   “健康的文化治愈破碎的过程”
*   “废物排斥文化”
*   不是河马
*   没有对错，都是取舍
*   1 > 2 > 0
*   Spotify 不是敏捷的天堂

# 持续改进引擎

如果目标是比其他人学得更快，不系统地将这些知识应用到工作的各个方面是不可接受的。Henrik 没有提到丰田生产系统(TPS)改进卡塔，但这似乎是 Spotify 文化的一部分。

团队应该知道他们的最大障碍是什么。
