在这一集里，我们为我们的测验项目建立了一个持续集成(CI)管道。

这个初始设置和与 GitHub 集成的目标是保证我们只合并不破坏构建并且通过所有测试的变更。对于 CI 服务器，我们可以做更多的事情，例如，自动构建、上传和部署应用程序到 TestFlight 甚至 AppStore。

*持续整合*只是开始，随着*持续打造*和*持续部署*(还有回滚！)是我们想要达到的更高的目标。这种自动检查可以节省我们宝贵的时间，并且在团队开发应用程序时，对于和谐的协作至关重要。它还可以显著提高**对业务部门的信任**和**透明度**，在开发过程中，我们可以**一天数次**向更广泛的团队部署应用版本。
