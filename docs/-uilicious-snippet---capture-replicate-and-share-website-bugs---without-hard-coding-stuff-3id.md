第一次试运行于[dev . to——当 youtube 宕机时](https://dev.to/uilicious/youtubecom-is-down---and-proving-it-with-testing-351)

经过数千小时的准备编码和咖啡。今天我们正式启动[snippet.uilicious.com](https://snippet.uilicious.com)到[的产品搜寻](https://www.producthunt.com/posts/uilicious-snippet)

[![snippets in action](../Images/5b462fcbb1d863cacf3c3fa62e9d289e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--b2UmCK0H--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/egdz7y2lo8myhh3uhrp9.gif)

这背后的想法是允许用户编写清晰的测试脚本，以及易于理解、可重复的错误报告，最重要的是可以公开共享！

通常情况下，编写 bug 报告会很乏味或者不准确。有时开发人员可能无法复制 bug，即使有提供的步骤！

因此 [snippets](https://snippet.uilicious.com) ，任何人都可以轻松地编写简单的测试脚本，无需复杂的 css 选择器硬编码，甚至无需登录。并附上包含结果的可共享链接，以便立即发送给开发人员。

更新:关于我们测试语言的更多细节可以在它的[伙伴帖子中找到——消除 UI 测试的恐惧😱](https://dev.to/uilicious/taking-the-horror-out-of-ui-testing--1fi6)
