# 告别个人项目

> 原文:[https://dev . to/Michael/say-bye-to-a-personal-project-3na 2](https://dev.to/michael/saying-bye-to-a-personal-project-3na2)

两年前，我开始寻求写一本书。这是一本关于静电发生器的书，Jekyll。我逐渐熟悉并非常喜欢的一款软件。这是我用来写[我的个人网站](https://michaelsoolee.com)的。

看到缺乏关于 Jekyll 的可靠资源，以及让 Jekyll 运行起来对我来说是多么的痛苦，我认为以此为主题写我的第一本书是个好主意。帮助人们通过一些更复杂的哲基尔导航。

最初的几个月能量很大。我已经[设计了一个登陆页面，有一个可爱的仿书封面设计和所有的东西](https://s3.amazonaws.com/i.michaelsoolee.com/20180619-field-guide-to-jekyll-archive.png)，整理了一份免费的赠品，里面有我关于如何在你的系统上安装 Jekyll 的说明，并开始为我的时事通讯收集电子邮件，让人们了解情况。

原来写书很难。我可能把注意力放在了我本应该在写作过程的后期关注的事情上。因为专注于获得完美的 PDF 布局，并建立一个流程来完全按照我想要的方式生成它，这有点过头了。

我也让这本书的范围失去了控制。我觉得有必要报道大量的事情，而不是专注于和 Jekyll 一起行动。这样做，我认为我为许多人创造了一个伟大的资源。但是我应该只关注一小部分人，关注他们的需求和痛点。

我没有马上终止这个项目，而是把它保留了两年...每次我想到它，我都会说服自己，我需要最终抽出时间来解决它。但这从未发生过。我的注意力总是分散在其他能让我更快获得成就感的项目上，因为对我来说，编码比写作更容易。

几个月前，我希望恢复我的个人时事通讯。我想给它应得的照顾。我也想[和那些已经签约的人](https://michaelsoolee.com/pizza-fund/#i-want-to-build-rapportrelationships)建立融洽的关系。所以我登录了 Mailchimp，看到了我的书的时事通讯列表。我已经收集了几百封电子邮件。我觉得有一种诱惑，想让名单上的每个人都知道我将结束这个项目，并给他们一个机会阅读我的个人通讯，但相反，我决定切断联系，继续前进。我删除了整个名单。

老实说，我早就该这么做了。这个项目的经历是积极的——与许多人有联系，有机会演讲——它占据了太多的空间。我没有从精神上解放自己，把精力集中在其他事情上，而是不断回到这本书上，为没有完成它而感到难过。

不久前，我已经从我的网站上删除了这本书的顶级导航链接，但今晚，我终于花了一些时间更新了[注册页面](https://michaelsoolee.com/jekyll-field-guide/)，让来的人发现我已经继续前进了。我还对我个人网站的回购进行了一些清理，以清除这本书的资产和内容。这是今晚有人在 Twitter 上提醒我，我的注册表格不再有效了。

我想通过写这篇文章来分享一个项目的结束，从而结束对我的书说再见的决定。我经常宣布我正在做的新的[项目](https://michaelsoolee.com/about/#things-ive-made),但是我认为分享一下关于告别一个项目也是有用的。一个看不到我设定的目标的人。

至于项目的剩余部分，在项目生命的某一点上，我曾想过开源这本书，并公开编写它。这样做，我希望得到贡献者。我已经着手[开源了这本书](https://github.com/michaellee/field-guide-to-jekyll),但不会积极地为它做贡献。我开源它的原因是，我认为仍然有一些内容将帮助人们在 Jekyll 的旅程中，但因为它是两年前写的，部分内容肯定是过时的。

我从写一本书的项目中学到了很多。也许有一天我会找到灵感和感兴趣的话题来写另一本书，但现在我要和《哲基尔野外指南》说再见了。如果你正在寻找一些 Jekyll 帮助，我已经在[书的曾经登陆页](https://michaelsoolee.com/jekyll-field-guide/)列出了我在生成器上写的所有文章。

你曾经不得不对一个个人项目说再见吗？如果是这样，我很想听听。请在下面留言，让我知道这对你来说是容易还是困难。
