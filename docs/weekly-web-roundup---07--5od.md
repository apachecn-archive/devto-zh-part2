## CSS 和 HTML

*   Roma Komarov 发表了一篇很棒的[文章，关于实验 CSS 变量来创建没有 JS 的可排序订单](http://kizu.ru/en/blog/variable-order/)。由于可访问性等原因，它并不真正意味着生产，但尽管如此，它仍然是对可能发生的事情的一次很酷的探索

## Java Script 语言

*   当您想要将数据库公开为 GraphQL API 时，最常用的工具之一是 Prisma。 [Divyendu Singh](https://twitter.com/divyenduz) 对 Prisma 做了一次[的深入研究，它变成了让你进行设置的一个很好的资源！](https://divu.in/prisma-deep-dive-3162dea2820c)
*   André Staltz 在[中写了一篇关于 JavaScript 承诺缺陷的有趣文章](https://staltz.com/promises-are-not-neutral-enough.html)。尤其是*“渴望，而不是懒惰”*部分是我经常遇到的事情，尽管我并不真的认为这是一个*那么*大的问题，但很难否认有些事情可以变得更好。
*   Craig Roblewsky 分享了一些关于如何用 GreenSock 库自然流畅地改造 SVG 的好点子。
*   用 JavaScript 加密货币是邪恶的！！或者是？ [Ber Kessels](https://twitter.com/berkes/) 为浏览器中可能的挖掘提供了一个很好的[案例，当然只要它是选择加入的。在你开始大喊大叫说你不想让你的电脑变成一个只有 3 个月寿命的烤箱之前，读一读这篇文章](http://berk.es/2018/02/15/making-a-case-for-javascript-mining/)

## 战略情报局

*   Axe 是对基于 HTML 的界面进行自动化可访问性测试最常用的引擎之一。我不会说谎，我从来没有听说过它，但它似乎应该得到更多的明星。[尼克科利](https://twitter.com/NickColley/)发布了 [`jest-axe`](https://github.com/nickcolley/jest-axe) ，一个库为了让 Axe 的可访问性需求玩了一个很好的玩笑。
*   Yangshun Tay 制作了一本前端面试手册，似乎对员工和雇主都很有用。当标语是*“几乎完整的‘前端工作面试问题’的答案，你可以用它来面试潜在的候选人，测试你自己或完全忽略”时，我怎么能不分享呢*

## PSA

尽管本周我没有在 Twitter 上花太多时间(因此有了这个小小的综述)，但我看到的看起来像是 playa 憎恨者的舞会。我看到了很多关于没有人被强制使用的图书馆的抱怨，关于*“不属于我们的行业”*的人的抱怨，以及关于实验不够开放的抱怨。讨论是伟大的，但让我们变得更好！
