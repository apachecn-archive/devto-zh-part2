#### CSS&SCS

##### [](#how-to-build-complicated-grids-using-css-grid)[如何使用 CSS 网格构建复杂网格](https://danwebb.co/journal/how-to-build-complicated-grids-using-css-grid)

使用 CSS grid 构建一个复杂的、非传统的网格，并使用一些时髦的偏移量和比率，这是一个非常有趣的演练。我喜欢 CSS grid 在我们可以考虑的网页布局类型中引发的复兴。非常酷！

##### [CSS 网格 9 大错误](https://www.youtube.com/watch?v=0Gr1XSyxZy0)

Jen Simmons 的精彩视频讲述了人们开始使用 CSS Grid 时出现的一些常见错误。对于每一个问题，她不仅会回顾错误，还会思考错误的来源，并强调更好的解决方案。我不是一个喜欢看视频的人，但这 14 分钟绝对值得。

##### [](#css-a-new-kind-of-javascript)[CSS:一种新的 JavaScript](https://medium.com/@Heydon/css-a-new-kind-of-javascript-fcf730d33ce7)

这篇文章对“JavaScript 中的 CSS”的争论进行了尖刻的评论，想象了一个 JavaScript 中的样式首先出现的世界，并展示了 CSS 如何解决这种方法可能带来的许多问题。这里没有什么太疯狂的，但它逗得我发笑。

##### [](#4-css-tricks-ive-learnt-the-hard-way)[4 CSS 招数我可是吃了苦头才学会的](https://itnext.io/4-css-tricks-ive-learnt-the-hard-way-aab1e7e8ff44)

针对极其常见问题的一套简便技巧。你肯定应该使用`box-sizing: border-box;`，我喜欢吸引注意力的边框脉冲动画，尽管这个例子相当激进；对大多数网站来说，更微妙的版本可能更好。

##### [](#the-complete-illustrated-flexbox-tutorial)[全图解 Flexbox 教程](https://medium.freecodecamp.org/the-complete-illustrated-flexbox-tutorial-d35c085dbf35)

对于视觉学习者来说，这是一个很好的掌握 Flexbox 一切的方式。这里的图形是惊人的，所以如果你一直在努力想象或理解 flexbox 的任何部分，看看这个。

#### [](#javascript)JavaScript

##### [JavaScript 反应性的最佳解释🎆](https://medium.com/vue-mastery/the-best-explanation-of-javascript-reactivity-fea6112dd80d)

在 Vue.js 中对反应如何起作用的一个很好的解释。我在 VueConf 的谈话形式中看到过这个。这简直太棒了，这篇文章让它更容易理解。也就是说，对于那些喜欢互动演示的人来说，也有视频可供选择。

##### [](#what-are-service-workers-and-how-they-help-improve-performance)[什么是服务人员，他们如何帮助提高绩效](https://www.keycdn.com/blog/service-workers/)

对服务工作者的可靠介绍，不仅介绍了概念，还带您从头开始编写服务工作者。也就是说，你真的不应该从头开始编写它们——谷歌的[服务工作者工具箱](https://developers.google.com/web/tools/workbox/)有即插即用的库，利用最佳实践来实现你想要构建到服务工作者中的各种功能。

##### [](#render-children-in-react-using-fragment-or-array-components)[使用片段或数组组件在 React 中渲染子组件](https://css-tricks.com/render-children-in-react-using-fragment-or-array-components/)

React 16 最大的变化之一是能够拥有多个根的组件。这对组件设计的灵活性来说是巨大的——有些情况下，子组件的集合应该组织在一起，但是 DOM 结构不允许包装它们。列表元素和表格行就是突出的例子。这篇文章讲述了如何在渲染函数中使用`React.Fragment`或数组来实现这一点。关于 Vue.js 中的类似功能，请查看我不久前写的这篇关于 Vue 中的[多根组件的文章。](https://dev.to/kball/creating-multi-root-vuejs-components-23j0)

##### [](#10-javascript-string-methods-you-should-know)[你应该知道的 10 种 JavaScript 字符串方法](https://dev.to/frugencefidel/10-javascript-string-methods-you-should-know-4l76)

在上一期的时事通讯中，我链接了一篇精彩的文章，题为“[你应该知道的 10 个 JavaScript 数组方法](https://dev.to/frugencefidel/10-javascript-array-methods-you-should-know-4lk3)”，突出了现代 JavaScript 数组中现在可用的令人难以置信的功能。同一个作者又回来了，带着一组内置的伟大的字符串功能(甚至可以通过 babel 在旧浏览器上访问)，所以越来越多的你不需要 lodash 或类似的东西就可以做。尽情享受吧！

##### [](#javascript-eating-the-world-desktop-edition)[JavaScript 吃天下，桌面版](https://changelog.com/jsparty/34)

在最新一期的 JSParty 播客中，我有机会与电子团队的一些核心成员交谈。了解更多关于 Electron 作为一个平台，他们如何改进它，以及它在未来的发展方向，这是非常有趣的。在你开车回家的路上绝对值得一听。

#### [](#other-awesomeness)其他牛气

##### [](#uiengine-10)[UIengine 1.0](https://dennisreimann.de/articles/uiengine-1-0.html)

UI 驱动的开发肯定是我听到越来越多的事情，我也听到了关于 [storybook](https://storybook.js.org/) 的非常好的事情，但这仍然是一个工具创新成熟的领域，我很高兴看到一个新的玩家进入这个领域。

##### [](#design-systems-at-github)[在 GitHub 设计系统](https://medium.com/@broccolini/design-systems-at-github-c8e5378d2542)

说到 UI 驱动的开发，这里有一个创建共享设计系统和组件库供整个公司使用的案例研究。我喜欢看到一家公司坦诚地介绍他们的方法以及他们所面临的不同扩展挑战。

##### [](#the-secret-to-connecting-designers-and-developers)[连接设计师和开发者的秘密](https://www.invisionapp.com/blog/connecting-designers-developers/)

另一个关于设计系统的力量的研究，这次来自于 vision，强调了这些工具在公司内部协调设计和开发的力量。请务必点击文章底部的[关于设计系统的电子书](https://www.designbetter.co/design-systems-handbook)(免费，无需电子邮件)!这篇文章是开胃菜，但电子书是主菜，它是🔥🔥🔥

##### [](#building-the-google-photos-web-ui)[构建 Google 相册 Web UI](https://medium.com/google-design/google-photos-45b714dfbed1)

在构建高性能 UI 的掩盖下，这是一个迷人的外观，它可以扩展到成千上万张照片，同时始终感觉敏捷和反应灵敏。许多许多金块的前端性能黄金埋在这里。

##### [](#create-your-own-serverless-api)[创建自己的无服务器 API](https://css-tricks.com/create-your-own-serverless-api/)

这是一个很好的例子，说明了使用微软 Azure 的无服务器功能实现来设置“按需 API 端点”是多么简单。我觉得我的脑袋仍然被“构建应用程序”的方法包裹着，但是当设置一个无限可扩展的端点如此容易时，JAMstack 方法(纯静态前端+API)开始看起来非常有吸引力。
