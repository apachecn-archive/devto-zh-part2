# 了解系统

企业是复杂系统中的一只鸭子

盖尔斯定律，所有有效的复杂系统都是从有效的简单系统进化而来的

流动是系统中输入到输出的运动
库存和闲置应该在价值流中得到有效管理。库存是物品的数量，而闲置是多余的库存。保持较低的松弛度可以降低库存管理成本，但需要较高的协调成本

约束理论认为，任何一个可管理的系统总是被限制在一个至少有一个约束的作弊多的目标上。如果你能识别在缓解约束哦，你就不会增加吞吐量。瓶颈之外的改进是一种幻觉

如果你不想重蹈恐龙的覆辙，你必须时刻考虑环境条件如何影响你的系统选择测试是系统和组织在整个环境中延续的过程。例如，没有获得足够的空气、水或食物的人不能通过选择测试。
如果你能识别市场上的选拔测试到底是什么，你就能在那个市场上更有效地竞争
黑天鹅事件可以在没有通知的情况下戏剧性地改变选拔测试

系统内的流程耦合越紧密，本地故障和延迟就越有可能影响系统的其余部分

正常事故理论，或简称“s*** happens ”,认为在复杂的系统中，某些东西会出错。对正常事故的过度反应会适得其反，当出现问题时，反应会变得过度敏感，封锁一切，并增加更多的控制措施来防止幸运事件再次发生。封锁只会让系统耦合得更紧密

# 分析系统

解构是将复杂系统分解成尽可能小的子系统的过程，以便理解事物是如何工作的
舒适系统可能很难用语言来描述。用图表来表示系统的流量、存量、输入和输出

测量是改进的第一步
被测量的得到管理
Kpi 是系统关键部分的测量
不能帮助你改进系统的测量比没有价值更糟，浪费你有限的注意力和精力
垃圾中的垃圾简单明了的原则是，如果你把无用的输入输入到系统中，你将得到无用的输出。如果你完成后不想要垃圾，就不要从垃圾开始

分析诚实是在精神上代表一个系统的 KP 眼睛，并尽可能恰当地保持最高的准确性
作为一般规则，不要孤立地看待任何测量，始终考虑系统的环境

普通测量、均值模式中值和中间范围
相关性不是因果关系。即使你发现一个测量值与另一个高度相关，这并不意味着它们彼此有直接的影响

改变测量方法，你就脱离了任何历史规范

# 改善系统

零假设，当出现问题时，避免您的偏见添加流程，评估情况，看看是否有必要进行系统范围的更改，或者是否需要立即取消个别变量的最小最大值，以提高系统性能。试图一次处理许多变量会很难区分相关性和因果关系
通过持续改进来重构你的系统。思考精益价值流优化
利用帕累托原则、80-20 法则，这些在向客户营销或设计产品时对你有利的关键要素。
减少摩擦，增加系统自动化。高流量流程中的少量摩擦可能是非常昂贵的周期测试，这些测试可以手动完成，也可以自动完成，但成本也很高，而且无法扩展
使用清单来帮助确保在完成流程的同时达到关键路径
商业世界需要的是更多的海龟，更少的老虎。在商业中，弹性是一种被低估的品质，就像乌龟一样。它可能不性感，但当你需要它的时候，它又活了一天。否则，故障保险的开发就太晚了。在你的关键路径上找到合适的准备量，以防止灾难性的失败

让你内心的恶魔肆意横行，在问题影响到客户之前解决好它们。混沌猴子测试
