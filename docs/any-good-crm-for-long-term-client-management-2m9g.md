嗨，

让我试着解释一个问题，我知道一定有解决方法，但不知道是什么。我希望你能帮我找到它。

在我的工作场所，我们围绕核心产品向每个客户(B2B)提供许多不同的 IT 服务和配置(*与此处解释*无关)。这些额外的功能都是可选的，但如果需要，我们可以提供。

例如，他们中的一些人回答:

*   他们有 FTP 服务器吗，或者我们应该提供一个？
*   他们能自己监控吗，还是我们应该监控？
*   他们有 AWS(任何其他)云帐户，还是我们设置它？
*   应用程序的所有服务器是基于云还是准系统，或者两者都是？
*   任何第三方许可证，是他们买的还是我们买的？
*   等等。
*   等等。

我想象一个系统，我可以去客户端配置文件，并添加一个循环服务，如 *FTP (500GB) (AWS)* 或*监控(基本到电子邮件)*，以及核心产品。这些都是简单的例子，但是还会有更多的细节。

我理想中的工作流程是:

*   销售人员输入新客户/ PoC 详细信息、联系信息(每个客户有多人)
*   然后我们(生产团队)实现一个解决方案，随着时间的推移改变事情
*   会计可以看到标准化的服务清单，并向每个客户发出账单

我检查过的 CRM 是直接面向销售/销售线索的，可以达成交易。再工作，卖个产品，但这完全不是那个。此外，这不是我要寻找的会计系统，许多选项不会产生成本，但仍然需要记住，像哪个(些)云是用于每个客户或简单的时区，他们操作。

建议可以是开放源码软件或付费订阅，也许有一些可用的 API

你跟踪你提供给客户的服务吗？有这方面的系统吗？你有什么建议吗？请帮忙！
