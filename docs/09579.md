### 自我介绍！你是谁？你在哪里工作？

你好😊我叫莎拉，是一名前端工程师，目前在巴黎一家金融科技初创公司工作。我专攻 CSS 和 JavaScript。

### 是谁或什么让你开始编程的？

从我记事起，第一次看到网站就想做一个。我是个书呆子，把东西拆开，用收获的碎片建造新的东西。我一直痴迷于*制造东西*。我记得第一次点击一个超文本链接:我被*吹走了*。所以你可以做一个东西，当你点击它的时候会带你去别的地方？这对我来说就像魔法一样。我*有*知道怎么做。

当我 15 岁的时候，我创建了一个 phpBB 论坛和我的朋友们一起玩。在某个时候，我们决定制作一个播客。我们需要一个网站来托管它，所以我上网查了一下，开始写我的第一个 HTML 页面。这也是 Flash 的黄金时代！我在制作我的第一个业余网站时获得了很多乐趣，这种热情从未离开过我。即使当我不得不为我不是特别喜欢的项目写代码时，我也一直热爱编码。

后来，我很幸运地找到了一份工作，在那里我找到了一位导师。我不能夸大它对我的影响。我希望每一个有抱负的开发人员都能找到能够帮助他们释放能量并迈出软件工程第一步的人。有一个导师对双方来说都是非常有益的经历，我强烈推荐。

### 巴黎的科技场景是怎样的？

它很棒，而且还在不断成长。法国长期以来拥有优秀的工程师，但我们最优秀的人才往往(也可以理解)去了硅谷。然而，许多有前途的初创公司在过去十年中成长起来，我们现在在巴黎有世界级的科技公司。

此外，还有很多聚会和会议。例如，互联网泡沫和互联网泡沫发生在巴黎。此外，在巴黎可以让你快速了解西欧的其他地方:你可以参加在伦敦、柏林、阿姆斯特丹、布鲁塞尔、巴塞罗那举办的活动...

我看到越来越多的外国人来巴黎寻找工程职位，我喜欢这样。多样性是伟大的。我们就是这样做出最好的东西的。

### 前端开发有什么让你兴奋的地方？

哇，我从哪里开始😀

我最喜欢的是能够用很少的一套语言与这么多不同的东西互动。当我进入开发领域时，对 HTML、CSS 和 JavaScript 的了解只帮了你这么多。我最初学习 PHP 是因为我想处理“沉重”的东西。现在完全不同了！

我喜欢逻辑和技术挑战，但我也喜欢设计和用户体验。如果我必须做出选择，那对我来说很难，因为对我来说，它们是紧密相连的。前端让我可以两者兼得，并成为一个充满活力的社区的一部分，不断创造令人惊叹的东西。

### 对设计的兴趣增加了开发人员的什么技能？是不是所有的开发者都应该学习一些设计理念和技巧？

我们是解决问题的人。归根结底，我们的使命是让别人的生活更轻松。UI/UX 设计师直接处理它，而开发人员则更多地“脱离”这一现实。如果你不小心的话，这会让你更关心代码、语言、工具和技术，而不是用户、结果和体验。

开发人员不需要成为设计师，但是熟悉设计原则有助于你考虑用户优先而不是机器优先。我知道很多开发者并不认为自己是设计师。他们认为自己没有“创造性的一面”。那是一个神话。我们整天都在与面向数字和物理用户的界面进行交互。因此，我们都对人体工程学和用户体验非常敏感。拥抱它会带来新的视角。

### 迄今为止，在你的软件生涯中，你学到的最难的一课是什么？

在我职业生涯早期的某个时候，我感觉很舒服，但我落后了。我有一份我处理得很好的日常工作，我认识所有的客户，事情不断重复。我不需要做太多就能快速完成工作，所以我变懒了，我停止了学习。我所做的只是重复我知道的模式。这很有效，客户很满意，我也很有效率，所以为什么要改变呢？

几年后，现实给了我沉重的打击:这个行业发展如此之快，而我却落后了。当你意识到这一点时，感觉一点也不好。我不得不快进，加倍努力才能赶上。因为这个，我错过了很多工作机会。冒名顶替综合症的感觉从未如此强烈。

掌握需要时间才能理解的东西感觉很棒，但这会让你固步自封。科技行业正以令人抓狂的速度前进。当你最终“得到”某样东西时，你很容易觉得你新获得的知识已经过时了。

现在听我说完:我不认为一个人应该*总是*处于事物的顶端，并匆忙学习最新的趋势语言或框架。那实际上是不可能的，你会精疲力尽，但你绝对不能让自己陷入另一个极端。成为一名开发人员是一次迷人的冒险，但是这个行业就是要拥抱变化。

### 对于成功的软件职业生涯，你的第一条建议是什么？

当我想到所有我认为对成功成为一名软件工程师至关重要的特质时，总是归结为好奇心。愿意不断发现，不断学习，不断提高。

有损你职业生涯的行为包括:

过于自负，认为自己是个天才，没有什么可学的了，

懒惰，固步自封，只做自己已经知道的事情，

沟通不畅，不愿意与同事或同行进行更多的讨论，

教条主义，绝对思维，拒绝替代方案和不同方法。

所有这些特征都直接或间接地源于缺乏好奇心。他们经常被冒名顶替综合症所驱使:这个行业可能相当精英化，而软件工程很难。许多开发人员觉得有必要“保护”他们的知识。我的建议来自两方面:

初学者和初级开发人员，和更有经验的同行在一起。抵制成为房间里最聪明的人的冲动，不要被虚假的头衔所迷惑。找到重视职业成长的公司，成为学长们想要辅导的那种开发者:好奇、热情、专注、豁达、感恩。

高级开发人员，接受你职位中的指导部分。成为大四学生不仅仅意味着更高的薪水和远离繁重的工作。如果你的公司不鼓励导师，或者在高风险的任务上过于依赖你，占用了你所有的时间，提醒他们教学的好处:工程团队内部更高的信任度，初级员工增长技能，以及短期、中期和长期的更好速度。

### 工作之外，你有什么爱好吗？你认为它们对你的科技事业有任何帮助吗？

软件开发是脑力劳动。如果你不停止看你的屏幕，如果你不通过做其他事情来充电，你筋疲力尽只是时间问题。而且那是时的*，不是*时的*。*

我喜欢任何与*制作*有关的东西；创意出路。当我不编码时，我主要写(在我的博客上)和画。多年来，我一直在研究字体和字体设计。我也喜欢看像版画一样的黑白交叉影线画，多年来一直很欣赏。我喜欢在空闲时间练习这个。
