# 我写了本书

> 原文:[https://dev.to/trickvi/i-wrote-a-book-lfg](https://dev.to/trickvi/i-wrote-a-book-lfg)

是的。我写了一本[书](https://www.manning.com/books/the-art-of-data-usability)。它还没有印出来，但既然我已经把它寄给了我的出版商，我就有时间消磨了。所以我想:为什么不反思一下我花了一年半的时间写这本书，并分享我的经验。如果你想写一本书，你也许可以了解这一旅程是什么样的。

这是背景故事。2016 年 9 月的一天，我收到一封电子邮件，对方自称是[曼宁出版社](https://www.manning.com/)的助理采购编辑。根据电子邮件，他们有兴趣出版一本关于数据质量的书，并想知道我是否有兴趣与他们交谈。我的第一个想法是:*“电子邮件骗局”*，然后经过一番思考，我决定回复，一年半后在一本完整的书中结束。

# 面向用户的图书写作

曼宁出版公司有一个非常好的写书方法。我不能和其他出版商相比，因为这是我的第一本书，也是我第一次尝试写作(除了我的博士论文)。曼宁的方法是非常面向用户的，整个过程旨在输出一本书，适合所有类型的读者。

在与曼宁初次通话后，我要做的第一件事就是定义我的目标受众(数据科学家)以及我想在书中教给这些受众的内容。我审视了这本书，权衡了每一章的难度，看看这本书是否真的适合我的读者。这是我的书的一种需求分析。当我写这本书的时候，我充实了每一章的内容，重新权衡了每一章，并且不断地将我所有的写作目标对准预定的读者。

在写作过程中，由我的目标读者组成的焦点小组定期审查这本书。这个小组给了我非常有价值的反馈，让我能够更好地将我的书集中在他们认为有帮助的地方，避免任何混乱或无聊的部分。这常常令人沮丧，因为在我找到教授数据质量的最佳方法之前，我不得不重新思考、重新构建和重写我所有的章节至少两次，而不受预期受众的先入之见的限制。

然后，一旦我把我的书的大部分章节发给了曼宁，我的书就进入了一个叫做*曼宁早期访问计划*或者简称为 MEAP 的项目。该计划发布了这本书的早期版本，并允许公众提供反馈，帮助我为所有可能的读者进一步改进这本书。作为 MEAP 的一部分，曼宁建立了一个图书论坛，我可以在那里与读者交流，并相应地修改这本书，但我发现在论坛之外与读者私下交谈最有帮助。

曼宁为我提供了一名开发编辑和一名技术编辑，他们帮助我构思这本书，并让我走上正轨。当我将曼宁的学习系统内化时，我的开发编辑非常棒，也很有耐心——他们发现的方法有助于人们最好地学习。这样，我就能根据读者阅读我的书的方式来调整我的风格。我认为我写作和内容结构的转变是最难克服的事情。我以前写过很多文字，但不知不觉我只为我自己和像我这样的人写。显然，我不是每个人，曼宁有办法让这本书适合更广泛的读者。

例如，我不喜欢在每一章重复解释，但我没有意识到的是，当我从第一页读到最后一页时。其他人*根据他们感兴趣的内容，以随机的顺序进入章节。如果我从来没有重复过我在前面章节中说过的话，那些读者就会错过上下文，得不到同样的体验。所以诀窍是这样写，使这本书对两个群体都是可接近的。*

鉴于我在数据质量和可用性方面的经验，除了了解用户需求的软件开发之外，了解用户需求也是非常重要的，承认这一点对我来说有多难几乎是令人尴尬的。

# 平衡工作、生活和写作

写书过程中第二个最困难的部分是让写书融入我的生活。我是两个女儿的父亲。当我开始写这本书的时候，我的两个女儿分别是 2 岁和 4 岁。我的妻子正在攻读博士学位，最近开始了一份全职工作。我的全职工作是冰岛统计局的信息技术和传播主管。我们还计划了 2017 年的暑假旅行。除了我决定要做的疯狂的事情之外，我还必须平衡我所有的其他兴趣，比如买一套新公寓并搬家，面对硬盘故障并从备份恢复到新的笔记本电脑，参加我女儿幼儿园的家长协会，在工作中进行重大战略转移等等。回想起来，我不知道我在想什么。

那么我是怎么写这本书的呢？首先:你知道作者倾向于感谢他们的家人和配偶的支持。我以新的眼光看待那件事。这不是他们出于对配偶的礼貌而做的事情。如果你有家庭，你需要一个支持你的配偶。我们必须计划如何分配周末，谁早上上班，谁下午上班(正如我说过的，我妻子正在兼职完成她的博士学位)。我们的家庭生活受到了影响。我们经常讨论连续几天见不到对方是多么糟糕。如果我们中的一个去幼儿园接女儿，做饭，给她们读书，哄她们睡觉；另一个人坐在咖啡店里用笔记本电脑打字。

咖啡店很棒。他们变得宾至如归。一个我可以集中注意力的地方。如果我在家，我很容易被打扰，这使我无法集中注意力。在咖啡店里，我很少被打扰，而且能集中注意力好几个小时。我特别喜欢一家营业到很晚的咖啡店，所以我白天喝咖啡，晚上喝啤酒。我在很多不同的地方写了这本书，这让我在修改这本书的时候一直保持着兴趣。

我发现最合适的写作节奏是花一两个星期去进一步研究我要写的章节；寻找有趣的故事，挖掘文献，材料，为章节和其他准备工作创建示例程序。然后用一个周末的时间来敲打周围的单词。然后，我把第一份草稿发给了我的优秀的开发编辑，他读了这份草稿，并帮助我完成了供用户审阅的章节，并做了大量的来回编辑和评论。这种节奏并不总是奏效-有些章节比其他章节更难计划和编写，所以他们需要更多的准备-但这是对我和我的家人最有效的安排。

# Git 统计

我所有的书都是用 Asciidoc 格式写的，这是一种基于排版文本的写书格式。我使用 Git 对这本书进行版本控制，并将其推送到曼宁托管的 Gitlab 服务器上。看看 Git 统计数据和 Gitlab 图表，就知道我是怎么写这本书的了。

纵观整个 git 库，我修改了 1208 个文件，插入了 113137 行，删除了 42037 行(一行不超过 80 个字符)。将这些插入和删除放入上下文中，这本书的主要章节总共有 9347 行(70077 个单词)。有些插入和删除是我为这本书创建并保存在 Git 中的 svg 图表，但很明显，大量的工作是重写、删除和添加文本。

如果我们看看按工作日划分的提交，那么在周末编写大部分内容的设置就很清楚了。我在周日做的 157 次提交中的大多数(总共 40 次)。这些是我在准备了一周或更长时间后敲定的大文本块。星期四的提交由两种类型的提交组成，一种是我在周末之前完成章节或章节结构的代码，另一种是我在从我的编辑器获得编辑后进行修改。

# 值得吗？

哦，是的，它是。即使我工作到很晚，在我的家人睡觉后，牺牲课外活动，我学到了很多，不仅是关于如何教学和写作，也是关于我的专业领域。我必须想出创造性的解决方案，使我的例子简单到足以在书中有用。

我对结果非常满意。这本书还在 MEAP(现在当我写这些文字的时候)，所以它还没有被印刷，但是所有的章节都已经出版了，它正在通过最后的审查。如果你感兴趣，你可以在曼宁的网站上购买- [数据可用性的艺术](https://www.manning.com/books/the-art-of-data-usability)(电子书现已上市，印刷书籍/pbook 一旦出版)。

如果我能够同时处理我生活中发生的所有事情，你也可以！因此，如果你肚子里有一本书的想法，没有什么可以阻止你开始(尽管我建议你的生活中发生的事情比我少——这样你更有可能保持理智)。

我希望这篇文章是鼓舞人心的，即使它主要是关于我的。我非常乐意回答你在评论中可能有的任何问题，以帮助和鼓励你把书写出来。
