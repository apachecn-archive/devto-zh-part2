# 我的 web 开发人员职业故事

> 原文:[https://dev . to/teraouralinna/my-web-developer-career-story-463 a](https://dev.to/teroauralinna/my-web-developer-career-story-463a)

我从 2005 年开始从事 web 行业，做前端和后端开发。我在 90 年代创建了我的第一个网站。这里有一个故事，讲述了我是如何陷入网络的，以及我是如何最终成为一名网络开发人员的。有三件大事为我的职业生涯铺平了道路。

## [](#the-beginning)开头

一切都始于 90 年代我十几岁的时候。我们和几个朋友开始创建网站来赚些零花钱。主要的想法是让大量的访问者点击广告并获得报酬。在那个时候，横幅广告的付费模式大多是基于点击的，所以如果你的网站设法吸引访问者是很容易的。那些是黑帽 SEO 策略的好时机。例如，你可以在页面底部使用一个巨大的关键字列表，与主体背景颜色相同。此外，网站链接目录仍然被大量使用。有一个芬兰的目录服务，在他们的首页上有一个按字母排序的新网站列表。所以很容易爬到顶端。

我还创建了一些乐队粉丝网站，并为我们的 QuakeWorld 和 Quake III Arena 战队维护网站。我使用像 Frontpage 或 Dreamweaver 这样的 WYSIWYG 编辑器进行开发。但是我总是觉得手工创建代码是正确的方法，可以得到更好的代码。让第一个自己的网站在互联网上公开可见是令人兴奋的。作为一个彻头彻尾的新手，输入一个地址，然后在网络浏览器上看到一些自制的东西，这种感觉既奇怪又迷人。

布局大多是用表格和图像构建的。这个 CSS 主要用于字体颜色和其他类似的非常简单的东西。

我编写了简单的 JavaScripts(主要是复制粘贴),并使用了用 Perl 编写的现成 CGI 脚本。还记得 FormMail.pl 吗？

我在我的网络连接提供商的虚拟主机上托管我的站点。有一次，网络提供商打电话给我，因为我的网站产生了太多来自芬兰以外的流量。那时候太贵了。我需要暂时关闭网站。

在某个时候，我们梦想有自己的公司制作网站，但我们从未真正开始它。我们有一个网站，我们甚至得到了一个真正的 RFP，但我们从来没有回答，因为我们不确定我们是否真的可以提供解决方案。

随着时间的流逝，很多年来，我并没有过多地考虑网站，但种子就在那里。

## [](#the-rise-of-the-programmer)程序员的崛起

我职业生涯的第二件大事是在职业学校学习视听传播。我最初申请了摄影，但最终我进入了多媒体领域。从职业角度来看，这可能是一个非常棒的决定。那时候感觉并不是很好，因为我梦想成为一名摄影师或者比电脑呆子更酷的职业。尽管我一直把摄影作为一种爱好。

我们学习了如何制作多媒体 CD-r om、DVD、Flash 动画和其他当时很流行的多媒体和网络素材。

然后我们有一门关于 PHP 编程的课程。老师是来自外部公司的客座明星，这真是一门很棒的课程。我已经知道一些 HTML 和 CSS，但我不知道如何使网站充满活力。在那门课之后，我开始真正学习编程，很快我就发现一切皆有可能。

此外，实习阶段的学习真的很有用，因为我从真实的项目中得到了大量的实践训练。实习期也给我带来了几个自由职业者的项目。

作为毕业作品，我用 PHP 创建了一个简单的网上商店。在书面部分，我集中在一个安全的网络编程和我如何使我的网上商店安全。我不记得这种安全感是从哪里来的，但它真的很重要，也很自然。我一直觉得有趣的另一件事是前端和后端的性能优化。

职业学校毕业后，我得到了第一份真正的网络开发工作，在一家摩托车商店做站长。我管理和增强他们的电子商务系统，并维护其他网站。

## [](#enhancing-skills-with-the-personal-project)用个人项目提升技能

对我的职业和学习来说，第三件真正重要的事情是我和几个朋友一起开发的名为 Kuvablogi.com 的个人项目。Kuvablogi 是芬兰语，意思是照片博客。

Kuvablogi.com 是一个芬兰的照片分享服务和社区，成员们可以上传、整理和分享他们的照片。还有博客和地理标记功能。有 **2 700 000** 张上传的图片和 **34 000** 个注册会员。当 Kuvablogi.com 关闭时，图片被浏览和下载超过 **246 530 000 次**。每周大约有 50 000 个独立访客和 364 000 个页面浏览量。

当我在职业学校的时候，我已经为自己实现了一个简单的图库。我开始为多用户服务改进它。当我们注意到第一批用户找到了 Kuvablogi.com 并注册了用户账户时，这是另一个令人惊叹的时刻。

我的职责是前端和后端开发。Kuvablogi.com 建在灯堆上。代码库实际上是一个可怕的东西，最终所有的东西都需要重写。这基本上是我们第一个更大的编程项目，我们没有使用任何框架来指导我们的应用程序结构或编程风格。当我们开始的时候，合适的 PHP 框架可能还不存在，或者我们不知道它们。我们也错过了设计技巧，所以用户界面和 UX 是一个相当大杂烩。

我们使用现在已经退役的 JS 库，比如[原型](http://prototypejs.org)和[script . aculo . us-web 2.0 Javascript](https://script.aculo.us)。

托管服务是一个持续的问题。带宽和存储需求巨大，而那时服务器托管并不便宜。我们有一些有趣的方法来扩展服务，并将映像分发到更便宜的硬件上。通过不断优化我们的代码库，我们能够用一个适中的硬件托管相当高流量的网站。即使代码库一片混乱，代码仍然运行得很好，因为没有复杂的层，我们可以非常容易地优化一切。

最终，一个腐烂的代码库、上涨的成本和其他生活活动占据了我们的空闲时间，Kuvablogi.com 无人维护。此外，视频开始变得越来越受欢迎，因此它会对存储容量产生指数级影响。整个服务后来完全关闭，因为我们认为这是最好的用户使用其他更活跃的服务。

## [](#afterword)后记

现在在 [Ambientia](https://www.ambientia.fi) 做前端开发。我大部分时间都是在客户的场所为出色的项目担任顾问，这也是一个了解更多和获得不同技术堆栈知识的好机会。

我的博客也是我最近作为一名专业网络开发人员成长的另一个步骤。

看到网络如何发展成为一个成熟的平台是一种荣幸。尽管我有时有点嫉妒年轻的 web 开发人员，他们可以从更成熟的标准和技术开始，而没有过去的负担。举几个历史上的例子，我真的不怀念 WAP 页面、IE6 黑客、Lotus Notes 的 HTML 时事通讯布局调整、PHP4、表格布局和创建带有图像的圆形按钮。
