# 2018 年网页设计

> 原文:[https://dev.to/bitersen/web-design-in-2018-456 f](https://dev.to/bitersen/web-design-in-2018--456f)

关于什么是网页设计，总是有很多困惑，尤其是在自学成才的设计师工作的圈子里。今天，这个术语很少使用，因为很少有其他更专注的职业从中涌现出来。这是完全可以理解的，因为流程比过去复杂得多，但你可以肯定这个职业仍然存在。

[![Web design](../Images/ad6fcc7c1fddc4bd7fbb6f31bed3bf7d.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--jNM4O7MV--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/pwgbz5do5p62uekvq2ys.png)

那么，今天的**网页设计**代表了什么？

关于网页设计师应该做什么，有两种不同的(业余的)观点。第一组人会告诉你，网页设计师是只专注于设计的专业人士，他们不应该有编程技能。第二组人会告诉你网页设计师也必须知道如何编码。

他们都错了，因为这个清单还远未完成。

作为一名毕业的平面设计师，作为一名拥有近 20 年专业经验的网页设计师和前端开发人员，我将尝试扩展网页设计的定义。这比人们想象的要多得多。

简而言之，网页设计是应用艺术的另一个分支，除了著名的平面设计和工业设计。无论你在哪里看到“应用”和“艺术”这两个词联系在一起，这意味着这个职业也涉及到很多工程技术。

例如，工业设计师必须了解创造闪亮新设备的材料和工艺流程，而平面设计师如果想为书籍、海报等创造美丽的设计，就必须了解印前、色彩和印刷材料。

和上面提到的两个职业一样，网页设计师必须知道如何设计、编码和交付他们想象的东西。

## [](#common-traits-of-design-professions-reflected-in-web-design)设计职业的共性在网页设计中的体现

所有的应用设计师在他们的职业中都有一些共同的特点。这些特征被分成 4 个主要的组，当它们结合在一起时，就构成了一个专门的设计职业。它们如下:

1.  美学
2.  技术
3.  工效学
4.  价值

### [](#esthetics)美学

这就是大多数人认为的设计——排版、色彩理论、构图、插图、视觉化。美学必须在视觉上吸引目标受众。根据你的喜好，给你足够的时间，你会形成自己的风格。为你创造的每个新产品开发独特的风格是非常重要的，因为你想让它在同类产品的海洋中脱颖而出。

不要认为使用其他设计师的成品风格会让你成为一名优秀的设计师。在这里，我特别考虑使用成品风格，比如 Twitter 的 Bootstrap、Google 的 Material design、微软的 Fluent design 系统等等。这些都是为这些公司设计的非常好的设计系统。他们提高了自己品牌的知名度，但不是你自己的品牌。请记住，当您尝试使用一些预制框架时。如果你有预算，就自己创造吧。

这本身就是一门完整的科学，这个领域非常广阔。例如，一些设计师可能会发现字体设计对他们很有吸引力，因此他们可能会选择专攻那个特定的主题，而其他人可能会发现专攻设计系统对他们来说更好。

优秀设计师应该对这个专业领域的所有重要课题有全面的了解。网页设计师的工作是关注那些对创建用户界面和向最终用户传递信息的最佳方式很重要的领域。

大多数设计师在 Photoshop 和 Illustrator 等程序中完成他们创作过程的这一部分，但根据职业的不同，工业设计师可能会使用 CAD 或 3D MAX 等 3D 程序，而网页设计师可能会使用 Affinity Designer、Sketch 或类似的程序。当然，没有人阻止他们在创作过程中混合使用这些程序，甚至使用更传统的方法，如笔和纸。

### [](#technology)技术

这一部分也非常重要，它让设计师在他们选择的领域成为真正的专业人士。正如我所提到的，这些设计领域中的每一个在最终产品和用于创造它的技术方面都是不同的。因为这篇文章是关于网页设计的，所以我将只关注这个领域。

对于一个网页设计师来说，了解他正在创造的新产品的技术是很重要的。在这种情况下，这是设计呈现的媒介和用于驱动我们产品的技术的知识——网络或移动应用。更具体地说，**网页设计者正在用网络技术创造他们的产品，这些技术使用不同的屏幕和设备作为媒介向最终用户传递信息。**

设计将如何适应不同的屏幕尺寸、像素密度，以及在极少数情况下有限的调色板和屏幕刷新率？网页设计师将如何准备他的创作？在不同的网络上，在较慢或较旧的设备上，这种创作会有怎样的表现？

因为我们创造了大部分产品，我们必须知道所有这些问题的答案。这就是为什么深入研究 HTML、CSS、JavaScript、web 协议、响应式 web 设计原则、设备功能等技术非常重要的原因。当然，如果你对这个极其广阔的领域感兴趣，并且把它作为你的重点，你可能会在前端开发中结束。

对于一个网站设计者来说，了解足够多的知识来创建至少一个具有功能性、响应性和性能界面的静态网站是很重要的。除此之外的一切都是专门的前端和后端开发人员的工作。

不要以为你的工作到此为止，我们还有两个更重要的领域；)

### [](#ergonomy)人机工程学

人机工程学的定义:*旨在通过最小化操作员疲劳和不适来最大化生产率的设计因素*。

当你设计你的应用程序时，它必须是直观的和易于使用的。用户应该能够用最少的时间、最少的努力找到他们需要的信息。

这听起来可能很容易，但实际上并不容易，因为网页设计师必须考虑不同的设计和技术因素，然后将它们与用户心理、习得的用户行为、从分析中提取的数据、A/B 测试和残疾等限制因素重叠。最终结果必须是大多数用户可以毫不费力地使用应用程序。

你可能也会发现这个领域的设计很吸引人，所以你可能会专攻 UX(用户体验)设计，但是作为一个网页设计师，你必须对这个领域有一个大致的了解。

如果美学和技术被用来设计你的应用程序的外观，工效学将是设计你的应用程序的灵魂。

### [](#value)值

不要把价值误认为价格。虽然我们产品的最终目标是盈利，但它必须为最终用户提供一些好处。它可以提供纯粹的知识，它可以用于娱乐或作为工具，但利益必须存在。否则，我们创造的只是纯粹的艺术，这不是我们作为网页设计师应该关注的。

创造价值对于任何产品来说都是最难实现的，因为你必须有一个好的想法和实现它的计划。如果你正在为你的客户创建网页设计，他们已经提供了他们的想法，但是如何使新的应用程序对用户有益和有吸引力的计划是由你来制定的。

所有提到的三个因素——美学、技术和工效学也增加了产品的价值。这就是为什么为了创造一个完整的、令人满意的产品，在这四个方面都做得好是很重要的。

## [](#web-design-vs-ui-ux-and-frontend)网页设计 vs UI、UX 和前端

与几年前相比，今天的网页设计不再被提及是有原因的:随着我们所使用的技术越来越复杂，以及它们的生命周期越来越短，保留这个专业头衔变得越来越困难。对于大多数人来说，成为一名网页设计师所必须掌握的大量主题是非常可怕的。

今天，我可以很容易地说，这个职业所需的知识量相当于 5 年的大学教育。这就是为什么我们有许多相对较小的专业从网页设计中涌现出来的原因:UI 设计、UX 设计和前端开发。

当我将这些职业与我在本文中提到的常见设计特征进行比较时，它看起来像这样:

[![Main differences between UI design and development professions](../Images/79cc4208b58b954e9fcfce723eb8c1ab.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--ABXhxS1L--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ln85f25fjujjjokzd927.png)

前端开发有不同的标记，因为与网页设计或 UI 开发相比，它是一个更广泛的开发领域。

为了更清楚，我将在 web 开发职业之间再做一个比较。

[![Main differences between web development professions](../Images/fb3e39e1369992574f4e0080d1e5f0c0.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--ZLrqK8ZS--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/063lf7rvbe3o4ubvmj9j.png)

当涉及到前端开发时，情况对大多数人来说是相当复杂和令人困惑的。我们有三个主要的专业参与前端开发:

1.  UI 开发人员
2.  网页设计师
3.  JavaScript 开发人员

主要区别在于他们工作的领域。

UI 开发人员设计和开发用户界面，但他们不像**网页设计师**那样参与产品设计的其他方面。这些职位的 JavaScript 要求与专门的 JavaScript 开发人员不在同一水平上，他们主要与动画和用户界面开发有关。当然，有一些网页设计师已经过渡到了现代的 JavaScript 框架和 ES6/7，但是这更多的是一个例外而不是规则。

另一方面，我们有 JavaScript 开发人员，他们主要使用这种语言，但根本不参与设计。他们是纯粹的程序员，设计任何东西都不在他们的工作范围内。他们最关注的领域是高级 JavaScript 框架、单页面应用程序、前端模板、高级性能优化等。

作为最后一个“职业”我们有独角兽。:)

我确信今天有这样的人存在，他们可以自己创造一切，但随着时间的推移和每个职业复杂性的增加，成为独角兽所需的智商和时间要求逐年提高。继续做网页设计师或全栈开发者越来越难，更别说独角兽开发者了。这就是为什么这些职业如今很少见的主要原因，也难怪——那些试图从事这些职业的人越来越受到发展分析的影响；)

...

我们，网页设计师还存在吗？我们当然知道，但我担心这个职业很快会变得像全栈或独角兽开发者一样神秘。当然，除非我们创建了 5 年制的大学，在那里我们会教新生如何成为应用网络艺术家。

这是一个值得追求的目标。)
