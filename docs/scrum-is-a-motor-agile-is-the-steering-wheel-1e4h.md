# Scrum 是你的软件项目的“发动机”，但是谁在掌舵呢？

> 原文：<https://dev.to/jaymeedwards/scrum-is-a-motor-agile-is-the-steering-wheel-1e4h>

当谈到 scrum(或看板)和敏捷开发之间的区别时，汽车的发动机和方向盘是一个有用的类比。

当你的团队在编写需求、编写代码、测试和部署你的软件版本时- *他们可能会遵循像 scrum 或看板这样的开发过程来完成工作*。

这项工作可以被认为是汽车的发动机，推动软件产品向前发展，改变或增加新的功能。

然而，软件公司在不断变化的市场中销售产品- *竞争对手可以迫使我们改变，客户可以迫使我们改变，我们自己的团队成员可以发现迫使我们改变的信息*。

这时，软件团队的领导层尤其是**需要信任他们的其他团队成员来做出改变**。

每次团队做出这些变化中的一个来响应计划外的事件，他们都是敏捷的。

在软件开发中保持敏捷就像使用汽车的方向盘一样。

当事情发生变化时，功能的方向需要改变，这样引擎的“动力”就会集中在不同的东西上。

在软件开发中，我们通过将新产品待定项(或看板票)放在高优先级来驾驶汽车，以使满足我们的客户变得重要。

当我们已经构建了我们认为他们想要的所有东西时，客户不会购买软件产品。

在这个视频中，我分享了一些关于这个类比的想法，以及下次你在做软件开发过程决策时如何用它来问这个问题:

“这种做法会有所帮助，还是会损害我们团队‘引导’工作方向的能力？”
