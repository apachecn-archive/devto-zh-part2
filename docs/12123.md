我 16 年前就开始学习编程，现在我每天都在学习。顺便说一下，我改进了我的学习方法。在这里，我将教你如何是我的过程后，多年的编程和学习了许多东西。

对我来说，在你想学的领域里有三个专业水平:

*   **基本流程和概念**，为此我推荐视频
*   **中级**，为此我推荐文字教程
*   高级水平，我推荐该领域专家的技术文档和博客文章

# [](#videos)视频🎥

当你不知道如何或在屏幕上点击哪里时，视频真的很好。当我不知道如何使用 IDE 或特定工具的工作流程等基本操作时，我会看视频教程。在这些情况下，在视频中看到的东西是有见地的。

# [](#textual-tutorials)文字教程📚

如果你已经学会了基础知识，你应该阅读并遵循文本教程。主要优点是看教程比看视频快。你也可以复制粘贴示例代码，自己尝试一下。

# [](#technical-documentation-and-expert-blog-posts%C2%A0)技术文档和专家博客文章📎

成为更好的程序员的最佳技巧是 IMO:

> *深入了解系统*。安德烈·斯塔尔茨

在试验并应用了你在文本教程中学到的知识后，你可以继续阅读技术文档，这将使你在该领域更深入。

此外，你应该看看专家博客，它们可以帮助你了解**高级概念和最佳实践**。跟着他们，不断搜索新的文章。

还有一个额外的步骤…

# [](#iterate-over-and-over%C2%A0)反复迭代➿

一旦你在某个领域有了更高的水平，就开始看你觉得不舒服的特定事物或**补充话题，并开始看视频、阅读文本教程等等。**

外面有许多令人惊奇的知识，如果你有 T2 的好奇心和 T4 的纪律性，你会学到很多。所以我鼓励你现在就开始学习你想要的东西！在你的屏幕后面，一个充满洞察力和敬畏的天堂正等着你。
