# 讨论——在面试中，你会问开发人员哪些行为问题？

> 原文:[https://dev.to/samjarman/discuss-你会在面试中问开发者什么行为问题- lmb](https://dev.to/samjarman/discuss---what-behavioural-questions-would-you-ask-developers-in-an-interview--lmb)

你会问些什么问题来确保你要雇佣的人是一个团队成员，和你想法相似，但挑战你，并帮助你、你的代码和你的团队改进？

#### 一些语句让球滚动起来

我想避免“文化契合”——这往往会变成工作中的回音室。我想雇用与我有相似价值观的人，但不一定是与我有相似实践和流程的人。我总是很兴奋地想知道我可以作为一个团队更快地前进或者更聪明地工作的方法。我要“文化加”。

我想寻找并雇佣团队成员，那些接受并宣扬团队决策的人，即使这不是他们投票/想要的方式。

我想避开那些保守的、不接受反馈或批评的开发人员，他们通常都固步自封。

我希望有人积极地将过程视为一种工具，而不是一个障碍。

我想要一个能教导别人和被教导的人。

#### 这里列出了最初的想法/问题

1.  告诉我一次你不同意团队的时候，发生了什么？
2.  告诉我你向你的团队推销一个想法的时候，你是怎么做的？
3.  告诉我你使用过什么样的开发方法，你在哪里看到了价值
4.  您熟悉代码审查吗？[如果是这样的话]您认为它的价值在哪里？
5.  告诉我你指导一个比你资历浅的工程师的一次经历。学习？
6.  告诉我一次你被一个比你级别高的工程师指导的经历。学习？
7.  你管理过团队吗，如果有，你学到了什么？
8.  您如何跟上现代工具、流程和技术的发展？

你有什么问题，为什么？拜托，我很想听听！
