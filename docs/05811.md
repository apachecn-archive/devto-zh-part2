# 利用特雷罗来优先安排我的时间

> 原文:[https://dev . to/matmooredev/using-trello-to-priorize-my-time-n3i](https://dev.to/matmooredev/using-trello-to-prioritise-my-time-n3i)

我是特雷罗的超级粉丝。在工作中，我的团队用它来管理我们积压的工作，但我也经常创建自己的 trello 板。

当我开始使用 trello 时，我认为它很简单，但我逐渐意识到它的简单性是它如此伟大的原因。简单灵活的工具。

现在我一直在使用它。这里有一些我用来帮助我更有效率的事情。

## 跟踪个人任务

在我现在工作的地方，我对自己的工作有一定程度的控制，但我也有很多不属于我团队积压工作的时间需求，这迫使我在优先处理我需要做的事情方面做得更好。

人们有不同的方法来跟踪他们需要完成的事情，但我喜欢用一个个人的 trello 板。只要有一个永久的地方来跟踪事情，就能帮助我抵制在每一个新请求到来时跳到它上面的冲动，这样我就可以最小化我正在做的上下文切换的数量。

确定优先级的一个很好的方法是艾森豪威尔矩阵。你把事情分为重要/不重要和紧急/不紧急，所以有 4 个象限。

在 trello 中，你可以用 4 列来完成，或者你可以创建两列，并使用卡片的垂直位置来指示事情的相对紧急程度。

提示:你可以使用分隔卡来分隔一列中的内容。这只是一张空白卡片，上面有一个图案作为封面图像。

[![A divider card](../Images/9b25abe344c9e2982cfd4327744b8607.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--pej9tTO7--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ilonl7y5ajbf30dh9cot.png)

拥有 4 个象限的目的是鼓励你以不同的方式应对不同类型的任务:

*   如果紧急重要，就去做。
*   如果它很重要但不紧急，为它安排时间
*   如果不重要但很紧急，看看其他人能不能做或者让它不那么紧急
*   如果不重要，不紧急，就不要做

这个想法是把大部分时间花在重要的事情上，而不是紧急的事情上，这样你就不会为了赶时间而四处奔波。

我有时会通过在谷歌日历中与自己预约会议来安排事情，这可以提醒我这样做，也可以告诉我的同事我正在忙着做一些事情。如果这可能与其他优先事项冲突，那么我可能会先征求我的经理的意见。

如果我正在做的事情在我的团队板上被跟踪，我通常不会将其添加到我的个人信息中，因为我通常一次只做一张票，我不会忘记这一点。我的个人留言板是用来写其他东西的，比如我需要与之交流的人，无聊的管理任务，或者我在直接团队之外所做的事情。

这种格式工作得很好，尽管我有时会弄乱它。我最近为“我与经理达成一致的事情”增加了一个专栏，我用它来写下我收到的反馈，或者我想思考的事情。

## 学习目标特雷罗板

我最近为自己创建的另一个 trello 板是学习 Java 的计划。之前做过一些 Java，但是没有太多的专业经验，知识差距巨大。我目前在一个使用 Java 的团队中工作，所以这是我想解决的问题。为了做到这一点，我留出了一些时间来学习，我创建了这个板来帮助我最好地利用这些时间。

[![Learning Java board](../Images/3990e2e34ff6cdee14f9871d3bd35f8b.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--40FMBBoO--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/3mnlo29a0byq4ala3t9z.png)

我从识别一堆我能接触到的资源开始，并准确地确定我在努力做什么。这帮助我将所有这些事情简化为我实际上有时间做的少量任务。

到目前为止，我只花了几个小时，但我觉得我现在有了明确的目标和实现这些目标的计划，所以在接下来的一个月左右，我会继续使用这个板来完成这个任务。
