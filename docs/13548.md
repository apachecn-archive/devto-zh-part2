# 教与学

> 原文:[https://dev.to/wiredferret/teaching-and-learning-4idl](https://dev.to/wiredferret/teaching-and-learning-4idl)

在 launch crystally，我们一直在非常积极地招聘员工，因为我们在做令人惊叹的事情。(来和我一起工作吧！)

这意味着我们也在进行一系列培训，我一直在为我们的非技术员工进行培训，比如营销、设计和对外销售人员。他们对我们的成功至关重要，他们拥有我所没有的大量领域知识，但是如果他们知道他们在销售什么，我们将会更加成功地合作。此外，自私地说，越多的人可以在会议上帮忙，每个人在会议上工作就越容易。

我已经做了 20 年的技术工作，涉及各个层面，包括从硬件到云。每次我开始一项新的工作，我都会带来大量的领域知识。但是对于这次培训，我们需要对软件生命周期进行足够的解释，以便能够谈论 launch crystally 解决的问题。这给了我一个真正深入研究的机会，并弄清楚如何以一种对那些没有意外饮用 devops 消防水管的人有用的方式解释这些基本知识。

*   开发者想做什么？他们的工作生活是什么样的？
*   什么是部署，有多难？
*   软件如何到达用户手中？
*   什么是瀑布和敏捷，这和我们有什么关系？
*   什么是 API、SDK、cdn、CI/CD、devops、特性标志、轮询和流更新？
*   什么是整体服务和微服务？
*   为什么不是每个人都构建自己的软件呢？
*   什么是软件即服务？

令人满意的是，我意识到我可以用简明易懂的术语来定义和描述所有这些。这和我写一份真正可靠的文件一样令人兴奋。我知道我是在帮助真实的人类，通过描述他们此刻需要的东西。

我没有想过内部培训是我工作职责的一部分，但当我们意识到我们需要它时，它似乎是合乎逻辑的，而且很容易做到。毕竟我的整个目的就是给人解释产品，给我的公司理解和解释人的痛点。这是很自然的搭配。另外，它非常有趣！

在我无限的空闲时间里，我在考虑用几张幻灯片记录这些解释，这样当人们想查阅时，我们就有了一点概念。
