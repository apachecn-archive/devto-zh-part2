# 我为什么转用 Linux，我现在在用什么

> 原文：<https://dev.to/theminshew/why-i-switched-to-linux-and-what-im-using-today--58e1>

几个月前，我艰难地切换到在我的工作笔记本电脑上运行 Linux，并且一次也没有回头。

我从 3.1 开始就是 windows 用户，从未热衷于苹果产品，我喜欢苹果的产品，偶尔会用我妻子的 Macbook，但大部分时间我都在使用 windows，因为我是一个游戏玩家(不想挑起一场操作系统战争)。我喜欢 Windows，对它很熟悉，但并不真的想要改变。在与 windows 并行运行的过程中，我偶尔会摆弄一下 Linux，但由于我并不真的喜欢编程，而总是喜欢游戏(我为在《国王的任务》和《横扫千军》上浪费了青春而感到羞耻)，因此我从未真正尝试过。

去年，我抱着从电子商务数据分析和项目管理转向开发的想法开始编程，并开始自学在 windows 10 上编程。我喜欢 windows 10，并把它作为我的非工作操作系统，但当我想安装或运行时，我总是会打嗝。在 Windows 上安装 c 是一件很麻烦的事情，无论我写什么代码，总感觉在我和代码之间有一层额外的东西。编码已经够难了，我开始感到沮丧。

去年早些时候，我一直在研究 Zed Shaw 的《艰难地学习 Python》。他推荐我学习命令行，并有一个快速入门教程。我坠入爱河，并立即尽可能多地切换到使用命令行，手动配置我的 Powershell 来运行我的东西。尽管有 DOS 的怀旧之情，但使用命令行导航还是很有趣，而且比我一直认为的要容易。(我会注意到配置 powershell 并不好玩)

去年发生了一些不寻常的事情。微软实际上允许 BASH(一种流行的 shell 和命令语言)和一个 linux 实例在 Win10 上运行，我开始摆弄它。我被说服了，我可以毫不费力地从同一个 shell 中运行几乎所有的语言。我几乎不费吹灰之力就可以安装 python，运行 c 程序，然后安装 python 程序。

唯一的问题是，我所有的工作都是在 windows 中的 Linux 环境下完成的，让我的代码进入 windows 中更正常的工作流程并进入 Github desktop 是很烦人的，我是在那个时候开始使用 Github desktop 的。所以，在一堆拖延之后，我终于从我的 windows 10 工作笔记本电脑上下载了所有代码，并安装了 Ubuntu Linux。一夜之间，我的工作流程得到了极大的改善，现在我几乎更喜欢将我的工作笔记本电脑用于个人日常使用。我强烈建议任何从事编码工作的人转到 Linux，它的易用性、安装和使用工具的简单性是令人难以置信的。我再次热爱 windows 10，但 Linux 正在慢慢偷走我的心。

我建议如果你转到 linux，从 Ubuntu 开始，尝试不同风格的操作系统，因为 Ubuntu 非常容易使用。我不喜欢 Ubuntu 的设置和桌面，所以我使用等离子 Kubuntu，我喜欢它。

虽然有很多很棒的 Linux 发行版，比如 CentOS 或 ARCH。我只是推荐 ubuntu/kubuntu，因为它非常容易上手，非常受欢迎，如果你需要一些帮助，它会得到很好的支持。此外，您可以将它下载到闪存驱动器，并从闪存驱动器演示运行它，而不必安装来体验操作系统。

我目前使用以下设置来运行我的大部分代码和工作工具。

操作系统:Kubuntu 16.04
桌面版:KDE 等离子版 5.8.8(在我看来比基础版的 ubuntu 漂亮多了，对 UX 也更友好。)

网页浏览器:Chromium 和 Firefox
版本控制:Git 搭配 Gitkraken 桌面平台。
Shell: Konsole (Bash)
文本编辑器:主要是 Atom，偶尔是 vim(慢慢掌握诀窍)几个小时前刚刚为 lisp 安装了 Emacs。
图片编辑器:Gimp
IM: Slack desktop 或 Google Hangouts
我正在使用的语言:C，Python3，Ruby，Clisp(今天才安装的)

我唯一怀念的是 windows 版的 notepad++我还没有找到一个完全相同的 Linux 版本，但这是一个小小的代价，因为它能让我更快地进入状态，更快更容易地安装软件，而且 Linux 给我带来了更好的整体体验。

此外，我可以轻松地做服务器工作，完全定制我的桌面的每个部分，轻松地设置键盘快捷键来运行程序或脚本(Windows 10 也可以做到这一点)，并且可以在一个简短的句子中安装 C 编译器(我不能强调在 Linux 上安装程序是多么容易，大多数程序都是在控制台中键入一个句子，然后快速按“y”键来确认安装。)

我希望这是有见地的，并鼓励其他人跳到 Linux 上做开发工作，我非常喜欢这个操作系统，我实际上正在学习 Linux 管理课程，并努力成为一个超级用户。这是我从来没有在 Windows 上做过的事情。感谢阅读并继续编码。
