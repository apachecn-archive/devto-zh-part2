## [](#knowing-a-programming-language)“懂得”一门编程语言

我觉得提供一些我认为“了解”一门编程语言所需要的上下文和洞察力是很重要的。这是一种观点，因此我对这个问题的想法不应该被视为 100%正确并适用于所有编程语言。

下面我提供了一个我认为是“了解”一门编程语言的支柱的概念列表:

*   具体细节

通过*“螺母和螺栓”*，我指的是使编程语言工作的核心特性和机制。这些特性与使用库和框架无关。这是最纯粹形式的语言。更具体地说，我指的是编译、类型系统、内存管理、构造、循环、评估机制、变量、范围和闭包等等。*【螺母和螺栓】*是编程语言的组成部分，除非一个人在理解如何使用和应用这些组成部分方面掌握得很好，否则他不能有效地使用这种语言。这是每个人学习一门新语言的起点。在我看来也是最容易的部分。对一些人来说，他们可能觉得一旦他们知道了*【螺母和螺栓】*，他们就知道了这门语言。但对我来说，这仅仅是一个开始，只是冰山的一角。

*   图书馆

大多数编程语言都打包了一组核心库。这些库是可重用的代码，可以共享、采用、扩展和应用于各种编程问题。通常，有成百上千的库可供选择。作为第三方和开源开发的结果，这种情况尤其普遍。了解与编程语言相关的所有库是不可能的。虽然这是一个好的实践，至少让自己尽可能多的熟悉。这需要大量的时间和承诺。

*   结构

我经常听到术语“库”和“框架”互换使用。然而，从我的角度来看，框架意味着比简单的库更大的东西。创建框架通常是为了帮助简化可能跨越应用程序架构多个层的开发。其中库易于添加、移除或替换。框架构成了一个人架构的固有部分，不容易被移除或改变。框架很大，它们需要投入更多的时间来完全理解和应用它们。通常，当你已经很好地理解了一个框架时，一个新的框架就会出现并取代它，你必须从头开始。

*   设计

关于有效编程的很多内容都是关于有效设计的。编程语言的设计通常决定了我们如何使用该编程语言设计系统。一个好的设计是一个长期“工作”的系统和一个最终失败的系统之间的关键区别。适用于一种编程语言的设计不一定适用于另一种编程语言。再说一次，只有通过时间和经验，人们才能决定什么设计可行，什么设计根本行不通。此外，不同的编程语言会出现不同的设计模式。这与一些编程语言本质上是命令式的，而另一些是函数式的这一事实有很大关系。正如我之前提到的，每种编程语言都有缺点。例如，令许多人沮丧的是，GO 编程语言目前不支持“泛型”。因此，如果用 C#/Java(支持泛型)编程，就需要使用不同的设计。在为现实世界设计程序时，有一个不断的权衡，每一种编程语言都不同。

*   应用和实践

每种语言都有自己的一套好的和不好的做法。了解这两者是很重要的。每种语言也会有微妙的差别和你应该意识到的“陷阱”。正是这种使用编程语言的实践性决定了你是否懂编程语言。只有通过不断地使用编程语言并将其应用到现实世界的场景中，人们才能很好地“了解”编程语言。

正如你所看到的(我相信很多人都知道)，学习一门编程语言是困难的。对我来说，随着时间的推移，编写越来越复杂的程序变得越来越困难。但是在学习一门新的编程语言时，存在一个有趣的“小”问题。这个“小”问题就是时间。你不能缩短时间，因此你也不能缩短经历。要“了解”一种编程语言，需要投入大量的时间来获得足够的经验，以便熟练掌握这种编程语言。我有大约 15 年的编程经验，用 C++、Java、C#、Javascript 和 SQL 编程。我可以诚实地说，我觉得我还是不“了解”这些语言。但这就是编程对我的意义。不断学习和提高。

在下一节中，我将提供一些我认为你应该知道的编程语言的见解。然而，根据我在这一部分所写的内容来判断，你可能已经意识到了我的微妙暗示，我认为不可能“知道”5 种编程语言。但我确实认为我们应该尝试，因为它让我们成为更好的思考者，从而成为更好的程序员。

## 我认为你应该知道的 6 种编程语言

看完*“大思考”*的视频后，我开始思考。我认为某个人(我指的是任何正在练习编程或对编程感兴趣的人)应该知道的五大编程语言是什么？经过思考，我决定实际上有 6 个，我想推荐。

在回答这个百万美元的问题之前，我想提供一些我的工作背景，因为它确实会影响我的选择。在过去的 15 年里，我在三种主要的行业中工作过。即金融服务(银行和保险)、医疗服务行业、在线游戏(赌博)和体育博彩行业(我仍在该行业工作)。我负责设计和构建的应用程序类型通常涉及系统的 3 个主要领域，如下所示:

*   **数据库**

没有某种形式的长期持久数据存储，就无法构建应用程序。如果说经验教会了我什么，那就是应用程序来来去去，但数据是永恒的。它是一种软件资产，往往比系统中的任何其他资产都要长寿。

*   **前端**

哪里有数据，哪里就需要方便地查看、存储、检索和管理数据。因此，我发现数据库和前端应用程序之间经常存在“一对多”的关系。我的意思是，对于每个数据库，可能会有许多应用程序与该数据库进行交互。我必须构建的前端应用程序的类型涉及到使用许多不同类型的语言。例如，通常我会使用一组语言来构建本地应用程序，而使用另一组语言来构建 web 应用程序。

*   **后端**

我很少被要求创建简单的 CRUD(创建、读取、更新和删除)应用程序。在一个系统中，总有管理不同流程的业务规则。一些业务规则非常复杂，并导致许多不同种类的领域模型的创建，这些领域模型涉及到数据库和服务的集成(例如 API)。通常，我被要求通过 API 公开领域模型。同样，某些编程语言在这个层次上比其他语言工作得更好。

基于我的背景和经验，我现在提供一些我认为重要的编程语言来了解或学习。我最喜欢的 6 种编程语言如下，排名不分先后:

**1。Javascript**

感觉现在 Javascript 无处不在，确实如此。Javascript 不再局限于客户端开发，它已经发展成为一种可以在服务器端使用的语言。一个人只需要阅读求职公告板和各种博客就能意识到 Javascript 将会长期存在。这总是让我想起伍兹定律，该定律指出:

> “任何可以用 Javascript 编写的应用程序，最终都会用 Javascript 编写”

事实是，很少有工作不受益于掌握一些 Javascript 知识的人。这是一门很容易学的语言，然而，你学得越多，它就变得越复杂。Javascript 语言充满了微妙的差别，即使是最有经验的 Javascript 开发人员也会被欺骗。

对我来说，了解 Javascript 与了解一种可以轻松转换成 Javascript 的语言是密不可分的。你想学习这种语言的原因是如果你正在构建一个大型的 javascript 代码库，并且想要提高可维护性。我的偏好是 [Typescript](https://www.typescriptlang.org) ，它是 Javascript 的超集。还有其他的 Javascript 替代品，比如:

*   Coffeescript 是一种转换成 JavaScript 的编程语言。它添加了受 Ruby、Python 和 Haskell 启发的语法糖，努力增强 JavaScript 的简洁性和可读性。

*   Elm 是一种特定于领域的编程语言，用于声明式地创建基于 web 浏览器的图形用户界面。Elm 是纯功能性的，开发时强调可用性、性能和健壮性。

还有一个叫做 [Flow](https://flow.org/) 的“类型检查器”，可以和 Javascript 一起使用。尽管目前我还没有看到任何接近 Typescript 的应用。

**2。面向对象编程语言**

在大学，我主修分布式计算，使用 Java 作为我的主要编程语言。不过，我也使用 C++和一种不太为人所知的编程语言做了相当多的编程，这种语言叫做。然而，Java 一直是我的最爱。学习 Java 当然使我能够很容易地适应其他编程语言。我能说的唯一一件我讨厌 Java 编程语言的事情就是“检查异常”。不，不管它们在技术上有什么优点，你都无法说服我。

这是我最后花了大部分时间使用的 C#编程语言。到目前为止，我已经用 C#作为我的主要语言进行了 10 多年的开发。对我来说，最大的缺点是 C#从来都不是开源或跨平台的。但值得庆幸的是，近年来这一切都改变了，我很高兴在 Linux 操作系统上开发 C#应用程序。这很奇怪，但最近，Javascript 语言发生了如此大的变化，以至于当我写 Javascript 时，我经常感觉自己在写 C#代码。例如，最近当 async/await 特性被添加到 Javascript 中时，我感觉就像在家一样，因为多年来我一直在用 C#使用非常相似的模式编写多线程异步程序。但是我必须承认，我不喜欢将“面向类”的概念引入 Javascript 语言。

所以我认为你应该学习 C#，Java，还是 C++等等？我个人的感觉是，没那么重要，但是选一个吧。就像比雅尼·斯特劳斯特鲁普在上面的视频中提到的。这些是一组语言，了解其中一种有助于你了解所有的语言。正如你从我的个人经历中看到的，它从 Java 和 C++开始，但最终主要是用 Javascript 和 C#编程。

**3。Python**

Python 已经存在很久了。有时被称为脚本语言，我将更进一步，称之为通用编程语言。最近，大约在 2017/2018 年，它已经获得了大量采用。在讨论中不提到 Python，就谈不上数据科学。另一个当然是 R，但目前我的首选是 Python。我主要是一个 Python 爱好者程序员，没有在任何生产环境中使用过它。这并不意味着我认为它是一种业余爱好者的语言。恰恰相反。再一次，你只需要访问一个工作论坛或者一些趋势统计就可以意识到 Python 是很多人的最爱。随着机器学习和人工智能的兴起，Python 似乎也成为这些领域的热门选择。

**4。SQL(结构化查询语言)**

是的，SQL 是一种语言。更确切地说，SQL 是更接近人类语言的第四代语言(4GL)。我有时怀疑 SQL 是否是世界上最普遍的编程语言。SQL 无处不在，在我所有的工作经历中，我从未不使用 SQL。无论是 MSSQL、Postgres、MySQL 还是 Oracle，SQL 都以某种形式存在。当然，SQL 不像新的 NoSQL 技术那样性感和耀眼，但是 SQL 还将存在很长一段时间。有趣的是， [Couchbase](https://www.couchbase.com/) ，一个 NoSql 数据库，使用 [N1QL(发音为“镍”)](https://www.couchbase.com/products/n1ql)查询语言。N1QL 有趣的地方在于它几乎完全符合 SQL ANSI-92。甚至像 [NoSqlBooster](https://mongobooster.com/blog/query-mongodb-with-sql/) 这样的工具也允许在 MongoDB 数据库上编写 SQL 查询。

**5。痛击**

BASH 是一种脚本语言，它是完全图灵化的。就像我之前选择的编程语言一样，BASH 出现在我的列表中的原因是它的广泛采用和社区。对我来说，成为 Linux 之王或之神的区别在于用 BASH 编程的能力。并且能够与 VIM 一起工作；)

此外，Linux 是目前服务器计算中使用最多的操作系统。因此，我认为尽可能多的掌握 Linux 是一个好主意，因为你迟早会使用它。学习 BASH 脚本是掌握 Linux 的一部分。了解 BASH 还将帮助您编写可重复的自动化任务，这些任务可以共享和使用。因此，除了是一个有价值的 sysadmin 工具，我认为它也是一个有价值的 devops 工具。

**6。函数式编程语言**

我没有使用纯函数式编程语言开发软件的经验。因此，我不能根据经验提出任何建议。然而，在缺乏经验的情况下，我转向数据来帮助推动我的决定。在过去的一年里，我一直在密切关注函数式编程语言。根据我的经验，似乎总是吸引最多注意力的 3 种函数式编程语言如下:

*   [Haskell](https://www.haskell.org/) - Haskell 是一种标准化的、通用的纯函数式编程语言，具有非严格的语义和强静态类型。

*   Scala 是一种通用编程语言，为函数式编程和强大的静态类型系统提供支持。为了简洁，Scala 的许多设计决策旨在解决对 Java 的批评。

*   F#是一种强类型、多范例编程语言，包括函数式、命令式和面向对象的编程方法。

我怀疑函数式编程在不久的将来会成为编程技能。我这样说的原因是因为编写并发和并行程序的需求增加了。这种需求是由微处理器不断变化的物理结构驱动的。简而言之，作为程序员，我们只需担心编写与单核处理器一起工作的程序。直到几年前。如果我们需要更快的速度，我们会提高处理器的速度。近年来，处理器工业达到了提供更高处理速度的极限。因此，处理器架构发生了变化，每个处理器提供 2 个、4 个或更多内核，有助于更快地运行程序。但是程序并没有运行得更快，因为没有人有编程语言可以有效地利用额外的内核来编写真正的并发程序，而没有副作用(稍后将详细介绍)。除了那些使用函数式编程语言的人。事实证明，函数式编程语言非常适合多核、多线程和分布式系统领域。人们经常听到被宣传为使用函数式编程的理由的最大优点之一是消除“副作用”。“副作用”是当你的一些代码正在访问或者改变一些数据(比如一个变量)的状态，这些数据超出了代码的范围。在多线程的情况下，与副作用相关的问题变得更加严重。应用于并行计算的另一个重要概念是阿姆达尔定律。解释阿姆达尔定律需要一整篇专门的文章，我不会在这里提供。然而，我想给你一个有趣的阿姆达尔定律的解释，可以在这里找到。我应该提到，像 GO、Rust、Java 和 C#这样的编程语言现在也将并发性(编写并发程序的能力)融入到语言中，并且确信这只会随着时间的推移而改进。

我鼓励任何编程新手或有经验的人花时间学习函数式编程语言。如果有的话，它将帮助一个人更好地思考如何正确地编写程序而没有副作用。此外，函数式范例也进入了许多流行的语言，如 Java 和 C#。

就函数式编程语言而言，我对 Scala 最感兴趣。对我来说，它是函数式编程领域的一颗新星。

这就结束了我对编程语言的思考，我认为你应该学习这些语言。在下一节中，我将总结这篇文章，并提供一些我认为也很重要的编程语言，它们中的任何一种都可以很容易地取代我已经提到的那种。

## [](#conclusion)结论

以我的经验来看，没有一种编程语言可以统治所有的人。我学习语言主要是出于以最简单的方式完成特定工作的需要。我所讨论的编程语言绝不是最流行的语言。但是这些语言有巨大的社区，大量的文档和学习材料，以及大量的工作。

最后，我想提几个我想尝试的编程语言。它们中的任何一种都很容易取代我在前面几节中讨论的编程语言。它们列举如下:

*   Rust 是一种由 Mozilla Research 赞助的系统编程语言，该公司将其描述为一种“安全、并发、实用的语言”，支持函数式和命令式过程范式

*   **[GO](https://golang.org/)**——GO 是谷歌在 2009 年创造的一种编程语言。它是一种继承了 Algol 和 C 传统的编译型静态类型语言，增加了垃圾收集、有限的结构类型、内存安全特性和 CSP 风格的并发编程特性。
