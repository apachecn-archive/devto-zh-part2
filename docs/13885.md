# 我们没有问的关于优步死亡的问题

> 原文:[https://dev . to/lethyglistic/the-questions-are-not-asked-about-the-Uber-death-fj5](https://dev.to/lethargilistic/the-questions-were-not-asking-about-the-uber-death-fj5)

你可能已经听说了亚利桑那州的死亡事件。一些人质疑这项技术是否安全。我在这里不是要谈论它们，所以我会马上打消那些恐惧。这项技术目前还处于起步阶段，你可以用你的想象力来描绘它的长远发展方向。汽车的未来是自主的，当你的后代称我们今天驾驶的汽车为“哑车”时，它们将是安全的，就像我们称功能手机为“哑手机”一样。

这篇文章讲述了一种不同的反动:技术专家急于捍卫这种独立的自动驾驶汽车，因为用于创造它的技术具有巨大的潜力。这些人中有许多是工程师，他们中有我非常尊敬的人。这些人不像工程师那样思考。他们的想法就像人们害怕他们的新玩具可能会被夺走一样。他们跳过了事故的具体细节，如果他们认为这是一个软件问题，他们会考虑的技术细节，以及我们会问任何撞了行人的*人类司机的基本问题。*

如果你没有问这些问题中的任何一个，你就没有资格对优步在这个女人的不幸死亡中是否应该承担任何责任发表意见。

## 汽车的速度是多少？

你知道吗？我问之前你想知道吗？我希望如此，但是请原谅我。

如果你有一个人类司机撞上行人的案例，你要做的第一件事就是试图想出司机可能疏忽的方式。我们一直在努力做的第一件事，也是几乎唯一的一件事，就是想办法让汽车免罪。极其常见的是，交通工具致死的案例具有很强的“幸存者偏见”,因为只有活着的人才能讲述关于如何无能为力的故事。

但是我们有视频！太酷了。它向我们展示了死亡前的视觉细节。它没有向我们展示其他细节，无论是关于汽车状态还是区域的定性还是定量。汽车行驶速度有多快？这辆车能纠正道路的任何缺陷吗？灯光是否尽可能明亮？左边有辆车吗？右边有肩部空间吗？

为什么我们要用一个比死去的人开车时更容易使用的规则来给这个工具评分呢？这些车应该是比我们更好的司机，对吗？为什么我们个人无法用我们的一种感官阻止死亡的想法意味着机器及其视觉和非视觉机械传感器在这种情况下不会做得更好？

顺便提一下，汽车正以每小时 40 英里的速度行驶。后来的新闻报道中也提到了这一点。你知道吗？

## 这辆车超速了吗？

与上述相关的原因显而易见，但值得自己推敲。我们假设答案是“是”,因为我们假设这些事情是不会错的，因为有一天它们*也会*不会错。这是谬论。当自动驾驶汽车成为道路上的大多数汽车时，这仍然是一个谬论。我们还没有解决这个问题；我们正处于解决这一问题的初级阶段。

那么，为什么我们假设它会限速，因为未来所有的自动驾驶汽车都会限速？

为什么我们假设摄像头是完美的，在这种情况下会识别出限速标志？如果摄像头在黑暗中无法看到她在路中间，我们怎么可能确定它看到了路边的限速标志？

也许汽车甚至不依赖于这方面的标志。为什么我们假设今天的方法是完美的？我们不应该，尤其是如果它不是完全公开的。

## 如果司机没有打电话，会发生什么？

我想直截了当地说，我是 ***而不是*** 试图让坐在方向盘后面的人感觉有责任。但是，如果他们一直在关注，并假设他们看到了这个女人，而摄像机没有看到，他们就会撞上刹车。这会有所不同吗？他们将带着这个问题生活，就像那些实际上能控制自己汽车的人不得不面对的那样。

我们已经在幻想坐在没有方向盘的汽车里，并且能够在早上多喝一杯咖啡。是卖点。自动驾驶汽车的出现也将是人们不注意道路的常态。在这个过渡阶段，这是否意味着可以预防的事故将会发生？难道我们就这样接受生命的损失是不可避免的，耸耸肩，说行人应该获得达尔文奖吗？

这是不可接受的。如果通过更好地保持驾驶员在方向盘后面的注意力，可以防止这一阶段的死亡，这是在这一过渡时期值得研究的一个途径。

## 镜头记录下了她。汽车发现她了吗？

让我们把这个交给观众中的工程学朋友。想象一下，有一天早上，你正在参加站立会议，一位高级开发人员宣布公司的旗舰软件拒绝加载。你有以下的交流:

```
"What investigations have we performed?"

"I saw an error message on the terminal."

"...I'd imagine so. What did the debugger say?"

"I don't have any debugger output." 
```

这是不可接受的。旗舰停机了，而你甚至还没有开始分享对确定问题至关重要的细节，更不用说解决问题了？现在想象一下，如果房间里的其他开发人员对此点点头，并确定系统运行正常，并且没有什么可以阻止这种情况发生。疯狂。

在视频中的某一点，你的人眼看到汽车前面的一个形式，并将其识别为一个人，更一般的是一个障碍。车内的电脑工作速度比你快很多，试图做同样的事情。我的问题是，汽车是什么时候识别出这个障碍物的？有过吗？我们那份有调试信息的视频拷贝在哪里，比如路在哪里，车前面有什么？

在这个阶段，我们无法辨别汽车撞上她时在想什么，因此我们不能完全免除汽车的责任。优步向警方透露过调试信息吗？如果我们要对责任做出决定，我们必须知道汽车对其读数的看法。我读了很多关于这方面的故事，甚至没有明确报道汽车*是否在无人干预的情况下*停下来。我*假设*如此。我*希望*如此。我*不* ***知道*** 所以。

如果这段视频显示汽车从未认出这个人，优步绝对应该对这种情况下的损失负责。如果不是一个人，而是另一个人拥有的某个物体，我们不会接受“我没有看到它”作为宣布没有过错的理由。此外，我还要进一步说，如果调试视频和复制它的原始数据没有公开，那么优步应该对全部损失承担惩罚性责任。这很重要。我们不能在这里开创先例，当我们可以访问他们的所有读数时，仅凭视频来判断这些机器是可以的。

## 该事故与之前的测试案例相似吗？

跳出来指责行人，拒绝正确分析这些事件，说“就是这样”也有可怕的二阶效应。试着暂时忘记死亡。忘掉所有我们归咎于人为错误的自动车辆事故。如果只寻找灾难性的失败让我们看不到还没有导致事故的错误，那会怎样？有哪些令人厌烦的错误，比如他们走错了方向，或者错误识别了什么东西，不恰当地调整了轨迹？如果没有这样的问题，软件是完美的(没有软件是完美的)，那么开发人员在做什么呢？如果它是完美的(它不是)，明天把它运送到世界各地。

如果不公开这些车辆的测试数据，我们就无法知道它们有多安全。

我反复强调我们假设它们是安全的，因为它们有一天会非常安全，但是为什么我们在没有任何实际数据的情况下认为它们今天是安全的呢？我们被告知，他们在道路上有良好的记录，大多数涉及他们的事故都是愚蠢的汽车及其司机的错。我们被告知，他们驾驶背后的系统已经在数十亿英里的虚拟道路和特别设计的课程上进行了测试。所有这些都可能是真的，但是如果我们不能独立地证实这些车辆是安全的，我们就不能相信它们。

呼吁这些公司公布这方面的数据。如果他们不这样做，我们需要让监管机构介入。人们的生命危在旦夕。

# 结束语

也许我看起来太专注于摄像机镜头了，但是视频让这更容易想象，这也是我们目前所得到的。优步也使用非视觉传感器。为什么他们没有检测到车辆正前方有一名女性？当车辆以每小时 40 英里的速度行驶时，它们的范围是否不足以对突然的障碍物做出反应？这是一个重要的安全问题，如果这将成为新的常态。

我一开始就说过，如果你不提出这些问题，你就没有资格发表意见。我肯定还有更多问题，但是现在你已经知道了。现在，我想让你回想一下伊莱恩·赫尔茨贝格死亡报道后的最初几个小时，并且永远记住，不管出于什么原因，你是否跳到了她导致自己死亡的假设上。

你没有回答所有这些问题的信息——我们仍然没有。你确实有能力开始*问*所有这些问题。我对那些以他们理想化的未来主义的名义决定对人类生命的损失视而不见和冷酷无情的人感到非常失望。如果我们不对我们正在创造的技术持批评态度，我们就不会创造这样的未来。
