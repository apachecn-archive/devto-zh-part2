# 四周的开发时间

> 原文:[https://dev.to/cindywu/four-weeks-as-a-develubber-cdj](https://dev.to/cindywu/four-weeks-as-a-develubber--cdj)

做一名开发者很难。在过去的两天里，我一直在研究从项目完成到支付的支付代码。这是第一个项目，我不得不在一个令我望而生畏的代码库中筛选现有的代码。

过去的两天对我来说也很艰难，因为除了这一项任务，我还在努力做其他事情。我已经和教练们开了 3 次会，给一个潜在的实习生打了电话(我应该拒绝这个)，选择了跟进我上周通过电子邮件和脸书见到的鲨鱼实验室的人，被奥弗德克基金会(Overdeck foundation)给 DonorsChoose 的 100 万美元转移了话题，写了我们自己的新闻稿，最重要的是，今天下雪了，所以我在家工作。我今天应该去办公室的。

如果我从这两天学到了什么，那就是如果我想成为一个更有效的开发人员，我应该做一件事，并把它做好。这意味着开发，对其他一切说不。这对我来说真的很难做到。

在过去的四年里，我的大脑已经知道，当我同时处理多项任务时，我的效率最高。开发的时候一心多用会让我变慢。除了极度自律，我不知道我是否有真正的解决办法。

比方说，我变得非常自律，不会说不，只会专注于发展。有什么新的东西很难开发？

#### [](#i-am-missing-the-fundamentals)我错过了基本面

关于 Rails 有一些我不熟悉的东西。这是我长期以来第一个不得不阅读大量 Rails 代码的项目。我想我应该花一个周末来读一本 Rails 的书，以此来恢复我的记忆。同样，我认为我错过了基本原则。

#### [](#learn-by-doing)边做边学

也就是说，我认为学习 Rails 的最好方法之一是边做边学。这是我在鲨鱼实验室看到的非常有效的方法。那里的每个人都是边做边学。如果你不知道如何从鲨鱼身上吸血，就不要准备书。尝试一下，然后向知道怎么做的人学习。这就是杰夫给我很大帮助的地方。在我第一次描述完成和支付之间发生的事情后，和 Jeff 一起回顾我的问题非常有帮助。不过，我认为如果我们坐在同一间屋子里，我们会更有效率。不过，也许像 ScreenHero 这样的东西可以解决我的问题。

#### [](#getting-overwhelmed-because-of-obvious-inadequacies-in-design)因设计上的明显不足而不知所措

我很容易分心，一旦分心，我就会不知所措。因为我对产品的每个部分都非常挑剔，所以我很快就会被吸进一个洞里。例如，我的任务是记录完成和支付之间发生的事情。我发现，当我看到设计糟糕的东西时(如支出表、收款启动邮件)，我倾向于立即记录这些东西。当实验的视觉描绘不是我所期望的样子时，我无法忍受。这让我觉得实验降低了我们自称的好设计的标准。因为我偏离了方向，开始沿着这些轨迹走下去，我错过了记录完成和支付之间发生的事情的主要任务。

#### [](#procrastination)拖延症

我发现我倾向于拖延那些看起来最难做的事情，或者看起来是我最不熟悉的代码部分。我注意到过去当我遇到一个使用 javascript 的 bug 时会发生这种情况。因为我没有使用 javascript 的经验，所以当类似的事情发生时，我会避免这个问题，因为我知道我们还有 700 多个错误可以修复。

为了成为一个更好的开发者，我真的需要强迫自己坚持最难的问题。我可能要花更长的时间才能弄明白，这可能会让我更沮丧，但我会学得更快。我想如果我有更长的不受干扰的时间，会对我有所帮助。而且，我应该在办公室，而不是在我的公寓，在那里我可以漫步到厨房做食物，然后陷入食物昏迷，昏死过去。

综上所述，我仍然非常喜欢做一名开发者。我觉得我每天都在学习，尽管我仍然觉得每天我的主要学习是有太多我不知道我不知道的东西，我真的很享受这种感觉。我学得越多，我知道我将来能学到的东西就越多。我知道如果我继续学习所有的东西，最终我会没有时间去学习我想学的其他东西。我想这是个不错的死法。
