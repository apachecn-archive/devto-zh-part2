# 我在过去几个月里写的 10 条软件工程谚语/语录

在过去的几个月里，我在推特上发布了一些有洞察力和启发性的软件工程谚语。如下图所示:

我浏览了我的 twitter 时间表，并将它们收集在一个地方。

## [](#here-are-the-quotes-with-visuals-)以下是带有视觉效果的报价:

### 要把自己变成一名优秀的程序员并不容易，需要大量的阅读、学习和实践。所以现在就开始。

[![](../Images/f4c55f3af06e116e16be72e4160a5091.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--9R-fLaJG--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/2048/0%2AFZKIObG03M5f0hgp.jpg)

### [](#an-open-pull-request-pr-is-a-liability-in-at-least-2-ways-1-its-a-featurefix-not-shipped-to-customers-2-it-will-invite-code-conflicts-soon)一个开放的拉式请求(PR)至少在两个方面是一种责任。1 这是一项未向客户提供的功能/修复。2 它将很快引起代码冲突。

[![](../Images/6c87e4e834d8456072e16a4a71699fe0.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--QXsH6YT9--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/2048/0%2AhdEm0UEY-1ENj0Qj.jpg)

### 谈论自动化测试和持续集成(CI)很容易，但作为一名网络商店经理，平衡它与交付和成本却很难。

[![](../Images/5f25ec566a96ff9666fcad9955a09893.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--LLsWzLcm--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/2048/0%2A4HXUU69Tc0ywAmgT.jpg)

### 拥有自动化测试却没有持续集成(CI)就像拥有一把剑却让它腐烂。

[![](../Images/ab94b66e9012c50dc90423199bde184a.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--2gwg6u2Q--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/2048/0%2A2KvvzeOdy6LZ1Hu7.jpg)

### 你不能事事循规蹈矩。它通常是一个 ish 实现，如 Agileish、Kanbanish 和 RESTish。

[![](../Images/4ae58c3b64c939d513a40d2b9f44654e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--AOwRssfI--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/2048/0%2A4rvj9_dfJ-8KOPmn.jpg)

### [](#if-you-are-thinking-of-a-software-product-idea-make-a-pain-killer-people-cant-live-without-not-an-optional-vitamin-people-can-and-will-skip)如果你正在思考一个软件产品的创意，制造一个让人无法生存的止痛药而不是一个让人可以并且会跳过的可选维生素。

[![](../Images/7a7568e27737cf99c60b68108179f836.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--3ua03Q0d--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/2048/0%2AdBLLq8Yd40Z5Kkvz.jpg)

### 软件工程师有能力创造伟大的解决方案，但是解决最不重要问题的最好软件是没有用的。

[![](../Images/b23b92ae3e8b728f90ed3ea2098d34f6.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--mUjRTne5--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/2048/0%2Axno1jKObRwNe-TYI.jpg)

### [](#programmersdevelopers-need-to-become-software-engineers-think-of-solution-then-technology-to-implement-it-think-of-abstractions-then-code)程序员/开发者需要成为软件工程师。先考虑解决方案，然后考虑实现它的技术。先考虑抽象，再考虑代码。

[![](../Images/e191187b4f0e8e90e53682610388fba9.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--XxEP8j4a--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/2048/0%2A_YP9cWrWtl_AeFNk.jpg)

### [](#software-developers-are-solution-providers-first-engineers-second-solution-should-have-business-value-than-using-the-latest-tech-fad)软件开发人员首先是解决方案提供商，其次是工程师。解决方案应该比使用最新的技术时尚更有商业价值。

[![](../Images/8f24cf519ec5645296b9f3105dd843e5.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--BICit28B--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/2048/0%2AWSE-RAjYU8Hzcy7C.jpg)

### 如果你不磨快你的斧子，砍树将会花费更多的时间和精力。软件开发也是如此。

[![](../Images/e059dc3dd5f817a1ecdea8d2a089b345.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--PmHGKzru--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/2048/0%2AEYuL1L772XoKpsrI.jpg)
