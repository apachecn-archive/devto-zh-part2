# 代码应该有截止日期吗？

在与一位软件开发人员的交谈中，他说所有的代码都应该有一个截止日期。我必须说，我喜欢为我写的代码设置一个截止日期的想法。

作者对代码质量的感知应该决定它的截止日期。

**需要考虑的一些示例问题:**

*   需求(功能性和非功能性)有多稳定、完整和有效
*   代码是匆忙投入生产的吗？或者它是经过精心开发并注重细节的？
*   它使用的是过时的或实验性的技术还是外来的/实验性的/未广泛使用的编程语言？
*   它有测试吗？
*   代码是在测试之后写的吗？
*   是否经过审查和/或结对编程？
*   ...

对上述示例问题和其他问题的回答有助于确定应该使用的截止日期代码。通过在代码上盖上截止日期，开发人员或团队就对产生代码的质量实践做出了声明。

另一个问题是:谁应该确定有效期？开发商？代码审查者？其他利益相关者？所有的结合？

我必须说这有些反常。到期日使得任何人在该日期后使用一段代码都要对自己的行为负责。喜欢吃过期的酸奶。你决定是否要冒险吃它。

到期日应该是动态的，如果代码被重新访问、重构，这将对到期日产生影响。
