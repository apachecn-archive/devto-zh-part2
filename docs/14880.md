# “每年至少学习一门新语言”是个坏建议

> 原文:[https://dev . to/bosepchuk/每年至少学一门新语言是不好的建议- 207p](https://dev.to/bosepchuk/learn-at-least-one-new-language-every-year-is-bad-advice--207p)

学习一门新语言需要很长时间。除非你的工作需要它(或者对你很重要的个人项目)，否则这是一项糟糕的投资。在这篇文章中，我将向你展示为什么*“每年至少学习一门新语言”*是个坏建议，以及你应该怎么做。

### “每年至少学习一门新语言”从何而来？

据我所知，这个想法是由安德鲁·亨特和戴维·托马斯在他们的书《务实的程序员》中推广开来的(顺便说一下，这是一本很棒的书)。

在关于投资知识的一节中，他们写道:

> 每年至少学习一门新语言。不同的语言用不同的方式解决同样的问题。通过学习几种不同的方法，你可以帮助拓宽你的思路，避免墨守成规。

引用的文本是程序员可以做来提高知识的事情列表中的第一个建议。在这本书的背景下，这是一个合理的建议。

然而，多年来，人们已经剥离了这种背景，变成了这样的东西: ***“每年至少学习一门新语言，否则你不是一个好的程序员。”*T3】**

而我在本帖要讲的就是这个扭曲的版本。

### 为什么这是坏建议

程序员，尤其是初学者和初级开发人员，已经把这个建议铭记在心。他们在互联网上跑来跑去，试图找出他们下一步应该学习哪种语言，并强调他们不知道足够多的语言这一事实。但这都是分散注意力，伤害人们和结果。让我告诉你为什么。

#### 它设立了不可能的期望

你不仅应该阅读博客，撰写自己的博客，为开源项目做出贡献，与其他程序员建立联系，从事令人兴奋的个人兼职项目，参加会议，阅读以跟上最新的技术发展，以及 ***做你实际的工作*** ，你还应该“每年至少学习*一门新语言”。谁需要睡眠或者生活，对吧？*

 *[![stressed computer programmer](../Images/e7ed65628bb8b3aa8e6f0e0d3c580053.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--uB3H5ExE--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/q6cz6kdazrib0am8ahai.jpg)

新开发人员几乎总是不知所措，他们需要基础知识方面的帮助。

他们了解他们正在进行的项目如何为企业带来价值吗？他们了解代码库吗？技术堆栈？他们能做出通过代码审查的有用的代码更改吗？所有这些其他的职业发展期望并没有帮助新开发人员掌握基础知识。

#### 有更简单的方法来拓宽你的思维

亨特和托马斯写道，学习一门新语言有助于拓宽你的思维，这样你就不会墨守成规。但我可以想到更简单的方法来拓宽你的思路。您可以:

*   学习设计模式
*   研究建筑模式
*   看看开源项目
*   与其他程序员交流
*   在线观看视频讲座

#### 如果你在批量消费编程语言，你真正学到了多少？

我只是对这里的机械有点模糊。人们是如何“学习”这些语言的？看书？学习语法？写一个“hello world”程序？复制该语言的教程？写一个 1000 行的项目？

根据我的经验，学习一门语言的语法是容易的部分。如果你想用这种语言编写产品质量代码，你还必须学习围绕这种语言构建的工具。这可能意味着您必须学习一种新的 IDE。您可能需要学习该语言的框架和库。和编译器，构建系统，静态分析器，linters，从哪里获得帮助，以及如何将小程序组装成更大的程序。测试工具等等。

那是一大堆工作。人们真的在做这些事情吗？我怀疑很多人在“hello world”和在教程中复制代码之间停下来。如果是这样的话，学习一门新语言有多大用处呢？

但是，即使你确实用你的新语言创建了一个小项目，我仍然怀疑这是否是对你时间的最佳利用。

#### 语言特有的知识是易逝的

假设你投入了真正钻研一门语言并深入学习它所需的时间。如果你不经常使用这种语言，这些知识还能有用多久？

四件事同时发生:

1.  你对语言细节的记忆随着你不使用它的每一天而消失
2.  语言进化
3.  工具、框架和库不断发展
4.  计算机科学的最佳实践不断发展

我在大学里学习了 Java 2，但是在过去的 15 年里发生了很多事情，我不确定在这一点上我能在多大程度上声称“我了解 Java”。

#### 有一天，你可能会在新的工作或项目中用到这门语言

如果你学习了一门新的语言，你可以把它写进你的简历。这也许能帮你找到工作，对吧？大概不会。

简历中的语言/技术膨胀非常严重。

我不怎么招人，但当我在简历上看到这些内容时，我会忽略它们。我遇到过太多自称精通一门语言的人，但在面试中却不能用这门语言写出一个简单的函数。如果我忽略了这些，你可以打赌 99%的招聘经理也会忽略你简历上的语言列表。

我能相信学习一门你真正想要的工作所需的新语言是个好主意吗？也许吧。如果你把它框定为“显示主动性”，我可能会买它。但即便如此，你的时间不是应该更好地用来练习面试技巧吗？还是学会协商更高的工资和更好的福利？

“学一门语言，万一有一天你需要它去工作”这种事让我觉得效率低下。你如何选择合适的语言来学习？你如何防止你的语言技能随着时间的推移而退化？

请继续阅读，寻找更好的方法。

### 专注于半衰期长的高杠杆技能是一种优越的策略

让我们打开这个。

一项技能是 ***高杠杆*** 如果它对你为获得它所付出的努力的结果有不成比例的影响。学会如何高效地学习是一项非常重要的技能。

一个技能有一个 ***长半衰期*** 如果长时间保持有用。领导力的半衰期大概有几百年。但是最新的 JavaScript 框架可能只有几个月的半衰期。

我鼓励你**专注于学习最能帮助你和你的雇主**取得杰出成就的东西**，优先选择半衰期长的知识。**

#### 如何决定下一步学什么

找到对你或你的雇主最有帮助的东西，如果你还不知道的话，学习一下，然后**用这些知识来改善你公司的业绩**。

然后找到新的最高杠杆的东西，重复。

这样做的人最终会得到令人印象深刻的简历，充满出色的成绩，这是雇主真正关心的。

在我职业生涯的几个阶段，我要学的最重要的东西与编程几乎没有关系。在你需要履行多重角色的小公司尤其如此。如果你发现了同样的事情，不要惊讶。进入其中；玩得开心。除了编程，世界上还有很多有趣的东西要学。

### 外卖

你不需要*每年至少学习一门新的编程语言*才能成为一名“优秀”的程序员。其实没有好的理由去追逐这样的目标是很傻的。

如果你想把学习编程语言作为一种爱好，那就去学吧(我自己也学过)。但不要指望它会对你的职业生涯产生巨大影响。这是一个你可以测试的假设。只要问问你的雇主，如果你学会了[在此插入新的编程语言]，你的公司会给你什么样的加薪。如果答案是否定的，那么你就证明了这种语言对雇主的市场价值。

**要点**:人们雇佣计算机程序员是为了帮他们解决问题，而不是为了他们懂多少种语言。最优秀的程序员**看到了更大的图景**和**优先考虑他们所学的**，以便他们能够 **[交付最好的结果](https://smallbusinessprogramming.com/working-on-the-most-important-thing/)** 。重要的总是结果。
