# 你应该从 Android 开始的三大理由

> 原文：<https://dev.to/brightdevs/top-3-reasons-why-you-should-start-with-android-5564>

开始职业编码冒险的人，无论是还在读书的，刚毕业的，还是转行的，可能经常会犹豫应该从什么编程平台入手。所以让我提几个我认为从 Android 开始是个好主意的理由。

# 平台可用性

与许多其他平台(包括 iOS)相反，你不需要任何特定的设备甚至操作系统来启动。如果你没有运行 Android 操作系统的智能手机，你可以使用仿真器*，对于大多数应用程序来说，这已经足够好了。Android Studio IDE 可以在 Windows、macOS 和 Linux 系统上运行。Google 为 Android APIs 提供了类似 JavaDoc 的文档和教程，因此通常可以相对容易地找到您需要的内容。即使你不能，也有一个巨大的 Android devs 社区。

# 爪哇和科特林

你可以开始用 Java 开发 Android 应用，Java 是有史以来最流行的语言之一。如果你以前有机会使用它，我不会感到惊讶。它很稳定，而且广为人知。但是，如果你想要一些新鲜、闪亮、难以置信的有用、灵活和充满功能的东西，这里就是: [Kotlin](https://developer.android.com/kotlin/index.html) 。它将把你的代码提升到一个全新的水平，仍然可以与 Java APIs 完全互操作。谷歌已经在用 Kotlin 编写的 Android 指南中提供了代码片段。

# 有效益的项目

每个人都认识一些从事大型项目的人，这些项目涉及数百名开发人员、数百万行代码和遗留 API。当你加入他们时，你通常会花几个月的时间弄清楚它是如何工作的，每个人都在看着你的手，这样你就不会弄坏任何东西。如果你花大部分时间试图在一个大的代码库中重现一个小 bug 或者阅读别人的代码(阅读和写作是不一样的，你知道这一点)，实际上学习编码是相当困难的。在 Android 开发中你不会看到这一点。大多数应用程序都相对小而简单，所以入门水平很低。例如，你可以很容易地想象每个应用组件及其行为。当然，这并不意味着你应该在一生中避免更困难的任务，它只是促进你的学习经验。此外，更小或更短的项目也意味着你可能会更经常地开始新的项目，有可能学习广泛的东西，并使用最新和最棒的解决方案。不是对每个程序员都有吸引力吗？:-)

如果你有一个 AMD 的 CPU，你可能需要一个 Linux 系统来加速模拟器。
