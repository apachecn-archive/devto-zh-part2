#### CSS&SCS

##### [](#how-to-create-a-simple-top-navigation-using-flex-box)[如何使用 Flex 框创建简单的顶部导航](https://blog.bitsrc.io/how-to-create-simple-top-navigation-using-flex-box-7be8c90bf461)

我有时倾向于跳过 flexbox 今天的文章作为“旧闻”，但偶尔我会记得 flexbox 刚出来时对我来说是多么令人兴奋。它完全解决了自 web 出现以来就存在的许多布局问题。如果你对 CSS 比较陌生，或者还不觉得 flexbox 直观，这是一个很好的应用 flexbox 实现设计的初级演练。

##### [2018 年选举日摇动 CSS 中加州的“我投票了”贴纸](https://css-tricks.com/rocking-californias-i-voted-sticker-in-css-for-election-day-2018/)

主题，也是一个伟大的一步一步演练如何考虑实现一个使用 CSS 的插图。对于那些想在家学习的人来说，每一步都有 codepen 示例。

##### [](#fun-tip-use-calc-to-change-the-height-of-a-hero-component)[趣味提示:使用 calc()改变英雄组件的高度](https://css-tricks.com/fun-tip-use-calc-to-change-the-height-of-a-hero-component/)

与基于媒体查询的响应度相比,“流体响应”布局和元素的兴起是过去几年中真正成熟的事情之一。Flexbox 和 calc 开创了它，CSS Grid 对它进行了大规模扩展，越来越多的人正在研究如何使用这些现代 CSS 工具来提高响应能力，使其适用于各种尺寸的设备。

##### [](#recreating-the-red-dead-redemption-2-tintype-loading-screen-effect-in-css)[在 CSS 中重现红色死亡救赎 2 Tintype 加载屏幕效果](https://dev.to/treyhuffine/recreating-the-red-dead-redemption-2-tintype-loading-screen-effect-in-css-2lnc-temp-slug-7607077)

如何将 CSS 动画与 CSS 遮罩和滤镜效果结合起来，以最少的代码创建一个超级酷的效果。

##### [](#how-to-organize-your-css-with-a-modular-architecture-oocss-bem-smacss)[如何用模块化架构组织你的 CSS(OOCSS，BEM，SMACSS)](https://dev.to/couellet/how-to-organize-your-css-with-a-modular-architecturebr-oocss-bem-smacss-33m1-temp-slug-2794467)

坚实的 CSS 架构文章。为您提供了一些最常用的 CSS 架构的概述，它们背后的一些核心概念，以及更多信息的链接。

#### [](#javascript)JavaScript

##### [](#listening-to-dom-changes-by-javascript-web-api-mutation-observer)[通过 Javascript Web API 监听 DOM 变化，突变观察者](https://medium.com/@shuvohabib/listening-to-dom-changes-by-javascript-web-api-mutation-observer-hint-its-the-best-practice-3ee92dc8aac6)

变异观察器是一个超级酷的 DOM API，当 DOM 元素被修改时，它允许您从浏览器注册回调。我以前在 ZURB Foundation 中见过它们，当组件的内容发生变化时，它们会自动更新组件，我还能看到许多其他潜在的用例，特别是对于库和框架开发人员。看看他们！

##### [](#the-evolution-of-async-javascript-from-callbacks-to-promises-to-asyncawait)[异步 JavaScript 的演变:从回调，到承诺，再到异步/等待](https://tylermcginnis.com/async-javascript-from-callbacks-to-promises-to-async-await/)

这可能是我所见过的理解 JavaScript 异步性的最好的“解释者”文章。不仅有很好的代码示例，还有一些非常好的人/物隐喻，让它变得有意义。

##### [](#vue-api-calls-in-a-smart-way)[Vue API 智能调用](https://blog.snap.hr/01/11/2018/vue-api-calls-smart-way/)

不要被标题中的“Vue”所迷惑——虽然有一个使用 Vue 组件的例子，但实际上这是一篇关于如何用 JavaScript 设计 API 访问的文章，特别是如果你背后有一个 REST API 的话。不是一个明确的“你应该这样做”，而是一个很好的例子，说明有人是如何做到这一点的。

##### [](#copying-objects-in-javascript)[在 JAVASCRIPT 中复制对象](https://smalldata.tech/blog/2018/11/01/copying-objects-in-javascript)

在浅拷贝和深拷贝之间导航困扰了很多开发人员。这篇文章对这些做了一个快速的分类，但是接下来真正深入到实现每一个的一些方法中。我对学习结构化克隆算法特别感兴趣。

##### [](#making-sense-of-react-hooks)[做出有意义的反应](https://dev.to/ben/making-sense-of-react-hooks-2in4-temp-slug-3153604)

作者(也是解释大师)丹·阿布拉莫夫对新 React 挂钩背后的思想进行了终极解释。

#### [](#other-awesomeness)其他牛气

##### [SmashingConf New York 2018:就是这样的感觉](https://www.smashingmagazine.com/2018/11/smashing-ny-roundup/)

##### [](#how-we-used-gatsbyjs-to-build-a-blazing-fast-ecommerce-site)[我们如何用 Gatsby.js 建立一个超快的电子商务网站](https://medium.com/harrys-engineering/how-we-used-gatsby-js-to-build-a-blazing-fast-e-commerce-site-a9818145c67b)

关于使用 Gatsby 静态站点生成器构建电子商务站点的非常酷的案例研究。看看这篇文章，然后快速浏览一下网站——好家伙，它很快吗？
