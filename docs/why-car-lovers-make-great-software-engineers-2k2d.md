# 为什么汽车爱好者会成为伟大的软件工程师

> 原文：<https://dev.to/dealeron/why-car-lovers-make-great-software-engineers-2k2d>

我们一些最好的软件工程师热爱汽车，我们不相信这是偶然的。自 DealerOn 于 2004 年成立以来，我们发现汽车行业与软件工程职业有许多相同的技能和挑战。在日常工作中，我们看到了与两种追求相关的特质的融合:独创性、对事物运作方式的好奇心、直觉思维，以及对完美不完美的近乎痴迷的渴望。通过利用我们在专业领域的个人激情**，** DealerOn 从汽车爱好者的角度塑造了汽车行业的软件开发。

汽车爱好者欣赏最终解决问题的满足感。在坐在我们的车库里，以强迫性的奉献精神试图找出并解决一个问题之后，我们为我们的工作感到自豪，我们喜欢炫耀它。这种热情深深植根于对实验的热爱——看看什么可行，什么不可行。我们喜欢我们的汽车是我们的粘土，可以塑造成我们想要的任何形状，拥有成果的所有权，并为全世界所见而自豪。在 DealerOn，我们努力创造一个环境，让我们的软件工程师进行试验，找到他们的位置，成为他们想成为的任何人。这可能意味着悄悄地建设我们未来的基础设施，或带头冲锋陷阵。

> 当汽车人保持主动的汽车维护时，许多其他人把它推迟到必要的时候…这就是技术债务。

虽然汽车人保持积极的汽车保养，但许多人把它推迟到必要的时候。这可能要等到指针悬停在**【E】**上方时才加满油，也可能要等到可怕的景象和声音从发动机舱中传出时才换油。嗯，技术债就是这样。汽车爱好者通过有组织、关心最佳性能和愿意付出额外的努力来解决这些问题，所有这些都是软件工程师所期望的品质。在 DealerOn，我们面临着过去十年指数级增长的挑战，同时积极维护我们的基础，通过利用公司的这些特征保持领先地位。

许多汽车爱好者喜欢通过更换更大的涡轮增压器或更大的排气管来让他们的汽车跑得更快。作为软件工程师，我们也为了性能而交换，比如交换一个小的去除空格的正则表达式以支持字符串替换来加速站点速度和稳定性( [Stack Overflow 在我们部署我们的更改后大约一周发表了一个类似的帖子](http://stackstatus.net/post/147710624694/outage-postmortem-july-20-2016))。

与此同时，其他爱好者专注于给他们的汽车进行美学翻新。DealerOn 目前正在准备公司历史上最大的 UX 改革，我们的目标是尽可能给用户提供最有凝聚力、最清晰、最吸引人的体验。这伴随着要克服的相当多的考验，我们将在博客中分享这方面的见解。

每个汽车发烧友都有过躺在地上，手里拿着扳手，被机械灾难压垮，准备认输的时候。在这一点上，一个人必须记住他们最初为什么会在那里结束——他们喜欢什么，是什么驱动了他们的热情。这是一个自豪地建造一些东西并找到一个独特的位置的机会。同样的原则也是让我们每天都期待来办公室的原因。无论你是拥有十年工作经验的终身制，还是获得第一份职业工作，DealerOn 都为你提供了追求职业和个人激情的机会。随着我们不断成长为我们的角色，我们巩固了我们开发团队的核心价值观，并将它们作为荣誉徽章佩戴。我们信任彼此，信任我们的过程，以及我们为用户创造最佳开发环境和体验的目的。

> 无论你是拥有十年工作经验的终身制，还是获得第一份职业工作，DealerOn 都为你提供了追求职业和个人激情的机会。

我们的文化正站在工程师激情的肩膀上，这一点显而易见。到了展示日，最重要的是产品质量——其他都不重要。我们面前有一条令人兴奋的道路，所以如果你想沿着前进，请继续关注。
