# Xamarin。表单确实需要加快速度

> 原文：<https://dev.to/jonstodle/xamarinforms-really-needs-to-pick-up-its-pace-4g7>

我最近在推特上写道:

液体错误:内部

这是 Morten Nielsen 在推特上的评论，他说微软在 MPV 峰会上展示的 MVP 非常令人兴奋。我希望这和 Xamarin 有关。但我担心这不太可能，因为 Xamarin。Forms 是开源的，如果在项目中突然出现一些意想不到的东西，那将是非常糟糕的形式。

我一直在和 Xamarin 合作。表单使用了几年，当我还在使用*原生外观和感觉*时，它不久前就开始变酸了。试图让不同的平台和 UI 栈以相同的方式运行会有太多的问题。

而像 Xamarin 这样的药物。Forms 给了我由 UIKit、Android widgets 和 Windows 生成的实际本机 Widgets。UI，越来越不是我实际想要的。我真正想要的是我的列表视图能在所有平台上同样好地工作。高要求？是的。不可能？号码

亚当·佩德利最近写道。表单给人一种*“原生的观感。人们常说的一句话，曾经是 Xamarin 的闪亮卖点…在 Xamarin 中，这句话正迅速失去相关性。表单应用程序。如今，客户不太注重让应用看起来像本地的……他们希望应用在所有平台上都像这样"*。对我来说，这已经变得越来越明显。

在过去的几个月里，我一直在断断续续地研究 Flutter，随着最近测试版的推出，它看起来真的开始有希望了。Flutter 将原生 UI 抛出窗口，自己完成所有工作。下面是 Skia，一个非常强大的 2D 渲染引擎，被 Chrome 等使用。

最重要的是 Flutter 重新实现了所有必要的 UI 组件。这既包括按钮、标签和文本字段等视觉元素，也包括触摸事件、布局和动画。通常我非常怀疑这种方法，但事实证明它确实有效。而且效果很好。

目前，我在工作中使用 Xamarin 来构建我们的移动应用程序，但我将密切关注 Flutter，以了解未来可能的变化。与此同时，我将在一些个人项目上与 Flutter 一起玩。
