首先，我需要警告你——不要当真。这是一个比喻，有时有点夸张。我没有任何在流水线工厂工作的经验，只是做了一点园丁，但是我有很多在不同组织文化的不同软件公司工作的经验。

我认为这个比喻很好地反映了在大公司工作和在小公司工作的区别，小公司重视信任胜过数字。

# 种工作

在装配线上，每个工人都有一套预定义的标准化任务。简单地说，你会一遍又一遍地钻一个洞，然后在同一个地方放一个螺丝。

在花园里，你更有创造力，你有时间思考并看到你作品的全貌。一天你在修剪篱笆，另一天你在给玫瑰施肥。当你看到自己的行为如何直接影响全局时，你也会从工作中获得更多的满足感。当你看到春天最早开放的花朵时，你会感到自豪。

你会对电脑主板上的下一个组装芯片感到多么自豪？

如果你去工厂仓库，你能找到由你组装的板的确切的计算机吗？不这么认为。工厂的规模与花园不同。

# 时间

在工厂里，你可以轮班工作，有时是白天，有时是深夜。与在大花园里工作相反，这里有电和灯。当然，你可以给自己买一个打火机，但从长远来看，随身携带并不舒服。此外，植物也有自己的生活节奏。它们在某些时候需要浇水，例如，你不能在强烈的阳光下浇水，否则它们会死。你需要调整，而不是他们。工作时间不再那么灵活了。

如果你和几个人一起做一个小项目，你们会非常依赖对方。经常需要同步沟通，需要找到一些共同的工作时间。当工作从未停止时，大组织通常选择跟随太阳的模式。当你上床睡觉的时候，别人正在接管你的部分。

# 协作

在装配线上，你需要加快速度。如果你太慢，你会减慢整条生产线的速度。你没有时间合作。要么你独自做你的工作，要么你根本不做。

在花园里，如果你有耐心，不要太匆忙，那就更好了。

假设你很不耐烦，担心你的植物长得太慢。你会更经常地给它们浇水，带更多的肥料。会有帮助吗？我担心你会杀了他们，而不是帮助他们成长。

类似的情况也发生在以指标为导向的公司中。当人们看到错误的数字，并推动在更短的时间内做更多的事情时，产品的整体质量会下降到非常低的水平，人们会筋疲力尽，员工流动率很高，知识损失巨大。

人们有不同的优先事项，并在一生中不断改变。因为有些人在工厂工作有更多的好处，其他人更喜欢花园。我自己是一名软件开发人员，我正在签署[软件园艺宣言](http://softwaregarden.io/manifesto/)。

*开心一天一醉方休。*
*结婚要幸福一年。*
*要幸福一辈子得到一座花园*
