# 游戏 JavaScript

> 原文：<https://dev.to/kayis/gaming-javascript--294>

*[图片由大卫·沃纳在 Flickr](https://www.flickr.com/photos/okaysamurai/) 上发布*

因此，你最终不知何故成为了一名 web 开发人员，编写 JavaScript 代码，年复一年，你会发现 JavaScript 可以做得越来越多。你用来为你的反恐精英部落页面创建闪烁文本的东西现在可以通过 Node.js 制作成熟的服务器，或者通过 React-Native 制作本地移动应用。

但是，如果你有点像我，你可能会梦想做游戏开发，然后用旧的无聊的应用程序赚很多钱。如果你能用你已经掌握的 JavaScript 技能创建你自己的游戏，那不是很好吗？请放心，因为有很多游戏引擎可以让你做到这一点！

## [相位器](http://phaser.io/)

2D 浏览器游戏引擎领域最著名的竞争者之一。代码非常多，但是如果这是你喜欢的，你一定会有宾至如归的感觉。它刚刚发布了一个完全重写的第三版。

## [诡计多端](http://craftyjs.com/)

另一个 2D 引擎，也是我开始游戏开发的那个。它比 Phaser 小一点，它的实体组件系统更固执己见，但我认为它也是一个坚实的基础。这个月它也发布了新的版本。

## [React 游戏套装](https://github.com/FormidableLabs/react-game-kit)

一个不太知名的竞争者，也是 2D，帮助你用 React 创建游戏。它提供了像游戏循环或阶段这样的基本概念。它还与 React-Native 配合使用，使原生手机游戏成为可能。可能不适合更大的项目，但肯定是 React devs 快速运行的一个简单方法。

## [巴比伦](https://www.babylonjs.com/)

这次是一个 3D 引擎，利用 WebGL、WebVR 和 WebAudio 的力量工作。如果你不是 2D 的类型，但也想保持在浏览器内的东西。正如我在这里提到的其他引擎一样，它比主要引擎更重编码，但如果你不喜欢虚幻引擎(C++)或 Unity (C#)这样的大东西，Babylon 是一个不错的选择。

## 结论

基于 JavaScript 的引擎越来越多，有些甚至允许你将游戏作为本地应用来开发。就我个人而言，我更喜欢自己控制大部分代码，这反映在我在这里列出的引擎中，但有些也带有图形编辑器。所以如果你是一个 JavaScript 专业人士，想要制作游戏，不要退缩！
