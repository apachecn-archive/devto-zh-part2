# 解释最常见的 CSS 度量单位

> 原文:[https://dev . to/flipped coding/most-common-CSS-units-of-measure-explained-2 kee](https://dev.to/flippedcoding/most-common-css-units-of-measure-explained-2kee)

当你浏览 CSS 并开始看到不同的度量单位时，确切地知道它们是做什么的会很有帮助。为了帮助你更好地理解 CSS 度量单位是如何工作的，这里有一些最常用的度量单位是如何工作的解释和例子。

## 像素

像素(px)是最常用的度量单位，因为网页设计中有许多基于像素的标准。就你自己所知，一个 px 映射到你屏幕上的一个像素。它非常小，但是拥有这个引用的目的是准确地知道一个元素或文本将填充多少物理空间。

当您使用像素时，请记住这些单位是相对于屏幕大小而变化的。12px 看起来会有点不同，这取决于它是在手机上还是在笔记本电脑上。像素是网站的基本度量单位。所有流行的浏览器都将网页的默认根大小设置为 16px。

```
div {
    height: 32px;
} 
```

## %

百分比在响应式设计中有很多用途，而且非常简单。你要记住的一件事是，大小是相对于父元素的。如果有一个

## 嵌套在一个中，并使用度量单位%给它一个宽度，那么

```

    width: 1250px;
}

h2 {
    width: 35%;
```

## 雷姆

这种度量单位与 em 度量单位非常相似，只是更容易理解。rem 度量单位基于根元素的字体大小。除非你改变字体大小，否则这个基本大小将是 16px。

这意味着所有有 rem 单位的东西都是 16px 的倍数。例如，假设你的页面上有一个侧边栏和一个主容器。这是你的 CSS:

```
.sidebar {
    width: 2.4rem;
}

.main-container {
    width: 7.2rem;
} 
```

既然你使用了 rem，你应该可以很容易地计算出你的元素有多大。与 em 相比，这是一个很大的优势，因为您不必跟上每个元素的字体大小，并且当在不同设备上查看页面时，它仍然会做一些缩放。
