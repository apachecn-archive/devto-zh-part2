# react 组件开发中需要使用的工具

> 原文:[https://dev . to/elaziziyoussouf/tools-you-need-to-use-in-your-react-components-development-13a 7](https://dev.to/elaziziyoussouf/tools-you-need-to-use-in-your-react-components-development--13a7)

[![alt text](../Images/dd7611e80b4193bd8eae0690b420638b.png "Photo by Fleur Treurniet on Unsplash")T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--jLVcR8wv--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/xvql6gn8ppd7npsnupi6.jpeg)

作为一名 react 开发人员，我相信从事 react 项目的每个人都必须单独开发所有组件，以便利用这个令人惊叹的前端框架背后的组件哲学。单独开发组件可以帮助您获得对项目工作流的控制，这允许您首先开发和测试每个组件，而不必担心整个项目的逻辑，不仅如此，使用这种方法您可以提高组件的重用性、可测试性和开发速度，这将简化团队内的组织和任务分配。

如果这是你第一次听说 storybook 或 react styleguidist，那么这篇文章是你最好的起点，但是如果你已经听说过它或者可能使用过它，那么这篇文章对于了解一些令人惊奇的 storybook 替代品也是非常有帮助的。

我们开始吧。

## [](#storybook%C2%A0-21k-stars%C2%A0)[故事书](https://storybook.js.org/) : (21K 颗星星)

[![alt text](../Images/6ac931eccf50dcc909c57b4dc8202880.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--8RXA5zxc--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/v9z9yzjj3kkck71f1bgn.gif)

构建组件的最著名和最强大的 react 工具之一，在 GitHub 中被描述为支持 react、react-native、vue 和 angular 的交互式 UI 组件开发和测试。换句话说，storybook 是 UI 组件的开发环境。它允许您浏览所有组件项目，并查看每个组件的不同状态。Storybook 附带了许多组件设计、文档、测试等附加组件。例如，您可以使用故事书道具附加组件轻松地玩您的组件道具。您可以使用 storybook-preview 和 storybook-deploy 将您的故事生成并部署为文档静态网站。

## [](#react-cosmos%C2%A0-4k%C2%A0starts)[反应宇宙](https://github.com/react-cosmos/react-cosmos) : (4k 启动)

Cosmos 扫描你的项目中的组件，并允许你在道具、上下文和状态的任意组合下渲染组件。这意味着如果你已经开始了你的项目，你需要在不失去理智的情况下给你的组件添加新的功能。Cosmos 将模仿每一个外部依赖，比如 rest API 或本地存储。此外，在与正在运行的实例交互时，您可以看到状态的实时演变，为了处理所有这些特性，Cosmos 使用了 fixture 和 proxy 架构。

## [React style guidist](https://github.com/styleguidist/react-styleguidist):(4k 颗星星)

[![alt text](../Images/b755c9dd6a8a2cc104e16f14711094ad.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--OSgJLjfw--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/wauyy3ez7q8syfsrqwxs.gif)

React Styleguidist 是一个组件开发环境，带有热重装开发服务器和一个生活方式指南，您可以与您的团队共享。它列出了组件原型，并显示了基于 Markdown 文件的实时、可编辑的使用示例，它可以与 react create 应用程序一起使用。

## [](#catalog%C2%A0-500%C2%A0stars)[目录](https://catalog.style/) : (500 颗星星)

[![alt text](../Images/c70a533fcffe6cdaeabd81ce622fbe26.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--qpygO8_5--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/w2rrz8y74bsz8s9kpzre.gif) 
目录是一个为数字产品创建生活方式指南的工具。它在一个地方将设计文档和资产与真实的组件结合在一起。我认为，当你需要在你的 markdown 页面中展示你的系统设计颜色、排版、音频、视频，当然还有你的 react 组件，以及更多具有多种屏幕尺寸选项的内容时，这个工具会更加强大。

还有其他工具像[atelier](https://github.com/scup/atellier)、[全权委托](https://github.com/carteb/carte-blanche)、 [React BlueKit](http://bluekit.blueberry.io/) 、 [React Cards](https://github.com/steos/reactcards) 、 [React-demo](https://github.com/rpominov/react-demo) 。
