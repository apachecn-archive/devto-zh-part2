我尝试每 2/3 年学习或重新学习一门编程语言。2015 年，我决定学习一门函数式语言，因为我对这种编程范式知之甚少。我尝试了一些语言，如 OCaml、Haskell、Clojure 和 F#，最后，我决定学习 F#。

学习一门新语言是一个缓慢的过程，你学习语法，然后开始用这门语言编写一些简单的代码。一开始，你使用新的语言语法，但你仍然没有用那种语言习语编码。对于第一阶段，我通常使用编码形，因为它们简单。我在 F#中做了一些动作，直到我觉得我已经理解了这种语言及其习语，如何编写测试，如何管理依赖关系，以及如何创建和构建简单的项目。如果你感兴趣，我的 F#形在这里。

随着时间的推移，你开始在语言语法和习语编程。当我达到这一点时，我从代码 katas 转移到一个宠物项目，这个项目有足够的规模来推动我真正学习语言和它周围的生态系统。我试着选择一个让我保持动力的宠物项目。对于 F#我决定写一个音乐理论和吉他库，我把它命名为 [Vaughan](https://github.com/pedromsantos/vaughan) 。

我选择和学习新编程语言的过程:

1.  决定我想学习或改进什么样的编程范式
    *   一些例子
        *   面向对象(Java，C#，...)
        *   函数式(Haskell，F#，Clojure，...)
        *   逻辑(序言，...)
        *   过程式(C，Pascal，...)
        *   声明性的(SQL，...)
        *   ...
2.  决定我想要构建什么类型的应用程序
    *   一些例子
        *   业务线应用程序(网站和 web 应用程序、移动应用程序、企业应用程序)
        *   系统(网络服务器、操作系统、网络、底层的东西:)
3.  尝试几种与应用程序的范例和类型相匹配的语言
    *   我通常会花几周/几个月的时间研究匹配语言
4.  选择语言并开始学习
    *   获得一本入门书或其他学习资源的推荐
    *   找有兴趣和你一起学习的人，一起学习
    *   找一个导师来指导你
        *   对于 F#来说，我有我的朋友和令人敬畏的人[Samir tal war](https://noodlesandwich.com/)指导我进入函数式编程的世界
5.  使用代码卡塔学习语言(主要是语法)和生态系统基础
    *   单一抽象形例子
        *   嘶嘶作响
        *   闰年
        *   罗马数字
    *   简单的多重抽象形例子
        *   保龄球
        *   Tic tac toe(微塔式机)
        *   生活的游戏
        *   火星漫游车
6.  用新语言构建一个宠物项目

构建 Vaughan 和学习 F#是一次非凡的经历。如果你对音乐和/或吉他感兴趣，去看看吧。

在 F#之前，我学习了 Swift 语言，并对 iOS 有所涉猎。

目前我又在选择要学的语言。这次我决定学习一门系统语言。我不知道是学习 Rust 还是重新学习 C++，特别是新的 C++11/14/17 标准。我在 90 年代和 2000 年初做了很多 C++，但是大部分都忘记了。两种语言看起来都令人兴奋。
