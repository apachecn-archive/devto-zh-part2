# 我们能停止假装计划软件开发是“确定的”吗？

> 原文：<https://dev.to/jaymeedwards/can-we-stop-pretending-planning-software-development-is-certain-22m1>

如果你已经开发软件几年了，你知道评估基本上是没有价值的。

除非你是在评估旧的技术，建造一些琐碎的东西，否则公司会要求你去做一些你以前从未做过的事情。

我以前听说过这样一种说法，只有当一个“敏捷”团队或公司从领导层一直到公司中的单个开发人员都接受它时，它才是可能的。

敏捷最难的部分是**我们仍然发现我们错了**，甚至更频繁！

问题是:*人错了怎么对待对方*？
