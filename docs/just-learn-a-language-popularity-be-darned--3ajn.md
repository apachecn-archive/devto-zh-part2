# 学一门语言就好。人缘太差了。

> 原文：<https://dev.to/ben/just-learn-a-language-popularity-be-darned--3ajn>

告诉我你以前是否听过这个前提:

## 文章不再可用

海报名为 [Abisekh subedi](https://dev.to/abisekhsubedi) ,专门处理闯入软件开发或职业飞跃带来的令人瘫痪的不确定性。有这么多的选择，大量的开发者告诉你为什么每一个都是最好的。当每个人都遇到一百万个声音，为你指出另一条路，而当你冒险走另一条路时，却发现同样的声音，那么一个新开发人员怎么可能确定他们的选择。

我认为这个评论是最完美的回应:

解决办法很简单。

忽略人们告诉你要学的所有东西，选择一门看起来有趣的编程语言，并在谷歌上搜索它的官方教程。

决定不了？然后把你正在考虑的所有语言写在纸条上，把它们放进帽子里，然后画一个。

综上所述， ***只是学习一门语言*** 。人缘太差了。

我不确定官方教程是否总是在正确的地方。您在这方面的收获可能会有所不同，但是建议是恰当的:在软件开发中有许多正确的答案。语言和领域的多样性意味着在许多方面都有发展的机会，你的选择并不重要。

除了在我们的领域掌握几乎任何技术都有足够的职业潜力之外，学习一门语言的重要部分*是，从头到尾学习某样东西的行为是你将在整个职业生涯中使用的实际技能。在软件开发中，学习和使用工具有很多阶段，包括阅读文档、调试问题、梳理代码库、联系社区成员、参加研讨会和会议、写博客和教授他人以及这之间的一切。在所有这些领域获得经验才是你真正学到的东西。在这一点上，技术只是一个细节。你在学习学习。*

 *话虽如此，你不必完全蒙着眼睛进去。做一些基本的咨询，找出适合你的方法，并在进行过程中随时进行试验，但在阿比塞克和许多其他人的情况下，他们对自己可能会进入的领域有一些想法，在这一点上他们可以投入并开始行动。

但是难道没有一些我应该避免的垂死技术吗？

有一些语言和领域真的消失了，但是大多数软件消亡的谣言被严重夸大了。很多时候“死”只是意味着“成熟”或者“无聊”。无聊往往是一件好事。我写了很多 Ruby 代码，我也曾担心过它在世界上的未来地位，但我逐渐意识到，它的好处在今天和过去一样真实，未来是光明的——即使它正在从黑客新闻的头版消失。从“狂热和不稳定”到“衰老和垂死”，中间有大约五分钟的突出时间，T2 看起来像是鲁比。但我向你保证这都是海市蜃楼。

我认为让 Ruby 一直存在的一点是，当你第一次学习编码时，它是多么容易。它的可读性，它的优雅，Ruby 社区——这些都降低了学习一门语言的门槛，它的相关框架，以及它最终成为合适工具的不同环境。它让你很容易就进入状态并开始工作，特别是如果你对这个领域还相当陌生的话。

不过，最终，我看到 Ruby + Rails 越来越多地被用作 API 层。虽然 JavaScript 之战意味着前端框架一直在不断变化，但对于构建人们能够轻松开发和贡献的 API 来说，Ruby 似乎是一个更稳定的选择。我很有兴趣看看 Ruby 如何适应和改变成高性能(读起来:快得多)的语言，比如 Rust。就我个人而言，我一直在关注像 [Helix](https://usehelix.com) 这样的东西，这让 Ruby 的未来看起来超级光明。

2006 年，David Megginson(Python 程序员)在一篇关于“编程语言周期”的博客文章中预言了 Ruby 的衰落。在那篇博文中，David 概述了一个理论，即编程语言是由精英程序员发明的，他们试图将自己与“痞子”区分开来。大卫在博客的最后写道:

> 这里最后也是最重要的一点是，一种编程语言被认为很酷，但它的成功总会让它受到影响。当成千上万的普通开发人员在 ACME 小部件的内部埋头苦干，用 Java 编写企业应用程序时，Java 不可能仍然很酷。事实上，如果 Ruby 在企业中取代了 Java(这种情况可能不会发生，因为 Ruby 在内存管理方面没有 Java 比 C++强的优势)，它将遭受同样的命运，我们可以期待 Bruce Tate 在五年左右的时间里写出一本超越 Ruby 的书。
> 
> 从这个角度来看，Python 的失败也是一种成功——只要它从未真正在工作场所站稳脚跟，它将永远与它有一点点区别，至少一些精英开发人员不会感到继续前进的压力。就像一部永远不会变得太受欢迎的电影或乐队一样，Python 将保持其势利的吸引力。

然而，David 的博客文章暗示，新的编程语言*不仅仅(甚至主要)是由技术问题驱动的，而是由精英程序员保持其社会地位的愿望驱动的。所以即使 Ruby 可能已经“衰落”，它仍然是一门伟大的语言...就像 PHP 一样...就像 Java 一样...就像 C++一样...等等。等等..*
