# 在雾蒙蒙的小径之外..

> 原文:[https://dev.to/sakshisangwan/beyond-the-foggy-path-4e 19](https://dev.to/sakshisangwan/beyond-the-foggy-path--4e19)

我相信，作为一个孩子，我们所有人都喜欢在我们最亲爱的电脑上玩，制作那些 MS-Paint 杰作，敲击键盘很难成为打字时的闪光灯，我们认为它们是当时最聪明的东西。
我姐姐的一句话粉碎了这个神奇的世界，“电脑是最笨的，它们需要被告知一切。”

## [](#first-encounter)初次相遇..

我记得我姐姐上高中的时候，我第一次看到她用奇怪的语法写东西，那是我第一次接触 C++。此后不久，一系列的邂逅开始了，我爱上了 C++。

我喜欢思考所有的逻辑和错误，这为我的未来提供了一些线索。这就是我最后一年成为 B.tech 学生的原因。

## [](#stumbling-upon)磕磕绊绊..

在我获得学士学位的 4 年里，我不断地在语言之间转换，从 C++到 C 到 Java 到 Python 再到 PHP，但是我从来没有真正知道我在寻找什么，也不知道我为什么要跳到这些语言中，或者从长远来看它们真正为我服务的目的。唯一让我坚持下去的是我喜欢花时间在我正在做的事情上。我会和人、长辈、老师、同学交谈，每个人都会提出这样或那样的建议。

## *现在的困境是什么？*..

前一年的 11 月，我在一家初创公司实习，获得了一个全栈开发职位。我从事 PHP 和 Laravel 工作，我喜欢从零开始创建应用程序，每天都学习新的东西。在课程中，我与从事不同工作的朋友们联系，并讨论他们如何迎接挑战。然后，经过评估，我得出结论，我没有公平对待代码。是的，代码起作用了，达到了目的，任务完成了。但真正的问题是，这段代码实际上应该是怎样的？我一直坚持这样一个事实，那就是值得爱的一件事就是质量代码，而我正在挑战我自己的信念。我知道首先花更多的时间在语言上比做一个蹩脚的产品更好。我离开了实习，尽管它给了我一直在寻找的答案。我终于能看到雾蒙蒙的小路的那一边。

## [](#what-now-)*现在怎么样了？*..

我从事 JavaScript 已经有一段时间了，我的下一个是 AngularJS2，这是我将坚持的。多年的困境告诉我，不要马上去钻研。**尝遍一切，失败。**这非常关键，你可能不知道自己想做什么，但你肯定会知道自己不想做什么。

## [](#spread-the-love)传播爱情..

我也成为了少数几个社区的一员，比如 ILUG-D、LinuxChix India、PyLadies Delhi 等等。这些社区帮助我获得了如此多的视角，他们教会了我传播知识和帮助他人成长是多么重要，因为归根结底，这就是作为一个整体，一个巨大的社区一起崛起。我欠这些社区太多了，我向自己承诺继续传播爱心，帮助他人。
我期待为这些社区做出贡献。

我在更多的失败和实现的希望中结束。

## [](#happy-coding)快乐编码！
