# 回到马背上

> 原文:[https://dev.to/viksaaskool/getting-back-on-the-horse-174p](https://dev.to/viksaaskool/getting-back-on-the-horse-174p)

2018 年已经过半，我学会了不要想当然。非常基本的事情，像走路和做运动。下面，我的左脚踝

二月初，我正在踢足球，以为铲球救球是明智之举——后来才知道，事实并非如此。写这篇博文的时候已经过去 6 个多月了——做日常活动感觉好像从来没有发生过。我已经开始跑步，做一些轻微的敏捷练习，可能两个月后我就可以再次踢足球了(是的，我必须回到球场上，否则我还没有真正战胜伤病，不是吗？).在大约 50 天的时间里，我不能走路，我没有工作，我知道——你用手编码，受伤应该不是问题——嗯，精神上真的很难，在阅读了互联网上充斥的大量关于脚踝骨折的恐怖故事后——我无法集中注意力。只是简单地读了一些与科特林相关的东西，博客，没什么太多的内容。大约 70 天没有编码。

另一件我认为理所当然的事情，也因为这次受伤而被夺走了，那就是编码形式和编码热情。

我在理疗课上学到的经验法则是，你需要再次正常行走并毫无问题地做基本事情的时间等于你不能行走的时间——我认为这在某种程度上适用于编码世界。如果你离开 IDEs 次，你需要 X 次才能回到游戏中。在我的情况下，两个月没有任何编码，真的意味着在 Android 工作室的两个月的辛勤工作，我仍然觉得我有一些工作要做。在此期间，我读了[这篇博文](https://is.gd/895IRP)，它让我对这种情况感到不安。这是一篇高质量的文章，请花时间阅读，此外还有有价值的评论部分。

[![get back on the horse](../Images/94ff22f787daf33eda0d0e12c1aabe23.png)T2】](https://i.giphy.com/media/3o6nUTwsa7SUMZSmDm/giphy.gif)

无论如何，这不是这篇博文的内容。这是关于如何感觉良好的编码和一些实用的步骤。请记住，编码(以及整个 IT)是一个发展非常迅速的领域，在跟踪行业脉搏方面稍有不慎就可能导致严重后果。所以，这里是，如果你因为任何原因发现自己处于类似的位置，现在有一个简短的列表:

**1。任何东西都有价格。**
重回马背会从你和你现在的状态中带走一些东西。恐慌，所以你可以组织起来。做出你的修正。你必须放弃一些东西——很可能是社交生活和睡眠。没有什么能代替你花在学习新东西上的时间，没有人能代替你。你得自己熬通宵了。耐心和奉献是关键。就我而言，我选择放弃睡眠、社交生活(周末的乐趣，不是全部)和电视节目。

**2。做你自己的项目。**
详细列出你想实施的想法，制定一个计划，可能的话，开一个董事会。为它设定粗略但现实的最后期限。当你能完成任务时，制定一个现实的时间表，但是要有要求。无论什么原因，你都需要弥补失去的时间——鞭策自己。如果你没有经验或者已经过了很长时间——在 github/android examples 上找到一个项目，浏览一遍并尝试复制它(尽可能少地复制粘贴，当你这样做时，确保你明白你在复制粘贴什么)。花点时间去理解你不懂的东西，如果你遇到困难，去问更有经验的人，不要自动驾驶，多走一步。

**3。参与进来，在工作中钻研**
,对自己要求更高——承担你以前没有做过的任务，实现你不太熟悉的功能。编写单元测试。把自己放在计算好的风险中，承担更多的责任。
如果你也离开了，回到社区谈话中去。大多数大型技术中心每周都有用户组会议，找到一个并加入——那里的人都是爱好者，分享想法，讨论问题很有帮助。如果这不是一个选项，检查互联网。

乍一看，这些可能很难，而且怎么强调都不为过，你必须保持头脑清醒——自律和奉献。这听起来像廉价的激励性谈话，但你必须付出 110%的努力，乘坐仓鼠轮，直到你再次感到舒适。那你可以低调一点。
