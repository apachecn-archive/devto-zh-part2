阅读时间:4 分钟

阅读源代码是软件开发人员的工作职责之一。然而，这种经历并不总是愉快的。不是每个人都喜欢阅读别人的代码，因为他们觉得无聊，有时令人沮丧。有些情况下，当你开始阅读别人的代码，但最终却因为无法理解或代码写得不好而感到痛苦。大多数开发人员希望专注于编写代码，而不是阅读，没有意识到阅读代码也是一项技能。

然而，阅读源代码有许多好处。阅读代码的回报是巨大的。人们可以将编码与文学相比较。你会看到，大多数作家在写作时都专注于阅读名著。原因是你不能仅仅用真空吸尘器创造任何东西。你必须吸收不同来源的东西，并在此基础上创造出你自己的杰作。编写代码与此没有任何不同。

为了能够写出好的代码，你必须阅读大量好的代码。通过这种方式，你可以了解其他开发人员是如何思考的，他们是如何解决某个特定问题的，以及他们的不足之处。你可以从阅读别人的代码中获得不同的想法、风格和模式，让你的知识逐渐增长。

如果要用简单的话来说——***读码技能提升编码技能*** 。

有几种方法可以提高阅读技巧，在这篇文章中，我想提几个-

1.**运行代码**–这是读取代码的第一步。能够立即运行一个特定的项目会给你一些精神上的认可来探索更多。然而，这可能不会给你很多关于项目的细节，但是，你会知道如何构建和运行它，你也会了解它使用的库，它依赖的框架，等等。这是提高您对特定项目的理解的好方法。如果您想编写自己的软件，类似于您正在探索的特定项目，您可能会对可能使用的框架或库有所了解。

2.**找到高层次的逻辑**——当你刚开始阅读一个项目时，你可能不会跳到每一个细节，而是应该关注高层次的结构。从入口点开始。大多数项目都有一个主方法，从那里开始。此外，如果这是一个 web 应用程序，开始查看不同的包，比如业务逻辑代码在哪里，UI 代码在哪里，控制器在哪里，等等。基本上，浏览整个项目，获得一个初步的想法，然后问自己一个问题，你想把重点放在哪里，你想先读哪一部分。您可能不会阅读整个代码库，但您可能会对其中的一部分感兴趣。当你找到自己的位置时，你可以从方法调用开始，看看它会把你带到哪里。

3.**使用/了解工具**–有很多工具可以用来阅读和探索源代码，帮助可视化代码。例如——intelli jidea 拥有强大的导航源代码的能力，你可以通过单词、单词的一部分甚至缩写进行搜索。你也应该学习键盘快捷键。用鼠标导航源代码可能会非常无聊和缓慢，而使用键盘快捷键可能会更快。您可以快速地从源代码的一部分跳到另一部分。

还有另一个很棒的阅读代码的软件，叫做 Sourcegraph，它是由斯坦福大学的两个毕业生 Quinn Slack 和 Beyang Liu 创建的，他们在花了几个小时搜索记录不良的代码后，决定开发一个工具来帮助他们更好地阅读和理解代码。

4.**了解语言/惯例**——深入了解一门特定的语言有助于提高你的代码阅读技能。每种语言都有自己的一套约定、风格和语法。这些知识有助于快速熟悉特定的代码。例如，在 Java 中，方法名以小写字母开头，而在 C#中，方法名以大写字母开头。了解了这种区别，有助于您从源代码中识别方法。

5.**阅读最佳实践/设计模式**–你正在阅读的代码，它的某个构造，可能看起来晦涩难懂或不为人知，这是很自然的。有很多好的实践和设计模式，人们用它们来正确地、优化地做事。例如，有一种称为 Singleton 的模式，其中构造函数是私有的。你可能会问，为什么地球上有人必须保持构造函数私有，我一直认为它是公共的，否则，我怎么会创建一个类的实例。嗯，这背后是有原因的。单例模式防止一个类被实例化超过一次，这有很多实际的用例。如果你知道这种模式，你就不会为此去撞墙。因此，了解某些模式——例如——Gang of Four 有 23 个文档化的设计模式，可以极大地帮助您的代码阅读技能。

6.代码评审–软件开发是一项非常协作的工作。没有人能够独自构建一个大型或重要的软件。每个软件都是在一个团队中构建的。在团队中，每个人都为项目的形成做出贡献。最终，每个人的贡献都融合在一起，成为一件对客户有真正价值的好作品。除了进行实际的编码，现在每个团队都在进行另一项实践，那就是检查彼此的代码，提出观察、建议和互相学习。这是一个强大的工具，用于构建代码库的知识，在团队中创建强大的纽带，并提高代码质量，从而减少系统中的错误，并让客户满意。

在进行代码审查时，你被迫阅读团队中其他人的代码，这最终会提高你的代码阅读技能。

7.临时 **重构**——临时重构也能帮助你提高代码阅读技巧。你可以从一个长方法开始，然后不断地把这个方法分成多个。继续这样做，直到你对这个更大的方法的意图有所了解。之后，您可以记下该方法做了什么，然后回滚更改。

这可以帮助你理解方法构造，也可以提高你的重构知识。
