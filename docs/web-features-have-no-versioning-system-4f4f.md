# Web 功能没有版本系统

> 原文：<https://dev.to/nektro/web-features-have-no-versioning-system-4f4f>

作为一名 web 开发人员，放眼广阔的开发平台，我注意到，作为一名 web 开发人员有一系列非常独特的挑战。我认为其中很多是由两个因素造成的。一个是我们最好的功能，另一个是可修复的问题。

1)“任何人都可以制作 web 浏览器”“
2)Web 没有版本控制系统

正如我前面提到的，第一个问题实际上是我们最大的特点，但问题的出现是因为用户期望我们的网站(除了完整的功能)需要以一种最起码的方式“工作”，不能优雅地告诉用户他们正在使用的设备的状态以及他们是否可以使用该网站。

第二个是可修复的。不，我不是说我们有 Web 2、Web 3 等等。令人惊讶的是，Web APIs 没有一个顺序，因此供应商可以按照对浏览器将运行的硬件最有意义的顺序来实现它们。然而，从开发者和潜在的新浏览器的角度来看，大多数规范的进展只能通过 Git 提交历史来跟踪。

> ECMAScript 本身和大多数 CSS 特性没有这个问题，但是 HTML 和 JavaScript 的许多 Web API 扩展遇到了上面的问题

WHATWG 说他们不想发布有问题的规范。我明白了。没有人知道，但是版本号的作用是给开发人员一个参考基线，让他们知道哪些功能和特性可供他们使用。它还会给新的和现有的浏览器开发人员提供一些功能检查点，以便他们可以更好地集中精力。这将为浏览器设置一个知道“嘿，这个网站需要 WebUSB 2，但我知道我只支持 v1，我应该告诉用户！”
