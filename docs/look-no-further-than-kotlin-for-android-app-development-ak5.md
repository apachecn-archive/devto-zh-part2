# 对于 Android 应用程序开发，没有比 Kotlin 更好的选择了

> 原文：<https://dev.to/rakesh_patel55/look-no-further-than-kotlin-for-android-app-development-ak5>

我已经在 Android 应用程序开发领域潜行了一段时间，我必须承认它似乎已经不再使用 Java 编程语言了。去年，谷歌宣布 Kotlin 将永远被支持为 Android 的一级编程语言。今天，它已经成为最受欢迎的编程语言。

Java 是面向对象的和精心制作的编程语言，具有有限的实现依赖性。用通俗的语言来说，它允许开发人员编写一次代码，就支持 Java 的平台而言，可以在任何地方运行。而 [Kotlin 是一种运行在 JVM 上的静态类型编程语言](https://en.wikipedia.org/wiki/Kotlin_(programming_language))。Kotlin 可以很好地编译成 JavaScript 源代码和本机可执行文件，它提供了一系列令人惊叹的特性，例如:

```
• It is an opensource
• Full Java Interoperability
• Kotlin compiles to JVM bytecode or JS
• Data Classes 
• Defaulted parameters
• imposes no runtime overhead
• Extension functions
• Swift
• Write less code in comparison to Java 
```

## 为什么 Android 要用 Kotlin？

由于支持数千种设备，安卓已经成为全球消费者的热门选择有一段时间了。尽管如此，Java 在应用程序开发中的应用似乎正在以一种狂热的速度扩大。但是同样重要的是，你要知道 Java 带有很多历史包袱。

Java 8 确实解决了一些语言问题，Java 9 和 10 纠正了更多问题。但是你很清楚一个事实，Android 开发者需要将最低 SDK 设置为 Android 24 才能使用 Java 8，这不是一个选项。另一方面，Kotlin 的目标是填补 Android 平台缺失的现代语言的空白。

它有几个值得考虑的重要原则，例如:

减少需要编写的样板代码的数量

当涉及到使你的代码更可读和更易理解时，非常有表现力

可以避免所有类型的错误，例如空指针异常

可互操作，利用 JVM 的现有框架和库，具有 100%的 Java 互操作性。

适用于构建服务器端应用、Android 应用或在浏览器中运行的前端代码。Kotlin 语言的缺点这项技术确实带来了好处，但也有相当多的缺点。

相当大的运行时间——kot Lin 的一个主要缺点是它的标准库和运行时间增加了文件大小。平均来说，一个. apk 文件扩展了 800 KB，这并不多，但是如果一些文件已经很大了，那么它可能会成为一个需要研究的问题。总的来说，它可以让用户在下载沉重的文件前思考。

复杂的代码解密——有时你最大的优势会变成你的弱点。即使有了很好的语法，你仍然可能会面临某些问题，因为一个小代码被注入了如此多的内容。就 Java 而言，它的代码并不那么紧凑。

缺乏官方支持——Android studio 被正式集成到谷歌的应用程序开发过程中，但它未能扩展对 Kotlin 的支持，因为该语言没有得到谷歌的官方支持。
