### 自我介绍！你是谁？你在哪里工作？

嗨！我是卡罗琳。我毕业于美国马萨诸塞州美丽的萨默维尔市的塔夫茨大学，获得了认知科学和计算机科学学位。

在过去的两年里，我在一家名为 Jibo 的社交机器人初创公司工作，在那里我开始构建机器人的 SDK，并过渡到编写更深远的后端系统，这些系统专注于将角色和个性注入 Jibo 的情感、语音和交互中。

我最近搬到了 Cogito(由一个来自惠灵顿的新西兰人创立！)，它做会话分析。虽然我在技术上是 web 团队的一员，但我的角色包括全栈开发和大量关于数据架构的思考。

当我不在咔嗒声过大的机械键盘上疯狂打字时，你会发现我在球场上玩极限飞盘，或者和朋友一起在山里磕磕绊绊。

### 是谁或什么让你开始编程的？

我所在大学的认知科学项目需要两门计算机科学课，这实际上是一种幸运。我在第一个学期就报名参加了入门课程，目的是“摆脱它”到了第二个项目，我知道我喜欢它，到了第三个项目，我准备主修它。我去告诉我的教授，他笑着拿出一张纸；他已经把我登记为他的顾问了！我想有时候你就是知道😍

### 你好像对人工智能和机器学习很感兴趣！这个领域的什么让你感兴趣？

我在认知科学方面的背景真正影响了我对计算的兴趣。我把编程作为一种工具，用来更好地理解大脑和我们自己的思维系统。直到我从事、开发和塑造了一个巨大的软件栈，我才学会欣赏大型软件架构的美丽和复杂。尽管如此，软件和人类思维系统在架构上的相似之处是惊人的。

人工智能和人工智能仍然是新兴领域，尽管正式的认知研究自 50 年代以前就已经存在(明斯基的《心智社会》被认为是[完全无法实现的]经典)，但科学家现在拥有的计算工具是无与伦比的，并开启了关于思维的全新思维方式。

在我看来，一旦人类知道我们自己是如何思考的，基本上就有无限的社会影响潜力。你知道你的大脑有时会做出很酷的事情吗？想象一下，如果我们知道如何一直调用它。想象一下，如果我们知道如何平衡和利用我们思维系统中的每一条路径。每一个进化上的侥幸或“生活帮”都会为我们所用。

### (在 Jibo)从事 SDK 工作让你学到了什么开发经验？开发者和普通消费者有哪些区别？

对于消费者，你可以或多或少地告诉他们想要什么。开发人员知道他们想要什么，尽管我总是发现他们非常友好和宽容。一般来说，响应性比立即正确赢得更多的分数，我认为这是因为社区的感觉。开发者喜欢帮助开发者，因为，嘿，我们都是开发者！在一天结束的时候，承认错误并表现出你在倾听和响应社区需求就足以赢得朋友。

我看到的群体之间最大的相似之处是，每个人都希望他们的互动简单。越来越多的现在，如果消费者(开发人员或休闲)需要考虑如何使用你的产品，你已经输了。一个简单的界面、合理的默认设置和一个万无一失的安全网(想想:即使你真的很努力，你也不会搞砸)大有帮助。基本上，“假装用户喝醉了”对开发者和普通消费者都适用。🍻

### 你在美国和新西兰都经历过计算机科学教育。有什么不同，你更喜欢哪一个？

最大的不同无疑是我在课外被期望做的大量工作。在美国，我的本科课程中至少有两门每周超过 40 小时。一节课。人们对花费大量时间在问题集和项目上有着巨大的期望，我真的相信我所做的 100%的工作使我成为了一名更好的工程师。在这种情况下，“更好”意味着更深思熟虑，并意识到实施背后的深层问题。所以在新西兰，我觉得，至少在时间上，我没有得到足够的重视。

我也觉得在课堂上举手提问很傻。在美国，我的课堂通常充满活力，完全由学生以我们认为准备好的方向和速度讲课，而在新西兰，尽管教授告诉我，他喜欢我提问，但我的同龄人很清楚，这不是怎么做的。老实说，我从未真正适应过，不管怎样，我还是问了。我的哲学是“我是一个普遍聪明的人，如果我不能理解这一点，那么其他人很可能也会迷失，如果不是这样，我就已经不知所措了，所以去他妈的。”

另一个巨大的差异是性别比例。在塔夫茨大学，计算机基础课程的男女比例是 50/50，我们的毕业比例通常是 65/35。在新西兰，虽然系里有其他女性，但我是两个计算机科学班里唯一的女性，而另一个班里有三名女性。这些都是大型的，讲座式的，30-50 人的课程！我最终找到了我的朋友们(holla atcha SAM)，但这比我想象的要可怕得多，并让我对不太敢说话的人口少数群体产生了很多同情。

### 迄今为止，在你的软件生涯中，你学到的最难的一课是什么？

所有的商业课程都是最难的。你可以做你喜欢的很棒的东西，但最终，它必须赚钱。为了在截止日期前完成任务，你不得不牺牲正确的做法；为了强化代码，你不得不牺牲实现很酷的新特性。基本上，软件相当于吃你的蔬菜。也就是说，现在我非常非常欣赏良好的测试覆盖率！

但我不得不说的一个更难的教训是，性别歧视绝对是一个问题，我讨厌这么说。它没有在个人层面上对我产生太大影响，但我确实经常看到它，而且无处不在。我非常幸运能在这么长的时间里远离它，但在现实世界中，我很难接受这样的事实:我必须更加努力地工作，才能让一些人认真对待我。

### 到目前为止，你对成功的软件职业生涯的第一条建议是什么？

团队合作，自主决策。如果你做了一个技术上很愚蠢的决定，承认它，并和你的团队一起修正它。不要找借口，只要学习，继续前进。另一方面，如果团队成员犯了错误，那也没关系！允许每个人都有空间来一起解决问题。如果做得好的话，编程是最社会化的工作之一，因为你必须不断地将不同的部分组合在一起。当你能和一个紧密团结的团队一起退后一步，看到你自己建立的这个巨人，这是你自己永远不可能完成的，这是多么令人满意。

此外，不要害怕自我推销。记住，如果你达到了工作描述中的每一个“要求”，你就大材小用了。承担风险，快速学习，对不知道答案和发现新的思维方式充满热情。
