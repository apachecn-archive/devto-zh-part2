# 为新而新

> 原文:[https://dev.to/satimewallin/new-just-for-new-4l19](https://dev.to/satimewallin/new-just-for-new-4l19)

我们都知道，如今软件和硬件每天都在创新(如果不是每小时的话)。如果您昨天使用的是 node 10.1，那么现在是 10.8 的时候了。那些不用 Docker 和 Kubernetes 之类的被视为古代遗物。作为一名前端开发人员，你需要跟踪 react、redux、respond、gulp、grunt、bower、yarn 等。

每天都有新的框架发布。对于 javascript，甚至有一个名为“自上次 javascript 框架以来的日子”([https://dayssincelastjavascriptframework.com/](https://dayssincelastjavascriptframework.com/))的网站，它总是一位数，最常见的是 0。

每天都有新的语言发布。只是在过去的几个星期里，我们都看到了关于 Joy-lang、Markdeep-lang、Pole-lang、Kitten-lang 等黑客的新闻帖子。

你可能认为这个咆哮暴躁的老男人(32 岁)已经厌倦了跟上时代，但这不是重点。我喜欢它，部分原因是因为不断发生的创新。我喜欢它，因为每个人都可以制作自己的东西，并在发布时得到广泛传播。

不过有两件事让我很恼火:

1)人们似乎把某种威望放在知识的新鲜度上。如果你在博客上写的是 15 分钟前发布的框架，那么你显然很棒。

那真正学点东西呢？无论你学得多快，几个星期后你都不可能真正擅长。在几个客户项目中使用新框架，在更长的时间内对其进行几次重构，然后你就可以开始宣称自己掌握了真正的知识，然后我很乐意阅读你的博客文章。

“针对正确问题的正确工具”怎么了？最新的语言或部署工具不可能是解决所有问题的合适工具。例如，Docker 不可能是所有部署或托管环境的最佳选择——即使 Docker 是一个很好的工具。

2)人们倾向于学习一个框架，并声称了解该语言。再一次，这在 Javascript 中表现得很明显。我见过很多前端开发人员，他们自称了解 Javascript，但实际上他们只知道 React 和 JQuery。他们知道用某种语言编写的框架和工具，但他们不懂这种语言。

另一个很好的例子是后端开发人员经常争论使用 ORM 的好处。为什么不学 SQL？！SQL 简单而强大。与 ORM 不同，您实际上可以使用您的知识来优化 SQL 查询。当使用 ORM 时，你可以简单地调用其他人根据他们的 SQL 知识编写的函数——而你几乎不知道实际发生了什么。
