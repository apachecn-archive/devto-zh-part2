# 我刚开始一份新工作，他们想让我从事我认为对我的职业没有好处的技术工作，我该怎么办？

> 原文：<https://dev.to/sloan/ive-just-started-a-new-job-and-they-want-me-to-work-with-technology-i-see-as-no-benefit-to-my-career-what-do-i-do-in9>

这是一个不愿意透露姓名的会员发来的匿名帖子。请仔细考虑你的回答，因为这些通常是很难写的帖子。

我的新公司在我身上“冒了一次险”(这让我非常兴奋),它正处于一个新项目的早期阶段，客户为几个静态网站提供 CMS 解决方案。

我被要求评估一些选项，其中一个他们甚至在与我交谈之前就似乎非常热衷，这个特定的选项被宣传为完全前端不可知的解决方案。我开始安装它，启动并运行演示站点。它是一个使用 tomcat 的 Java 解决方案，内置语言是某种形式的 xml 模板语言。你必须定义你的内容的“工作室”既笨重又不直观。

我很快就开始测试一些我更熟悉的东西，CMS 提供了一个 GraphQL API 来提取你的内容，一个静态站点生成器来模板化页面，我很自豪我在我必须使用它的短时间内成功地将它放在一起，并且可以看到它对这个项目非常有用，这个项目使用了其他开发人员想要使用的熟悉框架。

所以他们决定使用另一个 Java 笨拙而不直观的 studio 解决方案，我被告知我将接受使用它的培训，并且我应该开始熟悉它，因为我将在五月底交付一个 PoC。

不想在这里听起来有资格，但我真的不想浪费我的时间学习和过时的技术，这不会帮助我的职业发展。

我在这里还处于试用期，但我觉得这根本不是我想做的事情。

在找新工作之前，我能做到什么程度？
