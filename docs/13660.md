# 答。Linux 世界里的网人。

> 原文:[https://dev.to/rafalpienkowski/a-net-man-in-linux-world-i66](https://dev.to/rafalpienkowski/a-net-man-in-linux-world-i66)

在我看了几篇帖子之后(例如[)。网芯 2.1 亮点:站在巨人](https://dev.to/samueleresca/net-core-21-highlights-standing-on-the-shoulders-of-giants-278m)和[的肩膀上我的 2018 年 Linux 开发环境](https://dev.to/brpaz/my-linux-development-environment-of-2018-ch7)我已经开始思考一个小实验。

如果我作为一个. Net 开发者，哪个自然环境是 Windows 机器，开始在 Linux 机器上工作呢？

*   。Net Core 是 cors 平台解决方案，完全受 Linux 支持。
*   我不是桌面/移动解决方案开发人员(没有 WinForms，WPF，Xamarin)。
*   我喜欢在我的解决方案中使用 Docker，Linux OS 比 Windows 更适合 Docker。
*   我可以在 Linux 操作系统上使用 Visual Studio Core。

为什么不至少试一试我下班后负责的项目呢？

我不是 Linux 的高级用户。我有一点在学习中获得的经验。我任意选择了 Ubuntu 17.10 作为我的 Linux 机器。我已经安装了 [VS 代码](https://code.visualstudio.com/docs/setup/linux)，[。Net Core](https://docs.microsoft.com/en-us/dotnet/core/linux-prerequisites?tabs=netcore2x) 、 [Git 客户端](https://git-scm.com/download/linux)和 [Docker](https://docs.docker.com/install/linux/docker-ee/ubuntu/) 基于互联网上的教程。现在我出奇的容易。接下来几周的计划是将我所有的附带项目移植到 Linux 上，并将 Linux 作为我的主要“开发”环境。

我将分享我对操作系统变化的感受。请随意添加您的建议和问题。

所以就像斯汀的[英国人在纽约](https://youtu.be/d27gTrPPAyk)一样，我会成为 Linux 世界的. Net 人。
