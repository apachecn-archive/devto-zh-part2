# 3 个想法

> 原文:[https://dev.to/eduardsi/3-thoughts-2cj6](https://dev.to/eduardsi/3-thoughts-2cj6)

今年夏天，我 30 岁了，我决定分享一些我多年来作为软件开发人员、培训师和企业家所学到的东西。这篇文章没有具体的目标，只是一系列的想法。我不得不把这篇文章减少到只有 3 个想法，以使它具有可读性。如果你觉得其中一些有用，我会很高兴。接下来会有更多的帖子。

### [](#thought-1%E2%80%8A%E2%80%8Awe-are-problem-solvers-not-software-developers)🧠认为 1 -我们是问题解决者，而不是软件开发者

当我开始我的软件开发生涯时，我认为我的工作是写代码。这些年来，我意识到有时候写代码并不是解决问题的最好方法。通常，更多的代码意味着更多的问题。你还记得软件开发的普遍规律吗？🤔

```
e = mc² 
```

或者

```
errors = (more code)² 
```

我们写的代码越多，我们的团队和团队之后需要维护的代码就越多。更多的代码导致更多的复杂性和错误。更多的代码意味着不做其他重要的事情。编写经过良好测试的干净代码需要很多时间。**写代码本来就很贵**。

今天我不喜欢**软件开发者**这个称呼。我们是**问题解决者**。这是完全不同的心态。范式转变。
公司雇佣你来解决问题。一般营利性公司都想多赚钱，减少开支。我们的目标是提供帮助。不要从功能的角度考虑，而要从金钱的角度考虑。如果给你的同事带咖啡能让他们有效率(而效率是瓶颈)——做一个咖啡抢劫犯。谁负责什么并不重要。你的工作协议里有什么并不重要。你是 CTO 也没关系。
专注于职位、角色和职责会分散我们对客户真正重要的东西的注意力—**金钱**💰。

[![](../Images/3ead18f76145e84340b2db4d0f24b10c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--xGva9ggp--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1600/0%2AsOWXpj4V2IMFU4LS)

我们的工作是了解公司是如何赚钱的，帮助它**赚更多的钱**和**减少开支**。以这种方式思考和行动，你将很快成为公司最有价值的资产。

我也运用类似的策略来选择发展技能。例如:

✅无服务器-只为您消耗的资源付费，在容量规划上花费更少的时间=💰💰💰

✅ TDD -更短的反馈周期，更少的 bugs =💰💰💰

✅软技能——团结员工，让他们更有效率、更快乐=💰💰💰

如果你正在纠结下一步该学什么——回答一个简单的问题:

> 什么技能可以帮助我的公司赚更多的钱，减少开支？

希望它能让你不再学习一个 JS 框架。

### [](#thought-2%E2%80%8A%E2%80%8Afocus-on-fundamentals)🧠思想 2——关注基本面

我在技术领域工作了 10 多年，相信我这是有规律的——趋势来来去去。

与 Angular、React、Vue、Riot、Ember、Knockout 保持同步很有趣，但它会夺走你的生活。它让你远离真正重要的“东西”——朋友、配偶、孩子、父母。我们的生命是有限的。你必须充分利用你的时间。你必须开始把你的时间投资在那些能经受住下一波炒作的东西上。

理解**林迪效应**很有帮助:

> 林迪效应是一个概念，即一项技术或一个想法的未来预期寿命与它们当前的年龄成正比，因此每一个额外的存活期都意味着更长的剩余预期寿命。

应用于书籍:

> 如果一本书已经出版了 40 年，你可以预期它还会出版 40 年。但是，这是主要的区别，如果它能再生存十年，那么它将有望再出 50 年。

这就是我选择编程语言、工具和框架的方式。一项技术上市的时间越长，对我来说就越有利。新生事物有很大概率会消亡。让时间来决定哪种技术值得投资。时间是你最好的顾问。学会等待。

-

当我还是一名 Java 开发人员时，我将大部分时间投入到学习 Java 及其生态系统上。尤其是框架。我认为我了解的框架越多，我就是越好的开发人员。后来我遇到了我的导师，他说:

> 艾德，你在做什么？
> 
> –(自豪地)我正在读一本关于如何使用 GWT 构建 Java 应用的书
> 
> 为什么？
> 
> –显然是因为我是 Java 开发人员，想跟上潮流
> 
> –你读过的上一本潮流书籍是什么，GWT 之前的那本？
> 
> 这是一本关于 Tapestry 框架的书。挂毯是个东西。
> 
> 挂毯还是一个东西吗？
> 
> 不完全是。现在 GWT 是一个东西。
> 
> –你能使用 Tapestry 技能解决当前公司的问题吗？
> 
> 没有。
> 
> Tapestry 知识有助于理解 GWT 吗？
> 
> –几乎没有，但我看到了一些重叠的模式。
> 
> –是的，它们被称为设计模式。你能用它们来解决当前的问题吗？
> 
> 是的。(打勾！)
> 
> –趋势来来去去，但它们有很多共同点。正确设置优先级。**把 80%的学习时间投入基本面，20%投入框架等不稳定的东西**。记得优先考虑你不知道的东西。总有一天你会说谢谢。

这是一个改变人生的建议。我扔掉了大部分“框架书”，开始学习基础知识。不仅仅是基本面，还有我没有经验的基本面。比如模块化，干净的代码，简单的规则，测试驱动的开发，团队合作，心理学。

那是 2008 年。从那以后，我和 60 多个团队合作过。我感谢我的导师。因为我获得的 80%的知识在今天仍然是相关的。它可以跨公司、团队和领域移植。

➡️ 🔥最重要的技能是**知道如何与不同的人打交道**。现在去把非暴力沟通加入你的书架吧。

### [](#thought-3you-can-do-what-you-love%C2%A0)🧠思想 3——你可以做你爱❤️的事

我来自🇱🇻拉托维亚，这个神奇的国家首都有近 60 万人口。当我离开全职工作时，我不想再成为雇员。但我几乎找不到一个客户来咨询或培训。市场太小了。

我认为我的技能不受欢迎，我感到很痛苦。我是一名高薪员工，所以我被逃离不确定性，回到舒适区，重新成为一名员工的机会所诱惑。

但是我忘了一件重要的事情。熟练的开发人员总是受欢迎的。基本上，市场为我们服务。我们承担得起风险。如果我们失败了，我们总能找到工作。也许这不是理想的工作。但是如果我们梦想一份工作，也许我们的梦想不够远大？🤔

所以我决定拥抱不确定性，并遵循我最喜欢的一本书《反脆弱:

> 侵略性加上偏执狂的结合——抑制你的缺点，保护自己免受极端伤害，让优点，积极的黑天鹅，自己照顾自己。

坏处是什么？破产，付不起房租。所以我通过和前雇主保持良好关系来减少负面影响。以及削减开支。

就在那之后，我踏上了一段充满不确定性的旅程，**希望**有巨大的上升空间。

我开始在世界各地寻找客户。我学的是网络营销和销售。我开始公开演讲。我在伦敦呆了大约 3 个月来建立联系。我提供免费的咨询来寻找我的第一批客户。我创建了一个技术社区。我开了一个会议。我开始训练。我甚至卖掉了我漂亮的奥迪来付房租。🚘😭

经过反复试验，我意识到这个世界很大，没有国界。它比我想象的更大，更多样化。无论你宣扬什么，无论你擅长什么，你总能找到你的客户，过上体面的生活。这只是需要时间，一点勇气和适应不确定性。

你可以是培训师、顾问、会议组织者、初创公司创始人、宇航员等等。没有理由不做自己喜欢的事情。世界之大，回报之大。不要害怕去寻找(和做)你喜欢的事情，❤️.机不可失，时不再来。邦·乔维知道。🎸

### [](#bonus-thought)奖励思想

我相信我们是被我们周围的环境塑造的。我很幸运有导师，可以一起出去玩的人，没有废话的信息空间(我不看电视，从不在社交网络上争论，我最喜欢的社交网络是 [Goodreads](https://www.goodreads.com/eduardsi) )。

记住——环境塑造了你。

如果你周围有合适的人和高质量的信息空间，你可以成为你想成为的任何人。你有能力创造或找到一个让你成长的环境。

> 当一朵花不开时，是你修复了它生长的环境，而不是花。

有时这意味着获得新的联系，切断那些让你沮丧的联系。正如桑德罗·曼库索在《发展》杂志上的名言:

> 你不能总是改变你周围的人，但你可以改变你选择与之相处的人。

有时候，这意味着更喜欢一家有鼓舞人心的导师的公司，而不是一家稳定、富有的公司。

有时候意味着说再见，用自己的方式做事。

记住——我们的环境塑造了我们。对你的环境负责。成为他人的好环境。
