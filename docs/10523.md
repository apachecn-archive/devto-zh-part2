### [](#what-is-bootstrap)什么是 Bootstrap？

[Bootstrap](http://getbootstrap.com/) 是一个开源工具包，用于开发 HTML、CSS 和 JS 网站。对于它是否是一个框架或库仍然有一些争论，这对我们来说并不重要，不是吗？

### Bootstrap 为什么这么受欢迎？

一句话；Bootstrap 功能多样、简单且使用非常灵活。对 bootstrap 和 CSS3 的良好了解和结合将为开发阶段的开发人员和现场观看时的最终用户带来非常流畅和响应迅速的用户界面。

所以让我们开始吧。

# [](#5-important-things-about-bootstrap)关于 BOOTSTRAP 的 5 件重要事情

1.  **REBOOT(Reset Sheet)**:REBOOT 是一个引导 CSS 重置样式表。重启基于 [Normalize.css](https://necolas.github.io/normalize.css/) 。它完全去除了大多数元素中的顶部边距，它包含了 CSS 的继承值。你可以在这里了解更多

2.  响应系统:如果使用得当，Bootstrap 的响应能力非常强。它利用两种响应模式，并根据哪种服务更好，在各个组件上交替使用。

    *   网格系统:Bootstrap 利用移动优先的 flexbox 网格和十二列宽度系统来构建各种形状和大小的布局。它的响应能力分为五级`-` `-sm` `-md`等。
    *   Flex 系统:Bootstrap 使用 flex 系统来快速管理网格列、导航和组件的布局、对齐和大小调整。
3.  **响应断点** : Bootstrap 包括五组预定义的类，用于构建复杂的响应布局。这些类别涵盖超小型、小型、中型、大型或超大屏幕宽度。

4.  **组件** : Bootstrap 有很多方便的组件，你可以从中选择并轻松集成或修改，以满足你的需求。这些组件中的大多数都带有默认断点，以适应所有的屏幕宽度。这对于快速原型开发大有帮助；一个导航条在这里，一个大屏幕在那里，一些表格和容器在这里和那里，你可以走了。
    最棒的是，一些组件已经整合了它们的 javascript 行为，所以你不需要担心为它编写功能。
    这些组件包括:

    *   列表组
    *   模型
    *   下拉菜单
    *   警报
    *   旋转木马等。
5.  **Utilities** :这是 Bootstrap 我个人觉得有意思的部分，因为这个部分其实是负责编写较少的 CSS 样式的。它们也是可修改的；例如，如果一个实用程序`color`有一个值为`#14acf2`的`primary`类，由于**的重启**，你可以通过将 CSS 中的`primary`类设计成你想要的值来覆盖它。所以你的 CSS 将只是用于定制和 bootstrap 类不能处理的范围。一些非常方便的实用程序(我几乎每次都使用)是:

    *   边界
    *   颜色
    *   显示
    *   弯曲
    *   配置
    *   文本
    *   边缘
    *   填料

**给自己注**
`No matter how high you fly with bootstrap you will still need to land with CSS`。

让我解释一下:
不要试图通过拒绝编写任何样式来挫败 CSS 的目的，因为你将构建一个灾难性的网页，bootstrap 的本质不是停止编写 CSS，而是减少冗余和重复，以及我们的样式表的大小，知道它有多难调试。如果您需要编写样式，请尽一切办法编写您的样式表。

我还建议，如果你是一个 CSS 的铁杆粉丝，并且不想使用 bootstrap，你至少应该阅读并了解它，这不是徒劳的知识，因为它可以帮助你更好地理解 CSS 命名约定，同时反过来将提升你的代码可读性和风格。
