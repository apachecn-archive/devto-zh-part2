#### CSS&SCS

##### [CSS 布局入门](https://www.smashingmagazine.com/2018/05/guide-css-layout/)

对于 CSS 新手来说，这是一个很棒的指南，但是对于那些感觉没有掌握 CSS 布局或者没有跟上 CSS 布局最新发展的人来说也是如此。向您介绍 web 上的基本流程、浮动层和定位，然后介绍 Flexbox 和 Grid。非常好。

##### [用 CSS 解决人生难题](https://css-tricks.com/solving-lifes-problems-with-css/)

几周前，我链接了一篇由纯 CSS 肖像作者写的文章，该文章稍微探讨了 CSS 在 web 技术堆栈中的力量和位置，但也介绍了她是如何做到这一点的！我的意思是真的-那幅肖像是惊人的，你应该看看这篇文章。:D

##### [Z-Index 讲解:如何使用 CSS 堆叠元素](https://medium.freecodecamp.org/z-index-explained-how-to-stack-elements-using-css-7c5aa0f179b3)

Z-Index 是 CSS 的元素之一，看起来应该很简单，但是当你真正开始使用它时，会有各种各样的复杂情况。理解细微差别，尤其是堆叠上下文的概念，是让 z-index 成为你的朋友而不是沮丧地盯着你的屏幕的关键。这篇文章很好地解释了。

##### [知道你的唱段:“隐”vs“无”](http://www.scottohara.me/blog/2018/05/05/hidden-vs-none.html)

深入探究对屏幕阅读器隐藏内容的不同方式的细微差别。你知道`aria-hidden`和`role="none"`的区别吗？如果没有，你应该看看这个！(别难受，看这个之前我也不知道)。

##### [SCSS 可持续发展的 80-20 方法](https://dev.to/kball/the-80-20-approach-to-sustainable-scss-4g74)

(偏见警告——这是我自己的帖子)我是过去几年我们在 css 架构方面取得的巨大进步的忠实粉丝，这些进步使我们能够设计出真正强大的系统，这些系统比以前的 CSS 更容易维护，但这是以 CSS 和 SCSS 有时看起来让新人望而生畏为代价的。绝大多数项目不需要任何超级复杂的东西，所以这篇文章试图打破你可以做出的两个快速决定，这将带你走向可持续发展的 SCSS，而不需要深入的研究或疯狂的复杂系统。

#### JavaScript

##### [懒载动物模因与交集观察者](https://scotch.io/bar-talk/lazy-load-animal-memes-with-intersection-observer)

交叉点观察器 API 是我最喜欢的新浏览器 API 之一——当然，你可以用它来进行延迟加载，也可以用它来做各种有趣的事情，比如当某些东西滚动到视图中时添加动画，或者根据可见的内容有效地自动更新页面内导航。这个异想天开的教程带你通过它是如何工作的，并显示你想要的是可能的。

##### [ES6 在行动:增强的对象文字](https://www.sitepoint.com/es6-enhanced-object-literals/)

这是一个 Sitepoint 教程，介绍了我们在新版 JavaScript 中定义和操作对象的所有新功能。理解在一个对象中定义函数的所有新方法将帮助您快速理解 Vue 的单个组件文件语法(它非常依赖于此来导出定义所有组件逻辑的对象)，而其他功能，如参数中的默认值，无论您使用什么工具都非常有价值。

##### [Vue.js —注意事项和窍门](https://blog.webf.zone/vue-js-considerations-and-tricks-fa7e0e4bb7bb)

Vue 比很多 JavaScript 框架更容易学习，但是仍然有很多地方容易混淆或者不明显。这是一个很好的关于如何处理这些令人困惑/不明显的案例的资源的获取和链接。

##### [React Context API 能为你做什么？多语言文本、情态动词和主题转换器](http://www.js-craft.io/blog/15-What-can-React-Context-API-do-for-you-Multi-language-Modals-Themes/)

React 的新上下文 API 提供了一种简单的方法来跨多个组件共享状态，而不需要创建深钻道具，或者使用像 Redux 这样更复杂的库。是要完全取代 Redux 或者相关库吗？一点也不！但是对于需要一些共享状态的用例，不需要完全开放的 flux 架构带来的所有复杂性，这可能是一个很好的选择。这篇文章包含了一些你如何在你的应用中使用它的例子。

##### [JavaScript 启动优化](https://developers.google.com/web/fundamentals/performance/optimizing-content-efficiency/javascript-startup-optimization/)

我们都喜欢 JavaScript(好吧，也许不是我们所有人，但大多数人！)但是 JavaScript 是有成本的。Addy Osmani 的这篇指南向我们介绍了 JavaScript 成本的组成部分，尤其是在移动设备上(不仅仅是带宽！).然后，他介绍了一些方法，我们可以用这些方法来最大限度地减少手机上至关重要的互动时间。

#### 其他牛气

##### [玛丽·米克尔互联网趋势报告 2018](http://www.kpcb.com/internet-trends)

凯鹏华盈(Kleiner Perkins)的玛丽·米克尔每年都会发布一张幻灯片，记录互联网的发展趋势，这也是预测行业发展趋势的首选资源之一。对于那些在产品角色或初创公司的人来说，可能比纯粹的开发更有趣，但它太好了，不容错过，今年的报告刚刚上升！

##### [用现代前端技术让传统网站更具性能](https://www.lullabot.com/articles/making-legacy-sites-more-performant-using-modern-frontend-techniques)

我喜欢这个案例研究，因为它不仅揭示了许多优秀的前端性能技术，而且解决了我们许多人都遇到的一个问题:将这些技术应用于遗留代码库，而没有从头重写的奢侈。

##### [基于 React 的开源静态站点生成器 Gatsby 获得商业支持](https://thenewstack.io/gatsbyjs-the-open-source-react-based-ssg-creates-company-to-evolve-cloud-native-website-builds/)

我是静态站点生成器的超级粉丝(我的网站是基于 Jekyll 构建的，我也用 ZURB 栈构建了一些)，所以看到这个领域有更多的创新是令人兴奋的。Gatsby 是一个基于 React 的开源 SSG，它引起了足够的兴趣，核心开发人员也正在围绕它成立一家公司，以在云中提供并行构建。非常酷！

##### [2018 Node.js 用户调查报告](https://nodejs.org/en/user-survey-report/)

Node.js 基金会刚刚发布了第三次年度用户调查的结果。或许不出所料，他们发现 Node 用户喜欢使用 Node！:)说真的，超过 80%的调查受访者计划继续增加他们对 Node 的使用……在这份互动报告中还有很多其他有趣的花絮，或者如果你只是想要一份高水平的总结，你可以看看我在 InfoQ 上的[文章。](https://www.infoq.com/news/2018/05/node-user-survey-rapid-growth)

##### [动态贝塞尔曲线](https://www.joshwcomeau.com/posts/dynamic-bezier-curves)

超级酷的互动后期使用 React 和 SVG 创建动态贝塞尔曲线。它不仅很好地教你如何做，而且玩他的演示也很有趣。更不用说他的变形头球效果是 wicket 酷。
