# 从传统 PC 应用程序过渡到 Web 开发

> 原文：<https://dev.to/lyon/transitioning-from-traditional-pc-apps-to-web-development--1f34>

我有 6 年的 C#/专业经验。网络工作。(我学到的不仅仅是这些，但是为了这次讨论的目的，我把重点放在这一点上)

在短暂的教学工作之后，我正在重新寻找软件开发员的工作。当地的就业市场已经枯竭，很明显，我必须做两件事之一:搬家或远程工作。

搬到其他地方对我来说不是一个选择，所以我正试图冒险进入远程工作。到目前为止我学到的东西:很难找到我有专业经验的工作，如果不是不可能的话。我需要拥抱网络开发。

所以我开始钻研。当我拥有多年的工作经验时，不得不努力寻找入门级别的职位，这有点令人羞愧，而入门级的偏远职位本身就非常罕见。

我还了解到我需要建立一个 GitHub 组合。到目前为止，我的职业生涯一直在从事其他软件，我甚至不能 100%确定我应该为一个项目组合做些什么。我认为教程是不够的，因为它不是我自己的工作。

我知道 HTML/CSS 和一些 JavaScript，但知道这些还不够。我需要在各种流行技术(React，Ruby-On-Rails，Node.js 似乎经常出现)方面有 x 年的经验，这样我的简历才能在被求职者跟踪软件过滤之前到达人力资源部。

反正这是长篇大论。如果你读到这里，谢谢！我很乐意阅读你在职业生涯中实现飞跃的经历以及你的任何建议。
