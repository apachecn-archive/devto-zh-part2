**机器学习(ML)** 是每个人都在谈论的 dope 新事物，因为它真的很擅长从数据中学习，以便可以预测未来类似的事情。手工做 ML 很烦人，因为它通常涉及矩阵数学，而这在 JavaScript 中毫无乐趣可言(或者如果你问我:任何地方😅).谢天谢地， [TensorFlow.js](https://js.tensorflow.org/) 来帮忙了！这是一个开源库，有很多内置的机器学习工具，比如模型和算法，这样你就不用从头开始写了。

### 你的问题是机器学习问题吗？

机器学习擅长对数据进行分类和标注。每个机器学习问题的前提是:

*   有人给了我们一些数据，这些数据是根据一个**秘密**公式生成的。这些数据可能是一堆点(根据一些数学公式生成)，但也可能很有趣，比如图像(秘密公式可能是“这些图像中有些是吉娃娃，有些是 T2 蓝莓松饼”或公交时刻表)。
*   通过查看我们得到的这些数据，我们可以逼近秘密公式，这样我们就可以正确预测未来的数据点。例如，如果给我们一张照片，我们最终将能够自信地说出它是一只狗还是一块松饼。

### 一个好玩的 demo！

如果你想开始，预测数字往往比预测图像更容易，所以在这个例子中，我们试图用一条曲线来拟合一堆数据(这是来自 [TensorFlow](https://js.tensorflow.org/tutorials/fit-curve.html) 网站的同一个例子，但有更多的代码注释和更漂亮的图形)。

给我们一堆点(对于-1 到 1 之间的 x，根据 y = a * x<sup>3</sup>+b * x<sup>2</sup>+c * x+d 计算出一个 y——我们知道这是秘密公式但是我们不知道那些 a，b，c，d 系数的值。)我们的目标是学习这些系数，这样如果给我们一个新的 x 值，我们可以说 y 值应该是多少。

蓝色的点是我们得到的训练点。**红色**点将是我们的猜测，基于我们最初的默认系数(hella 不正确！).一旦你点击训练按钮，**绿色**点显示我们的系数如何变得更好。在你看到默认的例子之后，检查如果你改变数据的形状，或者我们得到更少的数据点或者更少的迭代会发生什么！

### 工作原理

大多数机器学习算法遵循这种模式:

*   我们必须弄清楚产生给我们的数据的秘密公式的**“特征”**，这样我们才能学习它们。在我看来，这相当于解决一个 ML 问题的 80%的复杂性。在这个例子中，我们被告知秘密公式的形状(它是一个立方体！)，所以我们要学习的特征就是多项式中的系数。对于像“这是一只狗还是一个蓝莓松饼”这样更复杂的问题，我们必须考虑像素、颜色和形状，以及是什么让一只狗成为一只狗而不是松饼。
*   一旦我们弄清楚了这些特征(在我们的例子中，是那些 a、b、c、d 系数)，我们就把它们初始化为一些随机值。我们现在可以用它们来做预测，但是它们是不可靠的，因为它们只是随机的。
*   (从现在开始，我将使用我们的实际例子，而不是狗)
*   我们开始查看给我们的每一份(x，y)训练数据。我们得到 x 值，基于这些我们估计的系数，我们预测 y 值是多少。然后，我们从原始训练数据中查看正确的 y 值，计算两者之间的差异，然后调整我们的系数，以便我们的预测值更接近正确值。
*   (这就是所谓的“随机梯度下降”，其中加入了更多的数学知识。“随机”的意思是概率性的，“梯度下降”应该让你想到走下一座小山，朝着一个陷坑走去——山越高，预测误差越大，这就是为什么你想朝着无误差的洞下降。)
*   这部分代码实际上相当混乱(因为矩阵和导数)，TensorFlow 为我们做了这些！
*   我们一直这样做，直到我们用完了所有的数据，然后重复整个过程，这样我们就可以一遍又一遍地迭代相同的数据，直到最后我们基本上学会了系数！
