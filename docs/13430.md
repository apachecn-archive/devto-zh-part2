# 3 您的项目中可能没有的友好文件。

> 原文:[https://dev . to/moopet/3-friendly-files-you-possible-not-have-in-your-project-40mm](https://dev.to/moopet/3-friendly-files-you-probably-dont-have-in-your-project-40mm)

所以你有一个 robots.txt 和一个 sitemap.xml 和一个许可证和一个 README 等等。但是你有什么东西给我们人类吗？

## humans.txt

如果你猜不到，humans.txt 是根据熟悉的 robots.txt 文件命名的，就像它的兄弟一样，它位于你的网站的根目录中。因此，它不太可能存在于非 web 项目中。

[![humans.txt logo](../Images/843ad9b1ba41559aea2462bde306083c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--UTR9dF0K--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/6thdsl0qopbdfg0t71yk.gif)

这是干什么用的？这是给你的。你可以在里面放任何你喜欢的东西- *这就是*它的用途:

*   相信你自己和你的团队。
*   大声喊出你最喜欢的文本编辑器。
*   把这个网站献给一个死去的同伴。

切记不要扒雇主的渣，没有客户的允许根本不要收录这个文件。

这是我的一个例子:[moopet.net/humans.txt](https://moopet.net/humans.txt)

[阅读更多关于人类的信息](http://humanstxt.org)

## foaf

朋友的朋友仍然存在，但它从来没有达到你需要的那种临界质量，使它看起来真的值得。这是文章、作者、主题和你能想到的东西之间的另一层联系；它比其他静态文件复杂得多。作为另一个特定于超文本的文件，它与非 web 项目无关。
