你真的一直做 TDD 吗？有时候？修复 bug 的时候？做 API 的时候，做 ui 的时候，什么时候？
或者你做 DDT，开发驱动测试？
好了，我刚刚补了最后一个。

我发现这是一个惊人的方法，我很少这样做。当我确切地知道我要编码什么的时候，是的，它就工作了。大多数时候，尽管我在代码中重复了太多，但感觉还是不对。我更喜欢完成我的代码，然后给它们添加测试。
这显然需要一些前瞻性的思考。

所以我会说:我认为在 TDD 中，我不做 TDD。

你们这些人呢？
