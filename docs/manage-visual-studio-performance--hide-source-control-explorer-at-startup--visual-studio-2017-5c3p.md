# 管理 Visual Studio 性能–启动时隐藏源代码管理资源管理器–Visual Studio 2017

> 原文：<https://dev.to/mandarbadve/manage-visual-studio-performance--hide-source-control-explorer-at-startup--visual-studio-2017-5c3p>

最后，我们可以选择在 2017 版中管理 Visual Studio 的性能。你可以从帮助菜单中看到它。

[![Visual Studio Help](../Images/9223e316aa6f1212de174bece9f03847.png)T2】](https://i0.wp.com/blog.mandarbadve.com/wp-content/uploads/2018/02/Visual-Studio-Help.png)

以下是管理性能的选项。

[![Visual Studio Help Modal](../Images/4dca330a4c64b110793de1b7ec55386d.png)T2】](https://i2.wp.com/blog.mandarbadve.com/wp-content/uploads/2018/02/Visual-Studio-Help-Modal.png)

它说，如果您设置它使用默认行为，即启动时加载源代码管理资源管理器，加载 visual studio 可能需要 4 秒钟。

推荐选项是:启动时不显示窗口。这将在 visual studio 启动时节省多达 4 秒的时间。
