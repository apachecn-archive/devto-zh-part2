## 我将从事什么工作？

谢天谢地，我已经有了一长串我一直想添加到 [Tron](https://github.com/HF-Solutions/Tron) 的东西，这是我很久以前开始开发的不和谐机器人。更具体地说，我已经收到了一些用户对机器人新命令的请求，比如收养命令，一些新的反应图像，以及一个潜在的[通知系统。](https://haveibeenpwned.com)

虽然我真诚地相信我可以在 **Tron** 上连续工作 100 天，并且仍然有更多的东西要添加，但我担心到最后我会讨厌我心爱的机器人，不再想为它工作😅因此，除此之外，我还想做一些其他的事情:

1.  扩展我的自由职业公司 **HassleFree Solutions** ，寻找新客户。

    *   因为这不是真正的编码，这不会是我每天 1 小时的实际时间。然而，它可以带来更多的项目。
2.  将现有的 **HassleFree Solutions** [网站](https://hasslefree.solutions)从 SquareSpace 转换为自托管解决方案。

    *   具体来说，我将使用 [Gatsby.js](https://www.gatsbyjs.org/) 静态站点生成器和 [Dimension](https://github.com/ChangoMan/gatsby-starter-dimension) starter 插件。
    *   我不打算再把博客放在网站上了，因为我所有的文章都贴在 dev.to 上，同时也保存在个人档案中。
3.  编写一个 Visual Studio 代码扩展来帮助我用 Markdown 写文章。

    *   这实际上是在写这篇文章时想到的。当添加链接时，我必须用括号将链接括起来，并回忆我最后使用的 URL 的数字是什么(3，6，9？)
    *   最好有一个简单的快捷方式，用括号将文本括起来，然后添加序列中的下一个数字。
    *   任何关于这方面的投入将不胜感激，因为我不会开始在这至少几个星期。
4.  扩展我的 PowerShell 模块， [Gamgee](https://github.com/Alcha/Gamgee) 以包含更多的函数，我已经为 Zeus 在 Trello 板中存储了不知道多长时间🤦‍♂️

## 我的规则

就像我之前看到的那些一样，我的规则将与挑战最初给出的两个规则“略有”不同。当然，我仍然会遵守这两条规则，与我注意到的大多数人的做法相比，我会做一点小小的调整:

1.  我注意到大多数人不希望把客户工作算在这个挑战中，这是可以理解的，原因很明显。然而，我将会把客户的工作计算在内，因为我还在做我的自由职业，我能为我的客户做的所有工作都是对我时间的极大利用。

2.  因为我还年轻，没有孩子，所以没有太多可以接受的理由来错过一天，但为了以防万一，我应该错过一天的唯一理由是:

    *   病入膏肓，不能下床(很少发生，但以防万一，我想有那个基地覆盖😅)
    *   家庭紧急事件
3.  我要加入最后一条规则，为 dev.to 写作(只要是技术文章，而不是无关紧要的文章)也可以算作一天中的一个小时，因为我通常会花大量时间为我的文章编写代码、调试和研究材料。
