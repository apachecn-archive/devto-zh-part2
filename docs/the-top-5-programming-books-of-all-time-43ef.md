# 有史以来排名前五的编程书籍

> 原文：<https://dev.to/redfred7/the-top-5-programming-books-of-all-time-43ef>

是什么让一本编程书真正伟大？浅见，两件事:

1.  内容质量
2.  内容交付

有很多聪明的人，他们对一些非常创新、复杂和有用的主题有很多知识。但是这些人中很少有人能够以一种易于理解和吸收的方式将这种知识以书面形式传授。写作风格和表达能力决定了知道一些事情和在写作中分享的区别。记住这一点，以下是我最喜欢的 5 本编程书籍(排名不分先后):

* * *

*   ![the C programming language](../Images/9c92332ef50300dbff7cbce88a2411b8.png)

这本书教了一代又一代的程序员 C 语言。这是一本严肃的书，简洁明了，但是告诉你你需要知道的关于 C 的一切。有些人会说，在这个时代，你不需要知道 C 来成为一名程序员。没错，但是你学习 C 不是为了成为一名程序员，而是为了成为一名更好的程序员。了解内存分配，指针如何工作，什么是 L 值...所有这些都让你对自己选择的编程语言有了更好的了解。用一个同事的话来说“在你知道 strcmp 和 strcpy 是如何工作的之前，你不能真正编程”。

*   [![Thinking in C++](../Images/7739680816f1c794dd9670cc45071cbf.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--XMRO_-Tp--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://images-na.ssl-images-amazon.com/images/I/61V67pi5UGL._SX377_BO1%2C204%2C203%2C200_.jpg)

    在这本书问世之前，大多数 C++(和其他编程)书籍都是作者毫无疑问的专业知识的简单知识宝库。布鲁斯·埃凯尔选择了一个非常复杂的主题(C++)，将其分解为基本原则，并以一种非正式的、对话式的方式解释每一个原则，这使得即使像我这样没有 Stroustrup 或 Scott Meyer 那样丰富智力的读者也能理解。C++为您的普通乔程序员！

*   [![Design Patterns in Ruby](../Images/9d2a2d522cbe76a010f805703fc1845c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--uUL5z_tn--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://i.gr-assets.com/images/S/compressed.photo.goodreads.com/books/1347345993i/2278064._UY630_SR1200%2C630_.jpg)

    这本书试图在介绍和展示 Ruby 编程语言的同时，让读者对设计模式有一个很好的理解。与 GoF 设计模式书不同的是，这本书是以对话和渐进的方式编写的，提供了关于如何以及何时应用设计模式的相关的真实世界的例子，并且还提供了一个关于模式如何仅仅是当前框架和语言约束的产物的很好的视角。不管你用什么编程语言，这都是设计模式的必读书目。

*   [![JavaScript: the Good Parts](../Images/9763869403c1285a769d0f9c72add6de.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--YnIOau6r--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://images-na.ssl-images-amazon.com/images/I/5131OWtQRaL._SX381_BO1%2C204%2C203%2C200_.jpg)

    这本书将一种因其不一致、丑陋的语法和设计缺陷而臭名昭著的语言剥离出来，呈现给你一种优雅、富有表现力和有效的语言。我还能说什么？

*   [![Refactoring](../Images/41adfef09e4bf58123b10755fce77731.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--nbTMk6mf--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://images-na.ssl-images-amazon.com/images/I/51K-M5hR8qL._SX392_BO1%2C204%2C203%2C200_.jpg)

    当这本书第一次出版时，像“重构”、“代码气味”和“TDD”这样的术语还停留在软件社区的边缘，被少数怪人实践和理解。这本书解释了为什么我们应该注意，以及如何在我们的日常编程中应用这些概念。一本改变思维过程的书。
