**Web 提取和 XPATH 初学者指南**

XPATH 是一种查询语言，用于选择 HTML 或 XML 文档的节点并计算其值。对于正在阅读这篇文章的一些人来说，很多信息都是抛给你的。你在这里可能是因为你掉进了网络提取的兔子洞，或者广泛称为数据搜集，在这一点上你只是四处点击。停下来。您已经到达了 web 提取和 XPATH 入门指南。让我们分支讨论什么是 web 抽取，以及抽取大部分数据所需的两个要素。

**数据抓取**

Web 抽取，又称 [web 抓取](https://www.webscraper.io/) 和 Web 收割，是用于从网站中抽取数据的数据抓取。抓取可以手动完成，也可以通过网络爬虫和机器人自动完成。这些数据是从互联网上收集的，并被复制到一个中央本地数据库或电子表格中。谷歌和亚马逊 AWS 免费提供网页抓取工具和公共数据。虽然这听起来有点侵犯性，但是有很多日常用法，比如:

*   在线价格变化监控
*   价格对比
*   房产列表
*   气象数据监控
*   产品审查

尽管如此，网站仍在寻找新的方法来阻止 [网络爬虫](https://www.sciencedaily.com/terms/web_crawler.htm) 和机器人提取数据。数据收集系统也在发展，但有时需要和需要人的参与。这就要求用户能够读写网络语言。

**网络语言**

网页是使用基于文本的标记语言构建的，例如 [HTML 和 XHTML](https://www.w3schools.com/html/html_xhtml.asp) 。当 web 浏览器接收 HTML 文档时，它们将它们呈现为多媒体网页。HTML 从语义上描述了网页的结构。在文本形式中，要通过离线解析的信息量很大。理解网络语言将为你充分利用网络抓取工具、机器人和网络爬虫打下基础。它还可以让你自己真正地阅读、书写和解决你所寻找的问题。仅举几个例子，这里还有一些其他的标记语言:

*   BNML(商业叙事标记语言)
*   CFML (ColdFusion 标记语言)
*   InkML
*   VoiceXML
*   XBEL (XML 书签交换语言)

**根、树和 XPATH**

在 [使用 Xpath:Xpath 教程](https://www.import.io/post/xpath-tutorial-intro-to-xpath/) 的介绍中，确立了 Xpath 是 web 的另一种语言。XPATH 基于 XML 文档的树表示。基于 XML 文档的层次结构，它们可以被解释为类似于树结构。XML 树必须有根元素。根元素是所有其他元素的父元素。每个元素可以有子元素。在处理 XPATH 时，元素被称为节点。XPATH 允许用户在树中导航并选择特定的节点。XPATH 的选择和计算能力是必不可少的，因为一些子节点可能包含值、条件、文本甚至其他数据结构。

如果你打算从事网络工作，学习语言是必不可少的一步。随着网站开发者找到新的方法来阻止机器人，了解这种语言将使你找到一个工作。在搜集数据时，知道并理解你在找什么是至关重要的。使用 XPATH，当您找到包含值的节点时，您可以毫不浪费地计算它们。用网络语言来建立你的基础会让你对树形结构有更深的理解。对树结构的深入理解会让你立刻收集大量的数据。
