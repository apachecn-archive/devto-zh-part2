# 本周(2018 年 3 月 16 日)我学到了什么

> 原文：<https://dev.to/jrop/what-i-learned-this-week-march-16-2018--2ee3>

在我工作的地方，我非常重视的一件事是，每周，我们的主管会发电子邮件给我“本周我学到了什么”。这让我想到，对我来说，反思我所学到的东西的一个好方法是每周五发表同样的东西。

因此，不再拖延，我向你们展示我本周学到的东西:

## 常见主题

我们最近在我工作的地方成立了 DevOps 委员会，我们一直在讨论未来六个月的发展方向。考虑到这一点，我个人一直在审查以下技术，以寻求可能的建议，作为我们可以研究的内容:

## 通过迷你库布的库布

上次我研究学习 Kubernetes(被社区亲切地缩写为`k8s`)时，它非常难以进入和学习，因为仅仅是为了实验就必须构建一个小型集群沙盒。想象一下，当我得知有一个工具 [minikube](https://github.com/kubernetes/minikube) 可以让你在你的开发机器上本地使用 Kubernetes 时，我是多么兴奋！说真的，去装吧，开始玩 Kubernetes。

## [哈希公司](https://www.hashicorp.com/)

哈希公司因开发了神奇的工具“流浪者”而闻名。事实证明，他们还有很多其他很棒的项目:

*   terraform——用代码表示你的基础设施
*   nomad——一个非常棒的部署工具
*   保险库-秘密管理工具

这些工具的主要优点是它们是模块化的和模块化的。然而，这也是他们的弱点:为了和他们一起玩，还需要更多的设置。

## [Ansible](http://docs.ansible.com/ansible/latest/index.html)

本周我还重新发现了 Ansible。通过重新发现，我的意思是我更深入地研究了它的工作原理，并再次爱上了它。我们已经在我的工作中使用了这个工具，但是到目前为止，它已经在我们的系统支持团队中取得了很大的成功。本周，它终于“点击”了 Ansible configuration 是如何工作的，现在它不再是一个如此(伟大)的秘密。我肯定会更频繁地将它集成到我的更多工作流程中。

## 结论

我只是想保持这些简短和甜蜜，所以这就是所有的人！
