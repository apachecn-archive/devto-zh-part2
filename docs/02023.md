这里有一个关于 DevOps 的一些常见误解的简单列表，以及为什么它们是错误的。

## “devo PS 不仅仅是工具”

### [](#false)假！

*   如果不仅仅是工具，为什么我们会有这么多工具？
*   没人在乎你的文化
*   DevOps twitter 上有很多工具

## “devo PS 不只是自动化”

### [](#false)假！

*   CAMS 语中的“A”是唯一的元音
*   元音比辅音更重要
*   因此，自动化是最重要的

## “Kubernetes 不会解决你所有的问题”

### [](#false)假！

*   Kubernetes 来自谷歌
*   Docker 又无聊又老
*   YAML 展示了你对精确的承诺

## [](#if-you-devops-you-still-need-testing)“如果你开发了，你还需要测试”

### [](#false)假！

*   消除人为因素和人为错误
*   如果你的微服务只做一件事，那就不能有错误。这将是一个额外的事情
*   反正 QA 的人都很烦

## [](#you-cant-buy-devops)“买不到 DevOps”

### [](#false)假！

*   你可能买不到，但我可以卖给你
*   说真的，我很想卖给你一些德文
*   怎样才能让你今天穿上 devops？

## “devo PS 不是头衔”

### [](#false)假！

*   没有人能告诉你在你的组织里应该使用什么头衔
*   我想你的人力资源部的人可以
*   但是 HR 到底知道些什么？

## [](#you-can-do-devops-on-windows)“可以在 Windows 上做 DevOps”

### [](#false)假！

*   DevOps 的一个关键组成部分是沾沾自喜的自我满足感
*   Windows 都是点击式的，对吧？
*   某物某物[斜线圆点](https://slashdot.org)

## [](#cats-are-more-aligned-with-devops-than-dogs)“猫比狗更符合 DevOps”

### [](#false)假！
