# 怎样才能成为一名优秀的前端开发人员？

> 原文:[https://dev . to/zellwk/what-makes-a-good-frontend-developer-4g4a](https://dev.to/zellwk/what-makes-a-good-frontend-developer-4g4a)

前端开发行业现在越来越混乱。更多的人认为前端开发人员是拥有很棒的 JavaScript 技能的人。

但是前端开发不仅仅是 JavaScript。

作为专业的前端开发人员，我们需要了解我们的工作是什么。

## 一个前端开发者是做什么的？

前端开发人员只做一件事。

我们为访问我们网站(或应用)的用户提供良好的体验。

我们知道我们的网站(或应用程序)应该做什么。我们精心制作它们(有时是从零开始)，并在发送给我们的用户之前对它们进行润色。

我们投入数小时的工作来完善我们建立的网站。只是希望用户会喜欢我们的网站。如果他们喜欢我们的网站，我们希望他们会再次访问我们。

但是伟大的经历意味着什么呢？我们需要哪些技能来为用户提供出色的体验？

## 一个前端开发者需要的技能

我认为前端开发人员需要掌握四种不同的技能。

1.  神入
2.  密码
3.  设计
4.  沟通

### 感同身受

什么是很棒的体验？

除非我们了解谁是用户，否则我们不会知道。

在互联网上，我们为各种各样的用户提供服务。他们和我们不一样。

人们使用不同的电脑。有的人用 4k 显示屏的 Mac，有的人用 Windows 机，还有的人用没有很好计算能力的旧手机。

人们使用不同的浏览器。Edge、Chrome、Safari 和 Firefox 是世界上最常见的四种浏览器。但是浏览器的数量比 T1 多。

人有不同的能力。有些人视力很好，有些人戴眼镜，有些人是色盲，有些人根本看不见。

人也有不同的喜好。有些人用鼠标浏览，而有些人更喜欢键盘。

关键是，每个人都不一样。

**我们在这里提供每个人都喜欢的体验**。

我们需要明白人们和我们不一样。即使他们与我们不同，我们仍然为他们提供了很好的体验。

“它在我的电脑上不起作用”不是我们可以逃避的借口。如果它对其他人不起作用，我们会深入挖掘，找出问题所在，然后解决它。

### 代码

我们需要了解 HTML、CSS 和 JavaScript。

是的。所有三种语言。HTML、CSS 和 JavaScript 是我们的工具。如果我们不了解自己的工具，如何成为优秀的前端开发人员？

但是当我们谈论代码时，许多开发人员关注的是错误的东西。他们关注代码质量。

我认为这与法典无关。如果体验很糟糕，代码是否干净和维护得好也没关系。见鬼，如果你想写最干净的代码，就不要写任何代码。提供一个没有 CSS 和 JavaScript 的 HTML 文件。我保证会干净的。

代码质量很重要，但我们提供的体验更重要。

当我们给每个人一个好的体验时，我们可以把注意力转向内部，清理我们的代码。

有经验的开发人员可以在制作网站的时候写出干净的代码。这是他们通过经验学到的技能。

年轻的开发人员应该首先关注他们能够提供的体验。一旦他们确定了经验，他们就可以学习改进他们的代码。我们可以两全其美。

### 设计

设计很重要。

当我谈到设计时，我指的是视觉设计和交互设计。

我们是从事设计的最佳人选，因为我们知道网络是如何运作的。我们知道什么能做，什么不能做。我们也知道人们如何使用网络。我们足够关心让自己保持更新。

我们不是来接管设计师的工作的。他们有他们的技能，我们有我们的。我们相互补充。

我们可以评论设计师的想法。

这让我想到了最后一个技巧。

### 沟通

开发人员素有反社会怪胎的名声。我们更喜欢整天坐在电脑后面敲代码。如果可以，我们甚至不想和任何人说话。

如果我们不想和任何人说话也没关系。

但是如果需要的话，我们必须能够沟通。以下是一些原因:

1.  如果你和一位设计师一起工作，*如果你不知道如何和他们交谈，你怎么能评论他们的设计(或指出一些东西)*？
2.  如果你和其他开发人员一起工作，*你如何分配工作*？你如何决定在项目中使用什么样的实践？
3.  如果你和一个项目经理一起工作，*你如何解释为什么一个特定的功能需要这么长时间*，或者为什么某些功能不能完成？你如何说服他们？
4.  如果你和用户交谈，*你如何发现他们在你的网站上遇到的问题？*

以上几点只是一些可能的原因。我相信你能想到更多。

关键是，如果我们想获得人们的认同，我们需要知道如何与他们交谈。如果我们不能很好地与任何人沟通，我们如何能创作出最好的作品？

## 包装完毕

很难擅长前端开发。我们需要根据情况戴上不同的帽子，我们需要很好地使用所有这些帽子。

这可能很难，但这是不可能的。前端开发是一项需要时间来掌握的技能。

我现在甚至还没有掌握前端世界的一切。我还在朝这个方向努力。

我自豪地称自己为前端开发人员。我希望我是个好孩子。
