# 自学的优势

> 原文：<https://dev.to/ajsmithsw/the-advantages-of-being-self-taught-2lcj>

在最近的一次 meetup 活动中，我与一小群与会者交谈，其中一人是一名有抱负的自学移动开发者。他充满激情和热情，但他心中有些疑虑。他一直在关注我们的谈话，当时我们的谈话充满了与技术相关的术语，他承认自己很难跟上。他担心自己缺乏经验，或者没有计算机科学学位，这意味着他在寻找第一份技术工作时处于非常不利的地位。

一年前，我处于完全相同的位置，这次谈话让我意识到，这些“缺点”最终都不会对我的职业前景产生不利影响。相反，我认为自学成才的开发人员有几个优点。

### 沟通技巧

敏捷技术团队很少纯粹由开发人员组成。你不仅要知道如何编码，而且你经常需要向产品经理、BAs、销售、市场甚至客户和顾客解释复杂的技术问题，这取决于你公司的规模。在一个开发环境中，很容易花 99%的时间与已经对您的挑战有深刻理解的其他开发人员交流。在调整你的语言以适应你的非开发同龄人的词汇方面，没有很多机会获得经验。

如果你是自学的，很可能你已经在另一个行业工作过。无论是在零售店为客户服务，还是在一屋子人面前展示统计数据，您都已经获得了大量宝贵的经验，能够与来自各种背景的人产生共鸣并进行交流，从而更容易让整个团队了解您在做什么以及为什么要做，并能够提出问题，以便您能够完全了解业务的需求。

### 学习速度更快

作家马尔科姆·格拉德威尔普及了一种理论，即要成为任何领域的专家都需要 10000 小时的练习和努力。这是一个不可靠且受到强烈质疑的理论，但“熟能生巧”这句箴言已经被应用于许多学科。如果你用这个来自学如何编码，你会期望你投入学习的时间意味着随着时间的推移你会成为一个更好的程序员。可能不太明显的是，你也在自学如何学习。你不会一页一页地跟着课本走，也不会在提交截止日期前苦苦挣扎。你正在发展自己的学习节奏，弄清楚如何快速掌握困难的概念，学习如何在没有导师或导师帮助的情况下找到问题的答案。你在犯错误，很多错误，而不是有人向你指出它们，你在经历它们的后果，并学习如何适应这些失败。

一旦你得到了第一份技术工作，你就会一头栽进你没有准备好的事情中。但是你在学习如何学习上投入的时间意味着你会很快适应，并且马上就能站起来。

### 职业技能

我花了很多时间和计算机科学的毕业生一起工作，要么是在他们的“工业年”，要么是作为毕业生就业计划的一部分。让我一直感到惊讶的是，他们的大学课程似乎都没有教授基本的职业技能，而这些技能几乎是每个开发人员在公司工作时都需要的。他们学习复杂的算法和概念，比如图灵完备性，但是他们并不被鼓励去学习诸如版本控制、单元测试、DevOps 或者如何编写干净的、可维护的代码。

自学的方法让你没有一个很好的丰富的资格来充实你的简历，所以另一种方法是建立一个出版作品的组合，要么将应用程序发布到产品中，要么在 GitHub 上发布你的代码(理想情况下两者都有)。如果你已经发布了自己的应用程序，那么你必须学习整个过程，从文件->新建一直到部署。在 GitHub 上维护一个投资组合至少需要基本的版本控制知识。当你试图将产品推向市场时，你会遇到很多公司面临的挑战，所以一旦你开始工作，你就会在这些事情上一帆风顺。

### 工作满意度

想从事软件开发职业的最可能的原因是因为你对编码有真正的热情。自学编程并不容易，只有当你深深喜欢它时，你才会坚持下去。如果你到了想把爱好变成职业的阶段，那么得到第一份工作会变得更加甜蜜；为了成为一名出色的程序员，你牺牲了太多的空闲时间，现在你在这里，做你喜欢的事情并获得报酬。这是一种不可思议的感觉，它激励你去做令人敬畏的工作。这让你对雇主来说非常有价值。
