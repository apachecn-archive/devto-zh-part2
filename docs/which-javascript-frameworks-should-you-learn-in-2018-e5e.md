# 2018 年应该学习哪些 JavaScript 框架？

> 原文：<https://dev.to/samerbuna/which-javascript-frameworks-should-you-learn-in-2018-e5e>

[![](../Images/01bfe63ff4092f1d0bd91befec78c256.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--PrxsE7Ph--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1600/1%2ApPJq-yvXA13X4g87Lc4NJQ.png)

让我先定义一下我认为的编码框架是什么，因为这个词是有争议的。我认为框架是一个库，它定义了关于每个方面或层的结构，可以使创建应用程序的任务更容易。所谓层，我指的是像数据库、模型、控制器、视图、演示者、网络等等..

框架试图解决它们的应用程序通常遇到的大部分已知的大问题。它们有内置的设计决策，您无需担心。他们也有精心制作的指导方针。

好的框架也有智能默认值，并遵循配置概念的惯例。这方面的领导者是 Ruby on Rails 框架，这是我最喜欢的框架之一。在 JavaScript 领域，领先的 JavaScript 框架是 **Angular** 。(不是 **React.js** 因为[我没把 React.js 当成框架](https://dev.to/samerbuna/yes-react-is-taking-over-front-end-development-the-question-is-why-4mad))

* * *

既然明确了这一点，那我来回答题目中的问题:**2018 年你应该学习哪些 JavaScript 框架？**

我的回答是:没有。

我来解释一下原因。

除非你从事的工作需要你使用框架，否则不要为了学习而学习框架。我说的是学习如何使用框架。这是一种很快就会失效的知识。

另一方面，你应该学习**框架如何工作**。这是非常宝贵的知识。如果你学习框架的动机是为了找到一份好工作，那么知道框架如何工作远比知道如何使用它们更令人印象深刻。

> 学习如何只在你真正需要的时候使用框架。

我认为，要成为更受欢迎的开发人员，还有更重要的技能需要学习:

*   尽可能地了解 JavaScript 语言本身，它的优缺点，以及它最近获得的所有现代特性。了解如何在 JavaScript 中构建和使用各种数据结构。
*   了解像节点和浏览器这样的 JavaScript 运行时，并了解它们是如何单线程的。了解它们的 API 和局限性。了解事件循环。了解虚拟机的调用堆栈。熟悉浏览器的开发工具。
*   了解如何在客户机和服务器之间共享代码以及预加载初始数据。了解如何最小化浏览器中的 JavaScript 加载和解析。了解如何按需加载 JavaScript。
*   学习函数式编程的优点，并尽可能使用它的概念。尽可能学习如何声明而不是命令。
*   了解小型 JavaScript 库，它们做一件事并且做得很好。选择 API 最小的库，不要关注 API，而是关注这些库能让你做什么。
*   了解如何构建可伸缩的数据 API(看看 GraphQL)。
*   了解 CSS 的强大功能以及如何使用它来最小化应用程序的 JavaScript 代码。了解新的 Flexbox 和网格布局。了解用户界面响应设计。
*   了解如何使用 TypeScript(或 Flow)将静态类型添加到 JavaScript 中，并了解应该将重点放在类型和测试的哪个方面。
