# 你如何应对胜人一筹的情况？

> 原文：<https://dev.to/brianbola/how-do-you-deal-with-one-upmanship--bn9>

[![](../Images/89957a05b858e4bef0d3216700ca0291.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--IUSHnmw---/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/e8v425nhjeqpixscftce.jpg) 
我先从这个说起。

*“电脑容易，人难”*

我们都经历过..你在 slack 或 hangouts 或 reddit 上..你试图讨论或强调某人可能会觉得有用的东西。你正在做的某个项目，或者你开发的某个工具，来节省重复工作的时间，或者第三方应用程序..你认为有价值的东西。

然后你得到的是 ***【专家】*** 的意见，这些意见没有提供反馈或改进。只是一个悬而未决的声明，强调一个不相关的观察。

这通常是一个胜人一筹的时刻。这些贬低是驱动开发社区中的骗子综合症的原因。

你如何将这些讨论引回到建设性的对话中，而不是变成一场自私的游戏？

我并不想在这里表现得很刻薄，但是在开发文化中确实存在这样的问题，这种问题会阻碍学习的心态。

我绝不是在贬低自己是某个学科的专家。但是，有了这些专业知识，最好还是把你的知识用于更大的社区。
