# 我的第一次会议

> 原文：<https://dev.to/jamessouth/my-first-conference-2icn>

我在最后一刻决定去，当时 Progress Software 慷慨地给了我一张免费票(谢谢！).Progress 正在波士顿举办第一场为期两天的双轨制 jsMobileConf。很棒的演讲者，很棒的话题，完美！我是在假期前的第一天到达的——我来这里是为了学习和结识朋友，并获得一些求职的想法。我有技能、动力、天资和一份[投资组合](https://jamessouth.github.io/Project-12/)，但还是缺少了一些东西。我拿到了我的徽章和一小盘切好的水果。

我走近几个开发人员，说“嗨”。他们来自一家为自闭症教育制作软件的公司，该公司赞助了当晚的欢乐时光。我告诉他们我一直在努力成为一名开发者。然后一个做过一些合同工作的开发人员加入了我们。他建议我研究相同的东西以获得一些经验，并建议我选择像 React 这样的东西，并专注于它以找到一份工作，而不是我目前的覆盖海滨的方法，在这种方法中，我一直在尽可能地转向新的东西。

午餐前的谈话时间到了！我参加了一个关于 JavaScript 中二进制数据的讲座。现在，我们都知道我们的 1 和 0，但是这里我们分解了组成 GIF 的代码！相当整洁！二进制也比 JSON 有一些优势(序列化速度/大小),这使它有时成为更好的选择。太棒了。

接下来是关于 [NativeScript](https://www.nativescript.org/) 的演讲，通过它你可以使用 JavaScript 构建本地移动应用。我认为它太酷了，我将为我的下一个项目学习它(以及 Angular 和 TypeScript)。我已经有主意了。所以我猜[的第三或第四反应项目](https://github.com/jamessouth?tab=repositories)将不得不等待😢。我没吃午饭去散步了。不要不吃午饭。

接下来的演讲是关于 webpack 和热模块替换(HMR)。不是重装，是替换。这一点得到了强调😊。我刚刚用 webpack 重构了我的[作品集网站](https://github.com/jamessouth/Project-12)，但是我用的是 dev-server 而不是 HMR。在开发周期中，我没有任何状态要维护，但是如果你有，HMR 是最好的选择。除非你用 Angular，显然他们现在合不来(Vue 和 plain JS 都可以)。这是正在[上工作的](https://github.com/NativeScript/NativeScript/issues/6398)。总之，这是一场精彩有趣的演讲。

之后是关于骗子综合症的演讲。作为(大概)唯一一个没有第一份开发工作的与会者，我感觉自己是个终极冒名顶替者。所以这个演讲很有帮助。这让我放松了一些，事实证明，我交谈过的每个人都很友好，我觉得我可以在我们讨论的技术话题上与他们保持一致。我确实知道一些事情！

休息时间！我认为这是爆米花出来的时候，这是一个伟大的呼吁。记得吗，我没吃午饭。总之，我遇到了一位顾问，告诉了他我的情况，他认为，就我在投资组合中所做的一切而言，问题可能更多的是我没有能力推销自己，而不是我的技能、资质和动力。他建议我得到一些简历帮助，我打算这样做。如果有人想看看，请告诉我！

当天的最后会谈是关于离线应用和开源软件(OSS)的。我已经接受了半打左右的 PRs，但它们大多是一些小东西，比如打字错误，还有一些文档和一个小的网站更新。但是不管是大是小，OSS 都是参与其中的好方法！

第二天有一些有趣的会议:上午是 AI 聊天机器人(通过 [NativeChat](https://www.progress.com/nativechat) )和 microapps(在容器应用程序中按需惰性加载的小程序)，下午是 Node.js 审查、机器驱动的重构(与 [ASTs](https://en.wikipedia.org/wiki/Abstract_syntax_tree) )和移动机器学习，演讲者在 Azure 上训练了一个模型，然后在他的应用程序中使用它。还不错！

午餐时，我和一位 dev 聊了很久，他提醒我要坚持找工作，即使申请工作似乎是浪费时间，但毫无进展。我承认我可能在通过 CFA 考试后太快停止了寻找；我只是不明白为什么我没有任何进展。但他说事情就是这样。你必须坚持到你赢为止。这对我是一个很好的教训。

因此，在所有这些演示和与这么多人交谈之后，我带着一个计划离开了 jsMobileConf:获得一些简历帮助，不要放弃，当然，还有代码！！！谢谢你，进步！！！😁
