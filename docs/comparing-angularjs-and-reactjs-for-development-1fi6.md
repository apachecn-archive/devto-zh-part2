# 比较 AngularJS 和 ReactJS 的发展

> 原文：<https://dev.to/bargavi2307/comparing-angularjs-and-reactjs-for-development-1fi6>

工程师倾向于选择角度超过反应，这是一个有利的位置，因为在这种情况下，与结构一起工作更有帮助。更重要的是，谷歌模式支持了我们关于棱角分明的名声的假设。

我们有许多 Angular 任务，每个任务都以相似的方式完成——Angular 无疑是快速和适度简单的 UI 安排的理想选择。按照这些思路，如果您有一个具有复杂业务原理的应用程序，那么选择 React + Redux 是更明智的。

另一个角度上的缺点是获得反应的高合理性。例如，如果你在工作过程中误解了某件事，这无疑会引发其他事情。此外，你还会让更多的工程师在这项事业上分一杯羹，而症状只会相互叠加。更重要的是，在扩大规模后，失去对质量的控制将变得极其简单。尽管可能有所预期，但在使用 Angular2 和 [ReactJS](https://mindmajix.com/react-js-training) 时没有出现这样的问题，因为状态区间和不变量不足为奇。

Rakish 同样有一个问题，我们称之为猴子/香蕉问题。也就是说，万一你需要给猴子一个香蕉并观察猴子如何处理它，你需要先建造整个荒野。与 Angular2 不同，使用 React，由于创建了环境，您可以在没有太多拉伸的情况下获得更多可测试的片段。
