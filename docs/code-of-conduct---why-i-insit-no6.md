# 行为准则——我为什么坚持

> 原文：<https://dev.to/arne_mertz/code-of-conduct---why-i-insit-no6>

*这是来自我的[个人博客](https://arne-mertz.de/2018/03/code-of-conducts-why-i-insist/)的交叉帖子。*

最近我收到了很多关于会议行为准则必要性的问题和讨论。我抵制任何没有会议或由于其他原因许多人无法参加的会议。这里有一个跑题的，带外的帖子，关于我对此的想法。

技术社区，尤其是 C++社区，是一个白人男性主导的文化。众所周知，在不同的会议上发生了各种各样的性骚扰、性别歧视、种族歧视和同性恋骚扰事件。有时情况并没有那么糟糕，但这并不意味着没有“更衣室谈话”或简单的感觉，即少数民族根本不受欢迎。

除了这种情况让我们行业失去了很多人才，也让很多人失去了他们喜欢或想做的事情。这就是为什么我认为迫切需要采取措施让我们的社区更具包容性。在会议上有一个行为准则是最容易的。

# 什么是行为准则，做什么

行为准则通常被视为一种工具，使会议组织者能够处理骚扰其他与会者的与会者。如果我们仔细观察，就不是那样，而是更多。这是给每个参加或有兴趣参加的人的一个信号。

首先，这是给每个与会者的一个信号，他们正在参加一个不容忍混球行为的活动。总会有人认为这种行为可以被普遍接受。行为准则告诉他们“不要在这里！”。这可能不会改变他们的想法，但它可能会使会议成为其他人更愉快的经历。

其次，这是对有兴趣参加会议的少数民族的一个信号。行为准则表明他们是受欢迎的，会议组织者打算保护他们免受骚扰。它能让人们在会议上感受到极大的安全和受欢迎。

最后，如果事情真的发生了，行为准则也是会议组织者关心的一个信号。它告诉骚扰的受害者，当他们告诉别人时，他们会被听到，会有事情发生。

# “但是……”

我听过很多反对行为准则的“但是”。以下是一些最常见问题的答案。

## ”...只有好人会来参加我的会议。”

所以你认为。在每个群体中，都有一些不按社会常规行事的人。会议规模越大，与会者中就越有可能出现一些害群之马。一个白痴就能毁掉一个或多个参与者的体验。最好不要冒险。

## ”...在*我的*社区从来没有发生过任何事情。”

据你所知。受到性骚扰或其他方式的骚扰会非常丢脸。大多数受害者不太可能告诉很多人发生了什么。尤其是在没有线索的情况下——比如说，行为准则——他们会被倾听。

## ”...反正我会对付混蛋，我不需要行为准则。”

当然，你会的，因为你可能是个正派的人。你知道这一点，任何见过你的人也知道这一点。害怕被骚扰的潜在与会者可能不知道这一点，因为他们还没有见过你。这意味着你必须告诉他们。猜猜怎么回事。

## ”...它没有解决根本问题。”

没错，不是。它也不能解决世界饥饿问题。顺便说一句，会议也没有。行为准则不能解决一些问题，并不意味着它是一件坏事。它也没有假装解决多样性、包容性和一些人是混球的问题。除了当我们在我们的社会中到处公开有一个“骚扰是不可以的”政策时，它可能会有一点帮助。至少比什么都不做好。

# “那么，我是不是应该把维基百科上的东西复制到我的会议网站上，就这样？”

虽然你可以在互联网上找到好的行为准则文本，你可以简单地复制或链接，但这只是一点点而已。你应该说到做到，也就是说，如果在会议上发生了什么事情，你应该做好实际行动的准备。我参加过的大多数会议都有一个简短的介绍性发言，组织者会指出有一个行为准则。

请记住，这主要是关于信号，并不是每个人都阅读会议网站上的所有内容。简短地告诉与会者 CoC 是关于什么的(你可能不需要它)，以及万一他们或其他人发生了什么事，他们可以联系谁。

如果你真的关心多样性，你当然可以更进一步，为关于包容性的会议安排专门的时段。

# 结论

在会议上要求有一套行为准则并不过分。把它放在网站上不需要太多时间，在会议开始时告诉你的与会者就更少了。就我而言，我不会参加没有行为准则的会议，甚至不会在会上发言。值得一提的是，我也不会去那些人们因为出生地、宗教、性取向、种族或任何其他琐碎原因而受到立法者、警察或边境官员骚扰的国家参加会议。

> 幸运的是，建立一个 CoC，并在会议开始时说几句话，真的只是一个小干扰。比骚扰事件少得多。而且，总比无休止地争论一件只有积极后果的小事要少得多。
