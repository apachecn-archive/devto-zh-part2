我的许多榜样之一曾经说过，她开始从事技术写作的原因是，作为一名软件开发人员，她写下了她在不断发展的旅程中的新学习和发现，以帮助她在“再次面对相同的问题/错误时迷失”的时候。

除了其他方法之外，教授、解释或分解逻辑的一种深入方法是通过写作，因为它允许你在方便的时候探索你所有的创造性思维，而不必担心怯场。

技术写作包括所有这些甚至更多，它包括将复杂的技术过程或意识形态记录成简单易懂的格式。它从技术产品或服务的用户或说明手册开始，后来发展到撰写技术或专业主题，如计算机应用程序和工具、软件开发语言和框架、医疗程序、环境法规等

### [](#the-skills)**技能:**

技术写作技巧对有些人来说是天生的，对另一些人来说是后天习得的，但总的来说，成为一名优秀的技术写作者所需的一些重要技巧包括:

1.  强大的写作和沟通技巧:用简单易懂的语言向你的读者传达复杂的意思和术语的能力是必须的，因为这是阅读你的文章的本质，即理解他们一直不理解的东西。
2.  **与领域相关的技术经验/技能**:传达信息意味着理解和掌握信息，你所写领域的知识和经验是必要的。假设你目前在所述领域工作或有所了解，因此了解你的洋葱，许多技术作家在他们的领域担任技术职务，如*软件开发人员/工程师，信息安全分析师等。在你非常熟悉的领域写作对写出一篇好文章大有帮助。*
3.  兴趣:对技术有(巨大的)兴趣会让你在从事技术写作时获得优势，因为它会激励并让你不断探索新的技术来学习和写作。
4.  **研究技能**:写作包括(大量)收集和比较来自不同来源的数据和发现，结合你对该主题的个人知识，转化为有价值的书面信息。

### [](#the-process-)**流程**:

写一篇技术文章可能与普通的文章有很大的不同，而且可能需要相当长的时间，因为大多数时候需要重新创建简单到复杂的实用技术步骤，以便传达你的目标。因此，在写作之前和写作过程中，为了不在文章中造成严重的缺陷，总是要采取以下精确的步骤:

1.  **了解你的读者**:读者永远是技术写作者心中的最前沿，了解你所写作的读者类别有助于简化剩下的任务。用不熟悉的技术或先进的说法，用虚假的术语写一篇给初学者的文章，不会达到传递信息的目的。你的受众定义了所使用的文本以及你的信息是如何传达的。为了更好地了解你的读者，在开始写一篇文章之前，应该考虑以下问题:

*   他们是谁？
*   他们需要什么(从你的文章中学习)？
*   他们会在哪里读到你的文章？
*   他们为什么要读书？
*   他们会如何阅读？

这些问题有助于理解你的听众。

1.  **规划你的文章**:写一篇技术文章看起来像是一项复杂的艰巨任务，对于从哪里开始有点困惑，但是有了明确的受众，创建一个规划或指南(开始< — >结束)就容易多了。概述你的读者应该从阅读中获得的必要信息，并可能将它们转化为要点/指南，一些作者喜欢概述所有这些，即在文章的开头创建一个概述，让读者了解他们在继续阅读时会期待什么。在你的文章中有一个强有力的介绍也有助于你知道你文章的下一条路。创建地图或指南有助于突出你可能没有想到或甚至不知道的其他领域，并有助于让你更加熟悉和有所准备。

2.  **反馈迭代**:没有哪个作家是知识的大山，就像在技术领域的职业人士中鼓励评论和反馈一样，在技术写作中也是如此。从同事、专业人士、领域专家等那里获得对你文章的技术评论和反馈，同时以读者的心态阅读你自己的文章，这有助于在发表你的文章之前发现可能逃过你审查的语法和技术错误。

3.  内容创意:现在你已经理解了你的读者，并且有了一个文章的地图或指南，通过保持一种写作风格来注入创意，同时在文章主体中清晰地传达你的信息，这将大大有助于保持你的读者的注意力直到最后。这里有几个例子:

*   选词:选择你的用词，并在必要的地方包含更多的细节，这有助于你的文章更容易理解。

***示例*** : *打开命令行，在“文档”目录下执行 XYZ 动作*

***改良*** : *打开你的命令行，通过输入“* *cd documents”进入“documents”目录，执行 XYZ 动作。*

*   经常使用主动语态:在文章中使用主动语态比被动语态更容易阅读和理解。

***举例*** :超过 *35 万个包被* ***包含*** *在 Node.js npm 包注册表*

***改进***:*node . js NPM 包注册表* ***包含*** *超过 35 万个包。*

*   **句子结构**:把重要信息放在第一句话里大有帮助。

***例如*** : *在进行数据库更新之前备份数据总是明智的。如果您不遵守这一点，很有可能会永久丢失数据*

***改进的*** : *建议在进行数据库更新之前备份您的数据，以免永久丢失数据。*

*   避免技术俚语/缩写/行话:我们很容易陷入写文章和炮制大量信息的困境，无意中使用自己和朋友或同事们熟悉的缩写/俚语/行话。当写文章并打算使用不熟悉的术语时，最好将它们完整地陈述出来，给它们下定义，然后用缩写来指代它们，而不是直截了当地让你的观众蒙在鼓里。

***示例***:****MVC****模式在各种框架中被广泛接受。**

 ***:****模型-视图-控制器*** *(MVC)* *模式在各种框架中被广泛接受。**

 **   **启动基于任务的方法**:大多数技术文章都是以教程或 D.I.Y .的形式提供给读者，所以以步骤的形式写文章也是使文章更全面的好方法。确保遵循正确的步骤顺序，让你的写作自然流畅。

### [](#the-benefits-)**好处**:

作为一名技术写作者有很多好处，其中包括:

1.  **不断学习**当你不断尝试跟上新技术，提高你的阅读和写作技能，投入到新的领域并获得不断的反馈。这是一个向上的增长规模。
2.  这是一种**珍贵且有利可图的技能**，因为任何真正擅长用简单的术语向用户传达技术复杂性的人，对于总是愿意花重金拥有他们的组织来说都是“有价值的”。一些申请付费技术写作的热门网站有[推手](http://www.pusher.com)、 [Scotch.io](http://www.scotch.io) 、 [logrocket](http://www.logrocket.com) 等
3.  这对技术领域的个人来说是一个职业提升和额外的技能组合，因为他们被认为能够快速掌握、分解并向用户传达技术用途和重要性。
