# 老狗学新把戏

> 原文：<https://dev.to/threedeeprinter/an-old-dog-learning-new-tricks-5b7h>

[![Dog using a laptop](img/94654b245ee9549869715ee14894bd4f.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--BulK2ClS--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/7yuyeznk5b1lgk1pbckx.jpg)

在经历了最近一轮裁员后，我决定将我目前的技能与 LinkedIn 上的求职者进行比较。我对四年间发生的巨大变化感到震惊。我也对自己没有及时更新技能感到有点失望。我对 Node.js、Angular 或任何反应式框架知之甚少。我不时地使用把手，但那是我最接近模板的时候了。除了工作之外，我也没有真正参与过“对话”。

我决定学习 Node.js，TypeScript 和 Angular。工作中有人试图介绍 Vue。JS 到我的团队，所以我也加上了。Udemy 对我了解这些技术帮助很大。突然间，编码又变成了一种乐趣。像 Vue 和 Bootstrap 这样的框架消除了 UI 编码的繁琐，因此你可以比以前更快地进入原型阶段。您必须编写 Javascript 代码来遍历一个表以聚集数据并将其发送回服务器的日子已经一去不复返了，现在所有这些都直接随着每个表单控件的更新而更新。我觉得自己像一个在糖果店的孩子。

最棒的事情是，我觉得我又成为了对话的一部分。我开始听开发者播客，并报名参加了几个当地的开发者见面会。我还觉得我在目前的职位上获得了优势，因为现在我知道的技能远远超出了我所做的。

我学到的另一件事是，你不应该把自己局限于你在当前职位上使用的东西。学习新技能有助于重新激发你成为开发人员的兴趣。

我忘了说我 46 岁了。我开始接触 VBA、HTML 和 SQL Server。在 ASP 或. Net 出现之前，我就已经开始编程了。有很多关于老开发人员的讨论，以及如果我需要的话，在以后的生活中找到一份工作会有多难。我认为，只要你对这个行业保持兴奋，让自己置身于谈话之中，并明白在你工作的地方之外还有一个有待探索的世界，你就永远是相关的。为自己不应该学习找借口是让自己被淘汰的最快方法。

不好意思，我要去上. Net Core/Angular 课。

为学习干杯！
