本文分享了三个讲座。我的是第二次谈话。它是关于 Flutter 和其他可用于创建移动应用的框架之间的比较。

我在演讲中的重点是比较将代码库编译成本地应用的框架。但为了显示这些框架的位置，我还添加了原生 Android 和 iOS 以及混合应用程序框架 Cordova/Ionic 进行比较。这就做了一个 Android/iOS 原生与 Cordova/Ionic 与 React 原生与 Xamarin 与原生脚本与 Flutter 的对比。

现在，所有这些工具都彼此大相径庭。所以对我来说，这就像拿橘子和苹果做比较。所以我放了两个主要目标来对比。

1.  使用相同的代码库为 iOS 和 Android 创建最好的应用程序。
2.  以最好的方式回答开发人员的问题。

所以每个框架都有不同的优缺点。在我的演讲中，获胜者是以最好的方式回答了上述目标的人。获胜者是颤振和亚军是反应土著。
