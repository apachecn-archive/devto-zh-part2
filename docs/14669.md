# 自我记录的代码不是

> 原文:[https://dev.to/grahamcox82/self-documenting-code-isnt-5097](https://dev.to/grahamcox82/self-documenting-code-isnt--5097)

我知道这将是一篇很有争议的文章，但我还是要写。

自我记录代码不是一个好的目标。在我看来恰恰相反。

代码中注释的存在有多种原因。其中之一是告诉*代码做什么，这是自文档化代码试图消除的。这是一个不错的目标，因为如果代码需要注释来告诉你它做了什么，那就太复杂了。*

然而，另一种形式的注释是告诉你*为什么*代码做它要做的事情。这比*什么*重要得多，因为这从代码本身来看并不明显。这可以归结为代码设计决策、产品设计决策、大规模架构决策或无数其他原因。

另一种形式的评论是外部资源的链接。这可能是问题跟踪编号、设计文档或许多其他内容。

最后，有时候你真的需要记录一些东西是如何工作的，因为整个算法是复杂的，并且让每个人都阅读所有的代码来获得大图概述是不可行的。

因此，在努力使你的代码干净、明显和容易理解的同时，不要在删除所有代码注释方面走得太远，因为它们还有其他用途。
