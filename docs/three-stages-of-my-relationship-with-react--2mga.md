# 我与 React 关系的三个阶段

> 原文：<https://dev.to/dellward/three-stages-of-my-relationship-with-react--2mga>

这可能过于简单，但 React 的主要目的是为开发人员提供一个如何构建应用程序 UI 的蓝图。剩下的就看开发商了。这意味着为服务器、路由、样式表、状态管理等找到并实现自己的工具。React 不握任何手。这使得学习曲线非常陡峭，但最终也是有益的。反应激烈但不苛求，友好但独立，我已经坠入爱河。

我喜欢反应。很晚了，我可能只是想睡觉，但我和 React 在一起的时间感觉我喜欢我和我妻子的关系的发展。听我说完:

## 配置

在这里 React 是欲擒故纵。它玩着强迫我了解它自己的游戏，而不放弃太多。它希望我成功，但它不确定自己需要什么，希望我弄清楚，有时会告诉我，无论我在做什么，都是错的。它爱一个人有坚持。所以我开始告诉别人我的困境，得到一些答案，然后我带着大量的工具回来，看看 React 是否最终能回报我的爱。

## 发展中

此时我们发现(配置？)对方出来了。我仍在学习 React，但我知道是什么让它运行，这是它最讨厌的地方。当我们继续朝着伟大的目标前进时，我们会彼此完善。有一些打嗝，但我们比我们仍然“交谈”时更快地克服它们，并且以一种更成熟的方式。

## 生产

当 React 和我对我们的关系充满信心时，我们将开始计划制造一大堆我们自己的快乐。是的，我知道这个包在技术上是在开发期间创建的，但这并不意味着它已经可以交付了。让我们说，到目前为止，它仍处于酝酿阶段。一旦你的应用发布到世界上，就没有回头路可走了。现在的工作是好好照顾这个应用程序，保持它的健康和安全。父母的工作永远不会结束，在这个阶段，开发者的工作也不会结束。我还没有达到 React 的这个阶段，但是我很期待它的出现。

说真的，到目前为止，React 还算不错。我在 React 工作才几个月，但我对它的兴奋让我受邀加入一个开发团队，成为一家潜在初创公司的联合创始人。我仍然称自己是一个初学者，这是我的第一个团队，所以谁知道结果会如何。但我知道这将是一次很好的学习经历。激动人心的时刻！
