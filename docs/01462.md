自从 WebAssembly 开始流行以来，开发人员中就出现了一个问题:它会终结 JavaScript 吗？然而，一些开发者和企业认为 WebAssembly 将补充 JavaScript 并加速 web 开发。

## [](#what-is-webassembly)什么是 WebAssembly？

WebAssembly (WASM)是一种二进制指令格式，用于编译 C、C++、Java、Python 和 Rust 等高级语言。它允许在 web 和服务器应用程序上部署。

用户用多种语言编写代码，然后在网上运行。WebAssembly 允许他们以本地速度在 web 上运行代码。

## WASM 和 JavaScript 的未来

创建 WASM 的 Mozilla 开发者将其定义为用户可以在网络浏览器中运行的代码。它提供了新功能，并在性能上有了重大改进。

像 JavaScript 一样，它也在沙箱环境中执行，并表现出相同的安全性。但是，说它将终结 JS 是非常不现实的。即使是 WASM 的创造者也否认这种意图。

根据 GitHub 的最新报告，JavaScript 是过去几年中使用最多的编程语言。此外，它拥有最多数量的公共和私有存储库的贡献者。

JavaScript 没有放缓的迹象，在不久的将来也不会被任何人取代。此外，随着像区块链这样的现代技术的流行，JS 将成为 web 开发人员的新时代语言，并在各行各业获得牵引力。

## WASM 会取代 JavaScript 吗？

让我们假设 JavaScript 被取代了。WebAssembly 是否能扮演 JS 的所有角色？别忘了，WASM 还是一名年轻球员，几年前才在 2015 年出道。然而，JavaScript 已经存在了大约 20 年了。它已经成为 web 开发的理想语言。

商业领袖说 WASM 将取代 JavaScript 有些言过其实。他们认为它不会保持垄断地位，并随着时间的推移而增长。

JS 的一个局限是它不是作为编译目标而构建的。另一方面，WebAssembly 已经被构建为一个高效的编译目标。它是现成可用的，提供了更小的有效负载、快速解析以及性能上的一致性。

根据 WebAssembly 的 FAQ 页面，它通过设计使 web 浏览器能够在其现有的 JavaScript 引擎中使用 WASM 来降低成本。因此，从成本角度来看，WASM 应该比得上一个新的 JS 功能，而不是浏览器模型的基本扩展。

目前，JavaScript 支持所有主流浏览器。为了使用 WASM， [web 开发公司](https://www.techuz.com/web-development/)需要放弃不支持 WebAssembly 的浏览器。直到 WebAssembly 带来对新浏览器的支持，JavaScript 生态系统将进一步发展，更多的开发人员将在他们的项目中使用它。

JavaScript 开发的生态系统非常庞大，大多数开发人员都在其中编写他们所有的库和框架——Bootstrap、Angular 和 React 等等。为了超越 JS，WebAssembly 将需要所有这些库和框架的替换选项，这似乎并不容易。

此外，将现有的 JS 代码替换为 WebAssembly 是不可行的。即使它发生了，公司也不会喜欢它，因为它会非常慢，而 JS 中的代码库将是巨大的。JavaScript 在市场上已经有很多竞争对手，但是没有一个能够取代它。

随着时间的推移，WebAssembly 将越来越受欢迎，以支持基于浏览器的游戏、VR(虚拟现实)应用程序和其他计算密集型任务。它只能减少 JavaScript 的市场份额，但不能取代已经用 JavaScript 编写的应用程序。

## [](#webassembly-complements-javascript)WebAssembly 补充 JavaScript

网站开发公司可以同时使用 WebAssembly 和 JavaScript，因为这两者可以相互补充。WebAssembly 可以使用 WASM JavaScript API 与 JS 一起运行。它会将 WASM 模块加载到一个 JS 应用程序中，并一起共享功能。

当一起使用时，开发人员将受益于 JavaScript 的表现力和灵活性以及 WebAssembly 的性能和功能。因此，WASM 有能力将 JavaScript 转移到脚本化的角色。如果企业和开发者认为 WebAssembly 是一个伙伴，它可以产生一个标志性的组合。

## [](#wrapping-up)包装完毕

WebAssembly 将凭借其核心特性和原生速度为 web 带来大量改进。但超越 JS 的可能性非常小。不管你喜不喜欢，JavaScript 将继续主宰 web 开发领域。

甚至 WebAssembly 的 FAQ 也声明 WebAssembly 将允许在 web 上编译多种语言，但是 JavaScript 已经获得了令人难以置信的发展势头。因此，JS 仍将是网络上唯一的特权语言。
