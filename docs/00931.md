# Java 还是免费的！

> 原文:[https://dev.to/lemuelogbunude/java-is-still-free-286](https://dev.to/lemuelogbunude/java-is-still-free-286)

我看到很多帖子哀叹 Java 现在是如何被付费的。我觉得很多人都理解错了...没错。

## [](#oracle-jdk)甲骨文 JDK

也许当你开始用 Java 编码时，你被要求下载 JDK (Java 开发工具包)。默认情况下，你去的是甲骨文 JDK。甲骨文不是唯一的提供商，尽管他们是最受欢迎的，所以当他们采取行动时，每个人都会感觉到。

甲骨文 JDK 公司现在每六个月发布一次功能，这一新的版本方案此前于 2017 年宣布。以前这需要 3 年多的时间。比如看看 Lambdas 在 Java 8 中用了多长时间才到了 Java。所以六个月的功能发布是个好消息！

Oracle JDK 可以在开发和测试环境中免费使用，但在商业环境中需要付费。

换句话说，更新通常会提供 6 个月，然后在下一个版本发布时停止。如果需要更长时间的更新/支持，或者用于生产用途，则必须从 Oracle 购买。

现在，如果他们以这样的速度前进，甲骨文收取费用实际上是有道理的，这背后有辛勤工作的开发人员，而不是机器人。

此外，企业收取企业支持费用并不是什么新鲜事。

如果您是一名刚刚学习 Java 的开发人员，这不会对您产生任何影响，因为您可能正在开发和测试环境中使用 Oracle JDK。

现在，如果你打算把它用于商业用途，你应该怎么做？

你有一些选择，但我会列出几个-

## [](#your-company-could-go-for-the-commercial-plan-and-longterm-support)贵公司可以寻求商业计划和长期支持:

-每个用户(台式机)每月 2.5 美元或更少
-每个处理器(服务器)每月 25 美元或更少

对于大型部署支持需求，折扣很大，例如，10k-20k 处理器单元的折扣为 50%。

## [](#openjdk)OpenJDK

Oracle 为 OpenJDK 做出了巨大贡献，它是 Oracle OpenJDK 版本和 Oracle JDK 的基础。OpenJDK 是免费和开源的。事实上，从 Java 11 开始，OpenJDK 的特性已经和 Oracle 的 JDK 不相上下。换句话说，你不应该认为 Oracle JDK 比 OpenJDK 版本“更好”。这是个好消息，不是吗！

您当然不“需要”选择付费支持。

但是，如果你在一个大的组织中，需要一个及时的修复程序，需要一个人来响应你的用户请求，或者如果你想确信你所使用的二进制文件是由一个供应商支持的，那么你可以去寻求商业支持。

值得思考的是，作为最终用户，您如何支持 Java SE 生态系统，以确保它有一个长久的未来！

## [](#summary)总结

我已经给出了这种情况的概述，如果你仍然有兴趣了解更多细节，你可以去查阅更多详细的文章和文档。

在我们的世界里，有很多优秀的语言和优秀的特性，Java 需要加速它的发布并更快地添加新的特性。因此，甲骨文决定投资推动 Java 向前发展。
