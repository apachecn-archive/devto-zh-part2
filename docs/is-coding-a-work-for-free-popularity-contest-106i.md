# 编码是免费工作的流行竞赛吗？

> 原文：<https://dev.to/makiten/is-coding-a-work-for-free-popularity-contest-106i>

我改变了我的职业生涯。2015 年，我不再做 dev 去做销售。我不会在这里进行冒险，但当我试图找出我职业生涯的下一个篇章时，大多数雇主会试图将我归类为开发人员，我似乎真的找不到我的目标客户只把我视为代码员的咨询工作。

无论如何，我离开开发的一个原因是，总有两个微妙的品质可以让我得到一个比我当时更好的开发角色:

1.  免费工作(在空闲时间)
2.  非常受欢迎

建立一个“品牌”需要很多时间，其中大部分似乎需要你的空闲时间去做。在另一个帖子中，有人说，人们期望你将 100%的时间投入到发展中，而我越老，就越不想这样。

如果不沉迷于最新的技术、聚会、演讲活动、会议等，你的职业生涯还能继续发展吗？？
