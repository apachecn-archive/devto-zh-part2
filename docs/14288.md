# 开发者工作申请评估是双向的

> 原文:[https://dev . to/Chris hunt/developer-job-application-assessments-is-a-two-way-thing-4m 20](https://dev.to/chrishunt/developer-job-application-assessments-are-a-two-way-thing--4m20)

当寻找新的开发人员工作时，相当数量的职位需要公司进行某种评估，以便开发人员能够证明他们声称的技能。

在我看来，这些评估也反映了公司的问题。作为一个开发者，这是一个早期的机会来看看一个公司在他们的开发者身上看重什么。在被邀请完成许多这样的测试后，我看到了光谱的两端，这调整了我对我申请的公司的看法。

这些评估可能很耗时，双方(申请人和公司)都应该珍惜投入的时间。申请人应该能够对公司要求他们做的事情有所了解。它还应该具有挑战性，以便申请人可以在完成评估时感受到一些回报。

最近，我被要求完成一个 50 分钟的多项选择测试。问题通常是关于以非等宽字体打印的代码的语法，没有突出显示语法。我坚持了大约五分钟，甚至更少的问题，然后我给公司发了一封电子邮件，解释说他们的工作规范强调需要富有想象力和多才多艺的开发人员，但他们的测试促进了某种类似机器人的代码猴子，我想撤回我的申请。这种回答对相关公司来说可能显得傲慢，但如果从本意来看，可能会凸显出在招聘过程中进行评估的必要性。

我认为，在试图评估申请人的能力时，公司可以也应该在这个过程中留下工作的“精神”印象。

我更喜欢的方法是解决项目类的问题，这种方法允许申请人发挥想象力，并展示为什么他们应该是某个角色的最佳候选人。它还允许在任何进一步的访谈中围绕思维过程和技巧进行讨论。这些通常会很长(3 或 4 个小时)，但一份高质量的申请几乎可以在进一步面试之前就获得这个职位。
