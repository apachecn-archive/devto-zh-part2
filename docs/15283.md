# 我如何开始为开源做贡献？

> 原文:[https://dev . to/captainsafia/how-do-I-get-started-contributing-to-open-source-3b0b](https://dev.to/captainsafia/how-do-i-get-started-contributing-to-open-source-3b0b)

我最近在一个会议上做了一个主题演讲，分享了我如何为开源做出第一份贡献的故事，以及从那以后的旅程。后来，人们问了我几个关于为开源做贡献的问题。我收到这些问题已经有一段时间了，我想我应该写一篇博文来回答我收到的一些问题。

> 我如何开始为开源做贡献？

这是我得到最多的问题，它相当模糊，很难回答。我要做的是，分解它的不同细节，并回答这些问题。

> 如果我的目标是用开源软件找到一份工作，我该如何开始为开源软件做贡献？

如今，很多招聘人员通过考察候选人在 GitHub 上的活动来评估他们。假设你在 GitHub 上提交的次数越多，你就是一个更活跃、更成熟的程序员。我不认为这种假设是正确的，我认为将 GitHub 活动纳入招聘过程的决定是有害的，但规则不是我制定的…

也就是说，如果你有兴趣为开源做出贡献，向潜在的招聘者展示你的开发技能，我会考虑另一种途径。

这有几个原因。首先，参与开源项目的起点可能非常陡峭。你可能需要相当长的时间才能做出贡献，展示你的全部技能。

此外，如果你的目标是利用你的开源贡献来获得一份工作，你应该考虑你长期参与开源会是什么样子。一旦被录用还会继续贡献吗？你会在多大程度上参与开源项目？在为开源项目做贡献之前，回答这些问题是很重要的。

总而言之，我想说的是，如果你正试图在潜在的求职中改善你的 GitHub 档案，我会建议你做一些个人项目(比如用一个你想展示技能的框架构建一个 Twitter 克隆)并把它们放在 GitHub 上。这将让招聘人员看到你的全部技术技能，并允许你建立一个伪产品组合来展示你开发完整软件产品的能力。

> 如果我的目标是寻找良师益友，我如何开始为开源做贡献？

如果你有兴趣将开源作为一个空间，在指导下提高你的技术技能，我建议你参与一个大中型项目。中型到大型是描述开源项目的一种奇怪的方式。又不是软饮料什么的。在这种情况下，当谈到规模时，我关注的关键指标是项目中活跃的维护者的数量。维护人员是已经为一个项目做了一段时间贡献的个人，他们负责审查拉请求。像 Python 或 Node 这样的项目。JS 将是大型项目，因为他们有大量的维护人员。React 和 Babel 将是中型项目，因为虽然他们有几个维护者，但他们没有上面提到的项目多。

如果你正在寻求指导，为什么这是决定什么项目有助于你的一个重要因素？简单地说，一个项目的维护者越多，你就越有可能找到适合你学习和开发风格的人。

> 我如何开始为开源做贡献？我的目标是提高我的技术技能吗？

如果你有兴趣为开源做贡献，并打算提高你的技术技能，那么确定你对提高哪部分技术技能感兴趣是至关重要的。你想成为更好的围棋开发者吗？您想在项目中更有效地利用 React 吗？

如果您有兴趣更好地使用一种特定的编程语言，那么从为与该项目相关的包做贡献开始，然后最终为项目本身做贡献，这将会很有帮助。这是有效的，原因有二:您将接触到特定语言的包生态系统，获得使用该编程语言构建包的技能，并了解语言本身的复杂性。

如果你对更好地使用某个特定的框架或数据库感兴趣，那么为利用该框架的开源 web 应用做贡献会很有帮助。例如，你可以看看由 Thinkster 构建的开源项目集合[，这些项目展示了如何使用不同的前端和后端 web 框架。](https://github.com/gothinkster/realworld)

一旦你有了使用特定框架的经验，你就可以开始为上面的框架做贡献，以深入了解它的优点和缺点。

> 如果我的目标是提高我的沟通和共鸣技能，我该如何开始为开源做贡献？

如果你有兴趣为开源做出贡献，以提高你的沟通和共鸣技能，你肯定是做了正确的选择。许多开源工具肯定会从文档、可访问性和福音部门的改进中受益。

这里的可能性几乎是无限的。您甚至不必从发出拉取请求开始。你可以从在你的博客上发布一个教程或者制作一些教育材料开始。在我看来(和许多其他人的观点)，这仍然算作对开源社区的贡献！

也可以浏览 GitHub 问题板或 StackOverflow，寻找自己有专长的特定开源项目，公开回答问题。这也算是对开源社区的贡献。

在交流和共鸣领域还有很多其他的机会。以下是我想到的一些例子:

*   就一个特定的开源项目进行会议演示
*   回答 StackOverflow 上的问题
*   帮助在问题跟踪器上对问题进行分类
*   创建教育博客帖子和教程
*   创建截屏或 YouTube 视频
*   使用开源项目或工具时在 Twitch 上进行直播
