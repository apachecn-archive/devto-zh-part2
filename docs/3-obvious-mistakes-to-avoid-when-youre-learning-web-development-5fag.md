# 学习 web 开发时要避免的 3 个明显错误

> 原文：<https://dev.to/flippedcoding/3-obvious-mistakes-to-avoid-when-youre-learning-web-development-5fag>

学习 web 开发可能有点棘手。你可以从这么多不同的来源以这么多不同的方式学习，很难决定哪一个是最好的，这就是为什么很容易犯错误。我无法告诉你有多少次我不得不重新学习概念，因为我用错误的方式学习它们。

对你来说好消息是，如果你知道应该避免做什么，你可以为自己节省几个月的时间。你将要学习的内容可能听起来很明显，但这些是我们最常看的东西。

1.  试图一下子学到太多东西。
    [![don't drown yourself](../Images/1849f4998462ee2091f3b3788d0b1e05.png)T3】](https://res.cloudinary.com/practicaldev/image/fetch/s--tYjUdCMn--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/3fkyezaijnxa8x8sba6c.jpg)

    *   你知道你需要学习 HTML、CSS 和 JavaScript，你可能想尽快学会它们。这并不意味着你应该同时开始学习它们。如果你想精通这些东西，慢慢来，一个一个地学习。这样做会给你真正理解每种编程语言是如何工作的深度。
    *   如果你想通过网络开发赚钱，不管是开始新的职业还是兼职自由职业，你都需要知道你在做什么。不管在线课程和编码训练营教给你什么，你都不会从新的文件开始项目。你将会被一些遗留的垃圾击中头部，你需要知道如何去挖掘它。
    *   慢慢来，学习 HTML、CSS 和 JavaScript，因为它们是 web 开发的基础。您需要能够单独查看这些代码，并知道如何阅读它们，在哪里进行更改，以及为什么需要进行这些更改。做到这一点的唯一方法是一次专注于一个概念。
2.  没有一起运用你们的技能。
    [![use the stuff you learn together](../Images/d340a5ab0807e2627b56ec22218b29a3.png)T3】](https://res.cloudinary.com/practicaldev/image/fetch/s--y2FpgyMi--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ttor99xyqcvwydywv1bt.jpg)

    *   这与上一点完全相反。你需要了解 HTML、CSS 和 JavaScript 的一切，但是你也需要知道它们是如何协同工作的。最好一开始就把所有的东西都分开学，因为这能让你对网站运行的细节有深入的了解，但是真正的力量来自于一起使用它们。当你作为初学者学习如何编程时，很容易区分不同的语言。
    *   一开始可能会有点挑战，但是你需要练习用这三种编程语言制作小网站，直到你可以用它们制作更大、更复杂的网站。有时候，课程太专注于学习个别的编程语言，以至于没有留给你足够的时间来练习。
    *   你应该花至少 4 个月的时间除了做网站什么都不做。不是练习个别技能，而是一起使用所有技能。这才是真正的 web 开发。想想看，如果你知道 JavaScript 中的所有概念，但你不能在真实的网页上使用它们，那么你真的了解 JavaScript 吗？
3.  完成一些教程或课程后，认为自己是专家。
    [![you're still considered a baby](../Images/07579af1c419202347b1d5e6ec0724ab.png)T3】](https://res.cloudinary.com/practicaldev/image/fetch/s--yeRQlaQQ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ca4kgc2mjitezmvasof5.jpg)

    *   抱歉，如果我打破了你的幻想，但你不会在 3 - 6 个月内成为一名熟练的 web 开发人员。差远了。你也许能在 3 - 6 个月内把一些基础的东西做得很好。甚至好到可以开始自由职业或申请工作。问题是没有人真正关心最基本的东西。这不会让你赚钱的。
    *   你需要一些实践和经验，这需要时间。而且不，你不需要有人给你机会练习。你只需要站起来开始做网站。你甚至不需要拿出一个原创网站！去你最喜欢的网站，试着复制一份。
    *   教程和课程给你一个良好的开端，你可以相对较快地完成它们。你不能就此止步。花最少 4 个月的时间做网站是个好主意，这样你才真正知道自己在做什么。你还可以建立一个作品集，展示给潜在的自由职业客户或大公司，这样你的实践就值得了。

这些是人们在学习 web 开发时犯的一些最明显的错误，现在你不必犯这些错误了。但是有一个大问题我故意漏掉了。如果你认为你会在不到一年的时间内精通 web 开发，你要么非常聪明，要么就像我刚开始时一样，只是不知道需要多少时间。

请记住，在你花了这么多时间学习之后，你看到的是平均起薪为 70，000 美元的~~(但实际上更像是 55，000 美元)~~，这是一个被标记为在未来 10 年将会增长的领域，你可以获得在任何其他领域都不会获得的机会。如果你住在美国，这一点尤其正确，因为熟练的程序员极度短缺。
