# 我们不知道自己在做什么。

> 原文:[https://dev . to/captainsafia/我们不知道在做什么 o4e](https://dev.to/captainsafia/we-have-no-idea-what-were-doing-o4e)

说到网络，我们不知道自己在做什么。

我的意思是在几个方面。

我们不知道我们的客户想从 web 产品中得到什么，我们经常最终构建出在可访问性、可用性和有用性上不达标的产品。我已经错过这个标记好几次了，我相信你也是。

我们不知道五年后网络会是什么样子。景观在不断发展，我相信我们的想象力太有限了，无法看到互联设备、AR 和 VR 等沉浸式技术以及网络的交叉点上可能存在的潜力。

我们不知道网络的基础是什么，因为基础在不断发展。

我自发地写了这篇博客来回应 Twitter 上最近的一些关于什么被认为是 web 开发的基础部分，什么不是。像这样的讨论通常是一种守门的形式，旨在使一种开发方法看起来不如另一种。在这种情况下，对特定语法(样式表的一个`<link>`)的重要性的度量被强调。

网络发展得如此之快，以至于我们经常忘记，就在不久前，网络才出现在(一些，而不是全部)人们生活的最前沿。见鬼，我们为商业目的编写计算机程序已经有大约 70 年了！

像许多新学科一样，软件工程正在努力定义自己。我认为，道德违规、安全缺陷和产品事故的混乱部分是因为我们还没有具体弄清楚软件如何融入社会结构，以及我们将如何在全球化的高速世界中使用它。

鉴于事情的快节奏和不确定性，我认为我们仍在努力从重要的事情中找出真正重要的事情。这是一种说法，我们作为一个行业和特定领域的专家在不断发展。

我认为，感觉自己已经想通了一个特定的概念是一种安慰。知道自己不知道一些事情，或者意识到人们对重要的事情有不同的看法，这种感觉可能会有点令人不安。共识和随和是我们作为人存在的重要因素，其他人不同看法的潜力可能会导致许多冲突(从而在恶性循环中放大其根源)。为了调和这些差异，我们经常试图排斥或“其他”我们不同意的人。如果他们做不到这一点，那么他们一定不是真正的 web 开发人员。有些人说这样的话正是因为这有助于他们坚持自己的世界观，尤其是当新信息挑战他们的世界观时。

每隔几个月左右(见鬼，甚至可能是几周)，开发社区中就会出现一场关于某某不是真正的编码或者某某不是真正的开发人员所做的事情的辩论。尽管有时这些陈述可以根植于事实，但它们通常是个人试图坚持他们对编写软件的特殊想法的结果。事情是这样的，我们对于写软件意味着什么都有不同的想法，而且我们离找出一个我们都同意的集体定义还很远。有一件事是肯定的；基本原理和语法没有任何关系。它们存在于一个更高的推理层面，并且有着更深远的影响。

说到网络，我们不知道自己在做什么。
