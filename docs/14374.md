我已经用 Python 写了将近 2 周的代码，我认为我已经很好地掌握了基本的东西，比如基本的语法，声明变量，打印它们，摆弄列表，字典等等，非常基本。

现在我非常擅长 PHP，我的意思是 PHP 是我比其他语言更了解的语言，我的大部分工作都使用它，但由于它在各种开发论坛上的流行，也由于我作为开发人员的好奇心，我得出结论，我最好了解一些 Python，如果它对我有帮助，我实际上可以开始用它做一些项目。

因此，当我设置好环境并准备开始我的 python 编码冒险时，当我已经对这种语言有了足够的理解并开始做一些数据库脚本测试时..连接到数据库，查询数据库以及诸如此类的事情，我遇到了一些错误，长话短说，这些错误与 mysql-connector 模块有关，我在我的脚本中导入了该模块，与我系统中的 Python 版本相对照(我在 Ubuntu 上运行 Python3.6)。据我所知，最稳定的版本是 Python2.7，它实际上是我的操作系统中的默认 Python 环境，所以当我运行我的小 DB 连接脚本时，它运行得非常完美。

我回到在线论坛寻找解决我的情况的方法，显然我在 DB 测试中使用 Python2.7 会更容易。困境是我实际上想写 Python 最进步的版本，现在是 3.6，可能是 3.7。就好像我现在被困在 2.7 版，对此我不太确定。

将 PHP 与 Python 进行比较，其中最基本的代码实际上是相同的(至少对我来说是就 DB connection 而言)，我不禁对这一切感到沮丧，如果有人知道我的一些好的起点，请随时分享您的意见。

谢了。:)
