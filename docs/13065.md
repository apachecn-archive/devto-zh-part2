# 合作

> 原文:[https://dev.to/brightdevs/collaboration-3n PD](https://dev.to/brightdevs/collaboration--3npd)

不是每个人都是外向的，当然也不是每个程序员都是，但公司是由团队创建的，团队对有效运营公司负有最大的责任。如果团队成员不合作，你的企业成功的机会微乎其微。

程序员的工作并不像许多人想象的那样可预测。程序员和机器一起工作并不意味着没有关系和协作的空间。然而，与其他人一起工作比与机器一起工作要困难得多，尤其是如果你每天都必须与人密切合作的话。

有很多功能失调的团队在分享知识方面存在巨大的问题。还有很多程序员很难与他人分享他们的代码，这最终可能会导致一场真正的灾难。程序员甚至不愿意向别人展示他们的代码，更不用说让别人“接触”它们了。但坦率地说，在专业人士工作的环境中，这种行为是不可接受的。在一个成功的软件公司中，拥有代码的是团队，而不是个人。每个团队成员都能够对代码进行适当的修改。专业开发人员让其他人处理他们的代码。他们不会在自己的代码周围筑起高墙。他们不会阻止其他人学习他们的代码。专业开发人员互相合作，互相学习。

而且他们经常通过结对编程来学习，这不仅是在紧急情况下解决问题的最有效的方法之一，也是审查你的代码的最好方法之一。通过结对，开发者分享他们的知识并交换位置。他们合作编写代码。他们不怕看到对方的错误，也不怕感受到对方的恐惧。他们谈论问题。他们交流。

当然，有时候独自工作是正确的，尤其是当你不得不深入思考问题，或者任务很琐碎，需要你自己完成的时候。但是永远记住，即使你一个人工作得更好，整个团队也不一定要这样。

对我们所做的事情充满热情是件好事，但关注我们周围人的目标也是件好事。与同事、经理、销售人员、商业分析师、客户等的合作与协作。，是创造成功的业务和有效的工作环境所必需的。我们应该经常思考我们工作的公司如何从我们的工作中获益。我们的工作是维持生意。我们应该和其他人一起工作，互相合作，而不是把自己埋在“技术的坟墓”里专业人士不会这么做。
