> 原文:[https://dev.to/codemouse92/the-programmers-workbench-4ijc](https://dev.to/codemouse92/the-programmers-workbench-4ijc)

人们不需要寻找太多关于各种编辑器和 ide 优点的讨论，它们无疑是程序员工具箱中最重要的应用程序。许多专业编码人员建议在选择 IDE 或编辑器时要认真思考和努力。一旦找到了自己的编辑器，就值得花时间去学习它，定制它，然后真正地*掌握*它所提供的高级用户特性。从长远来看，时间投资会带来巨大的回报！

同时，忽略许多对程序员来说非常有价值的其他应用程序变得相当容易。我认为我们应该像选择 IDE 一样，在这些方面投入相同的思想、培训时间和定制！

程序员需要的工具因学科而异。这些大多是我用过的工具。请在评论里添加你自己的！

# 计算器

这并不奇怪，编程涉及很多数学！虽然我们中的大多数人(希望如此)都非常精通心理代数之类的东西，但是当我们头脑中有 60 多条编程逻辑线索时，我们真的不想清除更多可用的心理记忆来做乘法。

然而，一个好的计算器应该不仅仅能够进行基本的计算！我发现以下所有内容在编码计算器中都相当重要:

*   复制/粘贴支持
*   计算机代数系统
*   基本转换，例如十进制到十六进制，八进制到二进制，等等。
*   三角函数
*   用户定义的函数和变量

**[【SpeedCrunch】](https://speedcrunch.org/)**符合要求，也是我长期以来最喜欢的计算器。这是我在工作的电脑上首先安装的东西之一。当然，我仍然有值得信赖的 TI-89，但复制粘贴的便利性使 SpeedCrunch 在许多情况下更加实用。

我最终将 TI 主要用于绘图，这在算法分析中非常有用。如果我在电脑上需要同样的功能，我推荐 **[ZeGrapher](https://www.zegrapher.com/)** 。

# 电子书阅读器

我喜欢书...可能有点过了。正如我的 Goodreads 个人资料可以证明的那样，我很容易在任何给定的时间阅读 3-5 本不同的书。虽然我喜欢纸质书，尤其是因为在我编码的时候少了一件占用屏幕空间的事情，我也可以欣赏一本好的电子书。多亏了几个简陋的包，各种免费的电子书，一些随机购买的东西，还有 [Packt 的每日免费电子书](https://www.packtpub.com/packt/offers/free-learning/)，我有了...几百个。

这就是为什么我知道一个好的电子书阅读器的价值！因为我没有 Kindle、Nook 或任何类似的电子阅读器，所以我必须依赖我的电脑。一个好的电子阅读器应用程序不仅仅是显示书籍；我希望能够管理和搜索我的收藏，书签页面，并(由于我的阅读障碍)控制页面显示颜色和字体。

根据我的经验， **[口径](https://calibre-ebook.com/)** 可以很好地处理所有这些任务。我可以存储 ePub、Mobi 和 PDF 电子书，维护其中的目录元数据，并(当然)阅读它们。

既然说到这个话题，你也应该找一个你喜欢的 PDF 浏览器。作为一个 Linux 用户，我喜欢。

# 记录员

我的桌子上有太多的便利贴和纸屑。这可能不是一件坏事，只是我很少整理它们。科学家们已经开始研究它们的迁移模式，它们成群结队地从我的桌子移动到我的大金属“收件箱”，最下面的抽屉，然后回到桌子。

实际上，我认为他们已经形成了一个原始社会。我很担心。

我非常相信模拟，但笔记是个例外。一个好的笔记应用程序可以帮助我控制纸质书的数量。

我试了几十个，但最终选定了 **[Simplenote](https://simplenote.com/)** 。这是一项免费、开源、基于云的服务，这意味着我可以在多种设备上同步我的笔记。

在此之前，我对 **[RedNotebook](https://rednotebook.sourceforge.io/)** 有着相当不错的体验，我现在仍然不时地使用它。它缺乏云同步，但标签和日历功能非常有用。

# 音乐播放器

如果你讨厌音乐，请举手。...是的，我看到后面的两个人。你好。你可以调到下一个标题。如果你想在那之前吃点心，后面有饼干。

我们大多数程序员都离不开对音频环境的完全控制。我们投资好耳机是有原因的！就我个人而言，我正在抨击*托比马克——目前我的降噪 JVCs 中的元素*。

找到合适的媒体播放器同样重要:它能让你快速找到你想要的音乐，按照你的方式建立播放列表，并轻松控制它——最好是用你键盘上那些方便的媒体键。

老实说，没有一款音乐播放器完全符合我的要求，但是在我完成自己的作品之前，我只能退而求其次。

因为我有很多真正的 CD、MP3 和 OGGs，所以我经常使用 **[Rhythmbox](https://wiki.gnome.org/Apps/Rhythmbox)** ，主要是因为它不碍事。(我想念 Banshee，但它患了严重的 bitrot)。

就流媒体音乐而言，我是 Spotify Spotify Spotify T2 T3 的忠实拥护者。他们的免费计划让我可以在我的桌面上播放我想播放的任何东西，并管理我数百个精心策划的播放列表。(当然，我必须定期听到商业广告，但这就是静音按钮的作用！)

对于定制的网络电台，我也有过很好的体验，比如使用 **[【潘多拉】](https://www.pandora.com/)** 和 **[iheart](https://www.iheart.com/for-you/)** 。

# 通讯工具

显然，你别无选择，只能使用你的团队喜欢的任何聊天服务，无论是 Slack、Telegram、Gitter、Discord 还是其他。但是你的社区在哪里呢？

我在 IRC 上有很多好朋友，所以我的聊天客户端是我工具箱中必不可少的一部分。 **[HexChat](https://hexchat.github.io/)** 是我最喜欢的，虽然我绝大多数的朋友都喜欢 **[Irssi](https://irssi.org/)** 。当然，还有更多 IRC 客户端可供选择。通过实验，我找到了自己的最爱；HexChat 正好满足了我的需求，我已经得到了我想要的方式。

对于你可能使用的所有其他聊天服务来说， **[Rambox](https://rambox.pro/)** 是一个真正的神智拯救者！任何时候我需要松弛，不和谐，Mattermost，Gitter，无论什么...我只是把它添加到我的 Rambox 客户端。它位于我的通知区，只在有什么需要我注意的时候才会亮起。如果我对某个特定的服务感到分心，我可以暂时关闭那个标签页；下次我登录时它会在那里。

快速提示:如果你在网络礼仪方面相对有经验，USENET 仍然是一个东西...这是一件非常值得报名参加的事情！我用 **[潘](http://pan.rebelbase.com/)新闻播报员**来访问 USENET。(要想在 USENET 上看到*，请看[永恒的九月](https://www.eternal-september.org/)。)*

# 网络浏览器

如果你从事 web 开发，这可能是你首先想到的。仅仅为了测试，在你的机器上安装尽可能多的主流浏览器是非常必要的，但是你也应该选择一个供你个人使用的浏览器。

以下是我在浏览器中寻找的内容:

*   相当快，没有吃掉所有可用的内存。
*   低界面干扰。
*   与我的基本网站配合良好，尤其是视频聊天服务！
*   适合我工作流程的书签工具。
*   我喜欢使用的开发工具。(有效地将此归入 IDE 类别。)
*   支持我的基本插件:反分心(Leechblock 或 StayFocusd)，广告拦截器(uBlock Origin)，隐私(Privacy Badger，HTTPS Everywhere)，安全(Web of Trust)，样式表控制(Stylus)。

目前，Firefox Quantum 非常适合这个目的，但我也很欣赏 Vivaldi 和 Brave！(面对现实吧——Chrome 是个内存猪。)

# 文本编辑器

如果你是一个 IDE 用户，选择一个轻量级的文本编辑器*和*是非常重要的！我倾向于在我的 ide 旁边使用文本编辑器...

*   快速编辑；我不想仅仅为了纠正一个简单的打字错误就启动一个新的 IDE 标签，
*   读取和编辑 ide 项目外的文件*，*
*   在您完全重写代码时查看旧版本的代码，
*   处理您的 IDE 不适合的文件格式。

不要吝啬这个。文本编辑器之间的差异是巨大的。为此我喜欢 **[Geany](https://www.geany.org/)** ，虽然我对自己在 Windows 上使用 **[Notepad++](https://notepad-plus-plus.org/)** 的经历记忆犹新。(Linux 的等价物现在已经存在了！查看 **[Notepadqq](https://notepadqq.com/)** 。

你也要在终端里找到自己喜欢的文本编辑器；你永远不知道你什么时候会需要它。Emacs 和 Vim 以及显而易见的选项，尽管也有人支持 Nano 和 Tilde。实验！

# UML 和流程图

图表比一些人让你相信的要重要得多。当你计划复杂的代码时，UML 图、流程图等可以节省你很多时间。当然，没有什么比纸笔更快的了，但是有时你想要一个更光滑、更精致的最终结果...并且更加可编辑。

我有很多使用的好经验。我可以在几分钟内做出一个像样的图表，并将其导出为 PDF 或图像，以便在编码时继续参考。除了所有标准符号，Dia 还允许您创建或下载自定义形状集。

顺便说一下，如果您从事网络或工程学科，Dia 也支持所有这些图表！还挺百搭的。

# 终端

大多数程序员，尤其是 Linux 系统上的程序员，在命令行上花费了大量的时间，所以找到一个真正*喜欢*使用的终端非常重要。有几十种可供选择，每一种都有支持者，所以找到哪一种符合你的需要的最好方法就是实验。不要只满足于系统默认；探索你的选择，找到一个你能爱上的。

我其实有三个喜欢的:

*   **[瓜科](http://guake.org/)** 是我的通用终端。我可以随心所欲地定制它，只需简单地点击`F12`即可调出。另外，标签。；)

*   **[终结者](http://www.linuxandubuntu.com/home/terminator-a-linux-terminal-emulator-with-multiple-terminals-in-one-window)** 是我的初级全屏终端。分割窗口和完全定制是我喜欢它的主要原因。

*   **[炫酷复古终端](https://github.com/Swordfish90/cool-retro-term)** 旨在模仿老式 CRT TTYs，让它成为我想要*感受*炫酷时最喜欢的终端。你可以自定义所有的显示设置，这样你就可以设计出你梦想中的终端。它缺少很多《瓜克》和《终结者》更实用的功能，但看起来*牛逼*全屏！

# GUI 原型制作

在我构建 GUI 之前，我想弄清楚它应该是什么样子。第一步几乎总是纸笔；第二步是 **[铅笔项目](http://pencil.evolus.vn/)** ！

虽然还有其他 GUI 原型应用程序，但我对 Pencil 的体验最好。页面功能尤其有用；我不仅可以在一个模型中创建多个视图，还可以重用和分层它们！

# 合并工具

无论您是否使用存储库，您迟早都需要看到两个文件之间的差异。 **[Meld](//meldmerge.org)** 是我最喜欢的视觉区分和合并工具，可能是因为界面刚刚有意义。
