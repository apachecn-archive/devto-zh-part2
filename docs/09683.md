### [](#introduce-yourself-who-are-you-where-do-you-work)自我介绍！你是谁？你在哪里工作？

我叫阿塔，是一名拥有 UX 设计背景的科技企业家。

我目前是 LWA 解决方案公司的首席执行官，这是一家精品应用程序开发机构，是我和我的好朋友迈克在 2010 年共同创立的。我们是世界上第一批专门在 Windows 上提供应用程序的公司之一，并且在此期间非常成功。不幸的是，Windows 应用生态系统从未起飞，这迫使我们跳槽。

今天，我们的使命是通过帮助企业提供员工喜欢使用的移动解决方案来“消费化”企业软件空间。

当我不在工作的时候，你很可能会在五人制橄榄球场的球门柱之间找到我。

### 是谁或什么让你开始编程的？

电脑游戏！在我高中的最后一年，我接触了一个叫做“反恐精英”的游戏，并很快沉迷其中。由于我好胜的天性，我开始学习计算机硬件、网络、脚本等等。尽我所能在游戏中获得优势。正是在这里，我找到了对技术的新激情，并放弃了童年时成为一名建筑师的梦想，转而攻读计算机科学学位。

### [](#what-lead-you-to-programming-for-windows-devices-and-the-wider-ecosystem)是什么让你开始为 windows 设备和更广泛的生态系统编程？

2010 年，迈克和我正在完成我们一直在做的 Windows 演示基础(WPF)项目。也是在这一年，微软推出了 Windows Phone 7，作为技术极客的迈克立即购买了一部。

我发现 Windows Phone 的用户界面非常令人耳目一新，当我们得知 Windows Phone 应用程序是使用 Silverlight(本质上是 WPF)构建的时，我们毫不犹豫地开始修补开发。我们一起开了一家公司，开了一点玩笑，然后在 Windows Phone 上开发了第一个 YouTube 应用。令我们惊讶的是，这款应用迅速走红，让我们走上了今天的发展道路。

### [](#can-you-talk-a-little-about-the-advantages-and-disadvantages-of-xamarin-based-development)您能谈谈基于 Xamarin 的开发的优缺点吗？

作为一个对 UX 充满热情的人，我不喜欢传统的跨平台框架和“一刀切”的用户界面。话虽如此，编写原生应用程序并不得不维护多个代码库并不理想，尤其是在企业业务解决方案的环境中。我们发现 Xamarin 是一个允许您“鱼和熊掌兼得”的框架，因为它在代码共享和促进本机外观和感觉之间取得了良好的平衡。我应该澄清一下，我指的是 Xamarin。本机开发而非 Xamarin。表单(类似于传统的框架)。

为了。Net 开发人员来说，Xamarin 是一个更容易进入 iOS 和 Android 开发的入口点，尽管理解原生开发是必不可少的，但这是一个更宽容的学习曲线。Xamarin 在 2015 年拯救了我们的业务，当时我们不得不将重点从 Windows 开发转移开来，并提升我们整个团队的技能。net 开发人员来处理 iOS 和 Android 项目。

至于缺点，我认为主要的一点是，在 Xamarin 中，交付偏离推荐的操作系统设计指南的高度定制化的 UI 是不容易实现的。例如，如果你想开发一个游戏，这可能不是一个明智的选择。

### [](#as-a-ceo-of-a-small-and-busy-company-what-does-your-typical-day-look-like)作为一家小而忙的公司的 CEO，你典型的一天是什么样子的？

考虑到在任何特定时刻我需要戴多少顶帽子，我没有一个“典型”的一天。话虽如此，我的大部分时间都花在业务发展上，确保我们有足够的项目进来，以保持运转。通常这包括寻找潜在的机会，撰写提案，审查和谈判合同，以及培养与现有客户和合作伙伴的关系。

尽管我很想全力以赴发展业务，但我仍然经常参与我的 UX 设计团队。我偶尔也是一名 Scrum 大师、软件测试员，直到最近，我还在周末担任首席维护官(也就是看门人)。最后，我还是人力资源、销售和营销部门的主管，领导着一个由我、我自己和我组成的团队！

我可能不应该在“开发聊天”上承认这一点，但开发人员已经不是我戴了很长时间的帽子了…

### 你如何为自己的企业找到新工作？用什么技巧？

我们的大部分工作都是通过口口相传来完成的，因此我们的主要目标是确保我们为客户和战略合作伙伴提供真正的价值，而客户和战略合作伙伴也会乐意向其他人推荐我们的服务。

帮助我们获得新项目的另一个领域是我们多年来获得的媒体报道以及我们在创新方面的声誉。虽然这更像是一种公司文化而不是一种战略，但我们努力利用最新的技术来寻找新的和创新的方法来解决客户的问题，这通常会发现我们提出的解决方案是“当前的”和媒体感兴趣的。

### 迄今为止，在你的软件生涯中，你学到的最难的一课是什么？

准确评估工作量的重要性。在一天结束时，每个软件项目都有一个截止日期和一个预算，一个项目的任务越关键，这些度量对每个参与的人就越重要。

无论你是开发人员、项目经理还是首席执行官，能够准确估计交付一个用户故事、一个 Sprint 或你的整个工作流程需要多长时间是非常重要的。

我发现这是软件开发中最具挑战和压力的方面之一。做对非常困难，做错的后果往往很严重。

### 对于成功的软件职业生涯，你的第一条建议是什么？

努力工作，公开分享你的知识。观众越多越有经验越好。这是知道你是否在正确轨道上的最佳试金石。

### 工作之外，你有什么爱好吗？你认为它们对你的科技事业有任何帮助吗？

我是一个非常好胜的人，所以我的大部分爱好最终都变成了职业。例如，在我“电子竞技生涯”的巅峰时期，我效力于新西兰顶级反恐精英队。当我迷上经典的宝马和赛车运动时，我定制了一辆在南岛获得“最佳 V8”的汽车。今天，我代表新西兰参加五人制足球(一种室内足球)比赛，最近还被评为新西兰年度五人制足球运动员。

五人制足球、赛车运动和电子竞技几乎没有共同点。这个培养我最初知之甚少的爱好并“掌握”它们的过程帮助我建立了信心，并提高了我应对未知的能力。

在科技行业，变化是唯一不变的，有信心接受这一事实会有很大帮助。
