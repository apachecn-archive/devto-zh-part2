# 对象有类

> 原文:[https://dev.to/tcelovsky/objects-have-class-3a0](https://dev.to/tcelovsky/objects-have-class-3a0)

在我用 Ruby 进行面向对象编程(“OOP”)之前，我的编码之旅基本上是一帆风顺的(所以在遇到第一个路障之前，我真的没有走多远)。我阅读课文，直到所有这些 getters、setters、classes 和 objects 在我的脑海中混杂在一起，生活变得毫无意义。

我不得不休息几天，在网上阅读一些其他的资源。我不得不承认，起初它们都没有意义，但是用我自己的话写下一些概念真的很有帮助，我开始看到 OOP 隧道尽头的一丝曙光。

所以我发现一个类实际上只是创建一个基础版本的蓝图。事物的所有属性都可以在它的类中找到，我们可以通过在类中定义新的方法来添加更多的属性。这意味着每一个新事物“出生”时都有一套预定义的标准属性。我认为这是非常巧妙的，因为你所要做的就是决定你想要你的东西看起来是什么样子，它们应该如何表现，然后每一个新的东西都会完全按照你的预期来表现。一个事物的标准名称是对象；所以对象的所有属性都存储在它的类中。

当我明白类只是蓝图，而 Ruby 中的所有东西都是对象时，我非常高兴。然后是实例变量。这些都是漂亮的小东西；我非常喜欢实例变量。一个方法中定义的局部变量不能被另一个方法访问。您可以通过在变量名前面加上@符号来解决这个问题。就是这样！您已经有了一个实例变量，可以被类中的所有方法访问。

Setter 和 getter 方法对我来说是最难理解的。我只是不明白为什么你需要两种方法来让一个东西做某事。[这个](http://instruction.learn.co/student/video_lectures#/231)视频讲座帮了大忙！我必须明白，我实际上定义了两个完全不同的方法，例如，一个是 name 方法，另一个是 name= (name equals)方法。name 方法是 getter 或 reader 方法，它只是返回存储在实例变量中的信息。name=方法是 setter 或 writer 方法，这是可以改变存储在实例变量中的信息的方法。一旦我理解了 getter 和 setter 方法之间的区别，事情就变得清楚多了！

这可能花了一段时间，但我终于对面向对象编程有了一个不错的理解。是时候应用我的新技能，用 OOP 重写我的井字游戏了。好家伙，花了点时间！我想应用概念比学习它们更难，但是我现在已经不再玩井字游戏了，至少在人工智能出现之前。我会让你知道那是怎么回事！
