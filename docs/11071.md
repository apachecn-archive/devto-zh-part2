# [](#idea)想法

几个月前，我开始玩《巫师 3:疯狂狩猎》并立刻爱上了它。可用的高质量内容的数量简直是疯狂的。不仅仅是主要的故事，还有成百上千的支线任务都提供了令人敬畏的体验。然而，有一个游戏内功能(或者更多，一个游戏)如此成功，以至于它目前被移植到一个独立的游戏中: *Gwent:巫师纸牌游戏*。

> Gwent 是两个玩家之间的回合制卡牌游戏，每局进行三轮。每个玩家每回合必须从 25 到 40 张牌中打出一张牌。这副牌最多可以包含四张不同的金卡和六张不同的银卡。金卡和银卡通常比铜卡拥有更强大的能力。一副牌最多可以包含三张青铜牌。每副牌都属于一个提供不同玩法的阵营。每个派别都有不同的“领导者”，他们的功能就像可以随时使用的金卡，每个人都有各自的能力。由于 Gwent 不像大多数传统的 ccg 那样使用魔法系统，所以牌的优势通常是你赢得游戏的原因。
> 
> 目标是通过玩牌和咒语赢得三轮中的两轮，以获得棋盘上称为“力量”的点数。一个玩家在棋盘上的点数比对手多，他就赢了一局。当两个玩家都进入下一轮，或者两个玩家都用完牌时，回合结束。第一个赢得三轮中两轮的人获胜。

*   [*【Gwent:巫师纸牌游戏*(维基百科)](https://en.wikipedia.org/wiki/Gwent:_The_Witcher_Card_Game#Gameplay)

这就是我第一次开发竞赛的灵感来源。甚至在我意识到将会有一个独立的 *Gwent* 之前，我就有了利用网络技术创造一个克隆的冲动。只是为了好玩。但我从未真正采取行动——直到两周前！从那以后，我每天花一两个小时来创造一个我称之为 *Tweng* 的克隆体。

Tweng 不应该是 100%的克隆，但基本的游戏应该是一样的:两个玩家之间的回合制卡牌游戏，每个玩家控制一支不同属性的军队。

# [](#whats-pusher-doing)推子在干嘛？

我使用 Pusher 在一个频道的玩家之间共享*动作*和有效载荷。例如，一个动作可以是*游戏卡*，它的有效载荷是正在播放的卡。另一个是*回合*告诉你的对手你通过了这一回合。

# [](#current-status)当前状态

有一个用 React 编写的客户端和一个使用 Express(用于 API 端点)和 PostgreSQL(持久化通道和用户)的节点后端。

在过去的这个周末，我在补间上做了很多工作，并且创造了第一个可玩的版本！然而，仍然有很多我想添加的功能缺失。如果幸运的话，在开发竞赛截止日期之前，但很可能很快之后。

我真正想在截止日期前完成的是一个合适的移动版本。在开发过程中，我试图同时关注桌面和移动设备。但过了一段时间后，我只专注于桌面，以满足最后期限。

匹配系统目前是基于邀请的。这意味着您可以创建一个频道，并与朋友共享邀请链接。你的朋友加入后，你们两个准备玩一个 *Tweng* 的游戏！
