## FSharp 导师计划

自从我开始学习 FSharp，我就想加入这个社区，学习更多的功能实践。我注册了 slack 社区，找到了许多有助于学习语言、工具和惯用实践的资源。在了解了 FSharp 导师项目后，我报名成为了一名学员，但令我震惊的是，没有足够多的导师可以分配给每个人。我决定作为导师参加下一次会议。在这段时间里，我确实对成为一名导师有所顾虑。我不仅仅为我的个人项目编写 FSharp，我也没有将它应用到广泛的应用程序中，只是我用来执行我的工作和个人事务的各种实用程序。我只是希望我能为我的学员提供一些东西。

## 开始师徒关系

在二月初的某个时候，我在导师表格上签了名，并耐心地等待着。我收到了两封电子邮件，内容是关于导师制如何运作以及我的学员的信息。第一封邮件详细解释了师徒关系流程的机制、流程的指导方针、如何开始关系、学员信息，以及如果时间表或其他问题禁止了关系，该如何处理。发出的第二封邮件介绍了这对搭档，并抄送了基金会。我热情地回复了邮件，介绍了我自己和我想从事的项目类型。我耐心地等待回应。不幸的是，我的学员过了一段时间才做出回应，FSharp 基金会跟进了我的情况，并联系了我的学员，看看事情是否顺利。这非常奏效，因为我的学员很快就回复了关于他自己和他的兴趣的信息。

## 设置框架

我们决定的第一步是我们想给现有的开源项目添加什么特性。对于各种 FSharp 系统，我们有许多选项可供选择。我的主要目标是做一些对用户有影响的东西，同时不需要大量的工作。指导关系正式来说只有六周。我们想要一些能在这段时间内持续的东西，并且对两人有足够的启发性。我浏览了 GitHub 上的各种资源库，通过带有“待价而沽”标签的问题以及类似的问题进行过滤。最后，我选定了 ionide fsharp ide 插件，特别是一个很棒的功能建议，可以通过类型签名进行搜索。第一步是指导学员建立我们的分叉 github 项目。他分叉了这个项目，添加了在我们的项目中包含问题的能力，并把我添加为合作者。我将我们的专题分解成几个不同的问题，这样我们就可以分小部分来解决它们。
