### [](#introduce-yourself-who-are-you-where-do-you-work)自我介绍！你是谁？你在哪里工作？

我是 Jethro Carr，我专门研究云基础架构和实施。有些人可能称之为 DevOps。我目前在新西兰惠灵顿办事处的移动产品团队为 Sailthru 工作。

### 谁或什么让你进入了科技行业？

当我还在上学的时候，我很幸运地收到了一些过时的电脑，比如 i386，IBM PS/2 等等。作为一个寻找事情做的时间充裕的年轻人，我过去常常把它们拆开，再把它们组装起来，了解 DOS 和 Win 3.11 的乐趣，并编写基本程序。

最终，我的父母意识到我对计算机这个东西有点兴趣，并最终给我买了第一台当代计算机。从那以后，这是一个全新的有趣的世界，我最终深深地陷入了 GNU/Linux 的世界。对我来说幸运的是，我的爱好变成了一条很棒的职业道路！

### [](#what-is-devops-and-what-does-it-mean-to-you)什么是 DevOps，对你有什么意义？

老实说，我最近很讨厌 DevOps 这个词，因为它被招聘人员和任何寻找新工作的人过度使用，因为它是现在流行的职位名称。装过一次 Linux？DevOps！写了 bash 脚本？DevOps！推到 Heroku？云 DevOps 专家！Sysadmin 今天仍然做着和 1990 年一样的事情吗？DevOps！嘿，谁说他们错了，这个术语现在已经变得如此混乱和毫无意义。就像“It 顾问”。这到底是什么意思？

如果我必须定义它对我的意义:我认为 DevOps 是将基础设施作为代码，在开发团队中紧密合作(理想情况下嵌入)以共同完成事情，以及指导/训练开发团队以构建大规模可靠运行的应用程序的组合。

### 写博客和公开演讲对你的职业生涯有什么影响？

最大的好处是谷歌搜索问题，并找到我自己的帖子，讲述我上次是如何解决我忘记的问题的；-)但除此之外，它在三个关键方面非常有价值:

首先，它对获得就业非常有帮助——几乎所有与我合作过的雇主都说，阅读我的博客(不要脸的 plug:[www.jethrocarr.com](http://www.jethrocarr.com))有助于他们知道，他们正在招聘一个对我推荐的职位充满热情、知识渊博的人。

第二，它对学习如何写好技术文档有很大的帮助——理解如何与具有各种技术技能的人清楚地分享知识的过程是非常有价值的。

最后，做公开演讲只会提高你的演讲技巧。我一开始并不特别擅长公开演讲，但是像大多数事情一样，重复和再重复可以帮助你做得更好。天生良好的演讲技巧有助于每天开会、开会、打客户电话，并有助于职业发展。

### [](#what-do-you-see-as-the-most-interesting-challenge-with-cloud-computing-in-the-next-few-years-and-how-might-that-affect-your-role-and-industry)您认为未来几年云计算最有趣的挑战是什么？这对你的角色和行业有什么影响？

最终，公共云提供商将继续专注于开发产品，使公司越来越容易地做他们想做的事情——构建和交付产品和服务——而不是在配置服务器和基础设施上浪费时间。最终，所有企业都希望找到最简单的方法，将宝贵的员工时间花在他们的业务需求上，而不是做那些不会让他们在市场中更具竞争力的基础设施工作。

这已经导致工作中断——传统上在大公司常见的角色，如“SAN 专家”或“邮件服务器管理员”正在被云存储和托管邮件服务取代——想想现在 G-Suite 和 Office 365 上的组织数量，与 10 年前所有运行 Exchange 或 Postfix 服务器的人相比。这些角色消失了，但新的角色为那些能够利用它们的人敞开了大门——比如 Azure 或 AWS expert

这种趋势将继续下去——一如既往——并冲击目前被视为理想技能组合的工作保障。

现在我不认为 DevOps 的游戏已经结束了，但是工作将会改变。我们已经看到提供商推出了第一代托管 Kubernetes 平台以及无服务器解决方案。你将不再花时间构建工具和管道来支持 Kubernetes 和 CI/CD，取而代之的将只是一个 3 次点击的设置任务。

但是作为一名 DevOps 工程师，您可以在其他方面增加可观的价值，例如努力指导开发团队开发良好的软件和安全架构，进行更多面向性能/可伸缩性的编程和任务，等等。仍然需要许多相同的技能，但是 DevOps 的从业者需要足够的灵活性来适应并能够填充这些领域。

我认为 DevOps 未来的角色更多的是“具有基础设施专长的开发人员”，而不是现在的“能够编码和与人交流的系统管理员”。

### 迄今为止，你在科技领域学到的最难的一课是什么？

发展人际交往技能(人才的“软技能”类别，这一名称并不恰当)。仅仅知道他们很重要或者能够与人交流是不够的。你每天都需要不断地练习。

这是一项很难改进和保持的技能，比任何技术技能都难。而且它们还在不断发展——你的人际工作方式会随着不同代人、不同多样性的同事以及不同的组织/部门而变化。他们都有自己的挑战和需要学习的沟通方式。

### 对于成功的科技职业生涯，你的第一条建议是什么？

准备好每 5 年彻底改造你的技能。科技发展很快，如果你在一条濒临死亡的职业道路上停留太久，你就会被困住，很难再次转变为一条更理想的道路。

当然，说起来容易做起来难，我在这方面的建议是，一些专业技能可以让你在市场上更有吸引力，但不要太局限于一种技术。对你感兴趣的其他技术保持一定程度的了解和参与。不要只看你最喜欢的平台/供应商，这样如果他们改变方向不利于你，你还有选择

我是“t 型开发人员”的忠实粉丝(与“全栈开发人员”相反，我认为“全栈开发人员”会导致冒名顶替综合症，因为任何一个人都不可能成为每个技术栈的大师，这就是太多人对“全栈”的理解)。

作为一名 t 型开发人员，你应该有一些你非常熟悉和有经验的技能，但也有一系列你知道一点的其他技能，你可以在需要的时候成长和扩展。

### 工作之外，你有什么爱好吗？你认为它们对你的科技事业有任何帮助吗？

我的爱好变成了我的工作，实际上，从 100%的时间里做 100%的电脑，到成为一个更加平衡的个体，这花了十年的大部分时间。有些人可能会说，我仍然是 100%的计算机，100%的时间。公平地说，他们可能没有错。

这些天来，我已经减少了花在编码或 Linuxy 上的个人时间，现在包括一些 DIY，最近已经演变成家庭自动化，我现在非常喜欢。所以我想我没有离屏幕太远，但现在也有电动工具参与。

也许这应该是对之前关于职业建议的问题的一个回答…成为一个 24x7 不间断的极客很好，但是，嗯，也许获得更全面的经验也很好。最近，我试图做更多新的体验，看看我喜欢什么，不是工作或家务。
