# 用任何其他名字

> 原文：<https://dev.to/angelariggs/by-any-other-name-3f38>

本周早些时候，Ash Coleman [在 Twitter 上问了一个问题](https://twitter.com/AshColeman30/status/1054903908567408640):“今天我带领一个团队进行了几个类比来描述测试，比如‘看门人’或‘守门员’…很好奇，你用哪些类比来描述测试和测试人员？”

我真的很喜欢这个问题！我没有立即回答，所以这是一个有趣的思考练习。我首先考虑了阿什使用的类比。在一个支持和广泛考虑质量的公司，我认为看门人是一个很好的类比！这让我想到测试人员是最后一道防线——这意味着测试人员是在产品上线之前增加质量或防止退化的最后机会，但是其他人也一直在努力支持测试和质量。

然而，这种情况并不符合我的看门人类比经验。我看到守门人的类比被用作其他人不在他们的工作中包括质量实践，或者不太注意细节的借口，因为它会被测试人员抓住。看门人的观点将测试和质量的责任放在测试人员的肩上，这导致了各种各样的问题。

其他人用一个关于印第安纳琼斯的伟大类比回复了 Ash 的推文:“我喜欢认为印第安纳琼斯…探索和激励并弄清楚它可能被使用的内容和方式”。我喜欢这个！我在很多方面看到测试人员就像琼斯博士一样——无畏的探索者，我们探索前进的道路；我们寻找陷阱和避免它们的方法；我们需要灵活，根据需要快速迭代。除了做得好，我们还努力做好事，经常教导和教育我们周围的人，并确保每个人都为旅程做好准备。我戴着软呢帽看起来也很时髦😉

我们通常对人们有票来捕捉正在完成的工作有强烈的感觉！)
