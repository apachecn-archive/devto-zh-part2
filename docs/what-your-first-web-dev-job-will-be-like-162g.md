# 你的第一份网站开发工作会是什么样的

> 原文：<https://dev.to/flippedcoding/what-your-first-web-dev-job-will-be-like-162g>

很有可能你的第一份网络开发工作将会是你有史以来最好的编程培训。这是因为您可能最终会使用某种遗留代码。你从头开始做一个项目的可能性很小，但这种情况在初创公司比在老牌公司更常见。

通常你的遗留代码工作可以分为几类:调试、重构或者两者的混合。这对初级开发人员的好处是能够使用其他人的代码。这就是你学习好代码和坏代码的区别。

您将获得通读任何编程语言的能力，并快速弄清楚一切是如何工作的。学习如何浏览代码的最大好处是，你还会发现一些非常聪明的做事方法。有一些才华横溢的程序员写出了令人信服的代码。

一般来说，新的 web 开发人员希望进来并开始一个新的项目，因为这是他们最熟悉的。为使用遗留代码做准备，您可以做的最好的事情之一就是在开发人员工具中浏览代码。这为您提供了一条捷径，因为它会突出显示您正在查看的元素，这样您就可以更多地了解 HTML 和 CSS 是如何交互的。

好的一面是大多数公司都有不止一个网站开发人员，所以如果你需要帮助，你可以联系你的同事。高级 web 开发人员通常不介意帮助你，因为他们知道遗留代码有多糟糕。

这就引出了另一个关于你第一份工作的期望。即使你远程工作，你可能会比你想象的和别人交谈更多。大多数 web 开发团队都有某种通信软件，比如 Slack 或 Skype，这样他们就可以保持联系，以防某个服务器出现故障，或者从事同一项目的人相互之间有快速的问题。

那些关于开发人员不和任何人交谈的刻板印象会比太阳中间的冰块消失得更快。你将和一群开发人员一起工作，每天至少要和其中一个人交谈。不过，这并不是一件坏事。通过倾听更有经验的开发人员解决他们遇到的问题，您可以学到很多东西。

在你的第一份 web 开发工作中，你会意识到你实际上知道的是多么少。了解概念和理解它们应该如何工作是一回事，让它们在现有代码中工作是另一回事。你将会学到你不知道自己不知道的东西，这将会帮助你成为一名优秀的开发人员。

老实说，你的第一份工作可能没那么好。当然，有可能你会喜欢它，但是你不喜欢它的可能性很大。不过，这不是你的最后一份工作。这是你迈出第一步的机会，你可以尽快开始尽可能多的学习。

尽你所能做到最好，不要害怕与其他开发人员交流。他们可能有点奇怪，但他们很好，通常不介意帮助新人。请记住，一旦你克服了遗留代码的恐惧，你会因此成为一个更好的开发人员。
