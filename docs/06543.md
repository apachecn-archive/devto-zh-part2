## 我爱 WordPress——它让我的生活变得更美好。

在过去的 18 年里，我一直在建立网站，我对 WordPress 的第一次体验是纯粹的快乐。这些年来，我已经用 WordPress 建立了数百个网站，对于那些为如此美妙的开源项目贡献时间、精力和热情的人们，我只有钦佩。

然而，变革之风就在这里，正如俗话所说，“天下没有不散的宴席”。我担心我们正处在一场灾难的尖端，这场灾难将永远分裂 WordPress 社区，并开始 WordPress 作为首选内容管理系统的缓慢衰落。

## 我害怕的灾难是什么？一句话:古腾堡。

Gutenberg 是一个拖放页面生成器，其令人钦佩的目标是让 Wordpress 的非技术用户更容易快速创建引人入胜的页面。原则上，这是一个很棒的想法，而且这个想法已经被成功地部署在各种非常流行的插件中。

我担心的是古腾堡还没有准备好迎接黄金时代。它充满了错误，用户界面效率低下，需要无休止的鼠标点击，而且它不能很好地与市场上的大多数主题兼容。WordPress 的团队已经决定强迫 Gutenberg 进入 WordPress 的 v5 版本，尽管被 WordPress 社区的[大力反对。](https://wordpress.org/support/plugin/gutenberg/reviews/)

我属于“推后”阵营。在我对古腾堡的反馈充耳不闻后，我意识到 WordPress 不再是一个社区主导的项目——主要的决定是由少数精英做出的。

遗憾的是，我决定是时候转移到核心代码中没有 Gutenberg 的分支了。快速搜索发现没有人采取主动，所以我决定停止抱怨并采取行动。

## 进入 ClassicPress

正是在这一点上, [ClassicPress](https://www.classicpress.net) 诞生了——一个没有 Gutenberg 的 WordPress 4.9.8 的维护分支，专注于社区主导的开发。我充分意识到 ClassicPress 可能会导致我之前提到的社区分裂，我希望[这份请愿书](https://www.change.org/p/petition-to-wordpress-no-gutenberg-in-wordpress-core)会让 ClassicPress 不再被需要，至少暂时如此。

在这个关键时刻，我想说明古腾堡项目背后的团队是敬业的，正在努力实现他们的愿景。尽管有负面的反馈，他们仍然昂首阔步，勇往直前，为此他们应该受到表扬——我知道这对他们来说肯定是一个非常困难的局面。请愿书和 ClassicPress 绝不是为了抹黑他们对 WordPress 的贡献。

话虽如此，如果请愿失败，我相信我自己和成千上万其他长期支持 WordPress 的人将别无选择，只能转向 ClassicPress，以阻止他们的旧网站崩溃(并为未来的网站保持他们的工作流)。如果这是结果，我对 ClassicPress 项目的目标很简单:

*   保持与 WordPress 生态系统的兼容性，让 ClassicPress 从 WordPress core 获得最新的安全补丁、错误修复和非 Gutenberg 特性。
*   让社区来决定 ClassicPress 的发展方向。

我希望这篇文章能提高对 ClassicPress 的认识，这样我们至少可以和 WordPress 团队进行一次公开的对话。马特-你会听吗？你最直言不讳的拥护者正在成为你最直言不讳的批评者。
