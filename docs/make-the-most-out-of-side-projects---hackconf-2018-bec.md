## 弊

既然我们对副业有了一个概念，让我们问一个问题:为什么有人愿意花时间在副业上？

[![Cons](../Images/3fed1dc9b3dbc1561dd061abfcd834fb.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--AcOH_ief--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://s3.amazonaws.com/media-p.slid.es/uploads/909178/images/5251531/pasted-from-clipboard.png)

(解释或更正刚说过的话)我是说...你可能会说你没有时间。我猜你们大多数人都是全职开发人员。你可以说在办公室工作了 8 个小时后，你累了，你一点也不在乎，或者你有一个家庭，你想花更多的时间和他们在一起。在周末，你想远离键盘，去山里远足或参加当地的足球队。

但是记住，人们说你有时间做你腾出时间做的事情。我知道这是一个被过度使用的说法，但它是非常真实的。有一次我看到一个帖子，说的是一位父亲在晚上学习如何编程，他甚至不能睡很多觉，因为他有一个新生的女儿要照顾，但几个月后，他得到了他的第一份开发人员工作。

但是想一想。他投入了时间，得到了回报。一份工作邀请。听起来很棒，不是吗？

你可能会说，没有人会为它付钱，如果你失败或放弃了它，也没有人会称赞你为一个次要项目付出的努力和时间。痛苦的事实是，毫无疑问，大多数副业都会失败。我们都知道这一点。他们中的大多数甚至不会达到第一个版本，而那些达到的，通常会成为废弃的软件，不再被开发者所支持。GitHub 是他们的墓地。

你也可以说它没用。花时间在虚拟的东西上。创建另一个笔记应用程序或替代数百万甚至数十亿人使用的应用程序，目的是什么？

## 优点

让我给你看看所有的优点！

[![Pros](../Images/85cc525ac8cafdb2bada362ed1ee3948.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--h764UkDE--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://s3.amazonaws.com/media-p.slid.es/uploads/909178/images/5251542/pasted-from-clipboard.png)

我想强调的第一个优势是学习新技术，新的开发技能。

例如，如果你是一名 Java 程序员，为什么不在数据科学项目中尝试 Python 呢？或者，如果你是后端开发人员，为什么不尝试一些新的前端框架呢？或者让我们说，你通常做编码的东西，但部署你自己的应用程序到 Heroku 也许和建立一个数据库，詹金斯乔布斯或 SonarQube 来分析你的代码可能是一个挑战！

我们生活在一个除了不断变化的技术之外一无所有的世界。所以你能做的和你应该做的就是跟上我认为的新趋势。最好的工具是通过做一些虚拟的或者不那么虚拟的项目来尝试一个新的库、框架或者工具。

举例来说，你可以有一个项目来学习如何编写干净的代码，如何重构遗留代码或者如何进行测试驱动的开发，而不是一直更新 JavaScript 领域的最新前端框架，因为我们都知道这些事情需要花费大量的时间来掌握。你可以试着读一本该题目的必读书籍，同时将知识落实。

[![Technologies](../Images/4e2175c94381d0587408efbdfce9263a.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--ONO08po2--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ap1zgnppxkv0e3g0wv4f.PNG)

兼职项目的另一个好处是，你可以在办公时间之外进行大量练习。也许如果你在工作和业余爱好项目中使用相同的技术，那么你可能会预先遇到典型的错误和陷阱，这样你就可以在工作中更加自信。您将了解最佳实践，因此您可能能够帮助其他人做出正确的决定，就使用库 A 而不是库 B 提供建议，并为您节省时间，甚至为您的公司节省资金。

我想在这里提到的下一个优势是简单的建造东西的快乐。当你可以向别人展示你的产品，或者看一眼某样东西，知道你就是那个创造它的人时，这种感觉很好，很有成就感。如果你完成了一个别人可以使用或下载的项目，你真的应该为自己感到骄傲。当其他人愉快地使用你的产品时，这种感觉很令人满意。我希望你们很多人已经感受到了。如果没有，我希望至少有一次你会。

如果你有一大堆被抛弃的 git 库，那么你一定知道我之前给你看漫画时在说什么。有趣的是，在你的电脑上发现一个早已被遗忘的文件夹，里面装满了你 17 岁时写的代码。我甚至有一个文本文件来收集有趣的代码名“滑稽代码”。

我的一个朋友曾经跟我说，作为一个开发者，看到自己成长是很正常的。如果你能嘲笑你的旧代码，请高兴，因为如果你不能，那么你肯定没有提高你的技能。

这是另一种可以通过这些旧代码片段简单提高的技能:重构。把它想成是在你必须做的遗留代码上工作，做它需要做的事情，通过把有缺陷的、易受攻击的代码行变成漂亮的东西，一行一行地做改变，小的，但从长远来看是强大的改进。

[![night owls](../Images/c85b68e1253ec68c9b49539b8c197551.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--kiipdKQm--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://s3.amazonaws.com/media-p.slid.es/uploads/909178/images/5261023/pasted-from-clipboard.png)

副业也很棒，因为它们在你的简历上看起来很棒，在面试中，当他们问你的项目时，你会很高兴地说你有这个，那个，甚至还有其他一些东西。这有助于建立网上形象，而且肯定不会对你的简历、职业、人脉和职业发展造成任何伤害。

你应该总是有一个兼职项目，因为这是推进你作为软件开发人员的职业生涯的最好方式之一，毫无例外，就是通过兼职项目。在我看来，这在你职业生涯的最初几年甚至更为重要。尤其是，当你是一名刚毕业的学生时，我希望面试官会更乐意听到你所有的应用程序、网站等等...即使是未完成的。

我的意思是，不仅仅是一长串成功的产品会把你带到那里。我的一个朋友讲述了一个小小的 Android 应用程序如何给他的生活和事业带来巨大变化的故事。

如果你的一个项目会变得严重，那么你甚至可以辞去你的工作，开始创业，成为一名企业家。虽然我不鼓励你现在就辞职，但这是有可能发生的...

从零开始建立一家公司还有其他优势。首先，如果你是一个人或者和朋友一起，那么你将不得不做几乎所有的事情...我指的是市场营销、销售、支持和系统运营。我认为这些都是很重要的事情，哪怕只是一点点。

我的一个朋友还说，一个好的副业能让你在睡觉的时候赚钱...但有时它可能会造成一些伤害，因为有些人在他们的项目开始赚钱时变得疯狂，即使是很少的钱，通常主要目标也不是赚钱，而是学习，进步，成长。

[![business](../Images/d11ea484368e73ce0019dd8c06fef020.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--PR_8ghUs--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://s3.amazonaws.com/media-p.slid.es/uploads/909178/images/5261382/pasted-from-clipboard.png)

我还有几个优势副业给你。例如，如果你在一个开源项目上合作，或者为一个社区做一些事情，那么你将能够与来自世界各地的其他开发者联系，结交一些新的朋友和关系。听起来真不错，不是吗？

还有，就是清爽。这是走出舒适区，做一些不同于日常工作的事情的好方法。

最后但同样重要的是，我会说这很有趣！想想吧！你有想做什么就做什么的自由，没有老板，或者客户来告诉你做什么，怎么做！你不必花几个小时在无用的会议上。你不需要说服任何人相信你自己的观点。你的代码将超级一致，并严格遵循所有的代码惯例:你的。

## 如何

现在我们知道了所有的缺点和优点，我想给你一些建议，告诉你如何最大限度地利用你愿意花在和他们一起工作的时间。

### 第一步。1.找个话题。

通常，当我想学习新技术时，我会建造一些真实的东西，因为弄脏你的手是确保你做对的唯一方法。你需要那些“啊哈！”仅仅通过阅读/观看教程，你不会有这样的时刻。大多数人创建待办事项应用程序。请不要制作另一个待办事项应用程序！如果你要把你超级宝贵的时间花在某件事情上，确保至少把它花在最不可能给别人增加价值的事情上。做点有用的东西。

另一个建议:使用一些笔记应用程序，因为想法可以随时出现，所以要做好准备！否则你会忘记它们....:(

如果你不知道你能做什么，我的网站上有一个[列表](http://practice.floppylab.com/projects/),里面满是项目想法，而且还在增加。我把链接放在那里。当然，你也可以在网上找到数以百万计的其他想法。

### 第二步。如果你已经决定了这个想法，第一个目标是一个计划

别担心，我并不是说你应该从写一份 100 页长的详细文件开始，我想你通常会习惯，因为你的日常工作。我说，它甚至可以是一个随机的文本文件，几行，具有核心功能，你的目标和里程碑。这通常是有用的分离文件的 MVP 水平和你的变化，新功能等。并且不要忘记限制范围和设置最后期限！

### 第三步。找到一个社区

你也可以在网上论坛、小组中找到分享和接收关于副业的建设性反馈。我最喜欢的一个是[边项目 subreddit](https://www.reddit.com/r/SideProject/) 。它非常活跃，适度，你甚至可以从这里获得一些新的想法，当你用它来进行头脑风暴或如果你没有主意了。

### 第四步。做一些简约的东西。我是说，真的。

相信我:如果你是一个有经验的开发人员，很可能你会高估自己的能力。感觉自己一个人比在组织里更有效率是很正常的，这在一定程度上是真的。另外，如果你没有把你的功能列表写得足够短，你很可能永远也完成不了这个项目，因为你会在短时间内失去动力和兴趣。

### 第五步。成为终结者！

尽管你在一个项目上投入了所有的努力，事情还是会发生，所以你总是有一个好的理由不去完成一个项目。比如去度假，上德语课，或者只是在网上阅读有趣的东西。但有时你必须自己说，“嘿，你要完成这个应用程序，并将其放入 Android 应用程序商店。不管花多长时间，你每天至少要花一个小时在这个应用上，直到完成为止。”

[![what a feeling](../Images/fbffad2b06550a978d41944f5ecd1459.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--G1GkeqsQ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://s3.amazonaws.com/media-p.slid.es/uploads/909178/images/5261144/pasted-from-clipboard.png)

### 第六步。去直播！

上线不仅仅是将你的应用发布到应用商店或者将你的网站部署到网络上。它可以是向 npm 发布一个库，甚至只是开源代码，这样人们就可以看到它，使用它，并从中学习。
亦传天下！在你的博客、subreddits、黑客新闻、开发社区甚至 ProductHunt 上发布。你会喜欢从你的实际用户/贡献者那里得到反馈。

### +1。再给你一个建议:去参加黑客马拉松！

黑客马拉松越来越受欢迎，也越来越频繁地被组织起来，因为它是让人们从事某项工作的好方法。你通常和几个人一起在一个项目上工作几天，甚至 3 或 4 天。最后，你会有一个团队，一个详细的概念，甚至是产品的演示版本。听起来太好了，不是吗？

### +2。最后但同样重要的是，尝试改变你的工作场所

在我另一个朋友工作的公司，他们每个月都有一个“实验日”。这对于尝试一些有趣的技术来说是很棒的。许多科技公司都有类似的想法，原因显而易见。你甚至可以成为组织这些活动的人，对吗？

另外，我听说在一些地方他们有演示日，公司的任何人都可以展示他或她在空闲时间做的东西。

还有谷歌著名的“20%政策”，意思是你可以用最多 20%的工作时间做任何事情。为什么一家公司会这么做？因为他们认识到这对雇员和雇主都有好处。

## 那么为什么不开始一个呢？

正如你所看到的，你花在兼职项目上的宝贵时间是有益的，你的小创意甚至可以成长为大公司。我希望你们现在都有动力开始并完成一个！最后，最重要的是坚持走下去，积累经验。
