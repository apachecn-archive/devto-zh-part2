这年头，[性能是特色](https://blog.codinghorror.com/performance-is-a-feature/)！所以，你应该现在就开始重构你的代码以获得更好的性能。对吗？不，你不应该！

在开始改变你的代码之前，花些时间思考一下。请确保您没有犯以下错误之一。

## 错误 1——没有定义什么是重要的

你说的性能是什么意思？您是否在寻找更好的 CPU 使用方式？您是否在尝试减少延迟(从开始到结束运行一个操作的时间)？吞吐量(应用程序在一定时间内可以处理的操作数量)？是否需要减少运行所需的内存量？您对更好地使用磁盘感兴趣吗？您想减少传输数据所需的网络带宽吗？

你需要明确你在努力提高哪些方面。你需要精确。正确的时机就是现在！

## 错误 2——试图在没有可靠测量的情况下优化代码

不管你多么有经验，你都不应该依赖你的直觉。

有一些优秀的工具你可以也应该使用。您需要准确的绩效指标来完成您的工作。不要忽略这个动作。

标杆管理很难(尤其是微观标杆管理)。不要试图聪明地创造你的工具。

## 错误 3——不验证结果

你怎么知道你成功了？唯一的办法就是反复测量。

再说一次，不管你多么有经验，你都不应该依靠你的直觉。

有时候，为了改善一个方面，你会让另一个方面变得更糟。仔细检查你是否准备好处理“改善”的后果。

## 行动时间

再次，[性能是一个特点](https://blog.codinghorror.com/performance-is-a-feature/)。认真对待。

*   精确定义你的绩效目标
*   选择正确的绩效指标，并选择合适的工具为您提供这些指标。
*   仔细检查你的结果
