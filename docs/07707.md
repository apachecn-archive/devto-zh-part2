# 使用 Codewars 提高您的编程技能

> 原文:[https://dev . to/donaldsebleung/improving-your-programming-skills-with-codewars-14a 1](https://dev.to/donaldsebleung/improving-your-programming-skills-with-codewars-14a1)

追溯到 2016 年 1 月——多亏了 [BSD Academy](https://www.google.com/url?sa=t&rct=j&q=&esrc=s&source=web&cd=1&cad=rja&uact=8&ved=2ahUKEwjfnO_N287cAhUbQd4KHRBZAUEQFjAAegQIABAD&url=https%3A%2F%2Fhk.bsdacademy.com%2F&usg=AOvVaw0l5Y06YkbS3QAdVNQ-5ihP) ，我第一次学会如何使用 HTML5 + CSS3 + JS 编写网站和简单游戏代码已经有一年半了。我会偶尔创建一些简单的项目，比如一个由客户端 JS 驱动的 60 秒乘法游戏，或者一个简单的带有背景音乐的圣诞快乐网页，同时不断地在网上搜索相关的主题和信息。然而，尽管我有能力创建简单的项目，但我对算法和数据结构的知识几乎一无所知——我甚至不知道基本的问题解决技术，比如递归，也不知道可以使用`Array.prototype.push`将元素添加到 JS 数组中！

*然后*我发现了[代码大战](https://www.codewars.com)。Codewars 是一个网站，程序员可以在那里接受名为 *Kata* 的编程挑战培训，只要他们通过了入学考试(一个名为“乘法”的调试练习)。在我加入 Codewars 的前两到三个月，我完全沉迷于它——我每天都会花*小时*在它上面完成一个又一个的形，并创作我自己的形(Codewars 是社区驱动的；几乎任何人都可以创造并发布他们自己的形，让其他人来完成。直接的结果是，我在很短的时间内学到了很多关于编程和问题解决的知识，以及诸如`Array.prototype.map`、`Array.prototype.filter`和`Array.prototype.reduce`等方便的 JavaScript 内置代码。即使在最初的两三个月之后，Codewars 仍然是我日常生活中不可或缺的一部分，我通过完成相关的挑战不断学习新的主题，如递归和链表。在加入 Codewars 的短短半年内，我从一个问题解决和算法思维的完全 n00b，变成了一个“精通”的程序员(定义在`1-2 kyu`范围内)。那么，Codewars 相对于 CodinGame、Hackerrank、Leetcode 等类似平台有什么优势呢？

首先，Codewars 上的内容丰富多样。一些 Kata 专注于特定语言的特性或某些流行的框架/库等。比如 React。其他形可能集中在典型的计算机科学主题，如链表，二叉树，递归，记忆和编译理论。然而其他形非常强调数学和形式定理证明。即使对于类似的主题，形的难度也可能大不相同——例如，一些数学形简单到“找出第一个`n`自然数的和”(在恒定时间内或以其他方式)，而其他的可能需要你证明自然数相加的交换性(这是我目前坚持的一个形)或在线性时间内解决 Fabérge 复活节彩蛋问题。相比之下，其他类似的平台，如 Hackerrank 或 Leetcode，可能会更加强调竞争性编程和/或算法思维。

Codewars 的另一个独特之处是它几乎完全由社区驱动——社区中任何有足够荣誉点的成员都可以创建自己的代码挑战(又名 Kata)并发布它们。然后，形进入测试阶段，其他社区成员可以试用、投票并提供反馈。然后，如果/当形被认为是足够的质量，一个主持人(某人一旦他/她达到一定的荣誉级别就自动分配角色)可以批准它，形离开测试阶段-它正式成为网站上的形，每个人都可以在其上训练并获得等级进步。相比之下，某些类似的平台可能不允许社区在其网站上创建新内容，甚至其他社区驱动的平台(如 CodinGame)可能对社区可以创建的内容类型有某些限制。

值得注意的是，Codewars 使用测试驱动开发(TDD)的高度直观的格式，这与你在工作场所看到的非常相似。每个 Kata 都从一个初始解决方案(一些样板代码供求解器开始使用)和一组样本测试用例开始，这些测试用例是使用流行的框架以 TDD 格式编写的断言，如 JUnit for Java 或 HSpec for Haskell(无论如何，对于大多数语言，某些语言如 JavaScript 使用自定义框架，这在 Codewars 的' T0 ' GitHub Wiki page 上有记录)。然后求解器继续(尝试)编写一个通过这些样本测试用例的工作解决方案。例如，在这个过程中，求解者可以编辑这些样本测试用例来编写他/她自己的断言。然后，一旦求解者确信他/她对所述形有一个可行的解决方案，他/她可以点击“尝试”,这将根据形作者提供的(隐藏的)测试夹具来运行它。一旦隐藏测试通过，求解者就被认为通过了形，并可以提交他/她的解决方案以获得等级和荣誉进展，以及查看其他人的解决方案以获得更多见解。相比之下，CodinGame 之类的平台完全依赖于文件 I/O，这是非常不直观且依赖于平台的。

最后，无论你用什么语言编程，你几乎可以肯定 Codewars 支持它——Codewars 目前支持 **36** 不同的编程语言，某些语言甚至支持多个版本，例如 Codewars 支持 Python 编程语言下的 Python 2 *和* Python 3。相比之下，CodinGame 只支持大约 25 种语言(这还是在考虑到 Python 2 和 Python 3 是两种不同的语言之后)，Hackerrank 似乎只显示了不到 20 种语言，我在 LeetCode(在链表上)上尝试的唯一挑战只提供 C、C++、Java 和 Python(甚至不是 JavaScript！).

总之，Codewars 是一个很棒的网站，可以让各种能力和背景的程序员进行代码挑战方面的培训，并提高他们的编程技能，这也提供了比类似(竞争对手)服务更多的优势。
