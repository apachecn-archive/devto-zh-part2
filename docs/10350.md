# 对微软收购 Github 的几点思考

> 原文:[https://dev . to/casivaagustin/some-thoughts-on-github-acquisition-by-Microsoft-2g k3](https://dev.to/casivaagustin/some-thoughts-on-github-acquisition-by-microsoft-2gk3)

在过去的几天里，微软收购 Github 引起了很大的轰动，在我所在的不同社区中有很多活动和评论，也有几个帖子与此相关，是的，这是另一个关于我个人观点的帖子。

值得一提的是，我写这篇文章是基于我的个人经历和这些天我在网上读到的东西，我个人没有任何内部消息来源来确定我在这里说的任何事情，所以请你做出自己的结论。

# [](#microsoft-have-learn-how-to-make-money-with-the-open-source)微软已经学会如何利用开源赚钱

我记得在 2013 年，在 PHPConference Ar，在活动结束时，有两位微软高管上台谈论他们公司的开源，每个人都在笑，并对他们开玩笑，但是他们能够展示所有准备在 Azure Cloud 中使用的开源产品以及当时 MS 对 PHP 开源社区所做的所有贡献。

众所周知，微软最大的业务部门之一是 Azure，他们提供了大量基于开源产品的服务，据我所知，他们用这些产品赚的钱比他们自己的产品(如 windows)还多。

众所周知，MS 是 JavaScript 的最大贡献者之一，拥有 VS Code、TypeScript 等工具。

让我澄清一下，微软不是特蕾莎修女，它是一家大公司，他们追求利润。很明显，他们已经找到了用开源赚钱的方法，这就是为什么他们把赌注压在开源上，为了继续这个策略，他们把目光放在了开源世界上。

# [](#microsoft-does-good-products-for-developers)微软为开发者做好产品

在 Borland 垮台后，MS 是开发人员的最佳选择，他们为开发人员提供了大量令人惊叹的工具，如 Visual Studio、Visual Basic、SQL Server、Visual C++和众所周知的. NET。

随着开源软件的不断发展，大量的新产品和语言如 Java、PHP、Ruby、Python、Gcc、MySQL、PostgreSQL 等进入市场与他们的产品竞争。他们打了一场漂亮的仗，让 MS loose 成为这个市场的一部分。

今天，他们带着像 VS Code 这样的产品回到了游戏中。NET core，SQL Server for Linux，我认为 Github 将在这个新的开源世界中为开发人员提供支持，他们知道新的开发人员需要开源产品。

# [](#github-business-model-works)Github 商业模式作品

Github 作为一个企业是令人惊讶的，因为早期的创始人决定不进行任何创建，而是用自己的资金建立自己的企业，自力更生。他们从第一天起就开始盈利，这在硅谷并不常见。他们的生意很好，很健康。

很明显，Github 的工作方式和销售方式就是这样，我不明白为什么微软会试图改变 Github 业务的本质，他们可能会改变一些规则和价格，但我认为业务的基础不会改变。

# 纳特·弗里德曼是 Github 中最棒的

新任 Github 首席执行官 Nat Friedman 是开源世界的传奇圣骑士，Nat 和 Miguel de Icaza 一起通过 Gnome、Bonobo、Mono 和现在的 Xamarin 等产品为开源做出了巨大贡献。

Together 是世界上少数几个能够理解如何利用开源赚钱的人之一，他们知道如何可持续发展并实现用户的期望。

他们已经和微软并肩工作了很长时间，我个人认为是这两个人挖掘了微软高管的头脑来推动开源的想法，换句话说，我认为微软因为他们而押注于开源。

自从汤姆·普雷斯顿·沃纳(Tom Preston Werner)退出 Github 首席执行官的职位后，我个人感觉该公司正在衰落，不再有新的想法或新的行动；我觉得 Nat 会做出新的更好的 Github。

在这里了解更多关于纳特的信息[https://natfriedman.github.io/hello/](https://natfriedman.github.io/hello/)。

# [](#moving-to-gitlab-because-the-acquisition-is-to-be-a-paranoid)搬到 Gitlab 是因为收购是为了成为一个偏执狂

当我看到这个的时候，我想，哇！

[![alt text](../Images/284f66ce7232b78831adbbba1e47a765.png "Gitlab Migration")T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--iubyyMe1--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/6y73g2bd7nlbzbkp4fe6.png)

让我澄清一些事情，我从 2013 年开始就是 Gitlab 的用户，那时我们已经建立了自己的 Gitlab 服务器，从那时起，我使用 Github 进行开源项目，或者如果我的一些客户要求我使用 Github。

我看到了 Gitlab 这些年来的发展，我认为它是一个令人惊叹的产品，在许多方面甚至比 Github 更好。

然而，我看到这些人转移到 Gitlab，我认为这是一个愚蠢的举动。

我听说有人说

“我将移动我的项目，因为 MS 将窃取我的代码”，真的吗？

我认为，如果你对这个平台感到满意，至少现在还没有理由离开 Github。他们没有改变条款和条件，也没有改变价格，一切还是一如既往。

好吧，但是我的代码呢？！！因为我手里有下一件大事

如果你有一个开源项目，每个人都能看到你的代码，不用担心。

如果你有一个私人项目，你认为 MS 会检查你的代码吗？，真的吗？

如果你有一个私人项目，并且你正在像 Pied Piper 一样做下一个互联网，你认为 Gitlab 是安全的举措吗？Gitlab 看不到你的代码？你在做什么，你还没有创建自己的存储库？

“哈哈，但是 Gitlab 是开源产品啊！”，现在你在乎了？几年前，Github 也不是开源产品。好吧。

在我看来，不要浪费时间转移到 Gitlab，至少你知道 Gitlab 的伟大，你想转移它，因为你知道它在技术上更好，或者在最坏的情况下，因为 MS 改变了规则，你不再喜欢 Github。

# [](#oracle-and-sun)甲骨文和孙

如果你和我一样老，你可能还记得甲骨文收购 Sun 的时候，随着 Sun Java，Virtual Box 和 MySQL 的出现，当时每个人都疯了“现在会发生什么”，“拉里·埃里森是邪恶的”，“我们注定要失败”，“让我们创建一个 MySQL 的分叉”以及许多其他评论。

八年后，整整一代开发人员对此一无所知，他们正在使用 MySQL、Virtual Box 和 Java。

甲骨文从来没有因为我使用 Virtual Box 或 MySQL 而向我收费。

Virtual Box 和 MySQL 比以往任何时候都工作得更好。

可能会有一些特殊情况，这些产品的新许可会影响用户，但这不是一般情况，尽管这些用户也有其他选择，如 Open JDK 或 Maria DB。

如果产品很好，而且产品背后有一个很大的社区，总会有替代方案。

# [](#conclusion)结论

我不是 MS fan，从来都不是，但是我想识别什么时候有东西是好的。我对这次收购有信心，我希望在未来看到更多更好的微软开源产品。

我将把这个伟大的成功留给你们，开发者，开发者，开发者
