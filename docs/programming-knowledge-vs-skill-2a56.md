# 编程知识与技能

> 原文：<https://dev.to/thejoezack/programming-knowledge-vs-skill-2a56>

擅长某事是什么意思？

知识是对一个主题的理解，技能是你应用知识的熟练程度。根据这个定义，你不可能精通你不懂的东西。但是知识本身只是战斗的一半。

当我与其他程序员交谈时，我们谈论我们希望在哪些方面做得更好——谈话通常会很快转向算法、设计模式和数据结构之类的东西。

有时候会有一个特定的语言或者框架，可能是一些比较笼统的比如“游戏编程”或者“网页设计”之类的。

[![](../Images/a04e3c3808b44369ac240ac0ec76873d.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--jMKore3a--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.codingblocks.net/wp-content/uploads/2018/03/Screen-Shot-2018-03-14-at-9.14.05-PM.png)

知识是你知道什么，技能是你能用它做什么。

这听起来像是一个知识问题，但是读一本书或者看一个关于这个主题的视频只会让你走一半。

如果你是一个正在进步的程序员，你不想*知道*算法——你想写它们。

你不想*了解* JavaScript 或游戏编程——你想做很酷的东西！

知道只是成功的一半！技能是另一半。
