# 如何与设计师沟通的技巧

> 原文:[https://dev . to/django _ stars/how-to-speak-the-same-language-as-designers-c1i](https://dev.to/django_stars/how-to-speak-the-same-language-as-designers-c1i)

花一天时间和设计师们一起工作，你会听到他们专业的 UX 术语:线框、模型、界面、网格等等。但是他们所说的并不神秘。为了与设计团队成功协作，您需要说一种共同的语言。这篇文章是为非设计人员量身定制的，将涵盖设计原则，解释它们的重要性，并定义基本术语。如果你是一名设计师，在启动一个新项目时，向你的客户展示这篇文章。

## UX 术语能说明什么？

你可能听过这个故事。一个国王带着他最信任的顾问，给每个人戴上眼罩，这样他们就什么也看不见了。然后他把他们带到一个黑暗的房间。国王对他的顾问们说:“我去了一个遥远的地方，我带回了一些你们从未知道的东西。它叫大象。”

什么是大象？”顾问们问。国王说，“摸摸大象，给我描述一下。第一个顾问揉了揉腿说:“大象是一根柱子。”另一个顾问摸了摸尾巴，说:“大象是一根绳子。”第三个顾问抚摸着它的肚子说:“大象是一堵墙。”最后一位顾问摸了摸象牙，宣布道:“大象是一根实心管子。”"“你们都错了，但同时又都是对的，”国王说。"因为你们每个人只触摸了大象的一部分。"

不同背景、教育和经历的人对设计有不同的看法。他们都像这个故事中的国王顾问。视觉设计师从一个角度进行设计，交互设计师从另一个角度。然后是业务负责人、信息架构师、经理、程序员等等。

## [](#basic-ui-ux-terms-for-design-communication)设计沟通的基本 UI / UX 术语

当你和设计师交谈时，可能经常感觉他们在说一种完全不同的语言。即使你能理解上下文，他们使用的词汇的确切含义可能会让你困惑。所以让我们翻译一些设计师的话。

这本 UX 词汇表将会给经理们一个诚实的总结，让他们对可用性和网页设计问题有更深的理解。它也为客户的设计师提供了可用性标准。

*   **素描**

一种设计者用来提出、探索、提炼和分享想法的绘图类型。UX 设计师将草图作为解决设计问题的第一道防线。

[![alt](../Images/74b8d38c78eb2c9d3f9257c23b78f2b0.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--m3dEYwwM--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://djangostars.com/blog/content/images/2018/05/Img-1%402x-1.png)

*   **线框** ![alt](../Images/5e91255f5f6b7778c61c53f5b750f6ab.png)一页中重要信息的简化草图。也称为页面架构、页面示意图或蓝图。它是设计的框架，应该包含最终产品的所有重要元素。

虽然线框看起来就像一组灰色块，但它是一个低细节的设计展示。这个设计框架包含了最终产品的所有重要元素。它显示了主要的内容组、数据结构以及用户与界面交互的描述。团队中的开发人员、设计师、文案、经理都需要一个精心制作的线框。

*   **能见度**

系统应该在合理的时间内，通过适当的反馈，始终让用户了解正在发生的事情。

*   **可视化**

显示器上信息对象及其关系的图示。战略可视化通过显示器上的对象的接近度来图形化地说明关系的强度。
[![alt](../Images/98b833ea663a5a6d0329798438629c7c.png)T3】](https://res.cloudinary.com/practicaldev/image/fetch/s--hc3fpbxj--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://djangostars.com/blog/content/images/2018/05/Img-3%402x-2.png)

*   **模型**

设计的中等或高细节静态表示。一个好的模型以静态形式展示信息结构、内容和基本功能。实体模型使得传达设计变得容易，并且与原型相比，实体模型的创建过程更耗时。

*   **原型**

[![prototype](../Images/f13ebfaa785ee78f52313e3529ebb4a7.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--gMrR6UYw--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://djangostars.com/blog/content/images/2018/06/1_wZN66CkQmY8mFoQ8XOPcjA.jpeg)

通常与线框相混淆，原型是最终产品的中等或高度详细的表示。它模拟用户与界面的交互。它应该允许用户对内容和界面进行评级，并测试处理应用程序的主要选项。它可能看起来不完全像最终产品，但它绝对不应该是灰色阴影的草图。交互的模型必须足够接近最终产品。

术语“原型”是指产品的样本模型，可以通过测试来确认解决方案是否有效。原型不应被视为最终产品的类似物。他们的主要目的是让设计师、客户和用户看到设计规划是否正确。

在过去的几年里，在网页设计和应用程序开发中，原型相对于线框的价值直线上升。即使是低保真度的原型也能让设计师、客户和测试人员更接近未来产品的外观。

原型是 UX 设计和 UI 设计之间的一个有效且有用的步骤。当你想感受这些元素的效率，并检查在设计过程中是否有任何东西丢失时，原型将是一个很大的帮助。

*   **用户界面**

用户看到的内容；这可以是一组命令或菜单，用户通过它们与程序进行通信。它也是人机互动的地方。

*   **用户体验(UX)**

几个学科的广义术语，研究设计对产品、场所或系统的易用性和满意度的影响。UX 这个术语是认知科学家唐纳德·诺曼博士发明的。

*注意* : UI 和 UX 不可互换！UI 是你的用户与之交互的东西；UX 就是他们做这件事时的感觉。

用户界面可以是用户与产品交互的最终交互区域。它包括了所有加速可用性和满足用户需求和愿望的工具。所有的视觉特征，以及声音和触觉元素，都应该被分析和优化。例如，调色板、类型和动画会影响最终产品的性能。

一般来说，UX 研究和线框图阶段是关于网站或应用程序如何工作，而用户界面是它看起来是什么样子。用户体验更多的是处理逻辑、连接和用户行为。用户界面提供了概念的可视化表示。设计师应该首先集中精力在 UX 部分的布局上，使其功能强大，经过深思熟虑，清晰易用。如果没有这项重要的工作，你可能会把用户界面弄得一团糟。

在通过原型测试了 UX 部件，并且接受了布局、过渡和其他特性之后，设计者开始 UI 设计部件。这是当你的产品的新生的心脏和大脑被皮肤和骨头覆盖的时候。在这里，产品得到了它真正的配色方案、布局的形式和特征、样式、动画等等。

所有的用户界面解决方案都会直接影响用户体验。因此，UX 线框和用户界面设计的过程应该相互支持。

通过让新用户放松来为他们设计一个受欢迎的体验。您站点的入职流程设计通常仅限于首次使用场景。

*   **登陆页面** ![alt](../Images/05361a2715ed274b0b2a2547cfd85a84.png)

用户点击链接后网站上的位置。也称为目标页面或目的页面。

这个概念是 10 多年前由美国的网络营销者创造的。登录页面是一个具有最佳导航和单一行动号召的网页。它的主要目标是增加潜在客户的请求数量，并在网络公司的竞争中脱颖而出。

## [](#how-to-avoid-mistakes-in-design-communication)如何避免设计沟通中的失误

[![alt](../Images/2f30b17572993ec296e4342402d7a457.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--fcLDYyZb--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://djangostars.com/blog/content/images/2018/05/Img-6%402x.png) 
术语对用户查找和理解信息的能力起着很大的作用。许多术语对设计师和内容作者来说是熟悉的，但对用户、管理者和客户来说却不熟悉。

当设计师知道他们在和其他设计师交谈时，他们会以特殊的方式使用词语。在这种情况下，他们正在交流设计概念，他们要么知道，要么假设将是共享的领域。

为了与设计团队建立有效的关系，请始终与设计师一起创建和审查设计概要。这样，你就进入了设计领域。UX 词汇表可能对不熟悉某个主题并希望为他们的项目构建技术路线图的客户有所帮助。

### [](#common-mistakes-in-design-communication)设计沟通中的常见错误

需要对方的两方之间经常会出现笨拙的沟通障碍。客户有时要求过高。相比之下，客户和设计师的关系是由共鸣和理解滋养的。每个人都有犯错的权利。设计师和客户都在同一条船上。

因此，如果你不告诉设计师某些东西必须改变，他们不会知道你在烦恼什么。花点时间仔细看看设计师交给你的初稿。列出你希望看到的改正，并在后续会议上回顾所有的改正。

**“我喜欢/我不喜欢”**

像“我不喜欢”或“这看起来很奇怪”这样的话让设计师听起来很沮丧。在没有准确描述你发现什么奇怪的情况下，说这样的话并不能帮助设计师解决问题。

反馈需要尽可能具体。解释您希望产品的外观和感觉如何。指向特定的元素、样式、字体、调色板或布局。给设计师提供几个你喜欢的设计样本。

你能把那个按钮向左上方移动吗？然后把它变小？”

事后批评和命令设计师应该如何工作会减少他们的信任。设计师无法读懂你的心思，所以不要指望他们有灵异能力。作为客户，你的工作是为设计师定义问题。设计师的工作就是找到解决方案。避免给出过多的详细说明。相反，告诉设计师你或你的观众可能有什么问题，并相信设计师会解决它。

对设计师的想法保持开放，即使它们与你的愿景不一致。允许设计师使用他或她的大脑。

**影响设计沟通的因素**

*   你的时间或预算有限；

*   你以前和设计师有过不愉快的经历；

*   你总是咨询你的个人设计权威；

*   你觉得不得不批评。

从你的平面设计师或插画师那里获得最大收益的最好方法是给他们诚实的反馈。不幸的是，来自客户的反馈通常听起来像是，“会计部的简喜欢这个，但市场部的莎拉说了那个，而我的老板想要完全不同的东西。”没有什么比听到几个不同的人相互矛盾的意见更糟糕的了。

向设计师提供反馈的最好方式是写一份问题清单。你可以和设计师一起仔细检查，讨论对产品进行适当的修改。

视觉艺术和设计领域的人习惯于一丝不苟的批评。但如果你在建议改变时态度苛刻，可能会引发设计师的思维障碍。这就像作家的瓶颈，当你感到卡住了，不知道如何进行。

谨慎而机智地处理你与平面设计师的关系。例如，当谈论改变一种字体时，不要指着你的设计师的作品说:“这种字体很俗气。”相反，你可以说，“我想在这里尝试一种不同的字体。”然后指定你想采取的下一步。

### [](#tips-for-preventing-communication-gap)防止沟通隔阂的小贴士

**问简单的问题来理解问题**

尽早并经常提供反馈。向设计师询问问题可以节省他们的时间，并产生更好的项目，因为元素不必在最后一刻改变。

**多问问题**

将不令人满意的最终产品归咎于设计师是很容易的。相反，给设计师提供他们成功所需的所有工具。作为客户，你的工作是帮助设计师准确理解你对产品的想法和目标。由客户提出更多的问题，以促进有效的沟通。

**立即回答设计师的问题**

尽量按时提供设计师要求的所有必要信息。解释你想要传达的信息。概述你的业务或产品。记住你的拖延会减慢设计过程。

### [](#good-feedback-about-design)关于设计的良好反馈

[![alt](../Images/99a630580aac098340e14403a94db46d.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--zwYnCniH--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://djangostars.com/blog/content/images/2018/05/Img-7%402x.png) 
试着问问设计师他们会有什么建议。以下是一些如何询问设计师专业意见的例子:

*   “我们能做些什么来应对这种情况？”

*   "你认为对我们公司来说最有效的解决方案是什么？"

*   “从你的专业角度来看，你认为哪一个更能引起观众的共鸣，为什么？”

*   "你能再渲染一下，让它更清楚些吗？"

*   “我们希望更多地强调标志。这会儿好像迷路了。”

*   “我担心人们看不到按钮。你会推荐什么让它更突出？”

*   “能不能再解释一下 RGB 和 CYMK 的区别？”

*   “这是一个聪明的开始。然而，我担心这种外观类似于我的一些竞争对手的视觉效果。有没有可能更好地区分我们自己？"

*   “你用的字体是个不错的选择。它非常清晰，会吸引我们年轻的观众。”

设计师感觉自己被微观管理的情况并不少见，尤其是如果客户提供了太多关于设计的细节。当一个设计师有太多的需求时，最终的产品看起来就像是通过在清单上打勾创造出来的。要灵活，抓住什么对你的产品是必要的，什么不是，然后让你的设计师用他们的技能来创造你的愿景。

### UI/UX 术语的知识如何让您受益

希望你有足够的信心参与更多这样的设计对话。询问关于设计过程的开放性问题，并练习及时给出明智的反馈。

要与设计师交流，你必须了解他们的想法。从头开始设计任何东西都是困难的，尤其是当你刚接触一个设计师或设计团队的时候。将概念转化为产品是一项费力的工作，客户需要从一开始就参与设计过程。

长话短说，更多地意识到设计中的沟通差距将有助于你更好地了解你的设计团队——甚至可能像设计师一样思考。
