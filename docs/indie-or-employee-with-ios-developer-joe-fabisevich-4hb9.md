### 自我介绍！你是谁？你在哪里工作？

嘿，山姆，谢谢你的采访！我一直在阅读您对其他开发人员的采访，很荣幸能和他们在同一家公司。

我是 Joe Fabisevich，是纽约市的一名 iOS 开发人员，目前没有具体的公司关系。我职业生涯的前五六年都在为 Timehop、Betaworks 和 Bitly 等初创公司工作。去年，我决定利用我学到的东西，通过咨询工作帮助其他公司成长。

我的工作是帮助公司度过他们目前面临的任何挑战。我最喜欢和早期创业公司一起工作，所以我的很多工作都集中在让创业公司进入 1.0 阶段。很多时候，他们只是从草图和线框开始。其他时候，我帮助企业制定他们的移动战略，或者与 iOS 团队合作提高他们的技能。这一点特别有趣，因为 iOS 最近已经转移到 Swift，到目前为止，该领域的专家很少。我想增加生活的灵活性，现在我能够应对各种不同的挑战。我的工作并不都是与工程相关的，这使得向多个方向发展变得很有趣。

### 是谁或什么让你开始编程的？

这有点老生常谈，但我进入编程是出于偶然。在高中我的成绩很差，我不得不在大三和大四选择一个专业。我妈妈是一名大型机程序员，她告诉我，如果我参加计算机科学课程，她会帮我。(请不要告诉我 11 年级的 AP Java 老师特纳先生)。大约过了两个星期，她宣布她已经完全不适应了，我只能靠自己了。

我从来不擅长死记硬背，但我总是擅长理解模式和系统。编程有助于动手学习，这让我终于对学习感到满意。在取得一些初步成功后，我非常着迷。随着病情好转，我能够开始帮助别人，这也是我以前无法做到的。

在大学里，我主修计算机科学，但相当漫无目的。当 iPhone 出来的时候，我真的很想和一个朋友一起为它开发一个应用程序，所以我鼓足勇气做了。代码很糟糕，UI 以今天的标准来看也很难看，但可能只有大约 10，000 人在做，所以这是一个相当大的成就。因为几乎没有什么 iOS 人才，所以我可以利用这个应用来实习。当我离开学校的时候(在课堂上花了大部分时间开发应用程序之后)，我能够把那次实习和随后的应用程序变成我的第一份创业工作。

### 纽约的科技场景是什么样的？有什么主要的优缺点吗？

纽约市的科技领域是一个大杂烩，但我对纽约的大多数事情都有这种感觉。科技不是这个城市的老大，这有它的优势和劣势。

最大的好处是，作为一个行业，我们基本上是脚踏实地的。我们用我们的技术提升其他行业，也为了技术而技术。这有助于我们避免你在硅谷看到的单一文化。

最大的缺点是，作为一个行业，我们基本上是脚踏实地的。这意味着我们吸引不了那么多的人才(尽管仍然有很多)，或者不像你在硅谷看到的那样有那么多的空想。这些天马行空的想法会成为脸书或 Airbnb，并以一种超乎寻常的方式影响我们的生活。

作为一个人，很难说我是否会用一个换另一个，但这总是一个有趣的思想实验。

### 你既为公司工作过，也为自己工作过——你有什么偏好吗？有什么优缺点？

就像生活中的任何事情一样，任何事情都有利弊。我没有偏好，也不认为我会一直做顾问，但也不认为我会一直全职工作。

作为一名顾问，我获得了难以超越的工作与生活的平衡。这是一个很好的机会，我可以利用额外的时间来投资自己的学习。我可以花更多的时间阅读关于产品、设计、管理，甚至像元认知这样的东西，这有助于我在职业生涯中成长。

另一方面，从长远来看，你可以在一家公司学到一些技能。我仍然在公司工作，但是在公司工作一段时间有助于你发展不同的技能。

我倾向于认为它是以广度优先的方式增长，而不是深度优先的方式。两者都可能让你到达你想去的地方，但是路径看起来会有所不同。最终，什么对你最有效取决于你自己。

### 迄今为止，在你的软件生涯中，你学到的最难的一课是什么？

我觉得职业很重要。开发人员往往不愿意考虑自己的职业生涯，转而考虑编程。这是有道理的，因为需要一段时间才能对自己所做的事情感到足够的自信，但结果是其他人的其他技能受到了影响。

我一直对产品、设计和编程感兴趣，所以选择一条线性道路对我来说很困难，但现在我作为一名顾问和前初创公司创始人也能够利用它。另一方面，直到最近，我才花了很多时间去思考管理或职业轨迹，并意识到我在这方面几乎没有经验，现在我正在迎头赶上。最终，这可能并不重要，因为你无法预测你的未来，但非常重要的是，要尽可能地与你的目标保持联系，不要平行转移。

### 对于成功的软件职业生涯，你的第一条建议是什么？

每个人都是从头开始，这是没有办法的。幸运的是，没事。在我软件生涯的最初几年，我阅读了任何听起来有趣的编程相关的书籍。即使它与我目前从事的工作无关，我还是会读它。当我开始扩展我的视野时，这最终得到了回报。尽管我并不完全理解，但我把所有这些知识都藏在了脑后，以备日后使用。

当我到了尝试 Python 的时候，我不需要从头开始。当我想建一个网站的时候，就像回想起一个月前看的那个教程一样简单。更好的是，我把我在其他语言和框架中学到的经验应用到我认为合适的工作中。这让我以同龄人没有的方式成长，并成为一名更自信的程序员。

虽然这个技巧对我有效，但我不能保证它对每个人都有效。每个人学的都不一样，你得找到自己的路。我的建议是尝试一下，看看它是否适合你！也许它不会是你成长的唯一方式，但它可以是你工具箱中的另一个工具。

### 你认为什么编程技能被低估了？

有两种性格特征是密切相关的。同理心和谦虚可以让你从一个优秀的程序员变成一个优秀的队友。成为一个好的队友可以把你变成一笔巨大的财富。如果你运用这些特质，你会更容易接受自己的缺点，这将让你在同事和社区成员面前更有力量。

移情和谦逊的一个被低估的方面是，随着时间的推移，你会变得更加自信和谦逊。自信和谦逊能让你把任何事情都变成学习的机会。你对自己或同事说“我不知道，但我想了解它”的次数越多，你就越有机会成长。随着时间的推移，它将成为你解决问题的固有部分。
