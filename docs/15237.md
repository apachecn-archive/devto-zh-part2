# 反思我的角色

> 原文:[https://dev.to/angelariggs/reflecting-on-my-role-cka](https://dev.to/angelariggs/reflecting-on-my-role-cka)

*本文最初发表在我的博客上，[http://Angela Riggs . github . io](http://angelariggs.github.io/blog/reflecting-on-my-role)。*

* * *

昨晚，我参加了一个科技小组，参加了一个名为的女性编程活动(太棒了！)，观众提出了一些关于测试和质量保证的非常好的问题。有一个问题引起了我的注意，因为回答这个问题让我思考我在团队中的角色。

我不记得确切的措辞，但它问的是如何处理工作没有通过质量保证时出现的紧张局势——我相信所有的质量保证工程师和开发人员都很熟悉这一点。

最核心的是，信任的基础对于维系这种关系至关重要:相信我会把客户的最佳利益放在心上；相信我会做好自己的工作，支持我的团队和他们的工作；相信我不会要求不合理的工作或返工；相信我的专业知识和建议。

这也需要团队接受和理解质量是每个人的责任。作为 QA 工程师，我可能有责任——但是确保我们在整个开发生命周期中关注质量的责任应该由团队共同承担。

没有信任，没有对责任的认同，这些自然的紧张关系就会变得有害。作为 QA 工程师，我的部分工作是通过帮助创建对团队期望的共同理解来建立信任和责任接受。在我的团队中，我们开始在这个问题上有点纠结，这个问题很好地提醒了我们，有时候你需要和你的团队坐下来，一起重新调整目标和期望。
