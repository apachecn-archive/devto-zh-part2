# 技术筛选:为什么面试过程会让应聘者失败

> 原文:[https://dev . to/techevangelista/tech-screenings-why-the-interview-process-fails-candidates-hc3](https://dev.to/techevangelista/tech-screenings-why-the-interview-process-fails-candidates-hc3)

技术筛选是令人畏缩的。一点也不夸张:当你亲自把它们带给科技界的人时，你通常会被赋予一些非常有趣的厌恶表情。甚至仅仅提到我正在写这篇文章就意味着我收到了一些非常发自内心的评论，没有一条是支持他们的。

如果你不熟悉技术筛选的概念，你应该开始知道它们的名声很糟糕，没有人同意它们应该如何发生，而且不管人们选择什么样的格式，通常有人会感到不便(充其量)。

从那些安排技术面试的人的角度来看，技术筛选是一项标准活动。这是面试官用来确保他们自己的候选人适合这个角色的方法，不管人们经常指出大多数人的日常工作有多么不同。他们很少亲自参加面试，通常是面试过程中的第二步或第三步(例如，经过企业招聘专家或招聘经理的筛选)。

这些技术筛选通常有四种主要方式:查看候选人的个人项目，给候选人一个挑战项目，管理定时挑战，以及现场互联网白板。

以下是每种类型的评论，以及为什么不管你怎么想，它们实际上并没有让你的招聘过程变得更好。再往下，你会找到替代方案的建议。

### [](#four-major-types-of-tech-screenings)四大类科技筛选

#### [](#the-personal-project)个人项目

*是什么:*

人们在 Github 上展示个人项目或者直接把代码发给面试官。然后，他们可能会要求在面试过程中向他们演示部分内容。

*为什么这么做:*

有些地方想看一个人做的代码，而不给出任何特定的挑战。这被视为一个人的兴趣和他们如何解决问题，以及他们对编码的奉献水平的良好指示。

有什么好处:

如果一个人已经有了一个要展示的项目，这给他们带来的压力会大大减少，还能节省他们的时间。这也是人们在现实生活中可能会做的代码类型的一个更好的指示器。

*有什么不好:*
T3】不是每个人都能有副业的。有些人已经在目前的工作岗位上长时间工作了。有些人在家里有责任，比如作为某人的主要照顾者。有些人压力很大，或者情绪低落，或者筋疲力尽。有些人身体上做不了更多。

许多人正在寻找新工作*因为*他们需要更好的工作生活平衡，或者他们目前的工作让他们痛苦，这基本上意味着他们的个人项目要么不太好，要么已经过时。

#### [](#the-challenge-project)挑战项目

*是什么:*

这通常是一种“这是一个挑战(或一组潜在的挑战可供选择)，你可以在[几个小时内]完成，并在[几天内]到期”的格式。

*为什么这么做:*

一般来说，人们认为这既创造了一个有点自然的环境(让人们在开始项目前想清楚，然后一点一点地做下去)，同时也展示了与工作相关的技能。

有什么好处:

这避免了上述个人项目的缺点，同时也让人们能够调整自己的时间来做这件事。

*不好的地方:*

它还是更青睐有更多空闲时间，在现状下更健康/更快乐的人。挑战何时开始/结束也往往没有太多的控制(有时，在初步筛选后，面试官会突然发来一封电子邮件，说这是一个项目，一周内到期)，这让日程繁忙或可能有其他承诺的人很难接受。

如果没有给出实际工作的时间限制(例如，“你应该花不超过 4 个小时”)，那么有焦虑或其他问题的人会对花多少时间在这件事上感到紧张。即使有时间限制，它仍然有利于那些有更多时间的人，因为对候选人/职位来说，一个具有适当挑战性的项目意味着他们至少需要更多时间来重构。时间限制也可能忽略了候选人可能不知道所涉及的技术，因为尽管许多挑战说要使用一个人最熟悉的语言，但他们可能需要候选人以前可能没有使用过的 API、库或其他技术，占用了他们更多的时间，只是简单地学习基础知识，以便面试官可以看到他们熟悉的东西。

此外，这些挑战中有许多都不是很具体，它们不是“这是我们希望你做的，这是我们希望你做的”，而是“这是一个你可以做的多种事情的模糊例子”，这不仅意味着它们不会对候选人进行并列比较，还意味着候选人不仅有项目的工作，而且还有首先提出项目的额外工作。此外，这些很少反映出员工将如何接受工作，或者候选人一旦接受工作后会被期望做什么样的工作，因此面试官不知道他们实际上会在工作中表现如何。

#### [](#the-timed-challenge)限时挑战

*它是什么:*

这种格式通常包括在特定时间必须在特定时间内(比如 2 小时)用 github repo 回复的电子邮件，或者是一个链接到一个专门用于计时编码挑战的网站，一旦你进入挑战就开始。

*为什么这么做:*

它返回足够的代码来评估候选人的技能，测试他们在压力下如何工作，并被视为不太耗时。

*它有什么好处:*
T3】与上述类型不同，每个人在它上面花费的时间是一样的，这对于手头时间较少的人来说更好，对于评估过程来说更好。它可以在一天/一周的任何时间进行，因此可以在候选人的工作时间之外进行。它还显示了某人在压力下的表现，就像常规测试一样。

*不好的地方:*

随着挑战项目的许多问题，这不利于那些不能不间断地花那么多时间的人(父母、有某些残疾的人，等等)。).压力更大。这不是一个“真实世界”的环境，也很少是候选人被要求解决的“真实世界”的问题。它还要求在工作之外定期、体面地访问互联网，这不利于农村地区和其他地区的低收入人群。

用于这类挑战的网站通常都很古怪，没有很好的 UX。他们可能需要相当多的准备时间，仅仅是在开始挑战之前了解他们的来龙去脉，这也有利于那些有更多时间的人。这些网站也可能不完全支持用户选择的语言，或者如果需要库，可能只支持非常有限的库(例如，如果有人面试并使用 Javascript 作为前端开发人员，这些网站可能要求他们拥有该工作不需要的节点知识)。通常他们的设置更多是为了让面试官而不是候选人感到舒服，这与某人喜欢的开发环境和工作空间的运行方式不一致。

#### [](#live-online-whiteboarding)在线直播白板

*是什么:*

应聘者和面试官都在线，看着同一个网站，同时应聘者也在完成一项挑战。

*为什么这么做:*

这基本上只是面对面白板练习的一种替代方式。通常要么作为公司不愿意投入大量时间和金钱的早期筛选过程，要么是为了那些不得不出差参加面试的候选人。

*有什么好处:*

如上所述，考生不用出差(甚至不用离开自己的家)。它允许候选人和面试官之间的实时对话，这也可以允许伪代码，而不是必须做工作代码，从而证明候选人知道该做什么，即使像在现实生活中一样，他们可能需要查找确切的方法。它也可以作为一个结对编程练习，而不是一个直截了当的挑战，向面试官展示候选人如何与他人合作。

*有什么不好:*
T3】就像现实生活中的白板练习一样，许多人发现这些甚至比面试的其他部分更有压力。这些挑战通常与现实世界不兼容，即使它们是兼容的，它们的性质也意味着候选人不应该搜索答案，更不用说从堆栈溢出中复制和粘贴了。面试官通常是一个候选人从未见过面或交谈过的人。有时他们甚至不在这个人将要一起工作的团队中，也不能回答问题，而是在那里盘问候选人。有时，即使这是白板上压力较小的结对编程，公司的员工实际上并不在工作中进行结对编程，这也不是一个很好的指标来衡量一个人作为员工会如何。它还需要不间断的时间和良好、可靠的互联网，就像其他类型的技术放映一样。

因为它是对面试官直播的，这意味着它发生在正常工作时间，有利于在家工作或可以请假的人。它基本上具有现场白板的所有缺点，但没有通常需要几个小时面试的好处，这将使过于紧张而无法在白板上表现良好的候选人仍然可以提出自己的理由。

### 为什么你要在意你的技术筛选是否糟糕？

糟糕的技术筛选有两个非常明显的结果:

首先，通过面试的候选人可能仍然决定不继续面试，或者可能因为糟糕的技术筛选而推迟接受工作邀请。技术筛选代表你对候选人的时间和努力的重视程度，这可以很容易地转化为你对你的*员工的*时间和努力的重视程度。如果你期望候选人做很多工作，当你声称“工作生活平衡”或类似的话，他们真的会相信你吗？如果你不能想出一个现实的项目，他们能相信给他们的任务是现实的吗？或者他们收到的票会写得很好吗？

其次，一个经历过许多糟糕的技术筛选的人会在未来抵制做技术筛选。这可能会直接影响到你(你希望在获得更多经验后申请的人即使受到鼓励也不会这样做)，也可能会间接影响到你(因为这个行业做得非常糟糕，一般来说，适合技术工作的候选人会申请技术含量较低的工作，以避免技术筛选)。如前所述，经常有人试图离开他们的工作，因为技术筛选的问题恶化，因此，这将是有意义的，他们去一个阻力较小的路径。

### [](#the-alternatives)替代品

在经历了这些之后，可能很难想象什么是好的技术筛选——因为实际上没有好的技术筛选。

最好的版本可能是一个灵活的挑战项目，有详细的说明，让每个候选人做基本相同的项目，但有自己的风格。

例如，候选人将被允许给出他们能够完成挑战的日期和时间，并获得生活障碍津贴，而不必向面试官解释个人信息。然后他们会被给予一个与他们将要做的实际工作相关的项目，不仅仅是一个随机的项目，而是可能是真实代码中的一个微小的特性或错误。因为他们在为公司工作，所以应该这样对待，候选人应该通过固定的费用获得报酬(以消除跟踪时间的必要性)。

然而，真正的最佳选择可能是根本没有项目。

编码是可教的，许多人有能力去做。能够在整个面试过程中与潜在的同事互动，并展示相关编程概念的知识，可以说比这些技术挑战所展示的更重要。不管技术筛选强调什么，如果候选人可以就某些话题进行讨论，他们几乎肯定可以在互联网上搜索这些话题，因此能够找出如何在工作中完成这些话题。
