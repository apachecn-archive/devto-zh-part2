# 编码是一门艺术吗？

> 原文:[https://dev.to/chrisvasqm/is-coding-an-art-2kjh](https://dev.to/chrisvasqm/is-coding-an-art--2kjh)

我在我去的大学参加了一个讲座，这个讲座涉及了一些高层次的话题，关于如何把我们的游戏从一个普通的程序员提升到一个伟大的程序员，其中一个演讲者用你在标题上读到的问题与公众分享了他的观点...

是啊，是啊，是啊，我知道我们应该知道干，吻，固体，和一堆其他原则或最佳做法，以实现编码“瓦尔哈拉”，但...编码真的完全没有创造性吗？

演讲者的意见可以恢复成:

> “像设计模式和某些规则这样的东西限制了我们(程序员)做任何形式的艺术。”

不，这与可能的结果无关，比如一个视频游戏或一个应用程序，试着关注代码本身。
