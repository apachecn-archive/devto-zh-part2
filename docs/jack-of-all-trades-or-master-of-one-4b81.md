# 什么都懂，还是什么都懂？

> 原文：<https://dev.to/aeiche/jack-of-all-trades-or-master-of-one-4b81>

我进入现在的职业有点意外。我接管了为一个朋友建立网页的工作，这个朋友去了我们的教堂，我不断遇到用 HTML 无法解决的问题。于是我开始学习 PHP。在做了几份自由职业后，我意识到我可以全职做这个工作。我在一家小公司找到了一份不仅仅是 PHP 的工作。有 HTML/CSS 和 Javascript，还有一种我不记得名字的奇怪语言。我学会了 MySQL 交互。我学习了一些 Apache 配置，并将自己命名为“全栈开发人员”。

在我的职业生涯中，我发现自己不得不学习越来越多的技术，并试图选择赢家。在我目前的职位上，我有时会构建 web 应用程序，有时会开发嵌入式系统原型。我已经在现在的公司工作了 4 年(这在 web 开发中有点奇怪)，因此我的技能范围变得更广了，但也更浅了。我经常想，在这件事上，我是不是在给自己帮倒忙。

### 狭窄却深邃

在我的职业生涯中，有几次我试图缩小我的工作范围。有时这是为了避免某些烦恼:有一段时间我想专门在后端工作，因为我厌倦了处理 IE 和其他浏览器之间的 CSS 问题。在其他时候，我想把所有时间都花在前端，因为我对动态前端体验非常兴奋——比如当我第一次用 jQuery 进行 AJAX 数据加载时，或者当我开始尝试 Angular 和 React 时。

成为某项技术或某项实践的专家，并能够对该技术的社区做出有意义的贡献，是一件值得做的事情。例如，如果你深入了解和理解 React 的工作方式，你就有可能以一种普通用户无法做到的方式为代码做出贡献。

不过，缩小技术堆栈的范围可能是一把双刃剑。在发现它已经被 JQuery 取代之前，我在 Prototype 中度过了许多日日夜夜。我在 jQuery 上花了很多年才意识到它不再流行，更喜欢 React 和 Angular 这样的框架。

### 宽而浅

正如我上面提到的，在我目前的工作中，我到处工作。我已经开始编写 Web 应用程序，微控制器的低级 C 固件，Amazon Echo 的技能，车载信息娱乐应用程序的 C++。这与我之前专门从事网站和网络应用的职位大相径庭。我非常享受这些额外的机会，尤其是在嵌入式领域，因为我非常喜欢电子产品。这也有助于消除我不知何故不是一个“真正的”开发人员的这种恼人的(但愚蠢的)恐惧，因为我没有用编译语言*编写软件。

宽而浅让你有机会探索你在工作环境中的想法和概念。这不是每个人都有的。我已经拒绝了许多工作机会，因为有人想让我在公司购物车或运输技术部门工作。我目前的工作给了我很多探索蓝天的机会。所以我知道的越多，我就越有可能从事新的有趣的工作。

当然，这也有它的缺点。不是某项技术的专家或“忍者”意味着当雇主寻找候选人时，你可能会被忽视。以我的经验来看，从事特定技术领域的工作是深入了解该技术的最佳机会，因为否则你就只能把业余时间花在 it 上了。如果你不专业，你可能会发现自己切换模式如此频繁，以至于你不能做你想要的深度探索。

### 跟随你的直觉

那么开发者该怎么做呢？我认为过去的许多 web 开发人员之所以成为“全栈”开发人员，是因为当时有这种需要。Web 开发人员不像现在这么多，这是让你引起别人注意的好方法。随着大量新的开发者涌入这个行业，这种情况已经发生了变化。我的一些同事仍然处于软件开发职业生涯的早期，他们向我表达了对他们的技能过于分散的担忧。我能给你的最好建议是:*不要因为害怕而做任何事情*。尽你最大的努力掌握你工作的领域，探索你的兴趣。

顺便说一句，这是一种冒名顶替综合症，知道很多人都有这种感觉很重要，即使是我们这些已经从事职业超过十年的人。
