# 他们的编程之旅是如何开始的

> 原文：<https://dev.to/brightdevs/how-their-adventure-with-programming-began-15co>

我曾经问过和我一起在 Bright Inventions 工作的三位开发人员，Adam、Agnieszka 和 Radek，关于他们编程生涯的开始、最初的挑战以及他们最喜欢自己工作的什么。发现他们的故事，看看我们聪明的团队。

## 亚当，全栈开发者

你的编程之旅是如何开始的？

很久以前，我有一个拨号上网连接，我必须为上网的每一分钟付费。我在浏览器窗口打开多个不同的网站，然后离线阅读我下载的所有内容。但是有些时候，当我阅读所有的东西，只是出于纯粹的无聊，我在检查所有这些东西是如何工作的。我的 HTML 之旅就是这样开始的。后来我也用 PHP 创建了自己的网站。

但老实说，中学毕业后，我认为这不适合我。我想学习地理，因为整天坐在电脑前对我来说并不特别令人兴奋。但是我申请了学习计算机科学，我被录取了，然后我就在这里了。我的第一份专业编程工作是用 PHP 创建页面，那是在我学习的第三年。

**开始第一份编程工作，最困难、最有挑战性的是什么？**

在我看来，当我们开始时，我们没有任何在团队中工作的经验，也没有与他人共享代码的经验。这可能是最困难和最具挑战性的部分。你在工作中得到的项目完全不同于你在大学里经常独自处理自己的代码。所以首先我们要学会如何阅读别人的 else 代码，如何不去伤害它。即使代码看起来很糟糕，你也不知道事情以这种方式进行的所有背景和原因。

**你最喜欢编程的什么？**

解决问题——这是我最看重的。如果我能解决一个真正的问题，并且有人能从中受益，我会感到满足。你可能会感到惊讶，但我喜欢“清洁”。我所说的清理是指让代码没有以前那么复杂。简化代码、重构、创建新规则——所有这些都让你摆脱一些多余的东西。当我能把事情变得不那么复杂，不那么复杂的时候，我就可以说我是幸福的。

**你相信编程的专业性意味着终生学习吗？**

看情况。总的来说，如果我们想发展并跟上所有科技新闻，学习的过程是最基本的。这也是我努力去做的事情，同时也是让我保持动力和对工作满意的事情。但我也可以想象一个人专注于一些利基遗留技术，并使他/她的市场价值和竞争优势，因为没有人想这样做。我知道这样的故事。

在光明发明公司工作，你最看重的是什么？

我认为 Bright 是一个我们处理各种任务的地方，同时每个人都感觉自己是团队不可或缺的一部分——你知道没有你这个项目可能不会成功。我们对项目有一个整体的看法。我们的任务是不仅从我们自己狭隘的角度，而且从客户的角度把事情做好。我们尊重彼此的工作，我们知道合作是关键。除此之外，我真的很喜欢没有正式结构和非公司的工作方式。还有一件事——我们可以对任何事情发表意见。没有禁忌话题。我觉得这很酷。

## 阿格涅斯卡，全栈开发者

你的编程之旅是如何开始的？

我在大学开始编程。在学习之前，我并没有写很多代码，但是我一直知道我会学习它。我的父母是信息技术专家，我总是擅长数学、物理或科学，所以这是一个很容易做出的决定。当然，在学习的时候，我可以学到一些理论上的东西，但是没有那么多的实践。我不确定它现在是什么样子。但有一点是肯定的——你必须去工作，开始编码，以获得一些实用和有用的技能。

当你开始第一份编程工作时，最困难和最具挑战性的是什么？

我想，不管第一份工作是否与编程有关，它的开始看起来都是一样的。你不确定你要做什么，你不知道你将和谁一起工作，你不了解公司文化。起初，它可能总是会引起一些担忧。

**你最喜欢编程的什么？**

我喜欢这样一个事实，当你开始编程时，你不必投入大量的金钱来更新和测试许多东西。当你是生物工程师或医生时，你需要有特殊的设备和预算来购买物质、材料等。去实验。在编程中，看起来不是这样的。你只需要有一台电脑(可能还有互联网连接:)就够了。最近，我在为女儿缝制枕头时，意识到我把别针放错了地方。我把它们拔出来是为了让它变得更好，但是后来我注意到我没有必要这样做，因为它可能已经起作用了，所以我又把它们粘住了。在编程中是不同的-你正在做一些事情，测试它，但是如果你想改变一些事情，你不必从头开始，你可以点击“恢复”然后继续，尝试一些新的东西。你不必浪费很多时间去尝试许多不同的解决问题的方法。太棒了。

**你相信编程的专业性意味着终生学习吗？**

是的。当然，不仅你一直在学习的事实证明你是专业的，而且这是专业的一个组成部分。通过学习，你可以了解许多技术，这使得为你的项目选择合适的技术更加容易。学习是至关重要的，但我们也应该合理地使用和参考我们所学的所有东西。

在光明发明公司工作，你最看重的是什么？

就我个人而言，我最喜欢我们平静祥和的环境。没有什么比这更奇怪的压力了。即使当我们犯了错误，出了问题，我们也保持冷静。没有有害的空气。没有人对你大喊大叫或唠叨。我们都知道有任务要做，我们总是尽力做好。

## 拉德克，移动开发者

你的编程之旅是如何开始的？

大概是在一所中学吧。我和高数、物理、IT 在一个班。那里的大多数人选择这种模式是因为数学和物理，但对我来说这是原因。我们有一个教我们 C++的 IT 老师。这是一种类似于数学的分类方法，我必须承认这很简单，但也很有趣。我记得我们解开了许多谜语。但是...等等，事实上我在初中的时候接触过一些编程，但当时似乎并不是特别兴奋。也许是因为我们的 IT 老师在谈论 Pascal 和其他古老的东西。就这样，我不得不等上几年，才能在中学爱上编程。然后就实际多了——我在学校学习，然后在家里重复不同的任务。然后我开始在格但斯克科技大学学习信息技术。但是说实话，我的第一份职业编程工作并不是在学习期间开始的。几年后，我真正的冒险开始了.....但那是另一回事了。

当你开始第一份编程工作时，最困难和最具挑战性的是什么？

从我自己的经验来看，我可以说，最困难和最具挑战性的部分是熟悉所有的工具，从长远来看，这些工具会使你的工作变得更容易，但在开始时，你必须同时面对它们。当你开始你的编程冒险时，你学习一门编程语言，而当你开始你的第一份编程工作时，你同时学习许多不同的东西，而不仅仅是一门语言。编程本身可能没有理解周围的一切那么大的挑战，例如 Git、新的 IDE、用于创建布局的工具(如 Sketch 或 Figma)、代理工具等。一开始可能会有点不知所措。

**你最喜欢编程的什么？**

谁说我喜欢编程了？现在完全严重——我不确定——这不是一个简单的问题。我喜欢不同的东西。但是我认为创造新事物的满足感是我最喜欢的。第一种满足感与创造小东西有关，给应用程序增加一些微小的改进，创造新的功能。你必须选择一些元素，这样它们才会起作用。后者是当你完成一个项目时的满足感，有人使用你创建的应用程序的满足感，以及它真正发挥作用的满足感。所以有日常的满足感和最终的满足感，但最重要的是你创造了以前从未有过的东西这一事实。编程确实是一种手艺。这非常令人满意。

**你相信编程的专业性意味着终生学习吗？**

我确信答案是肯定的，但我只是想知道我是否能找到一些反对的理由。嗯（表示踌躇等）...我不能，所以简单的回答还是“是的”。我们一生都在学习。编程领域是如此的动态，为了跟上时代的步伐，提供越来越好的解决方案，你必须一直学习。但是学习的过程也是每个项目不可或缺的一部分——通过做一些事情，甚至改进你的旧代码，你可以学到新的东西。这与医生的工作完全不同。医生不知道在手术台上给病人开刀时该做什么，我是说，他们以前就知道。在编程的过程中，你会边学习边把你的代码分割成小块，并添加一些新的组件。

在光明发明公司工作，你最看重的是什么？

从事光明发明有几个积极的方面，但我认为最大的价值之一是每个人都有一种合理的责任。我们的工作不仅仅是编写代码。我们负责创造产品的整个过程——我们联系客户，我们计划工作，我们做决定。我们一直和客户保持联系。我们问他们需要什么，不喜欢什么。他们和我们之间完全透明。我们不是被关在玻璃房子里。我们意识到我们每个人所负责的事情。这种意识是伟大的。它激励你变得越来越好。我们都知道更好的工作意味着更好的质量。
