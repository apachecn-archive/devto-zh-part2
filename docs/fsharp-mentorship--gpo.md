# FSharp 导师

> 原文：<https://dev.to/jdhorwitz/fsharp-mentorship--gpo>

[原帖](https://medium.com/@jdhorwitz/fsharp-mentorship-week-1-fe4bfc12ff86)

我还记得当我在网上搜索每一种我能找到的编程语言时，从 Nim 到 Pony，寻找一种能抓住我的语言。我偶然发现了 F#，我至今仍记得看到的一件事是由 FSharp 基金会([http://fsharp.org/](http://fsharp.org/))提供的导师项目。

作为导师/学员关系的大力支持者，这立即引起了我的共鸣。通过接受指导或指导他人，我获得了大量的知识和满足感。我想成为其中的一部分，也想成为 F#的一部分，因为我觉得 F #表达了社区的精神。但是唉！当我第一次发现它的时候，注册已经结束了。

大约一年后，在涉猎了现有的每一种语言，但都没有很深入之后，我上了 F#的网站，注意到目前申请导师是开放的。我急切地申请，并抱着最好的希望。几个星期后，我听到了一个好消息，我被选中参加，并将与一位导师配对！

我的背景是从 Java 开始的，后来转移到 JavaScript，Clojure，Ocaml，任何你能想到的语言。我在工作中花费的大部分时间都是在 Javascript 上，不管是 React 还是 Node，而且直到最近仍然是 Node。我很幸运地在一些伟大的创业公司和大公司工作过，并且有一些很棒的经历。我一直喜欢走在前沿，花大量时间学习新语言，并非常倾向于函数式编程。

我并不是一个编程或函数式编程的新手，但我对。NET 框架或 F#。

幸运向我微笑，因为我和一位伟大的导师 Nat Elkins 搭档，她在 Jet 以写 F#为生。他给了我很大的帮助，他有大量的知识和 F#的经验。我们讨论并提出了一个计划，第一周主要是。NET 生态系统以及他发给我的一些介绍性幻灯片和演示文稿。

。网络完全是另一种野兽。我使用 Linux 已经很多年了，但是我已经看到 Linux 的工具已经变得如此之好，我只是认为微软最近做得非常好。

我花时间学习了 FSharp，Paket，Forge，dotnet，Core CLR，Mono 等等..可以继续下去！这绝对是一个绊脚石，我的大部分时间都花在适应新的生态系统上。我觉得我可以开始发展了，但是还有很多很多东西需要我去学习。

熟悉函数式编程，新的概念如映射、过滤器、平面映射、不变性等..都不是问题，因为我已经有了经验。

我第一次开始经历一些打嗝时，探索歧视工会。我不太明白他们在做什么，也不知道如何在我熟悉的语言中找到相似的语言。在和我的导师谈过之后，他真的帮了我大忙，把它们看作是允许一个类型保存一个值的构造函数，我终于对它们是什么以及它们的用途有了清晰的理解。

昨天晚上，我们有一个很棒的电话，Nat 浏览了演示文稿，并澄清了我剩下的许多问题。总的来说，我度过了一个美好的第一周，并期待更多，真的很喜欢 F#和围绕它的社区。
