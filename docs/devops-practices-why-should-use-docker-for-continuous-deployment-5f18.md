### 什么是持续部署？

持续部署是一种软件开发实践，其中每个代码更改都通过自动化测试，从而成功构建应用程序，并通过多次生产部署投入生产阶段。

这种实践使程序员能够更快地创建、测试和部署高质量的应用程序。

### Docker 简介

Docker 是一款开源工具，允许自动化应用部署，并与云、Linux 和 Windows 提供商“协作”。

Docker 的工作方式如下:它将应用程序及其依赖项打包到独立的轻量级容器中，使它们可以轻松地移植到任何基础设施中。

Docker 有许多优势，首先是加速定制移动或 web 应用程序的开发过程，降低成本，更高质量和安全的应用程序。通过利用 Docker 的优势，程序员大大减少了编写代码和将应用程序投入生产之间的时间。

### 使用 Docker，您可以:

*   将软件开发流程(包括部署)自动化 13 倍

*   根据您的需求在云中或内部部署应用程序
    解决“在我的机器上工作”的问题

*   由于开箱即用的嵌入式安全功能和配置，开发风险最小化的安全应用程序

*   由于优化的基础设施使用和更快的软件开发，降低了成本

*   像管理应用程序一样轻松管理您的基础架构

现在，Docker 是唯一一个允许开发人员从开发到生产阶段创建、部署和管理他们的应用程序，并在云或内部运行它们的容器平台。
