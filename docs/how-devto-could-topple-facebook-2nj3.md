# 戴夫·托如何推翻脸书

> 原文：<https://dev.to/ben/how-devto-could-topple-facebook-2nj3>

我写这篇文章是作为一种个人日志，并不打算分享它，但随着对行动的需求越来越多，我想公开抛出这些想法。

空气中弥漫着许多反脸书的情绪。当然，最近的浪潮是由于脸书在他们的欺诈和掩盖行为被揭露之前无法说出重要事件的真相。但这也是一种更广泛趋势的一部分，即担心脸书的缺陷与他们在该领域的绝对主导地位相混合。

首先，我想澄清一点，我们的团队并没有被夸大的妄想分散注意力。但是，就描述我们当前道路的潜在可选性而言，这篇文章可以作为一个关于技术气候的有趣的思想实验。

* * *

如果你是这个网站 [dev.to](https://dev.to) 的会员或临时访客，你可能会看到我们的平台和脸书之间的一些相似之处。它有一个新闻提要，并侧重于联系和友谊。这是一个社交和/或职业网络。

它在许多方面也不同于脸书。它只面向软件开发人员，建立在开放的网络上——而不是一个有围墙的花园，最重要的是，它只是脸书面积的一小部分。

因此，尽管这种想法很愚蠢，但我确实花时间思考如何以及为什么怀有在脸书的空间中留下痕迹的愿景很重要。“为什么”对我来说很简单。脸书一贯做出道德上有问题的决定。充其量，脸书是权力和规模的必然结果。往坏里说，它是一家冷酷、邪恶的公司，运营商和投资者通常对他们可能造成的伤害漠不关心。

我开始从[发展到](https://dev.to)时，并没有想到脸书。我想在一个鼓励建设性和相互尊重的对话的环境中帮助主持好的技术写作和讨论。这仍然是目标。但是，如果你愿意的话，我想谈谈脸书的事情。

正如我提到的，在外形方面，我们打造的是脸书风格。我们很乐意复制任何社交网络的优点。但有趣的是:我们目前正致力于将我们的核心代码库作为开源项目向世界开放。我们想这样做有几个原因。

*   开源代码会随着时间的推移而改进，而封闭的源代码会衰退。腐烂的代码是昂贵的，并导致工程费用飙升。
*   热情的成员一直在要求这个大时间。社区乐于提供帮助，我们一次又一次地了解到给予社区更多的控制权是如何让我们受益的。
*   我们浪漫地依恋着 Linux 主宰操作系统空间的故事，以及众多看起来不再那么不可思议的开源成就。
*   我们的核心商业模式——为软件开发人员运营一个社区，通过会员和赞助来赚钱——不需要我们囤积代码。
*   我们希望用户对网站上保存的所有数据有一个全面的了解，以便为他们服务。我们实际上并没有保留那么多数据，但是从技术角度来看，最简单的透明方式是公开操作以供检查。

作为题外话——如果你一直在关注这个项目，你可能会对我一直在谈论这个问题感到沮丧。这是真的——在实际发展我们的社区和维持我们的运营的严格过程中，我们一直在努力开源。我不是来宣布什么不同的。事实上，我们越走越近，而不是更远，但我们还没有时间为开放代码做最后的努力。

在我们看来，唯一真正的障碍是彻底的安全审计。我们仍然有一些文档缺陷和入职痛点需要解决，但是我们可以公开地克服它们。开源代码最终比封闭代码安全得多，因为它的问题很浅，但是在让我们的代码呼吸的时候，我们需要小心。

但是回到脸书。我们没有将[开发的范围扩大到](https://dev.to)的计划。有一个区域来谈论技术，并在我们的全部光线下描绘我们的职业或职业相关的自我，这是非常美妙的。但是我们已经看到了一个强大的趋势，其他团体有兴趣支持他们自己的社区实例，基于他们想要的任何前提。

如果发生这种情况， [dev.to](https://dev.to) 将成为兼容但个性化的社交应用生态系统中的一个节点(希望是重要的)。

啊，权力下放的承诺。自从我成为观察者以来，像我这样的梦想家就一直在出售这些魔豆。这就是为什么我总是低着头，埋头研究[开发到](https://dev.to)特性。我不是技术乌托邦。权力下放被视为银弹，人们对待信任就像它是作为区块链的概念被发明出来的一样。我们的团队团结在社会细微差别的困难周围，努力应对社区发展的挑战，并以合乎道德的方式经营一家支持社区发展的企业。“技术上理想的”解决方案似乎在解释人类的社会细微差别方面做得不好。技术视野和专业知识至关重要，但同理心、外交、监管、政策等等也很重要。

这就是我们推翻脸书的方法。**不是一个人，也许会在一起**。我不认为我们的胜算有那么大，但将愿景留在脑海中很有趣。如果你有兴趣知道如何加入，你可以填写[这份表格](https://thepracticaldev.typeform.com/to/HkjWky)加入我们，因为我们逐渐让人们为代码做出贡献。请不要指望任何事情会进展得那么快。你需要耐心。我们更倾向于“稳步解决问题，一路深思熟虑”的态度，而不是一些知名公司创造的“快速行动，打破常规”的说法。

顺便说一句，脸书在我的脑海中是一个最伟大的影子巨人的例子，应该受到尽可能多的挑战，但我不能说社交领域有一个主要参与者没有前科。Twitter 一贯表现出对骚扰毫无价值的冷漠，以及对仇恨和滥用的容忍。LinkedIn 依赖于黑暗模式，通常采取的立场是从其市场领导地位中吸取价值，同时在其相对较长的生命周期中努力提供真正有用的功能。

我认识在所有这些公司工作的出色的人，他们认识到有时候画道德线是一件徒劳的事情。我不反对批评，但我也把建筑作为一种出口，而不是写作。代码是我改变(或大或小)的超级力量，到目前为止它对我很有帮助。我很兴奋能坚持下去。感谢阅读。
