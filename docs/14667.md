我真的同情他。他显然对进入这个领域和学习软件开发充满热情。我们开发社区中的许多人都有同样的困扰感，我们没有学到正确的东西，或者没有跟上。这是身处一个一切都在不断变化的行业的艰难之处。但是，当我想到他的帖子时，我反思了我的职业生涯，以及自从我在 PowerBuilder 中构建客户端服务器应用程序以来，有些事情是如何保持不变的。当我不能学到所有东西的时候，这些积木帮助了我的事业。

## [](#user-interface)用户界面

总会有一个用户界面。鉴于 Siri 和 Alexa 这些天来，它并不总是一个图形界面。但是，将需要从用户那里获得输入并对其采取行动。花时间了解输入数据类型的基础知识(字符串、数字、货币、...)、输入控件(文本框、下拉列表、复选框、单选按钮、...)，以及何时使用它们会很好地为您服务。

## [](#logic)逻辑

不管什么语言，基本的逻辑和流程控制都是必不可少的。每种语言都有机制来:

*   对于循环、 *while* 循环和列表迭代器，使用*迭代序列或列表*
*   创建和操作列表，无论它们是数组、链表、哈希表还是字典
*   使用 *if* 、 *if else* 进行条件分支(拼法有一百种！)，还有*别的*

正如许多人在评论中建议的那样，先深入学习一门语言会让你掌握这些基础知识，从而更容易应用到其他语言中。

## [](#data-storage)数据存储

大多数应用程序在某些时候都需要数据存储，无论是在内存中还是写入磁盘。理解为将来的检索编写数据的基础知识是开发人员职业生涯的主要组成部分。我会从一个好的 ole 关系数据库开始，然后学习 SQL。SQL 是一门美丽的语言。你会惊讶于你所能做的*加入*、*分组通过*、*子选择*。

## [](#grey-boxes)灰色方框

斯科特·汉瑟曼有一篇很棒的文章，他说互联网不是黑匣子。在软件世界中，真的没有多少真正的黑盒。当这些灰色盒子中出现问题时，不管技术堆栈如何，都有许多工具可以提供帮助:

*   监视文件或目录访问(Windows 上的进程监视器)
*   读取系统日志
*   检查服务器性能中的 CPU 或内存消耗
*   检查数据库进程中的慢速查询或锁定
*   查看 web 客户端行为(如 Scott 所解释的 F12)
*   了解 web 资源如何响应不同的 HTTP 动词和标题( [curl](https://curl.haxx.se/) 或我最喜欢的[HTTP pie](https://httpie.org/)
*   查看网络流量( [Wireshark](https://www.wireshark.org/) )

## [](#testing)测试

在我的职业生涯中，一个不变的事实是人们期望我的代码能够工作。多么粗鲁！学会永远不要相信你自己的代码是另一个有价值的职业技能。用空值、空字符串、错误的数据类型、错误的组合、无效的文件格式和错误的 URL 来敲打你的输入。幸运的是，随着单元测试工具的流行，现在这样做要容易得多。

## [](#soft-skills)软技能

你很有可能作为团队的一部分在一个包含人的组织中编写代码。在学习如何编码的时候，别忘了学习如何与人打交道。你需要赢得朋友，影响他人。你可能会发现 Jon Sonmez 的[软技能:软件开发人员的生活手册](https://www.amazon.com/Soft-Skills-software-developers-manual/dp/1617292397)很有用。

虽然学习一切的愿望可能会吸引你，但首先专注于基础知识将为职业发展提供一个良好的基础。
