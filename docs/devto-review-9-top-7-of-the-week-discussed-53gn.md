# 开发人员回顾#9:本周前 7 项，已讨论

> 原文：<https://dev.to/dangolant/devto-review-9-top-7-of-the-week-discussed-53gn>

Malik 和我进入了一个非常需要的安静的感恩节周，我们再次聚在一起谈论本周关于 dev.to 的 7 篇热门文章。

[https://w.soundcloud.com/player/?url=https://soundcloud.com/malikanddan/devto-review-episode-9&auto_play=false&color=%23000000&hide_related=false&show_comments=true&show_user=true&show_reposts=false&show_teaser=true](https://w.soundcloud.com/player/?url=https://soundcloud.com/malikanddan/devto-review-episode-9&auto_play=false&color=%23000000&hide_related=false&show_comments=true&show_user=true&show_reposts=false&show_teaser=true)

我们从[“为什么脸书的 api 以 for 循环开始”](https://dev.to/antogarand/why-facebooks-api-starts-with-a-for-loop-1eob)开始，作者是[安东尼·加兰](https://dev.to/antogarand)。安东尼向读者介绍了脸书和谷歌使用的漏洞缓解策略，以及“JSON 劫持”是如何工作的。

我们接着看[“程序员必读的 29 本书”](https://dev.to/bosepchuk/29-must-read-programming-books-2n45)，其中有[布莱恩·奥斯普丘克](https://dev.to/bosepchuk)的细节...嗯，他读过的 29 本书以及他为什么喜欢它们(有时不喜欢！).布莱恩提出了一个伟大的观点，他花了近 20 年的时间来阅读这些书，而不是把你所有的阅读都放在个人发展上。马利克和我开始讨论我们喜欢和不喜欢的类型。

在[“SQL 系列:关于 SQL 连接的一切”](https://dev.to/helenanders26/sql-series-all-about-sql-joins-15ol)，[海伦·安德森](https://dev.to/helenanders26)，*返回*给读者一个 SQL 连接的可视化分解。我们讨论你可能遇到的陷阱，以及面向初学者的材料对我们这些已经经历过几次的人的持久价值。

从另一位*回归冠军*那里，我们读到了[“dockering a Simple Python Process”](https://dev.to/thejessleigh/dockerizing-a-simple-python-process-2gdk)。在这篇文章中，[杰斯·乌瑞恩](https://dev.to/thejessleigh)讲述了 Docker 为开发者做了什么，以及一些常见的 Docker 文件字段的含义。Malik 和我讨论了我们在 Docker 上的经历，常见的陷阱，以及 Jess 在她的文章中使用的一种伟大的写作方法，这种方法使学习某种不可避免的东西成为可能。

在[“投资组合建议线程”](https://dev.to/aspittel/portfolio-advice-thread-56g9) , 创造性地翻转脚本，征集用户的投资组合网站，并通过对 50 多个投资组合提供反馈，实现了惊人的耐力壮举。Malik 和我讨论了我们在构建投资组合时的目标，为使它们保持最新而进行的永无止境的斗争，以及我们发现的常见模式和反模式。我们最喜欢的一些提交作品来自[斯塔克·博弗劳](https://dev.to/beau_dev)、[香农·克拉比尔](https://dev.to/scrabill)、[比伊](https://dev.to/kbyebea)和[露丝·雷耶](https://dev.to/ruthreyer)。

接下来，我们讨论[“全职、兼职、学习和保持理智”](https://dev.to/simoroshka/full-time-side-projects-learning-and-staying-sane-328l)。 [Anna Simoroshka](https://dev.to/simoroshka/full-time-side-projects-learning-and-staying-sane-328l) 分享了一幅坦率的图片，展示了不断学习和建设的压力会带来什么，并为读者提供了一些可行的建议，告诉他们如何创造稳定的生活方式，围绕这种生活方式，他们可以安排创造性的爱好。马利克和我讨论了我们的价值观，以及我们为稳定生活所做的努力。

最后，我们讨论[“编码时最喜欢听的专辑”](https://dev.to/hus_hmd/favourite-albums-to-listen-to-while-coding-25fb)。 [Hussein Al Hammad](https://dev.to/hus_hmd) 也要求观众参与，在开放一个非常活跃的评论区之前，他以一些器乐金属专辑的建议开始帖子。马利克和我讨论了我们对编码的音乐选择，给出了一些建议，我敞开了我对凯蒂·佩里的情境之爱。我建议人们在 Youtube 上看看[“欧陆节拍漂移”寻找一些笑点。如果你想听我的一个重点播放列表，请点击这里查看。](https://www.youtube.com/watch?v=E7oA3oaRMzs)

我们本周的“主持人选择”是[“我们开发者在帮助谷歌建立一个不可阻挡的垄断吗？”](https://dev.to/sarthology/are-we-developers-helping-google-build-an-unstoppable-monopoly--39ki)，一篇`#discuss`帖子，其中[萨尔萨克](https://dev.to/sarthology)提出了一个问题:开发者是否正在帮助谷歌进一步成为普通大众生活的一部分，这是否是一件好事。马利克和我讨论了我们认为哪些科技公司对人们生活的大部分有影响，以及它们可能产生的影响。
