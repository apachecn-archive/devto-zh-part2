# 为什么我更喜欢不和谐而不是懈怠

> 原文：<https://dev.to/liamhammett/why-i-prefer-discord-overslack-1i4k>

在过去，我使用了太多的聊天应用和解决方案，无论是个人使用还是工作使用。AIM、MSN、Skype、IRC、Pidgin、HipChat 和 matter most——仅举几个我用了至少一年的例子。

但这些年来，它们大多都逐渐消失了，其中两个站在最前沿:**专业用途的 Slack】，以及用于游戏和休闲用途的**Discord**。**

我每天都使用这两个选项，但强烈希望这两个选项不一致，无论是一般用途还是编码讨论。这里我要解释为什么不和谐更好。

## 多台服务器无缝协作

有很多聊天社区——我肯定不止在一个，可能会时不时地进出更多的聊天社区。

Slack 让这变得很痛苦，因为它试图将每台服务器分离成独立的。这当然在某些环境中有它的用处，但是当您在多个服务器中时，这并不理想。在 web 应用程序上，一次只能打开一个服务器，每次你加入一个新的服务器，你都必须重新“注册”它。

**Discord** 通过对所有事情都有一个单一的账户，使它变得轻而易举。您仍然可以获得每个服务器的昵称和身份，但是您只需点击一个链接就可以加入新的服务器，您的所有通知都以相同的方式得到整齐的处理，并且您可以快速创建新的服务器。它还让您能够查看哪些用户在共同的服务器中，在服务器本身之外创建 DMs 和组，以及许多其他方便的功能。

## 处理代码

我是一名程序员，所以在任何一天，我都有很好的机会编写或粘贴一段代码来与他人讨论。

Slack 有这样的选项——使用标准的 markdown，你可以编写内联或多行代码片段。或者，您可以从模式窗口插入一个代码片段，选择代码所在的编程语言，并让该语法在聊天时高亮显示它。

不和谐感觉自然多了。没有弹出窗口来插入代码——你可以从聊天栏完成所有的工作。你可以写 GitHub 风格的 Markdown 来告诉你它用的是什么语言，从而突出语法。也有一些小事情可以防止你弄乱，比如当你写一个多行代码块时，点击“返回”键会插入一个新行，而不是提交消息。

## 免费的

我不是一个会放弃为一个像样的产品付费的人，尤其是一个我每天都会使用的产品，但它肯定会成为问题。

**Slack** 有一个免费计划，但是它只让你存储最后的 10000 条消息。还记得几周前的一次讨论吗，你需要参考或获取对话的上下文？也许有人发布了一个代码片段来帮助其他人，现在你遇到了同样的问题？很难把它找回来。当然，Slack 会让你无限量保存消息，服务器中每个用户 6.30+的费用——这对于社区服务器来说是不实际的。

**不和谐**相反，是完全自由的。无限消息(我仍然有三年前*的消息和文件，不和谐地访问)，群组呼叫，自定义表情符号和你想从聊天应用程序中得到的一切。他们提供名为 Nitro 的付费订阅服务，但这只是提供了一些额外的装饰性好处，核心应用程序并不需要它。*

 *## 正常运行时间

令人有点失望的是，这甚至是一个问题，但肯定是一个重要的问题。

在过去的几个月里，Slack 让我多次经历了几个小时的停机时间，当这是与同事沟通的主要方式时，就有点麻烦了。Slack 确实提供了 99.9%的正常运行时间 SLA，但在*费用是付费计划的两倍*。

**Discord** 这些年来也有不少正常运行时间问题，尤其是在 2017 年，但即使 Discord 出现问题，也很少会影响整个平台。即使一个服务器关闭了，如果你需要联系的话，你仍然可以访问那个服务器上的人。

## DMs 中的隐私

**Slack** ，简单来说，允许服务器所有者导出整个聊天历史——包括你可能私下里发出的直接消息。想私下谈论一些你不想让老板知道的私事吗？不要懈怠。

**Discord** 不提供这些——当然，如果你所在的行业要求你存档所有信息，这可能不太理想，但你可能不会一开始就使用 Discord。

## 个性

一个应用程序如何才能有个性？有很多方法——事物如何表达，有什么图形，让你微笑的小事情，享受使用应用程序的体验。

**懈怠**有个性。它在某些地方很有趣，也很友好，比如当你给一条信息添加太多反应时的措辞。虽然它非常有限，但应用程序本身仍然给人一种公司化和非个人化的感觉——这对它的目标受众来说是完美的，但不是对每个人都是。

## 然而，这并不完美

虽然 Slack 使用起来很愉快，但它并不是万能的。游戏氛围可能不适合所有人，没有办法贴上白标或获得 SLA，轻松导出所有内容，但这不是我最关心的问题。

然而，在我的“想要”清单上，有些东西是最不和谐的:

*   Discord 很少有像 Slack 那样的**专业级集成**。有一个非常活跃的机器人社区，但仍然有很多工作要做，以使 JIRA，特雷罗，GitHub 等东西在同一水平的集成。
*   Slack 为机器人提供了简洁的交互信息功能，比如**问题按钮和菜单**，这使得与这些机器人的交互更加方便。
*   **线索消息**。我不经常在空闲时使用它们，但它们确实有助于保持对话的上下文，而不会扰乱对话的主流。
*   **键盘易用性**。我承认我没有深入研究过这个问题，但是你似乎不能用键盘控制轻松地在这个不和谐的界面中导航。

你对这些聊天应用有什么看法？我错过了让一个公司比它的竞争对手更好或更差的东西吗？*
