# 🍯JAMstack Conf 2018 的两个精彩日！

> 原文：<https://dev.to/takeshape/two-days-at-jamstack-conf-2018-3kg4>

我在[的联合创始人成型了](https://www.takeshape.io/)，我在旧金山的 [JAMstack Conf 2018](https://jamstackconf.com) 待了两天。这是我对事件的汇报。

TLDR:我们非常期待 JAMstack conf 2019！你去 JAMstack Conf 了吗？很想在评论里听听别人的经历！

什么是 JAMstack🍯？JAMstack 是 JavaScript、API 和标记。简而言之，JAMstack 可能是您已经做了一段时间的东西，只是有一个很棒的名字！

我第一次听到 JAMstack 这个词大概是在 2015 年读[“什么是 JAMstack？”](https://www.netlify.com/blog/2015/11/16/what-is-the-jamstack/)，在 Netlify 的博客上。几周前，Netlify 的创始人 Matt Biilmann 为 Smashing 杂志写了一篇名为[“为什么静态站点生成器是下一件大事”](https://www.smashingmagazine.com/2015/11/modern-static-website-generators-next-big-thing/)的文章，这篇文章探索了 JAMstack 背后的思想，但没有使用这个词本身。在 JAMstack Conf 2018 的舞台上，Matt 谈到了 JAMstack 这个名字的由来。像大多数伟大的想法一样，结果是几个朋友坐在他们的客厅里讨论想法。

我是一名前端开发人员，一听到 JAMstack 这个术语，我就爱上了它。作为一个前端开发人员，JAMstack 正在增强。它给你一个框架，把你的想法从前端一直带到曾经是后端开发者的领域。在 JAMstack conf 的演讲中，一个非常活跃和精力充沛的 Chris Coyier 宣称“我是一个前端开发人员，我不需要任何人...!"嘭！

## 如何知道自己是否已经在 JAMstack 上

*   你在为网络做东西？
*   你渴望完美。
*   你正在用普通的 JavaScript / react / vue 构建惊人的动态界面
*   你同意下面的说法:预渲染标记并将其放在 CDN 上是很棒的你知道使用静态站点生成器并不意味着构建一个无聊的体验。这意味着你要建立一个了不起的网站，而不是处理一堆胡扯，如如何建立一个灯栈
*   您嘲笑构建 EC2 实例的想法，并像文明人一样使用无服务器功能。
*   加分——当你听到无头 CMS 时，你会想“太好了，没有 WordPress 了！”，而不是考虑一个骑士。

JAMstack 网站有一个不那么愚蠢的定义。

## 我是 JAMstack 的信徒

我已经接受了 JAMstack 的好处。我希望网站速度快。我希望能够使用一流的服务来组装项目，而不仅仅是特定平台所支持的服务。我想花更多的时间创造，少花时间处理后端。我认为让更多的人创造更好的东西是我们发展一个更加互联的社区的方式。我非常相信 JAMstack，所以我联合创立了一家公司，名为[take shape](//takeshape.io)a headless graph QL CMS，帮助人们用 JAMstack 方法创造更多的东西。
