# 网页开发和网页设计有什么区别？

> 原文:[https://dev . to/flipped coding/web-development-and-web-design-j8g 的区别是什么](https://dev.to/flippedcoding/whats-the-difference-between-web-development-and-web-design-j8g)

人们会在不应该的时候互换使用网页开发者和网页设计师这两个术语，因为他们是完全不同的职业。混乱来自于~~组织不良的项目~~网页开发者做一点网页设计，网页设计师做一点开发。现实中，两人合作做一个网站。他们的角色只是不同。

**<u>网络开发者</u>**

开发人员负责编写代码，并确保网站正常运行。他们从网页设计师那里得到设计，并使之成为现实。如果设计者已经做好了他们的工作，网页开发者就不必担心可用性了，因为它已经融入到设计中了。

web 开发人员将使用 HTML、CSS、JavaScript 和后端语言来设计一个功能性网站。他们不担心网站是否好看，他们只想确保它能正常工作。Web 开发人员在准备编写代码时会使用 Atom 和 Visual Studio 之类的程序。值得庆幸的是，一个网页开发者不需要知道太多关于颜色如何搭配或者什么是最好的字体。

**<u>网页设计师</u>**

设计师关注网站的用户流量。他们会考虑在哪里放置按钮、图片、链接、文本、视频以及你在网页上看到的任何东西。网页设计师为网站上的所有东西设计布局，并且使其易于使用。

网页设计师会关注颜色、排版和用户体验。他们使用 Photoshop 或 Sketch 之类的软件来设计他们的设计，这样他们就可以把它们交给开发人员。与网页开发人员不同，网页设计师不需要有任何编码经验。这是有帮助的，但不是必要的。

**<u>现实</u>**

当然，没有哪个角色能保持如此干净。真正发生的是，网页设计师可能需要知道一点 HTML 和 CSS 来理解开发者在做什么。web 开发人员可能需要了解一点 web 布局，因为最后一分钟的更改并不罕见。

大多数优秀的设计师至少知道一点 HTML 和 CSS，大多数开发人员知道一些最好的设计实践。有一些独角兽在这两方面都做得很好，但在大多数情况下，最好专注于其中一个，因为它们需要不同类型的思维。

Web 开发的方法更符合逻辑。有规则可循，有事情要做的顺序~~(不管做没做)~~。网页设计更具艺术性，因为你试图吸引人们的情感，让你的网站脱颖而出，同时还能匹配你的品牌。
