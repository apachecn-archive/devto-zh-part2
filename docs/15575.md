# 重新开始

> 原文:[https://dev.to/jonesm13/starting-again-4869](https://dev.to/jonesm13/starting-again-4869)

那天是中午，我本该早早溜出办公室，跳上车去伦敦参加一个工作面试。我坐在书桌前，门开着，一种恐怖的感觉慢慢从我的内心升起，穿过我的胸膛，进入我的嘴里。我在一个编码网站上看到一个很小的、相当简单的 C#挑战，我被吓呆了。我甚至不知道如何开始这项任务，更不用说我能做什么来解决问题了。

当天晚些时候的面试是结对编程练习；你和潜在新雇主的一两个开发人员坐在一起，考虑他们描述的一项任务，然后一起着手解决问题，与此同时，他们正在评估你的方法和思维方式。我在日常工作中并没有真正做过多少结对编程；在那之前，我的雇主从来没有真正买过它，每次我与任何人一起看着我的肩膀时，我的手指(正常最高速度为 60+ wpm)变成了冷冻香肠。

我是一个自学成才的开发者；小时候，我们家里总有一台 BBC B/Vic 20/Spectrum 型机器，因为我的父母都是程序员，他们的工作就是在穿孔带上打孔。由于缺乏手眼协调，我从来不喜欢在电脑上玩游戏，所以我主要写小程序。8 位机变成了 PC；BASIC 变成了 VB6 变成了 OOP，最后变成了 C#和. NET，我从来没有，从来没有想过我会以此为生，开发软件会让我在近 20 年的时间里非常满意地衣食无忧。我真的站稳了脚跟，我非常感谢所有信任我的人，尽管我完全没有资格做我所做的工作。

我没有任何学位，更不用说 CS 学位了，我觉得我已经错过了很多我的受过正规教育的同事们所拥有的学术方法。如果不参考模型实现，我就无法编写二分搜索法方法，完成保龄球游戏 TDD kata 或任何其他你期望在这个行业工作了近 20 年的人完成的基本任务。直到最近，我甚至不确定比特、字节、无符号整数和浮点数是什么意思。这就像砖家不知道泥铲是什么，或者医生不知道骨头是什么。

我也对学习资源的彻底改变感到困惑；当我在工作中学习时，没有 Stackoverflow，只有很少的在线代码示例。唯一的资源是书籍；巨大笨拙的枯树，包含的智慧在印刷时可能是正确的，但其主题每天都在演变。如今，模型实现和示例代码无处不在——你只需在 Google 中键入异常消息或方法签名的几个关键字，就会出现那些已经去过那里、做过并为下一个旅行者留下面包屑的善良灵魂的智慧。这导致了一种感觉几乎是寄生的存在；我把别人的代码拿来，根据自己的目的进行修改，然后扔进锅里。这不是我们过去所知道的学习，当然也不会让个人满意。

这并不是说我在职业生涯中一无所获；我对干净、简单、面向对象的代码怀有深深的敬意。我热情地相信，高质量代码的价值不仅仅是“让计算机做事情”，也是对下一个必须继续做这件事的女孩/男孩的一种礼貌，或者实际上是对六个月后的我自己。我相信 TDD 是“足够”发展的关键，因此可以减少浪费和虚荣心。我相信学习是成为一个更自信的人的唯一途径，更不用说成为一名开发人员了，尽管我很想从那些已经走过这条路的人身上学到东西，但我也想成为那些跟随我的人。

我需要学习的是学习的行为本身，而不仅仅是发现。我需要练习这些简单的练习，并把它们当作一次旅程来尊重，而不是把目的地本身当作微不足道的小事而不予理会。我需要学会不被分阶段或恐慌或无聊，因为我不能自己解决它。我需要学习如何慢慢地思考，一步一步来，忘记其他的复杂性，直到我们到达那里。
