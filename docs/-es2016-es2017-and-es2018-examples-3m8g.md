# # ES2016、ES2017 和 ES2018 示例

> 原文：<https://dev.to/johnmunsch/-es2016-es2017-and-es2018-examples-3m8g>

这篇漂亮的文章提供了使用 ES2015 后的所有功能的示例，这些功能你很少听说:[这里是 ECMAScript 2016、2017 和 2018](https://medium.freecodecamp.org/here-are-examples-of-everything-new-in-ecmascript-2016-2017-and-2018-d52fa3b5a70e) 中所有新功能的示例

您可能想知道，“对这些特性的支持是什么？其实这年头 ES2015 有什么支撑？”如果你确实这样认为，那么不要忘记 Kangax 兼容性表: [ECMAScript 2016+兼容性表](http://kangax.github.io/compat-table/es2016plus/)

或者，如果你想了解 Node.js 对它的全部支持: [Node.js ES2015/ES6、ES2016 和 ES2017 支持](http://node.green/)

与这些信息相呼应的是这篇关于 JavaScript 三个可能即将到来的变化的文章:[这里有三个你会喜欢的 JavaScript 即将到来的变化](https://medium.freecodecamp.org/here-are-three-upcoming-changes-to-javascript-that-youll-love-387bce1bfb0b)
