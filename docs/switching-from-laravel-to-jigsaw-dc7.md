# 从 Laravel 切换到 Jigsaw

> 原文：<https://dev.to/wilburpowery/switching-from-laravel-to-jigsaw-dc7>

不久前，我给了自己一份重新设计个人网站的工作。第一个版本是使用 Laravel 框架制作的。

对于第二个版本，我开始思考是否 Laravel 真的是像我这样的简单投资组合网站所需要的。经过几天对我为该网站第一版编写的代码的分析，我发现我只是简单地使用了 Laravel，因为我喜欢它附带的刀片模板引擎。意识到这一点后，我专注于寻找一个轻便、可靠的好工具，我可以用它来简单地建立一个漂亮、醒目的个人网站，在那里我可以创建一个降价文件，并与每个人分享我的想法。在网上看了几天很棒的东西后，我找到了满足我需求的完美工具 [Jigsaw](http://jigsaw.tighten.co/) 。

### 什么是拼图

在其核心部分， [Jigsaw](http://jigsaw.tighten.co/) 是一个用于构建静态网站的框架。它帮助开发人员使用与构建动态 web 应用程序相同的现代工具快速构建静态站点。

### 拼图与众不同的地方。html 文件

使用 [Jigsaw](http://jigsaw.tighten.co/) 而不是创造一些。html 文件并在线提供给他们带来了很多很棒的特性。您可以访问 Laravel 框架的许多优秀特性，但是是以一种轻量级和最小化的形式。

#### [拼图](http://jigsaw.tighten.co/)提供的一些功能有:

*   刀片模板
*   降价页面
*   . blade.md 文件(是的，blade 和 Markdown 混合)
*   全局配置文件(非常方便)
*   许多辅助方法
*   收集
*   整理
*   页码
*   变量和助手方法
*   现成的 Laravel 酏剂支持

### 结论

Jigsaw 是一个创建静态网站的很棒的框架，在这里你只需要显示静态内容，但是你仍然想使用一些你喜欢的 Laravel 的特性，比如 Blade 模板。开始非常容易，他们的[文档](http://jigsaw.tighten.co/docs/installation/)很棒，而且它仍然是你喜欢和享受的 Laravel 社区的一部分。
