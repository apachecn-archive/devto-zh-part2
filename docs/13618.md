# 成为技术领导者:信任你的团队

> 原文:[https://dev . to/craignicol/being-a-technical-lead-trust-your-team-dff](https://dev.to/craignicol/becoming-a-technical-lead-trust-your-team-dff)

当我第一次成为技术主管时，我面临的最大挑战之一是放弃控制权。这并不是技术上不信任我的团队，而是我不想让他们负担过重，所以我承担了所有的计划、所有的客户互动，以及所有其他我觉得让我从编写代码的工作中分心的工作。

我错了。

我需要让我的团队参与进来，因为他们有我没有的想法，他们有能力，而我是一个瓶颈。我拖住了我的队伍。

但是我需要确保编码标准得到遵守，质量得到维护。你知道还有谁关心这个吗？我的团队。我把确保代码符合标准作为他们的责任，并推行代码审查。我把内部文档和代码放在同一个级别。任何人都可以编辑，但是每个人都可以审核，或者否决。

我在回顾会上要求[诚实的反馈](https://craignicol.wordpress.com/2017/08/01/the-importance-of-feedback/)，而[实施了变更](https://craignicol.wordpress.com/2015/05/19/i-dont-trust-change/)，这样他们就可以相信我在关注他们，我给他们设定了改进的目标，以及特色工作:“我们能一周发布两次吗？”(挑战—自动化解决棘手问题)；“我们能把测试中发现的错误减半吗”(挑战——更好的单元测试)

我越信任他们，支持他们，理解他们的局限性，他们就表现得越好。我也和不信任自己团队的[经理一起工作过](https://craignicol.wordpress.com/2015/08/18/dear-manager/)。那些团队工作得不是很好。

如果你不信任你的团队，他们就不是你的团队。他们只是在你身边工作的人。你今天做了什么来培养信任？
