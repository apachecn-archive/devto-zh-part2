# 我在父母家找到的书

> 原文：<https://dev.to/rhymes/books-i-found-at-my-parents-3192>

如果你有一个半游牧阶段，你知道你父母的房子是如何完美的长期储存场所。所以，我的大部分编程书都在那里。你应该看看他们有多“高兴”让它们躺在那里这么多年。

有的不见了(壁炉用 kindle？)但是我把今天找到的那些堆起来，决定把它们变成一个帖子。

让我们谈论一些书。

*   Python 食谱，第二版。曾经有一段时间，堆栈溢出是不存在的，像这样的一本书价值连城(所以是从 2008 年，从 2005 年的书)。它的许多内容都是在一个 Python 专用网站上收集的，但这本书得到了扩展，包含了更好的解释，并且充满了由 Python 大师 Alex Martelli 编写的代码。我现在不会推荐它，也不会推荐它基于 Python 3 的第三版。

*   尽管有这个标题，[算法简介](https://mitpress.mit.edu/books/introduction-algorithms-third-edition)并不是真正的简介。它由麻省理工学院出版社编辑，并在我的大学课程中。我讨厌它:它迂腐、无聊，使用伪代码和数学符号，而且...有更好的书可以帮助你从零开始，尤其是如果你不是数学家而是软件开发人员。尽管如此，它的价值在于它的深度。

*   [Head First: Design Patterns](https://www.amazon.com/Head-First-Design-Patterns-Brain-Friendly/dp/0596007124) 是关于设计模式最好的书。比经典的 GoF 要好。有那么好。

*   《Arduino 入门》是我在米兰参加了一个关于 Arduino 的工作坊，免费得到了套件，回家后在三纳秒内对硬件修补失去兴趣的结果。

*   [JavaScript:好的部分](https://www.amazon.com/JavaScript-Good-Parts-Douglas-Crockford/dp/0596517742)还是一本很扎实的书。它是 ES6 之前的版本但是语言的核心并没有太大的变化(毕竟 Babel 可以将 ES6 翻译成 ES-before-6)。在过去，这张照片流传很广:

[![](../Images/51ea7b23b3d95639062f486ed46cb00c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--eEk4_Kpq--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/xzb7jt1r70pehz7d0yl1.jpg)

关于 JS 以前有多诡异有话说:-D

*   《语言 C》是 T2 的书，是为数不多的以其作者名字首字母而闻名的书之一。这可能是我读的第一本关于编程语言的书。它真的真的很好，而且 C 可以装进你的脑袋。我几乎忘记了，该死的高级语言😂许多语言设计者不得不感谢这本书
*   我不知道它现在是否仍然有用(这些好的实践现在更加普遍)，但是[高性能网站:前端工程师必备知识](https://www.amazon.com/High-Performance-Web-Sites-Essential/dp/0596529309)是一本很好的读物。由史蒂夫·索德斯撰写，他对速度和网站略知一二。我相信那时他是谷歌的“首席性能工程师”或者类似的职位。他还编写了第一批衡量网站性能的工具之一，多年后，它的规则在 Chrome 的 Lighthouse 中仍有或多或少的相同名称。
*   编写 Erlang 是一股新鲜空气。我记得我第一次看著名的“ [Erlang: The Movie](https://www.youtube.com/watch?v=uKfKtXYLG78) ”并与其他开发者讨论时，我们都在想“WTF 是东西吗？！？太神奇了！!"(孩子们和他们的玩具:D)。没有人使用 Erlang(除了爱立信和一些不知名的公司)。我认为长生不老药甚至不是一个想法。这让我尝试了一下[端口](http://erlang.org/doc/tutorial/c_port.html)和其他让 Erlang 与 Python 对话的方法(或者反过来？)
*   [实用常用 Lisp](http://www.gigamonkeys.com/book/) 是最好的编程书籍句号。这甚至很吸引人，这在编程书籍中并不常见。我想我是在“学习随机语言只是因为”这个阶段读到这篇文章的。我无意成为一名 Lisp 程序员，我仍然讨厌括号，但这本书很棒。你甚至可以在某个时候写一个 MP3 文件的数据库。谈论实际的例子(Spotify 还不存在，人们的硬盘上有很多 MP3 文件，:D)
*   RESTful Web Services 是一本关于 REST 的书，写给向世界解释发明 REST 的[博士论文](https://www.ics.uci.edu/~fielding/pubs/dissertation/top.htm)，但也是一本可能没有人真正读完的书(包括我)，所以我们都部分实现了 REST，但我们称它为 REST，因为我们都是 Web 的潮人(是的，你也是)。我确信无论是发明它的人还是这本书的作者都对我们的 REST 架构印象深刻。Rails 的创始人 DHH 有一句名言*“每个从事 Web 工作的开发人员都需要阅读这本书。”*。他继续在 Rails 中实现 REST 的非完整版本，就像我们其他人一样。“谁需要超媒体”是当时的“640 KB 的内存应该足够每个人使用了”。
*   马丁·福勒的《重构》是 refactoring.com 的死树版本。网站就够了。
*   Twisted(天生的 Twisted Matrix)是我们在最近的服务器端框架中发现的许多概念的酷叔叔。本质上是一个框架，用于处理单进程事件驱动的网络事务。从 TCP 到 SSH 到 HTTP 到 IMAP 等等。这也是我所见过的反应器模式和事件循环的最佳实现之一。当我接触 Twisted 时，文档主要是一个教程，Luke 阅读源代码。哦，它已经推迟(像 JS 的承诺，但回到当 JS 既不是好的部分，也不是 ES6 :D)。它的一个前维护者就在 dev.to 上，但我不会说更多，因为我无权透露:D 的身份
*   我不认为我读过红宝石的方式，不知道我有:D
