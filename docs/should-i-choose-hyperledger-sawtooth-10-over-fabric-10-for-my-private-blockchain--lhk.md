# 我应该为我的私人区块链选择 Hyperledger 锯齿 1.0 而不是 Fabric 1.0 吗？

> 原文：<https://dev.to/skcript/should-i-choose-hyperledger-sawtooth-10-over-fabric-10-for-my-private-blockchain--lhk>

Hyperledger 区块链项目目前正处于高峰期。有了像 IBM 这样的大公司和像我们这样以技术为基础的公司，这个项目的开发周期比以前更快了。这绝对是我们引以为豪的事情。

最近宣布的企业就绪锯齿 1.0 很大。从英特尔转移到 Linux 基金会的锯齿项目与 [Hyperledger Fabric](https://www.skcript.com/svr/what-is-hyperledger-fabric-and-what-you-should-know-about-it/) 有点相似，但更成熟一些。

两个项目最大的不同在于项目背后的理念。

例如，Hyperledger Sawtooth 专注于创建非常安全的方式来处理您的智能合同，具有更严格的规则和共识。

另一方面，Hyperledger Fabric 可以选择性地执行这些规则，这有助于许多不同的私人区块链提供商。

如果您希望构建一个非常安全、严格分布的私有区块链，锯齿将是最佳选择。原因如下，锯齿 1.0 是 Hyperledger 项目历史上的一个主要版本:

**分布式台账；**锯齿的目标是拥有分布式账本，真正的分布式。

**安全智能合同；**智能合同采用严格的策略来真正分发您的信息，因此更加安全，适合企业使用。

**BFT 启用；**当我看到锯齿是企业就绪时，主要原因是它带有拜占庭容错特性。这种 BFT 特性比 Fabric 中的崩溃容错率高得多。

**全球国家协议强制执行；**该模块确保每个节点都拥有与区块链网络中其他节点完全相同的数据库副本。有加密技术的强大支持。

**并行交易；**与锯齿进行并行交易的能力决定了一切。从而减少系统中处理事务的时间。

### 那么，我应该选择面料还是锯齿呢？

这个决定完全取决于你的用例。一个经验丰富的区块链顾问将能够帮助你更快地做出这个决定。

如果你正在寻找一个拥有高度敏感数据和敏感智能合同的私人区块链网络，锯齿可能是你应该考虑的一个选择。
