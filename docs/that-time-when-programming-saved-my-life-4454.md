# 那次编程救了我的命

> 原文：<https://dev.to/acoh3n/that-time-when-programming-saved-my-life-4454>

[![banner](../Images/b06480c39df458bbb0c9c3dbac9ea59f.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--T3PaVF_s--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/zxjyjcdh14jey40zcbrk.jpg)

那是 2000 年，我是以色列国防军的一名士兵，开始了我强制性 3 年服役期的第二年。

我在军队的职位并不光彩。由于奇怪的机缘巧合，我发现自己在一个与电脑毫无关系的小部队里负责一个小裁缝办公室(我想所有的办公室里大概有 5 台电脑)。不用说，我的生活并不充实，实际上我很痛苦。

在参军之前，在高中时，我发现了自己对编程的热情，甚至做得相当好。不幸的是，我所在部队的副指挥官(他对我恨之入骨)在我要求重新任命为电脑化官员时，根本不听我的话。

此外，在构建任何可用的软件时，我不太基础的高中计算机科学培训没有给我任何实际技能。

有一天，我看到一本军队手册，上面解释了如何使用微软 Access 来构建应用程序。这本手册很小，只涵盖了基础知识，但它让我意识到军队已经批准微软 Access 作为构建应用程序的技术。

由于我对 Access 一无所知，下一次我在部队休假时，我就去我家附近最近的书店，买了我能找到的关于这个主题的最大的书。我想是这个(希伯来语):

[![MS Access Book](../Images/2341a620b93f101da175ca3db253f947.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--VUHEp6f4--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/fusbznjzerw5vu205sx6.jpg)

现在我只需要一个项目。

所以我在单位的各个办公室走来走去，寻找一个项目。稍微搜索了一下，我发现“Hamal”(接待员和电话接线员的混合体)到处都有成堆的手绘电话簿和其他纸片，上面写着电话号码。所以我决定建立一个电话簿数据库。似乎很容易。

我开始阅读这本书，并对它进行了深入的研究，足以构建最初的几个屏幕。一旦我卡住了，我再次打开书，继续阅读，直到我得到我的答案。然后，我回去继续写一些代码，修复了几个屏幕，又碰了一次壁。再次打开书，继续足够得到我的答案，然后回到代码。冲洗，重复。

请记住，互联网并不是无处不在，堆栈溢出甚至在接下来的 8 年里都不存在，谷歌也不是一个真正的东西。所以我唯一的选择是继续写我的书。

在构建了应用程序的重要部分之后，我决定和那个电话接线员一起尝试一下。他似乎很感兴趣，但有点怀疑。但是愿意尝试一下。

第二天，当我去拜访他的时候，他已经把他大部分的电话簿和纸片都输入了申请，脸上挂着灿烂的笑容。他说，这让他的生活轻松多了，也给了我第一份功能需求清单。

感觉自己好像有所发现，我决定大胆一试，给那个副指挥官演示一下。见鬼，我没什么可失去的。我又不会被解雇。

所以我走进他的办公室，问他是否可以给他看些东西。他不情愿地同意了。我启动了应用程序，我可以说我引起了他的注意。我继续解释手写电话簿的问题，电话号码如何因磨损而丢失，接线员与单位其他办公室共享数据有多困难，以及我从接线员那里得到的积极反馈。

他被卖掉了，几天后，我被重新任命为计算机化官员，这导致了许多更有趣的项目(其中一个被刊登在军队杂志上)，并最终开始了我的职业生涯。
