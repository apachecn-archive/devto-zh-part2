毕业伊始，许多信息技术专业的学生学习一门编程语言，并加入它的“倡导者团队”。在我的例子中，它使用的是最受批评，但即使如此，仍然流行的基于 vm 的语言:Java。

我们的老师是一个 Java 爱好者。在学习了面向对象的范例、JavaFX 的丰富接口和一点点 Android 之后，我非常兴奋，甚至没有考虑过在没有 Oracle 编程语言的情况下做其他事情的可能性...直到我目前的项目出现...

有了科学启动计划，我的学院有了由老师的想法启动的软件开发研究项目(或系统评审)。我的一位教授创建了一个框架来解决巴西高等教育的一个最大问题:处理教育部要求的自我评估报告，在理想的世界中，必须考虑构成该机构的所有领域，在 INEP(国家教育研究学院)公布的技术说明中指定的轴和维度。

还有，有什么问题？它需要一个软件工具...在 Django 开发的一个遗留应用程序中，称为 SUAP -公共行政统一系统。该应用程序的数据来自教师、行政人员、学生以及在两个机构学习、工作或具有某种职能的许多人:帕拉伊巴州联邦研究所(IFPB)和北里奥格朗德(IFRN)，该应用程序最初由后一个机构开发，并被前一个机构采用。但是在我的校园里，在不得不处理它的学生中，这个系统有着极其负面的名声。

作为一名 Java 程序员，一开始并不擅长处理基于缩进编程语言。对于一个如此大范围的项目，我们试图尽快开始。而事情，几次之后，变得越来越复杂。代码没有任何文档，开发环境需要在基于 Debian 的 Linux 发行版中进行如此复杂的设置。通过文档传递给我们的这个环境的配置是我们的第一个问题。一些依赖项已经过时了，我们对 Linux 的一点点了解让事情变得相当复杂。所以，我决定“激进化”:我在过去测试了一个非常友好的发行版，叫做 Deepin(来自中国)，我们试图在一个非常过时的 Ubuntu 版本上做一些事情。我从学术机器上抛弃了这个系统，并成功地尝试在它上面做这个程序。我们赢得了第一场战斗...但是还有另外一个。

一个开发者指南，以前的开发者提供的，太差了。不知道 Django 是如何工作的，以及应用程序本身是如何工作的。我们试着只阅读短期课程中的几张幻灯片来做我们的活动，答案堆积如山。我们试图尽可能遵循产品原型。但是后来，我开始意识到它是多么的低效:代码是如此的复杂、混乱和不稳定。我开始用下午的时间诅咒语言、代码和一切试图为随机问题找到解决方案的东西...有一天，我非常苦恼，以至于我不得不走出房间 30 分钟- 1 个小时，把我的头放在适当的位置，呼吸一些空气，喝点咖啡让自己平静下来(我在其他场合也苦恼过，但没有那么强烈)。我们的老师甚至考虑向资深团队请求 Rest-API，以便我们开始使用 Angular、PHP 或其他工具重新编写所有应用程序。幸运的是，这不是必要的。

最后，在阅读了一些遗留代码和一些 Django 文档后，我决定放弃所有“改编”的代码，并试图只通过文档和自己遗留代码的启发来做事。事情开始好转。几个月后，我们开始在 Django 开发上成熟起来。我开始明白我的憎恨，以及在我之前的开发人员的憎恨，绝不是缺乏适当的知识，而是由极其糟糕的代码文档证明的。因此，我不仅开始关心我的代码质量，并试图遵循 Django 上的良好实践，而且开始编写关于这个系统的文档，以帮助我的校园中的其他 Java 爱好者，他们可能在我结束后的某个时候，将不得不处理它。

现在，如果 Python 不是我最喜欢的编程语言，至少我不再讨厌它了。有时候我已经习惯了，当我写 JavaScript 的时候，我开始忘记在 if 语句的末尾加上“{ ”(实际上，我是如此的放松以至于让我发笑)。我开始对 Django 越来越好奇，并发现了它的一些特性，我开始有了和开始用 JavaFX 做接口时一样的感觉。我又迷上了用更合适的方式开发东西。

我仍然认为自己是一个非常普通的初学者，不太熟练的 Python 开发者。但至少我不再是苦恼的那个了。
