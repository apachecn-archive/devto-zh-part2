排名不分先后:

*   了解关系数据库是如何工作的(这总是有用的)
*   了解 HTTP 通常是如何工作的
*   学习如何用一种语言调试代码(用其他语言也差不多，你可以重复利用大部分知识)
*   熟悉命令行
*   知道如何查找代码(在命令行上使用 IDE 或 grep)
*   了解正则表达式的基础知识会让你走得更远(也是为了找到上一点中的代码)
*   知道如何使用搜索引擎找到解决方案
*   知道如何操作 git(我会说任何源代码控制，但 git 是事实上的标准，所以你最好从它开始)
*   问问题，尤其是当你认为这些问题不值得问的时候
*   了解时区是如何工作的(不是开玩笑，很多开发者对时区还是很模糊)
*   了解 unicode 和 utf-8 是如何工作的(时区也是如此)
*   大致了解缓存(CPU、内存、磁盘、HTTP)作为一个概念是如何工作的

请注意，除了 git 之外，我没有指定任何特定的工具，这是因为无论您最终使用哪种语言或工具，都会遇到这种东西。

我同意他们的观点...但是十二件事？我记不清十二件事。不，你不需要 12 样东西。你需要一样东西，一个黄金法则，一个变种:

> 问问题，尤其是当你认为这些问题不值得问的时候

更接近

## 一直提问。

这就是你所需要的。你需要学会问问题。你需要学会将你的无知武器化。

什么？把你的无知武器化？我到底什么意思？

作为一名新的软件开发人员(“初级”什么的——我开始的时候 35 岁左右)，在你的职业生涯中获得成功的最快方法是尽快认识到三件事:

1.  其他人应该比你更了解软件开发。
2.  你工作的一部分是学习更多关于软件开发的知识。
3.  教你是其他人工作的一部分。

真的是。真的。

所以，要让(2)发生，你需要让(1)去做(3)。你需要主动去做。不要等着其他人(“前辈”——他们都比我年轻)来为你创造学习的机会。抓住那些笨蛋的衣领说:

> 你说“宁静”是什么意思？
> 
> 依赖注入框架在家的时候是什么鬼？
> 
> 你“修改”了那个功能？你做了什么？
> 
> 为什么用的是`PUT`而不是`POST`？
> 
> 哇，哇，哇！你刚才用的那个`curl`命令是什么？
> 
> 你刚才用了什么捷径来做到这一点？
> 
> 你为什么总是打`git pull -r`？`-r`是干什么用的？
> 
> 这个变量来自哪里？

如果很难问这些问题，那就问:

> 为什么不能多结对编程？如果我看不到你在做什么，我很难学到更多。

如果你因为问了“愚蠢”的问题而受到指责:

> 为什么你对让我成为更好的开发者不感兴趣？
> 
> 你为什么认为我应该知道？为什么觉得真的很基础？你为什么不解释给我听？

头两年，在任何地方，任何时候都这样做。编程的时候做。开会的时候做。在你吃午饭的时候做:

> 为什么写在你的 t 恤上？

利用你在职业生涯的这个阶段本该一无所知的事实——让这种无知为你所用！什么都要问。这是你的黄金机会！

你会发现其他程序员(大多数情况下)喜欢谈论编程和炫耀，告诉你他们知道什么。他们积极地享受它！而且，通常情况下，他们不会知道——所以你们俩都有很好的借口通过搜索网络来一起寻找答案——在这一点上，你将了解到高级开发人员是如何得到答案的！

## 不要不问问题

但是无论你做什么，不要闭上嘴，让这个缩写/技巧/句子不被质疑。保持沉默对任何人都没有好处:对你自己没有好处，因为你不会学到任何东西；对你的同事也没有好处，他们会认为你知道他们在谈论什么，当后来发现你不知道时，他们会很生气。

## 培养强烈的好奇心

大声说出你的无知！对一切都要好奇！让你的口头禅是“我不知道——给我看看”。你需要知道的其他一切都会从这一件事中产生。抓住这个千载难逢的机会！将你的无知武器化。

因为，再过一年左右，你就做不到这一点了。大约一年后，你的团队中会有另一个新的开发人员问你这些问题。所以好好利用这个机会。
